## 摘要
用于多项选择和成对评估任务的大语言模型（LLMs）通常会因为选项位置和标签符号等非语义因素而表现出选择偏差。现有的推理阶段去偏方法成本高昂且可能损害推理能力，而逐点训练则忽略了同一个问题在不同排列下应产生一致答案的特性。为解决这一问题，我们提出了排列感知型群体相对策略优化（Permutation-Aware Group Relative Policy Optimization, PA-GRPO），通过强制执行排列一致的语义推理来缓解选择偏差。PA-GRPO 通过生成多个候选排列为每个实例构建排列组，并利用两种互补机制优化模型：(1) 跨排列优势，即计算相对于同一实例所有排列的平均奖励的优势；(2) 一致性感知奖励，鼓励模型在不同排列间做出一致的决策。实验结果表明，PA-GRPO 在七个基准测试中均优于强基准模型，在保持高整体性能的同时显著减少了选择偏差。代码将在 Github 上开源 (https://github.com/ECNU-Text-Computing/PA-GRPO)。

## Abstract
Large language models (LLMs) used for multiple-choice and pairwise evaluation tasks often exhibit selection bias due to non-semantic factors like option positions and label symbols. Existing inference-time debiasing is costly and may harm reasoning, while pointwise training ignores that the same question should yield consistent answers across permutations. To address this issue, we propose Permutation-Aware Group Relative Policy Optimization (PA-GRPO), which mitigates selection bias by enforcing permutation-consistent semantic reasoning. PA-GRPO constructs a permutation group for each instance by generating multiple candidate permutations, and optimizes the model using two complementary mechanisms: (1) cross-permutation advantage, which computes advantages relative to the mean reward over all permutations of the same instance, and (2) consistency-aware reward, which encourages the model to produce consistent decisions across different permutations. Experimental results demonstrate that PA-GRPO outperforms strong baselines across seven benchmarks, substantially reducing selection bias while maintaining high overall performance. The code will be made available on Github (https://github.com/ECNU-Text-Computing/PA-GRPO).

#### 1. 核心问题与研究动机
*   **核心问题**：大语言模型（LLMs）在处理多选题（MCQ）和成对评估（LLM-as-a-Judge）时，极易受到**非语义因素**的影响，产生“选择偏差”。例如，仅仅交换选项的顺序或改变标签（如 A/B 变为 B/A），模型的预测结果就会发生改变。
*   **研究动机**：
    *   **现有方法缺陷**：推理端去偏（如后验校准）计算开销大且不改变模型本质；训练端去偏（如 SFT 增强）通常将不同排列视为独立样本，忽略了同一问题在不同排列下应保持语义一致性的内在逻辑。
    *   **排列盲目性（Permutation-blindness）**：现有的强化学习（RL）对齐方法（如 GRPO）在优化时，将同一问题的不同排列变体视为独立事件，导致模型可能在某一特定顺序下获得高分，但在其他顺序下表现糟糕，且这种不一致性未受到惩罚。

#### 2. 方法论：PA-GRPO
*   **核心思想**：将同一实例的所有排列变体组织成一个“排列组（Permutation Group）”，通过强化学习强制模型实现**排列不变性（Permutation Invariance）**。
*   **关键技术细节**：
    *   **排列组构建**：对于多选题，采用结构化子集（4次循环位移+1次逆序，共5个变体）；对于成对评估，采用全排列（2个变体）。
    *   **跨排列优势（Cross-Permutation Advantage, APA）**：在计算优势函数（Advantage）时，基准值不再是单个 Prompt 的样本均值，而是整个排列组内所有样本的平均奖励。这确保了只有在所有排列下表现都优于平均水平的样本才会获得正向激励。
    *   **一致性感知奖励（Consistency-Aware Reward, $r_{con}$）**：在原始的正确性奖励基础上，增加一致性惩罚。如果模型在不同排列下做出了不同的语义选择，则给予负奖励。
    *   **优化目标**：基于 GRPO 框架，结合 KL 散度约束，优化剪切后的代理目标函数，使模型在保证正确性的同时，最大化决策的一致性。

*   **性能提升**：PA-GRPO 在所有基准测试中均优于强基准模型，尤其在“一致准确率（CA）”上提升显著（如在 Qwen3-8B 的 JudgeBench 上提升了 7.9%）。

#### 7. 优点
*   **原理直击痛点**：通过排列组的概念解决了 RL 训练中的“排列盲目性”问题。
*   **训练效率高**：基于 GRPO 架构，无需额外的价值网络（Value Network），节省了显存和计算资源。
*   **通用性强**：统一了 MCQ 和成对评估任务的去偏框架，适用于多种离散选择场景。

#### 8. 不足与局限
*   **任务局限性**：目前主要针对离散选择任务，如何扩展到开放式生成任务（语义等价性更难量化）尚待研究。
