## 摘要
多智能体大语言模型（LLM）系统跨越信任边界委托任务，但目前的协议无法在质量声明不可验证的情况下对委托行为进行治理。我们研究表明，当受托者（delegates）可以夸大自报的质量分数时，基于质量的路由会产生一种“溯源悖论”：它会系统性地选择最差的受托者，其表现甚至劣于随机选择。我们扩展了 LLM 委托协议（LDP），引入了通过明确目标、预算和失败策略来约束权限的委托合约；一种区分自报质量与验证质量的“声明 vs 认证”身份模型；以及支持自动恢复的有类型失败语义。在包含 10 个模拟受托者并经由真实 Claude 模型验证的受控实验中，按自称质量分数进行路由的表现差于随机选择（模拟：0.55 对 0.68；真实模型：8.90 对 9.30），而认证路由则实现了接近最优的性能（d = 9.51, p < 0.001）。跨 36 种配置的敏感性分析证实，当存在不诚实的受托者时，该悖论会稳定出现。所有扩展均向后兼容，且验证开销低于微秒级。

## Abstract
Multi-agent LLM systems delegate tasks across trust boundaries, but current protocols do not govern delegation under unverifiable quality claims. We show that when delegates can inflate self-reported quality scores, quality-based routing produces a provenance paradox: it systematically selects the worst delegates, performing worse than random. We extend the LLM Delegate Protocol (LDP) with delegation contracts that bound authority through explicit objectives, budgets, and failure policies; a claimed-vs-attested identity model that distinguishes self-reported from verified quality; and typed failure semantics enabling automated recovery. In controlled experiments with 10 simulated delegates and validated with real Claude models, routing by self-claimed quality scores performs worse than random selection (simulated: 0.55 vs. 0.68; real models: 8.90 vs. 9.30), while attested routing achieves near-optimal performance (d = 9.51, p < 0.001). Sensitivity analysis across 36 configurations confirms the paradox emerges reliably when dishonest delegates are present. All extensions are backward-compatible with sub-microsecond validation overhead.

随着多智能体 LLM 系统（如 Google 的 A2A、Anthropic 的 MCP）的普及，任务在不同代理间的委派变得频繁。然而，现有的协议存在一个关键缺陷：**质量声明不可验证**。
*   **研究动机**：当路由算法依赖代理自报的质量分数时，理性的代理为了获取更多任务，会有动机夸大其能力。
*   **溯源悖论（Provenance Paradox）**：论文揭示了一个反直觉现象——基于自报质量的路由会系统性地选择最不诚实（通常也是质量最差）的代理，导致系统整体表现甚至不如随机选择。
*   **核心目标**：通过扩展 LLM 委托协议（LDP），引入治理机制来解决这种信任危机。

#### 2. 论文提出的方法论
作者为 LDP 协议提出了四个关键扩展，旨在建立“有约束的委派”：
*   **委托合约（Delegation Contracts）**：在任务提交时附加明确的约束，包括目标（Objective）、成功标准、预算（Token/成本上限）、截止日期及失败策略（如 `fail_closed` 拒绝违规结果，或 `fail_open` 仅记录违规）。
*   **声明 vs. 认证身份模型（Claimed vs. Attested Identity）**：将质量指标分为四级：
    *   `self_claimed`（自述，低信任）；

    *   `issuer_attested`（第三方机构认证）；
    *   `externally_benchmarked`（外部基准测试验证，高信任）。
*   **类型化失败语义（Typed Failure Semantics）**：将模糊的错误字符串替换为结构化的 `LdpError`，包含类别（如政策违规、能力不足）、严重程度和是否可重试的标志，支持自动化故障恢复。
*   **验证状态与谱系（Verification & Lineage）**：记录结果是如何被验证的（如人工、工具或同行验证）以及完整的处理链条。
