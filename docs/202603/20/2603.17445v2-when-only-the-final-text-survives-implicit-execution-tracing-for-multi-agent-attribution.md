## 摘要
当多智能体系统产生错误或有害的回答时，如果执行日志和智能体标识符不可用，谁该负责？多智能体语言系统日益依赖于委派和迭代细化等结构化交互，然而最终输出往往掩盖了底层的交互拓扑和智能体贡献。我们引入了 IET（隐式执行追踪），这是一个独立于元数据的框架，能够直接从生成的文本中实现令牌级归因，并提供了一种简单的交互拓扑重建机制。在生成过程中，特定于智能体的密钥信号被嵌入到令牌分布中，将文本转化为一种仅能通过密钥检测的自描述执行追踪。在检测阶段，一种感知转换的评分方法会识别智能体交接点并重建交互图。实验表明，IET 在保持生成质量的同时，能以高准确度恢复智能体分段和协作结构，从而为多智能体语言系统实现隐私保护审计。

## Abstract
When a multi-agent system produces an incorrect or harmful answer, who is accountable if execution logs and agent identifiers are unavailable? Multi-agent language systems increasingly rely on structured interactions such as delegation and iterative refinement, yet the final output often obscures the underlying interaction topology and agent contributions. We introduce IET (Implicit Execution Tracing), a metadata-independent framework that enables token-level attribution directly from generated text and a simple mechanism for interaction topology reconstruction. During generation, agent-specific keyed signals are embedded into the token distribution, transforming the text into a self-describing execution trace detectable only with a secret key. At detection time, a transition-aware scoring method identifies agent handover points and reconstructs the interaction graph. Experiments show that IET recovers agent segments and coordination structure with high accuracy while preserving generation quality, enabling privacy-preserving auditing for multi-agent language systems.

#### 1. 论文的核心问题与整体含义
随着多智能体系统（MAS）的广泛应用，系统输出错误或有害内容的责任归因变得至关重要。然而，现有的归因方法高度依赖于“白盒”环境下的执行日志和智能体标识符（元数据）。在现实场景中，这些元数据往往因隐私保护、合规性要求或简单的“复制粘贴”操作而丢失，导致最终文本成为唯一的审计线索。本论文提出了**隐式执行追踪（IET, Implicit Execution Tracing）**框架，旨在解决在**无元数据、仅有最终文本**的情况下，如何准确识别每个文本片段由哪个智能体生成，并重建智能体间的交互拓扑结构。

#### 2. 方法论：核心思想与关键技术
IET 的核心思想是将执行身份直接绑定到生成过程中的 Token 分布上，使文本成为“自描述”的执行记录。
*   **信号注入（Signal Injection）：** 在生成阶段，利用特定于智能体的密钥（Secret Key）对语言模型的 Logits 进行调制。通过词表置换和微扰（Perturbation），在不改变文本语义的前提下嵌入统计特征。
*   **拓扑编码（Topology Encoding）：** 将智能体间的交互路径（如 A 委派给 B）表示为邻接矩阵，并将其编码为位掩码（Bitmask），随生成过程动态更新并嵌入到 Token 分布中。
*   **局部一致性评分（Local Consistency Scoring）：** 在检测阶段，利用滑动窗口计算每个 Token 与各智能体密钥的对齐程度，生成平滑的评分序列。
*   **竞争性变点检测（Competitive Change-Point Detection）：** 采用累积和算法（CUSUM）识别评分序列中的显著切换点，从而精确划分智能体交接的边界，并根据嵌入的位掩码还原交互图。

#### 7. 优点
*   **独立于元数据：** 解决了“文本离开服务器环境即失去追溯性”的痛点。
*   **隐私保护：** 归因信号仅对持有密钥的审计者可见，对普通读者表现为自然文本。
*   **系统级插桩：** 相比于事后推理，这种“设计即归因”的思路更加确定和可靠。
*   **拓扑感知：** 不仅能识别“谁说了什么”，还能还原“谁影响了谁”的逻辑结构。

#### 8. 不足与局限
*   **系统介入要求：** 需要在智能体生成阶段介入 Logits 处理，无法应用于无法控制生成过程的第三方闭源 API（除非 API 提供商集成此类技术）。
