## 摘要
具身机器人系统日益依赖基于大语言模型（LLM）的智能体，在与环境交互过程中支持高层推理、规划和决策。然而，调用 LLM 推理会引入显著的计算延迟和资源开销，这可能会中断动作执行并降低系统可靠性。过度的推理可能会延迟动作，而推理不足则往往导致错误的决策和任务失败。这为具身智能体提出了一个根本性问题：智能体何时应当推理，何时应当行动？在这项工作中，我们提出了 RARRL（基于强化学习的资源感知推理），这是一个用于具身智能体资源感知编排的分层框架。RARRL 并非学习底层控制策略，而是学习一种运行在智能体决策层的高层编排策略。该策略使智能体能够根据当前观测、执行历史和剩余资源，自适应地确定是否调用推理、采用哪种推理角色以及分配多少计算预算。大量的实验（包括使用源自 ALFRED 基准测试的经验延迟分布进行的评估）表明，与固定或启发式推理策略相比，RARRL 在降低执行延迟和增强鲁棒性的同时，持续提高了任务成功率。这些结果表明，自适应推理控制对于构建可靠且高效的具身机器人智能体至关重要。

## Abstract
Embodied robotic systems increasingly rely on large language model (LLM)-based agents to support high-level reasoning, planning, and decision-making during interactions with the environment. However, invoking LLM reasoning introduces substantial computational latency and resource overhead, which can interrupt action execution and reduce system reliability. Excessive reasoning may delay actions, while insufficient reasoning often leads to incorrect decisions and task failures. This raises a fundamental question for embodied agents: when should the agent reason, and when should it act? In this work, we propose RARRL (Resource-Aware Reasoning via Reinforcement Learning), a hierarchical framework for resource-aware orchestration of embodied agents. Rather than learning low-level control policies, RARRL learns a high-level orchestration policy that operates at the agent's decision-making layer. This policy enables the agent to adaptively determine whether to invoke reasoning, which reasoning role to employ, and how much computational budget to allocate based on current observations, execution history, and remaining resources. Extensive experiments, including evaluations with empirical latency profiles derived from the ALFRED benchmark, show that RARRL consistently improves task success rates while reducing execution latency and enhancing robustness compared with fixed or heuristic reasoning strategies. These results demonstrate that adaptive reasoning control is essential for building reliable and efficient embodied robotic agents.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）在具身智能领域的广泛应用，机器人获得了强大的高层推理和规划能力。然而，LLM 的推理过程伴随着高昂的计算成本和显著的时间延迟。在实时交互的具身任务中，**“过度推理”**会导致动作延迟、系统响应缓慢；而**“推理不足”**则会导致决策错误或任务失败。
本论文的核心问题是：**具身智能体如何自适应地在“思考（推理）”与“行动（执行）”之间取得平衡？** 研究旨在开发一种机制，让机器人在有限的计算预算和时间约束下，自主决定何时调用昂贵的 LLM 推理模块。

#### 2. 方法论：核心思想与技术细节
论文提出了 **RARRL (Resource-Aware Reasoning via Reinforcement Learning)** 框架，其核心思想是将推理的调用建模为一个高层的编排（Orchestration）问题。

*   **核心架构**：采用分层结构，将决策层（编排策略）与执行层（底层控制）解耦。编排策略不直接控制电机，而是管理认知资源的分配。
*   **MDP 建模**：
    *   **状态空间 ($s_t$)**：包含当前环境观测、执行历史记录（过去动作的结果）以及**剩余资源预算**（如剩余 Token 或时间）。
    *   **动作空间 ($a_t$)**：决定是直接执行动作（ACT），还是调用推理（THINK）。若选择 THINK，还需进一步选择推理角色（如 Planner 规划者或 Verifier 验证者）及计算预算（Token 限制等级）。
    *   **奖励函数**：$r_t = r_{task} - \lambda \cdot \delta_t - \mu \cdot \mathbb{1}_{fail}$。其中 $r_{task}$ 是任务成功奖励，$\delta_t$ 是执行延迟惩罚，$\mu$ 是动作失败惩罚。
*   **算法流程**：使用 **PPO (Proximal Policy Optimization)** 算法训练编排策略。策略网络输出调用推理的概率、角色分布和预算等级。在训练过程中，LLM 被视为行为固定的“黑盒”，仅更新编排策略。

#### 3. 实验设计
研究在多个维度上验证了 RARRL 的有效性：
*   **实验场景**：
    1.  **抽象任务**：包括长程任务分解、结构化导航和多步交付任务，用于验证逻辑有效性。
    2.  **ALFRED 基准测试**：在 AI2-THOR 物理模拟器中进行，结合真实的 GPT-4o-mini API 调用，评估真实运行时的延迟和成功率。
*   **对比基准 (Baselines)**：
    *   **No Reasoning**：从不推理，仅直接执行。
    *   **Full Reasoning**：每一步都调用推理。
    *   **Fixed/Heuristic**：固定频率推理（如每3步一次）或基于简单启发式规则（如不确定性超过阈值时推理）。
    *   **Constrained PPO**：带有显式预算约束的拉格朗日优化 PPO。

*   **软件**：Python 3.8, PyTorch 1.13.0。

#### 5. 实验数量与充分性
*   **实验规模**：所有结果均基于 **5 个随机种子** 的平均值，并标注了标准差。
*   **消融实验**：非常充分。作者分别验证了“预算状态”、“执行历史”、“推理角色选择（仅规划 vs 仅验证）”以及“自适应预算分配”对最终性能的影响。
*   **鲁棒性测试**：专门设计了“延迟不确定性”实验和“预算冲击（Budget Shock，即资源突然减少）”实验，模拟真实世界中计算资源波动的极端情况。
*   **客观性**：通过 ALFRED 真实 API 调用的运行时评估，证明了该方法在模拟环境之外的实用价值。

#### 6. 主要结论与发现
*   **效率提升**：RARRL 在 ALFRED 任务中比“全量推理”减少了 **60% 以上** 的推理时间，同时保持了相近甚至更高的成功率。
*   **资源感知**：实验证明，显式地将“剩余预算”加入状态空间是至关重要的，这能防止智能体在任务早期过度消耗资源。
*   **性能上限**：编排策略的性能受限于底层执行器和推理器的强度，但 RARRL 能比启发式方法更接近性能天花板。
*   **自适应性**：在资源受限或环境不确定性增加时，RARRL 能自动从“依赖推理”转向“谨慎行动”，表现出更强的鲁棒性。

#### 7. 优点（亮点）
*   **解耦设计**：将“何时思考”与“如何思考”分离，使得该框架可以轻松适配不同的 LLM 后端或底层控制器。
*   **资源敏感性**：在具身智能研究中，首次系统性地通过 RL 解决 LLM 推理的成本-效益权衡问题。
*   **现实意义**：考虑了真实 API 调用的延迟和 Token 消耗，对移动机器人等端侧资源受限设备的部署具有指导意义。

#### 8. 不足与局限
*   **环境抽象性**：虽然在 ALFRED 中做了验证，但大部分训练仍依赖于抽象的任务模型，可能无法完全捕捉物理世界中复杂的传感器噪声和动态干扰。
*   **推理黑盒化**：将 LLM 视为固定行为的模块，未考虑通过微调 LLM 本身来进一步优化编排效率。
*   **应用限制**：目前主要针对高层逻辑任务，对于需要极高频率闭环控制（如毫秒级反应）的底层操作任务，该框架的适用性有待商榷。
