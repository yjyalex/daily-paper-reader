## 摘要
我们介绍了 CRAFT，这是一个用于评估大语言模型在严格部分信息下语用通信（pragmatic communication）的多智能体基准测试。在该设定中，多个拥有互补但非完整视野的智能体必须通过自然语言进行协作，以构建一个任何单个智能体都无法完全观察到的共享 3D 结构。我们将该问题形式化为多发送者语用推理任务，并提供了一个诊断框架，将失败原因分解为空间定位（spatial grounding）、信念建模（belief modeling）和语用通信错误，同时对前沿模型和开源权重模型中的行为失败模式进行了分类。通过对包括 8 个开源权重模型和 7 个前沿模型（含推理模型）在内的多种模型进行测试，我们发现更强的推理能力并不一定能可靠地转化为更好的协作能力：较小的开源权重模型表现往往与前沿系统持平甚至更优，且个体通信能力的提升并不保证协作的成功。这些结果表明，多智能体协作对于当前的语言模型来说仍然是一个从根本上尚未解决的挑战。我们的代码可在 https://github.com/csu-signal/CRAFT 找到。

## Abstract
We introduce CRAFT, a multi-agent benchmark for evaluating pragmatic communication in large language models under strict partial information. In this setting, multiple agents with complementary but incomplete views must coordinate through natural language to construct a shared 3D structure that no single agent can fully observe. We formalize this problem as a multi-sender pragmatic reasoning task and provide a diagnostic framework that decomposes failures into spatial grounding, belief modeling and pragmatic communication errors, including a taxonomy of behavioral failure profiles in both frontier and open-weight models. Across a diverse set of models, including 8 open-weight and 7 frontier including reasoning models, we find that stronger reasoning ability does not reliably translate to better coordination: smaller open-weight models often match or outperform frontier systems, and improved individual communication does not guarantee successful collaboration. These results suggest that multi-agent coordination remains a fundamentally unsolved challenge for current language models. Our code can be found at https://github.com/csu-signal/CRAFT

#### 6. 主要结论与发现
*   **推理能力不等于协作能力**：前沿模型（如 GPT-4.1-Mini, Claude-Sonnet）在个体推理和空间定位得分上极高，但在实际任务完成率上却不如 Mistral-7B 或 Qwen-7B 等小型开源模型。
*   **纠错螺旋（Correction Spirals）**：低效协作的一个典型特征是模型陷入过度纠错。前沿模型往往因为过于关注局部差异而频繁要求“拆除”已建成的部分，导致在有限轮次内无法完成任务。
*   **语用质量与成功率负相关**：在某些前沿模型中，个体通信的“高质量”（更详细、更关注独特视角）反而与任务进度呈负相关，因为这增加了指令的复杂度和冲突概率。
*   **协作鸿沟**：多智能体协调对于当前最先进的模型来说仍是一个未解决的挑战，模型难以在多方信息流中建立有效的“共同信念（Common Ground）”。

#### 7. 优点与亮点
*   **理论完备性**：将多智能体协作问题提升到语用学高度，并给出了严谨的数学形式化定义（Multi-Sender BPS）。
*   **诊断深度**：不仅关注“是否成功”，还通过 SG/MM/PS 三个维度拆解了模型是在哪个环节（感知、建模或表达）出了问题。
*   **解耦设计**：通过 Oracle 辅助界面巧妙地解决了多智能体系统中的“信用分配（Credit Assignment）”难题，确保评估的是主管的表达而非建筑师的理解。

#### 8. 不足与局限
*   **模态限制**：目前仅限于纯文本环境，未考察多模态模型（VLM）在直接视觉观察下的表现。
*   **结构规模**：测试的 3D 结构相对较小（3x3x3 空间），更复杂的长程协作任务可能展现出不同的失效模式。
*   **同质化假设**：实验中所有主管均使用同一模型，未探讨异构智能体（如不同厂商模型混合）在信息不对称下的协作表现。
*   **Builder 局限**：虽然使用了 Oracle 辅助，但建筑师本身仍可能存在推理偏差，影响最终决策。
