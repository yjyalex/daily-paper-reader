## 摘要
将具有可验证奖励的强化学习（RLVR）扩展到多模态大语言模型（MLLMs）面临一个根本性挑战：它们的响应本质上交织着感知相关 token（用于定位视觉内容）和推理相关 token（用于构建推理链）。这些 token 类型实例化了不同但又相互依赖的能力——视觉定位和符号推理——这使得孤立的优化变得不足。通过 token 级别的实证分析，我们证明了仅优化感知或仅优化推理 token 的表现始终不如全面优化，这强调了它们内在的耦合性。为了解决这个问题，我们提出了一种即插即用的 Token 重加权（ToR）策略，该策略通过识别这两类关键 token 并在 RLVR 训练期间动态重新加权，显式地对这种相互依赖性进行建模。在现有方法（如 GRPO 和 DAPO）的基础上应用 ToR，在多个多模态推理基准测试中实现了持续的性能提升，在准确的视觉定位和连贯的推理方面均达到了最先进的性能。

## Abstract
Extending Reinforcement Learning with Verifiable Rewards (RLVR) to multimodal large language models (MLLMs) faces a fundamental challenge: their responses inherently interleave perception-related tokens, which ground visual content, with reasoning-related tokens, which construct reasoning chains. These token types instantiate distinct yet interdependent capacities -- visual grounding and symbolic reasoning -- making isolated optimization insufficient. Through token-level empirical analysis, we demonstrate that optimizing either perception- or reasoning-only tokens consistently underperforms full optimization, underscoring their inherent coupling. To address this, we propose a plug-and-play Token-Reweighting (ToR) strategy that explicitly models this interdependence by identifying critical tokens of both types and dynamically reweighting them during RLVR training. Applied on top of existing methods (e.g., GRPO and DAPO), ToR delivers consistent performance gains across multiple multi-modal reasoning benchmarks, achieving state-of-the-art performance with both accurate visual grounding and coherent reasoning.

#### 1. 核心问题与整体含义（研究动机和背景）
本文探讨了如何将**具有可验证奖励的强化学习（RLVR）**有效地扩展到多模态大语言模型（MLLM）。研究背景在于，MLLM 的生成响应中交织着两类功能不同的 Token：**感知相关 Token**（用于定位和描述视觉内容）和**推理相关 Token**（用于构建逻辑推理链）。
*   **核心挑战**：现有的 MLLM 强化学习方法往往孤立地优化这两类能力（如仅关注思维链推理或仅增强视觉感知）。
*   **研究动机**：作者通过实证分析发现，感知与推理在 Token 级别是高度耦合且相互依赖的。仅优化其中一类 Token 会导致性能下降（例如，仅优化推理会导致视觉误读，仅优化感知则逻辑混乱）。因此，需要一种能够同时兼顾并平衡两者的优化策略。

#### 2. 方法论：Token 重加权（ToR）
作者提出了一种名为 **ToR (Token-Reweighting)** 的即插即用策略，核心思想是在 RL 训练过程中识别关键 Token 并动态调整其权重。

*   **关键技术细节**：
    1.  **推理 Token 识别**：基于**预测熵（Entropy）**。高熵 Token 通常对应推理链中的关键决策点（Forking points）。
    2.  **感知 Token 识别**：基于**视觉敏感度（Visual Sensitivity）**。通过计算“有图像输入”与“无图像输入（空占位符）”时 Token 对数概率的差值（Log-probability difference）来衡量该 Token 对视觉信息的依赖程度。
    3.  **重加权机制**：在计算策略梯度（如 GRPO 或 DAPO 算法）时，为识别出的推理 Token 分配权重 $\gamma_r$，为感知 Token 分配权重 $\gamma_p$，而对非关键 Token 降低权重或设为零。
*   **算法流程**：在 Rollout 采样后，计算每个 Token 的熵和视觉敏感度，选取全局前 $\alpha$ 比例的 Token 作为关键集合，随后在损失函数中应用这些权重进行梯度更新。

*   **实验规模**：作者进行了多维度的消融实验，包括不同 Token 选择比例（20% 到 80%）、不同权重组合（$\gamma_p$ 从 0.1 到 1.5）、以及在不同模型规模（3B 和 7B）上的验证。
*   **充分性与公平性**：实验涵盖了从感知到复杂推理的多种任务，对比了当前最先进的 R1 类多模态模型。通过在相同基座模型（Qwen2.5-VL）和相同训练数据下对比 GRPO/DAPO 的原生版本与 ToR 版本，证明了改进的客观性。

#### 6. 主要结论与发现
*   **耦合性验证**：实验证明，孤立优化推理或感知 Token 的效果均差于全量优化，证明了两者在 MLLM 中的内在耦合。
*   **ToR 的优越性**：ToR 策略在所有基准测试中均实现了性能提升。例如，在 MathVerse 上将 GRPO 的表现从 50.8 提升至 53.0，在 HalluBench 上从 69.8 提升至 72.4。
*   **平衡点**：发现当感知 Token 权重 $\gamma_p = 0.5$（推理权重默认为 1.0）且选择比例约为 30% 时，模型在感知准确性和推理连贯性之间达到了最佳平衡。

#### 7. 优点
*   **即插即用**：不需要修改模型架构或引入额外的辅助模型，仅通过修改损失函数的权重掩码即可实现。
*   **自包含性**：关键 Token 的识别完全依赖模型自身的内在信号（熵和对数概率差），无需外部标注或先验知识。
*   **协同优化**：显式地建模了感知与推理的依赖关系，解决了传统 RLVR 在多模态场景下“顾此失彼”的问题。

#### 8. 不足与局限
*   **Token 识别精度**：目前的识别方式（熵和概率差）虽然有效，但仍属于粗粒度代理指标。未来可能需要结合如 SAM 等视觉模型进行更精细的视觉区域关联。
*   **上下文忽略**：Token 级别的重加权可能忽略了语义块（Chunks）或上下文之间的长程依赖关系。
*   **应用范围**：目前主要在数学和几何推理任务上验证，对于更通用的多模态对话或视频理解任务的泛化性尚待进一步探索。
