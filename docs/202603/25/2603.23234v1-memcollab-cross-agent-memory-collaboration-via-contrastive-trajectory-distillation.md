## 摘要
基于大语言模型（LLM）的智能体依赖记忆机制来重用过去解决问题经验中的知识。现有方法通常以单智能体的方式构建记忆，将存储的知识与单个模型的推理风格紧密耦合。在具有异构智能体的现代部署中，一个自然产生的问题是：单个记忆系统能否在不同模型之间共享？我们发现，在智能体之间直接迁移记忆往往会导致性能下降，因为此类记忆将任务相关知识与智能体特有的偏置纠缠在一起。为了应对这一挑战，我们提出了 MemCollab，这是一个协作记忆框架，通过对比不同智能体在同一任务上生成的推理轨迹来构建与智能体无关的记忆。这一对比过程蒸馏出了抽象的推理约束，这些约束捕捉了共享的任务级不变性，同时抑制了智能体特有的伪影。我们进一步引入了一种任务感知检索机制，根据任务类别调节记忆访问，确保在推理时仅使用相关的约束。在数学推理和代码生成基准测试上的实验表明，MemCollab 在包括跨模型家族设置在内的多种智能体上，一致地提高了准确率和推理效率。我们的结果表明，协作构建的记忆可以作为多种基于 LLM 的智能体的共享推理资源。

## Abstract
Large language model (LLM)-based agents rely on memory mechanisms to reuse knowledge from past problem-solving experiences. Existing approaches typically construct memory in a per-agent manner, tightly coupling stored knowledge to a single model's reasoning style. In modern deployments with heterogeneous agents, a natural question arises: can a single memory system be shared across different models? We found that naively transferring memory between agents often degrades performance, as such memory entangles task-relevant knowledge with agent-specific biases. To address this challenge, we propose MemCollab, a collaborative memory framework that constructs agent-agnostic memory by contrasting reasoning trajectories generated by different agents on the same task. This contrastive process distills abstract reasoning constraints that capture shared task-level invariants while suppressing agent-specific artifacts. We further introduce a task-aware retrieval mechanism that conditions memory access on task category, ensuring that only relevant constraints are used at inference time. Experiments on mathematical reasoning and code generation benchmarks demonstrate that MemCollab consistently improves both accuracy and inference-time efficiency across diverse agents, including cross-modal-family settings. Our results show that the collaboratively constructed memory can function as a shared reasoning resource for diverse LLM-based agents.

*   **研究动机**：现有的 LLM 智能体记忆机制通常是“单智能体”模式，即记忆是从特定模型的推理轨迹中提取并供其自身使用的。这种方式导致记忆与特定模型的推理风格、偏置（Bias）高度耦合。

*   **研究目标**：探索能否构建一种“智能体无关”（Agent-agnostic）的共享记忆系统，使不同模型能够协同利用过去的经验。

### 2. 方法论
MemCollab 框架分为两个核心阶段：**对比记忆构建**和**任务感知检索推理**。

*   **核心思想**：借鉴对比学习（Contrastive Learning），通过对比同一任务下不同智能体的“成功”与“失败”轨迹，识别出决定成败的“任务不变性约束”（Invariants），从而过滤掉模型特有的风格。
*   **关键技术细节**：
    1.  **轨迹配对与偏好选择**：对训练集中的每个任务，同时运行弱智能体（$A_w$）和强智能体（$A_s$），根据结果将其分为首选轨迹（$\tau^+$）和非首选轨迹（$\tau^-$）。
    2.  **对比轨迹蒸馏（Algorithm 1）**：使用后台模型分析 $\tau^+$ 和 $\tau^-$ 的差异，提取出：
        *   **违规模式（Violation Patterns）**：导致失败的系统性错误。
        *   **推理不变性（Reasoning Invariants）**：成功解决任务必须遵循的结构化原则。
        *   **记忆条目格式**：`mk = (enforce ik; avoid vk)`（强制执行某原则；避免某错误模式）。
    3.  **任务感知检索（Algorithm 2）**：
        *   **分类**：利用 LLM 将任务预先分类（如：代数 -> 线性方程）。
        *   **路由**：推理时先进行任务分类，仅在同类任务的记忆子集中进行相似度检索，以减少噪声干扰。
    4.  **记忆增强推理**：将检索到的抽象约束作为 Prompt 的一部分引导智能体生成。

### 6. 主要结论与发现
*   **协作优于单体**：MemCollab 构建的记忆在所有基准上均优于模型自有的记忆或直接迁移的记忆。

*   **推理效率提升**：由于记忆提供了明确的“避坑指南”，智能体解决问题所需的平均推理轮数显著减少（例如在代码任务中减少了一半以上）。

### 7. 优点
*   **解耦性强**：成功实现了记忆与模型架构的解耦，解决了异构智能体共享知识的难题。
*   **对比机制有效**：通过“找不同”的方式精准捕捉逻辑核心，比单纯存储成功案例更能抑制模型偏置。
*   **任务感知检索**：引入分类路由机制，有效解决了大规模记忆库检索时的噪声干扰问题。

### 8. 不足与局限
*   **依赖高质量配对**：构建阶段需要同一任务的成功与失败轨迹，且需要 Ground Truth 或验证器来判定正确性，这在无标注的开放域任务中较难实施。
*   **蒸馏模型偏置**：虽然旨在消除偏置，但负责“对比总结”的后台模型（如文中使用的 32B 模型）本身可能在总结过程中引入新的抽象偏置。
*   **应用领域受限**：目前主要集中在有明确逻辑结构的数学和代码任务，在文学创作、情感咨询等缺乏明确“对错轨迹”的领域效果尚待验证。
