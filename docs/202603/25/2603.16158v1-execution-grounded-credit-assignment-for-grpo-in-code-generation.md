## 摘要
无评论员（Critic-free）的带可验证奖励强化学习（RLVR）通过优化单元测试通过率来改进代码生成，但 GRPO 风格的更新面临粗粒度信用分配的问题：即使失败源于局部的语义错误，单一的结果信号也会均匀地分布在长程序中。我们提出了基于执行的信用分配（EGCA），它利用执行轨迹来定位 GRPO 更新。对于满足算法约束但未通过测试的程序，EGCA 在相同的插桩环境下执行候选方案和标准参考解（离线整理一次；用于分析而非监督），识别最早的语义分歧点，并仅为对应的 token 跨度分配优势（advantage），同时屏蔽下游 token。EGCA 是一种即插即用的修改，不需要评论员、辅助损失或学习型验证器，在 HumanEval 上实现了 82.1% 的 pass@1（比 GRPO 提高 3.1%），在 MBPP 上实现了 68.9%（提高 1.5%），且仅增加 18% 的实际运行时间开销。

## Abstract
Critic-free reinforcement learning with verifiable rewards (RLVR) improves code generation by optimizing unit-test pass rates, but GRPO-style updates suffer from coarse credit assignment: a single outcome signal is spread uniformly across long programs even when failure stems from a localized semantic error. We propose Execution-Grounded Credit Assignment (EGCA), which localizes GRPO updates using execution traces. For programs that satisfy algorithmic constraints but fail tests, EGCA executes the candidate and a canonical reference solution (curated once offline; used for analysis, not supervision) under identical instrumentation, identifies the earliest semantic divergence, and assigns advantage only to the corresponding token span while masking downstream tokens. EGCA is a drop-in modification requiring no critic, auxiliary loss, or learned verifier, yielding 82.1% pass@1 on HumanEval (+3.1 over GRPO) and 68.9% on MBPP (+1.5) with 18% wall-clock overhead.

### 1. 核心问题与整体含义
*   **研究动机**：在代码生成的强化学习（RL）中，传统的 GRPO（群体相对策略优化）等算法采用“序列级”奖励。这意味着即使一个长程序只有一行逻辑错误，负面反馈也会均匀地分配给所有 Token（即“信用涂抹”问题）。
*   **核心问题**：如何精准地识别导致程序失败的特定代码段，并将强化学习的梯度更新集中在这些“罪魁祸首”上，而不是惩罚那些正确的代码？
*   **背景**：随着模型能力的提升，代码错误正从语法错误转向微妙的语义逻辑错误。此时，粗粒度的反馈已成为提升模型性能的主要瓶颈。

### 2. 方法论（EGCA）
EGCA（Execution-Grounded Credit Assignment）的核心思想是通过**执行轨迹对比**来实现 Token 级的精准信用分配。

*   **核心流程**：
    1.  **约束引导采样**：利用一个离线的参考解（Reference）提取算法约束，引导模型生成结构相似的候选代码。
    2.  **故障模式路由**：将生成的代码按优先级归类为：`SYNTAX`（语法错误）、`CONSTRAINT`（违反约束）、`LOGIC`（逻辑错误，即执行通过但结果错）和 `CORRECT`（正确）。
    3.  **执行轨迹定位（针对 LOGIC 类）**：
        *   在相同输入下，同时执行候选代码和参考解。
        *   记录执行过程中的状态轨迹（State Traces）。
        *   识别**最早的语义分歧点**（Earliest Semantic Divergence）。
    4.  **Token 级优势分配**：
        *   对于逻辑错误代码，仅将 GRPO 的优势值（Advantage）分配给导致分歧的 Token 跨度。
        *   **屏蔽（Masking）**：分歧点之后的下游 Token 不参与梯度更新。
*   **公式体现**：修改了 GRPO 的目标函数，将标量优势 $A_i$ 转化为 Token 级优势 $a_{i,t}$。对于逻辑错误，只有在错误跨度内的 $t$，$a_{i,t}$ 才非零。

### 6. 主要结论与发现
*   **性能提升**：EGCA 在 HumanEval 上达到 **82.1%**（比 GRPO 提升 3.1%），在 MBPP 上达到 **68.9%**（提升 1.5%）。
*   **信用分配是关键**：实验证明，一旦模型能够生成语法正确的代码，信用分配的精准度比奖励的稀疏性更重要。
*   **即插即用**：EGCA 不需要 Critic 网络、辅助损失函数或复杂的学习型验证器，仅通过修改梯度应用的位置即可生效。

### 7. 优点
*   **精准性**：通过动态执行轨迹定位错误，解决了长代码中“好坏 Token 混杂”导致的梯度噪声问题。
*   **无需 Critic**：保留了 GRPO 的计算优势，降低了显存占用和训练复杂度。
*   **鲁棒性**：通过故障模式路由，能够处理从语法到深层逻辑的不同层级错误。

### 8. 不足与局限
*   **依赖参考解**：EGCA 需要每个题目至少有一个离线的参考解来做轨迹对比，这限制了它在完全开放式、无标准答案场景下的应用。
*   **阶段依赖性**：该方法在模型已经能生成“大致正确”代码的后期微调阶段最有效。如果模型初期连语法都写不对，该机制触发频率低，效果退化为普通 GRPO。
*   **调试器能力限制**：虽然实验证明弱教师也能提供信号，但如果调试器完全无法解析执行轨迹，定位信号的噪声会增加。
*   **单文件限制**：目前实验仅限于 Python 单文件程序，尚未验证在多文件、复杂工程项目中的可行性。
