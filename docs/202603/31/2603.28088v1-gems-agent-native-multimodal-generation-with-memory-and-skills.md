## 摘要
近期的多模态生成模型在通用生成任务上取得了显著进展，但在处理复杂指令和专业下游任务时仍然面临挑战。受 Claude Code 等先进智能体框架成功的启发，我们提出了 GEMS（Agent-Native Multimodal GEneration with Memory and Skills，具有记忆与技能的智能体原生多模态生成），这是一个旨在突破基础模型在通用和下游任务中固有局限性的框架。GEMS 构建于三个核心组件之上：Agent Loop（智能体循环）引入了一个结构化的多智能体框架，通过闭环优化迭代提升生成质量；Agent Memory（智能体记忆）提供了一种持久的轨迹级记忆，分层存储事实状态和压缩的经验总结，在减少冗余的同时实现了优化过程的全局视角；Agent Skill（智能体技能）提供了一个可扩展的领域特定专业知识库，支持按需加载，使系统能够有效处理各种下游应用。在五个主流任务和四个下游任务中，通过在多个生成后端上的评估，GEMS 一致地实现了显著的性能提升。最值得注意的是，它使轻量级的 6B 模型 Z-Image-Turbo 在 GenEval2 上超越了最先进的 Nano Banana 2，证明了智能体治理在扩展模型能力、突破其原始极限方面的有效性。

## Abstract
Recent multimodal generation models have achieved remarkable progress on general-purpose generation tasks, yet continue to struggle with complex instructions and specialized downstream tasks. Inspired by the success of advanced agent frameworks such as Claude Code, we propose \textbf{GEMS} (Agent-Native Multimodal \textbf{GE}neration with \textbf{M}emory and \textbf{S}kills), a framework that pushes beyond the inherent limitations of foundational models on both general and downstream tasks. GEMS is built upon three core components. Agent Loop introduces a structured multi-agent framework that iteratively improves generation quality through closed-loop optimization. Agent Memory provides a persistent, trajectory-level memory that hierarchically stores both factual states and compressed experiential summaries, enabling a global view of the optimization process while reducing redundancy. Agent Skill offers an extensible collection of domain-specific expertise with on-demand loading, allowing the system to effectively handle diverse downstream applications. Across five mainstream tasks and four downstream tasks, evaluated on multiple generative backends, GEMS consistently achieves significant performance gains. Most notably, it enables the lightweight 6B model Z-Image-Turbo to surpass the state-of-the-art Nano Banana 2 on GenEval2, demonstrating the effectiveness of agent harness in extending model capabilities beyond their original limits.

**GEMS** 的核心思想是将多模态生成重新定义为一个**智能体原生（Agent-Native）的迭代优化过程**，通过引入结构化的循环、持久化记忆和可扩展技能，突破基础模型的固有能力边界。

#### 2. 方法论：核心思想与关键技术
GEMS 框架由三个核心支柱组成：
*   **Agent Loop（智能体循环）：** 建立了一个闭环优化系统。
    *   **Planner & Decomposer：** 规划任务并将其分解为多个可验证的原子准则（Binary Probes）。
    *   **Generator & Verifier：** 生成图像后，由多模态大模型（MLLM）作为验证器，逐项检查准则是否达成。
    *   **Refiner：** 根据反馈和记忆，迭代演化提示词（Prompt）。
*   **Agent Memory（智能体记忆）：** 采用**分层压缩策略**管理历史轨迹。
    *   保留原始的事实工件（提示词、图像、反馈向量）。
    *   使用压缩器（Compressor）将冗余的思维链（Thought）提炼为高层级的“经验总结”（Experiences），在减少 Token 开销的同时提供全局优化视角。
*   **Agent Skill（智能体技能）：** 一个可扩展的专业知识库。
    *   采用**按需加载（On-demand loading）**机制，仅在 Planner 识别到特定需求（如空间智能、创意绘画）时才加载详细指令，降低了系统的认知负荷并增强了专业性。

#### 7. 优点（亮点）
*   **架构创新：** 将 Agent 架构深度集成到生成流程中，而非简单的 Prompt 工程。
*   **分层记忆管理：** 巧妙地解决了长上下文迭代中的信息冗余与噪声问题。
*   **高度可扩展：** Agent Skill 模块允许用户通过简单的 Markdown 文件添加新技能，具有极强的灵活性。
*   **模型无关性：** 作为一个“外挂”框架，可以适配各种 T2I 和 MLLM 后端。

#### 8. 不足与局限
*   **推理延迟：** 迭代过程（最多 5 轮）必然带来更高的推理延迟，不适用于对实时性要求极高的场景。
*   **依赖后端 MLLM：** 系统的上限在很大程度上取决于验证器（Verifier）和重写器（Refiner）所使用的 MLLM 的推理能力。
*   **任务覆盖：** 目前主要集中在静态图像生成，尚未扩展到视频生成或复杂的图像编辑任务。
*   **自主性限制：** 目前仍依赖预定义的流转逻辑，智能体在工具调用和自我管理方面的自主性仍有提升空间。
