## 摘要
提升语言模型数学推理能力的主流范式依赖于具有可验证奖励的强化学习。然而，现有方法孤立地处理每个问题实例，未能利用训练过程中出现并积累的可复用策略。为此，我们提出了 ARISE（通过内在技能演化的智能体推理），这是一个分层强化学习框架，其中共享策略既在顶层管理技能，又在底层生成响应（分别称为技能管理器和执行器）。管理器通过专门的技能生成展开（rollout）来维护分层技能库，该过程对成功的解题轨迹进行结构化总结（执行后），同时采用策略驱动的选择机制来检索相关技能，以指导未来的展开（执行前）。分层奖励设计引导了推理能力与技能库质量的协同演化。在两个基础模型和涵盖竞赛数学及 Omni-MATH 的七个基准测试上的实验表明，ARISE 一贯优于 GRPO 系列算法和记忆增强基线，在分布外（out-of-distribution）任务上的提升尤为显著。消融研究证实，每个组件都对观察到的改进有所贡献，且技能库质量和推理性能在整个训练过程中同步提升。代码可在 https://github.com/Skylanding/ARISE 获取。

## Abstract
The dominant paradigm for improving mathematical reasoning in language models relies on Reinforcement Learning with verifiable rewards. Yet existing methods treat each problem instance in isolation without leveraging the reusable strategies that emerge and accumulate during training. To this end, we introduce ARISE (Agent Reasoning via Intrinsic Skill Evolution), a hierarchical reinforcement learning framework, in which a shared policy operates both to manage skills at high-level and to generate responses at low-level (denoted as a Skills Manager and a Worker, respectively). The Manager maintains a tiered skill library through a dedicated skill generation rollout that performs structured summarization of successful solution traces (after execution), while employing a policy-driven selection mechanism to retrieve relevant skills to condition future rollouts (before execution). A hierarchical reward design guides the co-evolution of reasoning ability and library quality. Experiments on two base models and seven benchmarks spanning both competition mathematics and Omni-MATH show that ARISE consistently outperforms GRPO-family algorithms and memory-augmented baselines, with particularly notable gains on out-of-distribution tasks. Ablation studies confirm that each component contributes to the observed improvements and that library quality and reasoning performance improve in tandem throughout training. Code is available at \href{https://github.com/Skylanding/ARISE}{https://github.com/Skylanding/ARISE}.

### 1. 核心问题与研究动机
*   **核心问题**：当前的语言模型数学推理强化学习（RL）范式（如 DeepSeek-R1/GRPO）通常孤立地处理每个问题。模型在训练过程中产生的成功推理策略（Skills）在单次迭代后往往被丢弃，无法像人类一样积累并复用跨问题的通用策略。
*   **研究动机**：现有的“记忆增强”方法通常依赖外部检索器或静态知识库，导致技能管理与策略优化脱节。作者希望构建一个**内在技能演化**框架，让模型在提升推理能力的同时，自主发现、总结并优化一套可复用的技能库。

### 2. 方法论
ARISE 提出了一个**分层强化学习（HRL）**框架，将单一策略网络 $\pi_\theta$ 同时作为“技能管理器（Manager）”和“执行者（Worker）”：
*   **演化技能 MDP (ES-MDP)**：将技能库视为智能体状态的一部分。状态由（查询 $q$，技能库 $M$）组成。
*   **管理器-执行者架构**：
    *   **技能选择（下载）**：在执行前，管理器利用策略自身的条件对数概率（Log-probability）对库中技能进行评分，选择最相关的技能注入 Prompt。这种“策略驱动”的选择允许梯度直接回传，优化选择偏好。
    *   **解题执行**：执行者根据注入的技能生成解答轨迹。
    *   **技能生成（上传）**：在获得成功解题轨迹后，管理器执行一次额外的“总结展开（OG+1）”，将成功经验提炼为结构化的 JSON 技能文档。
*   **两层技能库架构**：采用“缓存（Cache）+ 仓库（Reservoir）”设计。通过 `ADD`（添加）、`UPDATE`（更新效用）、`EVICT`（驱逐）、`LOAD`（加载）和 `DELETE`（删除）五种操作动态维护技能质量。
*   **分层奖励设计**：设置奖励 $R \in \{0, 1, 2\}$。其中 $r_2$ 代表“使用技能且正确”，$r_1$ 代表“未使用技能但正确”，$r_0$ 代表“错误”。这种差异化奖励强制模型学习利用技能来获得更高的组相对优势（Group-relative advantage）。
*   **两阶段训练**：第一阶段为热身期，积累基础推理能力并初步填充技能库；第二阶段激活全流程分层 RL。

### 3. 实验设计
*   **基础模型**：Qwen3-4B-Instruct-2507 和 Phi-4-mini-instruct (3.8B)。

*   **基准测试**：

*   **对比方法**：
    *   **原生 RL 类**：GRPO、Dr.GRPO、DAPO、GSPO。
    *   **记忆/技能增强类**：EvolveR、SimpleMem、SkillRL。

*   **硬件环境**：使用了 **8 台 NVIDIA A100-80GB GPU**。
*   **训练细节**：总计训练 2000 个优化步（Steps）。

*   **开销分析**：相比原生 GRPO，ARISE 增加了约 16.6% 的训练耗时（主要来自技能评分和额外的总结展开）。

### 5. 实验数量与充分性
*   **实验规模**：在两个不同架构的基础模型上进行了验证，涵盖了从竞赛级到奥数级的 7 个主流基准。
*   **评估严谨性**：所有结果均为 **32 次独立运行的 Pass@1 平均值**，有效降低了 RL 训练的随机性干扰。
*   **消融实验**：针对分层奖励、随机选择、技能生成组件以及置信度门控进行了详细消融，证明了各组件的必要性。
*   **客观性**：对比了多种最新的 GRPO 变体，实验设计较为公平，且在 OOD 任务上的表现增强了结论的说服力。

### 6. 主要结论与发现
*   **性能领先**：ARISE 在所有基准上均优于 GRPO 系列和现有的记忆增强方法。在 Qwen3-4B 上，Omni-MATH 的平均准确率提升了 2.9%。
*   **协同演化**：实验观察到技能库的质量（效用值）与模型的推理准确率同步增长，证明了“推理能力提升”与“技能库丰富”之间的正反馈循环。
*   **技能饱和现象**：训练后期，技能库规模趋于稳定，但准确率继续提升，说明模型在后期更多地学习如何更精准地**选择**已有技能，而非盲目增加新技能。
*   **泛化能力**：ARISE 在未见过的奥数题目（Omni-MATH）上表现尤为突出，说明积累的技能具有跨领域的迁移价值。

### 7. 优点
*   **内生性**：技能管理完全由策略网络自身驱动，无需外部检索模型，实现了端到端的优化。
*   **结构化**：通过 JSON Schema 强制规范技能格式，避免了长文本注入带来的冗余和干扰。
*   **效率平衡**：通过两层缓存机制和硬性长度限制（220 字符），将 Token 开销控制在极低水平（约 3%），实现了极高的“增益/开销比”。

### 8. 不足与局限
*   **任务局限性**：目前仅在数学推理任务上进行了验证。在代码生成或多工具调用等更复杂的 Agent 场景下，技能的定义和演化可能更具挑战性。
*   **冷启动依赖**：虽然有自动生成机制，但初期仍依赖 5 个种子技能（Seed Skills）进行引导。
*   **计算开销**：虽然 Token 开销小，但每一步需要对缓存中的技能进行多次前向推理评分，在大规模部署时仍存在一定的计算延迟。
