## 摘要
大语言模型（LLMs）在许多推理基准测试中取得了强劲的表现，但这些评估通常聚焦于孤立的任务，与任务导向型对话（TOD）中的实际应用场景存在差异。在这种设定下，LLMs 必须在生成文本并遵循角色、格式和风格指令的同时进行内在推理。这种不匹配引发了关于基准测试性能是否能准确反映模型在 TOD 设定中推理鲁棒性的担忧。我们通过引入 BOULDER 研究了在 TOD 中构建推理任务对 LLM 性能的影响。BOULDER 是一个全新的动态基准，涵盖了八个与旅游相关的任务，要求在常识和形式层面进行算术、空间和时间推理。每个问题都以孤立和基于对话的两种变体呈现，从而在减轻数据污染的同时实现受控比较。对八种 LLM 的实验表明，孤立设定与对话设定之间存在显著且一致的性能差距。通过消融实验和定性分析，我们发现这一差距主要由对话的多轮性质驱动，并受到角色约束和工具使用要求的额外影响。我们的研究结果强调了在真实的交互场景中评估 LLM 推理能力的必要性。

## Abstract
Large Language Models (LLMs) achieve strong performance on many reasoning benchmarks, yet these evaluations typically focus on isolated tasks that differ from real-world usage in task-oriented dialogue (TOD). In this setting, LLMs must perform reasoning inherently while generating text and adhering to instructions on role, format, and style. This mismatch raises concerns about whether benchmark performance accurately reflects models' reasoning robustness in TOD setting. We investigate how framing reasoning tasks within TOD affects LLM performance by introducing BOULDER, a new dynamic benchmark covering eight travel-related tasks that require arithmetic, spatial, and temporal reasoning with both commonsense and formal aspects. Each problem is presented in both isolated and dialogue-based variants, enabling controlled comparison while mitigating data contamination. Experiments on eight LLMs reveal a substantial and consistent performance gap between isolated and dialogue settings. Through ablations and qualitative analysis, we show that this gap is largely driven by the multi-turn nature of dialogue, with additional effects from role conditioning and tool-use requirements. Our results highlight the need to evaluate LLM reasoning in realistic interactive scenarios.

*   **核心问题**：将推理任务嵌入到对话场景中，是否会削弱 LLM 的推理鲁棒性？如果性能下降，其背后的驱动因素是什么？
*   **研究意义**：揭示了现有推理评估与实际应用之间的巨大鸿沟，证明了对话环境本身会增加推理的认知负荷。

### 2. 方法论
*   **BOULDER 基准测试**：构建了一个动态、可自动验证的基准，涵盖 8 类旅行相关任务：
    *   **算术推理**：火车票价计算、酒店预订价格。
    *   **时间推理**：火车出发时间（基于日落）、出发频率计算、餐厅营业时间重叠。
    *   **空间推理**：场馆间距离、方位关系判断、最短步行路径（旅行商问题变体）。
*   **核心思想**：为每个推理实例设计两个版本——**孤立设置（Baseline）**和**对话设置（Dialogue）**。两者底层推理逻辑完全一致，但后者嵌入在包含工具调用记录和多轮历史的对话模板中。
*   **关键技术**：
    *   **动态生成**：利用模板和数据库随机生成实例，防止数据污染。
    *   **LLM 答案提取器**：由于对话回复是非结构化的，研究者设计了专门的 LLM 解析器（基于 Qwen3 30B）将自然语言回复转化为可验证的 JSON 格式。
    *   **偏差修正（PPI）**：使用“预测增强推理”（Prediction-Powered Inference）技术，结合人工标注的小样本对解析器的误差进行统计学修正，确保评估结果的客观性。

### 6. 主要结论与发现
*   **性能大幅下滑**：所有模型在对话设置下的表现均显著低于孤立设置。即使是顶级模型（如 Claude 4.5 Sonnet），在对话中的推理准确率也出现了明显下降。
*   **多轮对话是主因**：消融实验证明，对话的多轮性质（Multi-turn）对性能打击最大。将对话压缩为单轮后，推理能力会有所回升。
*   **角色偏见（Persona Bias）**：要求模型扮演“助手”角色会诱导模型生成更短、更具社交性但推理更草率的回复。
*   **工具调用的副作用**：即使模型不需要实际执行工具，仅在上下文中存在工具 Schema 和调用记录也会干扰推理过程。

### 7. 优点与亮点
*   **动态基准设计**：有效解决了 LLM 评估中日益严重的“数据污染”问题。
*   **受控对比**：通过最小化孤立任务与对话任务之间的描述差异，精准定位了“对话框架”本身带来的负面影响。
*   **评估方法创新**：结合 LLM 解析器与 PPI 统计修正，既实现了大规模自动化评估，又保证了结果的科学严谨。

### 8. 不足与局限
*   **领域局限性**：实验仅限于旅行相关的四个领域，可能无法完全代表医疗、法律等更复杂领域的推理挑战。
*   **零样本限制**：实验主要在 Zero-shot 设置下进行，未探讨 Few-shot 提示或专门的微调是否能缓解这一问题。
*   **推理模型的缺失**：研究未深入探讨专门的“推理型模型”（如 DeepSeek-R1 或 OpenAI o1）在对话嵌入场景下的表现（尽管提到其高延迟在 TOD 中不实用）。
*   **解析器依赖**：尽管有 PPI 修正，但评估过程仍高度依赖于另一个 LLM 的解析能力。
