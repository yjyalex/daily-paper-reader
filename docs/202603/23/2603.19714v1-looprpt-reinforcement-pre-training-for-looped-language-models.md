## 摘要
循环语言模型（LoopLMs）通过迭代隐层计算来优化内部表示，为显式思维链（CoT）推理提供了一种极具前景的替代方案。然而，现有的强化学习（RL）范式主要针对输出 token，这与推理过程隐式展开的循环架构在结构上存在不匹配。在这项工作中，我们提出了 LoopRPT，这是一个专为 LoopLMs 定制的强化预训练框架。通过将下一 token 预测重新定义为下一 token 推理任务，LoopRPT 利用 EMA 教师参考和噪声隐层展开（noisy latent rollouts），直接将强化信号分配给隐层步骤。这种形式化方法使 RL 能够直接塑造中间表示，从而将有效的推理压缩到更少的迭代次数中。我们在多种模型规模的 Ouro 架构上实例化了 LoopRPT。结果表明，LoopRPT 持续提升了每步表示的质量，在准确率与计算量的权衡中实现了帕累托主导（Pareto dominance）。值得注意的是，在困难 token 上的显著提升表明 LoopRPT 增强了早期推理，而不仅仅是鼓励过早退出。我们的研究结果强调，强化预训练是学习 LoopLMs 中高效隐式推理的一种原则性范式。

## Abstract
Looped language models (LoopLMs) perform iterative latent computation to refine internal representations, offering a promising alternative to explicit chain-of-thought (CoT) reasoning. However, existing reinforcement learning (RL) paradigms primarily target output tokens, creating a structural mismatch with looped architectures whose reasoning unfolds implicitly. In this work, we propose LoopRPT, a reinforcement pre-training framework tailored for LoopLMs. By reframing next-token prediction as a next-token reasoning task, LoopRPT assigns reinforcement signals directly to latent steps using an EMA teacher reference and noisy latent rollouts. This formulation enables RL to directly shape intermediate representations, compressing effective reasoning into fewer iterations. We instantiate LoopRPT on the Ouro architecture across multiple model scales. Results demonstrate that LoopRPT consistently improves per-step representation quality, achieving Pareto dominance in accuracy-computation trade-offs. Notably, significant gains on hard tokens indicate that LoopRPT enhances early-stage reasoning rather than merely encouraging premature exits. Our findings highlight reinforcement pre-training as a principled paradigm for learning efficient latent reasoning in LoopLMs.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：现代大语言模型（LLM）通常依赖显式的“思维链”（CoT）生成来进行推理，这会消耗大量 token 且推理成本高。**循环语言模型（LoopLMs）**（如 Ouro 架构）通过在隐空间进行迭代计算来优化内部表示，实现了“隐式推理”。然而，现有的强化学习（RL）范式（如 RLVR）主要针对最终输出的 token，无法有效解决循环架构中隐式推理步骤的**信用分配（Credit Assignment）**问题，导致难以优化其动态退出机制和中间推理质量。
*   **研究动机**：如何利用强化学习来激励 LoopLMs 在不增加 token 成本的前提下，通过更高效的隐空间迭代实现更强的推理能力？

### 2. 方法论：LoopRPT 框架
LoopRPT 是首个将**强化预训练（RPT）**引入循环架构的框架，其核心思想是将“下一 token 预测”视为一个“推理任务”，直接对隐层迭代步骤施加奖励。
*   **基于熵的困难 Token 选择**：并非所有 token 都需要复杂推理。LoopRPT 使用教师模型的输出熵来识别“困难 token”，仅在这些位置应用强化学习，以提高训练效率。
*   **EMA 教师参考与步进式奖励**：
    *   维护一个**指数移动平均（EMA）教师模型**作为稳定的基准。
    *   **准确率增益（Accuracy Gain）**：计算学生模型在每个迭代步的对数概率与教师参考步之间的差值。
    *   **难度感知时间惩罚**：引入惩罚项以鼓励模型尽早退出，且惩罚力度根据 token 难度动态调整（简单 token 惩罚重，困难 token 允许更多迭代）。
*   **噪声隐层展开（Noisy Latent Rollouts）**：在隐层循环中注入高斯噪声，通过多次采样（Rollouts）来模拟策略的变化，利用组得分强化学习（GRPO）的思想同时优化退出策略和骨干表示。
*   **联合优化目标**：结合了策略梯度损失（LPG）、步进加权表示学习损失（Lrep）、熵奖励（Lent）和 KL 散度约束（LKL）。

### 3. 实验设计
*   **模型架构**：在 **Ouro-1.4B** 和 **Ouro-2.6B** 两种规模上进行实例化。
*   **训练数据**：使用 **Omni-Math**（包含约 4,428 个竞赛级数学问题及解答）。
*   **对比基准（Baselines）**：
    *   **非循环模型**：Qwen2.5 (1.5B/3B/7B), Qwen3 (1.7B/4B/8B), Llama3.1/3.2, Gemma3 等。
    *   **循环模型基色**：原始的 Ouro-1.4B 和 Ouro-2.6B。
*   **评估任务**：
    *   **下一 Token 推理（NTR）**：评估在不同难度下的预测准确率与平均计算步数。
    *   **下游基准**：MMLU, MMLU-Pro, BBH, GSM8K（数学）, HumanEval/MBPP（代码）等。

### 4. 资源与算力
*   **硬件环境**：使用 **8 台 NVIDIA A100 (80GB) GPU**。
*   **训练时长**：
    *   Ouro-1.4B 训练约 **2 小时**。
    *   Ouro-2.6B 训练约 **4 小时**。
*   **超参数**：Batch size 为 8，训练 3 个 Epoch，使用 AdamW 优化器，学习率为 5e-6。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 1.4B 到 2.6B 的模型缩放，验证了方法的可扩展性。
*   **消融实验**：非常详尽，分别验证了高斯噪声、策略梯度损失、表示学习损失、Token 选择机制以及时间惩罚项的必要性。
*   **敏感性分析**：对序列长度、KL 系数、GRPO 组大小进行了参数敏感性测试。
*   **客观性**：实验不仅报告了最终准确率，还详细分析了“准确率-计算量”的权衡曲线（Pareto Curves）以及每一步迭代的质量提升，评估视角全面且客观。

### 6. 主要结论与发现
*   **帕累托主导**：LoopRPT 在所有难度级别上均实现了比原始 Ouro 模型更高的准确率和更少的平均计算步数。
*   **增强早期推理**：LoopRPT 显著提升了前 1-2 步迭代的表示质量，证明其并非单纯通过“早停”来节省计算，而是提高了每一步的计算效率。
*   **下游迁移能力**：在数学（GSM8K +3.6%）和代码（MBPP+ +2.9%）任务上表现出显著的性能增益，证明了强化预训练在隐空间推理上的有效性。
*   **退出策略校准**：模型在处理困难 token 时能更自信地在较早步骤达到高准确率，优化了推理动态。

### 7. 优点
*   **创新性**：首次将 RPT 范式与循环架构结合，解决了隐空间推理的密集信用分配问题。
*   **效率极高**：通过隐空间迭代代替显式 token 生成，大幅降低了推理延迟和计算成本。
*   **设计精巧**：引入 EMA 教师和难度感知惩罚，平衡了推理深度与计算速度。
*   **即插即用**：该框架为未来更大规模的循环模型训练提供了标准化的 RL 预训练配方。

### 8. 不足与局限
*   **架构依赖**：目前仅在 Ouro 这一特定循环架构上验证，其在其他变体（如不同循环连接方式）上的通用性有待观察。
*   **数据领域局限**：训练主要集中在数学竞赛数据（Omni-Math），虽然在通用任务上有提升，但在更广泛的自然语言理解或多模态任务上的表现尚未充分探讨。
*   **训练稳定性**：尽管引入了多种稳定机制，但 RL 预训练本身对超参数（如噪声强度、惩罚系数）较为敏感，在大规模扩展时可能面临挑战。
