## 摘要
多智能体应用通常将复杂任务执行为多阶段工作流，其中每个阶段都是一次 LLM 调用，其输出将成为后续步骤上下文的一部分。现有的 LLM 推理服务系统在很大程度上假设集群是同构的，且拥有相同的模型副本。这种设计忽视了异构部署的潜力，即不同规模和能力的模型可以实现延迟与性能之间更精细的权衡。然而，异构性在具有不同吞吐量和性能的模型之间引入了调度方面的新挑战。我们提出了 Chimera，这是一个用于异构 LLM 集群上多智能体工作流服务的预测性调度系统，可同时优化端到端延迟和任务性能。Chimera 应用语义路由来估计每个请求在各模型上的置信度分数，预测工作流的总剩余输出长度，并利用处理中的预测 Token 量来估计各模型的拥塞情况，从而实现负载均衡。我们在代码生成和数学推理的代表性智能体工作流上，使用多种异构 LLM 配置对 Chimera 进行了评估。在同类设置下，Chimera 描绘了最佳的延迟-性能前沿，与包括 vLLM 在内的竞争基准相比，端到端延迟降低了 1.2--2.4 倍，任务性能平均提高了 8.0-9.5 个百分点。

## Abstract
Multi-agent applications often execute complex tasks as multi-stage workflows, where each stage is an LLM call whose output becomes part of context for subsequent steps. Existing LLM serving systems largely assume homogeneous clusters with identical model replicas. This design overlooks the potential of heterogeneous deployments, where models of different sizes and capabilities enable finer trade-offs between latency and performance. However, heterogeneity introduces new challenges in scheduling across models with diverse throughput and performance. We present Chimera, a predictive scheduling system for multi-agent workflow serving on heterogeneous LLM clusters that jointly improves end-to-end latency and task performance. Chimera applies semantic routing to estimate per-model confidence scores for each request, predicts the total remaining output length of the workflow, and estimates per-model congestion using in-flight predicted token volumes for load balancing. We evaluate Chimera on representative agentic workflows for code generation and math reasoning using multiple heterogeneous LLM configurations. Across comparable settings, Chimera traces the best latency-performance frontier, reducing end-to-end latency by 1.2--2.4$\times$ and improving task performance by 8.0-9.5 percentage points on average over competitive baselines including vLLM.

*   **核心问题**：如何在异构大语言模型（LLM）集群中，针对多智能体（Multi-agent）工作流进行高效调度，以同时优化**端到端延迟**和**任务执行性能**。
*   **研究动机**：
    *   **多智能体趋势**：现代应用常将复杂任务分解为多阶段工作流（如：规划->编码->测试），各阶段依赖性强。
    *   **同构假设的局限**：现有推理系统（如 vLLM）多假设集群由相同模型组成，忽视了异构模型（不同尺寸、能力）在成本、速度和质量之间的权衡潜力。
    *   **调度挑战**：在异构环境下，简单的路由会导致负载不均；而追求高性能模型可能导致严重的队列拥塞，从而大幅增加延迟。

### 2. 方法论
Chimera 是一个位于应用层和推理引擎之间的中间件，包含四个核心组件：
*   **语义路由器 (Semantic Router)**：基于 ModernBert-large 的编码器，为每个请求预测各候选模型的“置信度分数”（即解决该任务的概率）。
*   **长度预测器 (Length Predictor)**：使用轻量级 CPU 随机森林回归（QRF），预测整个工作流**剩余阶段的总输出 Token 长度**。
*   **活动监测器 (Activity Monitor)**：实时跟踪各引擎中“处理中”的预测 Token 总量，用于估算各模型的实时负载和延迟。
*   **负载均衡器 (Load Balancer)**：
    *   **调度策略**：采用“延迟感知路由”，在满足一定延迟容差（Latency Slack）的范围内，选择置信度最高的模型。
    *   **优先级机制**：实施**最短总任务优先 (STJF)** 策略，即优先处理预测总长度短的工作流。
    *   **抗饥饿机制**：引入基于老化（Aging）的提升机制，防止长任务被无限期推迟。

### 3. 实验设计
*   **数据集/场景**：
    *   **APPS**：代码生成任务，模拟“规划者->编码者->QA”等多阶段工作流。
    *   **MATH**：数学推理任务，模拟“规划者->求解者->验证者”工作流。
*   **模型配置**：使用了 Qwen2.5 (1.5B/3B/7B/14B)、Ministral 8B 和 Llama 3.2 3B 的异构组合。
*   **对比基准 (Baselines)**：
    *   **vLLM**：标准的负载均衡推理引擎。
    *   **MLFQ**：多级反馈队列调度。
    *   **LTR (Learning to Rank)**：基于排序预测的短作业优先调度。
    *   **vLLM (STJF) Oracle**：使用真实长度信息的理想化调度。

### 4. 资源与算力
*   **硬件环境**：实验运行在 **NVIDIA RTX A6000 GPU** (48GB VRAM) 上。
*   **训练细节**：
    *   语义路由器在 ModernBert-large 上微调了 5 个 epoch。
    *   长度预测器在 CPU 上运行，推理开销极低。
*   **系统开销**：Chimera 的调度器（含路由和预测）仅占端到端延迟的 **0.2% - 2.2%**，证明了其轻量化特性。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 2 模型和 3 模型的多种异构组合，并在不同的请求频率（RPS = 8, 12, 16）下进行了测试。
*   **消融实验**：对比了有无语义路由器、有无长度预测器以及与 Oracle（真值）预测的差距。
*   **评价**：实验设计较为充分，通过 Pareto 前沿曲线展示了延迟与性能的权衡，对比方法具有代表性，结果客观反映了系统在不同负载下的鲁棒性。

### 6. 主要结论与发现
*   **性能提升**：相比 vLLM，Chimera 将端到端延迟降低了 **1.2-2.4 倍**，同时任务准确率提升了 **8.0-9.5%**。
*   **工作流感知的重要性**：证明了针对“整个工作流”预测长度（STJF）比针对“单个请求”预测长度（SJF）能更显著地减少排队时间。
*   **异构优势**：通过动态路由，系统能在负载高时自动回退到较小模型以保证延迟，在负载低时利用大模型提升质量。

### 7. 优点
*   **协同优化**：首次在多智能体推理服务中实现了模型路由、负载均衡和优先级调度的深度耦合。
*   **实用性强**：作为 vLLM 的插件层实现，兼容 OpenAI API，易于部署。
*   **高效轻量**：通过异步批处理路由和 CPU 预测，极大地降低了调度延迟。

### 8. 不足与局限
*   **冷启动/离线依赖**：语义路由器和长度预测器需要针对特定任务（如代码或数学）进行预训练或收集离线轨迹，对于全新领域的泛化能力未深入探讨。
*   **模型部署假设**：假设异构模型已预先加载在特定引擎上，未涉及动态模型交换或 KV 缓存跨模型迁移的复杂场景。
*   **任务类型局限**：实验主要集中在逻辑性强的代码和数学任务，在创意写作等输出长度波动极大的场景下，预测准确性可能下降。
