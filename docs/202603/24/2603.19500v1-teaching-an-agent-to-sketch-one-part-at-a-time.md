## 摘要
我们开发了一种逐部分生成矢量草图的方法。为此，我们在监督微调的基础上，使用一种新颖的多轮过程奖励强化学习来训练基于多模态语言模型的智能体。我们的方法得益于一个名为 ControlSketch-Part 的新数据集，该数据集包含丰富的草图部分级标注，是通过一种新颖且通用的自动标注流水线获得的，该流水线将矢量草图分割成语义部分，并通过结构化的多阶段标注过程将路径分配给各个部分。我们的结果表明，结合结构化的部分级数据并在过程中为智能体提供视觉反馈，可以实现可解释、可控且可局部编辑的文本到矢量草图生成。

## Abstract
We develop a method for producing vector sketches one part at a time. To do this, we train a multi-modal language model-based agent using a novel multi-turn process-reward reinforcement learning following supervised fine-tuning. Our approach is enabled by a new dataset we call ControlSketch-Part, containing rich part-level annotations for sketches, obtained using a novel, generic automatic annotation pipeline that segments vector sketches into semantic parts and assigns paths to parts with a structured multi-stage labeling process. Our results indicate that incorporating structured part-level data and providing agent with the visual feedback through the process enables interpretable, controllable, and locally editable text-to-vector sketch generation.

*   **研究动机**：现有的文本到矢量草图生成方法大多采用“一次性生成全图”的模式，忽略了人类绘画时逐步构思的过程。这种方式导致生成结果缺乏语义结构，难以进行局部编辑（如只修改椅子腿而不影响靠背），且在处理复杂指令时缺乏可解释性。
*   **核心问题**：如何训练一个能够理解语义部件（Parts）、根据文本指令逐步构建矢量草图，并支持交互式局部编辑的智能体。

#### 2. 论文提出的方法论
*   **核心思想**：通过构建高质量的部件级标注数据集，并结合多轮过程奖励强化学习（Multi-turn Process-Reward RL），使智能体学会“观察当前画布 -> 阅读部件指令 -> 绘制对应笔划”的循环过程。

    *   **自动标注流水线**：提出一种通用的 VLM 驱动流水线，将现有矢量草图（如 ControlSketch）转化为包含部件描述的数据集。过程包括：部件分解、自我批判与修正、路径到部件的分配、诊断性可视化反馈及最终描述生成。

        *   **路径计数奖励**：惩罚冗余笔划，使生成结果在简洁性上与训练数据对齐。

#### 6. 论文的主要结论与发现
*   **部件级建模的优越性**：逐部分生成不仅提高了视觉质量，还赋予了模型极强的局部编辑能力（如通过删除特定部件的笔划并重新生成来改变局部特征）。

*   **可解释与可控**：生成的草图具有清晰的语义结构，符合人类创作逻辑。
*   **强大的编辑功能**：支持基于文本的局部修改，这是以往“一次性生成”模型难以实现的。

#### 8. 不足与局限
*   **数据依赖性**：模型性能受限于 ControlSketch 数据集的类别覆盖范围，对于训练集之外的罕见概念（如特定形状的轮子）可能生成失败。
*   **路径计数限制**：由于训练数据路径数固定，模型有时会为了匹配路径数而提前停止绘制，导致部分细节缺失。
*   **缺乏自我修正**：目前智能体尚不具备在发现错误后主动回溯修改的能力，早期步骤的严重错误仍可能影响后续生成。
