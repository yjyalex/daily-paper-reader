## 摘要
工业界从业者和学术研究人员经常使用多智能体系统来加速工作，但这些系统运行的框架并未提供一种简单、统一的机制来可扩展地管理智能体控制台（harness）的关键方面，这影响了单个人机交互的质量，也限制了从业者通过共享智能体基础设施协作达成共同目标的能力。智能体框架已经实现了日益复杂的多智能体系统，但定义这些智能体行为的行为规范仍然分散在散文指令文件、框架内部配置以及像 MCP 服务器这样独立于单个智能体定义的机制中，使得这些规范难以在团队和项目之间共享、进行版本控制或协同维护。我们将辐射安全中的 ALARA 原则（暴露量保持在合理可达到的最低水平）应用于智能体上下文，引入了一个声明式的上下文-智能体-工具（CAT）数据层。该层通过相互关联的文件表达，将每个智能体的工具访问权限和上下文限制在其角色所需的最低限度，并开发了用于执行该系统的命令行 shell npcsh。由于系统以结构化方式解析并强制执行这些文件，修改智能体的工具列表会产生确定的行为变化，而非模型可能遵循也可能不遵循的建议。我们在涵盖文件操作、网络搜索、多步脚本编写、工具链调用和多智能体委派的 115 个实际任务中，评估了 22 个参数量从 6 亿到 350 亿不等的本地托管模型，通过约 2500 次总执行量，刻画了哪些模型系列在哪些任务类别中表现出色，以及它们在何处失效。

## Abstract
Industry practitioners and academic researchers regularly use multi-agent systems to accelerate their work, yet the frameworks through which these systems operate do not provide a simple, unified mechanism for scalably managing the critical aspects of the agent harness, impacting both the quality of individual human-agent interactions and the capacity for practitioners to coordinate toward common goals through shared agent infrastructure. Agent frameworks have enabled increasingly sophisticated multi-agent systems, but the behavioral specifications that define what these agents can do remain fragmented across prose instruction files, framework-internal configuration, and mechanisms like MCP servers that operate separately from individual agent definitions, making these specifications difficult to share, version, or collaboratively maintain across teams and projects. Applying the ALARA principle from radiation safety (exposures kept as low as reasonably achievable) to agent context, we introduce a declarative context-agent-tool (CAT) data layer expressed through interrelated files that scope each agent's tool access and context to the minimum its role requires, and \texttt{npcsh}, a command-line shell for executing it. Because the system parses and enforces these files structurally, modifying an agent's tool list produces a guaranteed behavioral change rather than a suggestion the model may or may not follow. We evaluate 22 locally-hosted models from 0.6B to 35B parameters across 115 practical tasks spanning file operations, web search, multi-step scripting, tool chaining, and multi-agent delegation, characterizing which model families succeed at which task categories and where they break down across $\sim$2500 total executions.

### 论文总结：ALARA for Agents

#### 1. 核心问题与整体含义（研究动机和背景）
当前多智能体系统（MAS）的开发面临**配置碎片化**的挑战：智能体的行为规范、工具访问和协作逻辑往往散落在自然语言指令（如 `.cursorrules`）、框架代码（如 LangGraph）和外部协议（如 MCP）中。这种碎片化导致系统难以版本化、难以在团队间共享，且由于依赖模型对“散文式指令”的解释，存在行为不可预测和安全风险（如提示注入）。
论文借鉴了辐射安全领域的 **ALARA 原则**（As Low As Reasonably Achievable，保持在合理达到的最低水平），将其引入智能体上下文工程，旨在通过**最小特权原则**（Least-Privilege）解决工具滥用、上下文污染和安全漏洞问题。

#### 2. 方法论：核心思想与技术细节
论文提出了一种声明式的 **CAT（Context-Agent-Tool）数据层**，通过结构化文件而非纯文本指令来强制执行约束：
*   **Context（上下文文件）**：在团队层面定义资源共享、指定编排者（Orchestrator）以及子团队层级，通过物理目录结构隔离上下文，防止无关信息干扰决策。
*   **NPC（智能体定义文件）**：使用 YAML 定义单个智能体，包括其模型配置和 **Jinx 列表**。Jinx 列表既是工具目录也是权限集，未列出的工具在架构上对智能体不可见，从而实现结构化（而非解释性）的权限控制。
*   **Jinxes（执行模板）**：基于 YAML 和 Jinja 模板的工具定义。Jinx 可以调用 Python、Bash 或其他 Jinx，形成**有向无环图（DAG）**。这种设计消除了模式定义与实际行为之间的偏差，并允许在不支持原生函数调用的轻量级模型上运行复杂工作流。
*   **npcsh**：一个专用的命令行 Shell，用于解析并执行上述 CAT 数据层，支持交互式、API 服务、桌面应用等多种接口。

#### 6. 主要结论与发现
*   **结构化约束的有效性**：通过 Jinx 列表进行的结构化权限控制比散文指令更可靠，能有效防止模型在长上下文中因注意力稀释而产生的误操作。
*   **家族差异胜过规模差异**：模型家族之间的性能差异（Between-family variance）大于同一家族内参数规模带来的提升。例如，专门针对工具调用训练的 4B 模型表现可能优于未优化的 27B 模型。
*   **性能预测指标**：**工具调用量**（Tool call volume）是预测智能体成功率的最强指标（r≈0.7），强于任务时长或重试次数。
*   **重试价值的类别差异**：网页搜索等任务从重试中获益显著（+21%），而**多智能体委派（Delegation）**任务重试获益极低（+3%），且委派是所有模型表现最差的类别。
*   **能力相关性**：MMLU 得分与智能体表现呈正相关（r≈0.8），但存在显著的离群值，表明工具使用是一种需专门训练的独特能力。

#### 7. 优点
*   **架构优雅**：将辐射安全原则（ALARA）巧妙转化为计算机安全中的最小特权原则，并落实为声明式的文件系统结构。
*   **高可移植性**：CAT 数据层不依赖特定框架内部状态，可在不同环境（Shell、API、IDE）间无缝迁移。
*   **赋能小模型**：通过 Jinx 的确定性脚手架，使 1B-8B 规模的本地模型也能参与复杂的工具链协作。
*   **开源贡献**：框架和 Benchmark 均已开源，具有较强的实用价值。

#### 8. 不足与局限
*   **委派能力瓶颈**：实验显示所有模型在多智能体委派任务上表现均较差，说明目前的本地模型在处理复杂的跨智能体协调时仍存在根本性限制。
