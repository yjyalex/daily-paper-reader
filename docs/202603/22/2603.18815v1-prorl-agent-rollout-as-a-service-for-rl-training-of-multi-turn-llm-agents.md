## 摘要
多轮大语言模型（LLM）智能体在解决复杂的交互式任务中变得越来越重要，而强化学习（RL）是改善其长程行为的关键要素。然而，强化学习训练需要生成大量的沙箱化 rollout 轨迹，且现有的基础设施通常将 rollout 编排与训练循环耦合在一起，导致系统难以迁移和维护。秉持“rollout 即服务”的理念，我们提出了 ProRL Agent，这是一个可扩展的基础设施，通过 API 服务为完整的智能体 rollout 生命周期提供支持。ProRL Agent 还提供了标准化且可扩展的沙箱环境，支持在无根权限（rootless）高性能计算（HPC）设置下执行多种智能体任务。我们通过在软件工程、数学、STEM 和编程任务上的强化学习训练验证了 ProRL Agent。ProRL Agent 已开源，并作为 NVIDIA NeMo Gym 的一部分进行了集成。

## Abstract
Multi-turn LLM agents are increasingly important for solving complex, interactive tasks, and reinforcement learning (RL) is a key ingredient for improving their long-horizon behavior. However, RL training requires generating large numbers of sandboxed rollout trajectories, and existing infrastructures often couple rollout orchestration with the training loop, making systems hard to migrate and maintain. Under the rollout-as-a-service philosophy, we present ProRL Agent , a scalable infrastructure that serves the full agentic rollout lifecycle through an API service. ProRL Agent also provides standardized and extensible sandbox environments that support diverse agentic tasks in rootless HPC settings. We validate ProRL Agent through RL training on software engineering, math, STEM, and coding tasks. ProRL Agent is open-sourced and integrated as part of NVIDIA NeMo Gym.

### ProRL Agent: 面向多轮 LLM 智能体强化学习训练的 Rollout 即服务

#### 1. 论文的核心问题与整体含义（研究动机和背景）
多轮 LLM 智能体在处理复杂、长程任务（如软件工程、网页浏览）时，通常需要通过强化学习（RL）来优化其决策行为。然而，现有的智能体 RL 训练框架存在一个核心痛点：**回测（Rollout，即生成交互轨迹）与训练循环（Policy Update）高度耦合**。
*   **资源冲突**：Rollout 是 I/O 密集型的（涉及沙箱创建、工具调用、异步协调），而训练是 GPU 密集型的。耦合设计会导致资源利用率低下。
*   **维护困难**：Rollout 逻辑嵌入在训练器中，导致更换训练后端或扩展新任务时需要重构大量代码。
*   **环境限制**：现有的沙箱多依赖 Docker，但在高性能计算（HPC）集群中通常缺乏 root 权限，难以部署。

该论文提出了 **ProRL Agent**，核心理念是将 Rollout 视为一种独立的服务（Rollout-as-a-Service, RaaS），通过解耦提升系统的可扩展性、可移植性和效率。

#### 2. 论文提出的方法论
ProRL Agent 采用解耦架构，主要包含以下关键技术细节：
*   **Rollout-as-a-Service (RaaS)**：通过 HTTP API 提供完整的智能体回测生命周期管理。训练器只需提交任务实例，服务器负责环境初始化、多轮交互、结果评估并返回轨迹和奖励。
*   **三阶段异步流水线**：服务器内部将任务分为 **INIT（初始化）**、**RUN（运行交互）** 和 **EVAL（评估得分）** 三个独立阶段，每个阶段拥有独立的线程池。这种“流水线”设计允许不同任务的各阶段重叠执行，极大提升了吞吐量。
*   **HPC 兼容的无根沙箱**：基于 **Singularity** 构建容器运行时，支持在无 root 权限的 HPC 环境下运行。通过分配独立的环回 IP 和 Unix 域套接字（UDS）优化工具调用延迟。
*   **LLM 后端负载均衡**：使用最小堆（Min-Heap）管理推理服务器池，根据在办任务数动态分配请求，并支持在训练过程中动态更换模型检查点。
*   **Token-in/Token-out 通信**：全流程采用 Token ID 而非纯文本传输，避免了重分词（Re-tokenization）导致的分布偏移（Drift），确保训练的稳定性。
*   **优化版 DAPO 调度**：针对动态采样策略优化（DAPO）实现了异步补充机制，减少了等待“有效样本”时的工时浪费。
