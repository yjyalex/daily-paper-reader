# Expected Value Alignment for Generative Reward Modeling in Formal Mathematics Verification 

Shihao Ji, Haotao Tan, Zihui Song, Mingyu Li 

# Abstract 

Large Language Models (LLMs) are increasingly used with formal interactive the-orem provers such as Lean 4. Scaling these systems with reinforcement learning or search methods requires process reward models (PRMs) that can evaluate interme-diate reasoning steps. Existing reward-model designs expose a practical trade-off. Value-head models provide continuous scores but modify the generative model interface, while generative reward models preserve textual rationales but are poorly matched to continuous floating-point regression because numeric values are split across tokens. We introduce Expected Value Alignment (EVA), a reward-modeling procedure that keeps the surface output discrete while extracting continuous scores from the model’s token distribution. The model emits integer scores in a structured JSON format, and EVA computes a continuous score as the expectation over the logits of the corresponding anchor tokens. Training combines the causal language modeling objective with an auxiliary mean squared error loss on these expected values. We instantiate EVA in Leibniz , a reward model for Lean 4 formal veri-fication, and evaluate it against zero-shot and reward-modeling baselines. The evaluation demonstrates that continuous logit-based scoring significantly reduces discretization artifacts while retaining the interpretability of generative critiques. 

# 1 Introduction 

Large Language Models (LLMs) are increasingly applied to formal mathematics. Recent systems have used language models for formal theorem proving in environments such as Metamath and Lean, including generative proof search and retrieval-augmented theorem proving [Polu and Sutskever, 2020, Yang et al., 2023]. Unlike many natural language tasks, proof development in interactive theorem provers (ITPs) such as Lean 4, Coq, and Isabelle requires consistency with type-theoretic constraints and the current tactic state. Although LLMs can generate plausible informal proof sketches, formal tactic generation remains prone to hallucinations: a model may propose tactics that look syntactically plausible but are invalid or irrelevant in the current proof context. One way to reduce these errors is to train or search over proof trajectories using feedback signals, as in human-feedback reward modeling and process supervision [Ouyang et al., 2022, Lightman et al., 2024]. A central component in this setting is the process reward model (PRM), which evaluates intermediate reasoning steps and provides scalar rewards for policy optimization methods such as Proximal Policy Optimization (PPO) [Schulman et al., 2017] or search procedures such as Monte Carlo Tree Search (MCTS) [Coulom, 2007]. Common reward-model designs fall into two broad categories. A value-head reward model appends a regression head to the final hidden states of an LLM. This produces continuous scalar outputs, but it also changes the model interface and can make the learned reward less interpretable because no critique or rationale is generated. A generative reward model, or LLM-as-a-judge [Zheng et al., 2023], instead prompts the model to output a numerical score as text, for example Score: 4.25 . This preserves the generative interface and can produce Chain-of-Thought (CoT) critiques [Wei et al., 2022], but it is not naturally aligned

with continuous regression. Since LLMs predict discrete tokens, a floating-point score is represented by several token decisions, such as 4, ., and 25 . As a result, the probability of the first digit alone is an incomplete proxy for the model’s confidence in the continuous score. We address this mismatch with Expected Value Alignment (EVA). EVA uses a "latent continuous, surface discrete" design. During inference, the model is constrained to generate integer scores, for example {1, 2, 3, 4, 5}, within a predefined JSON schema. Rather than using only the sampled textual score, EVA extracts the pre-softmax logits at the score positions and computes the expectation over the integer anchor set. This yields a continuous reward signal while preserving a discrete, parseable output format. To align this expected value with supervision, we add an EVA loss during supervised fine-tuning (SFT). The loss applies a mean squared error (MSE) penalty to the expected score computed from the anchor logits. This encourages the score-token distribution to place probability mass near the continuous target, while the language modeling loss trains the model to produce the critique and JSON output. This work makes three connected contributions. First, it introduces EVA as a way to combine generative critiques with continuous scores extracted from score-token logits, without adding a separate value head. Second, it formalizes categorical anchor-token expectations for obtaining continuous rewards from discrete score outputs. Third, it describes an exact token indexing procedure based on character-to-token mapping and instantiates the method in Leibniz , a PRM for Lean 4 tactic state evaluation with separate Logic, Alignment, and Clarity scores. 

# 2 Background and Motivation 

2.1 Formal Mathematics and Lean 4 

Lean 4 is a functional programming language and theorem prover based on dependent type theory [de Moura and Ullrich, 2021]. In Lean, a proof is constructed interactively by applying tactics that transform a tactic state, consisting of hypotheses and goals, until the goals are resolved. Translating an informal mathematical step into a Lean tactic requires attention to variable types, implicit constraints, and the current proof state. Benchmarks such as miniF2F [Zheng et al., 2022] have made this setting more standardized, and a PRM for Lean 4 should therefore detect both syntactic failures and contextual mismatches, such as applying a theorem from an unrelated mathematical domain. This setting differs from ordinary mathematical text evaluation because correctness is local to a proof state. A proposed step can be mathematically true in isolation while still being unusable for the current goal, for example when it introduces objects that are absent from the context or proves a statement with a different type. Conversely, a terse Lean tactic may be valid even when it contains little natural language explanation. A reward model for this domain must therefore evaluate the interaction between the proposed step and the formal state, not only the plausibility of the mathematical content. 

2.2 Tokenization and Continuous Scores 

To understand the motivation for EVA, consider a ground truth reward of RGT = 3 .8. If an LLM is trained to generate the text "3.8" , the language modeling objective minimizes the negative log-likelihood of the token sequence [3, ., 8] .During inference, one possible approach is to extract the probability of the first digit token. Tokenizers can also group digits differently; for example, 3.85 may be tokenized as [3, ., 85] or [3, ., 8, 5] .These tokenization choices make first-token probabilities a noisy proxy for continuous scores. Restricting the model to a single categorical token, such as 1, 2, 3, 4, 5, avoids multi-token fragmenta-tion but introduces quantization. For example, a target score of 3.8 must be rounded to 4.0 in the generated text. EVA is designed to keep the output categorical while using the full anchor-token distribution to recover a continuous score. The quantization issue matters most when the reward is used for optimization rather than only reporting. # 3 Related Work 

Language-model theorem proving has developed along two complementary directions. One line treats proof construction as generative search over formal proof states, where the model proposes candidate tactics or proof steps that are checked by a formal system [Polu and Sutskever, 2020]. Another line augments the prover with retrieval from theorem libraries and examples, improving access to relevant premises and proof patterns [Yang et al., 2023]. EVA is orthogonal to these systems: it does not propose tactics directly, but estimates the quality of candidate reasoning steps so that a prover or search algorithm can prioritize them. Reward modeling provides a second source of context. Instruction following with human feedback showed that learned reward models can guide language model behavior at scale [Ouyang et al., 2022], while process supervision studies reward signals for intermediate reasoning steps rather than only final answers [Lightman et al., 2024]. Formal mathematics is a natural setting for process-level feedback because the state of the proof is explicit and errors can occur long before the final theorem statement. At the same time, formal verification introduces stricter alignment requirements: a reward model must evaluate whether a step applies to the current tactic state, not only whether it resembles a plausible mathematical explanation. LLM-as-a-judge methods preserve a generative interface and can produce textual assessments in addition to scalar scores [Zheng et al., 2023]. This is useful for formal proof development because a critique can indicate whether a candidate fails due to logic, state alignment, or presentation. However, generative judges usually emit scores as text, which makes continuous supervision indirect. EVA addresses this interface mismatch by keeping the judge generative at the surface while extracting the reward from the score-token distribution. 

# 4 The Expected Value Alignment (EVA) Framework 

EVA separates the textual output from the reward value used by downstream optimization. The model produces discrete text for parsing and inspection, while the reward is computed as an expectation over selected token logits.

4.4 Exact Token Indexing via Character Mapping 

A practical challenge in implementing Equation 5 during batched training is locating the sequence index idx k for each score token. Training sequences vary in length because CoT rationales have different lengths, so fixed positional indexing is not reliable. Padding mechanisms, often represented as ignore_index = -100 in PyTorch, can also cause indexing errors when token positions are misaligned. We handle this with a deterministic character-to-token absolute mapping algorithm during data collation. For each training sample, a regular expression match on the raw string identifies the character offset of each target JSON value, such as the digit after "logic_score": . The fast tokenizer’s char_to_token() method then maps these character offsets to token indices. Because causal language models predict the token at position t using the hidden state at t − 1, we extract logits at sequence position idx k − 1. This keeps the language modeling objective and the expectation loss aligned at the score-token positions. This indexing step is part of the model interface rather than a preprocessing convenience. If the extracted logits correspond to the generated comma, quote mark, or next field name rather than the score digit, the EVA loss would optimize an unrelated distribution and could degrade both the language modeling objective and the reward estimate. The implementation therefore treats missing or ambiguous character-to-token mappings as invalid training examples for the auxiliary loss while retaining the standard language modeling loss where appropriate. 5Table 2: Dataset score dimensions. # 5 Experimental Setup 

5.1 Dataset Construction 

The training and evaluation datasets pair Lean 4 tactic states with informal reasoning proposals. The dataset includes both high-quality derivations and synthetically generated adversarial samples containing logical fallacies, hallucinated theorems, and syntax errors. Expert annotators provided CoT critiques and continuous ground truth scores ranging from 1.0 to 5.0 across the dimensions summarized in Table 2. For the textual generation target, the continuous scores are rounded to the nearest categorical integer. The original continuous values are preserved as RGT for the EVA loss computation. The dataset contains approximately 1,800 training records and an independent hold-out set of 400 test records. The hold-out split is constructed at the problem level where possible, so that near-duplicate proposals for the same tactic state do not appear in both training and evaluation. This split is intended to test whether the reward model learns state-sensitive evaluation criteria rather than memorizing a small set of theorem names or common proof fragments. Adversarial examples are included to make this distinction explicit: many negative samples are fluent and mathematically themed, but they fail because they do not discharge the current Lean goal. Continuous labels are used because the annotator often has enough information to distinguish borderline cases inside a five-point category. A proof idea that is locally relevant but missing a necessary side condition should not receive the same supervision as a fully correct derivation, even if both would round to the same integer for textual generation. EVA uses these fractional targets only through the auxiliary loss; the generated JSON remains categorical and easy to validate. 

5.2 Model Architecture and Training Details 

We initialize Leibniz using the Qwen2.5-1.5B-Instruct base model [Yang et al., 2024]. Fine-tuning is conducted using Low-Rank Adaptation (LoRA) with a rank of r = 16 and an alpha of 32, targeting all linear projection layers ( q_proj, v_proj, k_proj, o_proj, gate_proj, up_proj, down_proj ) [Hu et al., 2022]. The model is trained using bfloat16 precision, the AdamW optimizer with a learning rate of 2 × 10 −5, and a batch size of 8, accumulated over 3 epochs. The EVA loss balancing parameter is set to α = 0 .5.The training objective is applied to the same sequence used for ordinary supervised fine-tuning, so the auxiliary reward loss does not require a separate value head or a second model pass. The LoRA configuration is chosen to adapt both attention and feed-forward projections while keeping the number of trainable parameters modest. In the current implementation, the EVA loss is computed only at the score positions identified by the character-to-token mapping, which limits the auxiliary supervision to the fields where a continuous target is available.

# 6 Results and Discussion 

6.1 Reward Accuracy and Ranking Capabilities 

Table 4 presents the quantitative comparison of Leibniz against the selected baselines. The results demonstrate the trade-offs between different reward modeling strategies and validate the efficacy of the proposed EVA method. Zero-shot models exhibit broad mathematical knowledge but frequently miss Lean-specific tactic con-straints, resulting in higher Mean Squared Error (MSE) and lower ranking accuracy. The value-head reward model is highly competitive on scalar MSE, as its architecture directly optimizes continuous regression, but it lacks the textual critique necessary for diagnosing proof failures. Standard SFT gen-erative reward models preserve critiques but rely on discrete textual scores, introducing quantization artifacts that degrade continuous calibration. As hypothesized, Leibniz, utilizing EVA, successfully mitigates these artifacts, achieving the highest Pearson correlation ( r = 0 .824 ) and ranking accuracy (84.6%) while retaining the generative critique interface. A motivation for EVA is to obtain continuous reward values from a discrete generative model. In reinforcement learning, coarse reward steps can increase variance in advantage estimates. 7In contrast to a standard SFT generative RM that emits only integer scores such as 4.0 or 5.0, EVA can use the distribution over score tokens. For example, a generated JSON value of "logic_score": 4 may correspond to an extracted EVA score of 3.842 if probability mass is also assigned to adjacent anchors. This property provides a continuous reward signal that can be used by optimization algorithms such as PPO. The benefit is most direct near category boundaries. If the model is uncertain between anchors 3 and 4, a parsed integer score must choose one category, while the expected value can represent the intermediate belief. This does not make the reward differentiable with respect to the downstream policy’s discrete proof actions, but it does provide a smoother scalar target for reward-model training and a less quantized signal for policy optimization or search heuristics. 

6.3 Multi-dimensional Scoring 

We evaluate multidimensional scoring with adversarial examples. In one example, a Lean 4 tactic state required proving a basic natural number inequality ( n > 0 ⊢ n ∗ n > 0). The adversarial proposal applied the Pythagorean theorem to geometry hypotenuses to prove the integer algebraic statement. This example separates mathematical plausibility from tactic-state alignment. The algebraic goal is true, but the proposed geometric argument is not aligned with the Lean context. In the intended behavior, Leibniz should assign a moderate logic score while assigning a lower alignment score and explaining the contextual mismatch in its critique. The same separation is useful for ranking partially correct candidates. A candidate that follows the current tactic state but omits a side condition may be preferable to one that gives an elegant proof of an unrelated theorem, because the former can often be repaired by search or by adding a missing lemma. A single scalar score can encode this preference only implicitly. Separate Logic, Alignment, and Clarity scores make the failure mode visible and allow downstream systems to change the aggregation rule depending on whether they prioritize proof completion, data filtering, or human inspection. 

6.4 Deterministic Decoding 

In preliminary evaluations, higher sampling temperatures (e.g., T = 0 .7) sometimes changed the generated CoT and affected the subsequent JSON logits. Because the score tokens are conditioned on the preceding critique, decoding choices can affect the extracted reward. For formal reward modeling, we therefore use deterministic decoding with temperature T = 0 and greedy search when extracting EVA scores. Deterministic decoding also improves reproducibility of the reward extraction pipeline. Since the EVA score is read from logits at specific generated score positions, different rationales can shift token positions and change the context for the JSON fields. In applications where stochastic critiques are useful, one could average EVA scores across multiple sampled rationales, but this would increase inference cost and would require a separate analysis of variance across samples. 

6.5 Limitations 

The current study evaluates EVA as a reward-modeling component rather than as a complete theorem-proving system. The method also depends on reliable score-position identification and on score anchors that are single tokens for the chosen tokenizer. If a tokenizer represents the anchors with multiple tokens, the implementation must either choose a different anchor set or extend the expectation to multi-token score strings. Another limitation is that continuous expected scores are calibrated only to the annotation distribution. They do not provide a proof certificate and should not be interpreted as a substitute for Lean’s kernel. EVA is best viewed as a way to rank, filter, or train candidate proof steps before formal checking. Its effectiveness in closed-loop proof search will depend on how the reward is combined with exploration, tactic execution feedback, and theorem-library retrieval. 87 Conclusion and Future Work 

We introduced Expected Value Alignment (EVA), a training and inference procedure for extracting continuous scores from generative reward models. EVA computes the expectation over discrete score-token logits at identified sequence positions, reducing the dependence on parsed integer outputs while avoiding a separate value head. We applied this method to Lean 4 formal verification through Leibniz, a 1.5B-parameter reward model with generative critiques and continuous logit-based scores. Future work will evaluate EVA-trained models inside PPO loops for theorem proving and study larger or non-uniform anchor sets.
