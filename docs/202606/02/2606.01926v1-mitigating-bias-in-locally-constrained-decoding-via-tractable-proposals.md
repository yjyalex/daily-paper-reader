## 摘要
大型语言模型的生成往往难以符合预期的约束条件，如JSON模式。现有的局部约束解码方法通过短视地屏蔽下一个令牌来强制执行约束，这会导致采样偏差和性能下降。最近的工作使用顺序蒙特卡洛方法来缓解此类偏差，但设计有效的提案分布或势函数仍然是一个关键挑战。在这项工作中，我们提出了一种通用方法，为从$p_{\mathrm{lm}}( \cdot \mid \mathrm{constraint})$进行SMC采样构建提案和势函数。首先，我们展示了将有限自动机指定的约束向量化，以便在GPU上高效执行，我们利用这一点构建了全局约束解码提案。此外，利用张量化有限自动机与隐马尔可夫模型共享相同电路结构的事实，我们通过电路乘法获得了概率性全局约束解码提案，该提案编码了目标分布的逻辑和概率信息。我们在函数调用、基于关键词的生成和SQL生成任务上评估了(P-)GCD。实验表明，在相同的SMC采样设置下，与LCD提案相比，(P-)GCD以显著更少的粒子更快地收敛到目标分布。

## Abstract
Generations from large language models often fail to conform to desired constraints such as JSON schema. Existing locally constrained decoding (LCD) approaches enforce constraints by myopically masking out next tokens, resulting in biased sampling and degradation in performance. Recent work uses sequential Monte Carlo (SMC) methods to mitigate such biases, but designing effective proposal distributions or potential functions remains a key challenge. In this work, we propose a generic approach to construct proposals and potentials for SMC sampling from $p_{\mathrm{lm}}( \cdot \mid \mathrm{constraint})$. First, we show that constraints specified as finite automata can be tensorized for efficient execution on GPUs, which we use to construct globally constrained decoding (GCD) proposals. In addition, leveraging the fact that tensorized finite automata share the same circuit structure as hidden Markov models, we circuit-multiply them to obtain the probabilistic GCD (P-GCD) proposals encoding both logical and probabilistic information about the target distributions. We evaluate (P-)GCD on the tasks of function calling, keyword-based generation, and SQL generation. Experiments show that under the same SMC sampling setup, compared to LCD proposals, (P-)GCD converges faster to the target distribution with significantly fewer particles.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**: 大型语言模型（LLM）在生成时往往无法严格遵循硬约束（如 JSON schema、SQL 语法、关键词包含）。现有**局部约束解码（LCD）** 通过短视地屏蔽下一个 token 来强制约束，但这种贪婪方式导致两个缺陷：(1) 扭曲模型分布，引入采样偏差；(2) 无法保证在有限 token 预算内完成约束（例如 JSON 对象缺右括号）。
- **动机**: 为了解决 LCD 的偏差，已有工作采用序贯蒙特卡洛（SMC）方法通过重加权和重采样来纠正，但收敛速度依赖于提案分布的质量。LCD 作为提案分布质量差，需要大量粒子。
- **目标**: 构建更优的提案（proposal）和势函数（potential），使得 SMC 在更少粒子下快速收敛到目标条件分布 \(p_{\text{lm}}(x_{1:n} \mid C)\)。

## 2. 方法论：核心思想、技术细节、算法流程
- **整体思路**: 将约束表示为有限自动机（NFA/DFA），通过张量化（tensorization）实现在 GPU 上的高效推理，进而构建全局约束解码（GCD）。进一步利用自动机与隐马尔可夫模型（HMM）的电路同构性，通过电路乘法得到概率性 GCD（P-GCD），编码逻辑与概率信息。
- **核心技术**:
  1. **张量化有限自动机**：将 NFA/DFA 表示为三个二进制矩阵：出边矩阵 \(T_{\text{out}}\)、入边矩阵 \(T_{\text{in}}\)、标签矩阵 \(B_f\)，以及初始状态向量 \(\pi_f\) 和接受状态向量 \(f\)。
  2. **全局约束解码（GCD）**：定义提案 \(q_{\text{gcd}}(x_t \mid x_{<t}) \propto p_{\text{lm}}(x_t \mid x_{<t}) \cdot \mathbb{1}\{x_{1:t} \in X_{t,n}\}\)，其中 \(X_{t,n}\) 为可在剩下的 \(n-t\) 步内完成约束的前缀集合。通过前向-后向算法高效计算 mask：前向消息 \(\alpha_t\) 传递可达状态，后向消息 \(\beta_t\) 传递能否在剩余步内到达接受状态。
  3. **概率性全局约束解码（P-GCD）**：先蒸馏一个 HMM 来近似 LLM 的分布；利用 HMM 与自动机诱导的 HMM 共享电路结构，通过 Kronecker 积电路乘法得到乘积 HMM \(p_{\text{prod}} \propto p_{\text{hmm}} \cdot p_{\text{fa}}\)（当 FA 为 DFA 时，\(p_{\text{prod}} = p_{\text{hmm}}(\cdot \mid C)\)）。算法 1 给出如何利用前向-后向缓存高效计算下一 token 条件概率。
  4. **SMC 集成**：GCD / P-GCD 作为提案分布，P-GCD 也可作为势函数 \(\phi_{\text{pgcd}}(x_{1:t}) \propto p_{\text{lm}}(x_{1:t}) \cdot p_{\text{hmm}}(C \mid x_{1:t})\)，用于粒子重加权。
- **运算效率**：利用 Kronecker-vec 恒等式将每步复杂度从 \(O((HE)^2)\) 降至 \(O(H^2E + HE^2)\)。
