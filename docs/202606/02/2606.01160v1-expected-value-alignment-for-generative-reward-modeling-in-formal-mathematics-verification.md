## 摘要
大型语言模型（LLMs）越来越多地用于与形式化交互式定理证明器（如Lean 4）结合。通过强化学习或搜索方法扩展这些系统需要能够评估中间推理步骤的过程奖励模型（PRMs）。现有的奖励模型设计暴露了实际权衡。值头模型提供连续分数但修改了生成式模型接口，而生成式奖励模型保留了文本理由但难以匹配连续浮点回归，因为数值被分割到多个标记上。我们引入了期望值对齐（EVA），这是一种奖励建模过程，它在保持表面输出离散的同时，从模型的标记分布中提取连续分数。模型以结构化的JSON格式输出整数分数，EVA通过相应锚定标记的logits的期望值计算连续分数。训练结合了因果语言建模目标与这些期望值的辅助均方误差损失。我们在用于Lean 4形式验证的奖励模型\textit{Leibniz}中实例化EVA，并针对零样本和奖励建模基线进行评估。评估表明，基于logits的连续评分显著减少了离散化伪影，同时保留了生成式批评的可解释性。

## Abstract
Large Language Models (LLMs) are increasingly used with formal interactive theorem provers such as Lean 4. Scaling these systems with reinforcement learning or search methods requires process reward models (PRMs) that can evaluate intermediate reasoning steps. Existing reward-model designs expose a practical trade-off. Value-head models provide continuous scores but modify the generative model interface, while generative reward models preserve textual rationales but are poorly matched to continuous floating-point regression because numeric values are split across tokens. We introduce Expected Value Alignment (EVA), a reward-modeling procedure that keeps the surface output discrete while extracting continuous scores from the model's token distribution. The model emits integer scores in a structured JSON format, and EVA computes a continuous score as the expectation over the logits of the corresponding anchor tokens. Training combines the causal language modeling objective with an auxiliary mean squared error loss on these expected values. We instantiate EVA in \textit{Leibniz}, a reward model for Lean 4 formal verification, and evaluate it against zero-shot and reward-modeling baselines. The evaluation demonstrates that continuous logit-based scoring significantly reduces discretization artifacts while retaining the interpretability of generative critiques.

## 2. 方法论：核心思想、关键技术细节

### 2.1 整体框架：EVA（Expected Value Alignment）

- **“潜在连续，表面离散”范式**：模型仍生成离散的整数 JSON 分数（如 `{"logic_score": 4, "align_score": 2, "clarity_score": 3}`），但训练和推理时从模型在分数位置上的 token logits 分布中提取连续期望值作为实际奖励。
- **输入输出**：
  - 输入 X：系统指令 I + Lean 4 策略状态 S + 提议的证明文本 T。
  - 输出 Y：CoT 推理段落（`<think>...</think>`）+ JSON 对象（三个维度分数：Logic、Alignment、Clarity，各取 1-5 整数）。

### 2.2 数学公式

- 设锚定分数集合 A = {1,2,3,4,5}，对应 token ID 为 v_i。
- 在生成分数 token 的前一个位置 t，获取隐藏状态 h_t，经 LM head 得 logits z ∈ R^{|V|}。提取锚定 token 的 logits z_A = [z_{v1},...,z_{v5}]。
- 计算 softmax：P(i) = exp(z_{vi}/τ) / Σ_{j∈A} exp(z_{vj}/τ)。
- 期望分数 E[R] = Σ_{i=1}^5 i·P(i)。该连续值作为奖励信号。
- 温度 τ 训练时设为 1.0。

### 2.3 损失函数

- 联合损失：L_total = L_SFT + α·L_EVA
  - L_SFT：标准因果语言建模交叉熵损失（训练生成评论文本和 JSON）。
  - L_EVA：对 K 个维度（K=3）的期望分数与真实连续分数之间的 MSE：L_EVA = (1/K) Σ_{k=1}^K (E[R_k] - R_GT,k)^2。
  - α = 0.5 为平衡超参数。
- 训练时，通过字符到 token 的精确映射（char_to_token）定位每个分数 token 的位置，提取前一个位置的 logits。

### 2.4 算法流程简述

- 前向传播得到 logits → 计算 L_SFT。
- 对每个维度 k，定位分数 token 索引，提取锚定 logits，计算 softmax 得概率，再计算期望 E[R_k]。
- 累加 MSE 得 L_EVA，与 L_SFT 加权求和后反向传播。

## 6. 主要结论与发现

- **EVA 方法有效**：Leibniz（EVA）在 Pearson 相关系数（0.824）和 Ranking Accuracy（84.6%）上均优于所有基线，MSE 略高于 Value-Head RM（0.334 vs 0.312）但仍显著低于标准 SFT 生成式模型。
- **减少量化伪影**：通过期望分数，避免了整数截断，提供更平滑的奖励信号，有利于策略优化。
- **保持可解释性**：保留了生成式批评文本，可诊断失败模式（逻辑、对齐、清晰度）。
- **多维度评分有价值**：对抗样本示例显示，模型可区分数学逻辑正确但与当前状态不对齐的情况（高 Logic 低 Alignment）。
- **确定性解码推荐**：使用 temperature=0 的贪婪解码以保证奖励提取的稳定性和可重复性。

## 7. 优点

- **方法创新**：提出“潜在连续，表面离散”的设计，优雅地解决了生成式奖励模型连续分数与离散输出的矛盾，无需额外值头。
- **实现简洁**：基于单次前向传播即可同时计算语言建模损失和 EVA 损失，训练效率高。
- **可解释性强**：输出包含 CoT 批评和三维度分数，便于人类理解和下游调试。
- **适用于优化**：连续期望分数适合 PPO 等强化学习算法，避免整数奖励的方差问题。
