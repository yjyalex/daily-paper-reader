## 摘要
基于LLM的多智能体系统将复杂任务分解为交互角色，但大多数仍通过提示词、工具和控制规则手动编排，而智能体很少通过统一的强化学习接口进行优化。现有的RL后训练框架主要针对单策略优化，缺乏对用户定义的多智能体工作流、结构化交互、角色特定信用分配和可配置参数共享的抽象。我们提出UnityMAS-O，这是一个用于基于LLM的多智能体系统的通用RL优化框架。UnityMAS-O将完整工作流视为优化单位，而非单个响应或策略轨迹。它通过四个一级对象表示工作流：逻辑智能体角色、图轨迹、用户定义奖励和智能体-模型映射。这将逻辑智能体与物理模型参数解耦，支持完全共享、完全分离和部分共享，并在角色、轮次和轨迹层级分配奖励。UnityMAS-O基于Ray的星型拓扑运行时扩展了verl。中央控制器执行工作流、调用工具、记录结构化轨迹并组装奖励；模型本地工作线程组处理回滚、缓冲、优势计算和分布式PPO风格更新。用户可以定义智能体、工作流、模型映射和奖励，而无需重写优化基础设施。我们在检索增强问答、迭代智能体搜索和反思式代码生成上实例化了UnityMAS-O。在Natural Questions、HotpotQA和留出代码任务上，多智能体RL优化后改进了手动指定的工作流，特别是对于较小模型和严格代码全通过指标，增益尤其显著。这些结果表明，UnityMAS-O可以作为可复用基底，将多样化的基于LLM的多智能体工作流转化为可训练的多智能体RL系统。

## Abstract
LLM-based multi-agent systems decompose complex tasks into interacting roles, but most remain manually orchestrated by prompts, tools, and control rules, while agents are rarely optimized through a unified reinforcement learning interface. Existing RL post-training frameworks mainly target single-policy optimization and lack abstractions for user-defined multi-agent workflows, structured interaction, role-specific credit assignment, and configurable parameter sharing.   We present UnityMAS-O, a general RL optimization framework for LLM-based multi-agent systems. UnityMAS-O treats the complete workflow as the optimization unit, rather than a single response or policy trajectory. It represents workflows through four first-class objects: logical agent roles, graph trajectories, user-defined rewards, and agent--model mappings. This decouples logical agents from physical model parameters, supporting full sharing, full separation, and partial sharing, with rewards assigned at role, turn, and trajectory levels.   UnityMAS-O extends verl with a Ray-based star-topology runtime. A central controller executes workflows, invokes tools, records structured trajectories, and assembles rewards; model-local worker groups handle rollout, buffering, advantage computation, and distributed PPO-style updates. Users can define agents, workflows, model mappings, and rewards without rewriting the optimization infrastructure.   We instantiate UnityMAS-O on retrieval-augmented QA, iterative agentic search, and reflective code generation. Across Natural Questions, HotpotQA, and held-out code tasks, multi-agent RL improves manually specified workflows after optimization, with especially large gains for smaller models and strict code all-passed metrics. These results show that UnityMAS-O can serve as a reusable substrate for converting diverse LLM-based multi-agent workflows into trainable multi-agent RL systems.

## 1. 核心问题与整体含义（研究动机与背景）

当前基于LLM的多智能体系统（如CAMEL、AutoGen、ChatDev）通过角色分解（规划者、检索者、编码者等）解决复杂任务，但这些系统大多依赖**手动编排**（通过提示词、工具和路由规则定义工作流），智能体本身的参数很少通过统一的强化学习（RL）接口进行联合优化。与此同时，现代LLM后训练框架（如TRL、OpenRLHF、verl）主要针对**单策略优化**，缺乏对多智能体工作流所需的**结构化轨迹、角色信用分配、灵活的参数共享模式**等抽象支持。这一问题导致每个多智能体训练管线成为任务特定的实现，难以比较不同参数共享机制、跨角色分配信用或在各工作流间复用训练基础设施。论文因此提出**UnityMAS-O**，旨在提供一个**通用优化基底**，将用户定义的多智能体工作流转化为可训练的多智能体RL系统。

## 2. 方法论

### 核心思想
将**完整的多智能体工作流**（而非单个响应或单策略轨迹）作为优化单元。通过四个**一级对象**统一表示任意工作流：
- **逻辑智能体角色**（Logical Agent Roles）：工作流中的节点（如规划者、检索者、编码者等），每个角色有自己的提示模板、输入/输出接口、工具访问和停止条件。
- **图结构轨迹**（Graph-Structured Trajectories）：工作流执行产生的结构化轨迹，记录状态、动作、控制流依赖。
- **用户定义奖励函数**（User-Defined Rewards）：支持角色级、轮次级和轨迹级奖励，可组合局部和全局监督。
- **智能体-模型映射（Agent–Model Mapping）**：将逻辑角色映射到物理LLM实例，支持完全参数共享、完全分离和部分共享。

### 关键技术细节
- **解耦逻辑与物理**：同一工作流可通过改变映射φ实现不同参数共享体制，无需重写训练管线。
- **分布式运行时架构**：基于Ray的星型拓扑扩展verl。中央控制器执行工作流、记录轨迹、调用工具/环境、组装奖励；**模型本地工作线程组**负责推理、回滚、缓冲、优势计算和PPO风格策略更新。工作流控制数据（轻量）与模型训练数据（张量）分离，减少通信开销。
- **训练循环**：异步回滚 → 奖励组装与提交 → 批量构建 → 优势计算（GAE） → 每模型PPO更新。不同模型可拥有不同的学习率、优化器、精度模式。
- **奖励接口设计**：每个角色可定义自己的奖励函数 R_v(s, y_v, s', τ)，涵盖节点级（如格式惩罚）、轮次级（如中间上下文评估）和轨迹级（如最终答案F1）。支持加权组合或自定义规则。

### 算法流程（文字说明）
1. 用户定义逻辑角色、工作流图、智能体-模型映射和奖励函数。
2. 中央控制器在任务实例上异步执行工作流，将每个角色调用路由至映射的模型工作线程组。
3. 工作线程组生成输出（轻量动作返回控制），保留张量级回滚数据在本地缓冲区。
4. 控制器在角色调用、子工作流或完整轨迹后评估奖励，并将奖励提交至对应模型的缓冲区。
5. 当缓冲区积累足够数据，构建每模型独享的训练批次，计算旧对数概率、参考策略对数概率、值估计、回报和GAE优势。
6. 每模型执行PPO策略更新，同步权重到回滚后端（如vLLM）。

## 3. 实验设计

### 数据集/场景
- **检索增强QA**：使用**Natural Questions**（NQ，单跳开放域）和**HotpotQA**（多跳证据聚合）。
- **代码生成**：使用**DeepCoder**数据（约24K唯一编程问题-测试对，来自TACO-Verified、PrimeIntellect SYNTHETIC-1、LiveCodeBench v5）。

### 工作流模板
- **QA**：三种搜索工作流——
  - A：并行检索（分解→检索→答案）
  - B：检索-提取-答案（增加证据提取节点）
  - C：迭代搜索循环（M-ASK，含规划、搜索、总结、更新、答案节点，使用轮次级奖励）
- **代码**：D：反思验证循环（三轮计划→编码→验证→反思，基于验证器分数增量奖励）

### 对比方法与评估
- **对比基准**：每个工作流在**多智能体RL训练前**的初始性能 vs **训练后最佳性能**。
- **评估指标**：QA用标准化答案F1；代码用留出测试集**全通过率**（all-passed rate）。
- **模型规模**：Qwen3系列（0.5B、1.5B、3B、7B、14B用于QA；4B、8B用于代码）。
- **参数共享消融**：在M-ASK工作流上比较3B共享参数 vs 4x3B独立模型。
- **其他指标**：代码工作流还报告**平均验证轮次**（efficiency）。

### 主要结果
- QA：所有工作流和模型规模训练后F1均提升，**小模型增益巨大**（如0.5B在NQ上从0.022→0.445）。HotpotQA上迭代搜索（M-ASK）因能分配轮次信用而表现更优。
- 代码：3xQwen3-4B全通过率从0.255→0.686；3xQwen3-8B从0.290→0.738。同时平均验证轮次减少约40%（从2.5降至1.68-1.76）。
- 参数共享：3B共享参数在1800步内达到与4x3B独立相近的最终F1（0.520 vs 0.529），但收敛稍慢。

## 6. 主要结论与发现

1. UnityMAS-O能**有效优化用户指定的多智能体工作流**，在QA和代码任务上普遍提升性能。
2. **小模型受益更大**：0.5B模型从近乎失效提升到可用水平，表明训练可补偿初始能力的不足。
3. **参数共享是可行的**：多个逻辑角色共享同一物理模型仍能达到接近独立模型的最终性能，支持资源-性能权衡。

5. UnityMAS-O提供**可复用基础设施**，将手动编排工作流转化为可训练的多智能体RL系统，并支持灵活的研究（信用分配、共享体制、分布式执行）。

## 7. 优点
