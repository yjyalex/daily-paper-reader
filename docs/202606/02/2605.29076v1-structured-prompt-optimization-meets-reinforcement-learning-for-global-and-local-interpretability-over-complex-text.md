## 摘要
LLMs在文本分类上取得了进展，然而现有范式面临权衡：监督式（仅标签）微调具有可扩展性，但在复杂文本上的推理有限，且缺乏模型整体透明度；而离散提示优化提供人类可读的指令，但在性能和可扩展性上存在困难。我们提出了eXTC（可解释文本分类器），包含三个逐步推进的阶段：（1）通过新的结构化提示优化算法学习自然语言形式的标准操作流程（SOP，即规则手册）；（2）基于SOP的推理蒸馏，将大型教师LLM的知识蒸馏到紧凑语言模型中；（3）通过强化学习扩展超出初始SOP的推理能力。这种设计使eXTC能够：（i）通过紧凑语言模型实现快速推理，（ii）提供推理时的局部推理轨迹，以及对其学习到的领域规则进行全局模块化解释，同时（iii）在多个基准测试中，在分类性能和解释质量两方面显著优于现有范式，且性能逐阶段提升。

## Abstract
LLMs have advanced text classification, yet existing paradigms face a trade-off: supervised (label only) fine-tuning is scalable but offers limited reasoning on complex text and lacks broader model transparency, while discrete prompt optimization offers human-readable instructions but struggles with performance and scalability. We introduce eXTC (eXplainable Text Classifier) with three progressive stages: (1) learning a Standard Operating Procedure (SOP, or rulebook) in natural language via a new Structured Prompt Optimization algorithm; (2) SOP-grounded reasoning distillation from a large teacher LLM into a compact LM; and (3) expanding reasoning capabilities beyond the initial SOP via reinforcement learning. This design enables eXTC to provide (i) fast inference via a compact LM, with (ii) inference-time local reasoning traces, alongside a global, modular explanation of its learned domain rules, while (iii) significantly outperforming existing paradigms across diverse benchmarks in both classification performance and explanation quality, with stage-by-stage gains.

## 1. 论文的核心问题与整体含义（研究动机和背景）

**核心问题**：现有大型语言模型（LLM）在文本分类中存在一个根本性权衡：
- **监督微调（SFT/PEFT）**：可扩展、性能好，但无法提供推理轨迹或全局解释，缺乏模型透明度。
- **离散提示优化（PO）**：可生成人类可读的指令（如规则集），可作为全局解释；但性能通常不如微调，且计算成本高，难以大规模应用。
- **零样本思维链（Zero-shot CoT）**：虽然能生成推理，但在复杂文本上效果差，容易依赖表面线索导致错误分类。

**整体含义**：论文旨在桥接上述 gap，提出一个 **同时具备高性能、高效推理、局部解释（推理轨迹）和全局解释（结构化规则手册）** 的 LLM 文本分类框架。这一设计对高可靠性应用（如医疗、法律、安全审核）尤为重要，这些场景需要模型不仅预测正确，还要可审计、可理解。

## 2. 论文提出的方法论：核心思想、关键技术细节

论文提出 **eXTC（eXplainable Text Classifier）**，是一个三阶段渐进式框架：

### 阶段 I：通过结构化提示优化学习标准操作流程（SOP Learning via SPO）
- **目标**：学习一个自然语言规则的**决策集**（decision set），称为 SOP（标准操作流程），作为**全局解释**。
- **算法**：SPO4SOP（Structured Prompt Optimization for SOP）
  - 基于**口头化机器学习（Verbalized ML）** 和**文本梯度**。
  - 使用三个 LLM 角色：**分类器 LLM**（对单个规则判断是否触发）、**梯度 LLM**（分析错误原因，生成异常/错误模式）、**更新 LLM**（根据梯度修改现有规则或合成新规则）。
  - 优化两种错误模式：**错误覆盖**（规则错误触发）、**盲点**（没有规则触发且默认标签错误）。
  - 规则子集选择：通过**束搜索**从候选池中选出最大化验证集宏观 F1（带稀疏惩罚）的少量规则，且利用**决策可组合性**（每个规则独立预测，可高效缓存重组），LLM 调用次数仅与规则数量线性相关。
- **输出**：一个由自然语言规则构成的、可单独触发、可组合的决策集（SOP），作为全局解释。

### 阶段 II：SOP 引导的推理蒸馏（SOP-Grounded Distillation）
- **教师 LLM**：将阶段 I 学到的 SOP 与每个训练示例拼接，生成**基于 SOP 的推理轨迹**和标签。
- **拒绝采样**：每个示例最多生成 4 条轨迹，只保留预测标签与真实标签一致的（称为“容易”样本），其余为“困难”样本（供后续 RL 使用）。
- **推理监督微调（R-SFT）**：用 LoRA 对小型学生 LLM（Qwen3-4B）进行微调，输入为原始文本，输出为推理轨迹+标签。学生模型不再需要 SOP 作为上下文，知识已内化到参数中，从而实现高效推理，同时产生局部解释（推理轨迹）。

### 阶段 III：超越 SOP 的强化学习（Beyond SOP via RL）
- **动机**：阶段 I 的 SOP 基于少量数据学习，覆盖有限；阶段 II 只蒸馏了教师正确的“容易”样本，大量“困难”样本未被解决。
- **挑战**：标准 GRPO 在分类任务中面临**展开同质性**（rollout homogeneity）问题：当同一输入的所有展开预测相同标签时，组内标准差为零，优势为零，无梯度信号。
- **提出的方法**：BD-GRPO（Balanced Dynamic GRPO）
  - **类别平衡上采样**：按类别分别上采样候选样本。
  - **动态过滤**：对每个候选样本运行 G 个展开，只保留展开间标准差不等于零的组（即存在分歧的组），构成有效批次。
  - 结合两者：先按类别上采样，再动态过滤，实现**自引导课程学习**：自动专注于当前“信息带”（hard 样本为主），加速收敛。
  - **对比验证**：单独使用类别平衡（B-GRPO）或单独动态过滤（D-GRPO）效果均不如 BD-GRPO；D-GRPO 在类别不平衡严重时崩溃。
- **输出**：进一步提升后的学生模型，推理时仍产生局部解释；RL 学到的新模式还可反馈回 SOP 进行规则更新（附录 B.2.2）。

**整体流程**：SOP（全局解释）→ 教师推理 → 蒸馏至学生 → RL 提升硬样本 → 学生模型兼具高效推理、局部解释，且性能超越各基线。

2. **SPO4SOP** 学习到的结构化规则集本身可作为全局解释，且其推理形成清晰簇，优于非结构化提示优化。
3. **R-SFT** 显著提升了解释质量（相比仅标签 SFT），且保持了良好分类性能。
4. **BD-GRPO** 通过类别平衡+动态过滤解决 rollout同质性问题，实现更快的收敛（3-5×）和更高的峰值性能，且保持训练健康（熵不崩溃、KL 小、硬样本准确率提升、易样本准确率保持）。
5. 案例显示，eXTC 能正确应用规则及其例外，克服 Zero-shot CoT 的表面触发错误。
6. RL 学到的知识可以反向更新 SOP（附录 B.2.2），进一步提升整体覆盖。

## 7. 优点

- **方法创新**：首个将结构化提示优化、推理蒸馏和强化学习统一到一个框架中的工作，同时实现高性能、局部解释和全局解释。
- **工程亮点**：利用决策可组合性使规则集搜索的 LLM 调用量从指数级降至线性级；BD-GRPO 通过自引导课程高效训练，解决 GRPO 在分类中的独特困境。
- **可解释性双赢**：全局规则（SOP）+ 局部推理轨迹（学生模型生成），无需事后归因，自然可审计。
- **实用性**：学生模型仅 4B 参数，推理高效，且不依赖 SOP 上下文；教师和优化过程可离线完成。
- **实验充分**：多数据集、多基线、多消融、定性+定量分析，证据较强。

## 8. 不足与局限

- **依赖闭源 LLM**：阶段 I 和 II 依赖 gpt-4.1-mini，阶段 III 依赖 gpt-4o-mini（辅助奖励），复现需相应 API 访问权限和费用；未对开源 LLM 进行验证。
- **仅限纯文本**：未考虑多模态场景（如图表、表格），但论文承认这是局限。
- **解释质量评估**：虽然使用了 NLI 和 LLM 法官，但这些自动方法存在局限性（例如同模型偏好），且 ground-truth 证据仅基于标注（可能不完美）。
- **未见与更复杂的结构化解释方法对比**：如 LSP（ICLR'25）和 MAC（arXiv'26-03），仅在附录表格中说明但未直接实验对比。
- **RL 训练的额外负担**：BD-GRPO 需要上采样和多次展开，虽然收敛快但单步成本高于传统 GRPO；辅助奖励进一步增加成本。
- **标签不平衡处理**：仅在 RL 中通过类别平衡缓解，但未探讨对极度不平衡（如 90% vs 10%）的鲁棒性（即使 MIMIC 已有 22% 少数类）。
- **未讨论部署中的推理稳定性**：学生模型可能在某些未见样本上产生不稳定推理，且无 SOP 引导时 faithfulness 和一致性未充分评估。
