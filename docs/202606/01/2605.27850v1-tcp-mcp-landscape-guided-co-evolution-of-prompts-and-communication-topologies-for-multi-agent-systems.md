## 摘要
有效的多智能体系统无法通过孤立地选择提示或通信图来设计。智能体行为取决于智能体接收的信息，而通信边的有用性则取决于接收智能体如何解释和使用该信息。我们提出	extbf{TCP-MCP}（面向多智能体协作问题求解的拓扑耦合提示），这是一个将智能体提示和通信拓扑作为统一基因组进行搜索的协同进化框架。TCP-MCP使用初始化时的景观探测器来校准早期搜索行为，然后依赖帕累托前沿诊断在三个目标（任务性能、令牌成本和结构复杂度）下调整探索。在使用相同的DeepSeek-V3.2骨干网络的所有方法中，TCP-MCP在MMLU-Pro、MMLU和GSM8K上分别达到82.66%、89.96%和96.61%的准确率。在三个基准测试中，它始终优于自动图生成基线，并与辩论式系统的准确率相当，同时在报告的操作点上使用的令牌数比这些系统少多达5.69倍。这些结果表明，在受控评估中，联合进化提示和通信结构为成本感知和任务自适应的多智能体系统设计提供了一条实用途径。

## Abstract
Effective multi-agent systems cannot be designed by selecting prompts or communication graphs in isolation. Agent behavior depends on the information an agent receives, while the usefulness of a communication edge depends on how the receiving agent interprets and uses that information. We propose \textbf{TCP-MCP} (Topology-Coupled Prompting for Multi-Agent Collaborative Problem-Solving), a co-evolution framework that searches agent prompts and communication topologies as a unified genome. TCP-MCP uses an initialization-time landscape probe to calibrate early search behavior, and then relies on Pareto-front diagnostics to adapt exploration under three objectives: task performance, token cost, and structural complexity. Using the same DeepSeek-V3.2 backbone across all methods, TCP-MCP achieves 82.66\%, 89.96\%, and 96.61\% accuracy on MMLU-Pro, MMLU, and GSM8K, respectively. Across the three benchmarks, it consistently outperforms automated graph-generation baselines and achieves competitive accuracy relative to debate-style systems, while using up to 5.69$\times$ fewer tokens than those systems at the reported operating points. These results show that jointly evolving prompts and communication structure provides a practical route to cost-aware and task-adaptive multi-agent system design in controlled evaluations.

## 1. 核心问题与整体含义（研究动机与背景）
- **核心问题**：大规模语言模型（LLM）驱动的多智能体系统（MAS）的性能不仅依赖于各智能体的提示（prompt），还高度依赖于智能体之间的通信拓扑结构。然而，现有设计流程将提示选择与拓扑设计割裂：要么固定拓扑只优化提示，要么单独搜索拓扑而忽略提示的适配。这种分离忽略了**提示行为与通信拓扑相互耦合**的关键性质——一个在某种通信模式下有效的提示在另一种拓扑下可能失效；一条通信边的价值取决于接收端智能体是否被指示正确解读和使用传递的信息。
- **整体含义**：MAS设计应被视作一个系统级联合优化问题，需将智能体提示与通信拓扑同时优化，才能实现性能、成本与复杂度的有效权衡。本文通过提出TCP-MCP框架证实了这一观点，并展示了在多任务基准上优于传统分离式方法的结果。

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- 将MAS设计编码为统一基因组 **M = (G, P)**，其中G为有向通信图，P为节点级提示模板。提示模板本身不嵌入拓扑信息，而是在运行时通过函数 `BuildPrompt(pi, N^- (vi; G))` 注入拓扑上下文（接收邻居信息），从而实现“同一模板在不同拓扑下产生不同行为”的耦合效应。
- 采用**协同进化**策略，同时进化拓扑结构、角色分配和提示文本，而非分阶段优化。
- 引入**初始化时的景观探测**（Landscape Probe）校准早期搜索偏差，并在进化过程中基于**帕累托前沿诊断**（Pareto-front diagnostics）自适应调节探索强度，兼顾任务准确率、令牌成本和结构复杂度三个目标。

### 关键技术细节
1. **统一基因组与多目标优化**：
   - 搜索空间为混合空间 **S = G × Θ**（拓扑 × 提示配置），目标向量定义为 F(M) = (A(M), -C(M), -ln(1+K(G)))，其中 A 为准确率，C 为令牌成本，K = |V|+|E| 为图复杂度（取对数惩罚避免大图支配）。
2. **初始化与景观探测**：
   - 使用拉丁超立方采样（LHS）生成初始种群，覆盖图大小、边密度、拓扑模式、角色组合、提示风格等维度。
   - 计算标量偏好得分 S(M) = A(M)^k / ((C(M)/T0)^γ · K̄(G)^β)，并计算适应度-距离相关性（FDC）作为粗粒度景观信号，仅用于初始化阶段。
3. **进化操作**：
   - **结构交叉（Structural Crossover）**：使用提示最小继承（PMI）策略——交叉转移拓扑子图而非提示片段。保留父本节点保持原提示，移植节点继承供体提示（若无则使用默认轻量提示）。通过角色名精确匹配或边界度启发式（b(v)=in_degree+out_degree）确定交叉锚点。
   - **突变操作**：包括拓扑突变（增删边/节点，拒绝环）、角色突变（更换角色并重新生成提示）、激进突变（在停滞时引入整体拓扑模式如链、树等，概率由自适应控制信号决定）。
   - **环境选择**：基于带约束的NSGA-II，加入动态准确率阈值（best_acc(t)-δ）过滤低质候选，结合分箱小生境（bin-based niching）保持成本-结构空间多样性，设置结构配额避免过早坍缩到简单低节点解。
4. **自适应控制**：
   - 进化期间不重复计算FDC，而是监测跨代帕累托诊断（归一化超体积HV、间距、前沿间隙、区域覆盖），综合成响应指数 s = wHV·sHV + wacc·sacc + wtok·stok + wdiv·sdiv，并通过sigmoid映射为激进突变概率 p_stag = σ(αs+β)，动态调整探索强度。

## 5. 主要结论与发现
1. **TCP-MCP显著优于自动图生成基线**：在MMLU上89.96%（G-Designer 87.90%），MMLU-Pro上82.66%（79.32%），GSM8K上96.61%（95.81%）。
2. **与高成本辩论式系统相比，实现有利的准确率-成本权衡**：对比LLM-Debate，在GSM8K上准确率更高（+1.13pp）且令牌成本低4.09倍；在MMLU上准确率略低（-0.88pp）但令牌成本低4.59倍；在MMLU-Pro上准确率略低（-1.01pp）但令牌成本低5.69倍。
3. **进化后的结构具有可迁移性**：跨数据集转移仅导致轻微准确率下降（1-1.5pp），跨骨干模型转移也取得最佳GPT-4准确率，表明搜索到的通信模式捕获了可复用的协作偏向。
4. **协同进化优于分阶段优化**：消融实验AB4（先提示后拓扑）在所有数据集上均不如完整TCP-MCP，证实了提示与拓扑的耦合性必须同时处理。
5. **结构交叉和突变均不可或缺**：缺少交叉导致MMLU-Pro准确率下降2.51pp；缺少突变导致MMLU准确率下降2.59pp且令牌成本上升46%；自适应控制主要起稳定搜索作用。

## 7. 不足与局限
- **实验统计严谨性不足**：未进行多种子（multi-seed）进化运行以量化随机性；仅做一次二次推理作为稳定检查，未提供均值和方差。作者在附录中明确说明“these intervals estimate uncertainty due to finite test-set sampling only. - **搜索成本较高**：虽然最终评估成本合理，但搜索阶段需近10亿令牌和37万次API调用，对于资源有限的研究者可能负担较重。作者提及可复用已搜索模式以摊销成本。

- **任务范围有限**：仅评估了静态问答/数学推理任务，未涉及工具使用、长期交互、动态环境、生成式任务（如代码、创意写作）。拓扑为静态选择而非在线自适应。
- **开发子集选择固定**：开发子集（80/100/40题）较小且固定，可能引入选择偏差。未进行不同开发集划分的敏感性实验。

- **帕累托点选择规则为人工设定**：虽然固定且基于开发集，但阈值（如ϵ_acc=1.5pp）的合理性和泛化性未充分论证。
