## 摘要
为确保大语言模型的输出符合特定结构，上下文无关文法（CFG）解码引擎会强制选择能生成符合给定CFG字符串的下一个词元。尽管当前CFG受限解码引擎已高度优化，但由于每一步搜索空间（即整个词元词汇表）巨大而产生的固有成本，导致在更复杂的CFG上开销极高——而这正是CFG引擎最有用的情况。本文提出CFGzip，一种离线压缩词元搜索空间的技术，显著降低了CFG引擎的开销。实验表明，将CFGzip与当前最先进的文法引擎结合使用时，延迟最多可降低两个数量级，总受限生成时间加速比最高达7.5倍：借助CFGzip，复杂CFG下的受限解码现可大规模高效实现。

## Abstract
To guarantee that an LLM's outputs conform to a specified structure, context-free grammar (CFG) decoding engines force the selection of next tokens that produce strings that conform to a given CFG. While current CFG-constrained decoding engines are highly optimized, the inherent costs arising from the massive per-step search space -- i.e. the entire token vocabulary -- result in intractably high overhead for more complex CFGs: precisely the situation where CFG engines are most useful. In this paper, we introduce CFGzip, an offline technique for compressing the token search space, which massively reduces CFG engine overhead. In experiments, we report latency reduction of up to two orders of magnitude when CFGzip is used with a SoTA grammar engine, yielding an up to 7.5x speedup in total constrained generation time: with CFGzip, constrained decoding is now feasible at scale for complex CFGs.

# 论文详细中文总结

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：LLM 在结构化生成（如代码、JSON、XML）中常需要约束解码以确保输出符合指定上下文无关文法（CFG）。现有 CFG 解码引擎（如 XGrammar2）虽已高度优化，但在处理复杂 CFG（如 C++、Bython）时，每步需要检查整个词汇表（通常 10万–20万词元），导致开销高达 2–10 倍，使得复杂 CFG 的受限解码在实践中不可接受。
- **意义**：解决这一开销瓶颈，可使 CFG 约束解码在大规模、复杂文法场景下变得可行，从而提升 LLM 输出的语法正确性和功能正确性，尤其对低资源语言（如 Bython）效果显著。

## 2. 方法论
- **核心思想**：许多词元相对于给定 CFG 是等价的（语法同余），只需检查每个等价类的一个代表词元，即可判断整个类所有词元的合法性。通过离线预计算等价类，在线推理时文法引擎在压缩后的词元空间上操作，实现无损、零额外开销的加速。
- **关键技术细节**：
  - **等价关系定义**：采用“位移等价”（Displacement Equivalence）近似语法同余。词元 t 的位移 Δ(t) 定义为所有输入栈-输出栈对，使得在该 PDA 下用 t 消耗输入栈得到输出栈。若两个词元位移相同，则它们语法同余（定理2）。
  - **位移计算算法（Algorithm 1）**：将 CFG 转换为 Greibach 范式，得到简单 PDA。对每个词元 t，非确定性模拟 PDA 解析过程，记录输入栈和输出栈。当栈为空但 t 未解析完时，进行栈回溯，并用预计算的栈邻接关系（Stack Adjacency）限制搜索空间。
  - **预计算**：对每个 tokenizer/CFG 对，计算所有词元的等价类，缓存映射向量（词元 ID → 类 ID，类 ID → 代表词元）。代表词元选择字节最短的词元。
  - **在线推理**：
    - 初始化文法引擎，词汇表为等价类代表集。
    - 采样词元后，将其映射到代表词元传给引擎更新状态。
    - LLM 输出 logits 后，通过 gather 操作将 logits 按等价类索引乘以引擎生成的 mask，实现并行化，几乎无额外时间。
- **理论保证**：压缩是无损的（证明与语法同余等价），且引擎无关。

## 6. 论文的主要结论与发现
- **延迟大幅降低**：CFG ZIP 将 XGrammar2 的开销减少 10–20 倍（每例/每词元），总约束生成时间加速比最高 7.5 倍（Bython 任务）。
- **复杂 CFG 下接近零开销**：在 C++ 上，+CFG ZIP 的总时间接近未约束模型；Bython 上，从 12.5–19.5 秒降至 1.76–3.52 秒。
- **正确性显著提升**：约束解码对低资源语言（Bython）提升极为明显，语法正确性从 ~2%–13% 提升至 43%–72%，功能正确性从 ~1%–11% 提升至 9%–47%。对常见语言（JSON、C++）提升有限但仍有收益。
- **LLM 越小，约束解码收益越大**：Llama-3B 在约束下超越未约束的大模型在某些任务上的表现，表明小模型+约束是低成本替代方案。

## 7. 优点
- **高效**：离线预计算，在线推理无额外开销（仅一次 gather 操作）。
- **无损**：理论保证输出与原始引擎字节级一致。
- **引擎无关**：理论上可与任何 CFG 解码引擎结合使用（论文验证了 XGrammar2）。
- **压缩率高**：词汇表压缩比 40:1 到 800:1（如 GPT-20B/JSON）。
- **实用性强**：预计算时间可接受（几分钟），而推理节省的时间可达数小时。
- **代码开源**：提供 pip 包和实验代码，便于复现。

## 8. 不足与局限
- **离线预计算耗时**：对动态/临时 CFG（如 JSON Schema 突变）不适用，但这类简单 CFG 原始引擎已足够快。
- **引擎兼容性受限**：实际只评估了 XGrammar2，其他引擎（Outlines、llguidance、transformers-cfg 等）因不支持复杂 CFG 或左递归、或缺乏 Python 绑定而无法使用。论文未在 llama.cpp 上测试（C++ 实现）。
- **Bython 任务超时问题**：XGrammar2 下约 5% 的实例因陷入死循环超时（15 分钟），而 CFG ZIP 仅 0.2% 超时。论文将其视为异常值排除，但可能低估了 XGrammar2 的实际开销。
- **未与其他加速技术结合**：未消融是否可与 JIT 编译、缓存等进一步叠加。
- **实验覆盖有限**：仅使用了 HumanEval-X 和 structEval 的子集，未在更多领域（如 SQL、化学结构）验证。
- **理论近似**：位移等价是语法同余的可计算近似，虽然无损，但可能因栈回溯限制过于严格，极端复杂 CFG 下压缩率可能下降。论文未讨论该风险。
