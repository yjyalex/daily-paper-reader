## 摘要
强化学习（RL）天然适用于基于智能体的知识库问答（KBQA），其中模型必须执行可执行动作、观察知识库反馈并最终返回答案。然而，当前基于RL的KBQA系统主要优化来自最终答案的稀疏奖励，导致中间动作错误缺乏监督。这尤其限制了带有逻辑形式标注的KBQA基准：金逻辑形式可以转换为可执行动作序列，但现有管道主要将其用于热启动数据构建，而非用于在线策略RL更新。我们提出GAPD，一种训练时的金动作策略蒸馏框架，为基于结果的RL添加密集的令牌级指导。为了将金动作与在线策略的学生展开对齐，GAPD使用中间锚点匹配：它将学生探索和金牌执行期间达到的中间实体视为状态锚点，并通过这些探索到的实体集将学生状态与金牌状态匹配。当前策略在该对齐后的金动作上的条件分布作为停止梯度的教师，其令牌分布被蒸馏回生成的行动令牌跨度上的普通学生策略。GAPD在WebQSP、GrailQA和GraphQ上持续超越当前最先进水平。

## Abstract
Reinforcement learning (RL) is a natural fit for agentic knowledge base question answering (KBQA), where a model must issue executable actions, observe knowledge-base feedback, and eventually return an answer. However, current RL-based KBQA systems mainly optimize sparse rewards from the final answer, leaving intermediate action errors weakly supervised. This is especially limiting for logical-form annotated KBQA benchmarks: gold logical forms can be converted into executable action sequences, but existing pipelines use them mainly for warm-start data construction rather than for on-policy RL updates. We propose GAPD, a training-time Gold-Action Policy Distillation framework that adds dense token-level guidance to outcome-based RL. To align gold actions with on-policy student rollouts, GAPD uses MID-ANCHOR MATCHING: it treats the intermediate entities reached during student exploration and gold execution as state anchors, and matches student states to gold states through these explored entity sets. The current policy conditioned on this aligned gold action serves as a stop-gradient teacher, whose token distribution is distilled back to the ordinary student policy over generated action-token spans. GAPD consistently surpasses the current state of the art on WebQSP, GrailQA, and GraphQ.

### 1. 核心问题与整体含义（研究动机与背景）
- **研究问题**：在基于强化学习（RL）的智能体知识库问答（KBQA）中，模型需要与环境交互执行动作。现有方法（如KBQA-R1、KnowCoder-A1）主要使用基于最终答案的稀疏奖励（如答案F1和格式正确性）来训练策略，导致**中间动作的监督信号极其薄弱**——一条轨迹中的动作承袭相同的分组相对优势，无法区分哪个中间动作导致失败。
- **现有不足**：逻辑形式标注的KBQA基准（如WebQSP、GrailQA、GraphQ）提供了金标逻辑形式（gold logical forms），可转换为可执行动作序列。但现有pipeline仅在热启动数据构建、监督微调或离线轨迹合成中使用这些金标动作，而在**在线策略RL更新阶段完全丢弃了它们**，浪费了宝贵的细粒度监督信息。
- **论文目标**：提出一种训练时框架，在保持RL探索特性的同时，利用金标动作序列为中间步骤提供密集的token级指导，从而改善RL训练的信用分配问题。

### 2. 方法论：核心思想、关键技术细节
- **核心思想**：GAPD（Gold-Action Policy Distillation）通过在RL训练中融合**最终结果奖励**（通过GRPO）和**基于金标的密集token级指导**。具体而言，当学生策略到达某个中间状态时，先通过Entity-Anchor Matching将学生状态与金标执行状态对齐，然后基于当前策略在对齐后的金动作条件下的分布构建“自教师”（self-teacher），将该教师的token级偏好蒸馏回学生策略。
- **关键组件**：
  1. **实体-锚点匹配（Entity-Anchor Matching）**：将学生探索和金牌执行过程中达到的**中间实体集合**视为状态锚点，通过Jaccard相似度匹配学生状态与金标状态。只对匹配成功的状态（相似度≥0.95）施加蒸馏，避免状态偏离时错误引导。同时施加两个保守过滤器：跳过答案动作和相同下一状态过滤。
  2. **金动作条件自教师（Gold-Action-Conditioned Self-Teacher）**：在学生上下文基础上，插入对齐后的下一个金动作（作为教师侧上下文），当前策略在该上下文下的分布作为教师分布（stop-gradient）。教师不依赖外部大模型，而是复用当前策略，避免额外开销。
  3. **Token级优势融合**：对匹配到的学生生成token，计算教师对数概率减去学生对数概率，得到裁剪后的GAPD优势（Agapd）。然后将该优势与GRPO的结果级优势（Agrpo）相加，形成融合优势（A = Agrpo + λgapd * Agapd），用于PPO风格的裁剪损失更新。
- **训练流程**（Algorithm 1）：
  1. 学生策略产生一条轨迹，同时执行金标动作序列收集实体锚点。
  2. 对每个学生回合，使用Entity-Anchor Matching匹配金标状态，若匹配则构建教师侧上下文，标记该回合的生成token位置。
  3. 计算GRPO优势（基于最终答案奖励和格式奖励）。
  4. 计算匹配token上的GAPD优势。
  5. 使用融合优势更新策略参数。

### 6. 主要结论与发现
- GAPD在三个基准上**一致超越当前最先进方法**（包括KBQA-R1和所有商业模型Harness版本）：

  - GraphQ：F1从53.8提升到55.1（相对+2.4%）。
- 消融表明：Entity-Anchor Matching是关键（不使用它时性能急剧下降）；第一回合监督也有助于但不如完全状态匹配；使用外部教师模型（Qwen3）不如自教师有效，且训练开销更大。
- GAPD不仅提高了准确率，还**减少了平均交互次数**（从2.91降到2.68），说明策略更加精确高效。
- 训练过程更稳定，奖励曲线更高（Figure 2a）。
- 系数λgapd在0.1~1.0范围内表现稳定，最佳在0.5附近。

### 7. 优点
1. **方法创新**：首次在在线策略RL中利用金标动作提供密集token级指导，解决了结果奖励稀疏导致的中级动作信用分配问题。
2. **自教师设计**：不使用额外教师模型，而是复用当前策略在金动作条件化下的分布，降低了内存和计算开销。
3. **状态对齐**：Entity-Anchor Matching基于可达的实体集合进行匹配，处理了学生探索轨迹与金标轨迹之间的状态偏移问题，避免错误引导。
4. **保守滤波**：通过相同下一状态过滤保留探索性，不强制所有成功轨迹都模仿金标动作。
5. **实验充分公平**：在多个基准上与强基线（包括商业模型）比较，做了详尽的消融和敏感性分析，可视化案例清晰。
6. **实际效益**：在提高精度的同时减少了平均交互次数，表明策略更高效。

### 8. 不足与局限
1. **训练开销**：GAPD的训练时间约为outcome-only RL的1.5倍（12.4h vs 8.2h），虽然推理时无额外开销，但训练成本更高。
2. **依赖金标逻辑形式**：框架的蒸馏信号完全依赖于数据集中提供的金标逻辑形式；对于没有此类标注的数据集，GAPD无法直接应用。
3. **状态匹配阈值敏感**：使用固定的Jaccard阈值（0.95）可能在某些场景下过于严格或宽松，文中未讨论自适应阈值策略。
4. **商业模型对比的局限性**：商业模型对比仅在500样本子集上报告，可能不完全反映完整测试集上的表现，且商业模型调用成本高导致无法进行大规模全量对比。
5. **仅包含三个基准**：虽然是KBQA主要基准，但未在更复杂的多跳或时间约束问答数据集上验证；对动作空间的泛化性尚待检验。
6. **潜在偏差**：蒸馏信号可能偏向金标轨迹，尽管有滤波保留一些探索，但若金标动作并非唯一正确路径，可能限制策略发现更优路径的能力。
