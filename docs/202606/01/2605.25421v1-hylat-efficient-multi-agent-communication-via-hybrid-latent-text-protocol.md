## 摘要
通信协议设计是基于大型语言模型的多智能体系统中的核心挑战。现有单通道方法面临一个固有的通信三难困境：基于文本的方法可解释但冗长，而基于潜在空间的方法高效但不透明，且仅限于单向工作流。受多通道通信理论启发，我们提出了HyLaT，一种混合潜文本通信协议，通过潜在通道传输精密的认知信号以提高效率，同时用自然语言表达简洁的关键信号以保持可解释性和精确性。我们引入了一个两阶段训练框架，结合单智能体混合生成学习和多智能体交互协同训练，使智能体能够在多轮交互中生成和解释混合消息。实验表明，HyLaT在保持竞争性任务性能的同时显著降低了通信开销，并在不同设置下表现出强大的泛化能力和鲁棒性。

## Abstract
Communication protocol design is a central challenge in large language model-based multi-agent systems. Existing single-channel approaches face an inherent communication trilemma: text-based methods are interpretable but verbose, while latent-space methods are efficient but opaque and limited to unidirectional workflows. Inspired by multi-channel communication theory, we propose HyLaT, a hybrid latent-text communication protocol that transmits elaborate cognitive signals through a latent channel for efficiency, while expressing concise critical signals in natural language to preserve interpretability and precision. We introduce a two-stage training framework combining single-agent hybrid generation learning and multi-agent interactive co-training, enabling agents to generate and interpret hybrid messages across multiple rounds of interaction. Experiments demonstrate that HyLaT reduces communication overhead significantly while maintaining competitive task performance, with strong generalization and robustness across diverse settings.

- **目标**：提出一种混合双通道通信协议，在保持可解释性的同时大幅提升通信效率，并支持多轮多智能体交互。
- **整体含义**：通过区分“精密的认知信号”（推理、解释）和“简洁的关键信号”（答案、决定），分别用不同通道传输，从而打破三难困境，实现效率与透明度的平衡。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：双通道协议（HyLaDual Channel）：
  - **潜伏通道**：将繁复的认知信号（如解释、推理过程）压缩为连续向量序列（k=6个向量），由特殊token `<bot>` 和 `<eot>` 界定。
  - **文本通道**：将简洁的最后答案或决定用自然语言表达，保持可解释性。
- **关键技术细节**：
  - **输出格式**：
    - 反应 = `[<bot> z1,...,zk <eot>]`（潜伏部分） + `[y1,...,ym]`（文本部分）。
  - **两阶段训练框架**：
    - **阶段1：单智能体混合生成学习**
      - 输入：纯文本 `x`；输出：文本解释 `e` + 答案 `y`。
      - 两个前向过程：
        - 混合前向：模型自回归生成潜伏向量 `Z`，然后生成文本 `y`，计算损失 `L_hybrid`（仅对 `y` 的LM损失）。
        - 文本前向：监督生成全文本 `[e, y]`，计算 `L_text`。
      - 交叉通道对齐损失 `L_align`：对齐两个分支在预测最终答案前的隐藏状态（针对一个共享目标位置），传递文本解释的信息到潜伏空间。
      - 总损失：`L_stage1 = α L_hybrid + β L_text + γ L_align`，文中α=β=1，γ=20。
    - **阶段2：多智能体交互协同训练**
      - 构造多轮多智能体对话数据，每个智能体输入包含其他智能体的前期反应（包括潜伏向量和文本）。
      - 类似阶段1计算每个智能体的损失，然后对所有智能体平均：`L_stage2 = (1/N) Σ_i (α L_hybrid^(i) + β L_text^(i) + γ L_align^(i))`。
      - 通过此训练，智能体学会解读同伴的潜伏向量并生成恰当的混合反应。
  - **通道专业化**：通过训练数据自然实现——解释/推理进入潜伏通道，答案进入文本通道。

## 6. 主要结论与发现

- **效率-性能平衡**：HyLaT在显著降低通信开销（token减少至文本方法的1/10，时间减少至1/6）的同时，保持了与最佳训练文本方法（TextFullT）相当的竞赛性能，并优于纯潜空间方法。
- **双通道互补**：纯文本变体效率低，纯潜空间变体准确率下降且格式错误率上升；双通道最优。
- **两阶段训练的必要性**：缺少阶段1导致潜伏生成不稳定；缺少阶段2导致无法处理同伴潜伏向量，格式错误率高达22%。
- **可扩展性**：随着智能体数量或交互轮数增加，HyLaT的token消耗增长缓慢，优于纯文本方法；通过补充更大规模训练数据可缓解性能下降。
- **兼容性**：HyLaT可与NL智能体混合工作，且增加HyLaT比例仍能降低token并提升性能。
- **鲁棒性**：对潜伏向量噪声（σ ≤ 0.5）不敏感。
- **可解释性**：注意力机制显示潜伏向量确实携带其他智能体的信息；PCA聚类显示潜伏向量按数据集语义分离。
- **跨模型泛化**：在Llama-3.2-3B和Qwen2-1.5B上均表现良好，且支持异质智能体通信。

## 7. 优点（方法论与实验设计亮点）

- 两阶段训练框架设计合理：先单智能体学习混合生成，再多智能体协同学习解读潜伏向量；交叉通道对齐损失巧妙传递语义到潜伏空间。
- 实验设计全面：覆盖多种基线、消融、异质、噪声、可扩展性、社会模拟、跨模型验证；统计充分。

- 可解释性分析：注意力分析、PCA探针提供了对潜伏向量语义的直观证据。
- 实用性：保留了文本通道，可与现有文本协议智能体互操作，降低了部署门槛。

## 8. 不足与局限

- **模型规模有限**：主实验仅使用1B参数模型，更大模型（如7B/70B）上的效果未知。
- **部署兼容性**：当前修改的生成流程不兼容vLLM等推理框架，限制了大规模或长时应用的评估。
- **文本通道多样性受限**：训练数据以QA辩论为主，导致文本通道表达有限；固定答案提示可能约束了行为的丰富性。
- **潜伏可解释性仍不足**：虽优于纯潜空间方法，但潜伏向量的语义内容仍难以直接解码，平衡可解码性与通信性能是开放问题。
- **潜在偏差**：训练数据来源（推理数据集）可能导致通道专业化过度依赖特定数据分布，未评估其他类型通信场景（如开放域对话、指令执行）。
- **未讨论通信延迟或带宽实际限制**：仅以token和时间衡量效率，未考虑潜伏向量传输的额外开销（如模型推断计算成本）。
- **未展示与最先进的纯文本方法（如GPT-4级别的辩论）的竞争力**：基线模型较弱，可能无法代表前沿水平。
