## 摘要
基于大语言模型的多智能体系统需要大量难以先验确定的协调选择：调用哪个技能协议、哪个智能体角色应执行子任务、每个角色应绑定哪个模型、角色之间如何交互、何时使用检索或验证、以及何时完全省略某个步骤。这些选择与任务场景和操作约束相互影响，因此静态流水线和一次性模型比较只能提供设计空间的有限视角。本文介绍了AgensFlow，一个开源框架，它将多智能体协调视为部分可观测下的在线策略学习问题。该框架使协调决策可观测且可从重复轨迹中学习，而不是将技能、角色、模型、拓扑和评估选择视为固定的流水线设计。AgensFlow在两个语料库上进行评估：分布式系统事件任务和安全咨询任务。评估显示了三个主要结果：在协调密集型类别中，学习到的路由比固定流水线基线达到更高质量的工作点；skip：X将拓扑压缩隔离为基础的有意义部分；热启动策略图可以在保持平台质量的同时降低探索成本。总体而言，结果表明，与静态布线相比，可学习、可审计的路由可以改善协调密集型多智能体工作流。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

### 1. 论文的核心问题与整体含义

*   **研究动机**：当前基于大语言模型（LLM）的多智能体系统在构建时，面临大量难以预先固定的协调选择，例如：

    *   哪个智能体角色应执行哪个子任务？
    *   每个角色绑定哪个模型？

*   **核心问题**：这些选择与任务类型、操作约束（如成本、延迟）相互交织、动态变化。静态的流水线或一次性模型比较无法充分探索这个巨大的设计空间，导致系统在面对不同任务时表现不稳定或低效。
*   **整体含义**：论文论证了解决上述问题的关键在于将多智能体系统的**协调**本身视为一个可学习的、可观测的在线策略问题，而非一个静态的工程设计问题。为此，作者提出了**AgensFlow**开放源码框架，旨在提供一个可审计、可优化的协调策略基础。

### 2. 论文提出的方法论

*   **核心思想**：将多智能体系统的动态协调建模为部分可观测马尔可夫决策过程中的在线策略学习问题。系统的目标是学习一个路由策略，针对不同的任务“折叠签名”选择最优的技能、模型、拓扑和评估动作。
*   **关键技术细节**：
    1.  **状态抽象为折叠任务签名**：系统不直接观测任务的全部细节（如用户意图、模型内部状态），而是将任务特征、中间状态和信念估计量折叠成一个离散的“签名”，以实现跨任务的状态共享和高效学习。`st = ϕ(xt, ht, bt)`，包含：场景标签（`ℓt`，如 straightforward, ambiguous）、二值手状掩码（`zt`，描述是否有目标、子问题、证据等字段被填充）、以及连续信念估计（如正确性 `ĉt`、不确定性 `û_t` 等）。这是实现在线学习可行的关键压缩。
    2.  **动作空间**：每个路由步的动作包括：`invoke(k, m)`（调用绑定模型 `m` 的技能协议 `k`）、`skip:X`（跳过尚未执行的可选单元 `X`）、和 `terminate`（终止，但由系统隐式触发，非策略选择）。这使得**拓扑结构（skip:X）** 本身成为可学习的一部分。
    3.  **奖励函数**：采用混合奖励，结合任务质量与操作成本：`r(t) = wq*Q(t) - wc*C(t) - w_ρ*ρ(t)`。其中 `Q(t)` 是**RelativeJudge**给出的相对质量分数（0-1），`C(t)` 是归一化的Token成本，`ρ(t)` 是失败惩罚。默认权重显示将质量置于最高优先级。
    4.  **学习对象与算法**：学习的对象是一个关键任务签名的**可审计策略图**。策略选择采用**可靠感知的UCB1**变体：`score(s, a) = r̄(s, a) + cs*sqrt(log(Ns+1)/Ns,a) - λf(s,a)`。该算法在探索-利用平衡之外，还加入了退火探索系数和基于失败率的可靠惩罚项（`-λf(s,a)`），鼓励模型选择更稳定的动作。
    5.  **Runtime数据流**：任务执行流程从预飞检查开始，经特征提取和场景检测后，进入一个循环：路由（基于UCB1选择动作）->执行（调用智能体单元）->更新手状状态和签名。循环直至任务结束，最后通过**RelativeJudge**评分并更新策略图。框架分为6个依赖层（Schema/Regime, Agents/Transport, Policy Graph/Router, Reward/Judge, Pre-flight/Governance, Web Search），各层同时运行。

### 3. 实验设计

    *   语料库2：**60个合成安全咨询任务**，覆盖8个场景类别（C1-C8），如程序性任务(C1)、单文档任务(C2)、跨文档多厂商推理(C3)、合成任务(C4)、语料库外模糊任务(C5)等，旨在测试跨文档推理和协调能力。

*   **对比方法**：在安全咨询任务上对比了3种配置与基线：
    *   **无跳过消融**（no-skip ablation）：启用变体池学习，但强制禁用`skip:X`。
    *   **主方案**（main）：8轮训练，启用`skip:X`，冷启动。
    *   **热启动**（warm-start）：8轮训练，启用`skip:X`，使用在分布式系统语料库上学习到的策略图（443节点）初始化。

### 6. 论文的主要结论与发现

1.  **学习路由优于固定管线**：在8个场景类别中，学习策略在5个上优于固定管线，尤其是在**协调密集型**场景（C3跨文档推理、C8跨厂商对任务、C7缓解正确性）上提升显著（分别+0.181, +0.156, +0.131）。在程序性任务上则基本持平。
2.  **拓扑压缩（skip:X）是有效组件**：`skip:X` 动作被证实是学习框架中一个有意义的组成部分。启用 `skip:X` 的主方案在质量相近的情况下，Token成本显著低于无跳过消融方案（+15% vs +94%），表明模型学会了动态裁剪不必要的步骤。
3.  **热启动降低探索成本**：使用相似领域的预训练策略图进行热启动，在保持审计质量的同时，能显著降低学习初期的Token成本（~10%至~21%），加快了学习效率。
4.  **奖励信号审计至关重要**：跨评判员审计揭示了单评判员结果可能夸大热启动的质量优势。热启动在单评判员下质量提升显著，但在三评判员审计下，其质量优势大幅缩小，而成本优势保持不变。这证明**奖励信号本身也是需要被审计的协调问题的一部分**。

### 7. 优点

*   **创新性的问题定义**：将多智能体系统的核心瓶颈精准地识别为动态协调问题，并定义为在线策略学习问题，视角新颖。

*   **强调可审计性与鲁棒性**：

    *   **奖励信号审计**：将跨评判员审计设计为框架的一部分，正视并解决LLM作为评判者可能带来的偏见，提高了结论的可靠性。
*   **实用的消融与迁移实验**：`skip:X`消融和跨域迁移实验设计清晰，有力支持了提出的两个核心论点（拓扑可学习、策略可迁移）。
*   **与相关工作的关联**：
    *   借鉴了模块化专家协调（RIM）思想，将其从可微分的序列模型扩展至可审计的Agent协调策略。
    *   基于相对轨迹评估（RULER）设计其奖励模型。
    *   清晰解释了为何其不同于对话式MAS（如AutoGen）和工具使用学习（如Toolformer），定位明确。

### 8. 不足与局限

*   **实验覆盖有限**：
    *   **拓扑类别单一**：仅测试了线性+可跳过的拓扑。论文承认并指出，并行、层次化、分支-合并等更复杂的协调拓扑虽属于框架设计范畴，但未被评估。
    *   **语料库规模小**：60个任务/类别可能不足以代表真实世界协调的复杂性。最终策略图仅包含443个“节点”，状态空间相对较小。
    *   **学习轮次短**：8轮学习可能不足以让策略充分收敛。
    *   **单评判员实时奖励**：实时学习信号仅依赖单一评判模型（Claude Haiku 4.5），引入了已知的偏差风险。论文也指出，跨评判员审计的正确做法应是**实时**执行。
*   **偏差风险**：正如跨评判员审计所揭示，单评判员结果可能导致误导性结论。论文没有探索其他更鲁棒的奖励建模方案。
*   **应用限制**：框架本身不通过梯度传播进行端到端学习，而是依赖于一个启发式的信念估计系统来构建签名，其有效性和鲁棒性在更复杂的任务中可能面临挑战。
