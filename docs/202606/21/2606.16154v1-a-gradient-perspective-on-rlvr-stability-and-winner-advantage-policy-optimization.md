## 摘要
基于可验证奖励的强化学习（RLVR）提高了语言模型的推理能力，但GRPO式优化仍然容易崩溃。我们通过token级梯度动态分析这种不稳定性，推导出一个分类体系，预测更新如何影响下一token概率和熵。该分类表明，稳定性同时取决于优势符号和当前策略下的token分布。受此发现启发，我们提出胜者优势策略优化（WAPO），一种简单的在线截断策略梯度目标，仅对正优势的完成序列进行更新。在数学推理和多跳问答基准测试中，WAPO提高了训练稳定性，并在多个模型家族中匹配或超过基线。完整代码见https://github.com/layer6ai-labs/wapo。

## Abstract
Reinforcement learning with verifiable rewards (RLVR) improves language-model reasoning, but GRPO-style optimization remains prone to collapse. We analyse this instability through token-level gradient dynamics, deriving a taxonomy that predicts how updates affect next-token probabilities and entropy. The taxonomy shows that stability depends jointly on the advantage sign and token distribution under the current policy. Motivated by this finding, we propose Winner Advantage Policy Optimization (WAPO), a simple online clipped policy-gradient objective that updates only on positive-advantage completions. Across mathematical reasoning and multi-hop QA benchmarks, WAPO improves training stability and matches or outperforms baselines across multiple model families. Full code can be found at https://github.com/layer6ai-labs/wapo.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：基于可验证奖励的强化学习（RLVR，如 GRPO）是提升语言模型推理能力的主流范式，但在训练过程中极易发生崩溃（collapse），表现为输出退化为多语言、代码片段等不可读内容（高熵崩溃）或重复固定的错误模式（低熵崩溃）。
- **背景挑战**：现有分析将不稳定性归因于旧策略和新策略之间的漂移（drift）以及训练-推理引擎不匹配，并提出基于信任区域裁剪等方法。然而，作者发现**单纯抑制离群 token 并不能保证稳定**（见图 2），有时反而导致崩溃。因此需要从 **token 级别的梯度局部效应**出发重新理解不稳定性的根源。

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：不从“离旧策略有多远”出发，而是分析**当前策略下每个 token 在梯度更新后概率和熵的瞬间变化**，从而揭示哪些 token 更新会导致熵增（不稳定）或熵减（稳定）。
- **关键技术细节**：
  1. **token 梯度推导**：对于 softmax 分布，一个梯度下降步（步长 η）在非采样 token i 上的概率变化为：  
     ∆pi = ηA pi (C(p) − p_s − p_i) + O(η²)，其中 C(p) = Σ_j p_j²。
  2. **Peak/Valley 分类**根据采样 token 的概率 p_s 与参考水平 C(p) 的关系，定义 **peak**（p_s ≥ C(p)）和 **valley**（p_s < C(p)）。结合优势 A 的符号得到四个象限：Pos-peak、Pos-valley、Neg-peak、Neg-valley。
  3. **熵变化预测**：通过一阶展开，Pos-valley 和 Neg-peak 增加熵，Pos-peak 和 Neg-valley 减少熵。实验验证了该预测。
  4. **WAPO 算法**：仅保留正优势（A>0）的完成序列进行策略梯度更新。对于每个 prompt，采样 G 个完成，计算标准组内归一化优势 Ai，然后只对 Ai>0 的 token 计算截断重要性比损失，且不包含下界裁剪（因为已经筛选了正优势）。在二值奖励设定下，WAPO 的梯度方向与二值策略梯度 ∇q_x 相同，但带有自适应因子 (1−q_x)，随成功概率增加而衰减，自动平衡难易 prompt 的更新强度。
- **算法流程**（文字说明）：
  - 对每个 prompt x，从 rollout 策略 π_θ_old 采样 G 个完成 {y_i}。
  - 计算每个完成的奖励 r_i，然后组内归一化得优势 A_i。
  - 只保留 A_i > 0 的完成（称为“胜者”）。
  - 对胜者中的每个 token，计算重要性比 ρ_ij(θ)，并求 min(ρ_ij, 1+ε) × A_i（ε 为上界裁剪）。
  - 对所有 prompt 的该损失取平均（且用固定最大序列长度 T 归一化避免长度偏置）。

## 6. 论文的主要结论与发现

- 不稳定性根源是 token 级别梯度效应，而非简单的策略偏离程度。**Peak/Valley 分类**能准确预测更新后熵的增减方向。
- **Pos-peak 稳定但保守**，**Neg-peak 和 Pos-valley 导致熵增崩溃**，**Neg-valley 起初提升但最终过信崩溃**。
- 基于此，**WAPO（仅保留正优势更新）** 在几乎所有任务上提升了训练稳定性，在多跳 QA 上显著优于基线（如 OTT-QA 上相比次优方法提升 9.9%），在数学任务上性能与最强基线持平或略低但从不崩溃。
- WAPO 还能保持 pass@k 多样性，并具有良好的 OOD 泛化能力（如在 HotpotQA → 2Wiki 上跨模型均最佳）。

## 7. 优点（方法或实验设计上的亮点）

- **理论驱动**：从 softmax 一阶梯度给出了严格的数学推导，并由此提出简单有效的实用方法。
- **极简干预**：WAPO 仅改变优势符号的取舍，不引入额外正则化或复杂信任区域机制，易于实现和集成。
- **实验设计全面**：跨多种任务、模型、基线，包含稳定性、pass@k、OOD 泛化等角度，消融实验直接验证理论分类。
- **揭示了普遍现象**：多种模型和数据集上均观察到相似的不稳定模式，说明发现的普遍性。
- **开源代码和复现细节**：提供完整代码和超参数设置。

## 8. 不足与局限

- **数学任务上性能有限**：在 Math-500 和 NuminaMath-LEAN 上，WAPO 早期略低于 GRPO/GSPO，最终指标也仅持平或略低，未展示明显优势。作者推测自适应因子 (1−q_x) 早期可能过于保守。
- **仅覆盖中等规模模型**：实验限于 3B-4B 参数量的模型，未扩展到更大规模（如 7B、13B）或 MoE 架构，泛化性有待验证。
- **未涵盖编程任务**：如 text-to-SQL、代码生成等可验证领域，当前实验集中在数学和问答上。
- **单次运行风险**：未报告多次随机种子实验，结果可能受随机性影响。
- **理论假设**：梯度分析基于一阶近似，实际训练中高阶项可能在某些情况下主导；peak/valley 分类中的边界情况（如非常接近 C(p) 的 token）可能行为不确定。
- **WAPO 的适应性因子 (1−q_x) 依赖于组内成功概率估算**：当组大小 G 较小时，¯r 可能不准确，导致梯度方向偏差。
- **负优势信息完全丢弃**：可能丢失某些有用信号（如负优势 valley token 的抑制学习可帮助早期快速收敛），但作者认为在粗粒度奖励下筛选困难，故舍弃。这可能在某些任务上影响收敛速度。
