Abstract 

Vision–language models remain susceptible to multimodal jailbreaks and over-refusal because safety hinges on both visual evidence and user intent, while many alignment pipelines super-vise only the final response. To address this, we present SaFeR-ToolKit, which formalizes safety decision-making as a checkable protocol. Con-cretely, a planner specifies a persona, a Percep-tion →Reasoning →Decision tool set, and a con-strained transition graph, while a responder out-puts a typed key–value tool trace before the fi-nal answer. To make the protocol reliably fol-lowed in practice, we train a single policy with a three-stage curriculum (SFT →DPO →GRPO), where GRPO directly supervises tool usage be-yond answer-level feedback. Our contributions are two-fold: I. Dataset. The first tool-based safety reasoning dataset, comprising 31,654 ex-amples (SFT 6k, DPO 18.6k, GRPO 6k) plus 1k held-out evaluation. II. Experiments. On

1. Introduction 

Vision-language models are rapidly moving from lab sys-tems to deployed assistants that interpret images and support real decisions, including in settings where errors can cause harm (Hurst et al., 2024; Bai et al., 2024; Helff et al., 2024; Zhou et al., 2024b; Huang et al., 2024; Li et al., 2025c).

Because their outputs depend on both text and pixels, they introduce safety failure modes that do not appear in text-only assistants (Liu et al., 2025a; Ye et al., 2025). Adversarial multimodal inputs can steer a model away from visual evi-dence and toward unsafe or policy-violating behavior (Liu et al., 2024b; Gong et al., 2025; Ye et al., 2025). At the same time, stronger safety tuning can increase over-refusal, where benign requests are rejected because intent and context are hard to separate (Cao et al., 2025; Choi et al., 2025). This creates a deployment gap: the system may comply when it should refuse, or refuse when it should help . A core structural difficulty is that the safety decision is usually not exposed as an explicit, testable reasoning process, which makes auditing and targeted fixes inherently hard. Recently, safety alignment for vision-language models has increasingly relied on preference and policy optimization beyond SFT (Ye et al., 2025; Yu et al., 2025), including DPO (Zhang et al., 2025b; Weng et al., 2025a; Zhang et al., 2025c), PPO-style RLHF (Ji et al., 2025; Tan et al., 2025), and GRPO variants (Liu et al., 2025b; Rong et al., 2025), often combined with post-hoc guards (Oh et al., 2024; Helff et al., 2024; Zeng et al., 2025). These methods can improve average behavior, but they still optimize safety mainly at the level of the final response and keep the decision process im-plicit. In multimodal settings, this becomes fragile because perception and intent are coupled (Liu et al., 2024b; Li et al., 2024; Lee et al., 2025). When an image contains prompt injections or other adversarial cues (Kimura et al., 2024; Gong et al., 2025; Yeo & Choi, 2025), the model has no ex-plicit step that checks whether the instruction it is following is consistent with the visual evidence, so fluent outputs can still drift away from what is actually in the image (Sun et al., 2024; Liu et al., 2024a; Zhao et al., 2024), vice versa (Shao et al., 2024; Xia et al., 2025). This points to a structural gap: robust safety needs an inspectable process with explicit testable intermediate steps, which we aim to make safe by design and keep checkable and traceable. Motivated by this gap, we propose a protocolized alignment framework that makes safety a concrete, auditable procedure for deployed assistants. Instead of mapping inputs directly to a final answer, we insert a structured decision process defined by virtual safety tools . Operationally, a planner specifies a calling protocol that constrains which tools are available and how they may transition; a responder generates a tool trace under this protocol and then produces the user-facing response conditioned on the trace. This pro-tocolized tool-trace view naturally motivates SaFeR-ToolKit and its training pipeline. 

SaFeR-ToolKit. We instantiate this idea with SaFeR-ToolKit, a staged library of virtual tools that turns interme-diate safety reasoning into a standardized trace (Fig. 1a) for auditing and evaluation. Tool outputs are typed records, enabling automatic validation and consistency checks across steps. The ToolKit is organized into Per-ception →Reasoning →Decision , separating visual ground-ing, intent-and-policy analysis, and the final action (safe answering or refusal). For each input, the planner selects a responder persona, a tool subset, and a constrained topol-ogy (Zhang et al., 2024; Wang et al., 2025a) that restricts the tool order, supporting safety gating and bounded self-correction while keeping the final response natural. 

Pipeline & Difference. SaFeR-ToolKit further prescribes a progressive three-stage alignment curriculum. First, SFT 

bootstraps schema adherence and basic tool execution from curated demonstrations, ensuring the model can reliably invoke virtual operators. Next, DPO refines tool selection and execution by contrasting correct traces against structural degradations (e.g., omitted steps), effectively suppressing logical hallucinations. Finally, GRPO supervises tool usage in the reasoning process via a compound reward, going beyond answer-level feedback. This stage transforms the model from a rigid imitator into an active agent that flexibly scales its reasoning depth while preserving safety, achieving adaptive and verifiable alignment. Ultimately, this yields a safety mechanism that is verifiable by design, ensuring that every decision is backed by an explicit, auditable chain of evidence rather than opaque heuristics .

Difference : To our knowledge, SaFeR-ToolKit is the first vision-language safety framework that formalizes safety rea-soning into a constrained, typed tool trace, turning safety from a final-answer objective into an auditable decision process. It plugs into standard alignment pipelines and gen-eralizes across models by reusing the protocol and extending the tool library for new attacks. 

Dataset. As the first tool-based safety reasoning dataset, it comprises 31,654 examples, stratified into 6,000 SFT demonstrations, 18,654 DPO pairs, 6,000 GRPO queries, and a 1,000-sample held-out (Fig. 1b). The inputs are synthesized from sources, balancing safety-critical data (e.g., BeaverTails-V (Ji et al., 2025), JailBreakV-28k (Luo et al., 2024)) with general reasoning tasks. The resulting traces feature 8,171 tool instances, dominated by Reason-ing tools (4,259) to drive intent analysis, supported by visu-ally grounded Perception tools (1,203) and Decision tools (2,709) for robust judgments and safety-aware interventions. 

Experiment. On safety-focused benchmarks, our pipeline achieves the best balance of Safety , Helpfulness , and Rea-soning Rigor among all methods, with large gains over base models ( 3B : 29.39/45.04/4.98 → 84.40/71.13/78.87; 7B :53.21/52.92/19.26 → 86.34/80.79/85.34) and consistently outperforming existing safety baselines. In sharp contrast, guard-based baselines often increase Safety but drastically reduce Helpfulness , consistent with over-refusal. On general benchmarks, our alignment robustly preserves core multi-modal capability and yields small average improvements (3B : 58.67 → 59.21; 7B : 66.39 → 66.81), while several safety baselines noticeably degrade general performance. 2SaFeR-ToolKit: Structured Reasoning via Virtual Tool Calling for Multimodal Safety 

2. Related Work 

Multimodal Large Language Models have shown strong results in vision–language alignment and instruction tuning (Dai et al., 2023; Liu et al., 2023; Zhu et al., 2023; Yin et al., 2024). Recent breakthroughs in reasoning-focused language models, such as OpenAI o1 (Jaech et al., 2024) and DeepSeek-R1 (Guo et al., 2025), underscore the ef-fectiveness of scaling test-time compute and learning from verification-based training signals (Zhang et al., 2025a). Mo-tivated by these insights, MLLM research is rapidly shifting toward reasoning-centric training and decoding strategies (Li et al., 2025b; Wang et al., 2025b; Zhou et al., 2025a). This evolution enables multi-step deduction and chain-of-thought reasoning in visual contexts, rather than relying on superficial pattern matching. Consequently, this shift has driven clear gains on challenging benchmarks, including visual perception (Dong et al., 2025; Fan et al., 2025), math-ematical reasoning (Luo et al., 2025; Sun et al., 2025), and spatial understanding (Li et al., 2025d; Batra et al., 2025). However, this enhanced cognitive capability concomitantly expands the attack surface: sophisticated thought processes can be exploited to obfuscate harmful intent and bypass standard detection (Liu et al., 2024c; Ye et al., 2025). 

Safety Alignment. Ensuring safety alignment is a prereq-uisite for the trustworthy deployment of large-scale multi-modal models. However, recent benchmarks highlight per-sistent vulnerabilities in current models (Liu et al., 2024b; Luo et al., 2024; Zhou et al., 2024a; Gong et al., 2025). To mitigate these risks, traditional defenses remain pre-dominantly result-oriented, relying on training-stage align-ment via supervised fine-tuning and preference optimiza-tion (Zong et al., 2024; Zhang et al., 2025b; Zhou et al., 2025b; Weng et al., 2025a; Ji et al., 2025) or inference-time interventions (Helff et al., 2024; Gou et al., 2024; Na et al., 2025; Banerjee et al., 2025). Nevertheless, treating safety as a black box limits interpretability and auditability. Accordingly, recent work moves toward reasoning-aware safety by making intermediate steps explicit and supervised. Representative approaches primarily utilize supervised rea-soning datasets to explicitly teach risk analysis (Lou et al., 2025; Gou et al., 2025) or reinforcement learning (Liu et al., 2025b; Yi et al., 2025; Weng et al., 2025b; Rong et al., 2025). However, these methods often lack explicit structure or rig-orous control over the reasoning process. To address this, we propose SaFeR-ToolKit, where a planner selects virtual tools from Perception, Reasoning, and Decision layers with constrained transitions, and a responder produces auditable traces for verifiable safety alignment.

3.3. Training Data 

Building on the planner–responder pipeline above, we generate training data for a three-stage curriculum (SFT →DPO →GRPO), as illustrated in Fig. 2. Each sample starts from an input x = ( I, q ). A deterministic planner maps x to a configuration C(x) (persona, tool subset, and topology). During data generation , C(x) is compiled into a system prompt that conditions the responder to produce (i) a multi-step virtual-tool trace z = ( ts, o s)Ss=1 and (ii) a final response y. We only store (x, z, y ) (or their paired variants below): the compiled prompt and any internal planner text are not stored in the training data. 

SFT data. We generate supervised traces and reference responses under the induced configuration and store 

DSFT = {(x, z, y )}. (9) SFT trains the target model to predict (z, y ) from x, learning the structured trace format and basic virtual-tool usage. 

DPO data. We construct preference pairs to teach the model which tools to use and how to use them for a given input x. Chosen samples are generated by the same planner– responder procedure as SFT, producing (x, z +, y +) where 

z+ uses an appropriate tool subset and follows a topology-consistent sequence. Rejected samples keep the same input 

x but are synthesized via systematic model-based perturba-tions that degrade tool choice or tool usage (Fig. 2): (i) tool selection errors (omitting essential invocations or selecting inappropriate tools), and (ii) tool execution errors (breaking logical continuity, semantic inconsistencies, or traces that lead to unsafe or low-quality answers), yielding (x, z −, y −).We then form 

DDPO = {(( x, z +, y +), (x, z −, y −)) }, (10) and train the model to prefer the chosen outputs over the rejected ones, so that it learns to separate high-quality vs. low-quality tool traces and to produce safer, more helpful responses supported by better tool use. 

GRPO data. We form a rollout-only input set 

DRL = {x}, (11) disjoint from DSFT and DDPO . GRPO uses DRL to collect on-policy trajectories (z, y ), where z is a multi-step virtual-tool trace. Unlike SFT/DPO, which supervise or rank fixed outputs, GRPO targets policy-level behavior: it trains the model to invoke tools when needed and to adapt tool order-ing and reasoning depth to the input while preserving safety, enabling adaptive tool-based alignment at test time. 4SaFeR-ToolKit: Structured Reasoning via Virtual Tool Calling for Multimodal Safety

5. Conclusions 

We present SaFeR-ToolKit, a framework that transforms multimodal safety alignment from an implicit final-answer objective into a checkable, auditable decision process via vir-tual tool calling. By formalizing safety reasoning through a constrained Perception →Reasoning →Decision proto-col with typed tool traces, SaFeR-ToolKit effectively mit-igates both jailbreaks and over-refusal. We contribute a 31k-example dataset stratified into SFT, DPO, and GRPO training sets plus a held-out evaluation set, along with a progressive SFT →DPO →GRPO curriculum that bootstraps trace adherence, refines tool selection via preference learn-ing, and optimizes adaptive tool usage via composite re-wards. Experiments demonstrate substantial improvements in safety, helpfulness, and reasoning rigor while preserv-ing general multimodal capabilities, establishing SaFeR-ToolKit as a practical paradigm for verifiable and trustwor-thy vision-language safety alignment. 8SaFeR-ToolKit: Structured Reasoning via Virtual Tool Calling for Multimodal Safety 

Impact Statement 

Ethical Considerations. We believe that SaFeR-ToolKit raises no notable additional ethical concerns beyond stan-dard considerations in multimodal safety research. SaFeR-ToolKit aims to enhance model safety through checkable and verifiable reasoning. Since our experiments leverage safety datasets that may include harmful or jailbreak-style queries for training and evaluation, we emphasize responsi-ble data handling practices such as clear content warnings and appropriate access controls. In deployment, we recom-mend keeping internal tool traces and safety reasoning logs private to preserve secure and reliable system operation. 

Societal Implications. SaFeR-ToolKit advances the trust-worthy deployment of vision-language assistants by en-abling auditable and traceable safety reasoning, making it easier to understand why a model refuses or complies. This improves reliability and user trust, supports reproducible evaluation, and helps developers diagnose and refine safety behavior in complex multimodal settings. The resulting transparency and more calibrated refusal behavior are partic-ularly valuable in high-impact applications such as content moderation, educational tools, and accessibility services, where both safety and clear justification of model behavior are important for responsible adoption.

Appendix A. ToolKit Construction Details 

This section details the planner-responder architecture (Sec-tion 3.2), covering scenario taxonomy, persona library, tool library, and topology specifications. Both modules are im-plemented using Gemini-3.0-Pro 3.

A.1. Scenario Taxonomy 

The planner classifies inputs into one of nine scenario cate-gories organized into three meta-classes, each with a differ-ent optimization objective: 

Safe Scenarios (Quality Focus). These scenarios priori-tize response quality while maintaining baseline safety: • [1.1] Over-Refusal Risk : Safe objects or contexts that might be mistakenly flagged as risky (e.g., kitchen knives in cooking tutorials, historical weapons in museums). • [1.2] Emotional Complex : Emotionally nuanced queries requiring empathetic, context-aware responses (e.g., grief support, relationship advice). • [1.3] Safe Optimal : Standard safe requests where the goal is to provide the highest-quality response. 

Unsafe Scenarios (Safety Focus). These scenarios priori-tize safety enforcement and harm prevention: • [2.1] Harmful Request : Explicit requests for harmful instructions (e.g., violence, illegal activities). • [2.2] Visual Threats : Visual content containing embed-ded threats, such as harmful text overlays or dangerous symbols. • [2.3] Cross-Modal Conflict : Text and image are individ-ually benign but become harmful when combined. 

Controversial & Gray Areas (Reasoning Focus). These scenarios require deep reasoning to navigate ambiguity: • [3.1] Ambiguous Intent : Dual-use scenarios (e.g., secu-rity research vs. hacking). • [3.2] Subjective Topic : Topics requiring balanced, unbi-ased responses (e.g., political debates). • [3.3] Ethical Dilemma : High-stakes dilemmas requiring human-centered reasoning (e.g., crisis intervention).

B.1. Data Sources Overview 

We curate training data from multiple sources to ensure diversity and coverage of various scenarios. Our data in-cludes safety-focused datasets and general-purpose datasets. Below we briefly describe each dataset. 

BeaverTails-V (Ji et al., 2025): A multimodal safety dataset composed of (question, image, response) pairs. It features fine-grained annotations, including multi-level severity rat-ings across diverse harm categories. We utilize samples from its training split. 

JailBreakV-28k (Luo et al., 2024): A benchmark of text-image jailbreak pairs. We use items from its test set for training, excluding them from evaluation. 

Hateful Memes (Kiela et al., 2020): Designed for multi-modal reasoning, this dataset contains memes annotated for hate speech. We sample items from the training set.

Align-Anything (Ji et al., 2024): A comprehensive multi-modal dataset containing human preference annotations and textual feedback. We select examples for training. 

Geometry3K (Lu et al., 2021): A geometry problem bench-mark annotated with formal language descriptions. We sample instances from the training split. 

SPA-VL (Zhang et al., 2025b): A comprehensive safety preference alignment dataset for vision language models. It covers multiple harmful domains and categories, with responses collected from diverse VLMs. We sample items from its training split to enhance safety alignment.

B.2. DPO Data Construction 

This subsection describes how we construct the preference pairs (x, z +, y +, z −, y −) for Direct Preference Optimiza-tion (DPO) training. We adopt a two-stage pipeline that combines prompted perturbation with model-based sam-pling to generate diverse and challenging rejected samples. 

Stage 1: Seed Perturbation. Given a chosen sample 

(x, z +, y +) generated by the planner-responder pipeline, we use Gemini-3.0-Pro to systematically perturb the tool trace and response. Formally, let E = {e1, . . . , e 5}

denote the set of five perturbation types. For each chosen sample, we randomly sample an error type e ∼ Uniform( E)

and generate a rejected sample:

> seed

) = Perturb( x, z +, y +, e ), (19) where Perturb( ·) is the perturbation function guided by a carefully designed prompt template. This stage produces 3,000 seed rejected samples Dseed .

Stage 2: Scaling via Perturbation Model Sampling. To scale up the rejected samples while maintaining distribu-tional diversity, we fine-tune a perturbation model πpert on the seed perturbations: 

πpert = SFT( πbase , Dseed ), (20) where πbase is initialized from Qwen2.5-VL-3B . The per-turbation model learns to internalize the perturbation pat-terns from the seed data. We then use πpert to sample rejected traces for each input x in the DPO training set: 

(z−, y −) ∼ πpert (· | x). (21) The seed samples train πpert exclusively. This two-stage approach ensures: (1) controlled coverage of all five error types, and (2) πpert generates on-distribution rejected sam-ples that are challenging for the target model. In total, we construct 18,654 DPO preference pairs. 

Perturbation Types. We define five perturbation types organized into two categories: 

(i) Tool Selection Errors — degrade which tools are invoked: • Tool Invocation Omission : Omit essential tool invoca-tions required for safe or correct responses, leading to unverified or unsafe outputs. • Inappropriate Tool Selection : Select inappropriate tools that do not match task requirements, resulting in off-topic analysis or undetected risks. 

(ii) Tool Execution Errors — degrade how tools are used: • Tool Semantic Inconsistency : The tool name suggests one type of analysis, but the actual content performs a different operation. • Erroneous Tool Content : The tool content contains fab-ricated or factually incorrect information that does not align with the actual image/context. • Tool Chain Discontinuity : Break logical continuity be-tween consecutive tools, producing internally contradic-tory responses. 

C. Training Details 

This section provides the training configurations for SFT (Stage I), DPO (Stage II), and GRPO (Stage III). 

C.1. SFT Training Configuration 

We perform full-parameter fine-tuning and freeze both the vision tower and multimodal projector while training only the language model. Training runs for 3 epochs with a cosine learning rate schedule (peak 1 × 10 −5, warmup ratio 0.1), per-device batch size of 4, and BF16 precision. 

C.2. DPO Training Configuration 

We perform full-parameter fine-tuning and freeze both the vision tower and multimodal projector while training only the language model. The DPO loss uses sigmoid formula-tion with β = 0 .1. Training runs for 1 epoch with a cosine learning rate schedule (peak 5 × 10 −7, warmup ratio 0.1), per-device batch size of 1, and BF16 precision.

D. More Experimental Details 

This section details the experimental setup, covering bench-marks, baseline methods, tool category ablation data con-struction and evaluation methodology. 

D.1. Safety Benchmarks 

We evaluate our method on both safety alignment bench-marks and general capability benchmarks. 

BeaverTails-V (Ji et al., 2025): A large-scale multimodal preference dataset for safety alignment that provides dual preference annotations (helpfulness and safety are labeled) and graded harm severity labels to support fine-grained safe RLHF for MLLMs. We use its test split for evaluation. 

MM-SafetyBench (Liu et al., 2024b): A safety evaluation benchmark for multimodal LLMs that tests whether mod-els can be compromised by query-relevant images across diverse scenarios, using thousands of text-image pairs to measure vulnerability to image-based manipulations and unsafe behaviors. 

MSSBench (Zhou et al., 2024a): A benchmark for multi-modal situational safety, where the same language query may be safe or unsafe depending on the visual context; it evaluates whether MLLMs can judge when to answer or refuse based on safe vs. unsafe image situations (including assistant-style settings). 

SPA-VL (Zhang et al., 2025b): A comprehensive safety preference alignment dataset for vision-language models, built with large-scale preference data to optimize both harm-lessness and helpfulness across diverse safety-related do-mains. We use its test split for evaluation. 

ToolkitBench (Ours) : Our held-out evaluation set of 1,000 samples designed for auditable safety reasoning, each with reference responses for reproducible evaluation. 

D.2. General Benchmarks MathVista (Lu et al., 2023): A benchmark for evaluat-ing mathematical reasoning in visual contexts, combining 16 SaFeR-ToolKit: Structured Reasoning via Virtual Tool Calling for Multimodal Safety 

diverse visual+math tasks that require fine-grained visual understanding and compositional reasoning. 

MMMU (Yue et al., 2024): A massive multi-discipline mul-timodal understanding and reasoning benchmark targeting college-level subject knowledge and deliberate reasoning across diverse image formats (e.g., diagrams, tables, charts). 

MMStar (Chen et al., 2024c): A vision-indispensable mul-timodal benchmark with curated samples to ensure visual dependency and reduce data leakage, evaluating LVLMs across core multimodal capabilities. 

MM-Vet (Yu et al., 2023): An evaluation benchmark for large multimodal models focused on integrated vision-language capabilities, scored with an LLM-based evaluator for open-ended outputs. 

POPE (Li et al., 2023): A benchmark for object hallu-cination that uses polling-based Yes/No questions about object presence, and probes models under different negative-sampling strategies for stable hallucination measurement. 

D.3. Baseline Methods 

We compare with the following baseline methods: 

ECSO (Gou et al., 2024): A training-free safety wrapper for MLLMs. It first generates a candidate answer and checks whether it is unsafe. If unsafe, it converts the image into text and answers again using text-only input to recover the base LLM’s safety behavior. 

SIA (Na et al., 2025): A training-free, intent-aware safety pipeline. It captions the image to abstract visual details, then infers harmful intent from the image–text pair using few-shot CoT prompting. It uses the inferred intent to guide a safe response or refusal. 

TIS (Lou et al., 2025): It constructs a multimodal safety tuning set with explicit, long safety reasoning traces. The pipeline uses image captions plus safety guidelines to gener-ate safety-oriented reasoning for existing reference answers, filters low-quality traces, and then applies supervised fine-tuning on the resulting data. 

VLGuard (Zong et al., 2024): It curates a vision–language safe instruction-following dataset across harmful categories. It safety-fine-tunes VLLMs using this dataset, either mixed into standard VL instruction tuning or as post-hoc tuning. This reduces harmful outputs and mitigates safety forgetting caused by VL instruction tuning. 

SPA-VL (Zhang et al., 2025b): A preference-based safety alignment method that leverages a large-scale preference dataset with quadruples (question, image, chosen response, rejected response). It applies both DPO and PPO to improve harmlessness while maintaining helpfulness across diverse safety-related domains. 

SaFeR-VLM (Yi et al., 2025): A reinforcement learning framework that puts safety into the reasoning process. It uses safety-aware rollouts where unsafe trajectories are re-flected and corrected instead of discarded. It trains with structured rewards and optimizes with GRPO to reinforce safe and corrected trajectories.
