Abstract 

LLM-based shopping agents increasingly rely on long purchase histories and multi-turn inter-actions for personalization, yet naively append-ing raw history to prompts is often ineffective due to noise, length, and relevance mismatch. We propose M EM RERANK , a preference mem-ory framework that distills user purchase his-tory into concise, query-independent signals for personalized product reranking. To study this problem, we build an end-to-end bench-mark and evaluation framework centered on an LLM-based 1-in-5 selection task, which mea-sures both memory quality and downstream reranking utility. We further train the mem-ory extractor with reinforcement learning (RL), using downstream reranking performance as supervision. Experiments with two LLM-based rerankers show that M EM RERANK con-sistently outperforms no-memory, raw-history, and off-the-shelf memory baselines, yielding up to +10.61 absolute points in 1-in-5 accu-racy. These results suggest that explicit pref-erence memory is a practical and effective building block for personalization in agentic e-commerce systems. 

1 Introduction 

Recommender systems (RS) have long served as critical tools for guiding users through vast item collections in domains such as e-commerce, enter-tainment, and social media. Classical approaches, from collaborative filtering to deep neural mod-els, learn from historical interactions to predict a user’s preferences and rank candidates accordingly. While effective in specific contexts, they give users little way to inspect or correct what the system thinks they want. With the advent of large language models (LLMs), recommender systems are evolving be-yond static candidate lists toward agentic behaviour.

LLMs have progressed from passive text generators to agents capable of planning, remembering, and invoking tools, enabling interactive and adaptive recommendation scenarios (Sumers et al., 2024). Recent perspective papers on multi-agent recom-mender systems argue that the next generation of RS will be agentic : driven by a language core, a suite of tools, and a hierarchical memory that cap-tures short-term context, episodic experiences, and semantic knowledge (Maragheh and Deldjoo, 2025; Huang et al., 2025a). Within this framework, mem-ory is not merely a cache but a structured repos-itory: working memory retains the current turn, episodic memory stores past events, and seman-tic memory distils general user preferences. Such memory-augmented agents can recall prior recom-mendations and infer latent tastes even when the user does not explicitly state them. LLM-based recommendation agents must also integrate domain-specific knowledge and external tools. Recent systems, such as the Recommender AI Agent framework, position the LLM as the “brain” and conventional ranking modules as tools that consume long-term and short-term user profiles to personalise results(Huang et al., 2025c). These architectures enable interactive planning and reflec-tion, but note that naively injecting entire histories into the prompt introduces noise and quickly sat-urates the context window. Broader surveys on agentic recommender systems emphasise that bal-ancing autonomy with controllability, incorporat-ing external knowledge, and evaluating dynamic multimodal settings remain open challenges(Huang et al., 2025a). Despite the growing excitement around agentic recommender systems, simply appending purchase histories or multi-turn conversation transcripts to an LLM prompt often yields only marginal benefits and can even degrade performance. 2 Related Works 

2.1 LLM-based Rerankers 

LLM-based rerankers fall into three paradigms. 

Pointwise models compute a relevance score for each query–document pair using generative or dis-criminative scoring techniques(Ponte and Croft, 2017; Zhuang and Zuccon, 2021; Zhuang et al., 2021; Peng et al., 2024; Liang et al., 2023; Nogueira et al., 2020). Pairwise (and setwise) ap-proaches compare documents against each other and use sorting or sampling algorithms to aggregate wins into a final ranking(Qin et al., 2024; Gien-app et al., 2022; Mikhailiuk et al., 2020; Zhuang et al., 2024). Listwise methods produce a complete ranking in a single prompt and typically rely on zero- or few-shot prompting, sliding windows or tournament-style evaluation(Ma et al., 2023; Sun et al., 2023; Pradeep et al., 2023; Sun et al., 2023; Chen et al., 2025), while recent work fine-tunes LLMs via reinforcement learning to jointly gener-ate reasoning and ranking for complex tasks(Zhang et al., 2025; Su et al., 2025). We adopt a setwise reranker for efficiency (Peng et al., 2025) and fo-cus on improving its input via a concise preference memory rather than altering the ranker itself. 

2.2 Memory in Agentic Systems 

Memory is a core component of LLM-based agents. Cognitive architectures propose a hierarchy of working, episodic and semantic memories to en-able long-horizon reasoning and reduce halluci-nations(Sumers et al., 2024). General memory frameworks such as Mem0 (Chhikara et al., 2025) and MemAgent (Yu et al., 2025) implement these ideas for broad LLM applications: Mem0 acts as a universal memory layer that dynamically extracts, consolidates, and retrieves salient user informa-tion across sessions, while MemAgent equips an LLM with a fixed -length memory that is selec-tively updated via reinforcement learning when processing long text, summarising critical details for later reasoning. Agentic recommender frame-works such as MR.Rec (Huang et al., 2025b) trains the LLM via reinforcement learning to optimise both memory utilisation and recommendation rea-soning. Memory-R1 (Yan et al., 2025) adopts RL methods to further optimise memory controllers to decide what to store and retrieve. MAP (Chen, 2025) maintains external memory banks of user em-beddings for retrieval. Memento (Zhou et al., 2025) adapts memory modules without updating the base 2LLM weights. Long-term dialogue benchmarks re-veal that LLM agents struggle to recall information over hundreds of turns (Maharana et al., 2024), and recent work explores reflective or timeline-based memory management for Personalized dialogue agents (Tan et al., 2025; Li et al., 2025; Ong et al., 2025). Most of these systems aim to preserve co-herence or support general reasoning. In contrast, we design a preference memory tailored to down-stream product ranking. 

3 Methodology 

3.1 Dataset Construction 

We construct our benchmark from Amazon-Review-2023 and Amazon-C4 (Hou et al., 2024). The Amazon-Review-2023 dataset comprises over 570 million reviews and 48 million items from 33 categories spanning from May 1996 to Septem-ber 2023. For each review, the product metadata and the user ID are both available. The Amazon-C4 dataset is a product search dataset created by prompting ChatGPT to rewrite 5-star-rated user re-views from Amazon-Review-2023, each at least 100 words long, into complex contextual queries. For each rewritten query, the associated product is the positive product, and 50 negative products are sampled in the same category as the positive prod-uct. Amazon-C4 contains 21,223 queries and about one million products. We collect the purchase his-tory for each query q in Amazon-C4 from Amazon-Review-2023. Specifically, we first use the user ID to filter all products the user interacted with in Amazon-Review-2023, then restrict the products to those before the user’s search for the query q to obtain the user’s purchase history before q.Following MR.Rec (Huang et al., 2025b), we adopt the same query rewrite prompt as shown in Figure 6 to further rewrite Amazon-C4 queries with o3-mini to preserve the original shopping in-tent and constraints while removing unnecessary details, making them more suitable for a personal-ized product reranking setting. Following Amazon-C4, we use the product title and product descrip-tion to represent each product. For each query, we construct a candidate set containing one pos-itive product and four hard negatives. The four negatives are retrieved by Qwen3-8B-Embedding from the same category as the positive product, and we keep the top-4 negative products. This same-category hard-negative design makes the task challenging while keeping the candidate set con-trolled. Finally, each example is built around a target positive product, its associated query, 4 neg-ative products, and the user’s historical purchases. The task is to prompt the LLM to find the most relevant product among the 5 candidate products. This 1-in-5 task is the basic operation of the setwise reranking algorithm (Zhuang et al., 2024). Figure 12a shows the number of in-stances in each target category. The distri-bution is imbalanced, with some categories such as Industrial_and_Scientific ,

Clothing_Shoes_and_Jewelry , and 

Electronics containing substantially more examples than others. We choose Electronics as our main testbed because it has enough data for both training and evaluation. We split the data into training, development, and test sets with a ratio of 70:15:15, resulting in 905 training instances, 194 development instances, and 194 test instances. For each instance, we treat the user’s previously purchased products from the same category as the positive product as the within-category purchase history, and previously purchased products from different categories as the cross-category purchase history. Figure 12b reports the average lengths of within-category and cross-category purchase histo-ries. Across nearly all categories, cross-category history is much longer than within-category history. Based on these histories, our task is to generate query-independent shopping preference memory that can be reused across reranking instances. 

3.2 MEM RERANK Framework 

Our framework contains two stages: preference memory extraction and downstream reranking. Given a user’s purchase history, the memory ex-tractor, e.g., an LLM, generates a compact textual summary of stable shopping preferences. The in-put is purchase history, and the output is structured shopping preference memory enclosed in prede-fined tags. We study two extraction settings: (1) ex-tracting only within-category shopping preference memory, and (2) extracting both within-category and cross-category shopping preference memory. The extracted memory is then inserted into an LLM reranking prompt as shown in Figure 7 to-gether with the user query and the candidate prod-uct set. The reranker must select the most relevant product from five candidates. We use this 1-in-5 

task as the downstream training and evaluation ob-jective because it is the basic operation in setwise reranking: once the model can reliably choose the 3best item from a small set, the same operation can be repeatedly applied to larger candidate pools. In the current work, we focus only on this controlled 1-in-5 setting and do not yet study ranking more retrieved products end-to-end. Compared with directly appending raw purchase history into the reranking prompt, our framework explicitly compresses long and noisy historical be-havior into a concise, query-independent prefer-ence memory. 

3.3 RL Training with GRPO 

We train the memory extractor using GRPO, treat-ing preference memory generation as a policy opti-mization problem. For each training example, the model receives the user purchase history and gener-ates shopping preference memory. The generated memory is then scored by a reward function de-signed to encourage both valid formatting and the performance of the downsteaming 1-in-5 task. Our reward has two components. The first is a 

format reward , which checks whether the generated output follows the required structure, e.g., whether the memory is enclosed by the correct tags such as 

<within_memory> and </within_memory> . This reward encourages the model to produce well-formed outputs that can be reliably consumed by the reranker. The second is a reranking reward . We insert the generated memory into the downstream 1-in-5 reranking prompt and use an LLM reranker to select the best candidate product. The reward is positive if the reranker selects the ground-truth pos-itive product and negative otherwise. To make the reward smooth, we sample 5 responses and average the accuracy. In this way, the memory extractor is optimized not merely to generate fluent summaries, but to produce memories that are directly useful for personalized reranking. Formally, let x denote the purchase history, 

m ∼ πθ(· | x) denote the generated memory, and 

r(x, m ) denote the total reward. GRPO updates the extractor policy πθ to maximize expected reward:

The format term ensures struc-tural correctness, while the reranking term aligns the extracted memory with the downstream task objective. This training setup has two advantages. First, it avoids requiring gold preference-memory annota-tions, which are difficult and expensive to collect at scale. Second, it directly optimizes the extrac-tor for its actual role in the pipeline: helping the reranker pick the correct product under personal-ized constraints. 

4 Experimental Setup 

4.1 Baselines 

We compare our method against several types of baselines that differ in how user preference infor-mation is provided to the reranker. 

No memory. This setting performs direct per-sonalized product reranking without any explicit extracted preference memory. As shown in Fig-ure 7, we input the empty <within_memory> and 

</cross_memory> , so the reranker receives only the query and candidate products and serves as the primary reference point for assessing the benefit of memory augmentation. 

Product-context baselines. We first consider simple retrieval-based context augmentation base-lines. Within-category products augments the reranker with additional products sampled from the same category as the target candidate set, aim-ing to provide local preference evidence. Within + cross-category products further includes products from other categories, allowing the reranker to ac-cess broader behavioral signals beyond the current category. 

LLM-extracted memory baselines. We com-pare against preference memories directly extracted by off-the-shelf LLMs. Base-extractor memory 

uses the base memory extractor, Qwen2.5-7B-Instruct, with the original extraction prompt but without any RL training. GPT-5.2 memory re-places the extractor with GPT-5.2, testing whether a stronger proprietary LLM can produce more useful preference memory under the same general setup. For both settings, we also evaluate a think tag vari-ant, which prompts the reranker to explicitly reason before making the final decision. 

External memory baselines. We further com-pare with two recently published strong memory-based methods, MR.Rec and Mem0 . These meth-ods provide alternative ways of constructing mem-ory, allowing us to compare our approach against prior memory-centric personalization strategies. 4For MR.Rec , we first adopt the prompt, as shown in Figure 4, to extract the shopping preference for each category and then apply the prompt, as shown in Figure 4 to summarize all category-wise shopping preferences into a user profile. 4.2 Implementation Details 

We consider two memory extraction settings: ex-tracting only within-category shopping preference memory, and extracting both within- and cross-category shopping preference memory. The maxi-mum generation length of the memory extractor is set to 512 tokens for the former and 1024 tokens for the latter. For GPT-4.1-mini reranking, we use tempera-ture=0.0 and top_p=1.0. Under standard reranking, the maximum output length is 64 tokens. When think tags are enabled for interpretability, the maxi-mum output length is increased to 1024 tokens for within-category shopping preference memory and 2048 tokens for within- and cross-category shop-ping preference memory. For o4-mini reranking, we use temperature=1.0, top_p=1.0, and reasoning_effort=low. Under stan-dard reranking, the maximum output length is 512 tokens for within-category shopping preference memory and 1024 tokens for within- and cross-category shopping preference memory. With think tags enabled, the maximum output length is set to 1024 and 2048 tokens, respectively. Overall, think tags require larger output budgets to accommodate explicit reasoning, and within-and cross-category shopping preference memory requires a larger token budget than within-category shopping preference memory because it contains richer and longer preference information.

5.1 Main Results 

Table 1 reports the main results on personalized product reranking in the Electronics category. Overall, our method consistently achieves the best performance across both rerankers, and adding the think tag further improves the results. For GPT-4.1-mini, the no-memory baseline achieves 32.47 accuracy. Adding raw product context yields only limited gains: within-category products improve accuracy by +0.11, and adding both within- and cross-category products improves it by +1.55. In contrast, our RL-trained memory extractor improves accuracy to 38.14 (+5.67), and further to 39.07 (+6.60) with the think tag. This shows that distilled preference memory is substan-tially more effective than directly appending raw product context. For o4-mini, the same trend holds but with larger gains. Our method also outperforms external memory baselines. MR.Rec and Mem0 are weak for GPT-4.1-mini, and while they provide moderate gains for o4-mini, they still trail our method by a large margin. 6 Conclusion 

We present a memory-based framework for per-sonalized product reranking. Instead of directly feeding long raw purchase histories to the reranker, we first extract query-independent shopping prefer-ence memory from user purchase metadata, and then use this memory to support product selec-tion. Experiments show that extracted preference memory is consistently more effective than direct reranking, raw product-context augmentation, and external memory baselines. Our RL-trained mem-ory extractor further outperforms both the base extractor and GPT-5.2-based extraction. We also find that prompt design matters: a semi-structured, evidence-grounded extraction prompt works better than either a rigid fixed-aspect prompt or a fully unconstrained prompt. In addition, think tags fur-ther improve performance, indicating that better memory and explicit reasoning are complementary. 8Overall, the results suggest that explicit shopping preference memory is a promising way to convert historical user behavior into actionable signals for personalized e-commerce search. 

Limitations 

This work has two main limitations. First, we eval-uate only on the Electronics category and use only product metadata for memory extraction, so generalization to other domains and richer signals remains unclear. Second, we focus on offline 1-in-5 reranking and do not study larger-scale retrieval settings, like reranking the top 100 retrieved prod-ucts.
