## 摘要
开放式文本生成的强化学习受限于缺乏可验证的奖励，这使得模型必须依赖于需要标注数据或强大闭源模型的评判模型。受近期利用基于置信度的内生奖励进行数学推理的无监督强化学习研究启发，我们探讨了这一原理是否可以应用于开放式写作任务。我们发现，直接应用置信度奖励会导致“平庸性偏差”（Triviality Bias）：策略会向高概率输出坍缩，从而降低了多样性和有意义的内容。为此，我们提出了 TCER（Triviality Corrected Endogenous Reward，修正平庸性的内生奖励），它通过奖励专家策略与通用参考策略之间的相对信息增益来解决这一偏差，并辅以一种依赖于概率的修正机制进行调节。在多个写作基准和模型架构上的实验表明，TCER 在无需外部监督的情况下实现了持续的性能提升。此外，TCER 还能有效迁移至数学推理任务，验证了该方法在不同生成任务中的通用性。

## Abstract
Reinforcement learning for open-ended text generation is constrained by the lack of verifiable rewards, necessitating reliance on judge models that require either annotated data or powerful closed-source models. Inspired by recent work on unsupervised reinforcement learning for mathematical reasoning using confidence-based endogenous rewards, we investigate whether this principle can be adapted to open-ended writing tasks. We find that directly applying confidence rewards leads to Triviality Bias: the policy collapses toward high-probability outputs, reducing diversity and meaningful content. We propose TCER (Triviality Corrected Endogenous Reward), which addresses this bias by rewarding the relative information gain between a specialist policy and a generalist reference policy, modulated by a probability-dependent correction mechanism. Across multiple writing benchmarks and model architectures, TCER achieves consistent improvements without external supervision. Furthermore, TCER also transfers effectively to mathematical reasoning, validating the generality of our approach across different generation tasks.

*   **核心问题**：在开放式文本生成（如创意写作）中，缺乏像数学或代码那样客观、可自动验证的奖励信号（Verifiable Rewards）。
*   **研究背景**：目前的强化学习（RL）主要依赖“LLM-as-a-Judge”（如 GPT-4 评分），但这面临 API 成本高昂、标注数据稀缺以及人类主观偏见等问题。

### 2. 论文提出的方法论：TCER
TCER 的核心思想是通过衡量“专家策略”相对于“通用策略”的相对信息增益，并对高概率 Token 进行抑制，从而鼓励生成更有意义的内容。

*   **核心公式**：
    奖励函数定义为：$r_t = \log \pi_s + k \cdot w \cdot \phi$
    *   **$\pi_s$ (Specialist)**：在高质量领域数据上 SFT 后的专家模型。
    *   **$\pi_b$ (Base)**：未经过领域微调的通用参考模型。
    *   **$\phi$ (Information Gain)**：定义为 $\log(\pi_s / \pi_b)$，即专家模型相对于通用模型的对数似然比，突出领域特有的知识。
    *   **$w$ (Gating Weight)**：定义为 $(1 - \pi_s)^\lambda$，这是一个校正项。当模型对某个 Token 的预测概率极高时，$w$ 趋近于 0，从而抑制平庸内容的奖励，迫使模型探索更有信息量的表达。
*   **算法流程**：
    1.  **SFT 阶段**：在高质量数据上训练得到专家模型 $\pi_s$。
    2.  **RL 阶段**：使用 **Reference Augmented GRPO** 算法。在 GRPO 的采样组中加入 Ground Truth（参考答案）来稳定优势估计（Advantage Estimation）。
    3.  **奖励计算**：在 Token 级别计算 TCER 奖励，并进行序列长度归一化，作为强化学习的反馈信号。

### 7. 优点
*   **无需外部监督**：完全摆脱了对昂贵闭源模型 API 或人工标注的依赖，实现了真正的无监督强化学习。
*   **理论支撑**：通过信息论分解（将置信度分解为通用知识和领域增益）来解决平庸性问题，逻辑严密。
*   **即插即用**：可以作为现有 SFT 模型之后的后训练（Post-training）增强手段，且适用于多种架构。

### 8. 不足与局限
*   **性能天花板**：作为一种自监督方法，其上限受限于基础模型（Base）和专家模型（SFT）本身蕴含的知识，无法凭空产生模型完全未见过的知识。

*   **解释性挑战**：尽管在 Token 级别计算，但自然语言的复杂依赖性使得很难精确解释为什么某些特定表达被判定为“非平凡”的。
*   **超参数敏感性**：$k$ 和 $\lambda$ 的选择对结果有影响，虽然论文给出了推荐值，但在不同任务中可能需要微调。
