## 摘要
在开放世界任务（即完成边界未预先给出的任务）中运行的自主智能体面临“分母盲区”问题：它们会系统性地低估目标空间的范围。Forage V1 通过协同演化评估（由独立的评估器发现“完成”的含义）和方法隔离（评估器与规划器无法看到彼此的代码）解决了这一问题。V2 将该架构从单次任务扩展为学习型组织：经验在多次运行中积累，在不同能力的模型间迁移，且制度性保障可防止知识退化。我们在三种任务类型（网页抓取、API 查询、数学推理）中验证了两个主张。知识积累：在六次运行中，知识条目从 0 增长到 54，且随着领域理解的加深，分母估计趋于稳定。知识迁移：预置了更强智能体（Opus）知识的较弱智能体（Sonnet）将 6.6 个百分点的覆盖率差距缩小至 1.1 个百分点，成本减半（从 9.40 美元降至 5.13 美元），收敛轮数减半（平均 4.5 轮对比 7.0 轮），且三次独立的预置运行得出了完全相同的分母估计（266），这表明组织知识能够校准评估本身。V2 的贡献在于架构层面：它设计了诸如审计分离、契约协议和组织记忆等制度，使任何智能体在加入时都更加可靠。积累的经验是组织性的、模型无关的且可迁移的，以可读文档的形式存储，任何未来的智能体无论其供应商或能力水平如何均可继承。

## Abstract
Autonomous agents operating in open-world tasks -- where the completion boundary is not given in advance -- face denominator blindness: they systematically underestimate the scope of the target space. Forage V1 addressed this through co-evolving evaluation (an independent Evaluator discovers what "complete" means) and method isolation (Evaluator and Planner cannot see each other's code). V2 extends the architecture from a single expedition to a learning organization: experience accumulates across runs, transfers across model capabilities, and institutional safeguards prevent knowledge degradation.   We demonstrate two claims across three task types (web scraping, API queries, mathematical reasoning). Knowledge accumulation: over six runs, knowledge entries grow from 0 to 54, and denominator estimates stabilize as domain understanding deepens. Knowledge transfer: a weaker agent (Sonnet) seeded with a stronger agent's (Opus) knowledge narrows a 6.6pp coverage gap to 1.1pp, halves cost (9.40 to 5.13 USD), converges in half the rounds (mean 4.5 vs. 7.0), and three independent seeded runs arrive at exactly the same denominator estimate (266), suggesting organizational knowledge calibrates evaluation itself.   V2's contribution is architectural: it designs institutions -- audit separation, contract protocols, organizational memory -- that make any agent more reliable upon entry. The accumulated experience is organizational, model-agnostic, and transferable, stored as readable documents that any future agent inherits regardless of provider or capability level.

### Forage V2: 自主智能体组织中的知识演化与迁移

#### 1. 核心问题与整体含义
*   **研究动机**：自主智能体在处理“开放世界任务”（即没有预设完成边界的任务）时，普遍存在**“分母盲区”（Denominator Blindness）**。智能体往往会低估任务的总规模（分母），导致其在仅完成一小部分工作时就错误地汇报 100% 的覆盖率。
*   **核心目标**：Forage V2 旨在将 V1 的单次任务架构扩展为**“学习型组织”**。它不仅要解决单次运行中的评估失准问题，还要实现经验在多次运行中的积累，以及从强模型（如 Claude Opus）向弱模型（如 Claude Sonnet）的知识迁移。

#### 2. 方法论
*   **核心思想**：借鉴人类制度设计（如审计分离、合同协议、组织记忆），而非单纯提升算法性能。
*   **关键技术细节**：
    *   **协同演化评估（Co-evolving Evaluation）**：由独立的**评估器（Evaluator）**发现任务边界并编写 `eval.py`，**规划器（Planner）**执行任务并编写 `action.py`。两者在运行中同步进化。
    *   **物理隔离（Hardened Isolation）**：评估器和规划器运行在完全独立的物理工作空间，互不可见对方的代码，防止认知锚定和自我服务式评估。
    *   **知识演化（Knowledge Evolution）**：任务结束后进行“事后分析”（Post-mortem），将经验提取为结构化的 Markdown 文档（包含领域知识、源可靠性、策略建议等）。
    *   **知识迁移协议**：将强模型生成的知识库直接注入弱模型的系统提示词中，使其能够跳过探索阶段，直接采用成熟策略。
    *   **评估合同 (`eval_contract.md`)**：作为两个智能体之间唯一的公共接口，规定输出格式和质量标准。

    *   **Opus 组**：从零知识开始，连续运行 6 次以积累知识。
    *   **Sonnet Cold 组**：无先验知识的弱模型基准。

#### 6. 主要结论与发现
*   **知识积累效应**：在 NVIDIA 任务中，知识条目从 0 增长到 54 条，评估器的分母估计随着领域理解的加深而趋于稳定。

*   **能力边界**：在极难任务（Q6）中，知识能提高智能体的“生存率”（产生输出的概率），但无法突破模型本身的推理极限。

#### 7. 优点
*   **制度化设计**：通过结构约束而非微调模型来解决可靠性问题，具有极强的可解释性。
*   **模型无关性**：积累的知识是人类可读的文档，可以跨供应商、跨能力的模型进行无缝迁移。
*   **鲁棒的隔离机制**：从 V1 的文件名隐藏升级到 V2 的物理目录隔离，彻底杜绝了智能体窥探评估标准的风险。

#### 8. 不足与局限
*   **静态环境限制**：实验主要针对静态知识空间，尚未在动态变化（如实时新闻或市场数据）的环境中验证。
*   **样本量限制**：作为独立研究者的工作，实验重复次数（n=6）虽然足以说明趋势，但在统计严密性上仍有提升空间。
*   **知识污染风险**：如果早期运行产生了错误的“经验”，可能会在组织记忆中传播，目前缺乏自动化的知识纠错/清洗机制。
*   **上下文压力**：长周期的推理任务（如数学证明）容易导致上下文窗口迅速耗尽，限制了复杂任务的深度。
