## 摘要
函数调用使大语言模型（LLMs）能够与外部工具交互，但现有的基于强化学习（RL）的方法在推理过程与工具调用决策之间存在失配问题。我们提出了 R2IF，一种用于可解释函数调用的推理感知强化学习框架。该框架采用了一种复合奖励机制，整合了格式/正确性约束、思维链有效性奖励（CER）以及规范-修改-值（SMV）奖励，并利用 GRPO 进行优化。在 BFCL 和 ACEBench 上的实验表明，R2IF 的性能优于基准模型，提升幅度高达 34.62%（在 BFCL 上的 Llama3.2-3B），并获得了正向的平均思维链有效性（Llama3.2-3B 为 0.05），同时增强了函数调用的准确性和可解释性，为可靠的工具增强型大语言模型部署提供了支持。

## Abstract
Function calling empowers large language models (LLMs) to interface with external tools, yet existing RL-based approaches suffer from misalignment between reasoning processes and tool-call decisions. We propose R2IF, a reasoning-aware RL framework for interpretable function calling, adopting a composite reward integrating format/correctness constraints, Chain-of-Thought Effectiveness Reward (CER), and Specification-Modification-Value (SMV) reward, optimized via GRPO. Experiments on BFCL/ACEBench show R2IF outperforms baselines by up to 34.62% (Llama3.2-3B on BFCL) with positive Average CoT Effectiveness (0.05 for Llama3.2-3B), enhancing both function-calling accuracy and interpretability for reliable tool-augmented LLM deployment.

### 1. 核心问题与研究动机
*   **核心问题**：大语言模型（LLM）在执行函数调用（Function Calling）时，其**推理过程（Reasoning）与最终决策（Tool-call Decisions）之间存在严重的“失配”现象**。
*   **研究动机**：
    *   现有的强化学习（RL）方法多为结果导向（Outcome-driven），仅根据最终调用的正确性（如 AST 匹配）给予奖励。
    *   这导致模型生成的思维链（CoT）往往只是“事后解释”，并未真正指导参数构造或工具选择，导致模型在面对复杂参数格式或未见过的查询时鲁棒性差、不可解释且难以诊断错误。
    *   **目标**：建立一个推理感知的 RL 框架，使推理过程真正支持参数规范、修改和实例化，提升函数调用的准确性与可解释性。

### 2. 方法论
R2IF 提出了一种推理感知的强化学习框架，核心在于**复合奖励函数设计**，并使用 **GRPO（Grouped Proximal Policy Optimization）** 算法进行优化。

*   **核心思想**：通过三种维度的奖励信号，强制模型在推理阶段就关注工具的规范和参数的转化逻辑。
*   **关键技术细节（复合奖励 $R$）**：
    1.  **二元奖励 ($R_{binary}$)**：硬性约束。包括格式奖励（是否包含正确的 `<reason>` 和 `<tool>` 标签）和正确性奖励（工具名及参数是否与 Ground Truth 完全一致）。
    2.  **思维链有效性奖励 ($R_{cer}$)**：量化推理的“效用”。通过一个“学生模型”在给定当前推理前缀下的成功率提升（相对于无推理前缀的基准）来计算奖励，确保推理能引导正确的决策。
    3.  **规范-修改-值奖励 ($R_{smv}$)**：细粒度监督。
        *   **Specification (规范)**：推理是否识别了参数的特殊格式要求（如日期格式、单位）。
        *   **Modification (修改)**：推理是否描述了从原始查询到最终参数值的转换逻辑。
        *   **Value (值)**：最终参数值是否正确实例化。
*   **算法流程**：先进行 SFT 预热，随后在平衡后的数据集上利用 GRPO 进行大规模强化学习，通过群体采样计算优势函数（Advantage），避免了传统 PPO 对 Critic 网络的依赖。

### 3. 实验设计
*   **数据集/场景**：
    *   **BFCL (Berkeley Function Calling Leaderboard)**：行业标准的函数调用评测集，包含 Non-live（静态）和 Live（实时更新）场景。
    *   **ACEBench**：更全面的工具使用基准，涵盖原子任务、单轮任务和无关拒绝任务。
*   **基准模型（Backbones）**：Qwen2.5 (1.5B/3B/7B) 和 Llama3.2-3B。
*   **对比方法**：
    *   Raw Instruct（原始指令微调模型）。
    *   SFT（使用 GPT-4o 蒸馏的推理数据进行微调）。
    *   Binary Reward（仅使用硬性正确性奖励的 GRPO）。
    *   ToolRL（现有的针对工具调用的 RL 方法）。

### 4. 资源与算力
*   **算力设备**：使用了 **6 台 NVIDIA RTX PRO 6000 GPU**。
*   **训练框架**：使用 **Verl** 框架进行 GRPO 训练。
*   **训练参数**：训练 9 个 Epoch，Batch Size 为 512，学习率为 $1 \times 10^{-6}$。
*   **开销分析**：引入 CER 奖励使训练时间增加了约 2.9% - 11.4%，推理时的 Token 长度略有增加（为了更详尽的推理），但整体受控。

### 5. 实验数量与充分性
*   **实验规模**：在 4 个不同规模和家族的模型上进行了验证，涵盖了从 1.5B 到 7B 的参数区间。
*   **消融实验**：详细测试了去掉 SMV、CER 和 SFT 预热后的性能变化，证明了各组件的必要性。
*   **多样性测试**：包含了多轮对话案例研究、人类评估（针对可解释性的四个维度：可预测性、可理解性、解释性、忠实度）以及针对 CER 采样鲁棒性的分析。
*   **评价**：实验设计非常充分且客观，不仅关注准确率，还引入了 ACE（平均 CoT 有效性）指标来量化推理质量，对比基准涵盖了当前主流的 RL 方案。

### 6. 主要结论与发现
*   **性能飞跃**：R2IF 在 BFCL 上将 Llama3.2-3B 的准确率提升了 **34.62%**，在 ACEBench 上将 Qwen2.5-7B 提升了 **15.4%**。
*   **推理对齐**：R2IF 是少数能让 **ACE 指标转正**的方法（如 Llama3.2-3B 达到 0.05），而传统方法该指标通常为负，说明 R2IF 的推理过程真正对决策产生了正面贡献。
*   **规模效应**：性能随模型规模增大单调提升，且在不同模型家族间具有良好的泛化性。
*   **鲁棒性**：在处理“无关拒绝”（Irrelevance）任务时保持了极高的准确率（≥96%），证明模型没有因为强化学习而变得过度“激进”。

### 7. 优点
*   **创新性奖励设计**：CER 和 SMV 奖励解决了 RL 训练中“过程监督”缺失的问题，使模型不仅“做对”而且“想对”。
*   **客观评估推理**：通过学生模型性能增量来评估推理有效性，避开了 LLM-as-a-judge 的主观偏差。
*   **显著的可解释性提升**：人类评估显示 R2IF 在推理轨迹的忠实度和逻辑性上远超基准模型。

### 8. 不足与局限
*   **任务局限性**：主要针对单轮函数调用任务，虽然附录有少量多轮测试，但其在复杂多轮交互中的长期推理一致性仍需进一步验证。
*   **奖励设计成本**：SMV 奖励依赖于预先生成的参数规范文档，这在扩展到拥有数千个 API 的超大规模工具库时，准备 Ground Truth 文档的成本较高。
*   **推理冗余**：为了对齐决策，模型生成的推理 Token 较多，可能会增加实际部署时的推理延迟。
