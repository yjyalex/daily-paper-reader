## 摘要
智能体检索增强生成 (Agentic RAG) 系统使大语言模型 (LLM) 能够通过与外部检索工具的多步交互来解决复杂任务。然而，这种多步交互通常涉及冗余的搜索步骤，从而产生高昂的计算成本和延迟。先前的研究通过限制搜索深度（即搜索步骤的数量）来降低成本，但这往往导致对复杂问题的探索不足。为解决这一问题，我们首先研究了搜索深度如何影响准确性，并发现了一个定义准确性-效率权衡的最小充分搜索深度，该深度由问题复杂度和智能体能力共同决定。此外，我们提出了 AutoSearch，这是一个强化学习 (RL) 框架，它通过自生成的中间答案来评估每个搜索步骤。通过自回答机制，AutoSearch 能够识别最小充分搜索深度，并通过奖励达到该深度、惩罚过度搜索来促进高效搜索。此外，我们还引入了奖励机制以稳定搜索行为并提高复杂问题的回答质量。在多个基准测试上的广泛实验表明，AutoSearch 实现了卓越的准确性-效率权衡，在保持搜索质量的同时缓解了过度搜索。

## Abstract
Agentic retrieval-augmented generation (RAG) systems enable large language models (LLMs) to solve complex tasks through multi-step interaction with external retrieval tools. However, such multi-step interaction often involves redundant search steps, incurring substantial computational cost and latency. Prior work limits search depth (i.e., the number of search steps) to reduce cost, but this often leads to underexploration of complex questions. To address this, we first investigate how search depth affects accuracy and find a minimal sufficient search depth that defines an accuracy-efficiency trade-off, jointly determined by question complexity and the agent's capability. Furthermore, we propose AutoSearch, a reinforcement learning (RL) framework that evaluates each search step via self-generated intermediate answers. By a self-answering mechanism, AutoSearch identifies the minimal sufficient search depth and promotes efficient search by rewarding its attainment while penalizing over-searching. In addition, reward mechanisms are introduced to stabilize search behavior and improve answer quality on complex questions. Extensive experiments on multiple benchmarks show that AutoSearch achieves a superior accuracy-efficiency trade-off, alleviating over-searching while preserving search quality.

*   **搜索不足（Under-exploration）：** 为了省钱而限制搜索步数，导致复杂的多跳问题无法获得足够证据。

### 2. 方法论：核心思想与关键技术
作者提出了 **AutoSearch** 框架，其核心思想是利用强化学习（RL）训练智能体，使其在每一步搜索后评估是否已获得足够信息。

*   **自回答机制（Self-answering Mechanism）：** 在每一个检索步骤 $t$，智能体不仅生成搜索查询，还会尝试生成一个“中间答案” $a_{inter}$。
*   **能力感知最优深度 ($t_c$)：** 定义 $t_c$ 为智能体能够首次正确回答问题的最早步骤。
*   **奖励函数设计 ($R_{overall}$)：**
    1.  **基础奖励 ($r_{base}$)：** 包含格式奖励（确保输出结构正确）和结果奖励（最终答案正确给予高分）。
    2.  **搜索效率奖励 ($r_{efficient}$)：** 
        *   如果当前步 $t \le t_c$，给予正向奖励以鼓励有效搜索。
        *   如果 $t > t_c$，则视为过度搜索，给予负向惩罚。
        *   通过公式设计，使累积奖励在 $t = t_c$ 时达到峰值。
    3.  **搜索质量奖励 ($r_{quality}$)：** 计算中间答案与标准答案的 F1 分数增益，奖励那些能显著提升信息质量的搜索步。
*   **算法流程：** 使用 PPO（近端策略优化）作为主要的强化学习算法，同时验证了 GRPO（群体相对策略优化）的适用性。

*   **实验规模：** 论文在 3B 和 7B 两种规模的 Qwen2.5 模型上进行了验证，并在 6 个不同难度的任务上进行了测试。

### 6. 主要结论与发现
*   **自适应性：** AutoSearch 能够根据题目难度自动调整步数。在单跳任务上保持低深度（约 1 步），在多跳任务上自动增加深度（约 1.5-1.8 步）。
*   **效率提升：** 在保持或超过基线模型准确率的同时，显著降低了搜索深度和过度搜索比例（OSR）。
*   **性能领先：** 在 3B 和 7B 模型上，AutoSearch 的搜索效率（SE）均大幅领先于 Search-R1 和 StepSearch。
*   **算法通用性：** 实验证明该框架不仅适用于 PPO，在 GRPO 下也能取得相似的优异表现。

### 7. 优点（亮点）
*   **无需外部监督：** 与 HiPRAG 不同，AutoSearch 不需要昂贵的外部大模型来判断搜索必要性，而是依靠智能体自身的“中间答案”进行自评估。
*   **闭环反馈：** 将“能力”与“深度”挂钩，模型在训练过程中随着能力的提升，会自动学会用更少的步数解决问题。
*   **细粒度控制：** 奖励函数不仅关注结果，还通过 F1 增益关注每一步搜索的边际贡献。

### 8. 不足与局限
*   **搜索深度上限：** 实验主要集中在较低的搜索深度（最大 4 步），对于需要极深推理链（如 10 步以上）的超复杂任务，该方法的稳定性尚未验证。
*   **中间答案的开销：** 虽然减少了搜索步数，但在训练和推理过程中生成“中间答案”会带来额外的 Token 生成开销（尽管作者在分析中指出总 Token 长度最终是下降的）。
*   **检索器依赖：** 性能仍受限于底层检索器（如 E5）和文档库的质量，若检索器召回率极低，自适应机制可能会失效。
