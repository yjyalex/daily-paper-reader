## 摘要
知识蒸馏 (KD) 是压缩大语言模型 (LLMs) 的一种强大范式，其有效性取决于散度方向、优化策略和数据方案之间相互关联的选择。我们拆解了现有 KD 方法的设计，并提出了一个建立它们之间联系的统一视角，将 KD 重新表述为 token 级别的重加权对数似然目标。我们进一步提出了混合策略蒸馏 (HPD)，它整合了前向和反向 KL 散度的互补优势，以平衡模式覆盖 (mode coverage) 和模式寻求 (mode-seeking)，并将离策 (off-policy) 数据与轻量级、近似的在策 (on-policy) 采样相结合。我们在长生成数学推理以及短生成对话和代码任务上验证了 HPD，证明了在不同模型家族和规模上，该方法在优化稳定性、计算效率和最终性能方面均有提升。与本工作相关的代码可在 https://github.com/zwhong714/Hybrid-Policy-Distillation 获取。

## Abstract
Knowledge distillation (KD) is a powerful paradigm for compressing large language models (LLMs), whose effectiveness depends on intertwined choices of divergence direction, optimization strategy, and data regime. We break down the design of existing KD methods and present a unified view that establishes connections between them, reformulating KD as a reweighted log-likelihood objective at the token level. We further propose Hybrid Policy Distillation (HPD), which integrates the complementary advantages of forward and reverse KL to balance mode coverage and mode-seeking, and combines off-policy data with lightweight, approximate on-policy sampling. We validate HPD on long-generation math reasoning as well as short-generation dialogue and code tasks, demonstrating improved optimization stability, computational efficiency, and final performance across diverse model families and scales. The code related to this work is available at https://github.com/zwhong714/Hybrid-Policy-Distillation.

### 1. 核心问题与研究动机
*   **核心问题**：如何高效且稳定地将大型教师模型的知识迁移到轻量级学生模型中。
*   **研究动机**：
    *   **设计维度复杂**：现有的知识蒸馏（KD）方法在散度方向（前向 vs. 反向 KL）、优化策略（Loss vs. Reward）和数据方案（在策 On-policy vs. 离策 Off-policy）上存在多种选择，且这些维度相互交织，缺乏统一框架。
    *   **现有方法缺陷**：前向 KL（FKLD）虽能覆盖模式但易导致预测过度平滑；反向 KL（RKLD）虽能寻求模式但训练不稳定且易发生熵坍塌；在策采样（OPD）虽然效果好但计算开销巨大。
    *   **目标**：建立一个统一视角，并提出一种既能平衡模式覆盖与寻求，又能兼顾效率与稳定性的蒸馏方法。

### 2. 方法论：混合策略蒸馏 (HPD)
*   **统一视角**：论文将 SFT、FKLD 和 RKLD 统一表述为**token 级别的重加权对数似然目标**。通过调整每个 token 的权重 $w(a_t|s_t)$，可以实现不同的蒸馏效果。
*   **核心思想**：HPD 在 token 级别混合了前向和反向 KL 信号，并结合了离策数据与轻量级的在策采样。
*   **关键技术细节**：
    *   **负 $K_1$ 估计器**：利用 $K_1$ 估计器计算学生与教师在特定 token 上的概率差距。
    *   **专家 Token 权重 ($w^*_t$)**：针对离策数据中的专家 token，若学生低估了它（$k_1 > 0$），则结合 FKLD 权重进行强化；若学生已高估，则通过 RKLD 信号抑制过度学习。
    *   **采样 Token 权重 ($w_t$)**：让学生在当前状态下进行轻量级采样（非专家 token），若采样到的 token 被教师认为是不合理的（$k'_1 < 0$），则施加负权重予以抑制。
    *   **强化操作 (Reinforce)**：当一个不合理的采样 token 被抑制时，HPD 会加倍专家 token 的权重，将抑制掉的概率质量显式地引导至专家 token 上。
*   **算法流程**：输入学生、教师模型及数据集 $\rightarrow$ 采样离策轨迹 $\rightarrow$ 计算专家 token 的 KL 差距 $\rightarrow$ 学生进行单步采样并计算采样 token 的 KL 差距 $\rightarrow$ 根据混合逻辑更新权重 $\rightarrow$ 执行梯度下降。

### 6. 主要结论与发现
*   **性能卓越**：HPD 显著提升了小模型的性能。例如，Qwen 2.5 3B 经过 HPD 蒸馏后，数学推理能力提升了 41%，接近甚至超过了某些 7B/8B 规模的模型。

*   **训练-推理对齐**：HPD 减少了训练和推理之间的行为偏差，提高了模型在长序列生成中的稳定性。
*   **强力初始化**：HPD 产出的模型是进行后续在策蒸馏（OPD）或偏好对齐（DPO）的极佳起点。

### 7. 优点
*   **理论统一**：提供了一个简洁的重加权框架，统一了多种看似不同的蒸馏算法。
*   **效率与效果平衡**：通过单步采样而非全序列 rollout，实现了接近在策蒸馏的效果，但计算成本远低于后者。
*   **无需超参数调节**：HPD 方案在公式设计上自然融合了不同信号，实验中无需针对不同任务繁琐地调整超参数。

### 8. 不足与局限
*   **分词器限制**：要求教师和学生模型必须共享相同的分词器（Tokenizer），这限制了它在异构模型间的应用。
