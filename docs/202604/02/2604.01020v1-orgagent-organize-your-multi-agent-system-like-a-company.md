## 摘要
尽管基于大语言模型的多智能体系统在复杂推理方面展现出巨大潜力，但如何有效地组织多个智能体仍是一个悬而未决的问题。在本文中，我们介绍了 OrgAgent，这是一种公司式的分层多智能体框架，将协作划分为治理层、执行层和合规层。OrgAgent 将多智能体推理分解为三个层面：用于规划和资源分配的治理层、用于任务解决和审查的执行层，以及用于最终答案控制的合规层。通过在推理任务、大语言模型（LLMs）、执行模式和执行策略上评估该框架，我们发现以公司式分层结构组织的多智能体系统通常优于其他组织结构。此外，在大多数设置下，分层协调相比于扁平化协作还降低了 Token 消耗。例如，对于 GPT-OSS-120B，在 SQuAD 2.0 数据集上，分层设置相比扁平化多智能体系统性能提升了 102.73%，同时 Token 使用量减少了 74.52%。进一步分析表明，当任务受益于稳定的技能分配、受控的信息流和分层验证时，分层结构的帮助最大。总之，我们的研究结果强调了组织结构是多智能体推理中的一个重要因素，它不仅影响有效性和成本，还塑造了协调行为。

## Abstract
While large language model-based multi-agent systems have shown strong potential for complex reasoning, how to effectively organize multiple agents remains an open question. In this paper, we introduce OrgAgent, a company-style hierarchical multi-agent framework that separates collaboration into governance, execution, and compliance layers. OrgAgent decomposes multi-agent reasoning into three layers: a governance layer for planning and resource allocation, an execution layer for task solving and review, and a compliance layer for final answer control. By evaluating the framework across reasoning tasks, LLMs, execution modes, and execution policies, we find that multi-agent systems organized in a company-style hierarchy generally outperform other organizational structures. Besides, hierarchical coordination also reduces token consumption relative to flat collaboration in most settings. For example, for GPT-OSS-120B, the hierarchical setting improves performance over flat multi-agent system by 102.73% while reducing token usage by 74.52% on SQuAD 2.0. Further analysis shows that hierarchy helps most when tasks benefit from stable skill assignment, controlled information flow, and layered verification. Overall, our findings highlight organizational structure as an important factor in multi-agent reasoning, shaping not only effectiveness and cost, but also coordination behavior.

#### 2. 方法论：核心思想与关键技术
OrgAgent 提出了一个仿公司制的三层分层架构，将推理过程解耦：

*   **核心思想**：将多智能体协作划分为治理、执行和合规三个垂直维度，实现权责明确、流程受控。
*   **关键技术细节（三层架构）**：
    *   **治理层 (Layer A - Governance)**：由 CEO（战略）、CTO（技术方案）和 COO（资源效率）组成。负责根据任务风险和复杂度进行高层规划、分配技能角色并设定执行预算。
    *   **执行层 (Layer B - Execution)**：包含 Drafter（起草者）、Reviewer（审核者）和 Specialist（领域专家）。基于治理层的指令进行迭代式的答案生成与修正。
    *   **合规层 (Layer C - Compliance)**：由 CSO（方案官）和 CCO（合规官）组成。负责将执行结果转化为特定 Benchmark 要求的格式，并进行最终的结构化合规检查。
*   **技能池与策略**：
    *   **技能型工人池**：提供技术、定量、推理、领域、沟通、数据六大专业导向。
    *   **执行模式**：支持 DIRECT（直接生成）、LIGHT MAS（轻量协作）和 FULL MAS（深度协作）。
    *   **执行策略**：提供 STRICT（严控成本）、BALANCE（平衡）、NOCAP（不设限）和 AUTO（自动选择）四种策略。

*   **层级结构胜过扁平结构**：在大多数推理任务中，层级化的 OrgAgent 性能显著优于扁平化 MAS。在 SQuAD 2.0 上，性能提升甚至超过 100%。
*   **显著的降本增效**：层级结构通过治理层的预规划，避免了无效的对话冗余。实验显示，在提升性能的同时，Token 消耗最高可减少 **74.52%**。

*   **任务依赖性**：层级结构在需要稳定角色分配、受控信息流和多层验证的任务（如 SQuAD 2.0）中优势最明显；而在简单的多项选择题中提升有限。

#### 7. 优点（亮点）
*   **跨学科借鉴**：成功将组织行为学理论引入 MAS 设计，为解决智能体协作混乱提供了理论支撑。
*   **模块化与灵活性**：通过执行模式和策略的切换，用户可以根据预算和精度需求灵活调整系统行为。
*   **高效率**：打破了“多智能体必然带来高成本”的固有印象，证明了良好的组织结构可以同时实现高性能和低成本。

#### 8. 不足与局限
*   **任务覆盖面**：在受限答案空间（如 MMLU 多选题）的任务中，层级结构的复杂性可能变成负担，提升不明显。

*   **延迟问题**：虽然 Token 减少了，但多层级的串行逻辑可能会增加系统的端到端响应延迟（Latency），文中对此讨论较少。
