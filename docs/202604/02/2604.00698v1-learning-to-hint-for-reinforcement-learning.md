## 摘要
组相对策略优化（Group Relative Policy Optimization, GRPO）被广泛应用于具有可验证奖励的强化学习中，但它经常面临优势崩溃（advantage collapse）的问题：当一个组内的所有采样序列（rollouts）获得相同的奖励时，该组产生的相对优势为零，从而无法提供学习信号。例如，如果一个问题对于推理器（reasoner）而言过于困难，所有采样的序列可能都是错误的并获得零奖励。近期研究通过为此类难题添加提示（hints）或辅助支架（auxiliary scaffolds）来解决这一问题，使推理器产生混合的结果并恢复非零更新。然而，现有的提示通常是固定的，而非自适应于当前的推理器，且在有提示输入下产生学习信号的提示并不一定能改进测试时使用的无提示策略。为此，我们提出了强化学习提示学习（Hint Learning for Reinforcement Learning, HiLL），这是一个在强化学习过程中联合训练提示器（hinter）策略和推理器策略的框架。对于每个难题，提示器会根据当前推理器的错误采样序列在线生成提示，从而使提示生成能够适应推理器不断演变的错误。我们进一步引入了提示依赖性（hint reliance），用于衡量正确的提示轨迹对提示的依赖程度。我们推导出了一个可迁移性结果，表明较低的提示依赖性意味着从有提示成功到无提示成功的更强迁移，并利用该结果定义了用于训练提示器的迁移加权奖励。因此，HiLL 倾向于选择那些不仅能恢复信息丰富的 GRPO 组，还能产生更有可能改进原始无提示策略信号的提示。在多个基准测试上的实验表明，HiLL 一贯优于 GRPO 和此前基于提示的基准方法，证明了自适应且具备迁移感知能力的提示学习在强化学习中的价值。代码可在 https://github.com/Andree-9/HiLL 获取。

## Abstract
Group Relative Policy Optimization (GRPO) is widely used for reinforcement learning with verifiable rewards, but it often suffers from advantage collapse: when all rollouts in a group receive the same reward, the group yields zero relative advantage and thus no learning signal. For example, if a question is too hard for the reasoner, all sampled rollouts can be incorrect and receive zero reward. Recent work addresses this issue by adding hints or auxiliary scaffolds to such hard questions so that the reasoner produces mixed outcomes and recovers a non-zero update. However, existing hints are usually fixed rather than adapted to the current reasoner, and a hint that creates learning signal under the hinted input does not necessarily improve the no-hint policy used at test time. To this end, we propose Hint Learning for Reinforcement Learning (HiLL), a framework that jointly trains a hinter policy and a reasoner policy during RL. For each hard question, the hinter generates hints online conditioned on the current reasoner's incorrect rollout, allowing hint generation to adapt to the reasoner's evolving errors. We further introduce hint reliance, which measures how strongly correct hinted trajectories depend on the hint. We derive a transferability result showing that lower hint reliance implies stronger transfer from hinted success to no-hint success, and we use this result to define a transfer-weighted reward for training the hinter. Therefore, HiLL favors hints that not only recover informative GRPO groups, but also produce signals that are more likely to improve the original no-hint policy. Experiments across multiple benchmarks show that HiLL consistently outperforms GRPO and prior hint-based baselines, demonstrating the value of adaptive and transfer-aware hint learning for RL. The code is available at https://github.com/Andree-9/HiLL.

### 1. 核心问题与整体含义
*   **研究背景**：在具有可验证奖励（如数学题）的强化学习（RL）中，**GRPO**（组相对策略优化）被广泛使用。它通过比较一组采样序列（rollouts）的相对奖励来计算优势。
*   **核心问题**：**优势坍缩（Advantage Collapse）**。当题目太难时，推理器（Reasoner）产生的所有采样可能都是错误的（奖励均为0），导致相对优势为零，无法产生学习信号。
*   **研究动机**：现有的“提示（Hint）”方法多为固定或离线生成的，无法根据推理器当前的错误动态调整，且容易产生“捷径提示”（即推理器靠提示直接得到答案，而非学会了推理），导致模型在测试时（无提示状态）表现不佳。

### 2. 方法论
论文提出了 **HiLL (Hint Learning for RL)** 框架，核心思想是**联合训练**一个提示器（Hinter）和一个推理器（Reasoner）。
*   **在线自适应提示生成**：对于推理器无法解决的难题，提示器根据“题目 + 推理器的错误尝试 + 参考答案”在线生成提示。这使得提示能针对推理器当前的薄弱环节。
*   **提示依赖度（Hint Reliance, $\rho$）**：定义为有提示和无提示下轨迹概率的对数比。
    *   公式含义：$\rho(\tau; q, h) = \log \pi_\theta(\tau | q+h) - \log \pi_\theta(\tau | q)$。
    *   **迁移性结果**：论文推导出，提示依赖度越低，从“有提示成功”到“无提示成功”的迁移能力越强。
*   **迁移加权奖励（Transfer-Weighted Reward）**：提示器的奖励由两部分组成：
    1.  **信号创建**：提示是否让推理器产生了“有对有错”的混合结果（打破优势坍缩）。
    2.  **信号迁移**：通过 $\exp(-\rho/T)$ 对奖励加权，惩罚那些让推理器产生高度依赖的提示。
*   **算法流程**：
    1. 推理器采样，识别全错题目。
    2. 提示器生成 $M$ 个候选提示。
    3. 推理器在提示下重新采样。
    4. 计算提示奖励，选择最佳提示组更新推理器，同时利用 GRPO 更新提示器。

### 3. 实验设计
*   **模型**：推理器使用 Llama-3.2-3B-Instruct 和 Qwen2.5-7B-Instruct；提示器使用 Qwen3-4B-Instruct。
*   **数据集**：OpenR1-Math-220k 的 15k 子集（源自 NuminaMath 1.5）。
*   **Benchmark**：
    *   **分布内（数学）**：AIME24/25, AMC23, MATH-500, Minerva Math, OlympiadBench。
    *   **分布外（通用推理）**：GPQA-diamond, MMLU-Pro。
*   **对比方法**：Base 模型、标准 GRPO、LUFFY（离线引导）、Scaf-GRPO（外部教师提示）、SAGE（自生成提示）。

### 4. 资源与算力
*   **硬件**：使用了 **8 张 NVIDIA B200 GPU**。
*   **软件框架**：使用 `verl` 进行 RL 训练，`vLLM` 进行推理采样，`Ray` 进行多模型管理。
*   **训练细节**：训练 500 步，Batch Size 为 128，每题采样 $G=8$ 条轨迹。提示器生成 $M=4$ 个候选提示。

### 5. 实验数量与充分性
*   **实验规模**：在两个不同规模（3B 和 7B）的基座模型上进行了验证，涵盖了 8 个主流推理基准测试。
*   **消融实验**：对比了 HiLL 与“无迁移加权（w/o TW）”的版本，验证了迁移奖励的必要性。
*   **分析深度**：提供了训练过程中全错率的变化曲线、提示依赖度的演变、温度系数 $T$ 的敏感度分析，以及提示词长度和数学表达式数量的定性分析。
*   **评价**：实验设计充分且客观，通过分布外测试验证了泛化性，对比基准涵盖了最新的提示增强 RL 方法。

### 6. 主要结论与发现
*   **性能提升**：HiLL 在所有数学和通用推理任务上均显著优于标准 GRPO 和其他提示基准。
*   **迁移价值**：引入“提示依赖度”能有效防止提示器提供“答案捷径”。HiLL 生成的提示更短、更具概念性（Conceptual），而非直接给出计算步骤。
*   **动态适应**：随着训练进行，提示器能自动转向更难的题目，并根据推理器不断变化的错误模式调整策略。

### 7. 优点
*   **自适应性强**：解决了固定提示无法随模型进化而调整的问题。
*   **理论支撑**：为提示的“迁移性”提供了数学推导和量化指标（Hint Reliance）。
*   **资源高效**：通过 Ray 调度，提示器和推理器顺序执行，共享显存，未增加额外的硬件门槛。
*   **泛化性好**：数学训练带来的推理能力提升成功迁移到了 GPQA 等非数学领域。

### 8. 不足与局限
*   **计算开销**：由于需要对全错题目进行二次采样和提示生成，HiLL 的训练耗时是标准 GRPO 的 **2.6 到 3.8 倍**。
*   **依赖参考答案**：提示器在训练阶段需要访问标准答案（Ground Truth Solution）来生成高质量提示，这在某些缺乏标注数据的领域可能受限。
*   **奖励类型限制**：目前主要针对二元（对/错）的可验证奖励场景，在开放式生成任务中的适用性有待验证。
