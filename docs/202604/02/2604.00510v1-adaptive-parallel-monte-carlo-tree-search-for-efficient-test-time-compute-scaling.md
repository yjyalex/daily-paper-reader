## 摘要
蒙特卡洛树搜索 (MCTS) 是一种有效的测试时计算缩放 (TTCS) 方法，用于提升大语言模型的推理性能，但其高度多变的执行时间在实践中导致了严重的长尾延迟。现有的优化方法（如正向提前退出）在理想情况下可以降低延迟，但在搜索持续进行却无实质性进展时效果较差。我们引入了“负向提前退出”（negative early exit），用于剪枝无成效的 MCTS 轨迹，以及一种“自适应增强机制”（adaptive boosting mechanism），通过重新分配回收的计算资源来减少并发搜索之间的资源竞争。这些技术集成到 vLLM 中后，在保持推理准确性的同时，显著降低了 p99 端到端延迟并提高了吞吐量。

## Abstract
Monte Carlo Tree Search (MCTS) is an effective test-time compute scaling (TTCS) method for improving the reasoning performance of large language models, but its highly variable execution time leads to severe long-tail latency in practice. Existing optimizations such as positive early exit, reduce latency in favorable cases but are less effective when search continues without meaningful progress. We introduce {\it negative early exit}, which prunes unproductive MCTS trajectories, and an {\it adaptive boosting mechanism} that reallocates reclaimed computation to reduce resource contention among concurrent searches. Integrated into vLLM, these techniques substantially reduce p99 end-to-end latency while improving throughput and maintaining reasoning accuracy.

#### 1. 核心问题与研究动机
*   **核心问题**：蒙特卡洛树搜索（MCTS）虽然能显著提升大语言模型（LLM）的推理准确率，但其**高度可变的执行时间**和**固有的串行依赖**导致了严重的**长尾延迟（Tail Latency）**。
*   **研究动机**：
    *   现有的“正向早停”（Positive Early Exit）仅在找到高质量解时停止，无法处理那些持续消耗资源却无法产生好结果的“无效搜索”。
    *   传统的并行 MCTS 虽能加速单次搜索，但无法在多请求并发的生产环境中动态管理资源，导致资源竞争和系统吞吐量受限。
    *   论文旨在从系统架构角度重新审视 MCTS，将其视为需要主动资源管理的系统，而非单纯的算法过程。

#### 2. 方法论
论文提出了一个自适应并行 MCTS 框架，核心包含以下关键技术：
*   **负向早停（Negative Early Exit）**：
    *   **原理**：基于过程奖励模型（PRM）的特性，轨迹分数通常由路径中节点的最小分数或累积乘积决定，因此轨迹分数受限于其叶节点分数。
    *   **机制**：当搜索树中所有叶节点的分数都低于预设阈值（即判定为“无效节点”）时，立即终止该请求的搜索，避免浪费计算资源。
    *   **优化**：引入“选择性无效检查”，利用第一步奖励与最终分数的相关性，提前排除低潜力分支。
*   **自适应增强机制（Adaptive Boosting）**：
    *   **动态并行度控制**：根据任务的“资历”（等待时间）和“紧迫性”（当前最高分接近阈值的程度）计算优先级分数 $S(i, t)$。
    *   **资源重分配**：将早停释放的计算资源（GPU 算力槽位）动态分配给最有希望快速完成的任务，通过提高其并行度（DOP）来压缩长尾延迟。
*   **并行 MCTS 策略（WU-PUCT）**：
    *   扩展了 AlphaZero 风格的 PUCT 规则，引入“未观测计数”（Unobserved Counts）来处理正在进行中的并行 Rollout，防止多个并行任务重复探索同一路径。
