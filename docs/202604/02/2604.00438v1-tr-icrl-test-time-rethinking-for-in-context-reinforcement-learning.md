## 摘要
上下文强化学习（ICRL）使大语言模型（LLMs）能够直接在上下文窗口内从外部奖励中进行在线学习。然而，ICRL 的一个核心挑战是奖励估计，因为模型在推理过程中通常无法获取真实标签（ground-truths）。为了解决这一局限性，我们提出了面向上下文强化学习的测试时反思（TR-ICRL），这是一种专为推理和知识密集型任务设计的新型 ICRL 框架。TR-ICRL 的运作方式是首先针对给定查询从无标签评估集中检索最相关的实例。在每次 ICRL 迭代期间，LLM 为每个检索到的实例生成一组候选答案。接着，通过多数投票从该集合中推导出伪标签。该标签随后作为代理提供奖励信息并生成形成性反馈，引导 LLM 进行迭代优化。最后，将这些合成的上下文信息与原始查询整合，形成一个综合提示词，并通过最后一轮多数投票确定答案。TR-ICRL 在主流推理和知识密集型任务上进行了评估，展示了显著的性能提升。值得注意的是，TR-ICRL 在 MedQA 上使 Qwen2.5-7B 的平均性能提升了 21.23%，在 AIME2024 上甚至提升了 137.59%。广泛的消融实验和分析进一步验证了我们方法的有效性和鲁棒性。我们的代码可在 https://github.com/pangpang-xuan/TR_ICRL 获取。

## Abstract
In-Context Reinforcement Learning (ICRL) enables Large Language Models (LLMs) to learn online from external rewards directly within the context window. However, a central challenge in ICRL is reward estimation, as models typically lack access to ground-truths during inference. To address this limitation, we propose Test-Time Rethinking for In-Context Reinforcement Learning (TR-ICRL), a novel ICRL framework designed for both reasoning and knowledge-intensive tasks. TR-ICRL operates by first retrieving the most relevant instances from an unlabeled evaluation set for a given query. During each ICRL iteration, LLM generates a set of candidate answers for every retrieved instance. Next, a pseudo-label is derived from this set through majority voting. This label then serves as a proxy to give reward messages and generate formative feedbacks, guiding LLM through iterative refinement. In the end, this synthesized contextual information is integrated with the original query to form a comprehensive prompt, with the answer determining through a final round of majority voting. TR-ICRL is evaluated on mainstream reasoning and knowledge-intensive tasks, where it demonstrates significant performance gains. Remarkably, TR-ICRL improves Qwen2.5-7B by 21.23% on average on MedQA and even 137.59% on AIME2024. Extensive ablation studies and analyses further validate the effectiveness and robustness of our approach. Our code is available at https://github.com/pangpang-xuan/TR_ICRL.

### 论文总结：TR-ICRL (Test-Time Rethinking for In-Context Reinforcement Learning)

#### 1. 核心问题与研究动机
*   **核心问题**：上下文强化学习（ICRL）允许大语言模型（LLM）在推理时通过上下文信息进行在线学习和策略改进。然而，传统的 ICRL 严重依赖外部奖励或真实标签（Ground-truth）来提供反馈。在实际推理（测试时）场景中，真实标签通常是不可用的，这限制了 ICRL 的自主进化能力。
*   **研究动机**：受 OpenAI o1 和 DeepSeek-R1 等模型在测试时缩放（Test-Time Scaling）能力的启发，作者希望探索如何在大模型推理阶段，在**无标签**的情况下，通过自我生成奖励信号来实现自主的策略优化和错误修正。

#### 2. 方法论：TR-ICRL 框架
TR-ICRL 包含三个主要阶段，旨在通过迭代反思将测试时计算转化为性能提升：
*   **上下文检索（Context Retrieval）**：针对目标查询（Query），利用嵌入模型从无标签的评估集中检索出 $N$ 个语义最相关的实例。这些实例作为 ICRL 的原始素材。
*   **测试时迭代反思（Test-Time Iterative Rethinking）**：
    *   **Rollout（采样）**：对每个检索到的问题进行 $K$ 次并行采样，生成多个推理路径。
    *   **伪标签生成**：通过**多数投票（Majority Voting）**从 $K$ 个答案中选出共识答案，作为“伪标签”。
    *   **奖励与反馈**：将每个采样结果与伪标签对比，生成二元奖励信号（正确/错误）。随后，引导 LLM 根据奖励信息生成“形成性反馈”（即诊断错误原因或强化正确逻辑）。
    *   **内存更新**：将“问题-答案-奖励-反馈”这一完整轨迹存入上下文窗口，作为后续推理的参考。
*   **自一致性聚合（Self-Consistent Aggregation）**：将积累了 $N$ 轮反思经验的上下文与原始查询结合，再次进行多次采样并投票，得出最终答案。

#### 6. 主要结论与发现
*   **性能飞跃**：TR-ICRL 显著提升了模型性能。例如，Qwen2.5-7B 在 AIME2024 上的准确率从 7.9% 提升至 18.77%（+137.59%），在 MedQA 上提升了 21.23%。
*   **“幸运奖励”（Lucky Reward）现象**：即使多数投票产生的伪标签是错误的，只要它与模型当前的错误预测不同，仍能提供正确的负反馈信号。实验显示，在 AIME2024 上，尽管标签准确率仅 16.56%，但奖励信号的准确率高达 71.71%。
*   **上下文干扰**：随着迭代步数增加（超过 6-8 步），过长的上下文可能导致模型注意力分散，性能出现轻微波动或下降。

#### 7. 优点
*   **完全无监督**：无需任何真实标签或人工干预，即可在推理阶段实现自我进化。
*   **通用性强**：不仅适用于逻辑严密的数学推理，也适用于依赖背景知识的医学问答。
*   **即插即用**：无需微调模型权重，通过纯上下文交互实现强化学习的效果。
*   **高效性**：通过测试时计算的缩放，使小参数模型具备了挑战超大参数模型的能力。

#### 8. 不足与局限
*   **奖励信号单一**：目前仅使用基于多数投票的二元奖励（对/错），缺乏更细粒度的评分机制（如过程奖励 PRM）。
*   **上下文长度限制**：迭代步数受限于 LLM 的上下文窗口长度，且长文本下的信息干扰问题仍待解决。
*   **计算开销**：虽然比微调省力，但多次采样（Rollouts）和迭代反思显著增加了推理时的延迟和计算成本。
