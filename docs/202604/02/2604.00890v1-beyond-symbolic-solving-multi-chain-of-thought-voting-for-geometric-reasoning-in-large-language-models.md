## 摘要
几何问题求解（GPS）一直是增强大语言模型数学推理能力的核心，因为它需要结合图形理解、符号操作和逻辑推理。在现有文献中，研究人员主要关注于将图形描述与文本字面量同步并解决问题。在这一思路下，他们采用了神经、符号或神经符号方法。但这仅解决了前两个需求，即图形理解和符号操作，而逻辑推理仍未得到充分发展。逻辑推理通常局限于单一的思维链（CoT）。为了解决现有模型中的这一弱点，本文提出了 MARS-GPS。该方法生成多个并行的推理展开（rollouts），并辅以 Python 代码执行进行数值验证，利用标记级（token-level）熵作为置信度信号对它们进行排序，并通过多阶段投票和自我验证流水线聚合答案。实验结果表明，具有 8 个并行展开的 MARS-GPS 在 Geometry3K 数据集上达到了 88.8% 的准确率，比之前的最优方法（SOTA）提高了近 11%，且准确率随着展开数量从 1 增加到 16 而持续提升（在消融子集上提升了 6.0%）。我们在匿名仓库中提供了代码和数据：https://anonymous.4open.science/r/MARS-GPS-DE55。

## Abstract
Geometric Problem Solving (GPS) remains at the heart of enhancing mathematical reasoning in large language models because it requires the combination of diagrammatic understanding, symbolic manipulation and logical inference. In existing literature, researchers have chiefly focused on synchronising the diagram descriptions with text literals and solving the problem. In this vein, they have either taken a neural, symbolic or neuro-symbolic approach. But this solves only the first two of the requirements, namely diagrammatic understanding and symbolic manipulation, while leaving logical inference underdeveloped. The logical inference is often limited to one chain-of-thought (CoT). To address this weakness in hitherto existing models, this paper proposes MARS-GPS, that generates multiple parallel reasoning rollouts augmented with Python code execution for numerical verification, ranks them using token-level entropy as a confidence signal, and aggregates answers through a multi-stage voting and self-verification pipeline. Empirical results show that MARS-GPS with 8 parallel rollouts achieves 88.8% on Geometry3K, a nearly +11% improvement over the prior state-of-the-art, with accuracy scaling consistently as the number of rollouts increases from 1 to 16 (+6.0% on ablation subset). We provide our code and data in an anonymous repository: https://anonymous.4open.science/r/MARS-GPS-DE55.

### 论文总结：超越符号求解——大语言模型几何推理的多思维链投票 (MARS-GPS)

#### 1. 核心问题与整体含义（研究动机和背景）
几何问题求解（GPS）要求模型同时具备图形理解、符号操作和逻辑推理能力。现有研究（如神经符号方法）多侧重于前两者，即如何将图形转化为形式化语言，但在**逻辑推理**环节往往依赖单一的思维链（Chain-of-Thought, CoT）或确定性的符号求解器。这种单路径推理容错率低，一旦中间步骤出错（如定理误用或计算失误），最终结果便会失败。本文旨在通过**推理时计算缩放（Inference-time Scaling）**，利用多路径并行推理、代码辅助验证和置信度感知的聚合机制，提升大语言模型（LLM）在复杂几何推理中的鲁棒性和准确性。

#### 2. 方法论：核心思想与技术细节
论文提出了 **MARS-GPS** 框架，其核心流程分为两个阶段：

*   **阶段一：问题解析（Problem Parsing）**
    *   **文本解析**：使用基于规则的解析器将题目文本转化为一阶几何谓词（FT）。
    *   **图形解析**：利用预训练的 PGDPNet 提取图形中的几何基元和关系（FD），合并为统一的形式化上下文 $F^*$。
*   **阶段二：推理时集成推理（Inference-Time Ensemble Reasoning）**
    *   **并行采样（Parallel Rollouts）**：从冻结的 LLM 中并行采样 $k$ 条独立的 CoT 推理路径。
    *   **代码增强推理**：每条路径均可调用 Python 沙箱进行精确的数值计算，减少 LLM 的算术幻觉。
    *   **置信度估计**：利用 Token 级的对数概率计算 **Shannon 熵**，作为衡量该条推理路径可靠性的反向指标（熵越低，置信度越高）。
    *   **六步聚合算法**：
        1.  **早期共识**：若超过半数路径答案一致，直接采纳。
        2.  **硬接受**：检查弱多数共识。
        3.  **候选筛选**：剔除孤立答案。
        4.  **熵排序**：按平均熵值对候选答案排序。
        5.  **LLM 自我验证**：按排序依次让模型对候选答案进行“正确/错误”的二元校验。
        6.  **加权回退**：若上述均失效，根据投票数和熵值的加权得分选出最终答案。

#### 6. 主要结论与发现
*   **性能突破**：MARS-GPS 在 Geometry3K 上达到 **88.8%** 的准确率，比之前的 SOTA（Pi-GPS）提升了约 **11%**。

*   **推理开销**：错误预测的平均执行时间是正确预测的 4.2 倍，因为系统会耗尽所有验证步骤才回退。

#### 7. 优点（亮点）
*   **无需微调**：完全基于推理时策略，易于迁移到不同的基座模型。
*   **多模态解耦**：通过解析器将图形信息转化为文本谓词，规避了 MLLM 直接处理几何图形时精度不足的问题。
*   **鲁棒性强**：结合了代码执行和多步验证，有效解决了几何题中常见的计算错误和逻辑跳跃。

#### 8. 不足与局限
*   **解析器依赖**：系统性能高度依赖第一阶段的图形解析器（PGDPNet），若解析出的谓词有误，后续推理难以挽回。

*   **应用限制**：依赖于结构化的形式化表示，对于非标准或极其复杂的奥数级几何图形，解析难度较大。
