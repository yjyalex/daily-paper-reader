## 摘要
推理大语言模型（LLMs）的最新进展主要依赖于预先思考（upfront thinking），即推理发生在给出最终答案之前。然而，这种方法在代码生成中存在严重的局限性，因为问题的完整复杂性往往只有在代码实现过程中才会显现，导致预先思考往往不足。此外，它无法在难度显著变化的代码生成过程中自适应地分配推理资源。在本文中，我们提出了 Think-Anywhere，这是一种新颖的推理机制，使 LLM 能够在代码生成过程中的任何 token 位置按需调用推理。我们首先通过冷启动训练教 LLM 模仿推理模式，然后利用基于结果的强化学习（RL）奖励来驱动模型自主探索何时以及何处调用推理，从而实现 Think-Anywhere。在四个主流代码生成基准测试（即 LeetCode、LiveCodeBench、HumanEval 和 MBPP）上的广泛实验表明，Think-Anywhere 的性能优于现有的推理方法和最近的训练后方法，达到了最先进的水平，并在不同的 LLM 中展现出一致的泛化能力。我们的分析进一步揭示，Think-Anywhere 使模型能够在高熵位置自适应地调用推理，从而提供了增强的可解释性。

## Abstract
Recent advances in reasoning Large Language Models (LLMs) have primarily relied on upfront thinking, where reasoning occurs before final answer. However, this approach suffers from critical limitations in code generation, where upfront thinking is often insufficient as problems' full complexity only reveals itself during code implementation. Moreover, it cannot adaptively allocate reasoning effort throughout the code generation process where difficulty varies significantly. In this paper, we propose Think-Anywhere, a novel reasoning mechanism that enables LLMs to invoke thinking on-demand at any token position during code generation. We achieve Think-Anywhere by first teaching LLMs to imitate the reasoning patterns through cold-start training, then leveraging outcome-based RL rewards to drive the model's autonomous exploration of when and where to invoke reasoning. Extensive experiments on four mainstream code generation benchmarks (i.e., LeetCode, LiveCodeBench, HumanEval, and MBPP) show that Think-Anywhere achieves state-of-the-art performance over both existing reasoning methods and recent post-training approaches, while demonstrating consistent generalization across diverse LLMs. Our analysis further reveals that Think-Anywhere enables the model to adaptively invoke reasoning at high-entropy positions, providing enhanced interpretability.

#### 1. 核心问题与整体含义（研究动机和背景）
传统的推理大语言模型（如 OpenAI o1, DeepSeek-R1）主要采用**“预先思考”（Upfront Thinking）**模式，即在输出最终答案前进行一段长推理。
*   **研究动机**：作者发现这种模式在代码生成任务中存在两大局限：
    1.  **复杂性滞后显现**：代码的完整复杂性（如具体的边界条件、索引逻辑）往往在实际编写过程中才暴露，仅靠开头的全局规划难以覆盖所有实现细节。
    2.  **算力分配不均**：代码不同位置的难度差异巨大，简单的模板代码不需要推理，而复杂的算法逻辑需要深度思考。固定在开头的推理无法实现计算资源的按需分配。
*   **核心目标**：提出一种名为 **Think-Anywhere** 的机制，使模型能够根据即时上下文和局部复杂度，在代码生成的**任何 Token 位置**自主触发推理。

#### 2. 方法论：核心思想与关键技术
Think-Anywhere 通过将推理过程与代码生成过程深度交织，实现了“边写边想”。其核心流程分为两个阶段：
*   **核心思想**：将生成序列定义为初始思考块、代码片段和中间思考块（`<thinkanywhere>`）的交替组合。模型在生成过程中动态决定是否插入思考块。
*   **关键技术细节**：
    1.  **冷启动训练（Cold-start）**：由于模型不会自发在代码中间思考，作者利用强推理模型（如 Gemini 2.5 Flash）构造了约 5000 条包含中间思考过程的高质量样本，通过 LoRA 进行监督微调（SFT），教会模型这种“随处思考”的格式规范。
    2.  **专用触发 Token（Dedicated Trigger Token）**：为了提高触发效率，作者引入了专用的特殊 Token，并采用**语义感知初始化**（结合 "think", "anywhere" 的语义嵌入和模式切换 Token 的结构嵌入），使模型能更稳定地切换推理模式。
    3.  **基于结果的强化学习（RLVR）**：采用 **GRPO 算法**，以代码的**功能正确性（单元测试通过率）**作为核心奖励信号。通过强化学习，模型不再是简单模仿 SFT 样本，而是自主探索在哪些关键位置（如高熵位置）进行推理能最大化提升代码正确性。

*   **效率优化**：证明了“按需推理”比“全量预推理”在计算资源利用上更具优势。
