## Abstract
Large Language Models (LLMs) can reason well, yet often miss decisive evidence when it is buried in long, noisy contexts. We introduce HiLight, an Evidence Emphasis framework that decouples evidence selection from reasoning for frozen LLM solvers. HiLight avoids compressing or rewriting the input, which can discard or distort evidence, by training a lightweight Emphasis Actor to insert minimal highlight tags around pivotal spans in the unaltered context. A frozen Solver then performs downstream reasoning on the emphasized input. We cast highlighting as a weakly supervised decision-making problem and optimize the Actor with reinforcement learning using only the Solver's task reward, requiring no evidence labels and no access to or modification of the Solver. Across sequential recommendation and long-context question answering, HiLight consistently improves performance over strong prompt-based and automated prompt-optimization baselines. The learned emphasis policy transfers zero-shot to both smaller and larger unseen Solver families, including an API-based Solver, suggesting that the Actor captures genuine, reusable evidence structure rather than overfitting to a single backbone.

*   **核心问题**：大语言模型（LLMs）在处理长文本或高噪声上下文时，尽管具备强大的推理能力，但往往会忽略关键证据（即“Lost in the Middle”效应）。
*   **研究动机**：现有的上下文处理方法（如压缩、摘要或重写）往往是破坏性的，容易导致证据丢失或产生幻觉。
*   **整体含义**：本文提出了 **HiLight** 框架，旨在将“证据选择”与“推理”解耦。通过训练一个轻量级的“强调代理”（Emphasis Actor），在不改变原始文本的前提下，通过插入微小的结构化标签（如高亮标记）来引导冻结的“推理模型”（Solver）关注关键信息。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：模仿人类阅读时做笔记的行为，在原始上下文中插入 `<start_important>` 和 `<end_important>` 标签，而不对文本进行任何删减或重写。
*   **关键技术细节**：
    *   **Emphasis Actor**：一个轻量级的预训练语言模型（如 Qwen3-0.6B/1.7B/4B），负责为每个 token 生成重要性得分。
    *   **预算化选择（Budgeted Selection）**：通过预设比例 $\gamma$（如 15%-25%）限制高亮 token 的数量，防止过度高亮。
    *   **弱监督强化学习（RL）**：由于缺乏 token 级别的证据标注，HiLight 将高亮任务建模为决策问题。使用 **分组策略梯度（Grouped Policy Gradient）** 算法，直接利用 Solver 在下游任务中的表现（如 F1、准确率、NDCG）作为奖励信号来优化 Actor。
    *   **非破坏性操作**：Actor 仅负责标记，Solver 看到的是包含原始信息及高亮标签的完整文本。

### 6. 主要结论与发现
*   **性能提升**：HiLight 在所有任务上均优于最强的 APO 基准。在证据稀疏的推荐任务中，相比手动提示词提升了约 **27%**。
*   **非破坏性的必要性**：在需要复杂推理的任务（如 HotpotQA）中，简单的“剪裁”文本会破坏推理链条，而 HiLight 的高亮模式表现更好。
*   **零样本迁移性**：Actor 学习到的是通用的证据结构，而非对特定 Solver 的过拟合。它能显著提升从未见过的更大规模模型（如 Llama-3-70B）的表现。
*   **证据对齐**：尽管没有使用证据标注，Actor 自动学习到的高亮区域与人类标注的证据 F1 重合度高达 **0.78**。

### 7. 优点（亮点）
*   **解耦设计**：将复杂的证据定位从推理中分离，降低了 Solver 的认知负荷。
*   **黑盒兼容性**：不需要 Solver 的梯度或中间激活值，完美适配 API 形式的商业模型。
*   **可解释性**：高亮标签是人类可读的，用户可以直观看到模型推理的依据。
*   **低开销**：Actor 模型非常轻量，推理延迟仅增加约 1.3%-2.9%。

### 8. 不足与局限
*   **多轮对话挑战**：目前的实现主要针对单次查询，在多轮对话中如何高效管理 KV 缓存（因为高亮标签会改变输入序列）尚需进一步研究。
*   **训练成本**：虽然推理快，但训练过程需要多次调用 Solver 以获取奖励信号（尽管比某些 APO 方法更高效）。
*   **任务依赖性**：在证据非常集中、推理难度极高的任务（如 PubMedQA）中，提升幅度相对较小，说明该方法主要解决的是“找证据”而非“强逻辑”问题。
