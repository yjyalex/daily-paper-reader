# Reasoning Graphs: Self-Improving, Deterministic RAG through Evidence-Centric Feedback 

## Matthew Penaroza April 15, 2026 

Abstract 

Language model agents reason from scratch on every query, discarding their chain of thought after each run. This produces lower accuracy and high variance, as the same query type can succeed or fail unpredictably. We introduce 

reasoning graphs , a graph structure that persists per-evidence chain of thought as structured edges connected to the evidence items they evaluate. Unlike prior memory mechanisms that retrieve distilled strategies by query similarity, reasoning graphs enable evidence-centric feedback : given a new candidate set, the system traverses all incoming evaluation edges for each evidence item across all prior runs, surfacing how that specific item has been judged before. We further introduce retrieval graphs , a complementary structure that feeds a pipeline planner to tighten the candidate funnel over successive runs. Together, both graphs form a self-improving feedback loop: accuracy improves systematically and verdict-level variance collapses. This requires no retraining; the base model remains frozen and all gains come from context engineering via graph traversal. We evaluate on MuSiQue and HotpotQA using a sequential cluster protocol, a high-reuse deployment simulation, and a determinism experiment. At 50%+ evidence profile coverage, our system reduces errors by 47% compared to vanilla RAG on the same questions (controlled dose-response, p < 0.0001 ). On 4-hop questions, accuracy improves by +11.0pp ( p = 0 .0001 ). In high-reuse settings, the system achieves Pareto dominance: highest accuracy, 47% lower cost, and 46% lower latency. Evidence profiles improve verdict consistency by 7–8 percentage points ( p = 0 .007 , Wilcoxon); the full system drives all 11 hard probes to perfect consistency at both temperature 0 and 0.5 ( p = 0 .004 ). 

# 1 Introduction 

Every language model agent follows the same loop: retrieve context from a knowledge base, reason over the retrieved evidence, and produce an action or answer (Yao et al., 2023; Sumers et al., 2024). The quality of the action is bounded by the quality of the reasoning, which is in turn bounded by the quality of the retrieved context. At each step, the agent’s chain of thought (its evaluation of each piece of evidence, why it used or rejected it, and how it connected evidence to its conclusion) is discarded. The next similar query starts from scratch. In this work, we address this problem through reasoning graphs , which persist per-evidence chain of thought as graph edges with bidirectional traversal. This enables evidence-centric feedback that makes agent verdict behavior deterministic at the evidence evaluation level without retraining. Discarding reasoning produces two distinct problems. First, accuracy is lower than it needs to be: the agent repeatedly encounters the same evidence items and must re-derive how to evaluate them, with no benefit from prior correct evaluations. Second, and less commonly recognized, variance remains high : the same type of query can succeed or fail unpredictably across runs because the agent’s reasoning is unconstrained by prior experience. Recent work has quantified this problem directly: on HotpotQA, ReAct-style agents produce 2.0–4.2 distinct action sequences per 10 runs on identical inputs, and behavioral consistency is the dominant predictor of task success: tasks with consistent agent behavior achieve 80–92% accuracy, while inconsistent tasks achieve only 25–60% (Mehta, 2026a). We observe the same pattern in our experiments: running a baseline RAG agent on 100 queries of the same type yields no accuracy improvement and no variance reduction across runs (Section 4.4). Prior work on agent memory addresses the first problem but not the second. Reflexion (Shinn et al., 2023) stores verbal self-reflections in a flat text buffer, providing general lessons but disconnected from specific evidence. Rea-soningBank (Ouyang et al., 2026) distills generalizable reasoning strategies from agent experiences as flat triples, retrieved by embedding the new query and finding similar past strategies. While effective at improving accuracy, these approaches are query-centric : they retrieve past reasoning based on query similarity, with no direct connection to the 1

Unlike query-centric approaches that retrieve past strategies by query similarity, evidence-centric feedback starts from the specific evidence items in the current candidate set. specific evidence items the agent is currently examining. The feedback is approximate by construction; it depends on the query embedding finding a relevant match in the strategy store. We propose a different mechanism: evidence-centric feedback . Instead of asking “what strategies worked on similar queries?”, we ask “what does the system already know about these specific evidence items ?” Given a new candidate set, our system starts from each evidence item and traverses all incoming evaluation edges across all prior runs, constructing an evidence profile that summarizes how that item has been judged before: its typical verdict, the reasoning patterns associated with correct decisions, and its reliability signal. The feedback is automatically relevant because it is attached to the specific evidence, not dependent on query embedding similarity. Figure 1 illustrates the architecture. We formalize this mechanism through two complementary graph structures: 1. Reasoning graphs persist the agent’s per-evidence chain of thought as structured edges connecting agents to the evidence items they evaluate. Each edge carries the verdict (used or rejected), the natural language reasoning, and the confidence delta. Bidirectional traversal enables both chain-of-thought reconstruction (forward, from agent to evidence) and cross-run pattern discovery (backward, from evidence inward across all evaluations). This is the primary contribution. 2. Retrieval graphs capture retrieval pipeline metadata as graph edges, feeding a pipeline planner that optimizes filter configurations and excludes consistently-rejected evidence items. This tightens the candidate funnel over successive runs. Together, these structures produce three properties that no prior agent memory mechanism achieves simultane-ously: • Accuracy convergence : task accuracy improves systematically over successive runs as evidence profiles be-come richer and the candidate funnel tightens, with the largest gains on complex multi-hop questions with well-developed profiles. • Verdict-level variance collapse : decision consistency increases as the agent’s verdicts align with the majority signal in the evidence profiles, producing deterministic verdict behavior on well-characterized evidence. We validate this empirically: evidence profiles improve Verdict Vector Identity Rate (VVIR) by 7–8 percentage points ( p < 0.05 , Wilcoxon) in a confound-free comparison where both conditions evaluate identical passage sets. 2• Architectural auditability : by design, every decision is a graph traversal away from its complete chain of thought, enabling systematic failure diagnosis. These properties emerge with zero gradient updates; the base model remains frozen and all improvement comes from context engineering via graph traversal. The system self-improves autonomously as a side effect of normal operation. In high-reuse settings where passage overlap is dense, inference cost decreases over time as the candidate funnel tightens; in standard settings, graph traversal overhead can dominate until sufficient pruning develops. At cold start, with no evaluation edges, the agent reasons from scratch. Performance is comparable to vanilla RAG. Our contributions are: 1. Reasoning graphs : a graph structure capturing per-evidence chain of thought with evidence-centric bidirec-tional traversal (Section 3.2). 2. Retrieval graphs : a complementary structure for pipeline self-optimization via retrieval metadata edges (Sec-tion 3.3). 3. Self-improving feedback architecture : both graphs compound to make accuracy convergent and verdict-level behavior deterministic, without retraining (Section 3.4). 4. Empirical evaluation : experiments on MuSiQue and HotpotQA demonstrating dose-response accuracy im-provement (47% error reduction at high coverage), verdict-level determinism (VVIR 0.880 → 0.950 , p = 0 .007 ,confound-free), and Pareto-dominant efficiency in high-reuse settings, with ablations confirming that both com-ponents contribute independently (Section 4). 

# 2 Related Work 

Retrieval-Augmented Generation. RAG (Lewis et al., 2020) augments language model inputs with retrieved pas-sages, improving factuality on knowledge-intensive tasks. Subsequent work has identified consistent failure modes at scale: top-k instability, chunk blending, and precision degradation as corpus size grows (Gao et al., 2023). Self-RAG (Asai et al., 2023) addresses retrieval quality within a single run by training the model to generate reflection tokens that assess passage relevance and output factuality. Our work is complementary: Self-RAG optimizes retrieval 

within a run using learned reflection tokens, while reasoning graphs optimize across runs using persistent graph struc-ture. The two mechanisms operate on orthogonal axes and could be composed. 

Graph-Augmented Retrieval. GraphRAG (Edge et al., 2024) demonstrated that combining knowledge graph struc-ture with retrieval-augmented generation outperforms vector-only approaches for complex queries requiring multi-hop reasoning. Subsequent work has explored efficient graph construction (Zhuang et al., 2026) and conditions under which graph augmentation outperforms vanilla RAG (Xiang et al., 2026; Peng et al., 2024). These approaches struc-ture the source data as graphs to improve retrieval. We structure reasoning about the source data as graphs to improve agent decision-making, a different axis entirely. Our reasoning graphs do not modify the knowledge base; they aug-ment the agent’s context with evaluation history. 

Agent Memory and Self-Improvement. Reflexion (Shinn et al., 2023) stores verbal self-reflections in an episodic memory buffer and retrieves them by recency, demonstrating that agents can improve through linguistic feedback with-out weight updates. Voyager (Wang et al., 2023) maintains a skill library of successful action programs. MEM1 (Zhou et al., 2026) uses reinforcement learning to learn a compact memory state for long-horizon tasks. Most relevant is Rea-soningBank (Ouyang et al., 2026), which distills generalizable reasoning strategies from both successful and failed agent experiences, storing them as flat triples (title, description, content) retrieved by embedding similarity. Our work differs from ReasoningBank in both mechanism and capability. ReasoningBank is query-centric : given a new query, it embeds the query, retrieves similar strategies, and injects them. The strategies are distilled away from their source evidence. ReasoningBank cannot answer “how has evidence item ki been evaluated across all past runs?” because the connection between strategy and evidence is lost during distillation. Reasoning graphs are evidence-centric : given a new candidate set, the system starts from each evidence item and traverses all incoming evaluation edges, constructing a profile of how that specific item has been judged before. The feedback is automatically relevant because it is attached to the evidence, not dependent on query embedding similarity. We also measure and achieve variance collapse, a property that ReasoningBank does not address. 3Case-Based Reasoning. Case-based reasoning (CBR) retrieves past problem-solution pairs to inform new problem solving through a retrieve-reuse-revise-retain cycle (Aamodt and Plaza, 1994; Kolodner, 1993). We acknowledge this conceptual lineage. The key difference is structural: CBR stores cases as flat records indexed by problem features and retrieved by similarity. Reasoning graphs store structured edges connected to the specific evidence items they reference, enabling bidirectional traversal: from a piece of evidence, traverse all evaluations that ever referenced it, across all past runs. This backward traversal from evidence inward is a graph operation that flat case stores do not support. 

Episodic Memory in Cognitive Architectures. The Cognitive Architecture for Language Agents (CoALA) frame-work (Sumers et al., 2024) formalizes agent memory into episodic, semantic, and procedural types coordinated through a single decision loop. CoALA’s episodic memory describes the motivation for structured records of past actions in-forming future decisions. Generative Agents (Park et al., 2023) stores observations as flat natural language with recency, importance, and relevance scores. Our work is a concrete instantiation of episodic memory with a specific graph structure, formal traversal algorithms, and a dual feedback mechanism that feeds both the pipeline planner and the agent’s context window. 

Adaptive Retrieval Memory. GAM-RAG (Wang et al., 2026) accumulates retrieval experience across recurring queries using a Kalman-inspired gain rule that adaptively updates retrieval memory, achieving improved performance with reduced inference cost. Concurrent work on explicit evidence evaluation (Shu et al., 2026) converts retrieval quality assessment into an explicit action within each run, using process-calibrated rewards to train retrieval agents. Our retrieval graph shares the motivation of cross-query retrieval adaptation with GAM-RAG but differs in mechanism: we use graph traversal over explicit metadata edges with cross-graph optimization (reasoning graph rejection rates feeding into retrieval exclusion), whereas GAM-RAG uses statistical gain rules over a hierarchical index. Neither GAM-RAG nor EvalAct provides evidence-centric reasoning feedback; the reasoning graph, which is our primary contribution, operates on a different axis from adaptive retrieval and could be composed with either approach. 

Experience Replay and Memory Transfer. Experience replay (Lin, 1992; Mnih et al., 2015) stores past transitions 

(s, a, r, s ′) for gradient-based policy updates in reinforcement learning. Reasoning graphs store (a, k i, verdict , reason )

edges for context injection into the language model’s prompt. The parallel is conceptual: both reuse past experience to improve future decisions. The differences are in granularity (per-evidence reasoning vs. scalar rewards), feedback mechanism (context engineering vs. gradient updates), and the absence of any training loop in our approach. Recent work on the pre-conditions for memory-learning agents (Shah et al., 2025) found that weaker models benefit from memory induced by stronger models (64–87% success rate increase). Our experiments test this hypothesis for reason-ing graphs specifically; cross-model transfer did not improve accuracy in our setting ( −2.6pp; Section 5), suggesting that the mechanism may be architecture-dependent. 

Agent Behavioral Consistency. Recent empirical studies have quantified behavioral inconsistency in language model agents (Mehta, 2026a), finding that agents produce 2.0–4.2 distinct action sequences per 10 runs on identical HotpotQA inputs, with a 32–55 percentage point accuracy gap between consistent and inconsistent tasks. Follow-up work (Mehta, 2026b) demonstrated that consistency can amplify both correct and incorrect interpretations; 71% of failures in one model stem from “consistent wrong interpretation.” Our evidence profiles address this by filtering for correct outcomes only: the agent receives evaluation history from decisions that were verified as correct, avoiding the amplification of incorrect reasoning patterns.

## 3.2 Reasoning Graphs 

Definition 1 (Reasoning Graph) . A reasoning graph GR = ( V, E decided ∪ Eevaluated ) is a directed labeled property graph where: 

• V = A ∪ D ∪ K comprises agent nodes, decision nodes, and evidence item nodes. 

• Edecided = {(a, d, ϕ ) | a ∈ A , d ∈ D} are decided edges, where ϕ = ( confidence , timestamp , outcome ). The outcome field is populated after a downstream signal is received. 

• Eevaluated = {(a, k i, ψ ) | a ∈ A , k i ∈ K} are evaluated edges, where ψ = ( step , verdict , reason , confidence delta , action ref ).

Write Protocol. After the language model finishes reasoning and produces action at with structured chain of thought 

θt, all edges are written atomically: one decided edge (a, d t, ϕ t) and |Ct| evaluated edges {(a, k i, ψ i)}ki∈Ct . The agent framework prompts the language model to produce structured output (via tool calling, JSON mode, or a structured output schema) with per-evidence reasoning. Each structured object maps to one evaluated edge. The outcome field on the decided edge is populated when a downstream signal becomes available. 5Evidence-Centric Context Injection. This is the core mechanism that differentiates reasoning graphs from prior agent memory approaches. Given a new candidate set Ct = {k1, . . . , k m}, the system constructs an evidence profile 

for each item: 1. For each ki ∈ Ct, traverse all incoming evaluated edges: Ein (ki) = {e ∈ Eevaluated | e = ( ∗, k i, ∗)}.2. Filter for edges where the associated decided edge has outcome = correct .3. Construct the evidence profile P(ki): aggregate the verdict distribution, extract the most representative rea-soning for each verdict, and compute the reliability score R(ki, τ ) = |{ e ∈ Ein (ki) : e. verdict = used ∧

e. outcome = correct }| /|Ein (ki)|, where τ is the query type (implicit from the edges accumulated during same-type runs). 4. Inject P(ki) into the language model’s prompt alongside the raw evidence. The language model receives both the evidence item and its evaluation history. It does not reason from scratch about items that have been evaluated before. 

Why This Requires Graph Structure. The traversal starts from each evidence item ki and walks inward along all incoming evaluated edges, a reverse edge traversal with outcome-based filtering. A flat store of distilled strategies indexed by query embedding cannot support this operation because the strategies are disconnected from the specific evidence items. The evidence-centric approach guarantees that the feedback is pertinent whenever the same evidence item reappears in a candidate set, without relying on embedding similarity thresholds. 

Selection Policy. To bound context size: • If |Ein (ki)| = 0 , no profile is injected; the agent reasons from scratch for ki.• If |Ein (ki)| > T max , sample the N most recent evaluations from correct outcomes. • Total injected context is bounded by a token budget B. If the sum of profiles exceeds B, prioritize evidence items with the highest |Ein (ki)|.

Properties. Beyond the core injection mechanism, reasoning graphs provide architectural auditability (every deci-sion is traceable via graph traversal), require no retraining (the base model remains frozen), and self-improve as a side effect of normal operation. 

Evidence reliability scoring. The reasoning graph implicitly computes a dynamic quality score for every evidence item. Define the reliability score scoped to query type τ :

R(ki, τ ) = |{ e ∈ Ein (ki) : e. verdict = used ∧ e. outcome = correct }| |Ein (ki)| (1) This identifies consistently misleading items (low R), stale items (rejection reasons clustering around “outdated”), and high-value items (high R across many evaluations). The reasoning graph doubles as a knowledge base quality discovery tool. 

Graceful degradation. At cold start ( |Ein (ki)| = 0 for all ki), the agent receives no evidence profiles and rea-sons from scratch, identical to the baseline. Empirically, all methods are tied at ∼54% accuracy at zero coverage (Section 4.4), confirming no degradation from graph infrastructure alone.

6Pipeline Planner. Given query type τ , the pipeline planner optimizes future retrievals: 1. Traverse GP to find all retrieved via edges for past queries of type τ .2. For each distinct filter configuration f , compute the success rate: S(f ) = |{ e : e. filter = f ∧ e. outcome =

correct }| /|{ e : e. filter = f }| .3. Select f ∗ = arg max f S(f ) subject to |{ e : e. filter = f }| ≥ min support. 4. Cross-graph optimization : query GR for evidence items with rejection rate exceeding Rthresh across all evalu-ated edges for type τ . Add these to an exclusion list. 

Query Type Definition. We define query type τ as a categorical label assigned by a lightweight classifier. In our experiments, we use dataset-provided categories where available (HotpotQA: bridge vs. comparison) and embedding-based k-means clustering where not (MuSiQue: k = 25 ). Sensitivity analysis on clustering granularity appears in Appendix C. 

Cost Reduction. As the retrieval graph tightens the candidate funnel, fewer evidence items reach the agent, reducing both the token count in the prompt and the number of evaluated edges written per run. In high-reuse settings where passage overlap is dense, this produces substantial cost savings (47% reduction); in standard settings where pruning is less aggressive, graph traversal overhead can exceed savings. We quantify this trade-off through per-query token cost in Section 5. 

## 3.4 The Feedback Loop 

Both graphs compose into a self-improving system. On each run, the retrieval graph informs the pipeline planner (what to retrieve), while the reasoning graph informs the agent’s context (how to reason about what was retrieved). After the run, new edges are written to both graphs, making the next run better. 

Running Example. Consider a cluster of multi-hop questions about award-winning films (e.g., “Who directed the film that won the Palme d’Or in 2019?”, requiring retrieval of passages about the award ceremony and the winning director): • Run 1 : The agent retrieves 12 candidate passages and reasons from scratch. A passage about a similarly-titled but different film ( k37 ) is incorrectly marked used , but the agent arrives at the correct answer via other evidence. Evaluated edges and a retrieved via edge are written. • Run 2 : A related question is posed. The pipeline planner excludes 3 previously-rejected passages (funnel: 

12 → 9). For each remaining passage, the system injects an evidence profile. The agent sees: “ k37 was rejected in 1 prior correct decision because: describes a different film with a similar title, different year.” The agent avoids the same confusion. • Run N : The funnel narrows to 7 passages. Evidence profiles are rich. The agent consistently identifies the correct director without being misled by similarly-titled passages. To illustrate concretely, consider passage k37 , which contains: “The 2018 Palme d’Or was awarded to Shoplifters, directed by Hirokazu Kore-eda. . . ” . We observe that Aτ (n) improves over successive runs as profiles densify. Two mecha-nisms drive this trend: the retrieval graph gradually excludes evidence items with high rejection rates, tightening the candidate funnel; and the reasoning graph increases the evaluation context available per evidence item, reducing the probability of incorrect verdicts. Both effects compound when outcome signals are reliable and query types are stable, conditions that hold in our experimental setting and that we validate empirically in Section 4.4. We do not claim for-mal convergence guarantees; the trend is empirically demonstrated through the dose-response analysis (Section 4.5) and convergence curves (Figure 3). 

Why the Ceiling Is High. Define evidence profile coverage Pcov (n) as the fraction of items in Ct with at least one prior evaluation after n runs. As n grows, Pcov (n) increases toward 1 because the same items recur across queries of the same type. Once Pcov (n) ≈ 1, the agent never reasons from scratch on any evidence item. The remaining error rate is dominated by three sources: (a) novel items ( Pcov < 1), which shrinks over time; (b) misleading evaluation histories, which are rare because profiles aggregate across many runs; and (c) irreducible query ambiguity, the task-specific floor. For well-defined tasks, source (c) is small, and sources (a) and (b) shrink as the graph densifies. 

Variance Collapse. Define decision consistency Dτ (n) as the fraction of evidence items in Ct for which the agent’s verdict matches the majority verdict in the evidence profile. As profiles become richer, the majority signal strengthens and the agent’s decisions align with it more consistently. The variance of Aτ (n) decreases as Dτ (n) increases. This is what we mean by “deterministic”: not identical tokens, but identical verdicts on the same evidence, producing consistent outcomes. 

# 4 Experiments 

## 4.1 Setup 

Models. We use Claude Sonnet 4 as the primary (strong) model and Claude Haiku 4.5 as the secondary (weak) model for cross-model comparisons. Both are accessed via the Anthropic API with temperature 0 unless otherwise noted. The same base model is used for all configurations within each experiment to isolate the effect of the feedback architecture. 

Datasets. We evaluate on three experimental settings: • MuSiQue (Trivedi et al., 2022): 1,100 multi-hop questions requiring 2–4 reasoning steps, organized into 11 clusters of 100 questions each. This is the primary evaluation. • HotpotQA (Yang et al., 2018): 500 questions requiring reasoning over two Wikipedia passages (distractor setting), organized into 5 clusters (1 bridge, 4 comparison). • MuSiQue High-Reuse : 387 questions filtered from MuSiQue for high passage overlap, simulating deployment conditions with dense evidence reuse. Evaluated with both Sonnet and Haiku. 

Sequential Cluster Protocol. Standard benchmarks evaluate each query independently. To test the feedback loop, we introduce a sequential cluster protocol :1. Cluster the dataset into query types τ using embedding-based k-means clustering (MuSiQue: k = 25 ; High-Reuse: single cluster) or dataset-provided labels (HotpotQA: bridge vs. comparison). 2. Within each cluster, process queries sequentially: q1, q 2, . . . , q N . Reasoning and retrieval graph edges accumu-late across queries within each cluster. 3. Report accuracy (Acc: token F1 ≥ 0.8 for MuSiQue; exact match for HotpotQA) and mean token F1 across the full sequence. 8Outcome Signal. For all datasets: ot = correct iff token-level F1 > 0.8. Outcomes are computed automatically against ground truth answers. In deployment without ground truth, the outcome signal can come from user feedback, downstream task success (e.g., a generated SQL query that executes correctly), or an LLM-as-judge evaluator. The correct-outcome filter (Section 3.2) mitigates noise: profiles aggregate across many evaluations, so occasional incorrect outcome labels are diluted by the majority signal. Robustness to noisy outcome signals remains an empirical question (Section 6). 

Determinism Experiment. To directly measure verdict consistency, we run a separate experiment: 30 probe ques-tions evaluated 10 times each at 3 graph checkpoints (50, 200, 387 questions processed), across Vanilla RAG, Ours-RG, and Ours-Full, at temperatures 0, 0.5, and 0.7 (6,298/6,300 API calls succeeded). Probes are selected from baseline-incorrect questions using a greedy diversity algorithm that ensures pairwise Jaccard similarity ≤ 0.5 across probe passage sets, guaranteeing statistical independence for paired tests. 

## 4.2 Baselines and Configurations 

• Vanilla RAG : vector retrieval with no cross-run feedback. • Reflexion (Shinn et al., 2023): verbal self-reflection stored as flat text, retrieved by recency. • ReasoningBank (Ouyang et al., 2026): we reimplement the core mechanism (distilling strategy triples from each run’s chain of thought and retrieving the top-3 most similar strategies by cosine similarity of query embed-dings using BAAI/bge-base-en-v1.5) within our agent framework so that all methods share the same retrieval pipeline, prompt structure, and evaluation harness. This shared-framework design ensures that performance dif-ferences reflect the feedback mechanism, not confounding implementation choices (different retrievers, prompts, or LLMs). • Reasoning graph only (Ours-RG): evidence-centric injection via reasoning graph, no retrieval graph. This ablation isolates the contribution of evidence profiles. • Full system (Ours-Full): both reasoning and retrieval graphs with evidence-centric injection and pipeline plan-ner.

## 4.5 Dose-Response Analysis 

The overall +3.6pp improvement dilutes the signal across cold-start questions. To isolate the effect of evidence profiles, we stratify by evidence profile coverage Pcov , the fraction of candidate passages with at least one prior evaluation, comparing Ours-Full accuracy to Vanilla RAG accuracy on the exact same questions (controlled for difficulty). Table 2 and Figure 4 reveal a perfectly monotonic dose-response relationship. At zero coverage, all methods are tied at ∼54%, confirming no advantage without profiles. At 50%+ coverage, accuracy reaches 88.2% vs. 77.6% for Vanilla RAG on the same questions ( p < 0.0001 , 32:4 wins). The effect continues to strengthen at higher thresholds: +11.4pp at 60%+ coverage, +12.8pp at 70%+. Ours-Full significantly outperforms all baselines at 50%+ coverage: vs. Reflexion p = 0 .006 (23:7), vs. Ours-RG p = 0 .044 (15:5). The monotonicity, controlled design, and statistical significance collectively establish that evidence profiles cause 

accuracy improvement. The curve is unsaturated: the accuracy gap continues widening at the highest coverage levels we observe. Average evidence signal in this experiment is ∼30 evaluations per passage; in deployment with recurring queries, signal would be 100–500+, suggesting the observed gains are a lower bound. 

Component Decomposition. Table 3 decomposes the contribution of each graph component at each coverage level. The two components play complementary roles (Figure 5). At low coverage, the retrieval graph provides early benefit by pruning obviously irrelevant candidates even before profiles develop. At high coverage, the reasoning graph is the primary driver (64%), with the retrieval graph adding additional gains through pruning. Pruning aggressiveness scales with coverage: 4.3% of candidates pruned at 0% coverage → 29.8% at 50%+.

## 4.8 High-Reuse Experiment 

To test behavior under realistic deployment conditions with dense passage reuse, we construct a high-reuse subset of MuSiQue: 387 questions filtered for high passage overlap, evaluated with both Claude Sonnet 4 (strong model) and Claude Haiku 4.5 (weak model). 

Accuracy and Pareto Dominance. Ours-Full achieves the highest accuracy on both models while being 46% faster (8.6s vs. 15.8s) and 47% cheaper (1.6c vs. 3.0c/query) than Vanilla RAG. Across all 10 model ×configuration com-binations, only Ours-Full (Sonnet and Haiku) lie on the Pareto frontier; every other configuration is dominated on at least one axis. On 3-hop questions with 50%+ coverage ( n = 128 ), Ours-Full reaches 85.9% vs. VR 75.0% (+10.9pp, 

p = 0 .0005 , 14:0 wins), mirroring the standard MuSiQue multi-hop finding with stronger significance due to denser passage reuse. 

Model Capability and Weak-Model Results. Reflexion and ReasoningBank show negative trends on Haiku ( −2.6pp and −1.0pp respectively vs. Vanilla RAG), widening the Sonnet–Haiku gap from 6.7pp to 11.6pp and 10.3pp. Their 14 complex memory mechanisms disproportionately benefit the stronger model. Ours-Full is the only method that sig-nificantly improves Haiku (+4.4pp, p = 0 .017 ) without widening the capability gap (6.2pp, slightly narrower than baseline). Ours-Full vs. Reflexion on Haiku: p = 0 .0006 (42:15 wins); vs. ReasoningBank: p = 0 .003 (33:12). On the 148 questions where Haiku achieves 50%+ coverage, Haiku Ours-Full reaches 89.9% accuracy, matching Sonnet Vanilla RAG (89.2%) at ∼1/5 the cost ($2.27 vs. $11.47). Evidence profiles partially compensate for model capability. 

Efficiency and Pruning. Candidates are pruned from 20 to 6.1 over 387 questions (70% reduction vs. 22% in standard MuSiQue), driving 40% total token savings. Ours-Full vs. Ours-RG on accuracy is not significant ( p = 1 .0,11:10 wins), confirming that the retrieval graph’s role in high-reuse is pure efficiency: identical accuracy but 43% lower latency (8.6s vs. 15.1s). Speed gains are reuse-density-dependent; on standard MuSiQue (lighter pruning: 20 →15.5 candidates), Ours-Full is slower (42.3s vs. 15.8s for Vanilla RAG) because graph traversal overhead dominates. The efficiency story requires sufficient passage reuse for aggressive pruning to offset overhead. 

# 5 Analysis 

We probe the mechanisms behind reasoning graphs’ accuracy gains and examine the conditions under which the system succeeds or fails. 

Verdict Flip Mechanism. 47% of questions have ≥1 verdict flip, where evidence profiles cause the agent to evaluate a passage differently from Vanilla RAG. Questions with verdict flips improve by +6.6pp (69.2% vs. 62.6%); questions without flips improve by only +0.8pp (67.1% vs. 66.3%). Virtually all accuracy gain comes from questions where profiles changed passage judgments. Net +34 rescued questions (46 rescued vs. 12 damaged); rescued questions average 3.6 verdict flips vs. 2.7 for damaged. Of the 60 total rescued questions (Section 4.6), 73.3% had Vanilla RAG F1=0; the system rescues complete failures. 

Passage Overlap Predicts Improvement. Per-cluster passage overlap correlates significantly with accuracy im-provement (Pearson r = 0 .638 , p = 0 .019 ; Spearman ρ = 0 .631 , p = 0 .021 ; Figure 6). MuSiQue mean overlap is 0.504 vs. HotpotQA’s 0.055 (9 × higher), directly explaining why MuSiQue shows large improvements. The only negative cluster (musique 2, −2.0pp) has the second-lowest overlap (0.245). This provides a predictive framework: evidence profiles will benefit any domain where the same evidence items recur across queries. 

Per-Cluster Consistency. 10 of 11 clusters show improvement (paired t-test p = 0 .0049 , Wilcoxon p = 0 .0068 ). Hard clusters (Vanilla RAG < 65%) benefit more: +4.2pp vs. +3.0pp for easy clusters. The effect is not driven by outlier clusters. 

Convergence Trend. The advantage grows from +0.9pp in the first 10 questions to +5.1pp by position 26–50 as coverage develops, consistent with the dose-response relationship. At 0% coverage, all methods are tied at ∼54%. 

Decision Consistency ( Dτ ). Dτ climbs from 90.0% at Q1–25 to 97.4% at Q76–100 (Figure 7, right). Decision consistency strongly correlates with accuracy: Dτ ≥ 95% → 77.1% accuracy vs. Dτ < 50% → 25% accuracy (p = 0 .0005 ; Figure 7, left). In the high-reuse experiment, Dτ ≥ 90% (n = 228 , 59% of questions): Ours 85.1% vs. VR 78.1%, +7.0pp (McNemar p = 0 .0008 , 18:2 wins). 

Cross-Model Transfer. Transfer does not improve accuracy. In the high-reuse experiment, Haiku with Sonnet’s graph achieves 54.0% vs. Haiku standalone at 56.6% ( −2.6pp). Model-specific graph accumulation is preferred over cross-model sharing.

Sequential protocol. Our evaluation processes questions sequentially within clusters. Error reinforcement. If the outcome signal is noisy or systematically biased, the feedback loop could reinforce incorrect patterns (Mehta, 2026b). Our correct-outcome filter mitigates this; we validate robustness to 10–30% label noise in Appendix G. 

Context window limits. Evidence profiles consume tokens. The selection policy (Section 3.2) bounds total injected context, but aggressive truncation may lose useful signal. 

Profile granularity. Current profiles aggregate all past evaluations of an evidence item regardless of query context or recency. Structured filtering by query type, temporal weighting toward recent evaluations, or embedding-based profile matching could improve precision but are not yet implemented. 

When to Use Evidence Profiles. Our results identify four conditions under which evidence profiles provide the largest benefit: 1. High passage reuse (overlap > ∼0.3): validated by the passage overlap correlation ( r = 0 .638 ). Low-reuse domains (e.g., HotpotQA with overlap 0.055) show marginal improvement. 2. Sufficient query volume : 50+ questions per passage type are needed for profiles to develop meaningful cover-age. The dose-response is monotonic but requires data. 3. Multi-hop reasoning : 4-hop questions show +11.0pp vs. +1.7pp on 2-hop. The benefit scales with reasoning complexity. 17 4. Verdict consistency matters : in applications where deterministic evidence evaluation is valued (audit, compli-ance, safety-critical), the VVIR improvement of 7–8 percentage points ( p < 0.05 ) in a confound-free compari-son is directly relevant. 

# 7 Broader Impact 

Reasoning graphs produce more accurate agents with architecturally traceable reasoning, as every decision can be replayed via graph traversal. This is a safety and alignment benefit: organizations can audit agent decisions and identify systematic failures. The no-retraining property eliminates catastrophic forgetting risk. On the negative side, reasoning graphs store detailed evaluation traces that may surface sensitive information from the knowledge base. Deployments must consider data governance and privacy policies. The feedback loop could amplify biases present in the outcome signal: if certain evidence items are systematically misjudged early, the system may reinforce those misjudgments. 

# Ethics Statement 

All experiments use publicly available datasets (MuSiQue, HotpotQA) with no private or personally identifiable infor-mation. The risks identified in §7 (information leakage through stored traces and potential bias amplification) should be addressed through access controls on the reasoning graph and periodic audits of evidence profiles before production deployment.

# 8 Conclusion and Future Work 

We introduced reasoning graphs, a graph structure that persists per-evidence chain of thought as structured edges with bidirectional traversal, enabling evidence-centric feedback that improves agent accuracy and produces verdict-level determinism without retraining. Combined with retrieval graphs for pipeline self-optimization, the architecture forms a self-improving feedback loop with three validated properties: (1) dose-response accuracy improvement, reducing errors by 47% at high coverage; (2) verdict-level variance collapse, with profiles improving VVIR by 7–8 percentage points ( p < 0.05 , Wilcoxon) in a confound-free comparison across three temperature settings; and (3) Pareto-dominant efficiency in high-reuse settings. The system improves autonomously as a side effect of normal operation, with no gradient updates and no manual memory curation. Several directions remain for future work. Cross-model family evaluation : testing on GPT-4, open-source mod-els, and future architectures to validate generality. Cross-model transfer : our current results show negative transfer (−2.6pp); understanding why and developing effective transfer mechanisms is an open problem, and more broadly, whether multiple agents sharing a single reasoning graph could create shared institutional knowledge. Temporal awareness : the reliability score R(ki, τ ) currently weights all past evaluations equally. Recency-weighted profiles would bias toward recent evaluations, handling knowledge base evolution and evidence staleness in domains where facts change. Anomaly detection when evidence profile distributions diverge could flag distribution shift. Profile sophistication : evidence profiles could be sharpened through structured filtering by query context and passage meta-data, richer node labels beyond binary verdicts (e.g., reasoning step type, source reliability), and vector similarity over graph nodes to surface profiles for semantically similar but previously unseen passages. More broadly, modeling inter-item dependencies could capture which combinations of evidence produce correct answers, surfacing comple-mentary evidence sets alongside individual reliability signals. Retrieval strategy orchestration : the pipeline planner currently only excludes consistently-rejected passages. The retrieval graph already stores retrieved via edges 18 with pipeline metadata; extending this to track which retrieval strategy produced each candidate set (vector search, graph-based retrieval, BM25, hybrid approaches) would turn the pipeline planner from a passive filter into an active retrieval orchestrator that learns which strategy produces the best outcomes for each query type, optimizing across accuracy, latency, and cost. Answer-level determinism : our verdict-level determinism does not fully propagate to final answers (AIR remains lower than VVIR across all conditions); bridging this gap requires constrained answer synthesis.

# D Storage Growth and Pruning 

The reasoning graph grows linearly with the number of queries processed. Each query produces one decided edge and 

|Ct| evaluated edges (one per candidate passage). With |Ct|=20 , this is 21 edges per query. Over our experiments: • MuSiQue (1,100 queries): ∼23,100 edges (1,100 decided + 22,000 evaluated). • High-Reuse (387 queries): ∼8,127 edges before pruning. With retrieval graph pruning, |Ct| drops from 20 to 6.1 by the end of the run, reducing the edge write rate by 70%. • Retrieval graph : 1 retrieved via edge per query. Negligible relative to evaluated edges. Graphs are stored as JSON files. The largest graph (MuSiQue Ours-Full, 1,100 queries) occupies ∼12 MB on disk. Graph reads (backward traversal for evidence profiles) complete in <50 ms for all experiments, well within the latency budget dominated by LLM inference ( ∼3–15 s per query). For production workloads (millions of queries), a graph database backend (e.g., SurrealDB, Neo4j) would replace JSON storage. Edge pruning strategies (such as retaining only the N most recent evaluations per passage or decaying edge weights over time) would bound storage growth. Our selection policy (Section 3.2) already bounds read cost by sampling when |Ein (ki)| > T max ; a complementary write pruning policy would bound storage. We leave empirical evaluation of pruning strategies for future work. 

# E Prompt Templates 

The agent uses tool calling to produce structured per-evidence reasoning. The system message and tool schema are shown below. When evidence profiles are available, they are injected inline after each passage. 

System Message. 

22 You are a multi-hop question answering agent. You will receive a question and a set of passages. Your task is to: 1. Evaluate EVERY passage: determine whether it is relevant ("used") or irrelevant ("rejected") to answering the question. You must provide a verdict for every passage. 2. Provide a brief reason for each verdict. 3. Provide a confidence delta (-1 to 1) indicating how much each passage shifts your confidence in the answer. 4. Provide your final answer to the question. Some passages may include prior evaluation history from previous runs. This history shows how the passage has been judged before; use it to inform your evaluation, but apply your own judgment for the current question. 

Tool Schema. The model is forced to call a submit answer tool with two required fields: (1) evidence evaluations ,an array of objects each containing passage id (string), verdict (enum: used /rejected ), reason (string), and confidence delta (number ∈ [−1, 1] ); and (2) final answer (string). 

Evidence Profile Format. Profiles are injected immediately after each passage’s text:                           

> [EVIDENCE PROFILE] Evaluated 28 times in prior correct decisions. Verdict distribution: used 1/28, rejected 27/28. Reliability score: 0.04 Top reason for "rejected": "describes adifferent film with asimilar title"

ReasoningBank Distillation Prompt. After each query, the ReasoningBank baseline calls the model with a prompt containing the question, answer, outcome, and per-passage evaluations, and asks it to distill a reusable strategy triple 

{title, description, content } in JSON format. Strategies are embedded using BAAI/bge-base-en-v1.5 and the top-3 most similar strategies are injected into future queries. 

Reflexion Prompt. After each query, the Reflexion baseline appends a self-reflection to a text buffer. The 3 most recent reflections are injected at the top of the prompt for subsequent queries. Source code for all prompts and schemas will be released as supplementary material.

# G Outcome Signal Noise Robustness 

A concern with any feedback loop is sensitivity to noisy outcome signals. In deployment, the “correct” / “incorrect” labels used to build evidence profiles may come from unreliable downstream signals rather than gold labels. To test robustness, we re-ran the full sequential protocol (Ours-Full, Haiku, high-reuse dataset, 387 questions) with artificially corrupted outcome labels at three noise levels: 10%, 20%, and 30%. At each level, the specified fraction of outcomes are randomly flipped before being written to the graph (so evidence profiles accumulate noisy data) while the true outcome is preserved for accuracy measurement. At 10% noise, we ran three independent noise seeds to quantify variance from the stochastic flip pattern. At 10% noise, accuracy across three noise seeds averages 56.4% (std 3.5pp, range 53.0–59.9%), compared to 56.6% clean—a negligible −0.2pp mean difference. The high seed-to-seed variance (3.5pp) indicates that which specific labels are flipped matters more than the noise rate itself: some flip patterns happen to corrupt high-leverage early profiles while others leave them intact. At 20–30% noise, accuracy degrades by ∼4pp but remains at or above the Vanilla RAG baseline, confirming that evidence profiles degrade gracefully rather than catastrophically under noisy outcome signals. The profile aggregation mechanism provides inherent robustness: even when noisy labels cause some spurious profiles to be written, the majority of accumulated signal remains correct, and the profile selection policy dilutes bad data as more queries are processed. 24
