## 摘要
强化学习是真正扩展了大语言模型（LLM）智能体的能力，还是仅仅提高了它们的可靠性？对于静态推理，最近的研究倾向于后者：基础模型与强化学习模型的 pass@k 曲线在 k 较大时趋于收敛。我们探究这一结论是否适用于智能体工具使用场景，在这种场景下，T 轮交互能够实现重采样无法恢复的组合策略。我们引入了 PASS@(k,T)，这是一种联合改变采样预算 k 和交互深度 T 的二维指标，旨在将能力扩展与效率提升区分开来。我们的主要发现是，与静态推理的结果相反，工具使用中的强化学习确实扩大了能力边界：强化学习智能体的 pass 曲线高于基础模型，且差距在 k 较大时会拉大而非收敛。这种扩展专门针对组合式、序列化的信息采集任务；在更简单的任务上，强化学习的表现与先前研究的预测一致。在训练数据匹配的情况下，监督微调在同样的组合任务上反而导致了边界退化，从而确定了自主探索是能力提升的因果因素。机制分析表明，强化学习重新调整了基础策略分布的权重，使其向那些下游推理更常产生正确答案的子集倾斜，改进主要集中在智能体如何整合检索到的信息。这些结果调和了对 LLM 强化学习的乐观和悲观解读：两者在不同的任务类型上都是正确的。

## Abstract
Does reinforcement learning genuinely expand what LLM agents can do, or merely make them more reliable? For static reasoning, recent work answers the second: base and RL pass@k curves converge at large k. We ask whether this holds for agentic tool use, where T rounds of interaction enable compositional strategies that re-sampling cannot recover. We introduce PASS@(k,T), a two-dimensional metric that jointly varies sampling budget k and interaction depth T, separating capability expansion from efficiency improvement. Our main finding is that, contrary to the static-reasoning result, tool-use RL genuinely enlarges the capability boundary: the RL agent's pass-curve pulls above the base model's and the gap widens at large k rather than converging. The expansion is specific to compositional, sequential information gathering; on simpler tasks RL behaves as prior work predicts. Under matched training data, supervised fine-tuning regresses the boundary on the same compositional tasks, isolating self-directed exploration as the causal factor. Mechanism analysis shows RL reweights the base strategy distribution toward the subset whose downstream reasoning more often yields a correct answer, with the improvement concentrated on how the agent integrates retrieved information. These results reconcile optimistic and pessimistic readings of RL for LLMs: both are correct, on different task types.

### 1. 核心问题与研究动机
*   **核心疑问**：强化学习是真正扩展了 LLM 智能体的“能力边界”（即教会了模型原本完全不会的东西），还是仅仅提高了其“效率/可靠性”（即让模型更频繁地完成原本偶尔能做对的任务）？
*   **研究背景**：在静态数学推理领域，近期研究（如对 DeepSeek-R1 的分析）认为 RL 只是重新分配了概率质量，并未扩大能力边界（pass@k 曲线在 k 较大时趋于收敛）。
*   **研究动机**：作者怀疑这一“悲观”结论在智能体（Agentic）场景下可能不成立。因为智能体涉及多轮环境交互（T），这种组合策略的深度是单纯增加采样次数（k）无法弥补的。

### 2. 方法论：PASS@(k, T) 框架
*   **核心思想**：引入一个二维评估指标 **PASS@(k, T)**，将智能体的表现分解为两个维度：
    *   **k（采样预算）**：独立尝试的次数，用于衡量采样效率和可靠性。
    *   **T（交互深度）**：每轮尝试中允许与环境交互的最大步数，用于衡量处理复杂组合任务的能力。
*   **关键定义**：
    *   **能力边界 (Capability Boundary)**：当采样次数 $k \to \infty$ 时，模型能够解决的问题集合。
    *   **能力扩展 vs. 效率提升**：如果 RL 模型在 $k$ 很大时依然优于基础模型，则视为“能力扩展”；如果仅在 $k=1$ 时占优但在 $k$ 很大时被追平，则视为“效率提升”。
*   **算法流程**：使用 **GRPO**（群体相对策略优化）算法进行强化学习训练，通过环境反馈（正确与否）作为奖励信号，不依赖专家轨迹。

### 3. 实验设计
*   **任务分类**：
    *   **Category A (纯推理)**：MATH-500 数据集，不使用工具（对照组）。
    *   **Category B (独立检索)**：HotPotQA 比较类问题，需两次独立检索。
    *   **Category C (序列检索)**：HotPotQA 桥接类问题，需根据第一次检索结果决定第二次检索，具有组合依赖性。
*   **对比方法**：
    *   **Base**：Qwen2.5-7B-Instruct 基础模型。
    *   **SFT**：在 200 条专家轨迹上进行监督微调。
    *   **RL (GRPO)**：在相同的 200 个问题上通过自主探索训练。
*   **评测基准**：在 300 个测试问题上，针对不同的 $(k, T)$ 组合进行详尽采样（每个单元格采样 $n=64$ 次）。

### 4. 资源与算力
*   **硬件**：使用单张 **NVIDIA RTX 6000 Ada (48GB)** GPU。
*   **训练时长**：
    *   **RL 训练**：约 **55 小时**（1000 个梯度步，涉及大量环境交互回放）。
    *   **SFT 训练**：约 12 分钟。
*   **推理成本**：总计约 8 个 GPU 小时用于生成数万条评估轨迹。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 3 个任务类别、4 种模型变体、7 个采样梯度（k=1 到 64）和 5 个交互深度（T=0 到 5）。
*   **充分性分析**：
    *   通过 **Bootstrap 置信区间** 验证了结果的统计显著性。
    *   进行了**机制分析**（困惑度分解、策略多样性测量、跨策略交换实验），从因果角度解释了能力提升的来源。
    *   **公平性**：SFT 和 RL 使用完全相同的训练题目，排除了“数据暴露”导致的差异。

### 6. 主要结论与发现
*   **RL 确实扩展了能力边界**：在复杂的序列检索任务（Category C）中，RL 模型的 pass 曲线在 $k$ 增大时与基础模型的差距不断拉大，证明了其开发出了基础模型在海量采样下也无法触达的新策略。
*   **SFT 的负面效应**：在同等数据下，SFT 反而导致了能力边界的退化（在 Category C 上表现变差），这表明**自主探索**（Exploration）而非数据灌输是智能体能力进化的关键。
*   **任务依赖性**：在简单任务（Category A/B）上，RL 的表现符合“效率提升”模型；只有在需要深度组合策略的任务上，RL 才表现出“能力扩展”。
*   **作用机制**：RL 并没有发明全新的搜索词，而是优化了**如何整合检索到的信息进行推理**。它通过重新分配权重，让模型更倾向于选择那些后续推理更容易成功的路径。

### 7. 优点
*   **指标创新**：PASS@(k, T) 首次解耦了采样可靠性与交互深度，为智能体评估提供了科学的二维视角。
*   **严谨的对照**：通过匹配 SFT 和 RL 的训练数据，清晰地隔离了“学习信号”这一变量。
*   **调和矛盾**：成功解释了为什么之前的研究认为 RL 没用（因为任务太简单或非智能体化），而智能体领域觉得 RL 有用。

### 8. 不足与局限
*   **模型规模限制**：主要实验基于 7B 模型，尚不确定在 70B 或更大规模模型上，基础模型是否已经具备了足够的组合能力，从而缩小 RL 的扩展空间。
*   **环境单一**：仅使用了 BM25 检索工具，未在更复杂的 Web 浏览或代码执行环境中验证。
*   **训练量较小**：仅使用了 200 个问题的训练集，虽然足以证明原理，但未展示大规模强化学习的极限。
*   **温度敏感性**：实验固定在温度 0.7，未深入探讨解码策略（如 Top-p）对 PASS@(k, T) 表面形状的影响。
