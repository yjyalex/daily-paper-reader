## 摘要
尽管大语言模型（LLMs）取得了显著进展，但形式化软件验证中的自动证明生成在很大程度上仍未得到解决。虽然 LLMs 在自然语言处理（NLP）、计算机视觉和代码生成方面表现出色，但形式化验证仍需要大量的人力投入。交互式定理证明（ITP）要求在严格的逻辑约束下手动构建证明，这限制了其可扩展性；例如，验证 seL4 微内核耗费了数十年的努力。现有的基于 LLM 的方法试图自动化这一过程，但仍存在局限性。大多数方法依赖于单次生成或浅层检索，这可能适用于小型证明，但无法扩展到具有深层结构依赖的大型、相互依赖的验证任务。我们提出了 PROMISE（通过结构化嵌入进行证明挖掘），这是一个结构感知的框架，它将证明生成重新定义为证明状态转换上的有状态搜索。PROMISE 不采用表层检索，而是从证明状态和策略（tactic）转换中挖掘结构模式，从而在迭代搜索过程中实现兼容证明片段的检索与适配。我们在多个 LLM 后端上基于 seL4 基准测试对 PROMISE 进行了评估，并将其与 Selene 和 Rango 等现有系统进行了比较。结果表明，PROMISE 的表现始终优于以往的方法，实现了高达 +26 点的提升（186% 的相对增益），同时在不同模型间保持了鲁棒性，证明了结构感知证明挖掘在可扩展定理证明中的有效性。

## Abstract
Automated proof generation for formal software verification remains largely unresolved despite advances in large language models (LLMs). While LLMs perform well in NLP, vision, and code generation, formal verification still requires substantial human effort. Interactive theorem proving (ITP) demands manual proof construction under strict logical constraints, limiting scalability; for example, verifying the seL4 microkernel required decades of effort.   Existing LLM-based approaches attempt to automate this process but remain limited. Most rely on single-shot generation or shallow retrieval, which may work for small proofs but fail to scale to large, interdependent verification tasks with deep structural dependencies.   We present PROMISE (PROof MIning via Structural Embeddings), a structure-aware framework that reframes proof generation as a stateful search over proof-state transitions. Instead of surface-level retrieval, PROMISE mines structural patterns from proof states and tactic transitions, enabling retrieval and adaptation of compatible proof fragments during iterative search.   We evaluate PROMISE on the seL4 benchmark across multiple LLM backends and compare it with prior systems such as Selene and Rango. PROMISE consistently outperforms prior methods, achieving up to +26 point improvements (186% relative gain) while maintaining robustness across models, demonstrating the effectiveness of structure-aware proof mining for scalable theorem proving.

#### 2. 方法论
PROMISE（PROof MIning via Structural Embeddings）的核心思想是将证明生成视为一个**受结构相似证明引导的有状态迭代搜索过程**。
*   **核心流程**：
    1.  **结构化检索（Structural Retrieval）**：不只检索定理文本，而是检索“证明状态转换轨迹”。通过将当前证明目标（Goal）解析并嵌入（Embedding），在数据库中匹配具有相似目标演化模式的证明片段。
    2.  **名称检索（Name Retrieval）**：从当前 Isabelle 环境中提取可用的引理、定义和定理名称，并按功能（如简化、规则、WP 规则）分类，为 LLM 提供准确的上下文事实。
    3.  **迭代束搜索（Iterative Beam Search）**：
        *   从初始状态开始，利用 LLM 生成多个候选策略（Tactics）。
        *   通过 **Scala–Isabelle 桥接器**对候选步骤进行实时机器检查（Machine-checked Verification）。
        *   根据证明进度、简洁度和策略多样性进行评分，保留最优的 $B$ 个节点进入下一层搜索。
    4.  **接地式少样本提示（Grounded Few-shot Prompting）**：将检索到的结构化模板和验证过的名称组合成高密度的 Prompt，引导模型生成符合逻辑约束的步骤。

*   **数据集/场景**：基于 **seL4 微内核（l4v 项目）** 的 Isabelle/HOL 证明。

*   **实验规模**：在 3 种不同能力的 LLM 上进行了跨 P1/P2/P3 等级的全面测试。
*   **充分性**：实验涵盖了从开源小模型（7B）到闭源大模型（GPT-4）的对比，验证了框架的**模型无关性**。

#### 6. 主要结论与发现
*   **性能显著提升**：PROMISE 在大多数设置下显著优于基准。在 Qwen2.5-Coder-7B 上，P1 成功率达到 77%（比 Selene 提升 47 个百分点）；在 GPT-3.5 上，P2 成功率达到 40%（相对增益达 186%）。
*   **模型鲁棒性**：PROMISE 缩小了不同模型间的差距。即使是 7B 的开源模型，在 PROMISE 框架下的表现也接近甚至超过了在传统框架下的 GPT-4。
*   **结构信息的重要性**：实验证明，检索“证明状态转换轨迹”比单纯检索“引理文本”更能有效引导 LLM 完成复杂的系统级验证。

#### 7. 优点
*   **结构感知**：首次将证明状态的动态演化作为检索的核心信号，更贴近人类专家的思维方式。
*   **高可靠性**：结合了 LLM 的生成能力与 Isabelle 的形式化检查，确保生成的证明是 100% 正确且可重放的。
*   **自适应搜索**：通过束搜索和动态调整预算，能够处理长程推理任务。

#### 8. 不足与局限
*   **极高难度任务挑战**：在 P3 等级（最复杂的安全证明）上，虽然优于基准，但成功率仍仅为 30.4%，说明对于极长路径的推理仍有提升空间。
*   **Prompt 过度约束风险**：在 GPT-4.1 上的实验发现，过于严格的结构化 Prompt 可能会限制极强模型的自主推理能力（在 P2 任务中略逊于 Rango）。
*   **环境依赖**：高度依赖 Scala–Isabelle 桥接器，对于其他证明助手（如 Lean 或 Coq）的泛化性虽有讨论但未在本文中直接实验验证。
