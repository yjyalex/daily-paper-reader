# Abstract 

Existing multi-agent video generation systems use LLM agents to orchestrate neural video generators, producing vi-sually impressive but semantically unreliable outputs with no ground truth annotations. We present an agentic sys-tem that inverts this paradigm: instead of generating pix-els, the LLM constructs a formal Graph of Events in Space and Time (GEST) — a structured specification of actors, actions, objects, and temporal constraints — which is then executed deterministically in a 3D game engine. A staged LLM refinement pipeline fails entirely at this task (0 of 50 attempts produce an executable specification), motivating a fundamentally different architecture. The key architectural insight is a separation of concerns: the LLM handles nar-rative planning through natural language reasoning, while a programmatic state backend enforces all simulator con-straints through validated tool calls, guaranteeing that ev-ery generated specification is executable by construction. Our system uses a hierarchical two-agent architecture — a Director that plans the story and a Scene Builder that con-structs individual scenes through a round-based state ma-chine — operating over exploration and building tools that provide paginated access to the simulation environment’s capabilities. Dedicated Relation Subagents further popu-late the logical and semantic edge types of the GEST for-malism that procedural generation leaves empty, making this the first approach to exercise the full expressive ca-pacity of the representation. The system can operate from an optional text prompt or autonomously — the Director Agent explores the simulation world, conceives a narrative, and produces both the executable GEST and a natural lan-guage story description. We evaluate in two stages: au-tonomous generation compared against procedural random baselines via a 3-model LLM jury, where agentic narratives win 79% of text comparisons and 74% of video comparisons against procedural baselines; and seeded generation where the same textual specification is given to the agentic system, VEO 3.1, and WAN 2.2, with human annotations confirming that engine-generated videos substantially outperform neu-ral generators on physical validity (58% vs 25% and 20%) and semantic alignment (3.75/5 vs 2.33 and 1.50). The system produces multi-actor narrative videos with perfect spatiotemporal annotations (RGB, spatial relation graphs, event-frame mappings) at zero marginal cost — annotations that neural generators fundamentally cannot provide. 

# 1. Introduction 

Multi-agent AI systems for video generation have prolif-erated rapidly, with frameworks such as StoryAgent [7], MAViS [17], and DreamRunner [18] orchestrating spe-cialized agents across scriptwriting, character design, and video rendering. These systems have demonstrated increas-ingly sophisticated narrative planning capabilities. How-ever, they all share a fundamental limitation: every agent in the pipeline ultimately drives a neural video genera-tor , producing pixels with no semantic guarantees. Objects spontaneously appear and disappear, actors morph between frames, and temporal ordering is routinely violated [3, 16]. We propose an inversion of this paradigm. Instead of us-ing agents to generate pixels, we use agents to construct a 

formal event graph specification — a Graph of Events in Space and Time (GEST) [12] — that encodes actors, ac-tions, objects, locations, and temporal constraints in an ex-plicit, inspectable structure. This specification is then exe-cuted deterministically in a 3D game engine [4], producing multi-actor narrative videos with perfect spatiotemporal an-notations (RGB, per-frame spatial relation graphs, event-to-frame temporal mappings) as a byproduct of execution. The central challenge is that GESTs must be both narra-tively coherent (telling a meaningful story) and simulator-valid (respecting the engine’s action chains, object lifecy-cles, POI capacities, and temporal constraint rules). We first built a staged LLM refinement pipeline — six specialized agents chained through LangGraph, each validating struc-tured outputs via Pydantic — and found that it fails entirely at this task: across 50 generation attempts, zero produced a simulator-valid GEST. The failure modes compound across stages (hallucinated actions, invalid chains, object lifecycle violations, temporal cycles, context overflow), and post-hoc correction requires backtracking through dependent state at superlinear cost. Section 4 describes the pipeline and its failure modes in detail; this outcome motivated a fundamen-tally different architecture. Our key insight is a separation of concerns : the LLM handles narrative decisions (what should happen), while a programmatic state backend handles constraint enforce-ment (what is allowed to happen). The LLM operates ex-clusively through tool calls — creating actors, starting ac-tion chains, adding temporal constraints — and every tool call is validated and executed by the same procedural gen-erator [4] that guarantees executability for random genera-tion. Invalid operations are rejected before they can affect the graph. This means every GEST produced by the system is executable by construction, regardless of what the LLM attempts. Our contributions are: • The staged LLM pipeline and the analysis of its failure modes, which directly motivated the agentic architecture. • A hierarchical two-agent architecture for agentic video generation: a Director Agent that explores the simulation world and plans multi-scene stories, and a Scene Builder Subagent that constructs individual scenes through a round-based state machine with tool-enforced constraints, augmented by Relation Subagents that populate the logi-cal and semantic edge types of the GEST formalism that procedural generation leaves empty, making this the first approach to exercise the full expressive capacity of the representation. • A tool-based constraint enforcement layer that repurposes a procedural GEST generator as a state backend, pro-viding transaction-based chain construction, POI capac-ity tracking, object lifecycle management, temporal cycle detection, and interaction coordination. • A two-stage evaluation: (1) autonomous generation com-pared against procedural baselines via a 3-model LLM jury on narrative coherence, and (2) seeded generation where the same text is given to the agentic system, VEO 3.1, and WAN 2.2, with both LLM jury comparisons and human annotations showing that the engine substan-tially outperforms neural generators on physical validity and semantic alignment. Prior work demonstrated text-to-GEST generation as a proof of concept [12]; we present the first end-to-end sys-tem that uses LLM agents to autonomously construct ex-ecutable event graph specifications for deterministic 3D video generation — producing multi-actor narrative videos with dense spatiotemporal annotations (RGB, spatial re-lation graphs, event-frame mappings) and an architecture extensible to depth, pose, and instance segmentation [4], rather than driving neural pixel generators. This transforms video generation from a programming task, where story structure is determined by the procedural generator’s code, into a specification task, where natural language shapes the story without any code changes. The system achieves 80% end-to-end success in seeded mode but remains at approxi-mately 50% in fully autonomous generation, indicating that stability improvements are needed before unattended large-scale corpus production.

# 3. Background: GEST and the GEST-Engine 

A GEST [12] is a directed graph G = ( V, E) where nodes represent events and edges encode temporal, spatial, and se-mantic relationships. Each event node specifies an action, a performer, participating entities, a location, and proper-ties. Special Exists nodes declare actors (with gender and name) and objects (with type). Temporal constraints in-clude before , after , same time , and concurrent ,drawn from Allen’s interval algebra [1]. The GEST-Engine [4] executes GESTs deterministically within GTA San Andreas via Multi Theft Auto (MTA). It implements a four-stage pipeline: (1) graph parsing and val-idation via set-cover episode selection, (2) entity and action grounding with chain-ID-based object identity, (3) tempo-ral orchestration via Floyd-Warshall constraint satisfaction, and (4) execution with multi-modal artifact capture. The en-gine produces RGB video, per-frame pairwise spatial rela-tion graphs, event-to-frame temporal mappings, and natural language descriptions at zero marginal annotation cost. A procedural random generator [4] reads the engine’s capability registry—which catalogs all available episodes, POIs, action chains, object types, and actor skins—and pro-duces random GESTs that are executable by construction by following valid next-action sequences at each POI. In this work, we significantly extend the procedural generator into a state backend for the agentic system. While the core state tracking (actors, events, temporal constraints, object lifecycles, POI capacities) originates from the random gen-erator, we adapted it with a delegation interface that exposes validated operations to external agents, added transaction-based chain construction with rollback on failure, enriched the constraint enforcement with cycle detection and inter-action coordination, and introduced scene-level state man-agement for multi-scene stories. These extensions ensure that agentic generation benefits from the same executability guarantees as procedural generation while supporting sub-stantially more complex narrative structures. Procedural generation produces event-dense, structurally valid stories, but has no authored narrative intent: actors execute random sequences of available actions, and story-level structure — if present — is latent in the action se-quence rather than explicitly authored. The agentic sys-tem described in Section 5 closes this gap by introduc-ing an LLM Director that plans stories with explicit in-tent, motivation, and causal structure, while preserving the executability-by-construction guarantee of the underlying state backend. 

# 4. The Staged LLM Approach 

Before arriving at the agentic architecture, we built a staged LLM refinement pipeline: a LangGraph workflow chain-ing six specialized stages — Concept (abstract hierarchical 3GEST with parent/leaf scenes and actor archetypes), Cast-ing (skin assignment from the engine’s catalog), Episode Placement (mapping scenes to valid simulation episodes), 

Setup (off-camera preparation actions and backstage posi-tioning), Screenplay (translating abstract narrative into con-crete action sequences), and Scene Detail (expanding each leaf scene into a complete executable GEST) — each im-plemented as a single prompt-and-parse step with struc-tured output validation via Pydantic. The pipeline pro-duced narratively coherent GESTs with logical structure and character motivation, but across 50 generation attempts 

zero produced an executable specification. The failure modes compounded across stages: hallucinated actions 

(e.g., “WalkTo” instead of “Move”), invalid action chains 

(“TypeOnKeyboard” without a prior “SitDown” and “Open-Laptop”), object lifecycle violations (putting down objects never picked up, two actors holding the same object), tem-poral cycles that the Floyd-Warshall solver cannot resolve, and context overflow from the 14,000-line capability reg-istry. The fundamental problem is that LLMs cannot main-tain precise state coherence across the 50–200 events that a multi-actor story requires; each stage makes locally rea-sonable decisions, but accumulated state — which actors are where, what they hold, which POIs are occupied, which temporal constraints are in effect — drifts from validity as the GEST grows, and post-hoc correction requires back-tracking through dependent state at superlinear cost. The key lesson motivated the agentic architecture: the LLM should handle narrative decisions (what should happen), while a programmatic backend should handle state-tracking and constraint enforcement (what is allowed to happen) —operationalized in Section 5 through tool calls validated by the same procedural generator that guarantees executability for random generation. 

# 5. Method: Agentic GEST Generation 

5.1. Architecture Overview 

The system uses a hierarchical two-agent architecture built on LangGraph for workflow orchestration and deepagents 1

for subagent delegation. A Director Agent handles high-level story planning: exploring the simulation world, se-lecting episodes and regions, picking actor skins, creating actors, and determining the sequence of scenes. For each scene, it delegates detailed event construction to a Scene Builder Subagent , which builds action chains, interactions, and temporal constraints through tool calls. After all scenes, the Director finalizes the GEST by linking scene boundaries with temporal constraints. Both agents operate exclusively through function call-ing: every action—creating an actor, starting a chain, adding a constraint—is a tool call validated by the state

5.2. The Director Agent 

The Director receives a seed text (an optional story prompt) and a generation configuration (target scenes, actors). Its workflow has four phases: 

Exploration. The Director uses read-only, paginated ex-ploration tools to discover the simulation world: browsing episodes, examining regions and their objects, inspecting POI action chains, and browsing character skins by gender (each with a visual description, e.g . “a middle-aged white man with blonde hair in a black police uniform”). Pagi-nation is essential—the capability registry is too large for a single prompt, so the agent must actively navigate the space. 

Casting. The Director creates the story structure via 

create story and create actor calls, specifying each actor’s name, gender, skin, and starting region. All actors are created before any scene is delegated. 

Scene building. Scenes are processed sequentially. For each, the Director calls start scene with the episode, region, and participating actors, then delegates to the Scene Builder with a natural language task description ( e.g .“Build a scene where James and Sarah have lunch in the kitchen”). After the Scene Builder returns, the Director han-4dles follow-up tasks and calls move actors for location transitions. 

Finalization. The Director calls finalize gest 

to link scene boundaries with cross-scene temporal con-straints, producing the complete GEST. 

5.3. The Scene Builder Subagent 

The Scene Builder receives an isolated context: scene ID, episode, region, actors, and a narrative description. It does not see the full story—only its assigned scene—reducing context requirements and preventing accidental modifica-tion of other scenes. Events are constructed through a round-based state ma-chine: 1. start round() : begins a temporal unit, returning the current state of all actors (posture, held objects, loca-tion). 2. For each actor: start chain(actor, POI) 

(which returns the valid next actions at the selected POI) → one or more continue chain(actor, action) (each advancing the chain by one ac-tion and returning the next valid actions) →

end chain(actor) to commit. 3. do interaction(actor1, actor2, type) 

for synchronized two-actor actions (handshakes, hugs, conversations). 4. start recording /stop recording for camera control on committed events. 5. add temporal dependency or 

add starts with for cross-actor ordering. 6. end round() : closes the round, adding cross-actor temporal ordering. Multiple actors can have active chains simultaneously, and the subagent can interleave calls across actors. After all rounds, end scene() returns a summary to the Direc-tor. The Scene Builder cannot create actors, move actors between regions, or finalize the GEST — these responsibil-ities belong exclusively to the Director, enforcing a clean separation between story-level and scene-level concerns. 

5.4. Relation Subagents 

After each scene is built and after the final GEST is as-sembled, the Director optionally delegates to two additional subagents for enriching the graph with higher-level anno-tations. A Logical Relations Agent adds causal and de-pendency relations between events (causes, enables, pre-vents, requires), capturing why events occur in relation to each other. A Semantic Relations Agent adds narrative co-herence relations using free-text types (observes, interrupts, motivates, sets context for, contrasts with), capturing how events relate at the story level rather than the temporal level. Both subagents receive a list of committed event IDs and operate through the same tool interface, ensuring that only valid event references are used. These relations are optional — the GEST is executable without them — but they populate the logical and semantic edge types defined in the GEST formalism that procedural generation never produces, bridging the gap between the representation’s ex-pressive capacity and the generator’s output. To the best of our knowledge, this makes the agentic system the first ap-proach to exercise the full expressive capacity of the GEST formalism in a single end-to-end pipeline. 

5.5. Tool-Based Constraint Enforcement 

The building tools implement a transaction-based state machine with four states (IDLE , STORY CREATED ,

IN SCENE , IN ROUND ). Each tool call verifies the correct state before executing. Key enforcement mechanisms in-clude: 

Actor state tracking. Posture (standing/sitting/lying), held objects (with type and origin region), current location. Actors must be standing to start chains or interactions, and held objects can only be put down in their origin region. 

Transactional chains. continue chain creates events in temporary buffers; only end chain commits them to the GEST. Failed chains are silently discarded. 

POI capacity. Exclusive-use objects (chairs, beds, gym equipment) are tracked per region, preventing two actors from using the same object. 

Temporal cycle detection. Before adding a before re-lation, the system walks the full dependency graph to check whether the new edge would create a cycle. Circular depen-dencies are rejected with an explanatory error. 

Interaction rules. Both actors must be standing, in the same region, not holding spawnables. No consecutive in-teractions without an intervening chain. The Give action requires an explicit receiver identifier and automatically cre-ates a synchronized INV-Give event for the receiver. 

Spawnable lifecycle. Phone and cigarette actions are atomic sequences ( TakeOut →Use →Stash ) that lock the actor until completion. 

5.6. Exploration Tools 

Exploration tools provide read-only paginated access to the capability registry: • get episodes , get regions , get pois : browse environments. • get poi first actions , get next actions :discover valid action chains at specific POIs. • get skins : browse actor appearances by gender. • get spawnable types ,

get interaction types : discover available object and interaction categories. • get simulation rules : retrieve constraint docu-mentation. 5The explore-before-plan pattern prevents hallucination: the agent must verify through tool calls that a POI has a specific action before including it in a plan, addressing a primary failure mode of staged LLM approaches. 

# 6. Experiments 

6.1. Data 

Stage 1: Autonomous generation. We generated stories using the agentic system without text prompts—the Di-rector Agent autonomously explored the simulation world, conceived narratives, and built GESTs. From multiple gen-eration batches, we obtained 5 unique story graphs, each simulated 5 times in the GEST-Engine, producing 25 agen-tic videos. Each agentic story carries a root narrative (the Director’s authored intent, e.g . “Andrei receives an unex-pected phone call while in his bedroom. He answers and ex-its to the hallway. Iulia arrives and knocks on the door, wait-ing for a response.”) and a Video Description from GEST (VDG) description—a post-simulation textual description generated by the proto-language pipeline [4, 11]. As baseline, we selected procedurally generated sto-ries [4] matched 1:1 to the agentic groups by actor count. Procedural stories have VDG descriptions (same pipeline) but no authored narrative. To enable a fair narrative-level comparison, we generated reconstituted narratives for the procedural stories by prompting GPT-4o to infer a story-level intent from the VDG description, using one-shot prompting with an agentic example. 

Stage 2: Seeded generation. We selected 25 procedu-ral stories from the GEST-Engine sample corpus [4], di-verse by scene type (classroom, gym, garden, house, mixed) and actor count (2–6). The procedural success rate for this subset was approximately 74% (roughly 34 generation at-tempts to obtain 25 successful simulations, computed post-hoc for comparison purposes). For each, we generated a VDG description and used it as a seed text for the agen-tic system—the Director received the procedural story’s de-scription as narrative input and generated a new GEST re-interpreting that narrative. The same VDG description was also used as the prompt for VEO 3.1 [6] and WAN 2.2 [16]. This produced four video sources from the same textual specification: the original procedural video, the agentic re-interpretation, and the VEO and WAN neural generations. Of the 25 agentic generation attempts, 20 completed suc-cessfully (80%). We collected human annotations on the procedural, VEO, and WAN videos (34 annotations from 10 evaluators), evaluating physical validity (binary) and se-mantic alignment (1–5 Likert scale). Inter-annotator agree-ment was 66.7% for validity and 83.3% within ±1 scale point for semantic alignment.

Prior work on the same engine val-idated this approach through a human evaluation study [12] that established semantic quality rankings later confirmed by LLM jury evaluations, where the jury consistently pre-ferred human references over machine conditions, demon-strating alignment with human judgment. Using three fron-tier models from different providers reduces single-model bias. Position bias is controlled by randomizing which text/video is assigned to position A vs B using a determin-istic seed per pair. 

Text jury. Each pair of descriptions is presented anony-mously as “Text A” / “Text B”. Judges score both 0–100 with no ties, evaluating narrative coherence, character mo-tivation, temporal logic, and richness. # 7. Discussion 

7.1. Complementary Explicit and Implicit World Models 

Our results suggest that explicit world models and neural generators address fundamentally different aspects of video generation. Neural models excel at visual realism but can-not maintain semantic consistency across complex multi-actor narratives. Formal specifications executed in 3D en-8gines guarantee semantic correctness and produce dense spatiotemporal annotations as a byproduct, but at the cost of visual fidelity. The most promising direction is their com-bination: using GESTs as structured conditioning signals for neural video generators, preserving semantic reliability while achieving photorealistic output. 

# 8. Limitations and Next Steps 

8.1. Generation Stability 

Autonomous generation succeeds about half the time. The dominant failure mode is repetitive planning — the agent exhausts its alternatives for expressing a scene within en-gine constraints and gives up. This is an engineering prob-lem, not an architectural one: better variation strategies, ex-plicit loop detection, and checkpoint-resume mechanisms should address it without changing the fundamental archi-tecture. 

8.2. The Fidelity Gap 

Narrative fidelity (1.89/5 event coverage in autonomous mode, 3.12/5 when seeded) reflects a gap between what the Director narrates and what the engine can render, caused by expressive channels that the current platform does not ex-pose. 

Cinematography. The engine captures from a fixed camera, with no tracking shots, zooms, or rack focus. Real storytelling is expressed through camera work: a tracking shot on a character walking away, a rack focus shifting at-tention between two speakers, a lingering wide shot of an empty room. 

Temporal nuance. The GEST temporal model draws from Allen’s interval algebra — before , after ,

same time , concurrent — but cannot express frame-precise spatial triggers such as “as she enters, he looks away,” which demand synchronization of two actors rela-tive to a spatial event. 

Atmospheric expression. Time of day, weather, light-ing shifts — a sunset, rain starting, a room darkening — carry narrative weight at zero dialogue cost. The current en-gine does not modify environmental conditions during exe-cution. 

Action blending. Real humans perform actions simulta-neously and fluidly: smoking while talking, walking while looking at a phone. The MTA animation system plays atomic sequences — one action completes, then the next starts — producing the robotic quality that is the single largest contributor to the visual realism gap. These are platform constraints, not architectural ones. The agentic system transfers directly to any engine that ex-poses these channels, and a GTA V port via FiveM or im-plementations in Unreal or Unity with animation blending and cinematic cameras would substantially close the fidelity gap without changing the pipeline. 

8.3. Cost 

Each agentic story currently costs approximately $0.25 in frontier-model API calls, with clear headroom through smaller fine-tuned models that internalize the simulation constraints. 

# 9. Conclusion 

We presented an agentic system that constructs formal event graph specifications (GESTs) for deterministic 3D video generation, inverting the dominant paradigm of using agents to drive neural pixel generators. The separation of con-cerns between narrative planning (LLM) and constraint en-forcement (programmatic state backend) guarantees that ev-ery generated specification is executable by construction, achieving 80% end-to-end success rate with seeded genera-tion versus 0% for a staged LLM pipeline. Our two-stage evaluation reveals complementary find-ings. In both stages, the LLM jury consistently shows that procedural generation wins at the execution level (denser VDG descriptions), but agentic narratives carry genuine story structure that restores or reverses this advantage when used as context. Human annotations on seeded generation confirm that the same textual specification produces physi-cally valid, semantically faithful videos when executed by the engine (58% validity, 3.75/5 alignment) but physically implausible outputs when rendered by VEO 3.1 (25%, 2.33) or WAN 2.2 (20%, 1.50). Seeding the Director with a con-crete specification roughly doubles narrative fidelity com-pared to autonomous generation. Beyond these individual results, this work brings to-gether every direction the GEST formalism supports within a single system: the Director’s natural-language planning produces GESTs, the deterministic engine executes them into video with dense per-frame spatial and temporal an-notations, and the proto-language pipeline describes what was rendered back into natural language. With the Relation Subagents populating the logical and semantic edge types that procedural generation never produces, the agentic sys-tem is the first to exercise the full expressive capacity of the GEST formalism in a single end-to-end pipeline — closing the loop from story to specification to video and back, with every component active.
