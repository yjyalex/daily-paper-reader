## 摘要
强化学习 (RL) 已广泛用于训练多轮交互任务中的大语言模型 (LLM) 智能体，但其样本效率受到稀疏奖励和长时序的严重限制。同策略自蒸馏 (OPSD) 通过提供来自特权教师（可访问标准答案）的密集词元级监督来缓解这一问题。然而，这种固定的特权信息无法捕捉智能体任务中多样化的有效策略，且简单地将 OPSD 与 RL 结合往往会导致训练崩溃。为了解决这些局限性，我们引入了 Skill-SD，该框架将智能体自身的轨迹转化为动态的、仅限训练的监督信息。已完成的轨迹被总结为简洁的自然语言技能，用于描述成功的行为、错误和工作流。这些技能作为动态特权信息仅对教师进行引导，而学生始终在普通任务提示下行动，并通过蒸馏学习将指导内化。为了稳定训练，我们推导了一种重要性加权的反向 KL 损失，以提供梯度正确的词元级蒸馏，并动态地使教师与不断进步的学生保持同步。在智能体基准测试上的实验结果表明，Skill-SD 显著优于标准的 RL 基准，在 AppWorld 和 Sokoban 任务上分别提升了原始 GRPO (+14.0%/+10.9%) 和原始 OPD (+42.1%/+40.6%)。项目主页：https://k1xe.github.io/skill-sd/

## Abstract
Reinforcement learning (RL) has been widely used to train LLM agents for multi-turn interactive tasks, but its sample efficiency is severely limited by sparse rewards and long horizons. On-policy self-distillation (OPSD) alleviates this by providing dense token-level supervision from a privileged teacher that has access to ground-truth answers. However, such fixed privileged information cannot capture the diverse valid strategies in agent tasks, and naively combining OPSD with RL often leads to training collapse. To address these limitations, we introduce Skill-SD, a framework that turns the agent's own trajectories into dynamic training-only supervision. Completed trajectories are summarized into compact natural language skills that describe successful behaviors, mistakes, and workflows. These skills serve as dynamic privileged information conditioning only the teacher, while the student always acts under the plain task prompt and learns to internalize the guidance through distillation. To stabilize the training, we derive an importance-weighted reverse-KL loss to provide gradient-correct token-level distillation, and dynamically synchronize the teacher with the improving student. Experimental results on agentic benchmarks demonstrate that Skill-SD substantially outperforms the standard RL baseline, improving both vanilla GRPO (+14.0%/+10.9% on AppWorld/Sokoban) and vanilla OPD (+42.1%/+40.6%). ### 1. 核心问题与整体含义
在训练多轮交互的 LLM 智能体时，强化学习（RL）常面临**奖励稀疏**和**长时序决策**的挑战。现有的自蒸馏方法（如 OPSD）通常依赖固定的标准答案作为“特权信息”，但这不适用于路径多样、策略灵活的智能体任务。此外，直接将蒸馏与 RL 结合容易导致训练不稳定甚至崩溃。
**Skill-SD 的核心意义**在于：它将智能体自身的历史轨迹总结为动态的“技能”，作为教师模型的特权引导，通过一种改进的、梯度正确的蒸馏损失函数，让学生模型在不依赖外部提示的情况下内化这些复杂策略。

### 2. 方法论
Skill-SD 框架包含以下关键技术：
*   **技能引导的教师（Skill-Conditioned Teacher）：** 将完成的轨迹总结为包含“成功分析、错误分析、黄金工作流”的自然语言技能。在训练中，教师模型输入“任务提示 + 检索到的技能”，而学生模型仅输入“任务提示”。
*   **动态自演进：** 教师模型并非固定，而是定期从最新的学生模型中同步参数，确保特权信号随学生能力的提升而共同进化。
*   **重要性加权反向 KL 损失（Importance-weighted reverse-KL）：** 针对教师和学生使用不同 Prompt 导致的分布偏移，推导了重要性采样权重 $\rho$。该损失函数修正了传统 $k_3$ 估计器的梯度偏差，确保词元（Token）级别的蒸馏梯度是无偏的。
*   **异步技能生成：** 使用辅助 LLM 异步总结轨迹并存入基于 UCB（置信区间上界）算法索引的技能库，平衡技能的探索与利用。

### 3. 实验设计
*   **数据集/场景：**
    *   **AppWorld：** 模拟多应用 API 协调任务，侧重于工具调用和状态管理。
    *   **Sokoban（推箱子）：** 空间规划任务，具有高度不可逆性，侧重于长程逻辑规划。
*   **基准（Benchmark）：** 使用 Qwen3-4B-Instruct-2507 作为基础模型。
*   **对比方法：**
    *   **Vanilla GRPO：** 标准的群体相对策略优化。
    *   **Skill-Augmented GRPO：** 直接在学生 Prompt 中加入技能（不使用蒸馏）。
    *   **Vanilla OPD：** 仅使用同策略蒸馏，不含 RL 奖励。
    *   **Base Model：** 未经微调的原始模型。

### 4. 资源与算力
*   **硬件：** 使用了 **8 张 NVIDIA L40S GPU**。
*   **模型：** 基础模型为 Qwen3-4B。
*   **训练细节：** 采用了异步架构以重叠 API 调用（技能总结）与梯度计算的时间，减少了挂钟时间开销。具体训练时长未明确给出，但提到在 AppWorld 上训练了约 300 步，Sokoban 约 180 步。

### 5. 实验数量与充分性
*   **实验规模：** 在 AppWorld 的 90 个训练任务和 Sokoban 的 96 个关卡上进行训练，并在各自的开发集/测试集上验证。
*   **消融实验：** 进行了非常系统的消融，对比了“同策略 vs 异策略（教师采样）”以及“动态教师 vs 冻结教师”的 4 种组合，验证了同策略采样和动态同步的必要性。
*   **充分性评价：** 实验设计较为客观、公平。通过对比“技能增强型 GRPO”，有力证明了蒸馏（内化知识）比单纯在 Prompt 中堆砌信息更有效。

### 6. 主要结论与发现
*   **性能飞跃：** Skill-SD 在 AppWorld 上达到 64.9% 的准确率（比 GRPO 高 14.0%），在 Sokoban 上达到 62.5%（比 GRPO 高 10.9%）。
*   **技能应引导教师而非学生：** 直接给学生喂技能（Skill-Augmented）会导致严重的过拟合和推理能力下降，而通过蒸馏让学生“内化”技能效果最好。
*   **稳定性关键：** 异策略（教师拥有轨迹）会导致训练中途崩溃，尤其是在不可逆任务（如推箱子）中；动态同步教师比冻结教师能获得更高的性能上限。

### 7. 优点
*   **创新性：** 巧妙地将自然语言总结的“经验”转化为 RL 中的密集监督信号。
*   **数学严谨：** 针对跨 Prompt 蒸馏推导了重要性加权损失，解决了 Token 级梯度偏差问题。
*   **实用性：** 训练后的模型在推理时不需要检索技能，保持了轻量化和独立性。

### 8. 不足与局限
*   **任务池规模：** 实验使用的任务数量（约 100 个）相对较小，在大规模通用任务上的泛化性有待进一步验证。
*   **检索机制简单：** 目前使用 UCB 算法进行任务级检索，未来可能需要更复杂的语义检索（Embedding-based）来处理海量技能。
*   **依赖辅助模型：** 技能总结依赖于外部高性能模型（如 Seed1.8），这在完全闭环的自进化系统中可能是一个外部依赖项。
