## 摘要
大语言模型 (LLMs) 通过思维链 (CoT) 提示解决了许多推理任务，但较小的模型（约 7B 至 8B 参数）在计算和 token 预算紧张的情况下，仍然难以进行多步推理。现有的测试时推理方法，如自我一致性（采样多个推理过程并投票）、思维树（在中间思维上进行搜索）以及批判-修正循环，虽然提高了性能，但通常 token 成本较高，且缺乏细粒度的步骤级控制。本项目旨在填补这一空白：小型语言模型 (SLMs) 能否在消耗相同或更少 token 的情况下可靠地进行推理？这个问题既具有科学意义，也具有实用价值。从科学角度看，它探讨了过程监督和简单的测试时控制（如 token 预算和拒绝冗余步骤）是否可以替代模型规模或大量的采样次数。从实用角度看，许多部署场景（如端侧、低延迟或成本受限的环境）无法负担庞大的模型或针对每个查询采样数十个推理过程。因此，一种能在固定成本下提高 SLM 推理能力的方法将具有直接的应用价值。

## Abstract
Large Language Models (LLMs) solve many reasoning tasks via chain-of-thought (CoT) prompting, but smaller models (about 7 to 8B parameters) still struggle with multi-step reasoning under tight compute and token budgets. Existing test time reasoning methods such as self consistency (sampling multiple rationales and voting), Tree-of-Thoughts (search over intermediate thoughts), and critique revise loops improve performance, but often at high token cost and without fine-grained step-level control. This project1 aims to address that gap: can Small Language Models (SLMs) reason reliably using the same or fewer tokens? This question is both scientific and practical. Scientifically, it probes whether process supervision and simple test-time controls (such as token budgets and rejection of redundant steps) can substitute for model scale or large sampling counts. Practically, many deployments (on-device, low-latency, or cost-constrained settings) cannot afford huge models or dozens of sampled rationales per query. A method that improves SLM reasoning at fixed cost would therefore be directly useful.

*   **研究动机**：虽然大型语言模型（LLMs）在复杂推理上表现优异，但 7B-8B 规模的小型语言模型（SLMs）在多步推理中仍面临挑战。现有的增强方法（如思维树 ToT、自我一致性 Self-Consistency）通常需要消耗大量 Token 或进行多次采样，不适合端侧设备或低延迟、低成本的实际应用场景。
*   **核心问题**：能否在严格的 Token 预算和计算限制下，通过过程监督和细粒度的步骤控制，让 SLM 实现可靠的多步推理？

### 2. 方法论
论文提出了一种名为 **Dual-Track CoT** 的框架，核心思想是将推理过程分解为“生成”与“评估”两个专门化的角色：
*   **双代理循环（Collaborative Loop）**：
    *   **Decomposer（分解者）**：负责根据问题和已完成的步骤，生成下一个推理步骤（STEP）或最终答案（FINAL ANSWER）。
    *   **Evaluator（评估者）**：对生成的步骤进行 0-3 分的评分，并提供简短的自然语言反馈（Hint）。
*   **关键技术细节**：
    *   **步骤级监督**：将 GSM8K 数据集重新格式化为单步预测任务，对 Decomposer 进行微调；利用 PRM-800K 数据集对 Evaluator 进行微调，使其具备评分和提供反馈的能力。
    *   **拒绝缓存（Rejection Cache）**：引入基于“数学指纹”（提取数字和算子，忽略自然语言）的轻量级缓存。若新步骤与已接受步骤重复，或与之前低分步骤高度相似，则直接拒绝，无需调用 Evaluator，从而节省 Token。
    *   **预算感知控制**：设置全局 Token 预算。若步骤评分低于阈值，则允许有限次数的重试（结合 Evaluator 的反馈）；若预算耗尽则强制停止。

### 3. 实验设计
*   **数据集**：
    *   **训练**：GSM8K（用于 Decomposer 微调）、PRM-800K（用于 Evaluator 微调）。
    *   **测试**：从 GSM8K 测试集中随机抽取 50 个样本进行评估。
*   **对比方法（Baselines）**：
    1.  **Plain CoT**：直接输出完整推理链。
    2.  **Finetuned Decomposer only**：仅使用微调后的分解者，无评估环节。
    3.  **Dual CoT (Few-shot)**：不进行微调，仅通过提示词实现双代理协作。
    4.  **Finetuned Dual CoT**：本文提出的完整微调框架。
*   **评估指标**：最终答案准确率、Token 预算与准确率的关系曲线。

### 6. 主要结论与发现
*   **微调至关重要**：未微调的 Few-shot Dual CoT 准确率仅为 24%，远低于微调后的版本（72%）。
*   **双轨制的优越性**：微调后的 Dual CoT（72%）优于仅微调 Decomposer 的版本（68%），证明了引入 Evaluator 进行实时纠错的有效性。
*   **缓存的价值**：在 Token 预算有限（如 200-500 Token）时，拒绝缓存能带来约 2% 的准确率提升，有效减少了冗余计算。
*   **纠错能力**：通过错误分析证明，Evaluator 提供的针对性反馈能引导 Decomposer 修正错误的中间步骤，避免错误累积。

### 7. 优点
*   **预算感知设计**：直接针对端侧和成本受限场景，通过 Token 预算管理推理深度。
*   **细粒度控制**：相比于生成后再检查的传统方法，步骤级的“生成-评估-重试”循环能更早发现并阻断错误。
*   **拒绝缓存机制**：利用简单的词法指纹有效解决了小模型容易陷入的“循环生成”和“冗余表达”问题。

### 8. 不足与局限
*   **测试样本量过小**：仅 50 个测试样本限制了结论的统计显著性。
*   **评估器噪声**：错误分析显示，Evaluator 存在误判（惩罚正确步骤或批准错误逻辑）的情况，且有时无法识别全局概念性错误。
*   **反馈利用率**：Decomposer 有时会忽略 Evaluator 提供的正确反馈，重复同样的错误逻辑，说明模型对自然语言指令的遵循能力仍有待提高。
*   **任务局限性**：目前仅在数学推理（GSM8K）上验证，尚未在更广泛的常识推理或代码生成任务中测试。
