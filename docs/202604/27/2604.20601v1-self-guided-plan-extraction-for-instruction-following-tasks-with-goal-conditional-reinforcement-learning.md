## 摘要
我们介绍了 SuperIgor，一个用于指令遵循任务的框架。与以往依赖预定义子任务的方法不同，SuperIgor 使语言模型能够通过自学习机制生成并优化高层计划，从而减少了对人工数据集标注的需求。我们的方法涉及迭代协同训练：训练强化学习（RL）智能体以遵循生成的计划，同时语言模型根据 RL 的反馈和偏好调整并修改这些计划。这形成了一个反馈循环，使得智能体和规划器能够共同提升。我们在具有丰富动力学特性和随机性的环境中验证了我们的框架。结果表明，SuperIgor 智能体比基准方法更严格地遵循指令，同时也展现出对先前未见指令的强大泛化能力。

## Abstract
We introduce SuperIgor, a framework for instruction-following tasks. Unlike prior methods that rely on predefined subtasks, SuperIgor enables a language model to generate and refine high-level plans through a self-learning mechanism, reducing the need for manual dataset annotation. Our approach involves iterative co-training: an RL agent is trained to follow the generated plans, while the language model adapts and modifies these plans based on RL feedback and preferences. This creates a feedback loop where both the agent and the planner improve jointly. We validate our framework in environments with rich dynamics and stochasticity. Results show that SuperIgor agents adhere to instructions more strictly than baseline methods, while also demonstrating strong generalization to previously unseen instructions.

### 2. 方法论
SuperIgor 采用了一种**自引导（Self-Guided）的迭代协同训练机制**，主要包含四个阶段：
*   **计划生成（Plan Generation）**：利用大语言模型（LLM）从指令中提取子任务，并构建**本体图（Ontology Graph）**来描述子任务间的依赖关系。通过拓扑排序生成多个候选计划。
*   **策略学习（Policy Learning）**：使用 PPO 算法训练强化学习代理。引入了**技能课程学习（Skill Curriculum Learning）**：智能体先学习简单的单步任务，只有当某项技能的成功率达到阈值（如 0.7）后，才将其加入更复杂的组合任务计划中。
*   **计划验证（Plan Validation）**：智能体在随机环境中多次执行生成的计划，收集成功率等性能指标作为反馈信号。
*   **LLM 微调（LLM Fine-Tuning）**：利用**直接偏好优化（DPO）**，根据智能体的实际执行表现对 LLM 进行微调。这使得 LLM 能够学习到哪些计划结构更易于被智能体执行，从而实现规划器与执行器的协同进化。

*   **性能卓越**：SuperIgor 在原子和组合任务上的成功率远超仅依赖指令的基准方法（如 PPO-T）。
*   **泛化能力强**：在“新对象组合”测试中，SuperIgor 的表现甚至比 Oracle（专家计划）更稳健，因为它在训练中接触到了更多样化的候选计划。
*   **协同效应**：DPO 微调能有效重新排列计划的优先级，使 LLM 生成的计划分布逐渐向智能体可学习的技能区间靠拢。

### 7. 优点
*   **无需人工干预**：实现了从指令到执行的端到端自学习，极大地降低了数据标注成本。
*   **结构化规划**：通过本体图解决了 LLM 生成计划时可能出现的逻辑矛盾或步骤缺失问题。
*   **闭环反馈**：将 RL 的执行结果作为偏好信号反馈给 LLM，实现了认知（规划）与感知（执行）的解耦与对齐。

### 8. 不足与局限
*   **初始质量依赖**：框架仍依赖于 LLM 初始的零样本规划能力，如果 LLM 对环境的基本逻辑理解完全错误，本体构建可能会失败。
*   **诊断困难**：当训练停滞时，很难区分是 RL 策略的探索不足，还是 LLM 生成的计划空间本身存在系统性缺陷。
*   **环境限制**：目前主要在 2D 仿真环境中验证，在视觉复杂度更高、物理规则更复杂的 3D 真实世界环境中的表现尚待观察。
