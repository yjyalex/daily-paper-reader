## 摘要
长程交互环境是评估智能体技能使用能力的试验场。这些环境要求多步推理、跨多个时间步的多种技能链式调用，以及在延迟奖励和部分可观测性下的稳健决策。游戏是评估智能体在环境中技能使用的良好试验场。大语言模型（LLMs）作为游戏智能体提供了一种极具前景的替代方案，但由于缺乏在不同回合中发现、保留和重用结构化技能的机制，它们往往难以实现一致的长程决策。我们提出了 COSPLAY，这是一个协同演化框架，其中 LLM 决策智能体从可学习的技能库中检索技能以指导行动，而由智能体管理的技能流水线则从智能体的无标签轨迹（rollouts）中发现可重用技能以构建技能库。我们的框架不仅提升了决策智能体学习更好的技能检索和动作生成的能力，同时技能库智能体还不断提取、精炼并更新技能及其契约（contracts）。在六个游戏环境中的实验表明，使用 8B 基础模型的 COSPLAY 在单人游戏基准测试中，相比四个前沿 LLM 基准模型实现了超过 25.1% 的平均奖励提升，同时在多人社交推理游戏中也保持了竞争力。

## Abstract
Long horizon interactive environments are a testbed for evaluating agents skill usage abilities. These environments demand multi step reasoning, the chaining of multiple skills over many timesteps, and robust decision making under delayed rewards and partial observability. Games are a good testbed for evaluating agent skill usage in environments. Large Language Models (LLMs) offer a promising alternative as game playing agents, but they often struggle with consistent long horizon decision making because they lack a mechanism to discover, retain, and reuse structured skills across episodes. We present COSPLAY, a co evolution framework in which an LLM decision agent retrieves skills from a learnable skill bank to guide action taking, while an agent managed skill pipeline discovers reusable skills from the agents unlabeled rollouts to form a skill bank. Our framework improves both the decision agent to learn better skill retrieval and action generation, while the skill bank agent continually extracts, refines, and updates skills together with their contracts. Experiments across six game environments show that COSPLAY with an 8B base model achieves over 25.1 percent average reward improvement against four frontier LLM baselines on single player game benchmarks while remaining competitive on multi player social reasoning games.

#### 1. 核心问题与整体含义
*   **研究动机**：在长程（Long-Horizon）交互任务中，LLM 智能体通常难以保持决策的一致性。主要原因在于缺乏一种机制来**自动发现、保留和重用**跨回合的结构化技能。
*   **核心挑战**：现有的技能学习方法往往依赖人工标注或静态的技能库，导致技能库无法随着智能体策略的改进而同步演化，从而出现技能与决策脱节的问题。
*   **研究目标**：构建一个闭环系统，让决策策略与技能库通过交互不断相互促进，实现智能体的自我进化。

#### 2. 方法论：COS-PLAY 协同演化框架
COS-PLAY 包含两个核心智能体和一个动态更新的技能库：
*   **决策智能体 (Decision Agent, AD)**：
    *   **功能**：负责技能检索、意图更新和原子动作执行。
    *   **流程**：观察环境 -> 检索/切换技能 -> 更新内部意图 -> 执行动作。它不仅执行底层控制，还受高层技能协议（Skill Protocol）的引导。
*   **技能库智能体 (Skill Bank Agent, AS)**：
    *   **功能**：从无标签的交互轨迹中提取并维护技能。
    *   **关键技术细节**：
        1.  **边界提议**：通过意图变化、奖励波动等信号识别技能转换点。
        2.  **分割推断**：将轨迹切片并匹配现有技能或标记为新技能。
        3.  **契约学习 (Contract Learning)**：统计技能执行前后的状态变化（Added/Deleted Predicates），形成可验证的“效果契约”。
        4.  **库维护**：执行精炼、合并、拆分和退役操作，保持技能库的精简与高效。
*   **协同演化算法**：
    *   采用 **GRPO (Group Relative Policy Optimization)** 算法进行优化。
    *   使用**多 LoRA 适配器**设计：决策智能体和技能库智能体分别拥有独立的 LoRA 模块（如动作执行、技能检索、契约生成等），以减少不同优化目标间的干扰。

#### 3. 实验设计
*   **实验场景**：
    *   **单人游戏**：2048、Candy Crush（糖果传奇）、Tetris（俄罗斯方块）、Super Mario Bros（超级马里奥）。

*   **初始化**：使用 GPT-5.4 生成 60 条种子轨迹进行冷启动（SFT）。

#### 4. 资源与算力
*   **硬件环境**：使用了 **8 × A100 GPU** 集群。
*   **训练细节**：
    *   基础模型为 Qwen3-8B。
    *   不同游戏的演化步数不等（如 2048 为 10 步，Diplomacy 为 25 步）。
    *   使用了 LoRA 适配器进行高效微调，而非全参数更新。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了从简单益智到复杂社交博弈的 6 类环境。
*   **客观性保障**：
    *   单人游戏进行 16 次独立运行，多人游戏每玩家进行 10 次运行，并报告了 95% 置信区间。
    *   **消融实验**非常充分，验证了“协同演化”、“技能库”以及“GRPO 优化”各自的贡献。

#### 6. 主要结论与发现
*   **性能飞跃**：基于 8B 基础模型的 COS-PLAY 在单人游戏中的平均奖励比 GPT-5.4 提升了 **25.1%**。
*   **小模型逆袭**：在复杂的 Diplomacy 任务中，8B 的 COS-PLAY 表现优于 Gemini-3.1-Pro，证明了结构化技能重用比单纯增加参数量更有效。

*   **稳定性**：协同进化机制使得技能库能够根据策略分布的变化自动调整，避免了静态技能库在强化学习过程中的失效。

#### 7. 优点
*   **全自动技能发现**：无需人工定义技能，能从原始交互中自动总结出带有前置条件和效果契约的技能协议。
*   **协同进化闭环**：通过决策与学习的解耦与同步，解决了长程任务中“探索-学习-重用”的循环难题。
*   **高效适配**：多 LoRA 架构允许模型在保持通用能力的同时，快速适应特定领域的复杂逻辑。

#### 8. 不足与局限
*   **状态摘要依赖**：目前依赖文本形式的状态描述，如果环境极其复杂导致文本摘要丢失关键视觉信息，性能可能会受限。
