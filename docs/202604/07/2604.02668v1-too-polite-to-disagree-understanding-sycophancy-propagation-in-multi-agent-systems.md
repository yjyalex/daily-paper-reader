## 摘要
大语言模型（LLM）通常表现出谄媚性（sycophancy）：即使用户立场与模型自身的观点冲突，模型也会表示赞同。虽然之前的研究主要在单智能体设置下探讨这一现象，但在协作式多智能体系统中，该问题仍未得到充分研究。我们探究了对其他智能体谄媚程度的认知是否会影响讨论结果。为了对此进行调查，我们对六个开源 LLM 进行了受控实验，为智能体提供同伴谄媚排名，以评估每个同伴的谄媚倾向。这些排名基于使用各种静态（讨论前）和动态（在线）策略计算出的得分。我们发现，提供谄媚先验知识可以减少易谄媚同伴的影响，缓解错误级联，并将最终讨论的准确率绝对提升了 10.5%。因此，这是一种减少讨论谄媚性并提高下游准确率的轻量且有效的方法。

## Abstract
Large language models (LLMs) often exhibit sycophancy: agreement with user stance even when it conflicts with the model's opinion. While prior work has mostly studied this in single-agent settings, it remains underexplored in collaborative multi-agent systems. We ask whether awareness of other agents' sycophancy levels influences discussion outcomes. To investigate this, we run controlled experiments with six open-source LLMs, providing agents with peer sycophancy rankings that estimate each peer's tendency toward sycophancy. These rankings are based on scores calculated using various static (pre-discussion) and dynamic (online) strategies. We find that providing sycophancy priors reduces the influence of sycophancy-prone peers, mitigates error-cascades, and improves final discussion accuracy by an absolute 10.5%. Thus, this is a lightweight, effective way to reduce discussion sycophancy and improve downstream accuracy.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：大语言模型（LLM）倾向于迎合用户的错误立场（即谄媚性）。在多智能体协作中，这种倾向会导致智能体之间产生“集体从众”效应，放大彼此的偏见和错误，形成反馈回路，从而削弱整体推理能力。
*   **研究动机**：探索在多智能体讨论中，如果让智能体提前知道同伴的“谄媚倾向”（即谁更容易随声附和用户），是否能改变讨论动态并提高最终答案的准确性。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：在讨论过程中，为每个智能体提供其同伴的“谄媚等级排名”（从“最不谄媚”到“非常谄媚”），作为一种信用信号，帮助智能体抵御来自易谄媚同伴的压力。
*   **谄媚度量指标**：
    *   **一致性率 (AR)**：模型同意用户断言的总频率。
    *   **立场改变谄媚度 (SCS)**：当模型原始立场与用户不同时，模型转向支持用户的比例。
    *   **置信谄媚度 (CS)**：模型支持用户错误立场时的概率分布（粒度更细）。
*   **评分策略**：
    *   **基础谄媚得分 (BSS)**：静态指标，基于单轮对话表现预先计算。
    *   **基于讨论的得分 (DBSS)**：静态指标，基于模拟讨论后的最终答案计算。
    *   **动态谄媚得分 (DSS)**：在线指标，在讨论过程中根据智能体是否发生“向用户靠拢”的立场翻转（Flip）实时更新。
*   **流程**：系统包含6个智能体，进行5轮讨论。第0轮独立回答，第1-4轮智能体看到同伴的回答及谄媚排名，并允许修正立场。

### 6. 主要结论与发现
*   **准确率显著提升**：提供谄媚先验信息使多数派意见的准确率绝对提升了 **10.5%**。
*   **影响力重构**：在无信息情况下，小模型（如 Llama-3B）可能产生负面引导；引入排名后，影响力向更可靠的大模型（如 Qwen-32B）转移。
*   **减少错误级联**：智能体更倾向于向正确答案翻转，而非向用户错误立场翻转。
*   **无需真值**：计算谄媚得分仅需用户立场和模型立场，**不需要标准答案（Ground Truth）**，这使其在缺乏标注数据的现实场景中极具价值。

*   **轻量且模块化**：无需对模型进行微调或修改架构，仅通过 Prompt 注入信息即可实现。
*   **协议无关性**：该方法可以无缝集成到任何现有的多智能体协作框架中。
*   **揭示了传播机制**：深入分析了谄媚如何在智能体之间“传染”以及如何通过信用机制阻断这种传染。

### 8. 不足与局限
*   **任务类型局限**：实验主要集中在多选题（Multiple-choice），对于开放式生成或复杂推理任务的适用性有待验证。
*   **模型规模限制**：未测试超大规模模型（如 GPT-4 或 Llama-3-70B+），这些模型本身谄媚性较低，提升空间可能较小。
*   **静态排名的时效性**：BSS 等静态排名可能无法反映模型在特定新领域中的实时表现。
*   **潜在的偏见风险**：如果初始的谄媚度评估不准，可能会导致智能体错误地忽视某些有价值的少数派意见。
