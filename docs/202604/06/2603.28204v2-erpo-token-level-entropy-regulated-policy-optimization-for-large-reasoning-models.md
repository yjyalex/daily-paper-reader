## 摘要
基于可验证奖励的强化学习显著提升了大语言模型的推理能力。然而，群体相对策略优化（GRPO）通常为所有令牌分配统一的序列级优势值，从而忽略了推理链中内在的信息异质性。我们证明，这种粗粒度的信用分配会导致过早的熵坍缩，并促使模型生成冗余且低质量的推理路径。通过系统的实证分析，我们识别出了关键决策枢纽（CDPs）：即策略轨迹对扰动最为敏感的瞬时高熵状态。这些枢纽代表了“分叉路口”，在这些位置，有效的多路径探索至关重要，但往往被统一的优势信号所抑制。基于这些见解，我们提出了熵正则化策略优化（ERPO），将优化重点从粗粒度序列转向细粒度令牌动态。ERPO 引入了三个协同组件：(i) 熵感知门控（Entropy-aware Gating），通过自适应放大 CDP 处的探索来促进多样化路径的发现；(ii) 基于桶的隐式归一化（Bucket-based Implicit Normalization），通过对齐令牌进度窗口来减轻难度偏差；(iii) 结果锚定优势合成（Result-anchored Advantage Synthesis），通过结果驱动的锚点重新加权令牌级信号。在具有竞争力的数学基准测试上的广泛实验表明，ERPO 显著优于 GRPO。值得注意的是，ERPO 不仅提高了推理准确性，还生成了更加简洁且鲁棒的推导路径，同时实现了与参数量高出几个数量级的大模型相当的性能。

## Abstract
Reinforcement learning from verifiable rewards has significantly advanced the reasoning capabilities of large language models. However, Group Relative Policy Optimization (GRPO) typically assigns a uniform, sequence-level advantage to all tokens, thereby overlooking the intrinsic information heterogeneity along reasoning chains. We show that this coarse-grained credit assignment leads to premature entropy collapse and encourages the model to generate redundant, low-quality reasoning paths. Through systematic empirical analysis, we identify Critical Decision Pivots (CDPs): transient high-entropy states where the policy's trajectory is most sensitive to perturbations. These pivots represent the "forks in the road" where effective multi-path exploration is most crucial yet often suppressed by uniform advantage signals. Building on these insights, we propose Entropy-Regulated Policy Optimization (ERPO), which transitions the optimization focus from coarse sequences to fine-grained token dynamics. ERPO introduces three synergistic components: (i) Entropy-aware Gating, which adaptively amplifies exploration at CDPs to facilitate diverse path discovery; (ii) Bucket-based Implicit Normalization, which mitigates difficulty bias by aligning token progress windows; and (iii) Result-anchored Advantage Synthesis, which re-weights token-level signals via outcome-driven anchors. Extensive experiments on competitive mathematical benchmarks demonstrate that ERPO significantly outperforms GRPO. Notably, ERPO not only boosts reasoning accuracy but also yields significantly more concise and robust derivation paths, while achieving performance comparable to large models with orders of magnitude more parameters.

### 论文总结：ERPO（令牌级熵正则化策略优化）

#### 1. 核心问题与研究动机
*   **核心问题**：当前的强化学习方法（如 DeepSeek 提出的 GRPO）在提升大模型推理能力时，通常采用“序列级”的奖励分配方式，即给一段推理过程中的所有 Token 分配相同的优势值（Advantage）。
*   **研究动机**：作者发现推理链中的 Token 具有显著的**信息异质性**。通过实证分析，他们识别出**关键决策枢纽（CDPs）**——这些是推理过程中高熵、对扰动极度敏感的节点（如逻辑分叉点）。粗粒度的奖励分配会导致模型在这些关键点缺乏探索，产生“熵塌缩”，并诱导模型生成冗余、低质量的推理路径（即“车轱辘话”）。

#### 2. 方法论：ERPO 核心思想与技术细节
ERPO 旨在将优化重心从粗粒度的序列转向细粒度的 Token 动态，主要包含三个协同组件：
*   **熵感知门控（Entropy-aware Gating）**：利用 Token 级的策略熵来识别 CDPs。通过自适应门控函数，在模型不确定性高（高熵）的地方放大探索信号，在确定性推导步骤（低熵）则保持稳定。
*   **分桶隐式归一化（Bucket-based Implicit Normalization）**：将推理链按进度划分为 $K$ 个时间桶（Buckets）。仅在相同推理阶段的 Token 之间进行优势值归一化，从而消除因推理步骤先后顺序或任务难度导致的偏差。
*   **结果锚定优势合成（Result-anchored Advantage Synthesis）**：
    *   计算**隐式进度信号**：基于当前策略与参考模型之间的 log-probability 差异。
    *   **结果锚定**：使用序列级奖励的符号（正/负）作为“锚点”，确保 Token 级的微观调整与最终答案的正确性方向一致，防止模型通过优化局部 Token 概率来“刷分”（Reward Hacking）。

#### 3. 实验设计
*   **模型规模**：基于 Qwen2.5 系列（1.5B、3B、7B），使用 LoRA 进行微调。
*   **数据集**：
    *   **训练集**：MATH 数据集（筛选难度 3-5 级的样本，共 8,441 条）。
    *   **测试集（Benchmark）**：AMC23、AIME24、AIME25 以及 Minerva（前 40 题）。
*   **对比方法**：Base 模型、SFT（监督微调）模型、GRPO（基准强化学习算法）以及商业大模型（如 DeepSeek-R1-671B、Qwen3-235B）。

#### 4. 资源与算力
*   **训练硬件**：单台高性能计算节点，配备 **NVIDIA RTX pro 6000 Blackwell (96GB VRAM)**。
*   **推理硬件**：NVIDIA RTX 5090 (32GB VRAM)。
*   **训练参数**：训练 1 个 Epoch，全局 Batch Size 为 16，每个 Prompt 生成 8 个响应（$G=8$），最大序列长度 2048。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 1.5B 到 7B 的三个参数量级，验证了算法的可扩展性。
*   **实验维度**：包括最终准确率对比、Pass@k 采样性能分析、训练动力学分析（奖励、熵、梯度范数、KL 散度）、推理长度效率分析。
*   **充分性评价**：实验设计较为全面，通过多尺度模型和多个高难度数学竞赛数据集验证了方法的鲁棒性。消融实验隐含在与 GRPO 的对比中，充分展示了 Token 级调节的必要性。

#### 6. 主要结论与发现
*   **性能超越**：ERPO 在所有规模下均优于 GRPO。ERPO-7B 在 AMC23 上的表现甚至超过了参数量大得多的 DeepSeek-R1-671B。
*   **推理简洁性**：ERPO 模型生成的推理路径比 GRPO 更短、更简洁，有效减少了冗余 Token，提升了推理效率。
*   **防止塌缩**：ERPO 在训练后期能维持健康的熵水平（0.2-0.4），而 GRPO 往往会出现熵塌缩，导致生成模式单一。
*   **小模型特性**：在 1.5B 小模型上，ERPO 初期由于强化探索表现较慢，但后期提升斜率更高，显示出更强的潜力。

#### 7. 优点
*   **精细化控制**：成功解决了强化学习在长链推理中“信用分配不均”的顽疾。
*   **高效率**：无需额外的奖励模型（Reward Model）或复杂的计算，仅靠 Token 统计量即可实现，训练开销极低。
*   **帕累托改进**：同时提升了推理的“准确度”和“简洁度”，达到了更好的性能-效率平衡。

#### 8. 不足与局限
*   **训练时长限制**：实验仅运行了 1 个 Epoch，对于 AIME25 等极难任务，模型可能尚未完全收敛。
*   **梯度消失风险**：作者提到，当组内优势值方差接近零时，可能存在梯度消失问题，需进一步优化归一化策略。
*   **任务局限**：目前主要集中在数学推理（可验证奖励），在缺乏明确客观标准（如创意写作）的任务上效果尚待验证。
