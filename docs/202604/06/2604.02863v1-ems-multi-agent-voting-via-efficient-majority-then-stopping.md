## 摘要
多数投票是将多智能体响应聚合为最终决策的标准方法。然而，传统方法通常要求所有智能体在聚合开始前完成推理，这导致了显著的计算开销，因为一旦达成多数共识，许多响应就变得冗余。在这项工作中，我们将多智能体投票建模为一个可靠性感知的智能体调度问题，并提出了一种高效的“多数即停止”（EMS）方法来提高推理效率。EMS 根据任务感知的可靠性对智能体进行优先级排序，并在达成多数意见的瞬间终止推理流水线。EMS 主要由以下三个关键组件构成：具体而言，我们引入了智能体置信度建模（ACM），利用历史表现和语义相似度来估计智能体的可靠性；自适应增量投票（AIV），用于顺序选择智能体并实现提前停止；以及个体置信度更新（ICU），用于动态更新每个参与智能体的可靠性。在六个基准测试上的广泛评估表明，EMS 能够稳定地将调用的智能体平均数量减少 32%。

## Abstract
Majority voting is the standard for aggregating multi-agent responses into a final decision. However, traditional methods typically require all agents to complete their reasoning before aggregation begins, leading to significant computational overhead, as many responses become redundant once a majority consensus is achieved. In this work, we formulate the multi-agent voting as a reliability-aware agent scheduling problem, and propose an Efficient Majority-then-Stopping (EMS) to improve reasoning efficiency. EMS prioritizes agents based on task-aware reliability and terminates the reasoning pipeline the moment a majority is achieved from the following three critical components. Specifically, we introduce Agent Confidence Modeling (ACM) to estimate agent reliability using historical performance and semantic similarity, Adaptive Incremental Voting (AIV) to sequentially select agents with early stopping, and Individual Confidence Updating (ICU) to dynamically update the reliability of each contributing agent. Extensive evaluations across six benchmarks demonstrate that EMS consistently reduces the average number of invoked agents by 32%.

*   **研究动机**：作者发现这种方式存在严重的**计算冗余**。在 $N$ 个智能体的系统中，一旦有超过半数（即 $\lceil (N+1)/2 \rceil$）的智能体达成共识，后续智能体的推理结果对最终决策已无影响。
*   **核心问题**：如何在不损失投票准确性的前提下，通过识别并消除冗余推理来提升多智能体系统的推理效率？

#### 2. 论文提出的方法论：EMS 框架
论文将多智能体投票建模为一个**可靠性感知的智能体调度问题**，提出了 **EMS (Efficient Majority-then-Stopping)** 框架，包含三个核心组件：

*   **智能体置信度建模 (ACM)**：旨在为每个智能体计算任务相关的可靠性得分，以决定调用顺序。
    *   **历史可靠性 (EMS-Rel)**：基于智能体在过去任务中与最终共识一致的比例。
    *   **语义可靠性 (EMS-Sim)**：利用预训练编码器将当前查询与智能体历史成功解决的查询进行语义匹配，计算相似度。
*   **自适应增量投票 (AIV)**：
    *   **初始投票**：首先调用排序前 $\tau$（多数阈值）个智能体。若这 $\tau$ 个智能体意见一致，立即停止。
    *   **增量步骤**：若未达成共识，则按可靠性降序逐个引入后续智能体，每增加一个智能体就进行一次共识检查，直到满足多数条件或用尽所有智能体。
*   **个体置信度更新 (ICU)**：在每次任务结束后，动态更新参与投票智能体的历史表现记录和语义缓冲区，实现系统的持续进化。

#### 6. 主要结论与发现
*   **效率显著提升**：EMS 在保持与全量投票（Simple MV）几乎一致的准确率（约 86.38%）的同时，将调用的智能体平均数量从 9.0 减少到约 6.1，**推理成本平均降低了 32%**。
*   **任务难度相关性**：在简单任务（如 GSM8K）上，超过 90% 的题目在初始阶段（前 5 个智能体）即可停止；而在难题（如 MATH500）上，更多题目需要调用全部智能体。
*   **排序的重要性**：基于历史可靠性的排序（EMS-Rel）在减少调用次数方面略优于语义相似度（EMS-Sim）。

#### 7. 优点与亮点
*   **即插即用**：EMS 作为一个决策框架，不改变底层 LLM 的架构，易于集成到现有的多智能体流水线中。
*   **动态性**：通过 ICU 组件，系统能够根据智能体的实时表现动态调整优先级，具有自适应能力。
*   **理论结合**：借鉴了分布式系统中的拜占庭容错（BFT）法定人数原则，具有较强的逻辑支撑。

#### 8. 不足与局限
*   **排序依赖性**：EMS 的效率高度依赖于初始排序的准确性。如果最可靠的智能体被排在后面，效率提升将非常有限。
*   **语义缓冲区限制**：语义相似度计算依赖于历史数据的积累，在系统运行初期（冷启动阶段）效果可能不佳。
*   **极端情况处理**：对于极难或极具争议的问题，如果智能体意见极度分散，EMS 会退化为普通投票，无法节省成本。
*   **API 延迟**：虽然减少了总 Token 消耗，但由于采用了顺序/增量调用模式，在某些高并发场景下，总体的端到端响应时间（Latency）可能会因多次往返请求而增加。
