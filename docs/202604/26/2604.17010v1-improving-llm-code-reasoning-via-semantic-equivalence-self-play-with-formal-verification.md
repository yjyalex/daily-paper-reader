## 摘要
我们引入了一个针对 Haskell 语义等价性的自博弈框架，利用形式化验证来指导生成器与评估器之间的对抗训练。该框架利用 Liquid Haskell 证明来验证等价性，并利用基于执行的反例来验证不等价性，并通过难度感知的课程学习进行组织。为了促进这一研究，我们发布了 OpInstruct-HSx，这是一个包含约 2.8 万个经过验证的 Haskell 程序的合成数据集。实证实验表明，我们的评估器能有效迁移到下游任务，在 EquiBench 上实现了高达 13.3 个百分点（pp）的准确率提升，并在 PySecDB 上获得了持续的增益。针对 SEQ-SINQ 机制的消融研究表明，虽然不等价性监督提供了数据量，但等价性证明对模型的推理能力起到了独特的作用。整个训练流水线和数据集已分别在 GitHub 和 Hugging Face 上公开发布。

## Abstract
We introduce a self-play framework for semantic equivalence in Haskell, utilizing formal verification to guide adversarial training between a generator and an evaluator. The framework leverages Liquid Haskell proofs for validating equivalence and execution-based counterexamples for inequivalence, organized via a difficulty-aware curriculum. To facilitate this, we release \textbf{OpInstruct-HSx}, a synthetic dataset of $\approx$28k validated Haskell programs. Empirical experiments show that our evaluator transfers effectively to downstream tasks, achieving up to 13.3pp accuracy gain on EquiBench and consistent gains on PySecDB. Ablation studies on the SEQ-SINQ regimes indicate that while inequivalence supervision provides data volume, equivalence proofs are uniquely responsible for the model's reasoning capabilities. The entire training pipeline and dataset are publicly released on GitHub and Hugging Face respectively.

### 2. 方法论
该研究提出了一个名为 **Alice-Bob 自博弈** 的框架，包含两个核心任务：**语义等价游戏 (SEQ)** 和 **语义不等价游戏 (SINQ)**。
*   **核心思想**：
    *   **Alice（生成器）**：负责生成参考程序的变体。在 SEQ 任务中，生成等价代码及 Liquid Haskell 证明；在 SINQ 任务中，生成不等价代码及一个导致行为差异的输入（反例）。
    *   **Bob（评估器）**：在不看证明或反例的情况下，判断 Alice 生成的两个程序是否等价。
*   **关键技术细节**：
    *   **验证机制**：SEQ 任务通过 Liquid Haskell 编译器验证证明的正确性；SINQ 任务通过实际执行代码验证反例。
    *   **难度感知课程学习**：通过 Bob 的表现计算难度得分 $\hat{d} = 10 \times (1 - \frac{N_{success}}{N})$。Alice 目标是生成高难度实例，Bob 则通过学习这些实例提升推理能力。
    *   **训练策略**：采用拒绝采样微调（Rejection Sampling SFT）而非强化学习（RL），以克服 RL 在代码推理中奖励稀疏和计算成本高的问题。

### 5. 实验数量与充分性
*   **实验组设置**：作者设计了 4 种实验机制（E0-E3）进行消融研究：

### 6. 主要结论与发现
*   **推理能力提升**：自博弈显著提升了模型在 Haskell 上的代码生成能力（Pass@1 提升约 8-10%）并降低了编译错误。
*   **跨语言迁移**：在 Haskell 上训练的评估器（Bob）能有效迁移到 Python 的语义等价性判断（EquiBench 提升高达 13.3pp）和漏洞检测（PySecDB）。
*   **SEQ 的独特价值**：消融实验证明，尽管 SEQ（等价证明）的数据产出率远低于 SINQ（不等价反例），但它是提升模型深层推理能力的关键。单纯增加不等价数据无法替代等价证明带来的增益。

### 7. 优点与亮点
*   **形式化闭环**：首次将 Liquid Haskell 的形式化证明集成到 LLM 自博弈框架中，提供了比单元测试更严谨的监督信号。
*   **跨范式学习**：通过在函数式语言（Haskell）上训练，迫使模型超越标准的面向对象模式，学习更抽象的逻辑。
*   **开源贡献**：提供了高质量的 Haskell 合成数据集 OpInstruct-HSx 和完整的训练流水线。

### 8. 不足与局限
*   **证明合成瓶颈**：Alice 生成 Liquid Haskell 证明的成功率极低（通常只有个位数），导致 SEQ 训练数据严重不足。
*   **模型规模限制**：由于使用了较小的 7B 模型，Alice 在处理复杂结构证明时显得力不从心。
*   **领域局限性**：模型在涉及内存管理、指针运算等低级语义（如 C 语言的 CodeXGlue）的任务上表现提升不明显，因为 Haskell 的内存安全特性无法覆盖这些场景。
*   **Liquid Haskell 限制**：并非所有 Haskell 程序都能被 Liquid Haskell 反射和验证（如非终止行为或大规模代数重写）。
