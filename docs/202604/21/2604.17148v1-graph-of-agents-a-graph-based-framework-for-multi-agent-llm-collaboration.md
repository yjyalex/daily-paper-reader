## 摘要
随着大语言模型（LLM）和基准测试的不断增加，编排多个模型以提升任务性能的需求变得前所未有的紧迫。虽然像 Mixture-of-Agents (MoA) 这样的框架尝试协调多个 LLM，但它们在以下方面往往表现不足：(1) 选择相关的智能体，(2) 促进有效的智能体间通信，以及 (3) 高效地整合响应。在这项工作中，我们提出了 Graph-of-Agents (GoA)，这是一种用于建模多智能体 LLM 通信的新型基于图的框架。我们的方法从节点采样开始，通过利用总结了每个模型领域、任务专长和其他特征的模型卡片（model cards），仅选择最相关的智能体。接下来，我们通过相互评估所选智能体的响应来确定相关性排序，从而在它们之间构建边。然后，执行从高相关性智能体到低相关性智能体的有向消息传递以增强其响应，随后进行反向消息传递以优化高相关性智能体的原始响应。最后，通过基于图的池化（例如最大池化或平均池化）聚合更新后的响应，以生成单一、统一的答案。我们在多种多领域基准测试（MMLU、MMLU-Pro、GPQA）和特定领域基准测试（MATH、HumanEval、MedMCQA）上评估了 GoA，智能体池包含跨越多个领域的 6 个 LLM。令人惊讶的是，GoA 仅使用 3 个选定的智能体就实现了卓越的性能，优于最近同时利用所有 6 个智能体的多智能体 LLM 基准模型。通过采用图结构，GoA 通过结构化消息传递提供了可扩展性和有效性，使其成为应对日益增长的 LLM 动物园挑战的有力候选方案。代码可在以下网址获取：https://github.com/UNITES-Lab/GoA。

## Abstract
With an ever-growing zoo of LLMs and benchmarks, the need to orchestrate multiple models for improved task performance has never been more pressing. While frameworks like Mixture-of-Agents (MoA) attempt to coordinate LLMs, they often fall short in terms of (1) selecting relevant agents, (2) facilitating effective intra-agent communication, and (3) integrating responses efficiently. In this work, we propose Graph-of-Agents (GoA), a new graph-based framework for modeling multi-agent LLM communication. Our approach begins with node sampling, selecting only the most relevant agents by leveraging model cards that summarize each model's domain, task specialization, and other characteristics. Next, we construct edges between the selected agents by evaluating their responses against one another to determine relevance ordering. Directed message passing is then performed from highly relevant agents to less relevant ones to enhance their responses, followed by reverse message passing to refine the original responses of the more relevant agents. Finally, the updated responses are aggregated via graph-based pooling (e.g., max or mean pooling) to produce a single, unified answer. We evaluate GoA on diverse multi-domain benchmarks (MMLU, MMLU-Pro, GPQA) and domain-specific benchmarks (MATH, HumanEval, MedMCQA), with an agent pool of 6 LLMs spanning multiple domains. Surprisingly, GoA achieves superior performance using only 3 selected agents, outperforming recent multi-agent LLM baselines that utilize all 6 agents simultaneously. By adopting a graph structure, GoA offers both scalability and effectiveness through structured message passing-positioning it as a strong candidate for navigating the challenges of the ever-growing LLM zoo. Code is available at: https://github.com/UNITES-Lab/GoA.

### 1. 核心问题与整体含义
随着 LLM 数量的激增，如何有效整合多个模型以解决复杂任务成为难题。现有的协作框架（如 Mixture-of-Agents, MoA）存在三大痛点：
*   **代理选择难：** 通常盲目使用所有可用模型，导致计算浪费并引入无关噪声。
*   **通信效率低：** 缺乏细粒度的点对点交互，难以根据模型专长分配权重。
*   **整合成本高：** 简单的响应拼接会导致 Token 数量爆炸，增加推理成本。

**GoA 的核心意义**在于将多智能体协作建模为一个**动态有向图**，通过结构化的消息传递和筛选机制，实现“少而精”的高效协作。

### 2. 方法论：核心思想与技术细节
GoA 框架分为四个关键阶段：
*   **节点采样 (Node Sampling)：** 利用“模型卡片”（Model Cards，包含领域、任务专长、参数量等元数据）作为输入，由一个 Meta-LLM 根据查询（Query）从代理池中筛选出最相关的 $k$ 个代理作为图节点。
*   **边采样 (Edge Sampling)：** 选定代理生成初始响应后，进行“同行评审”。每个代理对其他代理的响应进行评分，构建相关性评分矩阵。通过设定阈值 $\tau$ 剪枝低相关性连接，并根据评分确定节点等级（源节点 vs 目标节点）。
*   **消息传递 (Message Passing)：** 采用双向流动机制：
    1.  **源到目标 (Source-to-Target)：** 高相关性（更专业）的代理将信息传递给低相关性代理，帮助其修正错误。
    2.  **目标到源 (Target-to-Source)：** 目标代理更新响应后反馈给源代理，通过群体共识进一步精炼原始高水平答案。
*   **图池化 (Graph Pooling)：** 
    *   **Max-Pooling：** 直接选取连接数最多、相关性评分最高的代理响应。
    *   **Mean-Pooling：** 由 Meta-LLM 综合所有更新后的响应，生成最终答案。

### 6. 主要结论与发现
*   **以少胜多：** GoA 仅使用 **3 个** 选定代理的性能，就超过了使用全部 **6 个** 代理的 MoA 等传统方法。
*   **结构化优势：** 双向消息传递（尤其是从高相关性到低相关性的引导）对消除噪声和提升逻辑推理至关重要。
*   **高效性：** 通过节点和边采样，显著减少了 LLM 调用次数和 Token 消耗，解决了多代理系统“计算爆炸”的问题。

### 7. 优点
*   **灵活性与通用性：** 纯 Prompt 接口设计，无需微调，兼容黑盒 API。
*   **相关性感知：** 引入模型卡片和同行评分，使系统能够根据问题动态调整协作拓扑。
*   **泛化性强：** 证明了 GoA 是 MoA 的一种更广义、更灵活的形式（当图为全连接且权重相等时，GoA 退化为 MoA）。

### 8. 不足与局限
*   **Meta-LLM 依赖：** 节点采样和最终聚合高度依赖 Meta-LLM 的判断力，若 Meta-LLM 选错代理，后续步骤可能失效。
*   **推理轮次：** 虽然 Token 总数减少，但双向消息传递增加了推理的轮次（Round），在极高实时性要求的场景下可能存在挑战。
*   **模型卡片质量：** 效果依赖于模型元数据的准确性，若模型卡片描述模糊，可能影响节点采样的准确性。
