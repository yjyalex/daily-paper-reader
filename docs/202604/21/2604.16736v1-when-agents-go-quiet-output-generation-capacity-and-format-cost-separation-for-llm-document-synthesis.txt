# Abstract 

LLM-powered coding agents suffer from a poorly understood failure mode we term output stalling :the agent silently produces empty responses when attempting to generate large, format-heavy docu-ments. We present a theoretical framework that explains and prevents this failure through three contributions. (1) We introduce Output Gener-ation Capacity (OGC), a formal measure of an agent’s effective ability to produce output given its current context state—distinct from and empiri-cally smaller than the raw context window. (2) We prove a Format-Cost Separation Theorem showing that deferred template rendering is always at least as token-efficient as direct generation for any for-mat with overhead multiplier μf > 1, and derive tight bounds on the savings. (3) We formalize 

Adaptive Strategy Selection , a decision framework that maps the ratio of estimated output cost to available OGC into an optimal generation strat-egy (direct, chunked, or deferred). We validate the theory through controlled experiments across three models (Claude 3.5 Sonnet, GPT-4o, Llama 3.1 70B), four document types, and an ablation study isolating each component’s contribution. De-ferred rendering reduces LLM generation tokens by 48–72% across all conditions and eliminates output stalling entirely. We instantiate the framework as 

Gen-Pilot , an open-source MCP server, demon-strating that the theory translates directly into a practical tool.

# 1 Introduction 

Tool-using LLM agents [23, 33, 15] now per-form complex software engineering tasks: writ-ing code [10], managing repositories [32], and pro-ducing formatted documents as part of multi-step workflows [28, 30]. Platforms such as Claude Code [4], Cursor [5], and SWE-agent [32] expose these capabilities through the Model Context Pro-tocol (MCP) [3]. A critical failure mode affects these agents when generating large, format-heavy outputs . Unlike hallucination [13, 12]—which produces visible (if incorrect) output— output stalling manifests as complete silence: the agent’s response is empty, no error is raised, and repeated attempts silently con-sume the context budget. We observed this fail-ure burning ∼50 ,000 tokens across five consecutive empty responses in a real agent session (Section 2). We argue that output stalling is not merely an engineering nuisance but reveals a fundamental gap in current agent architectures: the absence of 

generation-capacity metacognition —the abil-ity of an agent to reason about whether it can pro-duce a given output before attempting to do so. This paper makes three theoretical contributions and validates them empirically: 1. Output Generation Capacity (OGC): We formalize the concept of effective generation ca-pacity as distinct from context window size, and show empirically that OGC degrades non-linearly as context fills (Section 3). 2. Format-Cost Separation Theorem: We de-1

# 2 Motivating Incident 

We briefly describe the real-world failure that mo-tivated our theoretical investigation. An LLM coding agent (Claude 3.5 Sonnet, 200K context) evaluated nine research papers, ac-cumulating ∼90 ,000 tokens of analysis. When asked to generate a formatted Word document via python-docx, the agent produced five consecu-tive empty responses , burning ∼50 ,000 tokens with zero output. Switching to LaTeX with chun-ked generation succeeded immediately, producing a 14-page PDF in one attempt using ∼10 ,000 tokens. 

Key observation: The failure was not due to insufficient context window (200K tokens available) but to the agent attempting to generate ∼10 ,000 

output tokens in a single turn while ∼90 ,000 to-kens of context were already occupied. The effec-tive generation capacity at 45% context occupancy was insufficient for the output size, but no mech-anism existed to detect this before the attempt. Note that newer models with larger context win-dows (e.g., 1M tokens) shift the absolute occupancy at which stalling occurs, but do not eliminate it— the OGC framework (Section 3) shows that degra-dation is a function of the occupancy ratio o/C , not the absolute context size. This incident raises three questions that the re-mainder of this paper addresses formally: (1) How does effective generation capacity relate to context occupancy? (2) Can we reduce the token cost of formatted output without reducing content? (3) Given limited capacity, what generation strategy minimizes failure risk?

# 6 Implementation 

We instantiate the theoretical framework as Gen-Pilot , an open-source MCP server [3] that exposes seven tools organized into three layers correspond-ing to the three theoretical contributions. 

Layer 1 (Budget) implements OGC estima-tion: gp_estimate computes |c|f = |c|raw · μf us-ing the cl100k_base tokenizer [18, 24]; gp_budget 

computes OGC eff (σ) and returns the strategy rec-ommendation from Equation 13. 

Layer 2 (Planning) implements adaptive strategy selection: gp_plan solves Equation 12 and returns an ordered step sequence; gp_replan ap-plies the dominance ordering (Theorem 2) to re-cover from failures by descending the cascade. 

Layer 3 (Rendering) implements format-cost separation: gp_register_template stores Jinja2 [22] templates; gp_render performs the 

Ttpl = 0 rendering step; gp_list_templates pro-vides template discovery with JSON Schema for each template’s expected data structure, enabling agents to construct valid input without trial-and-error. 

Recovery bounds. The cascade in Equation 13 could in principle cycle between chunked and de-ferred strategies indefinitely. The implementation bounds recovery at three replans; beyond this, 

gp_replan returns an actionable error recommend-ing context compaction or scope reduction. 

Security. Templates are rendered inside Jinja2’s SandboxedEnvironment , preventing server-side template injection. HTML templates enable auto-escaping to prevent cross-site scripting when rendered output is viewed in a browser. Template names are validated against path traversal, directory separators, and null bytes. The implementation integrates with Resilient-Write [1] for atomic file writes. All state is persisted in .gen_pilot/ (templates, plans, configuration) with automatic garbage collection of stale plan files. The package ships with four pre-built templates and a calibration script for measuring μf on new content types. For-5mat multipliers are configurable per-workspace via .gen_pilot/config.json , allowing calibrated overrides to propagate consistently through budget estimation, planning, and replanning.

# 8 Related Work 

We situate our contributions at the intersec-tion of tool-using LLM agents, context man-agement, structured generation, and agent self-evaluation. To our knowledge, no prior work for-malizes generation-capacity planning as a theoret-ical framework. 

8.1 Tool-Using LLM Agents 

Toolformer [23] demonstrated self-supervised tool learning; ReAct [33] showed that interleaving rea-soning with actions improves task completion. MRKL [15] proposed modular neuro-symbolic routing; Gorilla [20] fine-tuned models for API call accuracy; ToolLLM [21] scaled tool learning to 16,000+ APIs. These works address which tool to call and how . Our framework addresses a comple-mentary question: given the agent’s capacity, can 

it generate the required output, and if not, what strategy should it use? 

8.2 Autonomous Agent Frameworks 

LangChain [9] provides composable tool chains; MetaGPT [11] assigns specialized roles in multi-agent pipelines; SWE-agent [32] designs agent-computer interfaces for software engineering; AgentBench [17] benchmarks agents across diverse environments. None include generation-capacity awareness. When output exceeds effective OGC, these systems have no detection or recovery mech-anism. Our framework is composable—any agent system can integrate OGC checking as a pre-generation step. 

8.3 Context Window Management 

Large context windows [2] mitigate but do not eliminate context pressure. Liu et al. [16] showed that models struggle with information in the mid-dle of long contexts, implying effective capacity is smaller than nominal. LongBench [6] benchmarks 8long-context tasks. MemGPT [19] treats the LLM as an OS with virtual memory; Xu et al. [31] com-bine retrieval with long context. These manage 

input context. Our OGC concept addresses out-put capacity—the ability to generate tokens given current context state, a distinct and previously un-formalized problem. 

8.4 Structured and Constrained Gener-ation 

Outlines [29] uses finite-state machines to guide to-ken sampling; LMQL [7] provides a query language for constrained outputs; SGLang [34] offers efficient structured generation with parallelism. These con-strain individual tokens . Format-cost separation operates at the document level, selecting the gen-eration strategy rather than constraining the de-coding process. The approaches are complemen-tary: constrained generation could enforce JSON schema compliance within our deferred strategy. 

8.5 LLM Failure Modes and Self-Evaluation 

Hallucination—generating fluent but incorrect con-tent [13, 12]—is the most studied LLM failure mode. Output stalling is fundamentally different: the model produces no output , leaving no arti-fact for correction. Kadavath et al. [14] showed models can partially assess their knowledge bound-aries; Reflexion [25] enables verbal self-correction. Our OGC framework extends self-evaluation from 

knowledge (“do I know this?”) to capacity (“can I express this?”)—a novel metacognitive dimension. 

8.6 Tokenization and Cost Estimation 

BPE tokenization [24] varies across model fam-ilies [8, 26], meaning format multipliers are tokenizer-dependent. Code-optimized models [10] may exhibit different μf for python-docx boiler-plate. Our calibration methodology (Section 4) is designed to be re-run per deployment, and the the-oretical results (Theorem 1, Theorem 2) hold for any μf > 1.

# 9 Limitations 

OGC measurement. The capacity degradation function α (Figure 1) is measured empirically and may vary with prompt structure, system instruc-tions, and fine-tuning. A theoretical derivation of 

α from attention mechanics would strengthen the framework. 

Context access. Gen-Pilot cannot directly read the LLM’s token counter; all OGC esti-mates are approximate. Agent platforms expos-ing context_tokens_used() would enable exact OGC computation. As a consequence, the current implementation approximates Equation 13 using raw headroom ratios rather than the α-degraded 

OGC eff , since computing α requires knowing the true context occupancy. 

Multiplier stability. Format multipliers are measured per-document-type and may shift with content characteristics (e.g., code-heavy vs. prose-heavy text). Our calibration script addresses this but requires manual re-measurement. 

Template quality. Deferred rendering is only as good as the template. Complex lay-outs (multi-column, figures with captions) require sophisticated templates that may themselves be costly to generate. Pre-built template libraries and JSON Schema–annotated variable declarations mitigate but do not eliminate this concern. 

Cascade termination. The replan cascade (Section 6) is bounded at three iterations as a prac-tical safeguard. A formal analysis of convergence conditions —when no strategy can succeed—would allow the bound to be derived rather than chosen empirically. 

Evaluation scale. While we test three mod-els and four document types, the evaluation uses 5 runs per condition. Larger-scale studies with more models, document types, and occupancy lev-els would strengthen statistical claims. 

# 10 Conclusion 

We have presented a theoretical framework for un-derstanding and preventing output stalling in LLM coding agents, grounded in three contributions: 1. Output Generation Capacity (OGC) —a formal measure of effective generation capacity 9that degrades non-linearly with context occu-pancy, explaining why agents stall at context levels well below the nominal limit. 2. Format-Cost Separation Theorem —a provable decomposition showing that deferred rendering eliminates format tokens from LLM generation, with savings that grow with format complexity ( ρ = 48 –72% empirically). 3. Adaptive Strategy Selection —a decision framework with provable dominance ordering (defer ⪰ chunk ⪰ direct) that maps OGC to optimal strategy. Across three models, four document types, and 180 experimental runs (60 per strategy), deferred rendering achieved 100% success rate while re-ducing generation tokens by 48–72% . The abla-tion study shows that OGC and format-cost sep-aration are each independently essential, and that the framework is model-agnostic. The broader implication is that LLM agents need 

metacognitive tools —not just tools that act on the world, but tools that help agents reason about their own generation capabilities. Output Genera-tion Capacity formalizes a previously implicit con-straint; format-cost separation provides a princi-pled technique for working within that constraint. Together, they transform output stalling from an invisible failure into a preventable one.
