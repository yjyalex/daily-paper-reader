# Phase-Scheduled Multi-Agent Systems for Token-Efficient Coordination 

## Mohit Dubey 

Open Gigantic 02mohitdubey@gmail.com

Abstract. Multi-agent systems (MAS) powered by large language models suffer from severe token inefficiency arising from two compounding sources: (i) unstructured parallel execution, where all agents activate simultaneously irrespective of input readiness; and (ii) unrestricted context sharing, where every agent receives the full accumu-lated context regardless of relevance. Existing mitigation strategies—static pruning, hierarchical decomposition, and learned routing—treat coordination as a structural allocation problem and fundamentally ignore its temporal dimension. We propose Phase-Scheduled Multi-Agent Systems (PSMAS) , a framework that reconceptualises agent activation as continuous control over a shared attention space modelled on the circular manifold S1.Each agent i is assigned a fixed angular phase θi ∈ [0 , 2π) derived from the task dependency topology; a global sweep signal φ(t) rotates at velocity ω, activating only agents within an angular window ε. Idle agents receive com-pressed context summaries, reducing per-step token consumption. We implement PSMAS on LangGraph, evaluate on four structured benchmarks (HotPotQA-MAS, HumanEval-MAS, ALFWorld-Multi, WebArena-Coord) and two unstructured conversational settings, and prove stability, convergence, and optimality results for the sweep dynam-ics. PSMAS achieves a mean token reduction of 27.3% (range 21.4–34.8%) while maintaining task performance within 2.1 pp of a fully-activated baseline ( p< 0.01, n=500 per configuration), and outperforms the strongest learned routing baseline by 5.6 pp in token reduction with 2.0 pp less performance drop. Crucially, we prove that scheduling 

and compression are independent sources of gain: scheduling alone accounts for 18–20 pp of reduction, robust to compression degradation up to α=0 .40. 

Keywords: multi-agent LLMs, token efficiency, phase scheduling, circular manifold, attention control, dynamical systems. 

1. Introduction 

The deployment of LLM agents at scale has produced multi-agent systems (MAS) in which specialised agents collaborate to decompose, execute, and validate complex tasks [Wu et al., 2023, Hong et al., 2023, Chase, 2024]. A na¨ ıve MAS activates all agents simultaneously and provides each with the full accumulated context. For 

n agents and context length L, total token consumption is O(n×L)—growing quadratically when L itself grows with agent count (as in broadcast architectures). Empir-ical measurements on production five-agent code-review pipelines record 42,000–71,000 tokens per invocation, of which 29–38% is redundant context consumed by agents that do not act on it. 

1.1 Why Existing Approaches Fall Short 

Static pruning [Liu et al., 2023] removes context after the fact; hierarchical decomposition [Hong et al., 2023] localises it by topology; learned routers [Shen et al., 2024, Ong et al., 2024] direct tasks to specialists. All three treat coordination as a static allocation problem and cannot control when each agent is active. Even the strongest adaptive baseline (RouterLLM [Ong et al., 2024]) makes discrete per-step routing decisions without temporal structure, leaving systematic redundancy un-addressed. 

1.2 PSMAS: Continuous Attention Control 

> Core Thesis — Control Not Scheduling

PSMAS is not a scheduling heuristic. It is a contin-uous control system operating over the shared attention space of a multi-agent network. The circular manifold S1

represents activation capacity geometrically ; the sweep signal φ(t) is a control input allocating that capacity con-tinuously across time. This framing—continuous, geo-metric, principled—admits stability proofs, convergence guarantees, and analytical optimality conditions that no discrete scheduler can provide, and is why PSMAS out-performs learned routing approaches that are scheduling heuristics.

2. Why the Circular Manifold? 

A natural objection is: “Why not simply use linear (time-slot) scheduling?” We address this with a formal argument that S1 is the necessary topology, not an aes-thetic one. 

(1) Boundary effects and aperiodicity. Linear scheduling has hard endpoints: the last agent at slot tn

produces output that the first agent needs for the next iteration. Feeding back requires either explicit wrap-around logic (making the schedule circular de facto) or restarting the timeline, introducing latency discontinu-ities. On S1, periodicity is intrinsic: φ(t + 2 π/ω ) = φ(t), and every agent’s output is automatically in scope for the next cycle with no boundary handling. This matters practically: LLM agent pipelines are almost universally iterative (refinement, verification loops), and each iter-ation benefits from periodic scheduling without restart overhead. 

(2) No natural proximity metric. Linear time gives a total order but no meaningful notion of activation 

proximity independent of ω. On S1, the circular distance 

|θi −θj |2π is a bi-invariant metric that encodes activation proximity independently of sweep speed, enabling the window parameter ε to be specified in natural angular units and compared across systems of different sizes. 

(3) No connection to stability theory. Linear schedules admit no stability analysis beyond deadline feasibility. S1-based dynamics connect to the driven Ku-ramoto model [Acebr´ on et al., 2005] and to control theory on compact manifolds [Bullo & Lewis, 2004], providing Lyapunov functions, phase- locking conditions, and bi-furcation analysis as ω varies—tools that are structurally unavailable for linear or discrete-ring alternatives.          

> Necessity Claim
> Among all one-dimensional coordination manifolds, S1is the unique topology satisfying simultaneously: (a) peri-odic recurrence (iterative pipelines), (b) bi-invariant met-ric (scale-independent ε), and (c) differentiable structure (stability analysis). The real line Rsatisfies (b,c) but
> not (a). A discrete ring Z/n Zsatisfies (a) but neither (b) nor (c).

3. Related Work 

> 3.1 Multi-Agent LLM Frameworks

AutoGen [Wu et al., 2023], LangGraph [Chase, 2024], MetaGPT [Hong et al., 2023], and CrewAI [Moura, 2024] establish the multi-agent paradigm. None treat token efficiency as a first-class design concern. Agent-Prune [Tao et al., 2024] prunes the agent graph by con-tribution score—orthogonal to our temporal approach. GPTSwarm [Zhuge et al., 2024] learns communication topologies; we instead control activation timing over a fixed topology. 

> 3.2 Learned and Adaptive Routing

HuggingGPT [Shen et al., 2024] uses an LLM plan-ner to route tasks to specialists; RouterLLM [Ong et al., 2024] trains a lightweight classifier to select between model tiers. We include both as baselines ( §9.2) and show that PSMAS outperforms them because continu-ous temporal control reduces redundancy more precisely than per-step discrete routing. 

> 3.3 Context Compression

LLMLingua [Jiang et al., 2024] compresses prompts via learned token removal. We use this as a summarisa-tion primitive for idle agents; we prove in §6 that schedul-ing gains are independent of the compression quality achieved by this primitive. 

> 3.4 MoE and Sparse Activation

MoE [Shazeer et al., 2017, Fedus et al., 2022] acti-vates expert subsets per token within a single model. Unlike MoE, PSMAS operates across separate LLM in-ference calls with continuous time-indexed scheduling rather than per-token discrete routing. 

> 3.5 Synchronisation and Dynamical Systems

The Kuramoto model [Kuramoto, 1984] describes emergent synchronisation on S1. PSMAS imposes a driven external signal to prevent synchronisation and maintain phase diversity—formally the driven Kuramoto model [Acebr´ on et al., 2005]. Stability analysis tools from Bullo & Lewis [2004] underpin our convergence proofs.

Summarisation is a primitive that PSMAS leverages to reduce the residual cost of idle agents. The core contri-bution is scheduling, which determines the idle fraction (1 − f (ε)). 9.3 Main Results 

PSMAS-WPA achieves the best token reduction (34.8%) and lowest latency (29.7 s) with only 2.1 pp per-formance drop. RouterLLM-RA, the strongest prior adaptive baseline, achieves 29.2% reduction with 4.1 pp drop. PSMAS-WPA outperforms it by 5.6 pp in token reduction with 2.0 pp less performance drop ( p < 0.05), confirming that continuous temporal control is superior to per-step discrete routing. 

9.4 Per-Benchmark Breakdown 

ALFWorld-Multi’s smaller reduction (21.4%) confirms that highly parallel tasks benefit less from phase schedul-ing (few dependency edges to exploit). HumanEval-MAS and WebArena-Coord, with deeper sequential chains, show the largest reductions.

10.3 Results and Honest Assessment 

> Honest Assessment

In unstructured settings, PSMAS achieves 14–19% to-ken reduction— roughly half the structured gain. Per-formance drops are slightly higher (2.9–3.1 pp) but still below all non-PSMAS token-reduction baselines. PS-MAS is most effective when task structure is known or inferable; it degrades gracefully—not catastrophically— when structure is latent. Three factors reduce unstructured gains: (1) wider ε needed to prevent F1 violations; (2) graph-inference latency (0.8 s overhead); (3) dynamic dependency emergence requiring mid-task phase re-assignment. 

11. Discussion 

11.1 PSMAS Is Not a Scheduling Trick 

We state this explicitly because it is the most likely misreading. A scheduling trick assigns time slots and activates agents in order. PSMAS is different in every dimension that matters: 

• It defines a continuous activation state over the sys-tem’s attention space, parameterised by θi ∈ S1.

• The sweep signal φ(t) is a control input with proven stability (Theorem 7.3), not a counter. 

• The system has a convergence guarantee (Theo-rem 7.5) w.r.t. the full-activation fixed point—no dis-crete scheduler provides this.

11.2 Relation to Kuramoto Dynamics 

PSMAS prevents synchronisation via a driven exter-nal signal, formally equivalent to the driven Kuramoto model [Acebr´ on et al., 2005]. Failure mode F3 (con-vergence stall in cyclic graphs) is equivalent to Ku-ramoto phase frustration [Strogatz, 2000], suggesting that Kuramoto stability analysis could derive tighter safe-ω bounds for graphs with cycles. 

11.3 Scope of Applicability 

PSMAS benefits are largest for: deep sequential de-pendency chains, long per-agent contexts, predictable agent latency. Gains are smallest for: embarrassingly parallel tasks, short pipelines, highly dynamic communi-cation graphs.

13. Conclusion 

We presented PSMAS, a framework that reconcep-tualises multi-agent coordination as continuous control over shared attention space on S1. By assigning agents phases derived from task dependency topology and con-trolling activation via a global sweep signal, PSMAS achieves a mean token reduction of 27.3% (up to 34.8%) across four structured benchmarks and 14–19% in un-structured settings, while preserving task performance within 2.1 pp of full activation. We proved that the circular manifold is the neces-sary topology for this approach; that sweep dynamics are stable under latency noise; that the system con-verges to the full-activation fixed point; and that the op-timal window ε∗ can be derived analytically. We further showed that scheduling and compression are independent 

sources of gain—scheduling alone accounts for 18–20 pp of reduction even when compression quality degrades to 

α = 0 .40. PSMAS is not a scheduling heuristic. It is a principled control system with formal guarantees, a connection to dynamical systems theory, and empirical superiority over state-of-the-art learned adaptive baselines. We hope this work motivates a shift toward geometrically grounded, temporally continuous coordination as a first-class design principle in LLM agent architectures.
