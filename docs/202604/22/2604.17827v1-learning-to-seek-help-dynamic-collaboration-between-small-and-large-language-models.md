## 摘要
大语言模型（LLMs）具备强大的能力，但存在成本和隐私方面的顾虑，而小语言模型（SLMs）虽然支持高效且私密的本地推理，但其能力有限。为了协同这些互补优势，我们引入了一个动态协作框架，在该框架中，SLM 学习在多步推理过程中主动决定如何向 LLM 寻求帮助，而 LLM 则提供自适应反馈，而非仅作为一个被动工具。我们还系统地研究了 SLM 和 LLM 的能力以及效率和隐私约束对协作策略的影响。评估结果揭示了显著的缩放效应：更强大的 SLM 表现出更高的自立性，而更强大的 LLM 则能实现更少且更具信息量的交互。此外，所学习的动态协作策略显著优于静态流水线和独立推理，并能稳健地迁移至未见过的 LLM。

## Abstract
Large language models (LLMs) offer strong capabilities but raise cost and privacy concerns, whereas small language models (SLMs) facilitate efficient and private local inference yet suffer from limited capacity. To synergize the complementary strengths, we introduce a dynamic collaboration framework, where an SLM learns to proactively decide how to request an LLM during multi-step reasoning, while the LLM provides adaptive feedback instead of acting as a passive tool. We further systematically investigate how collaboration strategies are shaped by SLM and LLM capabilities as well as efficiency and privacy constraints. Evaluation results reveal a distinct scaling effect: stronger SLMs become more self-reliant, while stronger LLMs enable fewer and more informative interactions. In addition, the learned dynamic collaboration strategies significantly outperform static pipelines and standalone inference, and transfer robustly to unseen LLMs.

### 2. 方法论
*   **核心思想**：将大小模型协作建模为异构智能体之间的动态博弈。SLM 作为主动决策者，通过多步推理决定是本地解决还是请求 LLM；LLM 不再是死板的工具，而是提供自适应反馈（如指出请求不明确或提供知识）。
*   **关键技术细节**：
    *   **主动请求机制**：SLM 在推理每一步生成 `think` 块，判断是否需要 `message`（请求）。
    *   **自适应反馈**：LLM 根据请求质量决定是要求更多上下文（Feedback）还是直接回答（Information）。
    *   **强化学习优化**：使用 **VAPO 算法**（一种高效的在线强化学习算法）对 SLM 进行端到端训练。
    *   **多目标奖励函数**：
        *   $r_{quality}$：基于答案正确性的结果奖励。
        *   $r_{efficiency}$：基于交互轮数的效率惩罚（鼓励简单问题本地化）。
        *   $r_{privacy}$：基于隐私泄露检测的惩罚。
        *   $r_{feedback}$：基于 LLM 反馈质量的过程奖励。
    *   **隐私注入流水线**：通过 LLM 将公开 QA 数据集改造成包含个人隐私信息的 **PrivQA 数据集**，用于训练 SLM 的脱敏能力。

### 3. 实验设计
*   **数据集**：构建了 **PrivQA**，源自 6 个公开数据集：NQ、TriviaQA、PopQA（通用 QA）以及 HotpotQA、2Wiki、MuSiQuE（多跳推理）。
*   **Benchmark 与对比方法**：
    *   **独立推理**：SLM (CoT), LLM (Direct/CoT)。
    *   **静态协作**：PAPILLON（基于重写的静态流水线）。
    *   **路由/工具化**：Router-R1（将 LLM 视为检索工具）。
*   **评估指标**：准确性（EM, BERTScore）、效率（平均交互轮数、INScore 交互必要性得分）、隐私（隐私泄露率 Priv-sample）。

### 4. 资源与算力
*   **硬件环境**：使用了 **8 台 H20 96G GPU** 工作站。
*   **训练细节**：训练批次大小为 256，最大上下文长度为 8192（SLM）和 16384（LLM），总训练步数为 600 步。
*   **模型规模**：SLM 涵盖 3B 到 8B（Qwen3, Llama-3.1/3.2），LLM 涵盖 70B 到 235B 及闭源的 Qwen3-Max。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 4 种 SLM 与 3 种 LLM 的交叉配对实验，并在 6 个数据集上进行了跨领域验证。
*   **消融实验**：针对 LLM 反馈的作用、效率惩罚权重、隐私惩罚权重分别做了详细的参数敏感性分析。
*   **充分性评价**：实验设计非常充分，不仅验证了性能提升，还深入探讨了“缩放效应”（Scaling Effect）和“策略迁移性”，并包含了人工评估（200 样本）来校验 LLM 判别隐私的准确性，具有较高的客观性和说服力。

### 6. 主要结论与发现
*   **性能卓越**：动态协作框架在 EM 指标上比独立 SLM 提升 14.5%~17.4%，比静态流水线提升 2.8%~9.9%。
*   **缩放效应**：
    *   **SLM 越强越自立**：更强的 SLM 会减少对 LLM 的请求。
    *   **LLM 越强交互越精简**：更强的 LLM 能通过更少的交互轮数提供更精准的信息。
*   **隐私保护**：通过强化学习，SLM 学会了在不牺牲性能的前提下，将隐私请求重写为客观事实请求，隐私泄露率降低了 24.3%~32.4%。
*   **鲁棒迁移**：在较小 LLM 上训练的协作策略，可以直接迁移到更强大的未见 LLM（如 Qwen3-Max）上，且性能进一步提升。

### 7. 优点
*   **范式创新**：将大小模型协作从“工具调用”提升为“动态沟通”，引入了 LLM 的主动反馈机制。
*   **多目标平衡**：成功在单一框架内解决了性能、成本（效率）和隐私三个核心痛点。
*   **实用性强**：证明了协作策略的迁移性，意味着开发者可以用较小成本训练 SLM，然后部署到高性能云端环境。

### 8. 不足与局限
*   **冷启动依赖**：实验发现指令遵循能力极弱的模型（如 0.6B）无法直接通过 RL 学习策略，必须先进行有监督微调（SFT）。
*   **隐私定义的局限**：隐私保护依赖于预定义的隐私实体标签和 LLM 的判断，对于极其隐晦或长程关联的隐私泄露可能仍存在风险。
*   **计算延迟**：虽然减少了云端调用次数，但 SLM 的多步推理（Think 过程）本身会增加端侧的响应延迟。
