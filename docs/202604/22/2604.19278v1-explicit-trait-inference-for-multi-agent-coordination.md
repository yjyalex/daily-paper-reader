## 摘要
基于大语言模型（LLM）的多智能体系统（MAS）在处理复杂任务方面展现出巨大潜力，但仍容易出现协作失败，如目标漂移、错误级联和行为失调。我们提出了显式特质推理（Explicit Trait Inference, ETI），这是一种基于心理学理论的改进协作的方法。ETI 使智能体能够从交互历史中推断并追踪合作伙伴在两个既定心理维度——热情（如信任）和能力（如技能）——上的特征，并以此指导决策。我们在受控环境（经济博弈）中对 ETI 进行了评估，结果显示其将收益损失降低了 45-77%；在更现实、复杂的多智能体环境（MultiAgentBench）中，相对于思维链（CoT）基准，其性能提升了 3-29%（取决于具体场景和模型）。进一步的分析表明，性能提升与特质推理密切相关：ETI 画像能够预测智能体的行为，且信息丰富的画像是性能提升的关键驱动力。这些结果表明，ETI 是一种在多样化多智能体环境中改进协作的轻量级且鲁棒的机制，并提供了首个系统性证据，证明 LLM 智能体可以：(i) 从交互历史中可靠地推断他人的特质；(ii) 利用对他人的结构化特质认知来优化协作。

## Abstract
LLM-based multi-agent systems (MAS) show promise on complex tasks but remain prone to coordination failures such as goal drift, error cascades, and misaligned behaviors. We propose Explicit Trait Inference (ETI), a psychologically grounded method for improving coordination. ETI enables agents to infer and track partner characteristics along two established psychological dimensions--warmth (e.g., trust) and competence (e.g., skill)--from interaction histories to guide decisions. We evaluate ETI in controlled settings (economic games), where it reduces payoff loss by 45-77%, and in more realistic, complex multi-agent settings (MultiAgentBench), where it improves performance by 3-29% depending on the scenario and model, relative to a CoT baseline. Additional analysis shows that gains are closely linked to trait inference: ETI profiles predict agents' actions, and informative profiles drive improvements. These results highlight ETI as a lightweight and robust mechanism for improving coordination in diverse multi-agent settings, and provide the first systematic evidence that LLM agents can (i) reliably infer others' traits from interaction histories and (ii) leverage structured awareness of others' traits for coordination.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究动机**：尽管基于大语言模型（LLM）的多智能体系统（MAS）在处理复杂任务上表现出色，但它们经常遭遇“协作失效”，如目标漂移、错误级联、信息共享不足以及行为不一致。
*   **核心问题**：如何让智能体像人类一样，通过评估合作伙伴的稳定行为特征（特质）来优化协作决策？
*   **整体含义**：论文引入了心理学中的“社会评价”机制，提出通过显式推断他人的“热情”（Warmth）和“能力”（Competence）维度，来增强智能体对环境和他人的感知，从而实现更稳健的协调。

### 2. 方法论：显式特质推理（ETI）
*   **核心思想**：ETI 是一种轻量级的提示词（Prompting）流程，要求智能体在交互后对伙伴进行结构化的特质画像，并将该画像注入到后续的决策上下文中。
*   **关键技术细节**：
    *   **双维度框架**：借鉴社会心理学，将特质分为**热情**（目标一致性、协作性、诚实性、恶意性）和**能力**（执行能力、可靠性、适应性、效率）两个维度，共 8 个具体指标。
    *   **推理流程**：
        1.  **观察**：收集交互历史（动作、通信、结果）。
        2.  **推断**：使用 LLM 生成 JSON 格式的画像，包含 1-7 分的 Likert 评分及具体的行为证据。
        3.  **注入**：将最新的伙伴画像附加到智能体的规划和行动提示词中。
        4.  **决策**：智能体根据画像调整策略（例如：对低能力的伙伴增加验证，对低热情的伙伴保持警惕）。
*   **算法特点**：无需微调模型，完全基于上下文学习（In-context Learning）。

### 3. 实验设计
*   **实验场景**：
    1.  **受控经济博弈**：迭代囚徒困境（PD）和猎鹿博弈（SH），引入“能力参数”模拟执行失败，用于测试推断的准确性和决策最优性。
    2.  **复杂基准测试（MultiAgentBench）**：包含编程（Coding）、科研协作（Research）、商务谈判（Bargaining）和狼人杀（Werewolf）四个具有代表性的复杂场景。
*   **Benchmark 与对比方法**：
    *   **基准（Baseline）**：标准的思维链（CoT）提示词，智能体仅基于任务历史进行推理，不进行显式特质画像。
    *   **对比维度**：测试了不同模型（Qwen 3-8B, GPT-4o-mini）作为“特质推断器”和“行动执行器”的各种组合。

### 4. 资源与算力
*   **算力设备**：使用了 AWS EC2 g5.12xlarge 实例，配备 **4 张 NVIDIA A10G GPU**（总显存 96GB）。
*   **训练/推理时长**：总计消耗约 **300-400 GPU 小时**。
*   **API 使用**：GPT-4o-mini 的实验通过 OpenAI API 完成。
*   **成本参考**：在重度通信场景下，ETI 会增加约 2 倍的 Token 消耗，但单次运行成本增量极低（约 $0.01–$0.13）。

### 5. 实验数量与充分性
*   **实验规模**：
    *   经济博弈：每种配置重复 **25 次**。
    *   MultiAgentBench：每个场景重复 **100 次** 随机种子运行。
*   **充分性评价**：实验设计非常充分且客观。作者不仅对比了最终得分，还进行了：
    *   **消融实验**：验证特质画像的准确性。
    *   **跨模型实验**：探讨不同模型生成画像质量的差异。
    *   **适应性实验**：测试当伙伴特质突然改变时系统的反应。
    *   **统计检验**：所有核心提升均标注了显著性差异（p < 0.05）。

### 6. 主要结论与发现
*   **性能提升**：在经济博弈中减少了 **45-77%** 的收益损失；在 MultiAgentBench 中提升了 **3-29%** 的任务表现和 **6-42%** 的协作质量。
*   **可靠推断**：LLM 能够仅凭交互历史准确推断伙伴的隐藏特质（如在狼人杀中识别反派）。
*   **模型差异**：Qwen 3-8B 生成的特质画像比 GPT-4o-mini 更具判别力和客观性；GPT 存在明显的“正向偏差”（倾向于给好评），这反而限制了其协作提升。
*   **因果关联**：回归分析证明，特质评分能显著预测智能体的后续行为（如投票淘汰恶意评分高的对象）。

### 7. 优点（亮点）
*   **理论支撑**：将成熟的社会心理学框架（热情/能力模型）引入 AI 领域，具有很强的解释性。
*   **轻量高效**：无需昂贵的微调，通过简单的 Prompt 工程即可实现显著的性能飞跃。
*   **透明度**：显式的特质画像让多智能体系统的决策过程变得可解释、可审计。
*   **泛化性**：实验证明特质画像可以在不同任务间迁移（如从 PD 迁移到 SH）。

### 8. 不足与局限
*   **模型偏差**：ETI 的效果高度依赖于底层 LLM 的推理能力，某些模型（如 GPT 系列）的“讨好型”偏差会降低画像的诊断价值。
*   **适应滞后**：由于 ETI 强调“稳定特质”，当合作伙伴的行为发生剧烈且突然的转变时，系统更新画像的速度可能较慢。
*   **Token 开销**：虽然绝对成本不高，但在大规模部署时，额外的特质推断步骤会增加延迟和 Token 消耗。
*   **潜在偏见**：如果输入包含人口统计学信息，显式特质推理可能会放大模型固有的社会偏见。
