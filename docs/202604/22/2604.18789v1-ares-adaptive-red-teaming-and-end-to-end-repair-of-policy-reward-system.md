## 摘要
基于人类反馈的强化学习（RLHF）是实现大语言模型（LLM）对齐的核心，但它引入了一个关键漏洞：当不完美的奖励模型（RM）未能惩罚不安全行为时，它可能成为单点故障。虽然现有的红队测试方法主要针对策略层面的弱点，但它们忽视了我们所称的“系统性弱点”——即核心 LLM 和 RM 同时失效的情况。我们提出了 ARES 框架，旨在系统地发现并缓解此类双重漏洞。ARES 采用一个“安全导师”（Safety Mentor），通过组合结构化组件类型（主题、人格、策略、目标）动态生成语义连贯的对抗性提示词，并生成相应的恶意与安全回复。这种双重目标方法同时揭示了核心 LLM 和 RM 的弱点。利用发现的漏洞，ARES 实施了一个两阶段修复过程：首先微调 RM 以更好地检测有害内容，然后利用改进后的 RM 优化核心模型。在多个对抗性安全基准上的实验表明，ARES 在保留模型能力的同时显著增强了安全鲁棒性，为全面的 RLHF 安全对齐建立了新范式。

## Abstract
Reinforcement Learning from Human Feedback (RLHF) is central to aligning Large Language Models (LLMs), yet it introduces a critical vulnerability: an imperfect Reward Model (RM) can become a single point of failure when it fails to penalize unsafe behaviors. While existing red-teaming approaches primarily target policy-level weaknesses, they overlook what we term systemic weaknesses cases where both the core LLM and the RM fail in tandem.   We present ARES, a framework that systematically discovers and mitigates such dual vulnerabilities. ARES employs a ``Safety Mentor'' that dynamically composes semantically coherent adversarial prompts by combining structured component types (topics, personas, tactics, goals) and generates corresponding malicious and safe responses. This dual-targeting approach exposes weaknesses in both the core LLM and the RM simultaneously. Using the vulnerabilities gained, ARES implements a two-stage repair process: first fine-tuning the RM to better detect harmful content, then leveraging the improved RM to optimize the core model. Experiments across multiple adversarial safety benchmarks demonstrate that ARES substantially enhances safety robustness while preserving model capabilities, establishing a new paradigm for comprehensive RLHF safety alignment.

*   **研究动机**：RLHF 依赖奖励模型（RM）来引导核心 LLM 走向安全行为。然而，如果 RM 本身存在盲点（即无法识别某些有害行为），它就会成为单点故障。
*   **核心问题**：现有的红队测试（Red-Teaming）大多只关注策略模型（Core LLM）的弱点，而忽视了**系统性弱点（Systemic Weaknesses）**——即核心 LLM 生成了有害内容，而 RM 却给出了高分奖励。这种双重失效会导致模型在对齐过程中被错误地强化。
*   **整体含义**：ARES 提出了一种协同发现与修复的方法，通过同时针对 LLM 和 RM 进行压力测试，实现端到端的系统加固。

### 2. 方法论
ARES 框架分为两个主要阶段：

#### 第一阶段：自适应漏洞发现（Adaptive Vulnerability Discovery）
*   **安全导师（Safety Mentor）**：使用一个强大的 LLM 作为“导师”，基于结构化分类法生成对抗性提示。
*   **组合式攻击生成**：攻击向量由四个组件构成：**主题（Topic）**、**人格（Persona）**、**目标（Goal）**和**策略（Tactic）**。通过这些组件的组合，生成语义连贯且具有欺骗性的提示。
*   **漏洞分类**：
    *   **Type A (RM 误判)**：RM 对有害内容打高分，但 LLM 本身不生成此类内容。
    *   **Type B (策略漏洞)**：LLM 生成有害内容，但 RM 能正确识别并打低分。
    *   **Type C (系统性弱点)**：LLM 生成有害内容且 RM 给出高分（最危险）。
*   **层次化自适应采样**：系统会根据攻击的成功率动态调整组件的采样权重。如果某种组合成功诱导了模型失效，其权重会通过公式进行乘性增加，从而实现针对性探索。

#### 第二阶段：端到端系统修复（End-to-End System Repair）
*   **奖励模型修复**：首先利用发现的 Type A 和 Type C 数据微调 RM，提升其辨别有害内容的能力。
*   **核心模型优化**：使用修复后的 RM 作为奖励信号，通过强化学习算法（如 Dr. GRPO）优化核心 LLM，确保策略调整基于正确的安全评估。

*   **安全性显著提升**：ARES 在 StrongReject 和 HarmBench 上的安全率分别达到 0.97 和 0.95，远超基线模型。
*   **能力保持良好**：在大幅提升安全性的同时，MMLU 和 GSM8K 等能力指标几乎没有下降，且过度拒绝率（XSTest）保持在较低水平（0.10）。

### 7. 优点
*   **系统性视角**：首次明确提出并解决了 LLM 和 RM 协同失效的“系统性弱点”问题。
*   **自适应性**：通过动态权重调整，能够自动发现特定模型最脆弱的领域，而非盲目搜索。
*   **端到端闭环**：将红队测试与模型修复紧密结合，形成了一个从发现到解决的完整工程闭环。

### 8. 不足与局限
*   **算力开销**：漏洞发现阶段需要大量的推理计算，对于资源受限的团队可能存在门槛。

*   **覆盖范围**：目前主要针对单轮文本交互，尚未扩展到多轮对话、长文本或多模态安全场景。
*   **对抗性后缀**：未针对 GCG 等基于梯度的离散对抗后缀攻击进行专门设计。
