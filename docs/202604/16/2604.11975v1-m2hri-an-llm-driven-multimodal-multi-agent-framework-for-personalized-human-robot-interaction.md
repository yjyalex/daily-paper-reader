## 摘要
多机器人系统在家庭和医院等社会环境中具有巨大的应用前景，然而现有的多机器人研究往往将机器人视为功能相同的个体，忽视了机器人的个体身份如何影响用户感知，以及在存在这种个体差异时，协调机制如何塑造多机器人行为。为了解决这一问题，我们提出了 M2HRI，这是一个基于大语言模型构建的多模态多智能体框架，它为每个机器人赋予了独特的个性和长期记忆，并设计了基于这些差异的协调机制。在一项针对多智能体人机交互（HRI）场景的受控用户研究（n = 105）中，我们发现：由大语言模型驱动的性格特征具有显著的可辨识性并能提升交互质量；长期记忆改善了个性化程度和偏好感知；而中心化协调机制在显著减少行为重叠的同时提升了整体交互质量。这些结果共同表明，智能体的个体特性和结构化协调对于实现连贯且符合社交规范的多智能体人机交互至关重要。项目网站和代码可在 https://project-m2hri.github.io/ 获取。

## Abstract
Multi-robot systems hold significant promise for social environments such as homes and hospitals, yet existing multi-robot works treat robots as functionally identical, overlooking how robots individual identity shape user perception and how coordination shapes multi-robot behavior when such individuality is present. To address this, we introduce M2HRI, a multimodal multi-agent framework built on large language models that equips each robot with distinct personality and long-term memory, alongside a coordination mechanism conditioned on these differences. In a controlled user study (n = 105) in a multi-agent human-robot interaction (HRI) scenario, we find that LLM-driven personality traits are significantly distinguishable and enhance interaction quality, long-term memory improves personalization and preference awareness, and centralized coordination significantly reduces overlap while improving overall interaction quality. Together, these results demonstrate that both agent individuality and structured coordination are essential for coherent and socially appropriate multi-agent HRI. Project website and code are available at https://project-m2hri.github.io/.

#### 1. 核心问题与研究动机
随着机器人进入家庭和医院等社交环境，交互模式正从“单机对单人”向“多机对单人/多人”转变。然而，现有的多机器人交互（HRI）研究通常将机器人视为功能相同的“克隆体”，忽视了机器人的**个体身份（Individuality）**（如性格、记忆）对用户感知的影响。此外，当机器人具备不同个性时，如何协调它们的行为以避免冲突并保持社交适宜性也是一个挑战。本论文旨在探索如何通过大语言模型（LLM）赋予机器人独特的个性和记忆，并研究这些因素如何影响交互质量。

#### 2. 方法论：M2HRI 框架
M2HRI 采用了一种结合多模态感知、个体化建模和中心化协调的架构：
*   **智能体架构 ($a_i = \langle \phi_i, \Lambda_i \rangle$)**：
    *   **身份配置 ($\phi_i$)**：包含长期记忆（语义和情境记忆）、基于“大五人格”（Big Five）模型的性格向量，以及动作能力集。
    *   **认知循环 ($\Lambda_i$)**：包含感知模块（利用 VLM 将视觉和语音转化为语义描述）、推理模块（LLM 驱动的规划器）和执行模块。
*   **关键技术细节**：
    *   **性格映射**：将大五人格维度的数值（1-5）转化为自然语言描述符，注入 LLM 的系统提示词中。
    *   **记忆管理**：使用向量数据库（ChromaDB）存储用户偏好和历史交互，通过 LLM 控制器进行上下文相关的检索。
    *   **中心化协调机制**：引入一个共享的协调器（Coordinator），它根据所有机器人的观察、性格和对话上下文，为每个机器人计算“响应似然得分”。只有得分超过阈值的机器人才能发言，从而确保轮替的有序性。

#### 6. 主要结论与发现
*   **性格辨识度（RQ1）**：用户能显著区分机器人的性格，尤其是“开放性”和“神经质”特征。但“尽责性”在短期交互中较难被察觉。
*   **记忆的作用（RQ2）**：长期记忆显著提升了机器人对用户偏好的感知和召回准确率，使交互更具个性化，但对“交互自然度”的提升相对有限。
*   **协调的重要性（RQ3）**：中心化协调机制极大地减少了机器人之间的发言重叠（行为冲突），显著提升了对话的流畅度和响应的适宜性。

#### 7. 优点
*   **系统性强**：首次将 LLM 驱动的性格、长期记忆与多机器人协调机制整合进统一的 HRI 框架。
*   **社交属性突出**：通过性格差异化解决了多机器人系统中“功能同质化”的问题，使机器人更像独立的社交主体。
*   **协调方案优雅**：利用 LLM 作为协调器，能够理解复杂的社交语境来分配发言权，而非简单的固定规则。

#### 8. 不足与局限
*   **评估形式限制**：采用了视频观察而非实地交互，可能无法完全捕捉用户在真实场景中的生理和心理反馈。
*   **性格表现力不均**：某些性格维度（如尽责性）需要长期观察才能体现，在短视频实验中效果不明显。
*   **扩展性挑战**：目前的实验仅限于两台机器人，在更多机器人或更复杂的动态环境中，中心化协调器的计算延迟和决策复杂度可能会增加。
