## 摘要
通过扩展思维链来缩放推理时计算已成为提升大语言模型推理能力的主流范式。然而，现有研究隐含地假设更长的思考总能带来更好的结果，而这一假设在很大程度上尚未得到检验。我们系统地研究了随着计算预算的增加，额外推理 token 的边际效用如何变化。研究发现，在较高的预算下，边际收益显著递减，且模型表现出“过度思考”现象，即延长的推理过程往往会导致模型放弃先前正确的答案。此外，我们证明了最佳思考长度随问题难度而异，这表明统一的计算分配并非最优。我们的成本感知评估框架表明，在适度的预算下停止可以显著减少计算开销，同时保持相当的准确率。

## Abstract
Scaling test-time compute through extended chains of thought has become a dominant paradigm for improving large language model reasoning. However, existing research implicitly assumes that longer thinking always yields better results. This assumption remains largely unexamined. We systematically investigate how the marginal utility of additional reasoning tokens changes as compute budgets increase. We find that marginal returns diminish substantially at higher budgets and that models exhibit ``overthinking'', where extended reasoning is associated with abandoning previously correct answers. Furthermore, we show that optimal thinking length varies across problem difficulty, suggesting that uniform compute allocation is suboptimal. Our cost-aware evaluation framework reveals that stopping at moderate budgets can reduce computation significantly while maintaining comparable accuracy.

### 1. 核心问题与研究动机
*   **核心问题**：挑战“推理时间越长，模型表现越好”的普遍假设。
*   **研究动机**：随着 DeepSeek-R1 和 OpenAI o1 等模型的成功，通过延长思维链（CoT）来提升性能成为主流。然而，作者借鉴经济学中的“边际效用递减法则”，怀疑过长的推理可能导致模型“过度思考”（Overthinking），即模型在冗长的思考过程中可能会推翻原本正确的直觉，最终转向错误答案。此外，极高的推理成本也使得研究“何时停止思考”具有重要的实际意义。

### 2. 核心方法论
论文提出了一套系统化的分析框架，主要包含以下关键技术细节：
*   **预算强制（Budget Forcing）**：通过在模型试图结束时添加“Wait”标记或在达到预算时强制解码结束符，将推理长度控制在 500 到 16,000 token 之间。
*   **边际效用（Marginal Utility, MU）**：定义为每增加 500 个 token 带来的准确率变化。
*   **翻转事件跟踪（Flip Event Tracking）**：
    *   **正向翻转**：错误 $\rightarrow$ 正确（有益思考）。
    *   **负向翻转**：正确 $\rightarrow$ 错误（过度思考）。
    *   **翻转比率**：负向翻转与正向翻转的比率，大于 1 表示思考弊大于利。
*   **过度思考指标**：监控犹豫标记（如“Wait”、“Actually”）、答案震荡次数和置信度轨迹。
*   **成本感知评估（Cost-Aware Evaluation）**：引入效用函数 $U_\lambda(t) = Acc(t) - \lambda \cdot \frac{t}{t_{max}}$，其中 $\lambda$ 代表对计算成本的敏感度。

### 3. 实验设计
*   **模型对比**：选取了两个具有代表性的 32B 参数开源模型：**DeepSeek-R1-32B** 和 **s1-32B**。

### 6. 主要结论与发现
*   **边际效用递减**：随着计算量增加，准确率提升速度迅速放缓。在 AIME 上，R1 模型在 12K token 之后边际效用转为负值。
*   **过度思考确实存在**：当预算超过约 7K token 时，负向翻转开始超过正向翻转（翻转比 > 1）。
*   **难度决定最优长度**：简单问题（Level 1-2）在 1K-2K token 时即达到峰值，而难题（Level 5）在 7.5K-8K 左右达到峰值。

*   **提前停止的潜力**：使用基于指标的自适应停止策略，可以用 60% 的计算量实现 97% 的峰值准确率。

### 7. 优点与亮点
*   **视角新颖**：首次系统性地挑战了“思考越长越好”的直觉，并引入经济学概念进行量化。

### 8. 不足与局限
*   **领域局限**：主要集中在数学和科学推理，对于创意写作、法律分析等其他领域的过度思考现象尚未探讨。
