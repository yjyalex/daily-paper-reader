## TOOL SPEC : Accelerating Tool Calling via Schema-Aware and Retrieval-Augmented Speculative Decoding 

Heming Xia 1, Yongqi Li 1* , Cunxiao Du 1, Mingbo Song 2, Wenjie Li 11 Department of Computing, The Hong Kong Polytechnic University 2 Peking University

Abstract 

Tool calling has greatly expanded the practi-cal utility of large language models (LLMs) by enabling them to interact with external appli-cations. As LLM capabilities advance, effec-tive tool use increasingly involves multi-step, multi-turn interactions to solve complex tasks. However, the resulting growth in tool interac-tions incurs substantial latency, posing a key challenge for real-time LLM serving. Through empirical analysis, we find that tool-calling traces are highly structured, conform to con-strained schemas, and often exhibit recurring invocation patterns. Motivated by this, we pro-pose T OOL SPEC , a schema-aware, retrieval-augmented speculative decoding method for accelerating tool calling. T OOL SPEC exploits predefined tool schemas to generate accurate drafts, using a finite-state machine to alternate between deterministic schema token filling and speculative generation for variable fields. In ad-dition, T OOL SPEC retrieves similar historical tool invocations and reuses them as drafts to fur-ther improve efficiency. T OOL SPEC presents a plug-and-play solution that can be seamlessly integrated into existing LLM workflows. Ex-periments across multiple benchmarks demon-strate that T OOL SPEC achieves up to a 4.2×

speedup, substantially outperforming existing training-free speculative decoding methods.

training-free solution that can be seamlessly integrated into existing LLM workflows. • Extensive experiments across diverse models and tasks demonstrate that T OOL SPEC consis-tently achieves a 3.5×∼ 4.2× speedup, with up to a 61% relative improvement over the previous state-of-the-art SD method. • Further analysis validates the effectiveness of our proposed components and highlights the impact of format adherence on T OOL SPEC ’s speedup, indicating its efficiency potential as LLM capabilities continue to advance. 

2 Related Work 

Efficient Tool Calling As large language mod-els (LLMs) increasingly rely on multi-step, multi-turn tool-calling workflows, inference latency in LLM serving has grown substantially. To address this, recent studies leverage directed acyclic graphs (DAGs) to model inter-tool dependencies, enabling parallel execution of independent API calls and thereby reducing latency (Kim et al., 2024; Zhu et al., 2025). Another line of work executes tool calls proactively during generation to minimize sys-tem idle time (Xu et al., 2024; Nichols et al., 2025). In contrast to these prior methods that focus on tool execution, this work shifts attention to accel-erate tool-calling generation, to further improve inference efficiency of tool-use LLMs. 

Speculative Decoding In recent years, Specu-lative Decoding (SD) has emerged as a widely adopted paradigm for accelerating LLM infer-ence (Leviathan et al., 2023; Xia et al., 2023). At each decoding step, SD first generates a sequence of draft tokens, which are subsequently verified in parallel by the target LLM (Xia et al., 2024). Re-cent advances in SD, such as the Eagle series (Li et al., 2024b,a, 2025), introduce lightweight draft models layered on top of the target LLM, achiev-ing promising speedups. While these approaches require additional training or parameters, an al-ternative line of research focuses on training-free SD methods (Luo et al., 2025; Saxena, 2023; Xia et al., 2025). 3 Preliminary Study 

In this section, we conduct an empirical analysis of tool-calling generation, covering (i) the latency breakdown between generation and tool execution, and (ii) analysis of tool-call patterns. 

3.1 Latency Breakdown 

We begin by analyzing the latency bottlenecks in tool calling. Experiments are conducted using Qwen2.5-Instruct series on ToolBench (Qin et al., 2024b). As shown in Figure 2, tool execution la-tency remains nearly constant across model scales, whereas tool-calling generation latency increases significantly, accounting for up to 96 % of the end-to-end latency for Qwen2.5-72B-Instruct. These results indicate that tool-calling generation is the dominant bottleneck in tool-calling pipelines, par-ticularly in multi-tool, multi-turn scenarios, under-scoring the importance of accelerating generation.

4 TOOL SPEC 

In this section, we introduce T OOL SPEC , a plug-and-play SD approach for accelerating tool-calling generation. The method leverages the highly struc-tured nature of tool calls and recurring historical patterns to construct high-quality drafts with negli-gible overhead. As shown in Figure 5, T OOL SPEC 

integrates two complementary drafting strategies: 

schema-aware drafting and retrieval-augmented speculation , which we detail below. 

4.1 Task Formulation 

Given a user query and an input context containing system prompts and tool documentation, T OOL -SPEC extracts tool specifications from the doc-umentation and converts them into a structured schema. The schema defines (i) the set of schema tokens Σschema , (ii) a set of callable tools Ctools (e.g., 

ForgotPassword ), and (iii) the parameter fields as-sociated with each tool (e.g., Status , Username ). Let M denote the target model, x the input context, and y = ( y1, y 2, . . . , y L) the tool-call se-quence to be generated. At decoding step t, fol-lowing the standard SD paradigm, T OOL SPEC first proposes a sequence of draft tokens ˜yt:t+γ−1 =(˜ yt, ˜yt+1 , . . . , ˜yt+γ−1), as predictions of the subse-quent decoding steps. The target model M then verifies these drafted candidates in parallel and ac-cepts the longest prefix that is consistent with its probability distribution. When the drafting process incurs significantly lower latency than verification, maximizing the acceptance length of ˜y directly improves the effective decoding parallelism and reduces the end-to-end latency. 7 Conclusion 

This work presents T OOL SPEC , a schema-aware, retrieval-augmented speculative decoding approach for accelerating tool-calling generation. It requires no additional training or auxiliary models, enabling seamless integration into existing LLM workflows. Extensive experiments across various LLMs and tool-use benchmarks demonstrate that T OOL SPEC 

achieves a 3.5×∼ 4.2× speedup while preserving output distributions. Further analysis highlights the effectiveness of the proposed components and the impact of format adherence on the efficiency of TOOL SPEC . We hope this work advances efficient tool use in LLMs and provides practical insights into scalable tool-calling generation. 8Limitations 

Due to computational constraints, our experiments do not include larger-scale language models such as Qwen2.5-72B-Instruct (Yang et al., 2024). Never-theless, we expect T OOL SPEC to generalize well to these models and achieve further improvements in tool-calling generation efficiency. Additionally, our approach assumes strong adherence to predefined tool schemas and the existence of recurring invoca-tion patterns, which are consistent with real-world application scenarios characterized by frequent tool usage. As discussed in Section 3 and 6.2, we an-ticipate that the effectiveness of T OOL SPEC will continue to improve as LLM capabilities advance.

A Details of T OOL SPEC 

In this section, we provide details of our proposed TOOL SPEC , including the implementation of paral-lel verification and the detailed algorithm. 

A.1 Parallel Verification 

Following prior work (Li et al., 2024b; Cai et al., 2024), T OOL SPEC adopts a tree-based attention mechanism to verify multiple candidate draft se-quences in parallel. Specifically, the candidate drafts are compacted into a single flattened se-quence and fed into the target model M. The model then performs a single forward pass to ver-ify all draft sequences simultaneously using a tree-structured attention mask, as illustrated in Figure 1. To support different drafting strategies, T OOL -SPEC employs two types of draft-tree architectures: (1) Static trees. For open-ended states (e.g., qv

and qo), T OOL SPEC adopts a general SD strategy (e.g., TR) for text generation, where a pre-defined static tree architecture is used for drafting; (2) Dy-namic trees. For structural states (e.g., qt and qp)and retrieval-augmented speculation, T OOL SPEC 

constructs dynamic tree masks that adapt to the length of each draft sequence, thereby reducing unnecessary verification computation. 

A.2 Algorithm 

We present the pseudo code of T OOL SPEC in Algo-rithm 1. Concretely, at each decoding step, T OOL -SPEC checks whether the current suffix matches any sequence in historical calls. If a match is found, it extracts the corresponding continuations as high-quality drafts. Otherwise, T OOL SPEC employs a finite-state machine (FSM) to guide schema-aware drafting, thereby accelerating tool-call generation.
