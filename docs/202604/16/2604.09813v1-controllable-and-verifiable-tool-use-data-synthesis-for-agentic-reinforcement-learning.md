## 摘要
现有的合成工具使用语料库主要针对离线监督微调（SFT）设计，而强化学习（RL）则需要支持奖励可检查在线回放的可执行环境。我们提出了 COVERT，这是一个两阶段流水线：首先通过带有多级验证的自我演化合成生成可靠的基础工具使用轨迹，然后应用保持真值（oracle-preserving）的增强手段，系统性地增加环境复杂度。这些增强引入了干扰工具、间接或模糊的用户查询，以及噪声、多格式或错误的工具输出，同时严格保留原始工具调用和最终答案作为地面真值。这种设计使得标准情况可以通过参考匹配进行自动奖励计算，而对于错误检测等特殊行为，则可以通过轻量级的评判辅助验证，从而支持工具调用策略的 RL 优化。在 Qwen2.5-Instruct-14B 上，COVERT-RL 将 BFCL v3 的整体准确率从 56.5 提升至 59.9，将 ACEBench 的准确率从 53.0 提升至 59.3，且在通用能力基准测试上的退化极小；当在 SFT 之上叠加时，准确率进一步达到 62.1 和 61.8，证实了叠加增益。这些结果表明，保持真值的合成环境提供了一个实用的 RL 精炼阶段，与 SFT 互补，能够提高在模糊性和不可靠工具反馈下的工具使用鲁棒性。

## Abstract
Existing synthetic tool-use corpora are primarily designed for offline supervised fine-tuning, yet reinforcement learning (RL) requires executable environments that support reward-checkable online rollouts. We propose COVERT, a two-stage pipeline that first generates reliable base tool-use trajectories through self-evolving synthesis with multi-level validation, and then applies oracle-preserving augmentations that systematically increase environmental complexity. These augmentations introduce distractor tools, indirect or ambiguous user queries, and noisy, multi-format, or erroneous tool outputs, while strictly preserving oracle tool calls and final answers as ground truth. This design enables automatic reward computation via reference matching for standard cases and lightweight judge-assisted verification for special behaviors such as error detection, supporting RL optimization of tool-calling policies. On Qwen2.5-Instruct-14B, COVERT-RL improves overall accuracy on BFCL v3 from 56.5 to 59.9 and on ACEBench from 53.0 to 59.3, with minimal regressions on general-ability benchmarks; when stacked on SFT, it further reaches 62.1 and 61.8, confirming additive gains. These results suggest that oracle-preserving synthetic environments offer a practical RL refinement stage, complementary to SFT, for improving tool-use robustness under ambiguity and unreliable tool feedback.

### 论文总结：面向智能体强化学习的可控且可验证工具使用数据合成 (COVERT)

#### 1. 核心问题与整体含义（研究动机和背景）
当前大型语言模型（LLM）的工具使用能力主要通过**监督微调（SFT）**提升，但 SFT 依赖静态的离线轨迹，难以应对现实中复杂的动态环境（如模糊查询、工具故障、噪声输出）。**强化学习（RL）**被认为是进一步优化策略的有效手段，但其面临两大挑战：
*   **缺乏可执行环境**：合成数据通常是静态的，不支持在线回放（Rollout）。
*   **奖励验证困难**：在复杂或异常场景下，难以自动且准确地判定模型行为的对错。
本文提出 **COVERT** 框架，旨在构建一个“RL 就绪”的合成环境，通过可控的扰动和可验证的奖励机制，提升模型在工具调用中的鲁棒性。

#### 2. 方法论：核心思想与技术细节
COVERT 采用两阶段流水线构建强化学习环境：

*   **第一阶段：可靠基础轨迹生成（Stage I）**
    *   **自我演化合成**：从少量种子数据开始，利用 LLM 迭代生成新的工具使用案例。
    *   **多级验证**：通过**格式检查器**（JSON 规范）、**规则检查器**（参数约束）和**语义检查器**（LLM 判定逻辑一致性）确保基础轨迹的 100% 可靠性。
*   **第二阶段：保持真值的增强（Stage II - Oracle-Preserving Augmentation）**
    *   **核心思想**：在改变环境组件（如增加干扰工具、改写查询、注入工具错误）的同时，严格保留原始的“真值（Oracle）”调用和答案，作为奖励计算的基准。
    *   **增强类型**：
        *   **精确可验证类**：引入干扰工具、间接查询、多格式/噪声输出。奖励通过参考匹配（Reference Matching）自动计算。
        *   **评判辅助类**：引入错误输出（如连接失败）、无关查询。奖励由轻量级 LLM 评判员根据预设的“真值标签”进行判定。
*   **强化学习算法**：采用 **GRPO**（Group Relative Policy Optimization），通过计算组内相对优势进行优化，无需额外的价值模型。
*   **奖励函数**：$R = \lambda_{fmt} R_{fmt} + \lambda_{cor} R_{cor}$。其中 $R_{fmt}$ 负责格式规范，$R_{cor}$ 负责基于真值匹配或评判员反馈的正确性。

#### 3. 实验设计
*   **模型**：以 Qwen2.5-Instruct-7B 和 14B 为基础模型。
*   **工具使用基准测试**：
    *   **BFCL v3** (Berkeley Function Calling Leaderboard)：评估单轮、多轮及并行调用。
    *   **ACEBench**：评估原子能力、正常场景、特殊场景（噪声/错误）及智能体规划。
*   **通用能力基准测试**：LiveCodeBench (代码)、AIME/Math500 (数学)、MMLU-Pro (知识)、IFEval (指令遵循) 等，用于检测是否存在“灾难性遗忘”。
*   **对比方法**：Base 模型、TOUCAN-SFT（大规模 SFT 训练后的模型）、COVERT-RL（直接在 Base 上做 RL）、TOUCAN-SFT + COVERT-RL（SFT 后再做 RL）。

#### 4. 资源与算力
*   **硬件**：使用 **8 × NVIDIA H200 GPU**。
*   **训练时长**：7B 模型单次 RL 训练约 **28 小时**；14B 模型约为 7B 的 **1.6 倍**（约 45 小时）。
*   **框架**：使用开源 RL 框架 **VERL**。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了 7B 和 14B 两个参数量级，并在 10 多个主流基准测试上进行了验证。
*   **消融与组合**：对比了 RL 独立作用以及 RL 与 SFT 叠加的效果，验证了两者之间的互补性。
*   **公平性**：所有模型（包括基线）均使用相同的“最小化提示词（Minimal Prompt）”进行评估，排除了提示词工程带来的偏差，实验设计较为客观、严谨。

#### 6. 主要结论与发现
*   **显著提升**：COVERT-RL 在 14B 模型上将 BFCL v3 准确率提升了 3.4%，ACEBench 提升了 6.3%。
*   **SFT 与 RL 互补**：在 SFT 模型基础上叠加 RL 效果最好（14B 达到 62.1% 和 61.8%），证明 SFT 提供结构先验，而 RL 强化语义正确性和鲁棒性。
*   **鲁棒性增强**：在处理“无关查询检测”和“特殊错误处理”方面进步最明显，这直接归功于 Stage II 的针对性增强。
*   **通用能力保持**：RL 训练后，模型在数学、代码和知识类测试中的表现几乎没有下降，甚至在某些指标上略有提升。

#### 7. 优点
*   **可验证性强**：通过“保持真值”的设计，解决了合成数据在 RL 训练中难以自动评分的痛点。
*   **可控性高**：能够针对性地模拟现实中的各种“坏情况”（如 API 报错、干扰项），使模型学会处理异常。
*   **效率高**：采用 GRPO 算法和轻量级评判员，在有限算力下实现了显著的性能增益。

#### 8. 不足与局限
*   **长程规划限制**：目前的模拟环境主要针对短到中程交互，对于需要极长步数、状态高度耦合的复杂智能体任务（Long-horizon planning）覆盖不足。
*   **评判员偏差**：虽然引入了 LLM 评判员处理特殊行为，但尚未进行大规模的人类一致性研究来验证评判员的绝对可靠性。
*   **环境真实度**：尽管模拟了噪声和错误，但与真实的、具有复杂状态机变化的生产环境 API 相比，仍存在一定的分布差异。
