## 摘要
语言模型推理可以通过在测试时使用支架来显著改进，这些支架在不同原语之间扩展推理计算——单条轨迹内的序列推理、独立采样的并行轨迹，以及将多条推理轨迹聚合为最终响应。然而，在后训练过程中，语言模型仅针对单条轨迹内的序列推理进行优化。我们引入了序列-并行-聚合强化学习（SPIRAL）框架，在该框架中，语言模型被训练使用所有三种原语，作为统一推理计算流程的一部分。具体来说，语言模型首先并行采样一组独立轨迹，每条轨迹通过序列思维链推理产生，然后基于这些轨迹生成最终的聚合轨迹；所有组件都针对最终聚合响应的奖励进行端到端优化。为了训练该系统，SPIRAL使用集合强化学习来教导模型生成一组对聚合器集体有用的轨迹，并使用标准强化学习来教导模型将集合聚合为改进的最终响应。我们在推理任务上的实验表明，SPIRAL能够有效地随推理计算扩展，在三种计算原语均被扩展时，其扩展效率比GRPO高出高达11倍，性能高出15%。

## Abstract
Language model reasoning can be substantially improved at test time via scaffolds that scale inference compute across different primitives -- sequential reasoning within a trace, independently sampled parallel traces, and aggregation of multiple reasoning traces into a final response. During post-training, however, language models are optimized only for sequential reasoning within a single trace. We introduce Sequential-Parallel-Aggregative Reinforcement Learning (SPIRAL), a framework in which a language model is trained to use all three primitives, as part of a unified inference compute pipeline. Concretely, the language model first samples a set of independent traces in parallel, each produced through sequential chain-of-thought reasoning, and then generates a final aggregation trace conditioned on those traces; all components are optimized end-to-end against the reward of the final aggregated response. To train this system, SPIRAL uses set reinforcement learning to teach models to produce a set of traces that are collectively useful for an aggregator and standard reinforcement learning to teach models to aggregate the set into improved final responses. Our experiments on reasoning tasks show that SPIRAL effectively scales with inference compute, outperforming GRPO by up to 11$\times$ scaling efficiency and 15% higher performance when all three compute primitives are scaled.

## 1. 核心问题与整体含义（研究动机和背景）

- **现有瓶颈**：语言模型推理在测试时可以通过三种计算原语提升性能：单轨迹内的顺序推理、独立采样的并行轨迹、以及聚合多条轨迹生成最终响应。然而，**后训练阶段模型仅针对顺序推理（单条链式思维）进行优化**，完全忽略了并行采样和聚合协同的能力。
- **不良后果**：当测试时扩展顺序计算，模型常会将计算预算浪费在常规操作上；扩展并行计算时，样本倾向于坍塌为冗余的重复尝试；而在聚合阶段，模型难以有效验证、筛选和组合异见思路。因此，当前需要手工设计复杂的支架来协调多个专用模型（如验证器和修订器）。
- **研究目标**：提出一个统一的后训练框架，让模型在训练时**同时学习顺序、并行和聚合三种推理原语**，从而在测试时能够自适应地、高效地协调这些计算，减少手工设计。

## 2. 论文提出的方法论

- **核心思想**：将推理过程建模为一个两层强化学习问题：
  1. 给定问题 \( x \)，策略 \( \pi_\theta \) 先并行采样 \( n \) 个搜索轨迹 \( y_{1:n} \)。
  2. 然后策略以同一模型（或不同模型）在 \( x \) 和 \( y_{1:n} \) 的条件下生成聚合轨迹 \( y^* \)。
  3. 目标函数只依赖于最终聚合轨迹的奖励 \( r(x, y^*) \)，优化参数 \( \theta \) 使得期望奖励最大化。
- **关键技术细节**：
  - **集合强化学习（Set RL）**：用于优化搜索轨迹集合。集合奖励 \( f_{\text{spiral}}(x, y_{1:n}) = \mathbb{E}_{y^* \sim \pi_\theta(\cdot|x, y_{1:n})}[r(x, y^*)] \) 衡量一组搜索轨迹对聚合的集体贡献。通过 marginal set advantage 为每个搜索轨迹分配共同的学习信号（详见式 8）。
  - **标准强化学习**：用于优化聚合轨迹。每个聚合轨迹的 advantage 在其所属集合内计算（式 9），使用 per-set 基线降低方差。
- **算法流程（文字描述）**：
  - 对每个输入 \( x \)，采样 \( N_1 \) 个搜索轨迹。
  - 均匀无放回地抽取 \( K \) 个大小为 \( n \) 的集合 \( G_1, \ldots, G_K \)。
  - 对每个集合采样 \( N_2 \) 个聚合轨迹 \( y^{G_i}_j \)。
  - 计算每个集合的经验奖励均值 \( \hat{f}_{\text{spiral}}(x, G_i) \)，并得到全局集合基线 \( \hat{f}_{\text{spiral}}(x) \)。
  - 计算集合优势 \( A^\sharp(x, G_i) \) 以及每个搜索轨迹的边际集合优势 \( A^\sharp_{\text{marg}}(x, y_j) \)。

## 3. 实验设计

- **数据集**：过滤后的 **POLARIS-53k** 数学推理数据集。每次 batch 包含 256 个问题。
- **基准模型**：Qwen3-4b-Instruct-2507。
- **对比方法**：**GRPO**（Group Relative Policy Optimization），一种仅使用顺序推理的标准强化学习方法。
- **训练设置**：每问题 24 次 rollout，2 轮 RL 训练。SPIRAL 使用 \( N_1=8, K=4, n=4, N_2=4 \)，最大令牌数 4096；GRPO 每问题 12 条轨迹，最大令牌数 8192，使得两者单问题令牌预算相同（约 98k tokens）。
- **评估场景**
  1. **pass@k**：独立采样 k 个搜索轨迹，用 oracle 验证（无需聚合）。
  2. **递归自聚合（RSA）**：多轮分组-聚合流程，比较不同步数下的 pass@1。
  3. **顺序计算扩展**：增加单轨迹最大长度，评估 pass@1。
  4. **与多数投票对比**：在相同并行轨迹数下比较自聚合 vs 多数投票。
  5. **token 使用效率**：固定 token 预算下比较不同方法的性能。

## 6. 论文的主要结论与发现

- **SPIRAL 在 pass@k 上扩展效率比 GRPO 高 11 倍**，且保持更高熵值（未过早坍塌），表明搜索轨迹更具多样性。
- **在递归自聚合场景下，SPIRAL 比 GRPO 性能提升高达 13.5%**，说明其学习到了更适合聚合的搜索与精炼策略。
- **顺序计算扩展下所有方法表现相近**，但并行和聚合计算扩展时 SPIRAL 显著领先，凸显协同三种原语的重要性。
- **自聚合优于多数投票**，且 SPIRAL 的自聚合收益最大，表明模型内化聚合策略比手工规则更有效。
- **固定 token 预算下 SPIRAL 的 RSA 方法性能曲线最陡峭**，token 利用率更高。

## 7. 优点

- **创新性**：首次将集合强化学习（Set RL）系统性地应用于训练推理模型的并行搜索与聚合，形成端到端统一框架。
- **理论严谨**：提供 Set RL 梯度估计的无偏性证明，并讨论排序偏差的可处理性。
- **实用性**：训练计算预算与标准 GRPO 一致，且训练后模型在多种测试时计算扩展策略上均表现更优，无需额外手工设计支架。
- **消融深入**：通过对比自聚合 vs 多数投票、顺序 vs 并行扩展等，清晰地分离了不同原语的贡献。

## 8. 不足与局限

- **实验规模有限**：仅使用 4B 参数模型，未在更大模型（如 8B+）上验证，结论的泛化性存疑。
- **任务单一**：仅评估数学推理，未在代码生成、科学推理或开放领域任务上测试，实际应用范围待拓展。
- **训练复杂度**：两层生成（搜索+聚合）增加了训练内存和推理开销，文中未详细分析训练稳定性与超参数敏感度。
- **排序偏差假设**：算法在训练时假设集合目标函数对称（无序），但语言模型对输入顺序敏感，论文虽在理论上证明可通过有序采样克服，但实现时仍用无序集合采样，可能导致偏差。
- **聚合深度受限**：训练时仅两级聚合（搜索→聚合），测试时递归聚合虽获益但训练未直接优化多步聚合，可能存在训练-测试失配。
- **奖励函数单一**：仅使用最终答案正确性作为奖励，未考虑中间步骤质量或探索成本，可能鼓励短视行为。
- **消融不够完整**：未独立分析搜索轨迹的多样性度量、聚合阶段的验证能力、以及不同集合大小的效果。
