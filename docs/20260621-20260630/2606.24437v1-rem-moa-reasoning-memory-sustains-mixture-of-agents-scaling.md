## 摘要
混合智能体架构通过将多个LLM智能体组织为分层推理流水线来改善推理时扩展。然而，现有的MoA变体在深度增加时无法维持性能提升，表现出退化、早期平台期或饱和。我们提出ReM-MoA，一种记忆增强的MoA框架，通过两种机制维持扩展：（1）排名推理记忆，使用比较性评审智能体持久存储并排序所有层的推理轨迹；（2）精选多样化记忆路由方案，将不同的智能体暴露于成功和失败轨迹的不同组合中，在传播高质量推理的同时保持探索多样性。我们进一步引入可选的多领域评审精炼流水线，通过前沿模型监督提高排序质量。在涵盖数学、形式逻辑、代码、知识和常识的五个推理基准上，ReM-MoA在深度和宽度扩展方面均始终优于先前的MoA变体，且其优势随深度增加而扩大，确立了结构化的跨层推理记忆作为可扩展多智能体推理的关键缺失机制。

## Abstract
Mixture-of-Agents (MoA) architectures improve inference-time scaling by organizing multiple LLM agents into layered reasoning pipelines. However, existing MoA variants fail to sustain gains as depth increases, exhibiting degradation, early plateauing, or saturation. We propose ReM-MoA, a memory-augmented MoA framework that sustains scaling through two mechanisms: (1) a Ranked Reasoning Memory that persistently stores and ranks reasoning traces from all layers using a comparative Reviewer Agent, and (2) a Curated Diversified Memory Routing scheme that exposes different agents to distinct combinations of successful and failed traces, preserving exploration diversity while propagating high-quality reasoning. We further introduce an optional multi-domain Reviewer distillation pipeline that improves ranking quality through frontier-model supervision. Across five reasoning benchmarks spanning math, formal logic, code, knowledge, and commonsense, ReM-MoA consistently outperforms prior MoA variants across both depth and width scaling, and its advantage widens with depth, establishing structured cross-layer reasoning memory as a key missing mechanism for scalable multi-agent inference.

### 1. 核心问题与整体含义

- **研究动机**：Mixture-of-Agents (MoA) 架构通过将多个 LLM 智能体组织成多层流水线来提升推理时计算扩展。然而，现有 MoA 变体在增加层数（深度）时，性能会出现 **退化、早期平台期或饱和**，无法持续受益于深度扩展。这种缩放崩溃的原因包括：错误积累、局部通信导致优质推理轨迹丢失、以及多样性崩溃（智能体输出趋同）。
- **整体目标**：提出一种能够维持 MoA 深度扩展性能的新框架，核心是引入结构化的跨层推理记忆，同时保持智能体间的探索多样性。

### 2. 方法论

- **核心思想**：将 MoA 缩放视为“质量-多样性”联合保持问题，而非单纯的聚合或交互问题。
- **关键技术细节**：
  1. **排序推理记忆（Ranked Reasoning Memory）**：每层智能体生成推理轨迹后，由 **Reviewer Agent** 对所有轨迹进行 **对比评分**（1-10分）并给出针对性的理由（rationale）。评分和理由连同轨迹一起存入记忆，记忆可跨所有层访问。
  2. **精选多样化记忆路由（Curated Diversified Memory Routing）**：从第2层开始，每个提议智能体接收一个 **精选参考集**，包含高质量轨迹（Top n）、低质量轨迹（Bot n）或对比组合（Top a ∪ Bot b）。同一层内不同智能体的参考集不同，以保持探索多样性。路由函数按固定分布分配（例如 N=3 时：一个对比、一个正向、一个负向）。
  3. **可选的多领域评审精炼**：利用 **前沿模型（GPT-5.5）** 对框架自身产生的轨迹进行监督打分，然后用 LoRA 微调一个较小的评审模型（Qwen2.5-7B-Instruct），使其学会判断多种推理域的轨迹质量。训练数据包含 5 个不同领域的 500 个问题（共 1500 个元组）。
- **公式与流程（文字说明）**：
  - 记忆累积：`M_{≤l} = ∪_{k=1}^l M_k`，其中 `M_k` 是第 k 层提交的条目（轨迹+评分+理由）。
  - Reviewer 输出：`(s_l, ρ_l) = R(x, r_l)`，为每个轨迹赋予分数和理由。
  - 记忆条目：`M_l = {(r_{l,j}, s_{l,j}, ρ_{l,j})}_{j=1}^N`。
  - 提示构造：层1只有问题；层2+问题与参考集拼接。
  - 路由函数实例（N=3）：智能体1使用 Top1 ∪ Bot1（对比），智能体2使用 Top2（正向），智能体3使用 Bot2（负向）。

### 6. 主要结论与发现

- **深度/宽度扩展**：ReM-MoA 在所有配置下均优于所有基线，且 **优势随深度增加而扩大**（而非缩小）。在深度缩放时，Standard MoA 性能退化，RMoA 和 AttMoA 平台或饱和，而 ReM-MoA 持续提升。
- **蒸馏增强**：ReM-MoA*（使用蒸馏后的评审）进一步提升了性能，且在未见的领域上也能泛化（如 HellaSwag 从平台转为持续提升）。
- **关键组件**：跨层访问和路由多样化是维持缩放性能的核心，理由（rationale）提供细微改进。
- **确立结构化跨层推理记忆为 MoA 缩放的关键机制**。

### 7. 优点

- **方法创新**：首次将结构化跨层记忆引入 MoA，并同时关注质量保持和多样性保持，解决了多个缩放失败模式。
- **实验严谨**：深度和宽度双轴缩放，消融实验系统，蒸馏转移验证充分。
- **实用性强**：蒸馏可选，无蒸馏版本（ReM-MoA）已超越所有基线；使用开源模型，可复现。
- **结果显著**：在数学、形式逻辑、代码等需要深度推理的基准上，优势尤其突出（如形式逻辑在 L=9 时高于最强的 AttMoA 7.3 个百分点）。
- **清晰的工程设计**：路由函数设计合理，训练数据直接来自框架自身输出，避免分布失配。

### 8. 不足与局限

- **宽度扩展有限**：宽度仅探索到 N=8，未验证更大宽度（如 N=16, 32）下的表现。
- **提议池规模单一**：仅使用 7-8B 模型，未验证对更大（30B-70B）或更小（1-3B）模型的适用性。
- **评审开销**：每层需一次评审调用，总成本为 L·N + L + 1 次 LLM 调用，与 RMoA 相当但高于单次生成。蒸馏增加了额外训练成本。
- **继承 LLM 固有偏差**：评审智能体本身是 LLM，可能存在位置偏差、自我偏好等问题，对比评分和对比路由虽能减轻但无法消除。
- **计算成本高**：框架需要多次 LLM 调用，能量足迹较大。
- **蒸馏训练数据量较小**：仅 500 个问题（1500 个元组），可能限制评审模型的泛化极限。
