Abstract 

Large Language Models are being increas-ingly deployed as the decision-making core of autonomous agents capable of effecting change in external environments. Yet, in con-versational benchmarks, which simulate real-world customer-centric issue resolution sce-narios, these agents frequently fail due to the cascading effects of incorrect decision-making. These challenges are particularly pronounced for open-source LLMs with smaller parame-ter sizes, limited context windows, and con-strained inference budgets, which contribute to increased error accumulation in agentic settings. To tackle these challenges, we present the 

Failure-Aware Meta-Agentic (FAMA) frame-work. FAMA operates in two stages: first, it an-alyzes failure trajectories from baseline agents to identify the most prevalent errors; second, it employs an orchestration mechanism that ac-tivates a minimal subset of specialized agents tailored to address these failures by injecting a targeted context for the tool-use agent before the decision-making step. Experiments across open-source LLMs demonstrate performance gains up to 27% across evaluation modes over standard baselines. These results highlight that targeted curation of context through special-ized agents to address common failures is a valuable design principle for building reliable, multi-turn tool-use LLM agents that simulate real-world conversational scenarios.

Our experiments (§5.2) show that such errors are heterogeneous: dif-ferent model backbones, particularly open-source models (Yang et al., 2025; Abdin et al., 2024) with limited context windows, exhibit distinct domi-nant failure modes, indicating that static prompting strategies or agentic scaffolding architectures are insufficient. Prior work has largely sought to improve tool-using agents through task-specific supervised fine-tuning (Chen et al., 2023; Zeng et al., 2024), or reinforcement learning–based optimization (Acik-goz et al., 2025; Zhao et al., 2025b; Jiang et al., 2025). For multi-turn tool-use/tool-calling tasks, where trajectories are lengthy, partially observable, and highly variable (Kate et al., 2025; Wang et al., 2025b): collecting sufficient high-quality supervi-sion or reward-aligned experience becomes pro-hibitively expensive (Saeidi et al., 2026; Golubev et al., 2025; Qi et al., 2025; Chen et al., 2025b). Re-inforcement learning methods (Zhang et al., 2025b) are not overall effective for these challenges, as training requires large-scale curated trajectories, re-peated execution of complex tool interactions and long episodes to propagate sparse and delayed re-wards, leading to substantial compute and engineer-ing overhead (Wang et al., 2023, 2025c). Moreover, because training objectives are typically optimized over learning patterns of correct behavior, these methods rarely target the specific failure patterns exhibited by a given agent (Rauba et al., 2024), lim-iting their effectiveness in addressing systematic errors observed in practice. We have also observed that there has been lim-ited exploration (Nekoei et al., 2025; Fu et al., 2024) of training-free improvement techniques that extract failure signals from agent trajectories and use them to modularly orchestrate agentic systems built specifically on smaller open-source language models (Belcak et al., 2025). While larger models benefit from extensive pretraining and long context windows, their deployment is often impractical in real-world settings due to cost, latency, and pri-vacy constraints (Liang and Tong, 2025). More-over, scaling alone can mask, rather than resolve, fundamental deficiencies in long-horizon decision-making and tool use. Small language models ex-pose these limitations more clearly (Shen et al., 2024; Sinha et al., 2025), making them an ideal testbed for studying systematic agent failures un-der realistic resource constraints. In this work, we argue that reliable multi-turn tool use requires failure-aware agent composi-tion. We introduce a two-stage framework, named the Failure-Aware Meta-Agentic (FAMA) frame-work, that explicitly identifies the most prevalent failure modes in language agent conversation trajec-tories and dynamically adapts the tool-use agent’s internal prior context to mitigate those failures. FAMA operates in two stages. First, it analyzes fail-ure trajectories produced by a baseline agent and categorizes errors into a small set of common fail-ure types. Rather than treating all errors equally, this analysis surfaces the dominant errors that most strongly impact performance. 

In the second stage, the FAMA framework em-ploys a lightweight routing mechanism consisting of an orchestrator and a mitigation agent (Figure 2) that selectively activates a set of specialized LLM helper-agents designed to address these specific weaknesses (see §4.1). By dynamically construct-ing the agent’s prior context using only the nec-essary agents, FAMA avoids the overhead of ex-cessive context injection while directly addressing the agent’s most critical shortcomings (Figure 1). This design motivates our use of the term ‘ meta-agentic ’, as the proposed system does not act in the environment directly, but operates at a higher level by reasoning over agent behaviors, diagnosing their failures, and dynamically composing a minimal set of specialized agents to shape the decision-relevant context of the underlying tool-use agent. We evaluate FAMA across τ -bench, τ -trait and ACEBench using a set of open-source LM back-bones with parametric sizes ranging from 4-72 bil-lion. Our results demonstrate consistent improve-ments in tool-use accuracy, trajectory reliability, and context efficiency. FAMA yields noticeable task-success gains of up to 25 % in τ -bench, 27 %in ACEBench and 24 % in τ -trait over baselines for open-source model backbones, highlighting the effectiveness of targeted error mitigation over brute-force context injection. In summary, our main con-tributions are: 1. We propose the Failure-Aware Meta-Agentic (FAMA) framework, which operates in two stages: first, identifying the predom-inant errors of baseline agents, and then dy-namically constructing the prior context for the tool-use agent by selectively enabling the appropriate agents designed to mitigate these errors. 2. We demonstrate that the FAMA framework’s optimized context construction significantly improves tool-use performance of diverse open-source LLM agents across the τ -bench, 

τ -trait and ACEBench benchmarks. 3. We provide a comprehensive analysis of the improvements observed from FAMA over other baseline agents. 

2 Related Works 

LLM-based Tool-Use Benchmarks While early benchmarks primarily evaluated isolated tool calls in response to single-turn queries (Patil et al., 2024; Li et al., 2023; Qin et al., 2023), recent work has increasingly shifted toward multi-turn, interactive tool-use benchmarks (Yao et al., 2024; Barres et al., 2025; Chen et al., 2025a; He et al., 2025; Lu et al., 2025a; Liu et al., 2024). These benchmarks expose long-horizon reasoning challenges for language agents (Cemri et al., 2025; Sinha et al., 2025) under partial observability, and sustained environment and task constraints, highlighting the need for agent designs that explicitly address failure accumulation over extended interaction trajectories as a valuable direction. 

Training-Free Techniques for Tool-Using Agents 

A range of techniques, including structured prompt-ing (Yao et al., 2023; Wei et al., 2022), and modu-lar agent orchestration (Mishra et al., 2025; Zhang et al., 2025e; Zhao et al., 2025a), have been devel-oped to improve language agents without updat-ing model parameters. However, these approaches are predominantly evaluated on strong or propri-etary LLM backbones, with large context windows and robust reasoning capabilities (Qu et al., 2025; Chowa et al., 2025). This remains relatively under-explored for smaller open-source models remains comparatively, particularly in tool-use settings (Bel-cak et al., 2025) where limited context, weaker planning, and error accumulation pose significant challenges (Shen et al., 2024), and training tech-niques are dominantly explored (Jhandi et al., 2025; Zeng et al., 2025; Ma et al., 2025). FAMA attempts to tackle these challenges by specifically focusing on agentic design through failure modes of smaller open-source LLM agents. 

Failure-Aware Agentic Orchestration Recent work has begun to systematically analyze agent failures in multi-agent LLM systems (Cemri et al., 2025; Lu et al., 2025b), uncovering diverse fail-ure causes (Mishra et al., 2025; Sinha et al., 2025) and proposing taxonomies and annotation datasets to enable more principled debugging and evalu-ation (Zhu et al., 2025; Banerjee et al., 2025). Other efforts automate failure tracing and attribu-tion (Zhang et al., 2025c; Ge et al., 2025) to local-ize error sources. Although reflective (Shinn et al., 2023; Madaan et al., 2023) and experiential (Zhang et al., 2025d; Huang et al., 2025) feedback tech-niques exist, and recent episodic guidance/hinting works (Nekoei et al., 2025; Fu et al., 2024; Wang et al., 2025d) come close in principle, they do not leverage observed failure patterns to dynamically compose model-specific modular agent scaffolds under resource constraints, an angle we explore with our FAMA framework. 

3 Problem Statement 

For simulating ideal testbeds of tool-use in multi-turn conversations, we utilize tasks from bench-marks such as τ -bench (Yao et al., 2024), τ -trait (He et al., 2025), and ACEBench (Chen et al., 2025a). These benchmarks are specifically de-signed to assess the decision-making capabilities of language agents in realistic, multi-turn interac-tion settings. In these setups, both the user and the customer-service assistant are simulated by LLMs, enabling a controlled yet dynamic envi-ronment for analyzing interactive behavior. The customer-service agent is the language agent that generates the tokens signifying which tools are to be invoked, while following the specific domain policies. As explained in Shekkizhar et al. (2025), the benchmarks used here do not evaluate standard multi-agent settings (Stone and Veloso, 2000) as the distinct agents simulating different roles, such as the user or the assistant, can have private internal states and competing utilities. Further details on these benchmarks are provided in Appendix B. 

4 Method 

Although frontier models in agentic frameworks demonstrate strong performance on complex, dy-namic tasks, their deployment is often prohibitively expensive. Additionally, integrating these models into private or proprietary products is often imprac-tical due to cost and security constraints. Open-source models, by contrast, offer a viable alterna-tive; their smaller size enables local deployment, making them more cost-effective and better suited for privacy-sensitive applications. Recent works (Liu et al., 2025b; Rrv et al., 2025; Handa et al., 2025b) show that open-source models, along with inference-time frameworks built around them, can achieve performance comparable to fron-tier models on mathematical and reasoning tasks; however, their use within agentic frameworks to solve a complex task remains challenging. Due to their smaller size and the nature of the data on which they are fine-tuned, these models exhibit dis-tinct failure modes and limitations. Additionally, strict context constraints limit their applicability in complex multi-agent frameworks, where multiple agents must collaborate to solve a task, and rapidly fill up the context windows. These factors lead open-source models to exhibit weaker decision-making capabilities compared to frontier models. To address these challenges, we propose the 

Failure-Aware Meta-Agentic (FAMA) frame-work, a general and dynamic framework designed to identify failure patterns in the behavior of open-source model agents and determine an effective set of agents that achieves improved performance un-der resource constraints. In the following sections, we present and analyze the FAMA framework in detail. 

4.1 Failure-Aware Meta-Agentic Framework 

We assume the availability of a predefined set of agents A, where each individual agent is denoted by a ∈ A , and subsets of agents are denoted by 

A′ ⊆ A . The full agent set A is adapted from Mishra et al. (2025), which proposes a collection of specialized agents for improving performance in tool-calling environments, and is further extended with additional modules tailored to the benchmarks evaluated in this work. Specifically, A includes a Domain Constraints Extractor Agent (DCE), a Tool Suggestion Agent (TSA), a Tool Output Reformu-lator Agent (TOR), a Planner Agent, a Decision Verifier Agent, and a User Context Manager (Mem-ory) module. We further assume the availability of a prede-fined set of general error categories representing common agent failures, denoted by E, where a real-ized error instance is denoted by e ∈ E . Based on prior studies (Mishra et al., 2025; Shekkizhar et al., 2025; Kokane et al., 2024) and our own analysis, we categorize failures in tool-calling environments into four classes: (1) Domain Policy Violation, (2) 

Incorrect Retrieval from Complex Tool Outputs, 

(3) Contextual Misinterpretation and Hallucination, and (4) Incomplete Fulfillment or Early Stopping. Detailed definitions of these error categories are provided in Appendix E. Given the agent set A and the error category set E, we define |E| independent error analysis agents, each responsible for detecting the cause of failure associated with a specific error category e.These agents operate over a predefined list of fail-ure causes derived from empirical analysis of base-line trajectories. Additional details on the causes of failure for each error category are provided in Ap-pendix E. Each error analysis agent produces both a categorical decision and a supporting rationale in textual form. The textual outputs and rationales produced by all error analysis agents are concatenated into a single input and, together with the full interaction trajectory between the user and the tool-calling agent for a given task τ , are passed to the orchestra-tor agent for final failure attribution. This process explicitly accounts for cases in which an agent may initially make incorrect decisions but later recover after receiving feedback from the environment, en-suring that failure attribution reflects the overall trajectory rather than isolated errors. In the final stage, conditioned on the error cate-gories identified by the orchestrator agent and the functional definitions of agents in A, a dedicated mitigation agent determines an efficient subset of agents A∗ ⊆ A that can mitigate a specific error category e. Aggregating the mitigation agent’s rec-ommendations across the selected error categories yields a final agent subset that represents the mini-mal configuration required to address the dominant failure modes. We collect these recommendations across tasks and deploy multi-agent frameworks composed only of the selected agent subsets (See Algorithm 1). The results reported in Tables 1-4 demonstrate that FAMA consistently achieves better performance across different settings compared to IRMA, ReAct, and FC on both τ -bench, τ -trait, and ACEBench across different models.

Using all agents leads to inconsistent perfor-mance across models. Consistent with prior work, we evaluate open-source models on τ -bench in the IRMA setting. Table 1 shows that IRMA outperforms other baselines in a limited number of cases, indicating that while multi-agent frame-works can be beneficial, indiscriminately using all agents does not consistently improve performance across models and can, in some cases, degrade the base agent’s performance. 

FAMA consistently outperforms baseline meth-ods across different benchmarks. Tables 1–4 show that the effectiveness of an agentic frame-work is highly sensitive to both agent selection and domain, and that naively including all agents can degrade performance. By selectively enabling agents based on dominant failure modes, FAMA avoids unnecessary context injection and focuses mitigation on the most impactful errors. As shown in Table 1, FAMA outperforms ReAct, FC, and IRMA by 4.63%, 11.57%, and 5.27% in the Air-line domain and by 5.30%, 8.96%, and 6.15% in the Retail domain, averaged over five runs. 

FAMA is a generalizable method. Generaliz-ability is a key limitation of static multi-agent frameworks, which are typically tailored to frontier-model failures in specific domains and require ex-tensive manual redesign to adapt to changes. In contrast, FAMA is a dynamic framework that over-comes this limitation. Experiments across multiple domains and benchmarks, including ACEBench and τ -trait, show that FAMA consistently outper-forms other methods by adapting to agent failure behavior (See Tables 3 and 4). For detailed results on the τ -trait and ACEBench benchmarks, refer to Appendix D and C. The modular components in FAMA enable the easy addition of specialized agents to address specific error patterns. To further contextualize our results, we also evaluate reasoning-enabled (“thinking”) variants of Qwen3 models. As shown in Figure 6, while such variants can improve certain baseline meth-ods, they incur higher token usage (see Table 2) and frequently lead to context window overflows in multi-turn settings, where accumulated reasoning traces consume a significant portion of the available token budget. We quantify this effect by counting instances (refer to Table 8) that exceed the maxi-mum token limit across runs and treat them as fail-ures, as mitigating them would require truncating critical context (e.g., domain constraints or prior tool outputs), potentially leading to incorrect de-cisions. In contrast, FAMA achieves stronger and more stable performance across k without relying on expensive internal reasoning steps.

6 Conclusion 

In this work, we present the Failure-Aware Meta-Agentic (FAMA) framework, a dynamic multi-agent approach designed to overcome the limita-tions of open-source language agents in interactive environments such as tool calling. FAMA operates in two stages: first, it analyzes agent failures to identify dominant error patterns; second, it selects a minimal, task-relevant subset of agents via an or-chestrator framework to mitigate these errors. We evaluate FAMA across three interactive tool-calling benchmarks and four open-source LLMs. Results and analyses demonstrate the potential of FAMA as a generalizable, failure-aware orchestration frame-work for constructing optimized prior contexts as a key design principle for improving and building de-pendable, resource-efficient conversational agents. 

Limitations 

While the proposed Failure-Aware Meta-Agentic (FAMA) framework achieves consistent improve-ments across multiple benchmarks, several limita-tions remain. First, FAMA operates over a predefined pool of specialized agents inherited from the underlying agentic framework (§4.1). This design reflects a common setting in existing user-facing, modular tool-use systems, where a fixed set of components is orchestrated rather than dynamically generated. As a result, FAMA’s effectiveness is bounded by the coverage of this agent pool: failure modes that are not well captured by existing agents may not be adequately mitigated. Importantly, this assump-tion does not imply rigidity. FAMA is designed to be extensible. New specialized agents targeting emerging failure patterns can be incorporated with-out modifying the orchestration mechanism. How-ever, the current framework does not address the automatic discovery or synthesis of such agents, which remains an important direction for future work. Furthermore, the benchmarks considered in this study primarily focus on structured conversational environments. While these settings provide a con-trolled testbed for analyzing failure-aware orches-tration, they do not capture the full spectrum of in-teractive agent deployments. Extending FAMA to more diverse settings, including embodied, multi-modal, or open-ended environments with less well-defined failure taxonomies, is an important avenue for future investigation.

B Detail of Benchmarks 

Each task in τ -bench, τ -trait and ACEBench is formulated as a Partially Observable Markov Deci-sion Process (POMDP) (Åström, 1965), in which an assistant agent interacts with a user entity/agent through a sequence of natural language conversa-tions and tool invocations. At each step, the agent must decide which function calls to issue based on the current conversational context and partial ob-servations derived from the external environment. Executed function calls produce structured outputs that are incorporated into the evolving dialogue and inform subsequent decisions. This interaction proceeds over multiple turns until task termination, at which point the agent is evaluated using the ap-propriate benchmark-prescribed metric centered around successful conversation/task completion. 

τ -bench τ -bench evaluates tool using conver-sational agents in two real-world-style customer-service domains—retail and airline, where an agent must interact with a simulated user, follow domain policy rules, and use API tools to update/query a realistic backend state. In retail, the benchmark in-cludes a database with 500 users, 50 products, and 1,000 orders, exposes 15 tools total (7 write + 8 non-write), and provides 115 tasks spanning actions like cancellations, returns/exchanges, address changes, and information requests; in airline, it includes 500 users, 300 flights, and 2,000 reservations, has 13 tools (6 write + 7 non-write), and contains 50 tasks around booking/modifying/canceling reservations and refund-related workflows.

E Error Categories and Causes of the Failures 

In this section, we first describe the error categories and then elaborate on the underlying causes of fail-ure associated with each category. The definition of each error category is provided below. 

Domain Policy Violation: This error refers to a failure mode in which a tool-using agent issues an action or tool invocation that contravenes the explicit operational rules, constraints, or policies of the simulated domain defined by the environment. These policies govern which actions are permissi-ble under specific states and preconditions. A vio-lation occurs when the agent ignores, misinterprets, or incorrectly applies these domain rules, such as invoking a return operation that is disallowed by the retail domain’s return policy in τ -bench, re-sulting in an invalid state transition. Because the environment enforces domain policies strictly, such violations cause the task execution trajectory to ter-minate unsuccessfully, even if the agent’s intent aligns with the high-level task objective. 

Incorrect Retrieval from Complex Tool Outputs: 

refers to a failure mode in which a tool-using agent fails to correctly extract relevant information from prior tool responses stored in its context. This error typically arises when tool outputs are large, un-structured, or weakly structured (e.g., verbose logs, nested text fields, or mixed natural language and metadata), making it difficult for the agent to re-liably identify the correct entities, values, or state variables required for subsequent actions. As a re-sult, the agent conditions later tool calls or reason-ing steps on incorrect or incomplete information, leading to invalid decisions or task failure despite the necessary information being present in the en-vironment output. 

Contextual Misinterpretation and Hallucina-tion: refers to a failure mode in which a tool-using agent incorrectly infers the user’s underlying intent due to nuanced, ambiguous, or adversarial natural language instructions. Instead of grounding its interpretation in the domain’s available oper-ations and constraints, the agent over-literalizes phrasing, overlooks implied intent, or hallucinates an incorrect task formulation. This results in the selection and invocation of an inappropriate tool, such as executing a return operation when the user implicitly intends an exchange, despite the correct intent being inferable from contextual or domain knowledge. 

Incomplete Fulfillment or Early Stopping: 

refers to a failure mode in which a tool-using agent does not fully satisfy all components of the user’s task requirements before terminating or effectively abandoning the interaction. This error commonly arises in long-horizon, multi-turn trajectories where the agent must track multiple obligations across conversational turns while operating over large, unstructured tool outputs. As the interaction pro-gresses, the agent may lose track of pending sub-goals, become over-focused on intermediate tool calls, or prematurely conclude the task, resulting in unmet user requests despite partial progress. To enable judgment agents to detect task fail-ures, a detailed list of failure causes is required. Accordingly, we analyze the trajectories produced by baseline methods and provide a comprehensive list of failure causes for each error category, as described below. 

Cause of Domain Policy Violation      

> 1. The assistant performs an action that the policy explicitly forbids
> 2. The assistant skips a step that the policy explicitly requires
> 3. The assistant proceeds without meeting conditions the policy requires
> 4. The assistant makes commitments that contradict policy rules
> 5. The assistant uses data/arguments not provided by user or tools (fabrication)

Cause of Wrong Retrieval from Complex Tool Outputs       

> 1. The agent states information that differs from what the tool output actually contains
> 2. The agent miscounts items or provides wrong totals from tool outputs
> 3. The agent selects wrong items when correct op-tions exist in tool output
> 4. The agent misses relevant data in tool output that would change the action
> 5. The agent attributes data to the wrong entity when multiple exist in output
> 6. The agent uses wrong values (IDs, prices, dates, etc.) from tool output

Cause of Incomplete Fulfillment or Early Stopping     

> The user may have multiple requirements, questions, or actions they need completed. This error occurs when the agent:
> 1. Addresses only some parts of a multi-part request
> 2. Stops after encountering a difficulty without trying alternatives
> 3. Receives confirmation to proceed but never exe-cutes the action
> 4. Terminates the conversation with unfinished tasks

Cause of Contextual Misinterpretation and Hallucination            

> The user’s request may have specific intent, context, or nuances that the agent must correctly understand to take appropriate actions. This error occurs when the agent:
> 1. Understands the words but misses the actual intent
> 2. Confuses which entity/item/order the user is refer-ring to
> 3. Misapplies criteria or constraints the user specified
> 4. Takes an action that doesn’t match what the user actually wanted
