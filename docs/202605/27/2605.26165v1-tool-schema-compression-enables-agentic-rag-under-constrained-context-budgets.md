## 摘要
配备语言模型数十到数百个工具定义的智能体RAG系统面临一个关键资源冲突：工具模式消耗了检索增强生成所需的同一上下文窗口。我们首次系统性地研究了这种工具-上下文的权衡，评估了14个模型，涵盖1.5B-32B本地模型加上一个前沿API模型，在三个上下文预算（8K、16K、32K）下进行了6,566次可控API调用，使用了28个工具定义。应用TSCG保守配置文件压缩（节省44-50%的模式令牌），我们观察到一种二元启用效应：在8K令牌时，JSON模式工具定义完全溢出上下文窗口，导致近乎零的精确匹配率（平均2.6%），而压缩模式恢复了RAG功能，所有八个模型的平均精确匹配提升+20.5个百分点（在六个完全启用模型中提升+24.7个百分点）。在32K（两种格式都适用）时，五个测试模型中的四个显示差异<=1个百分点，确认该效应纯粹由预算驱动。在HotpotQA（50个多跳问题）上的外部验证显示，在相同溢出场景下精确匹配提升+48个百分点。前沿扩展测试表明，JSON模式在大约494个工具时溢出，而压缩模式在超过800个工具时仍可运行。我们的结果确立了工具模式压缩作为受限上下文部署中智能体RAG的必要基础设施层。所有代码、数据和检查点均已公开。

## Abstract
Agentic RAG systems that equip language models with dozens to hundreds of tool definitions face a critical resource conflict: tool schemas consume the same context window needed for retrieval-augmented generation. We present the first systematic study of this tool-context trade-off, evaluating 14 models spanning 1.5B-32B local models plus one frontier API model across 6,566 controlled API calls at three context budgets (8K, 16K, 32K) with 28 tool definitions. Applying TSCG conservative-profile compression (44-50% schema token savings), we observe a binary enablement effect: at 8K tokens, JSON-schema tool definitions overflow the context window entirely, yielding near-zero EM (2.6% average), while compressed schemas restore RAG functionality with +20.5 pp average exact-match lift across all eight models (+24.7 pp among the six exhibiting full enablement). At 32K -- where both formats fit -- four of five tested models show delta <= 1 pp, confirming the effect is purely budget-driven. External validation on HotpotQA (50 multi-hop questions) shows +48 pp EM under the same overflow scenario. Frontier scaling tests demonstrate that JSON schemas overflow at ~494 tools while compressed schemas remain operational beyond 800 tools. Our results establish tool-schema compression as a necessary infrastructure layer for agentic RAG in constrained-context deployments. All code, data, and checkpoints are publicly available.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：Agentic RAG 系统中，语言模型需要同时容纳工具定义（JSON Schema）和检索增强生成的上下文，二者竞争有限的上下文窗口。在部署中，工具模式可能占用 300–500 token/工具，28 个工具即约 11,000 token，超过了 8K 窗口的全部容量，导致 RAG 功能失效。
- **研究背景**：此前工作聚焦于自然语言提示压缩，忽略了结构化工具模式的压缩需求；同时，现有 RAG 评估均默认工具模式能放入上下文，但论文首次揭示在工具数≥28 时这一假设失效。
- **整体含义**：工具模式压缩不再只是可选的优化手段，而是受限上下文部署中 **必要的使能基础设施** — 它能将上下文溢出（无法运行）转换为功能性 RAG（可运行）。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：通过确定性、规则驱动的 JSON Schema 压缩（TSCG conservative profile），节省 44–50% 的 schema token，从而释放上下文预算给 RAG chunk，使系统在低预算下仍能运作。
- **关键技术细节**：
  - TSCG 采用结构感知压缩：保留参数名称、类型、枚举值等关键信息，移除描述文本、冗余元数据等非必需部分（平衡配置可进一步节省至 53%）。
  - 上下文预算模型：  
    `B_RAG = B - B_sys - B_schema(f, n) - B_hist - B_out`  
    其中 `B` 为总窗口，`B_sys ≈ 350`, `B_hist = 1,500`, `B_out = 512`。  
    可容纳的 RAG chunk 数 `k = floor(B_RAG / c̄)`，`c̄` 为平均 chunk 长度（350 token）。
  - Agent 架构：ReAct 风格，最多 3 轮迭代，每轮将压缩后的模式与检索到的 chunk 塞入上下文。
- **算法流程**：固定系统提示与对话历史 → 根据格式计算 B_schema → 从 RAG 池中填充最多 k 个 chunk → 模型输出中间操作（工具调用）或最终答案 → 循环。

## 6. 论文的主要结论与发现
1. **二元启用效应（Binary Enablement）**：8K 预算下，JSON 模式溢出（100%），平均 EM 仅 2.6%；TSCG 压缩使 6/8 模型启用，平均 EM 提升 **+20.5 pp**（完全启用模型提升 +24.7 pp）。这一效应是 **类别性** 的：从无法运行变为可运行。
2. **16K 下的 null 结果**：TSCG 提供了 3 倍 chunk（9→26），但 12/14 模型 |ΔEM| ≤ 3 pp，无显著差异。说明 NovaTech-28 在约 9 chunk 时已饱和，额外 chunk 带来干扰。
3. **32K 天花板控制**：4/5 模型 Δ ≤ 1 pp，确认效应由预算驱动。唯一异常（Qwen2.5-Coder +12 pp）归因于格式翻译偏好。
4. **HotpotQA 外部验证**：Phi-4 在 8K 下 EM 提升 **+48 pp**（0%→48%）。
5. **前沿扩展**：JSON 在 ~494 工具时溢出；TSCG 在 >803 工具时尚可运行（操作范围扩展 63%）。
6. **干扰稀释效应（Distractor Dilution）**：小模型（≤8B）在获得更多 chunk 时准确率下降（如 Qwen3:4B ΔEM = -8 pp），因为注意力被无关信息稀释。

## 7. 优点：方法或实验设计上的亮点
- **首次系统性研究**：在工具-上下文竞争这个空白领域进行大规模控制实验。
- **精妙的 paired 设计**：确保所有差异归因于预算机制，而非提示变化。
- **外部验证 + 前沿扩展**：从合成基准扩展到真实数据集（HotpotQA），再到大规模工具数（800+），增强泛化性。
- **揭示机制**：通过上下文利用模型 C(k) 和干扰稀释分析，解释为什么哪怕一个 chunk 就产生巨大收益，而更多 chunk 可能有害。
- **实用洞察**：指出在充裕预算下压缩无收益，但可节省成本；在受限预算下压缩是必要基础设施。为不同部署场景提供明确指导。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制
- **基准合成性**：NovaTech-28 为构造基准，真实世界涉及更复杂交互与多轮对话。
- **仅评估一种压缩方案**：只使用了 TSCG conservative 和 balanced 两种 profile，未比较其他压缩策略（如自然语言压缩、学习型压缩）。
- **小模型覆盖不完整**：Tier C 模型仅 16K 测试，未覆盖 8K 低预算，可能错过小型模型的启用效应。
- **前沿扩展单一**：仅在 Sonnet 4 和 200K 上测试，未在其他模型/预算下验证溢出阈值。
- **静态上下文填充**：实验使用固定 chunk 填充策略，未探索动态检索（迭代检索、自适应选择）与压缩的交互效应。
- **统计偏差**：虽然使用 Wilcoxon 等，但多个假设检验未做多重比较校正（但作者声称主要结论远低于校正阈值，影响较小）。
