## 摘要
上下文或提示级别的重加权已成为基于验证奖励的强化学习（RLVR）中提升大语言模型推理能力的一个核心算法杠杆，但决定最优加权原则的原理尚不明确。我们通过将提示重加权公式化为一个在通过率函数空间中定义的效用泛函的泛函导数来解决这一空白，从而构建了一个统一的最优性框架，适用于包括REINFORCE和GRPO在内的现有方案。基于该最优性框架，我们提出了一种分布感知的提示重加权方法，称为CurveRL，其基于分位数坐标变换，其中每个提示的权重不取决于通过率的绝对值，而是取决于其在学习动态中的排名和密度，以反映通过率的分布结构。在多个基准上的大量实验表明，我们提出的CurveRL持续优于GRPO和其他RLVR基线。我们的研究将上下文分布控制识别为分析和设计提示重加权RLVR算法的一个原则性轴。代码已发布在https://github.com/zhyzmath/CurveRL。

## Abstract
Context or prompt-level reweighting has emerged as a central algorithmic lever in Reinforcement Learning with Verified Rewards (RLVR) for improving the reasoning capability of large language models, yet the principle determining what constitutes an optimal weighting remains poorly understood. We address this gap by formulating prompt reweighting as a functional derivative of a utility functional defined in the pass-rate function space, yielding a unified optimality framework that accommodates existing schemes, including REINFORCE and GRPO. Building on this optimality framework, we propose a distribution-aware prompt reweighting approach, called CurveRL, based on a quantile coordinate transform, in which the weight assigned to each prompt depends not on the absolute value of pass rates but on its rank and density to reflect the distributional structure of the pass rates in the learning dynamics. Extensive experiments across multiple benchmarks demonstrate that our proposed CurveRL consistently outperforms GRPO and other RLVR baselines. Our study identifies context-distribution control as a principled axis for analyzing and designing prompt-reweighted RLVR algorithms. The code is released in https://github.com/zhyzmath/CurveRL.

- **核心问题**：什么是最优的提示重加权原则？如何从理论上定义并推导出最优的权重函数？
- **整体含义**：该工作将提示重加权重新理解为效用依赖的上下文分布控制（context distribution control），并提出一个基于泛函导数的最优性框架，揭示了现有方法本质上是该框架下不同效用泛函的特例，并指出了点式效用函数存在的“权重坍缩”缺陷。

## 2. 论文提出的方法论

### 2.1 核心思想

- 将提示重加权视为对有效上下文分布的直接操控（Policy-Reweighted Contextual Bandit，PRCB），定义最优权重为某个效用泛函 $U_\theta$ 在通过率函数 $p_\theta(x)$ 上的泛函导数 $\delta U_\theta / \delta p_\theta(x)$。
- 指出最优权重取决于效用函数的选择（risk-dependent），没有普适最优的效用函数，但可以通过引入分布信息来系统改善点式效用函数的不足。

### 2.2 关键技术细节

- **分布感知的效用泛函**：利用参考分布 $F_{\text{ref}}$（由滞后策略的通过率估计）对通过率 $p_\theta(x)$ 进行分位数坐标变换，定义效用函数 $U_\theta(F_{\text{ref}}) = \mathbb{E}_{x \sim d_0}[\psi(F_{\text{ref}}(p_\theta(x)))]$，其中 $\psi$ 为扭曲函数（distortion function），文中取 $\psi(u)=\log u$。
- **CurveRL 算法**：
  - 在每次更新前，维护一个滑动窗口 $\mathcal{W}$（大小为 $t_0 \times |B|$），存储最近 $t_0$ 个批次中各提示的估计通过率。
  - 基于窗口内的历史通过率，用直方图方法估计参考 CDF $\hat{F}_{\text{ref}}(p)$ 和密度 $\hat{f}_{\text{ref}}(p)$。
  - 对于当前批次每个提示 $x$，采样 $N$ 个回复，估计通过率 $\hat{p}$。若 $\hat{p} \in (0,1)$，则计算权重 $w_t(\hat{p}) = \hat{f}_{\text{ref}}(\hat{p}) / \hat{F}_{\text{ref}}(\hat{p})$，并利用基线 $\hat{p}$ 进行梯度估计。
  - 梯度形式为 $\hat{g}(x) = \frac{1}{N} \sum_{i=1}^N w_t(\hat{p}) \, (r_i - \hat{p}) \, \nabla_\theta \log \pi_\theta(y_i|x)$。
- **理论优势**：
  - 分布感知、数据驱动，能够避免点式权重的“权重坍缩”问题（当 $p_\theta(x)\approx0$ 或 $1$ 时，点式权重难以区分提示潜力）。
  - 满足单调校准不变性：对通过率的单调变换不影响梯度方向。
  - 理论上可将点式权重（如 GRPO、MaxRL）视为某固定先验分布 $F_{\text{ref}}$ 的特例（Theorem 1）。

## 6. 主要结论与发现

- CurveRL 在大多数基准上同时提升了 pass@1 和 pass@64，改善了 pass@1 与 pass@k 之间的权衡（部分方法提升 pass@k 时可能损害 pass@1）。
- 在 pass@k 曲线中，CurveRL 在较大 k 下优势更明显（例如在 HMMT Feb 2026 上，1B 模型 k=1024 时比 MaxRL 高约 7.3%；4B 模型 HMMT Feb 2025 上 k=512 时高约 26.8%），表明其有效拓宽了正确推理轨迹的搜索空间。
- 机制分析显示，CurveRL 能够将更多“不可解”提示转化为“中等难度”或可解提示，从而扩大模型的实际求解边界，并在多数投票（Majority@2048）中表现更好。
- 权重动态可视化表明，CurveRL 的权重自适应于通过率分布的演变，且更激进地关注低通过率提示，相比 MaxRL 更具风险偏好性。
