## 摘要
通信是多智能体强化学习（MARL）中缓解部分可观测性的关键组成部分，然而以往的方法往往依赖于低效的信息交换，或无法传输足够的状态信息。为此，我们提出了一种基于大语言模型驱动的多智能体通信（LMAC）方法，该方法利用LLM的推理能力设计一个通信协议，使所有智能体能够尽可能准确且一致地重建底层状态。LMAC使用显式的状态感知准则迭代优化协议，在提高状态恢复能力的同时缩小智能体间知识的差异。在多种MARL基准上的实验表明，LMAC改善了智能体间的状态重建，并相较于先前的通信基线取得了显著的性能提升。

## Abstract
Communication is a key component in multi-agent reinforcement learning (MARL) for mitigating partial observability, yet prior approaches often rely on inefficient information exchange or fail to transmit sufficient state information. To address this, we propose LLM-driven Multi-Agent Communication (LMAC), which leverages an LLM's reasoning capability to design a communication protocol that enables all agents to reconstruct the underlying state as accurately and uniformly as possible. LMAC iteratively refines the protocol using an explicit state-awareness criterion, improving state recovery while narrowing differences in agents' knowledge. Experiments on diverse MARL benchmarks show that LMAC improves state reconstruction across agents and yields substantial performance gains over prior communication baselines.

## 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：在多智能体强化学习（MARL）中，部分可观测性严重制约智能体的协作性能。现有通信方法（如广播、基于注意力或价值分解的智能体间通信）存在信息交换低效或无法充分传递全局状态信息的问题，导致智能体无法准确且一致地重建底层状态，进而影响协调决策。
- **整体意义**：利用大语言模型（LLM）的推理能力，设计显式的通信协议，使所有智能体能尽可能准确且一致地恢复关键状态信息，从而提升协作效率和任务成功率。

## 2. 方法论

### 核心思想
- 基于LLM（GPT-4.1等）的链式思维（Chain-of-Thought）推理和Reflexion迭代自省机制，设计并优化一个执行式的基于代码的通信协议 `f_C`。
- 协议以智能体局部观测为输入，输出各智能体专属消息。通过两步迭代细化：**恢复增强**（Recovery Enhancement）和**失衡缓解**（Imbalance Mitigation），使状态重建更准确、知识分布更均匀。

### 关键技术细节
1. **输入提示设计**：包含任务描述（目标任务、状态与观测维度的自然语言描述）和协议设计指令（输出格式、设计原则——仅共享其他智能体重建状态所必需的信息）。
2. **离线数据集**：从QMIX训练过程中收集5k条轨迹（带状态、观测、动作）作为基准数据。
3. **辅助解码器训练**：对每个迭代协议 `f_C^{(k)}`，训练一个解码器 `D^{(k)}_φ` 以从智能体轨迹+消息中重建全局状态，用于计算状态感知指示器（SAI）：
   - 定义阈值α，比较有/无消息时的重建误差，得到指示向量 $χ^{(k)}_{i,l,d,t}$。
   - 在第一步（k=0→1）使用**恢复成功率** $E_t[χ^{(0)}_{l,d,t}]$ 引导增加缺失信息。
   - 在第二步（k=1→2）使用**智能体间知识方差** $Var_i[χ^{(1)}_{l,d,t}]$ 引导增加均匀性。
4. **反馈生成**：LLM根据上述统计指标生成结构化的JSON反馈（评估、缺失信息假设、改进建议），用于更新推理链，进而生成下一版协议。
5. **元认知表示学习框架**：将最终协议 `f_C` 集成到CTDE训练（基于QMIX，可扩展到VDN/QPLEX）：
   - 编码器-解码器对 `(Enc_ψ, Dec_ψ)` 将轨迹+消息压缩为潜在表示 `z^i_t`。
   - 联合训练重建状态和预测SAI $χ_{d,t}$（作为元认知信号）。
   - 引入**循环一致性损失**：解码后再编码得到的 `ẑ` 应与原 `z` 一致，从而抑制冗余信息。
6. **训练总体损失**：`L = L_TD + L_recon + λ_cons L_cons`，其中重建损失包含MSE和交叉熵（用于SAI预测），一致性损失为MSE。

### 算法流程（文字说明）
1. 输入任务提示x，生成初始协议 `f_C^{(0)}`（k=0）。
2. 训练辅助解码器，计算恢复成功率，生成反馈 `c^{(1)}`，并更新推理链得到 `f_C^{(1)}`（恢复增强）。
3. 训练辅助解码器，计算知识方差，生成反馈 `c^{(2)}`，更新得到 `f_C^{(2)}`（失衡缓解）。
4. 最终协议 `f_C = (f_C^{(0)}, f_C^{(1)}, f_C^{(2)})`。
5. 在在线CTDE训练中，使用编码器-解码器及联合损失进行表示学习和策略优化。

## 6. 主要结论与发现

- LMAC **在所有11个场景上均显著优于所有现有通信基线**，并且**接近或超过QMIX+State上界**（在GRF和SMACv2上甚至超越）。

- LLM生成的协议能够**准确识别任务关键信息**（如敌我位置、相对偏移、锚点坐标、可见性标志等），并逐步补充缺失信息。

- 对LLM变体、提示简化、阈值等均表现鲁棒，说明**方法核心是迭代细化流程而非特定模型**。

## 8. 不足与局限

- **额外开销**：协议优化阶段需要训练辅助解码器（约27分钟/次）和LLM调用，虽然单次成本不高，但在需要频繁重新设计协议时可能累积。

- **安全性与偏差**：未讨论LLM生成协议可能引入的偏见或安全风险（例如错误推理导致协议失效）。
