## 摘要
基于可验证奖励的强化学习（RLVR）被广泛用于提升大语言模型的推理能力，但其仅对最终答案的正确性给予奖励，而缺乏对中间步骤的监督。基于评分准则的方法，如作为奖励的评分准则（RaR），通过根据结构化标准对生成序列进行评分来引入更细粒度的监督，但这些评分仍然被聚合成一个单一的标量应用于整个响应，导致三个弱点：多准则结构的丢失、对正确与错误步骤的均一监督，以及通过无限制的自我修正导致的奖励造假。在1000个问题上，我们发现正确答案响应中有18.2%的步骤是错误的却得到了正向奖励，而错误答案响应中有49.9%的步骤是正确的却受到了惩罚。我们引入了逐步评分作为奖励（SRaR），这是一个RLVR框架，它（i）使用LLM评委将每个评分项目归因于特定的推理步骤，（ii）在多个生成序列中对每步评分进行归一化，使得只有质量有差异的步骤产生学习信号，以及（iii）通过解耦优势估计器将每步奖励与结果奖励相结合，从而保持结果基线的稳定。我们进一步构建了一个包含16K问题的评分数据集，通过对比式提炼，从强模型采样的正确和有缺陷的推理路径中提取评分项目。在六个数学推理基准测试中，SRaR相比RaR在Qwen3-8B上平均准确率提升了3.57个百分点，在Qwen3-32B上提升了2.75个百分点，将AIME 2025上的忠实推理率从34.5%提高到46.7%，并将自我修正循环率从48.1%降低到26.5%。

## Abstract
Reinforcement Learning with Verifiable Rewards (RLVR) is widely used to improve reasoning in large language models, but rewards only final-answer correctness with no supervision over intermediate steps. Rubric-based methods such as Rubrics as Rewards (RaR) introduce finer-grained supervision by scoring rollouts against structured criteria, yet the rubric scores are still aggregated into a single scalar applied to the entire response, causing three weaknesses: loss of multi-criterion structure, uniform supervision of correct and incorrect steps, and reward hacking through unbounded self-correction. On 1,000 problems, we find 18.2% of steps in correct-answer responses are wrong yet positively rewarded, while 49.9% of steps in incorrect-answer responses are correct yet penalized. We introduce Step-wise Rubrics as Rewards (SRaR), an RLVR framework that (i) uses an LLM judge to attribute each rubric item to a specific reasoning step, (ii) normalizes per-step rubric scores across rollouts so only steps whose quality varies produce a learning signal, and (iii) combines the per-step reward with the outcome reward through a decoupled advantage estimator that keeps the outcome baseline stable. We further build a 16K-problem rubric dataset by contrastively distilling rubric items from correct and flawed reasoning paths sampled from a strong model. Across six mathematical reasoning benchmarks, SRaR improves average accuracy over RaR by 3.57 points on Qwen3-8B and 2.75 points on Qwen3-32B, raises the Faithful Reasoning Rate on AIME 2025 from 34.5% to 46.7%, and reduces self-correction looping from 48.1% to 26.5%.

# 论文详细总结：Step-wise Rubric Rewards for LLM Reasoning

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有 RLVR（Reinforcement Learning with Verifiable Rewards）方法仅根据最终答案的正确性提供奖励，缺少对中间推理步骤的监督。虽然基于评分准则（Rubric）的方法（如 RaR）通过结构化的评价标准引入更细粒度的信号，但所有评分最终被聚合成一个单一的标量，应用于整个回答序列，导致三个结构性问题：
  - **多准则结构丢失**：二元评分抹去了不同准则类型的区分（如 SUGGEST、PITFALL、BONUS），且聚合后丢失了步骤归属信息。
  - **均一监督**：同一回答中正确和错误的步骤获得相同的奖励信号，导致大量步骤被错误地奖励或惩罚。诊断研究发现：在正确答案的回答中，18.2% 的步骤是错误的却获得正奖励；在错误答案的回答中，49.9% 的步骤是正确的却受到负奖励。
  - **奖励破解**：模型学会通过无限制的自我修正行为（如反复重写和验证中间步骤）来虚增 rubric 分数，导致严重的循环问题（AIME 2025 上 RaR 的循环率达 48.1%）。
- **整体含义**：当前 rubric 方法在提供细粒度监督上迈出了一步，但奖励分配方式仍不合理。本文提出 SRaR，旨在实现真正步骤级别的、可解释的、稳定的奖励信号，从而提升推理质量和训练稳定性。

## 2. 方法论：核心思想、关键技术细节

### 2.1 整体框架
SRaR 基于 GRPO（Group Relative Policy Optimization）框架，但将原有单一的响应级奖励替换为步骤级奖励，主要包含三个协同设计：

- **步骤归属的 Rubric 评判（Step-attributed Rubric Judging）**：  
  使用一个 LLM 评判器（GPT-OSS-20B）对每个 rubric 项进行二元评判（满足/不满足），并将其归因到具体的推理步骤编号。每个 rubric 项带有类型标签（SUGGEST、PITFALL、BONUS），提供不同符号和性质的奖励信号。

- **跨 Rollout 的步骤归一化与 Token 分配（Per-step Token Assignment with Cross-rollout Normalization）**：  
  每个脚步获得原始 rubric 奖励 `d_{k,i}`（由该步对应的 rubric 项累加得到）。然后在同一 prompt 的所有 rollouts 中，对每个步骤 k 的原始奖励进行跨 rollout 归一化：  
  `\bar{d}_{k,i} = (d_{k,i} - μ_k) / (σ_k + ε)`  
  这样只有质量在不同 rollout 间有差异的步骤才会产生有效学习信号。归一化后的奖励广播给该步骤的所有 token，实现步骤对齐的信用分配。

- **解耦优势估计器（Decoupled Advantage Estimator）**：  
  将最终优势拆解为两部分：  
  `\hat{A}^{(t)}_i = A_{base,i} + \tilde{r}^{(t)}_i`  
  其中 `A_{base,i}` 是基于结果正确性的 GRPO 优势（来自基组奖励 `r_{base,i}`），`\tilde{r}^{(t)}_i` 是来自 rubric 的步骤级偏移。两者分别归一化后相加，使得 rubric 噪声不会污染结果基线，防止模型通过游戏 rubric 来虚增优势，从而消除自我修正循环的动机。

### 2.2 Rubric 设计
每个问题的 Rubric 包括三种类型：
- **SUGGEST**（建议）：强化标准推理步骤，贡献正偏移。
- **PITFALL**（陷阱）：惩罚已知错误，贡献负偏移。
- **BONUS**（奖励）：奖励优秀洞察，贡献额外正偏移。
每个类型有独立的预算上限，确保其贡献可控。

### 2.3 对比式 Rubric 蒸馏
由于没有大规模公开的 rubric 数据集，作者设计了三阶段蒸馏流水线：
1. **推理路径生成**：使用 GPT-5 对每个问题采样多条逐步推理轨迹。
2. **轨迹验证**：根据正确答案将轨迹分为正确和错误两组。
3. **对比式蒸馏**：将两组路径同时输入模型，模型通过对比结构差异，提炼出具体、可验证的 rubric 项（包含 SUGGEST、PITFALL、BONUS、ANSWER 标签），每个项都基于实际观察到的失败模式。最终构建了一个包含 16K 问题的 rubric 数据集。

### 2.4 训练与推理流程
- 训练时，策略模型被要求以显式步骤格式（`### Step N:`）生成回答，以便提取 token 跨度。
- 算法流程：每个 prompt 采样 n 个 rollout，计算基组奖励、执行 LLM 评判、计算步骤级原始奖励、跨 rollout 归一化、得到 token 级优势，最后使用裁剪的 PPO 更新策略。

## 6. 论文的主要结论与发现

1. **SRaR 显著提升推理准确率**：在 Qwen3-8B 上平均准确率从 RaR 的 68.96% 提升至 72.53%（+3.57），在 Qwen3-32B 上从 73.06% 提升至 75.81%（+2.75），在所有 6 个基准上均优于所有对比方法。
2. **忠实推理率大幅提高**：AIME 2025 上，具有正确最终答案且所有中间步骤都正确的轨迹比例（FRR）从 RaR 的 34.5% 提升至 46.7%，且 SRaR 是唯一达到 A✓S×=0（即所有正确答案轨迹的步骤都正确）的模型。
3. **自我修正循环显著减少**：循环率从 RaR 的 48.1% 降至 26.5%，且循环样本的准确率(42.5%)远高于 RaR 循环样本(24.2%)，表明剩余的循环是更有效的反思。
4. **步骤级信用分配是主要增益来源**：消融实验表明，即使仅使用 SUGGEST 型 rubric，SRaR 也优于 RaR（+2.86），全类型进一步提升（+3.57）。移除步骤归一化导致训练崩溃（平均准确率降至 56.76），移除解耦优势估计则下降至 72.06。
5. **长步骤、难问题获益更多**：推理步骤数多的基准（如 AIME 2025、Olympiad Bench）改进幅度更大，与步骤数呈强相关性（r=0.91）。在 Olympiad Bench 上，最难的五分位问题改进达 +13.4 百分点。

- **问题诊断深入**：通过 1000 条样本的诊断，清晰量化了现有 rubric 方法中步骤级监督的严重误对齐（18.2% 错误步骤获正奖励、49.9% 正确步骤受惩罚），强有力地支撑了动机。
- **奖励设计新颖且完备**：
  - 步骤归属 + 类型区分 + 跨 rollout 归一化 + 解耦优势估计，形成了一个逻辑自洽的奖励分配系统。
  - 理论性质证明（Appendix O）：零和性、有界性、方差对齐、无偏性，确保了训练的稳定性。
- **对比式 Rubric 蒸馏**：摆脱了人工标注，利用强模型自动生成基于实际失败模式的 rubrics，且经过质量验证（98% 有效性），具有可扩展性。
- **实验涵盖全面**：不仅报告主结果，还包括消融、鲁棒性、数据规模、训练动力学、循环分析、难度分析等，提供了对方法的深入理解和证据链。
- **代码开源**：提供了 GitHub 仓库，便于复现和进一步研究。

## 8. 不足与局限

- **对 LLM 评判器的依赖**：SRaR 需要调用外部 LLM 来评判 rubric 满足情况和归因，引入额外计算开销和潜在噪声。尽管跨 rollout 归一化缓解了系统偏差，但单个样本的评判错误仍可能影响训练。
- **额外推理成本**：每个训练迭代需要对每个 rollout 进行 LLM 评判，计算成本随 rollout 数和 rubric 项数线性增长，相比标准 GRPO 或轨迹级 rubric 方法更高。
- **领域局限**：实验仅限于数学推理，虽然方法原理上可迁移到其他多步推理任务（如代码生成、科学推理），但尚未验证。
- **Rubric 质量受限于强模型**：蒸馏流水线依赖 GPT-5 生成路径和 rubrics，在只有较弱模型可用时，效果可能下降。
- **需要显式步骤格式**：SRaR 假定策略模型生成带步骤标题（`### Step N`）的回答，这需要在训练时加入格式约束，可能不适用于无结构推理。不过实验表明格式约束本身几乎不影响性能（DAPO 有无格式结果相近）。
- **未报告训练总时长与能耗**：虽然说明了使用 8 × H200，但未给出具体的训练 wall-clock 时间，不利于评估实际计算开销。
