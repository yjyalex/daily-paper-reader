## 摘要
过程奖励模型（PRMs）为推理提供步骤级反馈，但当前的PRMs通常仅为每个步骤输出一个奖励分数。因此，下游方法必须将不完美的步骤级奖励预测视为可靠的决策信号，而无法得知何时应信任这些预测。我们提出了BetaPRM，一种分布式的PRM，它同时预测步骤级的成功概率和该预测的可靠性。在来自蒙特卡洛延续的步骤成功监督下，BetaPRM学习一个Beta信念，通过Beta-二项式似然解释观察到的成功延续次数，而不是将有限样本的成功比率作为点目标进行回归。这种学习到的可靠性信号指示何时应信任步骤奖励，使下游应用能够区分可靠奖励和不确定奖励。作为一项应用，我们引入了用于PRM导向的Best-of-N推理的自适应计算分配（ACA）。ACA利用学习到的可靠性信号，在可靠的高奖励解决方案出现时停止，并对不确定的候选前缀投入额外计算。在四个骨干网络和四个推理基准上的实验表明，BetaPRM在保留标准步骤级错误检测的同时，改进了PRM导向的Best-of-N选择。基于这一信号，ACA在固定预算的Best-of-16上改善了准确率-令牌权衡，在提高最终答案准确率的同时，减少了高达33.57%的令牌使用。

## Abstract
Process Reward Models (PRMs) provide step-level feedback for reasoning, but current PRMs usually output only a single reward score for each step. Downstream methods must therefore treat imperfect step-level reward predictions as reliable decision signals, with no indication of when these predictions should be trusted. We propose BetaPRM, a distributional PRM that predicts both a step-level success probability and the reliability of that prediction. Given step-success supervision from Monte Carlo continuations, BetaPRM learns a Beta belief that explains the observed number of successful continuations through a Beta-Binomial likelihood, rather than regressing to the finite-sample success ratio as a point target. This learned reliability signal indicates when a step reward should be trusted, enabling downstream applications to distinguish reliable rewards from uncertain ones. As one application, we introduce Adaptive Computation Allocation (ACA) for PRM-guided Best-of-N reasoning. ACA uses the learned reliability signal to stop when a high-reward solution is reliable and to spend additional computation on uncertain candidate prefixes. Experiments across four backbones and four reasoning benchmarks show that BetaPRM improves PRM-guided Best-of-N selection while preserving standard step-level error detection. Built on this signal, ACA improves the accuracy--token tradeoff over fixed-budget Best-of-16, reducing token usage by up to 33.57% while improving final-answer accuracy.

# 基于学习可靠性的过程奖励（BetaPRM）论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **现有过程奖励模型（PRM）的缺陷**：当前 PRM 通常仅输出一个标量步骤奖励分数（如步骤正确概率），不提供该预测的置信度或可靠性。下游方法（如 Best-of-N 选择、策略优化）不得不将不完美的标量分数视为可靠决策信号，无法区分哪些预测可信、哪些充满不确定性。
- **训练数据噪声问题**：常用的 PRM 监督信号来自蒙特卡洛（MC）延续：从某个前缀采样 N 条完整轨迹，统计成功数 K。标准 PRM 训练直接回归经验比率 K/N 作为点标签，忽略了 K/N 是有限样本估计，同一前缀重复采样可能得到不同比率，导致模型拟合噪声。
- **本文对应的解决思路**：希望 PRM 不仅输出奖励均值，还输出对该预测的可靠性估计（即不确定性）。这样下游算法能区分可靠与不可靠的步骤奖励，从而更智能地分配计算资源。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想：BetaPRM
- 将每个推理前缀的成功概率建模为一个 **Beta 分布**（Beta belief），其参数由模型预测。
- 输出两个量：
  - **均值 μₜ**：前缀成功概率的期望，即传统 PRM 的标量奖励。
  - **集中度 κₜ**（concentration）：控制 Beta 分布的尖锐程度。κₜ 越大，分布越集中（预测越可靠）；κₜ 越小，分布越平坦（预测越不确定）。
- 通过 **Beta-Binomial 似然** 来训练：模型输出的 Beta 分布应能解释观测到的计数（K 次成功、N 次尝试），而非直接逼近 K/N。

### 2.2 关键技术细节
#### （1）模型参数化
- 对每个 `<prm>` 标记位置，语言模型输出隐藏状态 hₜ 和词汇 logits zₜ。
- **奖励均值 μₜ**：通过 Yes/No 两个 token 的 softmax 得到：  
  μₜ = exp(z_Yes) / (exp(z_Yes) + exp(z_No))
- **集中度 κₜ**：通过一个轻量线性头 g_ϕ(hₜ) 经 softplus 激活后加上最小值 κ_min 得到：  
  κₜ = softplus(g_ϕ(hₜ)) + κ_min
- 然后计算 Beta 参数：αₜ = μₜ κₜ，βₜ = (1-μₜ) κₜ。

#### （2）Beta-Binomial 训练目标
- 观测到 Kₜ 次成功、N 次尝试，Beta-Binomial 的负对数似然损失为：
  ```
  L_Beta-Binomial = - (1/|P|) Σ_t log p(Kₜ | N, αₜ, βₜ)
  ```
  其中 p(·) 为 Beta-Binomial 概率质量函数。
- 该损失鼓励模型在预测的 Beta 分布下，赋予观测计数高概率。若分布集中且与观测一致则低损失；若分布集中但与观测偏移则高损失；若分布平坦则损失适中。

#### （3）可靠性校准正则项
- 增加辅助正则项，惩罚“高集中度与预测均值 μₜ 偏离观测比率 Kₜ/N”的情况：
  ```
  L_reg = λ_reg * (1/|P|) Σ_t |sg(μₜ) - Kₜ/N| * κₜ
  ```
  其中 sg 表示 stop-gradient，防止 μₜ 被拉向 K/N（避免退化回点回归），只调节 κₜ。
- 总损失：L = L_Beta-Binomial + L_reg。

#### （4）自适应计算分配（ACA）
- 利用 BetaPRM 提供的可靠性信号（标准差 σₜ = √[μₜ(1-μₜ)/(κₜ+1)]），设计了 ACA 算法用于 PRM 引导的 Best-of-N 推理。
- **风险调整候选得分**：S(y) = (1/T) Σ_t (μₜ - λ σₜ)，其中 λ 控制不确定性惩罚。
- **渐进式批量生成与早停**：
  - 先生成 n₀ 个完整候选，计算每个候选的 S(y) 及置信区间（LCB/UCB）。
  - 若当前最佳候选的 LCB 大于所有其他候选的 UCB，则认为结果可靠，停止计算。
  - 否则，继续生成下一批（最多 N 个），并选择最可能改变结果的非获胜候选（最高 UCB）进行修复。
- **不确定性引导的前缀修复**：对选中的候选，找到最早不满足低质量阈值（μₜ - c_cut σₜ < p_bad）的步骤作为切割点，保留前缀，重新采样后续步骤。

## 6. 论文的主要结论与发现
- **BetaPRM 提升 Best-of-N 选择质量**：在全部 4 个 backbone × 4 个 benchmark 上，BetaPRM 优于 Standard PRM，平均提升 1.29~3.37 个百分点。

- **集中度 κ 的演化规律**：训练初期 κ 下降（模型保持保守），后期逐渐上升并出现上尾（模型对可靠预测赋予高置信度），说明模型学到了自适应的置信水平。

## 7. 优点：方法或实验设计上的亮点
- **方法创新**：
  - 将 PRM 输出从点估计扩展为分布估计，同时提供奖励均值和可靠性，填补了现有 PRM 缺乏不确定性表征的空白。
  - 使用 Beta-Binomial 似然而非点回归，自然处理了 MC 监督的计数性质，避免了拟合噪声。
  - 辅助正则项有效校准了集中度，且 stop-gradient 设计避免了退化。
  - ACA 利用可靠性信号实现早停和定向修复，实现了智能计算分配。
- **实验设计**：
  - 对比了多个骨干网络（7B~14B），验证方法通用性。
  - 消融实验分离了 Beta-Binomial 损失、正则项、早停、不确定性来源等贡献，结论清晰。
  - ACA 的 token 节省在四个数学推理基准上一致，且准确率不降反升，强有力。
  - 公平性：所有选择器使用同一候选池，训练与评估数据分离。

## 8. 不足与局限
- **数据依赖**：BetaPRM 需要保留 MC 计数（K, N）的监督数据。目前仅 VisualPRM400K-v1.1 满足要求，限制了实验范围到多模态数学推理。未来需在其他领域（如纯文本数学）验证。
- **实验覆盖**：主要评估在数学推理任务上，未涉及代码生成、网页代理等更广泛的应用场景。
- **可靠性信号的性质**：学习到的可靠性是模型内部信念，并非数学保证的高斯校准误差。在高风险应用中仍需人工监督和额外校准。
- **ACA 的参数敏感度**：超参数（λ, c_stop, p_bad 等）在当前网格上表现良好，但论文未讨论其对不同场景的鲁棒性。
- **计算开销**：训练时需额外预测 κ（加一个线性头），推理时需额外计算 σₜ，但开销很小；ACA 因减少采样反而降低总计算。
- **与自一致性等其他方法的对比**：论文主要对比了固定预算 Best-of-N，未与 Beam Search、树搜索等更复杂的 test-time scaling 方法比较。不过 ACA 作为框架可与这些方法结合。
