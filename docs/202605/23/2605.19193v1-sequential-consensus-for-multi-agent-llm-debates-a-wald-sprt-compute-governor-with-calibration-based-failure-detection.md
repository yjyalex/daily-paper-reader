## 摘要
多智能体LLM辩论可提升事实准确性与推理能力，但多数方法采用固定轮次，导致简单问题计算浪费而复杂问题计算不足。我们将Wald的序贯概率比检验（SPRT）适配为LLM辩论的插件式计算调控器。每轮辩论后，LLM评判者对最新智能体立场输出[0,1]共识分数；Wald监测器在Beta似然族假设下累积“有效收敛”与“尚未有效”的对数似然比，并在任一边界被跨越时停止，或在达到最大轮次R_max时返回有上限的最佳结果。在独立同分布假设下，该规则继承SPRT的第一类/第二类错误保证；实际部署中，校准本身更为重要，因为它能估计在特定领域中评判者分数是否真正区分了有效收敛与无益收敛。我们评估两条路径：(i) 在标定Beta模型下进行蒙特卡洛研究，刻画工作曲线、错误率、截断行为及敏感性；(ii) 在真实LLM评估中，使用三个异构智能体（gpt-5、claude-opus-4-6、gemini-2.5-pro）和claude-opus-4-6评判者，对200个MMLU题目和200个GSM8K题目进行测试，并采用不相交的40题校准子集。在GSM8K上，该规则平均1.01轮（4.06次LLM调用）停止，准确率97.0%，而固定5轮辩论需15次调用准确率99.0%：调用减少3.7倍，准确率下降2个百分点。在MMLU上，标定KL散度接近0，规则在99.5%的题目上以2.1倍成本截断。关键结论并非SPRT使辩论更准确，而是经典序贯检验可作为多智能体LLM系统中低成本的计算控制与故障检测层。

## Abstract
Multi-agent LLM debate improves factuality and reasoning, but most recipes pick a fixed round count, over-spending on easy items and under-spending on hard ones. We adapt Wald's Sequential Probability Ratio Test (SPRT) as a plug-in compute governor for LLM debates. After each round, an LLM judge emits a [0,1] consensus score on the latest agent positions; a Wald monitor accumulates the log-likelihood ratio of "useful convergence" vs "not yet useful" under a Beta likelihood family, and stops when either boundary is crossed or returns a capped best-effort outcome at R_max. Under i.i.d. assumptions the rule inherits SPRT type-I/type-II error guarantees; in deployment the calibration itself is the more important object, since it estimates whether the judge score actually separates useful from unhelpful convergence in a given domain. We evaluate two tracks: (i) a Monte-Carlo study under calibrated Beta models characterising working curves, error rates, capping behaviour, and sensitivity; and (ii) a real-LLM evaluation on 200 attempted MMLU and 200 attempted GSM8K items with three heterogeneous agents (gpt-5, claude-opus-4-6, gemini-2.5-pro) and a claude-opus-4-6 judge, using disjoint 40-item calibration subsets. On GSM8K the rule stops in 1.01 average rounds (4.06 LLM calls) at 97.0% accuracy vs 99.0% for fixed-5 debate at 15 calls: a 3.7x call reduction at -2pp accuracy. On MMLU the calibrated KL collapses to about 0 and the rule caps on 99.5% of items at 2.1x cost. The takeaway is not that SPRT makes debate more accurate, but that a classical sequential test serves as a cheap compute-control and failure-detection layer for multi-agent LLM systems.

- **整体含义**：不是提出新的辩论方法，而是为现有辩论协议添加一个轻量级计算控制器和校准诊断层。

## 2. 论文提出的方法论
- **核心思想**：将Wald序贯概率比检验（SPRT）应用于多智能体辩论的轮次控制。每轮结束后，LLM裁判输出一个[0,1]共识分数 \(s_r\)，假设在两种假设（H0：未有效收敛；H1：有效收敛）下 \(s_r\) 服从不同的Beta分布，积累对数似然比 \(\Lambda_R = \sum_{r=1}^R \log[f_1(s_r)/f_0(s_r)]\)，当 \(\Lambda_R \ge A\) 时接受H1（停止并输出答案），当 \(\Lambda_R \le B\) 时接受H0（停止并标记为无共识），否则继续下一轮。设置最大轮次 \(R_{\max}\) 避免无限运行。
- **关键技术细节**：
  - 使用Beta分布建模共识分数（[0,1]区间，自然选择）。
  - 校准阶段：在小规模领域内辩论数据集（约40~50个）上，通过矩估计拟合H0和H1下的Beta参数。校准基于正确性条件：H1对应最终答案正确的序列，H0对应错误的序列（实际部署中可用验证集）。
  - 阈值：\(A = \log((1-\beta)/\alpha)\)，\(B = \log(\beta/(1-\alpha))\)，其中\(\alpha,\beta\)为设定的第一类/第二类错误率（默认0.05）。
- **算法流程**（文字描述）：
  1. 输入问题、K个智能体、Beta似然模型、误差参数、最大轮次。
  2. 初始化各智能体初始立场（第一轮独立回答）。
  3. 对每轮 r=1..R_max：
     - 运行辩论一轮（智能体交换观点并更新立场）。
     - LLM裁判生成共识分数 \(s_r\)（裁剪至[ε,1-ε]避免数值问题）。
     - 更新累积对数似然比 \(\Lambda_r\)。
     - 若 \(\Lambda_r \ge A\) → 输出当前聚合答案，标记“共识”，停止。
     - 若 \(\Lambda_r \le B\) → 输出当前聚合答案，标记“无共识”，停止。
     - 否则继续下一轮。
  4. 到达R_max → 输出聚合答案，标记“截断”。

## 7. 优点
- **简洁插件式**：算法极小（第4节），仅需每轮一次O(1)检查，可无缝接入任何辩论协议。
- **理论保证**：基于Wald经典SPRT，在独立同分布假设下提供第一类/第二类错误有界保证。
- **校准诊断突出**：通过校准阶段检测裁判信号质量，提供实际部署的预警机制，这是最关键的工程贡献。
- **成本效益**：在裁判信号有信息时大幅节省LLM调用；无信息时自动消耗更多计算并发出警告。
- **可解释性强**：提供“共识”、“无共识”、“截断”三种决策类型，便于事后分析和路由。
- **鲁棒性验证**：对参数误差、非独立性、不同难度均有敏感性分析。

## 8. 不足与局限
- **实验覆盖不足**：
  - 真实LLM评估仅限于MMLU（多选）和GSM8K（算术），缺少对数对比类（如JudgeBench）和长文本生成等更复杂任务。
  - 模型种类有限（仅gpt-5, claude-opus-4-6, gemini-2.5-pro），裁判和智能体存在重叠（claude既是智能体又是裁判），缺乏跨骨干网系统研究。
  - 样本量（200）导致统计功效低，无法可靠检测小差异。
- **假设偏离**：真实辩论序列存在时间依赖性（非i.i.d.），违反SPRT核心假设，会导致错误率轻微膨胀（论文通过模拟量化）。
- **对抗性脆弱**：若智能体针对裁判分数优化，可能产生表面一致但错误的答案，SPRT会提前停止。论文未测试对抗场景。
- **静态似然**：Beta参数固定，不随辩论过程自适应更新，可能错过继续优化的机会。
- **天花板效应**：当前强模型在基准上已饱和，无法有效对比协议间的准确率提升空间。
- **应用限制**：要求裁判具备良好的信心校准（KL>0），否则SPRT会退化为固定最大轮次；若裁判信号不可靠，需人工介入重新设计裁判。
