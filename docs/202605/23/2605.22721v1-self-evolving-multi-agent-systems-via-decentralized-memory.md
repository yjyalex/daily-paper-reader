## 摘要
自我进化的多智能体系统（MAS）已成为一种有前景的途径，使LLM智能体能够不断从经验中改进，其基础是持久记忆。然而，现有设计几乎完全采用跨智能体共享的集中式存储库，这会带来通信和协调开销，引发隐私问题，并削弱智能体多样性。我们提出DecentMem，一种去中心化记忆框架，其中每个智能体维护自己的双池记忆——一个利用池，包含整合的过去轨迹；一个探索池，包含针对未见上下文的LLM生成候选。这两个池基于来自LLM作为评判者的阶段性反馈进行在线重新加权。理论上，我们证明该设计保证了解决方案空间的全局可达性，并实现了$O(\log T)$的累积遗憾，与随机臂下界在常数因子内匹配。在实践中，跨三个MAS框架（AutoGen、DyLAN、AgentNet）、三个Qwen3骨干网络（4B/8B/14B）、两个Gemma4骨干网络（E2B/E4B）以及涵盖数学、代码、问答和具身任务的五个基准测试，DecentMem相比最强的集中式记忆基线平均准确率提升高达23.8%，相比无记忆基线提升高达52.5%，同时令牌用量减少高达49%。

## Abstract
Self-evolving multi-agent systems (MAS) have emerged as a promising route to LLM agents that continually improve from experience, with persistent memory at their foundation. However, existing designs almost exclusively adopt a centralized repository shared across agents, incurring communication and coordination overhead, raising privacy concerns, and collapsing agent diversity. We propose DecentMem, a decentralized memory framework in which each agent maintains its own dual-pool memory -- an exploitation pool of consolidated past trajectories and an exploration pool of LLM-generated candidates for unseen contexts. The two pools are reweighted online based on stage-wise feedback from an LLM-as-a-judge. Theoretically, we prove that this design guarantees global reachability of the solution space and achieves $O(\log T)$ cumulative regret, matching the stochastic bandit lower bound up to constants. In practice, across three MAS frameworks (AutoGen, DyLAN, AgentNet), three Qwen3 backbones (4B/8B/14B), two Gemma4 backbones (E2B/E4B) and five benchmarks spanning math, code, QA, and embodied tasks, DecentMem improves average accuracy by up to 23.8% over the strongest centralized memory baseline and by up to 52.5% over the no-memory baseline, while reducing token usage by up to 49%.

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- **去中心化双池记忆**：每个智能体 $a_m$ 拥有两个私有池：
  - **利用池（E-pool）**：存储过去成功轨迹，通过相似性检索驱动局部行走，复用历史策略。
  - **探索池（X-pool）**：由LLM为未见上下文生成候选，实现类似“启发式传送”的全局探索。
- **在线路由**：基于**LLM-as-a-Judge**的阶段式反馈，动态调整两个池的选择权重 $w_{m,\text{E-pool}}$ 和 $w_{m,\text{X-pool}}$。
- **框架无关性**：可嵌入AutoGen、DyLAN、AgentNet等任意MAS框架。

### 关键技术细节

1. **记忆表示**：每个记忆片段 $z = (\xi, r^*)$，$r^*$ 包含动作轨迹和智能体评论。
2. **在线路由概率**：  
   $\alpha_{m,\text{E-pool}} = \frac{w_{m,\text{E-pool}}}{w_{m,\text{E-pool}} + w_{m,\text{X-pool}}}$，$\alpha_{m,\text{X-pool}} = 1 - \alpha_{m,\text{E-pool}}$。
3. **检索与生成**：
   - 若选择E-pool：检索相似度高于阈值 $\tau$ 的记忆片段。
   - 若选择X-pool：LLM直接生成新颖候选。
4. **更新规则**：根据阶段评分变化 $\Delta_t$ 调整 $w_{m,\text{E-pool}}$：
   - 利用成功：$w_{m,\text{E-pool}} \leftarrow w_{m,\text{E-pool}} + \alpha$。
   - 利用失败：$w_{m,\text{E-pool}} \leftarrow \max(1.0, \beta w_{m,\text{E-pool}})$。
   - 探索时规则反之。
   - 任务完成后，X-pool内容合并到E-pool，X-pool清空。

### 理论保证
- **全局可达性**：在搜索空间图上，混合转移矩阵 $M_{m,\ell} = \alpha_{m,\ell} T_m + (1-\alpha_{m,\ell}) h_m \mathbf{1}^\top$ 严格正定，马尔可夫链不可约且非周期，保证任何状态可达。
- **对数遗憾**：将路由建模为随机臂问题，在严格凹性假设下，累计遗憾 $\mathbb{E}[R(T)] = O(\log T)$，匹配下界。
