## 摘要
基于熵的深度推理已成为提升大语言模型（LLM）推理能力的一个有前景的方向，但现有方法往往要么不加区分地增加响应长度，要么以牺牲准确率为代价缩短响应。为了更好平衡这一权衡，我们引入了条件熵塑造（CES），一种动态控制词级响应熵的框架，使LLM能在简单问题上生成简洁解，同时在难题上鼓励更深入探索。CES基于DAPO，将词级熵作为不确定性信号，并应用条件双向策略：在正确推理路径上惩罚高熵的“分叉点”词元以提高简洁性，在错误路径上奖励它们以鼓励探索和纠错。我们在DeepSeek-R1-Distill-7B上实现CES，并在12个数学基准上评估。与DAPO相比，CES在提高平均准确率的同时缩短了响应长度，补充实验在较小的1.5B骨干网络和域外基准上显示出类似趋势。

## Abstract
Entropy-based deep reasoning has emerged as a promising direction for improving the reasoning capabilities of Large Language Models (LLMs), but existing methods often either increase response length indiscriminately or shorten responses at the cost of accuracy. To better balance this trade-off, we introduce Conditional Entropy Shaping (CES), a framework that dynamically controls token-level response entropy, enabling LLMs to produce concise solutions on simple problems while encouraging deeper exploration on hard ones. Built on DAPO, CES uses token-level entropy as an uncertainty signal and applies a conditional bidirectional policy: it penalizes high-entropy "forking point" tokens on correct reasoning paths to improve conciseness, and rewards them on incorrect paths to encourage exploration and error correction. We implement CES on DeepSeek-R1-Distill-7B and evaluate it on 12 mathematical benchmarks. CES consistently improves average accuracy while reducing response length relative to DAPO, and supplementary experiments show similar trends on a smaller 1.5B backbone and on out-of-domain benchmarks.

### 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大语言模型（LLM）在处理复杂推理任务时，通过链式思维（CoT）显式生成推理步骤，取得了高性能，但也带来了严重的“过度思考”现象——对简单问题也生成冗长的推理路径，导致计算效率低下（高延迟、高成本）。现有方法在提升准确性和提升效率之间存在难以调和的矛盾：提升准确性的方法（如鼓励高熵探索）往往进一步拉长响应，而提升效率的方法（如强制约束长度）又可能损害复杂任务的准确性。
- **整体含义**：本文旨在解决这一权衡，提出一种能根据问题难度动态调整推理深度的框架——在简单问题上保持简洁，在困难问题上进行深度探索，从而在提升准确性的同时降低响应长度。

### 2. 方法论：核心思想、关键技术细节

- **核心思想**：基于**条件熵塑造（CES）**，利用**token级别的熵**作为不确定性信号，对推理路径的正确性进行条件性双向调整：
  - 对于**正确**的推理路径，对高熵的分叉点（forking point）token施加**惩罚**，抑制不必要的探索，鼓励简洁直接的推理；
  - 对于**错误**的推理路径，对高熵token施加**奖励**，鼓励模型探索替代路径，纠正错误。
- **关键技术细节**：
  1. **基础框架**：基于DAPO（Decoupled Clip and Dynamic Sampling Policy Optimization）强化学习算法，CES在其基础上引入token-level advantage shaping。
  2. **熵计算**：对每个token，计算其在预测分布上的熵 H(t_j | y_{i,<j})。
  3. **动态选择高熵token**：每个响应中选取 k_i 个最高熵token。k_i 由响应长度、基础选择率 τ 和动态乘子 b_i 决定：
     - b_i = a（组准确率）若响应正确；b_i = 1−a 若响应错误。
     - 这样，在正确响应上，若组准确率高（问题简单），则惩罚力度更大；在错误响应上，若组准确率低（问题困难），则奖励力度更大。
  4. **优势塑造**：对每个选中的高熵token，修改其优势值：
     - 正确路径：A’ = A − β₁·H（惩罚）
     - 错误路径：A’ = A + β₂·H（奖励）
     - 其他token维持原有组标准化优势 A。
  5. **梯度保留**：熵项 H 参与计算图，使模型能通过梯度下降直接学习降低/升高熵，实现双向控制。
- **算法流程**（文字说明）：
  1. 对给定问题生成 N 个响应；
  2. 计算每个响应的奖励（正确性 + 格式）及组准确率 a；
  3. 对每个响应，标准化优势；
  4. 根据正确性设置动态乘子 b，计算 k_i；
  5. 选出每个响应中 entropy 最高的 k_i 个 token；
  6. 对这些 token 根据正确性施加惩罚或奖励，得到最终 token-level 优势；
  7. 将重塑后的优势代入 DAPO 损失函数进行梯度更新。

### 6. 主要结论与发现

- **准确性-效率双赢**：CES 在12个数学基准上，平均准确率提升 +2.5%（从69.6%到72.1%），平均响应长度缩短 411 tokens（从2376到1965）。相比无条件熵优势方法，CES在准确率上显著更好且更短。
- **自适应分配推理预算**：CES 在困难数据集（如AIME24）上大幅提升准确率并缩短长度；在简单数据集（如CMATH）上，对困难问题增加推理长度换取准确性提升，对简单问题则保持简洁——说明它学会了按需分配计算资源，而非一味缩短。
- **关键组件验证**：
  - 动态 token 选择（基于组准确率）对避免过度惩罚正确但脆弱的解至关重要；
  - 熵梯度保留是双向控制的核心，否则模型无法学会降低熵。
- **训练动力学**：在约1000训练步之后，CES模型的长度和熵开始显著下降，而准确率保持与基线相当，表明CES在不牺牲性能的前提下实现了效率提升。

### 7. 优点：方法或实验设计的亮点

- **细粒度条件控制**：从序列级 advantage 深入到 token-level，并且根据正确性和组难度动态调整，实现精准的探索-利用平衡。
- **训练效率高**：额外计算开销极小（+0.7% 时间），且无需额外模型前向/后向。
- **强鲁棒性**：在多个超参数配置（4/5组）下同时优于基线；跨模型规模（1.5B）和跨领域（通用推理、代码）均能迁移。
- **解释性强**：通过训练动力学分析、简单/困难问题分层分析，揭示了模型如何自适应分配计算预算。
- **算法简洁**：基于成熟的 DAPO 框架，只修改 advantage 计算部分，易于实现和集成。

### 8. 不足与局限

- **依赖可验证正确性**：方法需要明确的正确/错误信号（如数学答案检查），对于开放生成、主观或弱可验证的任务（如创意写作、对话）难以直接应用。
- **超参数敏感性存在**：尽管在合理范围内鲁棒，但最优的 τ 和 β 仍可能需要针对不同骨干或任务分布进行微调。
- **训练规模限制**：仅使用 2500 个训练样本（来自 DeepMath），可能无法充分释放 CES 的潜力，且未使用复杂数据策略（如拒绝采样、课程学习），导致准确率在训练过程中波动无明显上升。
- **领域局限性**：训练数据仅来自数学推理，虽验证了代码和通用推理的泛化，但领域跨度有限，未来需在更多样化的任务（如科学推理、规划）上测试。
- **缺少与其他效率导向方法的直接对比**：基线中仅与无条件熵优势方法比较，未与 Ada-CoT、Ada-R1、GRPO-LEAD 等近期长度控制方法对比，尽管这些方法可能侧重不同（如长度修剪或混合策略）。
- **评估设置**：非贪婪解码（温度0.4）导致一定随机性，虽取了4次平均，但可能不如贪婪解码稳定。
