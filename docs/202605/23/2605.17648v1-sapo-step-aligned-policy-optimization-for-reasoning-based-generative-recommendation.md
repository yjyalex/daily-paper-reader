## 摘要
生成式推荐将下一个项目预测视为自回归的项目标识符生成。具体来说，项目被编码为语义标识符（SIDs），这些是短粗到细的标记序列，其早期标记捕获广泛语义，后续标记进行细化。最近的工作通过推理轨迹增强了这一范式，并通过带有可验证奖励的强化学习进行优化，通常是对生成的SID使用精确匹配反馈的结果奖励算法。然而，在大规模目录推荐中，对生成的SID的精确匹配反馈仅报告最终项目是否正确；当生成的SID不匹配时，结果奖励无法识别哪个SID标记预测导致了不匹配，并可能将匹配的SID标记位置与不匹配的位置一起惩罚。我们发现，在这种设置中，信用分配的自然单位是单个推理步骤（一个思考块与一个SID标记配对）。我们在SAPO（步对齐策略优化）中实例化了这一想法：SAPO不为整个响应广播一个优势，而是为每个推理步骤计算一个单独的组相对优势，并将其仅应用于相应的思考块和SID标记。在三个真实世界推荐数据集上，SAPO稳定了强化学习训练，并持续改进现有的生成式推荐基线，在稀疏精确匹配反馈使得推理步骤信用分配重要的地方获得最大增益。我们的结果表明，结构化生成的强化学习目标应反映解码器自身的输出分解。

## Abstract
Generative recommendation treats next-item prediction as autoregressive item-identifier generation. Specifically, items are encoded as semantic identifiers (SIDs), which are short coarse-to-fine token sequences whose early tokens capture broad semantics and later tokens refine them. Recent work augments this paradigm with reasoning traces and optimizes them via reinforcement learning with verifiable rewards, typically outcome-reward algorithm with exact-match feedback on the generated SID. However, in large-catalog recommendation, exact-match feedback on the generated SID only reports whether the final item is correct; when a generated SID mismatches, outcome-reward cannot identify which SID-token prediction caused the mismatch and may penalize matched SID-token positions together with the mismatched position. We identify that the natural unit of credit assignment in this setting is a single reasoning step (one thinking block paired with one SID token). We instantiate this idea in SAPO (Step-Aligned Policy Optimization): rather than broadcasting one advantage to the whole response, SAPO computes a separate group-relative advantage for each reasoning step and applies it only to the corresponding thinking block and SID token. Across three real-world recommendation datasets, SAPO stabilizes reinforcement-learning training and consistently improves over existing generative recommendation baselines, with the largest gains where sparse exact-match feedback makes reasoning-step credit assignment important. Our results suggest that reinforcement-learning objectives for structured generation should mirror the decoder's own decomposition of the output.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：生成式推荐（Generative Recommendation）将下一个物品预测建模为自回归的语义标识符（Semantic Identifier, SID）生成。SID 是由粗到细的短 token 序列（如 3 级码本），早期 token 捕获广义语义，后期 token 细化具体物品。近期工作引入了推理轨迹（reasoning traces）并用强化学习（RL）优化，通常采用基于精确匹配（exact-match）的结果奖励（outcome reward）进行信用分配。
- **核心问题**：精确匹配反馈仅告知最终 SID 元组是否正确，当生成错误时，无法定位是哪一个 SID token 预测出错。现有方法（如 GRPO）将单一标量优势广播到整个响应中的所有 token，导致两种失败模式：① 匹配的 SID token 位置与不匹配位置一起被惩罚；② 不匹配 token 的错误信号被扩散到无关思考块，造成训练不稳定、长度漂移和 KL 发散。这种“动作粒度不匹配”（action-granularity mismatch）是主要研究动机。
- **整体含义**：论文提出，在结构化生成中，信用分配的自然单位是单个推理步骤（一个思考块 + 一个 SID token），而非整个序列或单个 token。以此为出发点，设计了 SAPO 方法，以提高稀疏反馈下的训练稳定性和推荐质量。

## 2. 方法论

**核心思想**：将每个推理步骤 (`τ^(k)`, `s^(k)`) 作为 RL 中的行动单元，为每一步计算独立的组相对优势，并仅应用于对应的思考块和 SID token，实现步对齐的策略优化。

**关键技术细节**：
- **推理步骤分解**：模型首先生成 K 个思考块（thinking blocks）及其对应的 SID token（格式为 `<think>τ^(1)</think> ... <think>τ^(K)</think> s^(1)...s^(K)`）。每一步 `(τ^(k), s^(k))` 被视作一个推理步骤。
- **步匹配奖励**：为每个步骤分配奖励 `r_{i,k} = α·𝟙[s_i^(k)=s_gt^(k)] + β·𝟙[k=K]·b_i`。第一部分是 SID token 级别的匹配奖励，提供密集监督；第二部分是最后一个步骤的格式奖励（当响应结构正确时）。
- **步骤级优势**：在每个推理步骤 k 上，基于组内该步骤的奖励计算组相对优势 `A_{i,k} = (r_{i,k} - mean_i')/std_i'`，然后除以步骤长度得到归一化优势 `\tilde{A}_{i,k} = A_{i,k}/|y_i^(k)|`。
- **步对齐策略代理损失**：优化目标为 JSAPO（公式 9），其中每个 token 的损失使用对应步骤的归一化优势和重要性比率，通过步骤归一化的 token 聚合，确保更新幅度取决于步骤质量而非 token 数量。
- **梯度分析**（公式 10）表明，SAPO 替换了全局标量优势，仅对引发差异的步骤产生对比更新，并且通过长度归一化避免了冗长思考块的过度影响。

**算法流程**（文字描述）：
1. 对每个 prompt，采样 G 个 rollout，每个 rollout 包含 K 个思考块和 K 个 SID token。
2. 对于每个 rollout i 和每个步骤 k，计算匹配奖励（基于 SID token 正确性）和格式奖励。
3. 在步骤 k 上，对组内 G 个 rollouts 的奖励进行标准化，得到步级别优势 A_{i,k}。
4. 将 A_{i,k} 除以该步骤包含的 token 总数（思考块长度+1），得到归一化优势。
5. 优化策略参数 θ，使用 clipped policy gradient，其中每个 token 的梯度权重为重要性比率 × 归一化优势 × (1/|y_i^(k)|)。

## 6. 主要结论与发现

- SAPO 在 NDCG 指标上在所有数据集上取得最优，Recall 指标上排名第一或第二，显著优于传统方法和无推理生成式基线。
- 与 SIDReasoner（使用形状奖励的 GRPO）相比，SAPO 在 NDCG 上有系统性提升，表明推理激活本身不足，还需步对齐的信用分配。
- 训练动态显示 SAPO 的 reward 轨迹更平稳，响应长度和 KL 散度控制更好，SID 匹配率更高，而 GRPO 出现不稳定、长度漂移和 KL 快速增长。
- 消融实验表明，步骤级优势（advantage localization）和步骤级匹配奖励（dense feedback）互补，单独移除任一都会降低性能，同时移除则导致严重退化（梯度爆炸，reward 不稳定）。
- 案例研究表明 SAPO 能产生更精细的推理，保留关键线索（如从耗材到运动控制的转变），而 GRPO 由于全局广播优势，容易忽视特定步骤的信号。
- 结论：结构化生成的 RL 目标应匹配解码器自身的输出分解。

## 7. 优点

- **自然信用分配单位**：利用 SID 码本的层次结构，将每个推理步骤作为自然单位，无需外部过程奖励模型，仅凭已有 SID token 的正确性即可获得密集监督。
- **稳定性提升**：步骤级优势计算和长度归一化有效缓解了 outcome-reward 训练中的长度漂移、KL 爆炸和 reward 不稳定问题。
- **与 rollout-level 目标一致性**：理论证明（命题 1）累积步匹配奖励与精确匹配目标在可实现条件下有相同最优解，同时提供更密集的梯度信号。
- **计算开销低**：步级奖励和优势计算仅针对 K=3 个 SID 位置，几乎不增加 wall-clock 时间（<0.2% 增速）。
- **通用性**：方法可推广到其他具有自然层次解码的结构化生成任务（如工具调用、格式化答案）。

## 8. 不足与局限

- **实验覆盖有限**：仅覆盖三个英文 Amazon 类别，且 SID 层级固定为 K=3。未评估更大目录、更深/不平衡码本、跨语言场景或不同交互模式（如视频、新闻）。
- **反馈设置限制**：仅使用 ground-truth SID 作为可验证反馈，未扩展到多个可接受物品、内隐反馈（点击、停留）或生产级信号（转化、多样性）。
- **解码布局约束**：采用所有思考块在前的解码格式，限制了交替推理/解码的灵活性。未来可探索交错的思考和 SID 生成。
- **案例分析显示偶尔失败**：在物品家族突变的场景下（如从荧光笔到文件夹），SAPO 可能过度利用局部连续性而错误预测（附录 N 案例 4）。
- **计算资源较高**：尽管单步开销低，但 RL 阶段需要大量 rollout 样本（G=16），总训练时长约 48 小时/类别，对资源有一定要求。
- **未探索与多样性、公平性等目标的结合**：论文未讨论与 filter bubble、多样性奖励或公平性约束的集成，这在实际部署中可能需要额外处理。
