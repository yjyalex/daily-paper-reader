## 摘要
大型语言模型（LLMs）被广泛用于生成结构化输出，如JSON、SQL和代码，然而，对于同时满足行业标准XML和领域词汇约束的生成任务的评估，公开资源仍然有限。本文提出了石垣-IDS基准（Ishigaki-IDS-Bench），用于评估从建筑信息模型（BIM）信息需求生成信息交付规范（IDS）XML的能力。该基准包含166个由BIM/IDS专家编写并验证的示例，这些示例通过将83个实际场景扩展为日语和英语创建，并配有对应的黄金IDS文件，以及输入格式、语言、轮次设置、IFC版本和建筑领域的元数据。其评估结合了基于IDSAuditTool的可处理性、结构和内容审计，以及与黄金IDS文件的内容一致性评估。在对10个LLMs进行的零样本评估中，最佳模型的内容一致性的宏F1得分达到65.6%，而只有27.7%的输出通过了内容审计。这些结果表明，当前的LLMs能够将部分信息需求表达为IDS，但在稳定生成满足IDS标准和IFC词汇约束的XML方面仍存在困难。石垣-IDS基准支持比较评估、失败分析以及符合领域约束的结构化生成方法的发展。我们在GitHub和Hugging Face上以CC BY 4.0许可协议发布了评估脚本和基准数据。

## Abstract
Large language models (LLMs) are widely used to generate structured outputs such as JSON, SQL, and code, yet public resources remain limited for evaluating generation that must simultaneously satisfy industry-standard XML and domain vocabulary constraints. This paper presents Ishigaki-IDS-Bench, a benchmark for evaluating the ability to generate Information Delivery Specification (IDS) XML from Building Information Modeling (BIM) information requirements. The benchmark contains 166 BIM/IDS expert-authored and verified examples created by expanding 83 practical scenarios into Japanese and English, corresponding gold IDS files, and metadata for input format, language, turn setting, IFC version, and construction domain. Its evaluation combines IDSAuditTool-based Processability, Structure, and Content audits with content-agreement evaluation against gold IDS files. In zero-shot evaluation over 10 LLMs, the best model reaches 65.6% macro F1 for content agreement, while only 27.7% of outputs pass the Content audit. These results show that current LLMs can express part of the information requirements as IDS, but still struggle to stably generate XML that satisfies the IDS standard and IFC vocabulary constraints. Ishigaki-IDS-Bench supports comparative evaluation, failure analysis, and the development of constrained structured generation methods that conform to domain standards. We release the evaluation scripts and benchmark data under the CC BY 4.0 license on GitHub and Hugging Face.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：构建一个高质量、专家验证的基准数据集，并设计两阶段评估来分别衡量生成的 IDS 的**形式合法性**（是否符合 IDS 标准和 IFC 词汇）和**内容一致性**（是否与输入需求对应）。
- **关键技术细节**：
  - **数据集构建**：由 6 位 BIM/IDS 专家通过 5 步流程完成：①脑暴收集约 100 个用例，筛选为 83 个场景；②为每个场景手动创建输入需求与对应黄金 IDS；③专家审查语义一致性；④用 IDSAuditTool 验证黄金 IDS 的形式正确性；⑤将日语输入翻译为英语并双语审查。最终得到 166 个示例（83 场景 × 两种语言），并标注输入格式（自然语言/CSV）、语言、轮次（单轮/多轮）、IFC 版本、施工领域等元数据。
  - **评估协议**：
    - **第一阶段**：使用 **IDSAuditTool** 对生成的 IDS 进行三层次审计：
      - **Processability**（可处理性）：能否成功提取并完成审计。
      - **Structure**（结构）：是否符合 IDS 的 XML Schema。
      - **Content**（内容）：是否符合 IDS 标准与 IFC 词汇约束。
    - **第二阶段**：**Facet Scorer** 将黄金 IDS 与生成 IDS 分解为“比较块”（按适用性/需求、面类型、约束内容），通过精确匹配计算精确率、召回率和 F1（宏平均）。该评分不依赖元素顺序，并进行了命名空间、空白、大小写等归一化处理。
  - **评估面**：当前覆盖 entity、attribute、property 三个面（未含 classification、material、partOf）。未使用 LLM-as-judge 以避免噪声。

## 6. 论文的主要结论与发现

- **核心发现**：即使最佳模型 GPT-5.5，Facet F1 仅为 65.6%，Content 审计通过率仅 27.7%。说明当前 LLM 能够捕捉部分信息需求，但难以稳定生成同时满足 IDS 标准、IFC 词汇和属性集约定的 XML。
- **重要差异**：
  - 多数模型 Processability 高（>90%），但 Content 低（10-33%），表明形式合法性与内容准确性之间存在巨大鸿沟。
  - 多轮设置（编辑模式）效果显著优于单轮生成（GPT-5.5 多轮 F1 84.8% vs 单轮 58.6%），提示交互式更新 IDS 是更可行的流程。
  - CSV 输入比自然语言输入产生更高 F1，说明表格化需求有助于映射面和值约束。
- **失败模式**：主要集中在 property set 名称、基数（cardinality）、值约束表达，以及自然语言输入中目标和约束之间的歧义性。

## 7. 优点：方法或实验设计上的亮点

- **领域专业性**：基准由 BIM/IDS 专家精心构建并双重验证，使用标准审计工具确保黄金 IDS 的正确性，质量高。

- **丰富的元数据**：标注了输入格式、语言、轮次、IFC 版本、施工领域，支持细粒度分析和诊断。

## 8. 不足与局限

- **规模较小**：仅 166 个示例，相比通用 NLP 基准较小，难以作为训练数据使用。

- **零样本局限**：未评估 few-shot、fine-tuning 或语法约束解码（如 XGrammar）等更强方法的潜力，当前结果代表“瓶颈”而非上限。
- **无真实项目数据**：输入基于实际用例但已抽象化，不直接评估处理真实嘈杂文档或大型 IFC 模型的能力。
- **算力资源未报告**：缺失实验硬件细节，不利于复现和比较效率。
- **无一致性统计**：由于依赖专家单独修订而非多标注者投票，未报告标注者间一致性，对于非严谨专家场景可能存在一定偏差风险。
