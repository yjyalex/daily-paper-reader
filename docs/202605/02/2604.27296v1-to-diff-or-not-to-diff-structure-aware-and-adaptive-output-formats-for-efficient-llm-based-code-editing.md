## 摘要
大语言模型（LLMs）越来越多地被用于代码编辑，然而现行的全量代码生成范式面临严重的效率瓶颈，给要求低延迟和低成本的交互式编程助手带来了挑战。尽管目前的研究主要集中在扩展模型能力上，但编辑格式本身在模型训练中很大程度上被忽视了。在本文中，我们首先对传统的差分（diff）格式进行了系统研究，并揭示了脆弱的偏移量（offsets）和碎片化的代码块（hunks）使得生成过程对 LLMs 而言非常不自然。为了解决这一问题，我们引入了 BlockDiff 和 FuncDiff，这是两种结构感知的差分格式，它们将更改表示为对控制结构和函数等语法连贯单元的块级重写。此外，我们提出了 AdaEdit，这是一种通用的自适应编辑策略，通过训练 LLMs 在给定的差分格式和全量代码之间动态选择最具 Token 效率的格式。大量实验表明，AdaEdit 与结构感知差分格式相结合，在长代码编辑任务中能够持续匹配全量代码生成的准确度，同时将延迟和成本降低了 30% 以上。

## Abstract
Large Language Models (LLMs) are increasingly used for code editing, yet the prevalent full-code generation paradigm suffers from severe efficiency bottlenecks, posing challenges for interactive coding assistants that demand low latency and cost. Despite the predominant focus on scaling model capabilities, the edit format itself has been largely overlooked in model training. In this paper, we begin with a systematic study of conventional diff formats and reveal that fragile offsets and fragmented hunks make generation highly unnatural for LLMs. To address it, we introduce BlockDiff and FuncDiff, two structure-aware diff formats that represent changes as block-level rewrites of syntactically coherent units such as control structures and functions. Furthermore, we propose AdaEdit, a general adaptive edit strategy that trains LLMs to dynamically choose the most token-efficient format between a given diff format and full code. Extensive experiments demonstrate that AdaEdit paired with structure-aware diff formats consistently matches the accuracy of full-code generation, while reducing both latency and cost by over 30% on long-code editing tasks.

### 1. 核心问题与整体含义
*   **研究动机**：当前大语言模型（LLM）在代码编辑任务中主要采用“全量代码生成”模式，即无论修改多小都重写整个文件。这导致了严重的效率瓶颈（高延迟、高 API 成本），且容易引入无关的错误修改。
*   **核心挑战**：传统的差分（diff）格式（如 Unified Diff）虽然节省 Token，但依赖精确的行号偏移，且生成的代码片段（hunks）在语法上是破碎的，这与 LLM 的自然语言生成特性不匹配，导致编辑准确率大幅下降。
*   **研究目标**：设计一种既能保持全量生成的高准确率，又能显著提升效率的 LLM 代码编辑输出格式。

### 2. 方法论
论文提出了两项关键技术：
*   **结构感知差分格式 (Structure-Aware Diff Formats)**：
    *   **核心思想**：利用抽象语法树（AST）将代码修改映射到语法连贯的“块”上，而不是任意的行片段。
    *   **BLOCK DIFF**：在细粒度语法节点（如控制结构、循环、分支）级别进行重写。
    *   **FUNC DIFF**：在函数级别进行重写，提供更稳定的结构上下文。
    *   **关键流程**：使用 `tree-sitter` 构建块树，计算最小 diff 后，将其扩展至包含该修改的最小语法块，并确保锚点内容在全文中唯一以实现精确补丁（Patching）。
*   **自适应编辑策略 (ADA EDIT)**：
    *   **核心思想**：没有一种格式是万能的。当修改范围极大时，diff 格式的开销反而可能超过全量生成。
    *   **技术细节**：在训练阶段，为每个样本计算 diff 格式和全量代码的 Token 长度，将更短的格式设为 Ground Truth。通过监督微调（SFT），使模型学会根据编辑意图和代码长度动态选择最经济的输出格式。

*   **结构感知的价值**：`BLOCK DIFF` 和 `FUNC DIFF` 显著优于行级 diff，准确率接近甚至在某些情况下超过了全量生成（FULL CODE）。

### 7. 优点（亮点）
*   **自然性回归**：通过将 diff 限制在语法块内，巧妙地解决了 LLM 生成破碎代码片段时的“不自然感”。
*   **工业实用性**：`ADA EDIT` 的设计直接面向 IDE 插件等实时交互场景，平衡了准确率与响应速度。
*   **鲁棒性**：提出的格式不依赖精确行号，对模型生成的轻微波动具有更强的容错性。
