## 摘要
在单流自回归接口中，相同的 token 既更新模型状态，又构成不可逆的公开承诺。这种耦合产生了一种“沉默税”：额外的深思熟虑会推迟首个任务相关内容的出现，而天真的早期流式输出则面临过早承诺的风险，从而使后续生成产生偏差。我们引入了并排（SxS）交替推理，使披露时机成为标准自回归生成中一个可控的决策。SxS 在同一上下文中将部分披露与持续的私有推理交替进行，但仅在内容得到目前推理支持时才发布。为了在不激励填充内容的情况下学习这种节奏，我们通过将答案前缀与支持性推理前缀匹配，构建了蕴含对齐的交替轨迹，然后通过监督微调（SFT）获取双重动作语义，并通过强化学习（RL）恢复新格式下的推理性能。在两种 Qwen3 架构/规模（MoE Qwen3-30B-A3B，稠密 Qwen3-4B）以及领域内（AIME25）和领域外（GPQA-Diamond）基准测试中，SxS 在诸如更新间等待等 token 级代理指标下，改善了准确率与内容延迟之间的帕累托权衡。

## Abstract
In single-stream autoregressive interfaces, the same tokens both update the model state and constitute an irreversible public commitment. This coupling creates a silence tax: additional deliberation postpones the first task-relevant content, while naive early streaming risks premature commitments that bias subsequent generations. We introduce Side-by-Side (SxS) Interleaved Reasoning, which makes disclosure timing a controllable decision within standard autoregressive generation. SxS interleaves partial disclosures with continued private reasoning in the same context, but releases content only when it is supported by the reasoning so far. To learn such pacing without incentivizing filler, we construct entailment-aligned interleaved trajectories by matching answer prefixes to supporting reasoning prefixes, then train with SFT to acquire the dual-action semantics and RL to recover reasoning performance under the new format. Across two Qwen3 architectures/scales (MoE Qwen3-30B-A3B, dense Qwen3-4B) and both in-domain (AIME25) and out-of-domain (GPQA-Diamond) benchmarks, SxS improves accuracy--content-latency Pareto trade-offs under token-level proxies such as inter-update waiting.

### 1. 核心问题与整体含义
论文针对大语言模型（LLM）在推理任务中的**“沉默税”（Silence Tax）**与**“过早承诺”（Premature Commitment）**之间的矛盾展开研究。
*   **研究动机**：在标准的单流自回归生成中，模型生成的每一个 token 既是内部状态的更新，也是不可逆的公开输出。这导致模型若想通过长链条推理（CoT）提高准确率，用户就必须忍受长时间的等待（沉默税）；若为了降低延迟而过早输出结论，则可能因缺乏充分推理而产生偏差或错误（过早承诺）。
*   **核心目标**：使模型能够自主决定“何时思考”以及“何时发言”，在保证输出内容有推理支持的前提下，尽早向用户披露阶段性成果。

### 2. 方法论
论文提出了 **Side-by-Side (SxS) 交替推理**框架，将披露时机建模为一个可学习的控制问题。
*   **核心思想**：在同一个 token 流中引入特殊的标签（如 `think` 和 `speak`），使模型在同一上下文中交替进行私密推理和公开披露。
*   **关键技术细节**：
    1.  **蕴含对齐的轨迹构建（SFT 阶段）**：利用一个强大的教师模型（如 GPT-OSS-120B）作为蕴含检查器，将原始的“推理-答案”对进行切片对齐。只有当某段推理前缀能够逻辑蕴含（Entail）某段答案前缀时，才将该答案块插入到推理流中。这确保了早期披露的内容是“有据可查”的。
    2.  **双阶段训练**：
        *   **SFT（监督微调）**：学习 `think/speak` 的双重动作语义和交替格式。
        *   **RL（强化学习）**：使用 GRPO 算法。由于交替格式改变了数据分布，模型初始准确率会下降，RL 旨在恢复推理性能并优化准确率与延迟的权衡。
    3.  **奖励设计**：主要采用基于结果的正确性奖励。此外，可选地引入“粒度奖励”，通过二次规划（QP）在不损害正确性信号的前提下，鼓励模型进行更频繁、更短小的交替（即提高输出频率）。

### 6. 主要结论与发现
*   **打破沉默税**：SxS 显著降低了用户等待首个有效内容的时间（AIRW 指标大幅下降），实现了更优的准确率-延迟帕累托权衡。
*   **RL 的关键作用**：SFT 虽然能教会模型格式，但会导致准确率骤降（分布偏移），必须通过 RL 才能找回推理能力。
*   **鲁棒性提升**：在领域外任务（如 GPQA）上，SxS 表现出比标准 CoT 更强的抗灾难性遗忘能力，可能是因为强制性的“推理-输出”对齐减少了无意义的奖励黑客行为。
*   **可控性**：通过奖励重塑，可以有效控制模型输出的频率（粒度），且不会显著损害最终性能。

### 7. 优点
*   **架构兼容性**：无需修改模型结构，仅通过标签和数据工程即可在标准自回归模型上实现。
*   **逻辑严密性**：通过蕴含检查器构建数据，避免了模型为了降低延迟而生成无意义的填充内容（Filler）。
*   **交互体验优化**：将 LLM 推理从“黑盒等待”转变为“实时进度披露”，更符合人类交互习惯。

### 8. 不足与局限
*   **预处理成本高**：构建蕴含对齐的数据集需要调用大型模型进行大量推理和逻辑检查，计算开销较大。
*   **对教师模型的依赖**：数据质量高度依赖于用于蕴含检查的教师模型，若教师模型判断失误，可能导致训练数据中出现逻辑断层。
*   **性能恢复成本**：模型在 SFT 后需要较长时间的 RL 训练才能恢复到基准准确率，增加了训练复杂度和资源消耗。
