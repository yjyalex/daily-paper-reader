## 摘要
大语言模型（LLM）多智能体系统通常依赖于僵化的编排方式，要么采用扁平的单次查询路由，要么采用手工设计的任务分解，导致分解深度、执行者选择和推理预算无法在单一目标下进行联合优化。我们提出了 Uno-Orchestra，这是一种统一的编排策略，它能够选择性地分解任务，并将每个子任务分派给一个可行的（模型，原语）对，这两个决策都是通过基于真实执行者交互的精选强化学习（RL）轨迹共同学习得到的。在涵盖数学、代码、知识、长上下文和智能体工具使用的 13 个基准测试集上，Uno-Orchestra 与 22 个基准模型相比，达到了 77.0% 的宏观 pass@1，比最强的工作流基准高出约 16%，而单次查询成本降低了约一个数量级，推进了选择性委派的准确率-效率前沿。

## Abstract
Large language model (LLM) multi-agent systems typically rely on rigid orchestration, committing either to flat per-query routing or to hand-engineered task decomposition, so decomposition depth, worker choice, and inference budget are not jointly optimized under one objective. We introduce Uno-Orchestra, a unified orchestration policy that selectively decomposes a task and dispatches each subtask to an admissible (model, primitive) pair, with both decisions learned together from curated RL trajectories grounded in real worker interactions. Against 22 baselines on a 13-benchmark suite spanning math, code, knowledge, long-context, and agentic tool-use, Uno-Orchestra reaches 77.0% macro pass@1, roughly 16% above the strongest workflow baseline, at roughly an order of magnitude lower per-query cost, advancing the accuracy-efficiency frontier of selective delegation.

### 1. 核心问题与整体含义（研究动机）
当前 LLM 多智能体系统存在两种极端的编排模式：
*   **扁平路由（Flat Routing）：** 简单地将整个查询分配给一个模型，无法处理需要并行或多步协作的复杂任务。
*   **僵化编排（Rigid Orchestration）：** 依赖手工设计的任务分解流，分解深度和执行者选择是预设的，无法根据任务难度动态调整，导致资源浪费。
*   **核心动机：** 提出一种“选择性委派（Selective Delegation）”机制，让一个统一的策略模型能够自主决定：**是否需要分解任务、分解到什么程度、以及将每个子任务分配给哪个（模型, 原语）对**，从而在单一目标下优化准确率、效率和推理成本。

### 2. 方法论：核心思想与技术细节
Uno-Orchestra 将编排视为一个多轮决策过程，核心技术包括：
*   **统一策略架构：** 使用单个因果语言模型（Causal-LM）作为编排器。在同一个助手回合内，它同时生成任务分解计划（DAG 图）和每个子任务的路由决策（模型 $m$ + 原语 $s$）。
*   **动作空间（模型, 原语）：** 原语（Primitive）涵盖了模型内部认知操作（如直接回答、思维链）、多步技能调用（如文档阅读）和外部工具调用（如代码执行）。
*   **两阶段训练流程：**
    1.  **验证器门控课程 SFT：** 从 38 个公共数据集中提取任务，利用强老师模型（如 GPT-4o）生成轨迹，仅保留通过验证器（Verifier）检查的正确轨迹进行监督微调（约 6.1 万条）。
    2.  **Agentic-GRPO 强化学习：** 针对多轮智能体交互改进了 GRPO 算法。引入了**回合级信用分配（Turn-level Credit Assignment）**和过程奖励，解决了长程强化学习中的信用分配难题，使模型能学习到何时该“偷懒”（直接回答）或何时该“勤奋”（深度分解）。
*   **成本感知目标：** 奖励函数结合了验证器正确性信号和归一化的推理成本，强制模型在保证正确的前提下选择最廉价的路径。

### 6. 主要结论与发现
*   **性能领先：** Uno-Orchestra 达到了 77.0% 的宏观 Pass@1，比最强的工作流基准（AgentOrchestra）高出约 16%。
*   **成本极低：** 在性能大幅领先的同时，其单次查询成本比重型工作流系统降低了约一个数量级（约 1/10）。
*   **选择性委派的有效性：** 模型学会了对简单问题直接回答（零编排成本），对复杂问题进行并行分解。
*   **泛化能力：** 在分布外（Out-of-domain）任务上的表现甚至优于分布内任务，证明了策略的通用性。

### 7. 优点与亮点
*   **架构优雅：** 将任务分解和路由决策合并到单个模型的一次前向传播中，消除了多个模块间的冗余上下文传递。
*   **RL 创新：** Agentic-GRPO 解决了智能体在多轮交互中难以获得中间反馈的问题，使强化学习在复杂编排场景下变得可行。
*   **效率前沿：** 成功证明了“更聪明地分配任务”比“堆砌更多智能体”更能提升系统整体效能。

### 8. 不足与局限
*   **验证器依赖：** 训练高度依赖自动验证器（如数学的符号匹配、代码的单元测试）。对于缺乏客观评价标准的开放式任务（如创意写作），该方法难以直接应用。
*   **API 不稳定性：** 实验基于特定时间点的商业模型 API 价格和性能，随着 API 的更新或价格调整，最优路由策略可能需要重新训练。
*   **延迟问题：** 虽然成本降低了，但多轮编排（最多 8 轮）在某些实时性要求极高的场景下可能仍存在延迟挑战。
