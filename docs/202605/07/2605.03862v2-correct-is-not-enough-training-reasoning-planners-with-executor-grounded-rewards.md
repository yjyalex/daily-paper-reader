## 摘要
带有可验证奖励的强化学习已成为提高大语言模型显式推理能力的常用方法，但仅凭最终答案的正确性无法揭示推理轨迹是否忠实、可靠，或者对使用它的模型是否有用。这种仅关注结果的信号可能会强化“因错误原因而得出正确结果”的轨迹，通过奖励捷径夸大推理收益，并在多步系统中传播有缺陷的中间状态。为此，我们提出了 TraceLift，这是一个将推理视为可消耗中间产物的规划器-执行器训练框架。在规划器训练期间，规划器输出带标签的推理。一个冻结的执行器将该推理转化为最终产物以获取验证器反馈，同时利用基于执行器的奖励来塑造中间轨迹。该奖励将基于评分标准的推理奖励模型（RM）分数与在同一冻结执行器上测得的提升值相乘，从而奖励既高质量又有用的轨迹。为了使推理质量直接可学习，我们引入了 TRACELIFT-GROUPS，这是一个基于数学和代码种子问题构建的、带有评分标准注释的纯推理数据集。每个样本都是针对同一问题的分组，包含一个高质量参考轨迹和多个看似合理但有缺陷的轨迹，后者通过局部扰动降低了推理质量或对解法的支持度，同时保留了任务相关性。在代码和数学基准测试上的广泛实验表明，这种基于执行器的推理奖励比仅基于执行的训练更能提升两阶段规划器-执行器系统，这表明推理监督不仅应评估轨迹是否“看起来不错”，还应评估它是否对使用它的模型有所帮助。

## Abstract
Reinforcement learning with verifiable rewards has become a common way to improve explicit reasoning in large language models, but final-answer correctness alone does not reveal whether the reasoning trace is faithful, reliable, or useful to the model that consumes it. This outcome-only signal can reinforce traces that are right for the wrong reasons, overstate reasoning gains by rewarding shortcuts, and propagate flawed intermediate states in multi-step systems. To this end, we propose TraceLift, a planner-executor training framework that treats reasoning as a consumable intermediate artifact. During planner training, the planner emits tagged reasoning. A frozen executor turns this reasoning into the final artifact for verifier feedback, while an executor-grounded reward shapes the intermediate trace. This reward multiplies a rubric-based Reasoning Reward Model (RM) score by measured uplift on the same frozen executor, crediting traces that are both high-quality and useful. To make reasoning quality directly learnable, we introduce TRACELIFT-GROUPS, a rubric-annotated reason-only dataset built from math and code seed problems. Each example is a same-problem group containing a high-quality reference trace and multiple plausible flawed traces with localized perturbations that reduce reasoning quality or solution support while preserving task relevance. Extensive experiments on code and math benchmarks show that this executor-grounded reasoning reward improves the two-stage planner-executor system over execution-only training, suggesting that reasoning supervision should evaluate not only whether a trace looks good, but also whether it helps the model that consumes it. Our code is available at: https://github.com/MasaiahHan/TraceLift

### 1. 核心问题与研究动机
*   **核心问题**：当前基于可验证奖励（如答案正确性）的强化学习虽然能提升模型表现，但存在“结果导向”的弊端。模型可能因为错误的原因得到正确答案（歪打正着），或者通过走捷径、掩盖中间错误来获得奖励。
*   **研究动机**：推理轨迹（Reasoning Trace）不应仅仅是事后解释，而应作为一种“可消耗的中间产物”来辅助后续执行。作者认为，评估推理的好坏不仅要看它是否符合逻辑，更要看它是否对后续的“执行器”（Executor）真正有用。

### 2. 方法论：TraceLift 框架
*   **核心思想**：将推理过程视为规划器（Planner）与固定执行器（Executor）之间的接口。规划器负责生成推理，冻结的执行器根据推理生成最终结果。
*   **关键技术细节**：
    *   **执行器锚定奖励（Executor-Grounded Reward）**：奖励函数由两部分组成：
        $$R = 0.5 \times R_{exec} + 0.5 \times RM_{\phi} \times u_{exec}$$
        其中 $R_{exec}$ 是最终结果的验证分数；$RM_{\phi}$ 是基于评分标准（Rubric）的推理奖励模型分数；$u_{exec}$ 是**执行器提升值（Uplift）**，即“有推理”相比“无推理”在执行器成功率上的差值。
    *   **TraceLift-Groups 数据集**：为了训练推理奖励模型（Reasoning RM），构建了包含 6,000 个组的数据集。每组包含一个高质量参考轨迹和多个经过局部扰动（如算法错误、计算失误、遗漏边界情况等）的有缺陷轨迹。
    *   **训练流程**：
        1.  **阶段一**：在 TraceLift-Groups 上训练一个能够识别推理质量（而非仅看答案）的推理 RM。
        2.  **阶段二**：使用 GRPO 算法训练规划器，利用上述复合奖励函数进行优化。

### 6. 主要结论与发现
*   **性能提升**：TraceLift 在所有基准测试中均优于“仅结果导向”的训练方法。例如在 Qwen2.5-7B 上，代码平均分提升了 2.61%，数学平均分提升了 4.51%。
*   **推理质量**：TraceLift 训练出的模型生成的推理轨迹更短、更精炼（比 Exec-only 平均短 30 个 token），且对执行器的辅助作用更强。
*   **避免冗余**：实验证明，性能的提升并非来自于推理长度的增加（Verbosity），而是来自于推理内容的实用性。

### 7. 优点
*   **视角新颖**：将推理视为“执行器的输入”，引入 Uplift 概念来量化推理的实际效用，解决了信用分配（Credit Assignment）难题。
*   **数据质量高**：TraceLift-Groups 通过局部扰动生成负样本，强迫 RM 学习细粒度的逻辑差异，而非简单的表面模式匹配。
*   **鲁棒性强**：即使推理 RM 不完美，由于有最终验证器（Verifier）的锚定，系统仍能保持任务目标的正确性。

### 8. 不足与局限
*   **执行器依赖**：该框架假设执行器是冻结的。如果执行器本身能力极弱，可能无法提供有效的 Uplift 信号。
*   **合成数据偏差**：TraceLift-Groups 的负样本是人工/模型合成的扰动，可能无法完全覆盖真实场景中人类或模型可能犯的所有逻辑错误。
*   **计算开销**：在训练期间需要多次调用执行器来估计 Uplift（如 K=3），这增加了训练时的计算成本。
