## Abstract
Reasoning in Large Language Models incurs significant inference-time compute, yet the token-level information structure of reasoning traces remains underexplored. We observe that reasoning tokens split into two functional types: low-entropy \textit{structural} tokens (recurring phrases that scaffold the reasoning process) and higher-entropy \textit{organic} tokens (problem-specific content that drives toward a solution). This asymmetry motivates a simple, model-agnostic compression pipeline: apply cross-word BPE merges on a model's own reasoning traces to derive \textit{supertokens} that capture frequent structural patterns, then teach the model to adopt them via supervised fine-tuning. Across three model families and five mathematical reasoning benchmarks, our approach shortens reasoning traces by 8.1\% on average with no statistically significant accuracy loss on any model--benchmark pair. Beyond compression, supertokens act as interpretable reasoning-move annotations (backtracking, verification, strategy shifts), exposing the model's high-level strategy at a glance. Analyzing transitions between structural categories reveals systematic differences between correct and incorrect traces: correct traces show productive recovery (backtracking followed by strategy shifts and verification), while incorrect traces are dominated by confusion cycles (repeated hedging and unresolved contradictions). These diagnostic signals suggest applications in reward shaping and early stopping for RL-based reasoning training.

*   **核心问题**：思维链（CoT）推理虽然显著提升了 LLM 的性能，但会产生极长的推理轨迹，导致推理成本（计算量和延迟）大幅增加。
*   **研究背景**：现有的压缩方法通常是“内容级”的（如删除步骤或蒸馏），这往往会破坏推理的完整性或可读性。
*   **核心观察**：推理 Token 可分为两类：**结构化 Token**（如“Wait, let me think”等重复出现的引导词，熵值低、可预测性强）和**有机 Token**（如具体的数学公式、变量，熵值高、承载核心信息）。
*   **整体含义**：通过将低信息的结构化短语合并为单个“超级 Token”（Supertoken），可以在不损失推理质量的前提下，实现推理过程的物理压缩。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：利用信息论中的熵值差异，识别并合并推理轨迹中的高频、低熵短语，通过微调让模型学会使用这些“速记符号”。
*   **关键技术流程**：
    1.  **超级 Token 提取**：在模型生成的推理语料上应用跨词 BPE（SuperBPE）算法，提取频繁出现的 n-gram。
    2.  **结构化过滤**：设计过滤器，仅保留符合推理逻辑的短语（如以大写字母开头、标点符号结尾的短语），剔除无意义的语法碎片。
    3.  **词表扩展与初始化**：将选定的超级 Token 加入原词表。新 Token 的嵌入向量（Embedding）通过其组成部分的向量平均值进行初始化。
    4.  **自适应微调（SFT）**：在重构后的数据集上对模型进行有监督微调。为了保持模型能力，通常只训练嵌入层、输出头和少量 Transformer 层（如前 N 层和后 M 层）。
*   **理论支撑**：利用条件熵公式证明，一旦结构化短语开始，其后续 Token 的确定性极高，因此合并它们具有极高的压缩潜力。

### 3. 实验设计
*   **实验模型**：QwQ-32B、Qwen3-30B-A3B、DeepSeek-R1-Distill-Llama-70B。
*   **数据集/Benchmark**：涵盖 5 个数学推理基准：AIME '24、AIME '25、MATH-500、Minerva、OlympiadBench。

*   **评估指标**：准确率（Accuracy）、Token 减少率、超级 Token 采用率、端到端推理延迟。

### 5. 实验数量与充分性
*   **实验规模**：在 3 个主流模型家族和 5 个难度极高的数学竞赛数据集上进行了验证，实验组数充足。
*   **消融实验**：对比了不同的微调模式（仅 Embedding、LoRA、部分层解冻）、LoRA Rank 的影响、以及跨模型的熵值验证。
*   **统计严谨性**：所有准确率和 Token 减少量均提供了 **95% 置信区间（CI）**，并进行了显著性检验（p < 0.05），确保结果不是随机波动。
*   **充分性评价**：实验设计非常全面，不仅验证了压缩效果，还通过 IFEval 验证了模型通用指令遵循能力未受损。

### 6. 主要结论与发现
*   **压缩效果**：在保持准确率不变的情况下，推理轨迹平均缩短了 **8.1%**（部分任务达 17%）。
*   **性能保持**：在所有模型-基准对上，准确率的变化在统计学上均不显著（即实现了无损压缩）。
*   **可解释性发现**：超级 Token 充当了“推理动作”的标签。研究发现：
    *   **正确推理**：表现为“生产性恢复”（回溯后紧跟策略转变和验证）。
    *   **错误推理**：表现为“混乱循环”（反复的模棱两可、未解决的矛盾）。
*   **效率提升**：实际推理延迟（Wall-clock latency）的降低与 Token 减少量基本同步。

### 7. 优点
*   **模型无关性**：该方法不依赖特定架构，可广泛应用于各类推理模型。
*   **保持可读性**：与潜空间压缩不同，超级 Token 解码后仍是人类可读的自然语言，保留了推理的透明度。
*   **诊断价值**：通过超级 Token 的转移矩阵，可以实时监控模型的推理状态，为强化学习（RL）中的奖励建模或早期停止提供信号。
*   **正交性**：可以与现有的内容级压缩方法叠加使用，进一步提升压缩比。

### 8. 不足与局限
*   **压缩率上限**：相比于直接删除推理步骤的方法（可达 30-70%），8.1% 的压缩率较为温和。
*   **分词器敏感性**：在 Llama 系列分词器上的表现略逊于 Qwen 系列，说明基础词表的结构会影响超级 Token 的集成效果。
*   **训练成本**：虽然是轻量级微调，但仍需要针对每个模型进行专门的 SFT 过程，无法像某些剪枝方法那样实现完全的即插即用。
