## 摘要
结构化 LLM 工作流（其中专门的 LLM 子智能体根据预定义的图执行）已成为解决复杂任务的强大抽象。优化此类工作流，即为每个子智能体选择配置以平衡准确率和延迟，由于模型选择、推理预算和工作流结构构成的组合设计空间而具有挑战性。现有的成本感知方法在很大程度上将工作流优化视为路由问题，在推理时根据训练期间使用的准确率-延迟目标为每个查询选择配置。我们认为结构化 LLM 工作流也可以从编译的角度进行优化：在部署之前，系统可以全局探索工作流设计空间，并构建一组可重用的工作流级配置，涵盖多种准确率-延迟权衡。受机器学习编译器的启发，我们推出了 FlowCompile，这是一种结构化 LLM 工作流编译器，它执行编译时设计空间探索，以识别高质量、可重用的权衡集。FlowCompile 将工作流分解为子智能体，在不同配置下对每个子智能体进行性能分析，并通过结构感知代理组合这些测量结果，以估计工作流级的准确率和延迟。它在单次编译过程中即可识别出多种高质量配置，无需重新训练或在线自适应。在不同工作流和具有挑战性的基准测试中进行的实验表明，FlowCompile 始终优于启发式优化的工作流配置和基于路由的基准方法，实现了高达 6.4 倍的加速。编译后的配置集进一步作为可重用的优化产物，支持在不同的运行时偏好下进行灵活部署，并支持下游的选择或路由。

## Abstract
Structured LLM workflows, where specialized LLM sub-agents execute according to a predefined graph, have become a powerful abstraction for solving complex tasks. Optimizing such workflows, i.e., selecting configurations for each sub-agent to balance accuracy and latency, is challenging due to the combinatorial design space over model choices, reasoning budgets, and workflow structures. Existing cost-aware methods largely treat workflow optimization as a routing problem, selecting a configuration at inference time for each query according to the accuracy-latency objective used during training. We argue that structured LLM workflows can also be optimized from a compilation perspective: before deployment, the system can globally explore the workflow design space and construct a reusable set of workflow-level configurations spanning diverse accuracy-latency trade-offs. Drawing inspiration from machine learning compilers, we introduce FlowCompile, a structured LLM workflow compiler that performs compile-time design space exploration to identify a high-quality, reusable trade-off set. FlowCompile decomposes a workflow into sub-agents, profiles each sub-agent under diverse configurations, and composes these measurements through a structure-aware proxy to estimate workflow-level accuracy and latency. It then identifies diverse high-quality configurations in a single compile-time pass, without retraining or online adaptation. Experiments across diverse workflows and challenging benchmarks show that FlowCompile consistently outperforms heuristically optimized workflow configurations and routing-based baselines, delivering up to 6.4x speedup. The compiled configuration set further serves as a reusable optimization artifact, enabling flexible deployment under varying runtime preferences and supporting downstream selection or routing.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：随着 LLM 系统从单模型推理演变为由多个子智能体（Sub-agents）组成的结构化工作流，如何为每个子智能体选择最佳配置（模型类型、推理预算、工作流结构）以平衡准确率和延迟，成为了一个极具挑战性的组合优化问题。
*   **研究动机**：现有的优化方法（如路由算法）通常在推理时为每个查询动态选择配置，这往往需要针对特定的权衡目标进行重新训练或调整。作者受机器学习编译器（如 TVM）的启发，提出将工作流优化视为一个**编译时（Compile-time）**问题，即在部署前全局探索设计空间，生成一套可重用的优化配置集（Pareto 前沿），以适应不同的部署需求。

### 2. 方法论：核心思想与技术细节
FlowCompile 的核心思想是将复杂工作流分解为独立的子智能体进行性能分析，然后通过结构感知代理进行全局合成。其流程分为三个阶段：
*   **阶段 1：子智能体数据归纳与性能分析（Profiling）**
    *   使用高能力参考模型（如 GPT-5）执行工作流，收集执行轨迹并诱导产生子智能体级别的伪标签数据集。
    *   在不同模型（如 Qwen-3 系列）和推理预算（Reasoning Budget，即 Token 限制）下，独立测试每个子智能体的准确率和延迟，构建底层成本模型。
*   **阶段 2：结构感知的工作流级性能估计（Compositional Estimation）**
    *   提出一种**结构感知代理（Structure-aware Proxy）**，利用解析规则（如顺序执行的乘法规则、并行执行的概率规则、条件分支的加权规则）将子智能体的性能指标合成为工作流级的准确率和延迟估计。
    *   这种方法避免了对数百万种组合进行穷举式的端到端测试，将搜索复杂度从组合级降低到线性级。
*   **阶段 3：设计空间探索与部署**
    *   对子智能体配置进行帕累托剪枝（Pareto Pruning），剔除被支配的低效配置。
    *   对剩余空间进行非支配排序，产出一组涵盖从“极速”到“高精”不同权衡点的优化配置集。

### 6. 主要结论与发现
*   **性能卓越**：FlowCompile 产出的配置在相同准确率下实现了显著加速，在 LiveCodeBench 上最高达到 **6.4 倍加速**。
*   **帕累托前沿更优**：相比于现有的路由算法，FlowCompile 发现的配置集在整个准确率-延迟曲线上都表现更好。

### 7. 优点
*   **范式创新**：首次将“编译”概念引入结构化 LLM 工作流优化，将昂贵的在线搜索转化为高效的离线分析。
*   **可扩展性**：通过分解与合成，成功处理了包含数百万甚至数十亿种可能性的组合设计空间。
*   **灵活性**：不仅优化模型选择，还同时优化推理预算和工作流拓扑结构。
*   **实用性**：产出的优化配置集是静态的、可重用的，非常适合实际生产环境的部署。

### 8. 不足与局限
*   **适用范围限制**：目前仅适用于**结构化**工作流（预定义图），对于像 ReAct 这种动态、开放式的智能体系统（执行轨迹随查询剧烈变化）支持有限。
*   **代理估计偏差**：虽然相关性很高，但结构感知代理仍是近似模型，可能无法完全捕捉子智能体之间复杂的非线性交互或分布偏移。
*   **依赖参考模型**：初始的数据归纳依赖于一个高能力的参考模型，如果参考模型在某些领域表现不佳，可能会影响后续的性能分析质量。
