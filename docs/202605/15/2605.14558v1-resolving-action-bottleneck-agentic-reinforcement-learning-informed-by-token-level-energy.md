## 摘要
智能体强化学习利用多轮轨迹训练大语言模型，这些轨迹交织了长推理链与短促的面向环境的动作。常用的策略梯度方法（如 PPO 和 GRPO）对轨迹中的每个 token 一视同仁，导致了均匀的信用分配。在本文中，我们批判性地证明了这种均匀的信用分配在很大程度上错配了 token 级的训练信号。从基于能量的模型视角出发，我们表明 token 级的训练信号（通过其与从给定提示词采样出的不同 rollout 的奖励方差之间的相关性来量化）急剧集中在动作 token 而非推理 token 上，尽管动作 token 仅占轨迹的一小部分。我们将这一现象称为“动作瓶颈”（Action Bottleneck）。受此启发，我们提出了一种极其简单的 token 重加权方法 ActFocus，该方法降低了推理 token 的梯度权重，并结合了一种额外的基于能量的再分配机制，进一步增加了具有更高不确定性的动作 token 的权重。在四种环境和不同模型规模下，ActFocus 始终优于 PPO 和 GRPO，在不增加任何额外运行时间或内存开销的情况下，最终步的增益分别高达 65.2 和 63.7 个百分点。

## Abstract
Agentic reinforcement learning trains large language models using multi-turn trajectories that interleave long reasoning traces with short environment-facing actions. Common policy-gradient methods, such as PPO and GRPO, treat each token in a trajectory equally, leading to uniform credit assignment. In this paper, we critically demonstrate that such uniform credit assignment largely misallocates token-level training signals. From an energy-based modeling perspective, we show that token-level training signals, quantified by their correlations with reward variance of different rollouts sampled from a given prompt, concentrate sharply on action tokens rather than reasoning tokens, even though action tokens account for only a small fraction of the trajectory. We refer to this phenomenon as the Action Bottleneck. Motivated by this observation, we propose an embarrassingly simple token reweighting approach, ActFocus, that downweights gradients on reasoning tokens, along with an additional energy-based redistribution mechanism that further increases the weights on action tokens with higher uncertainty. Across four environments and different model sizes, ActFocus consistently outperforms PPO and GRPO, yielding final-step gains of up to 65.2 and 63.7 percentage points, respectively, without any additional runtime or memory cost.

#### 1. 核心问题与整体含义（研究动机和背景）
在智能体强化学习（Agentic RL）中，大型语言模型（LLM）生成的轨迹通常包含长篇幅的推理过程（Reasoning）和极短的动作指令（Action）。目前的策略梯度方法（如 PPO 和 GRPO）普遍采用**均匀信用分配（Uniform Credit Assignment）**，即对轨迹中的每个 token 分配相同的权重。

论文指出，这种做法会导致**“动作瓶颈”（Action Bottleneck）**现象：
*   **信号稀释**：动作 token 虽然只占轨迹的极小比例（如在 Sokoban 任务中仅占 4%），但它们是决定环境状态转移和最终奖励的关键。
*   **错配问题**：作者通过能量模型分析发现，训练信号（与奖励方差强相关的部分）高度集中在动作 token 上，而推理 token 的梯度贡献大多是噪声。
*   **不稳定性**：均匀分配导致梯度被大量无关的推理 token 淹没，引发训练崩溃或性能退化。

#### 2. 论文提出的方法论：ActFocus
为了解决动作瓶颈，论文提出了一种名为 **ActFocus** 的 token 级重加权机制，核心思想是“抑制推理，聚焦动作”。

*   **核心思想**：利用轨迹的结构化信息（如 `<think>` 和 `<answer>` 标签）和冻结参考模型的**自由能量（Free Energy）**来重新分配梯度权重。
*   **关键技术细节**：
    1.  **推理降权**：为推理 token 分配一个较小的权重因子 $\alpha$（如 0.1），减少其对梯度更新的干扰。
    2.  **动作增强与能量重分配**：对于动作 token，利用冻结参考模型的输出对数几率（logits）计算 token 级能量 $E_t = -\log \sum \exp(f^{ref}_{t,v})$。能量越高，代表模型对该动作越不确定。
    3.  **权重公式**：
        $$w_t = \begin{cases} \alpha, & t \in \text{Reasoning} \\ 1 + \beta \cdot \text{sigmoid}(\text{normalized } E_t), & t \in \text{Action} \end{cases}$$
        其中 $\beta$ 控制能量调制的强度。
*   **算法流程**：在计算 PPO 或 GRPO 的代理损失函数时，将原本均匀的 $w_t=1$ 替换为上述计算出的 $w_t$，从而引导模型关注那些高不确定性的关键动作。
