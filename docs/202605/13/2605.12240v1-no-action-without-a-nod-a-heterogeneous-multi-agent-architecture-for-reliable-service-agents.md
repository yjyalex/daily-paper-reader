## 摘要
大语言模型（LLM）智能体日益推动了服务应用的发展，例如预订机票。然而，这些服务智能体在长程任务中表现出不可靠性，经常产生策略违规、工具幻觉和动作失配，这极大地阻碍了它们的实际部署。为了应对这些挑战，我们提出了 NOD（Navigator-Operator-Director），一种用于服务智能体的异构多智能体架构。与以往工作中在对话上下文中隐式维护任务状态不同，我们将结构化的全局状态（Global State）外置，以实现显式的任务状态追踪和 Navigator 的一致性决策。此外，我们在关键动作之前引入了选择性的外部监督，允许独立的 Director 智能体验证执行情况并在必要时进行干预。因此，NOD 有效地缓解了长程任务中的错误传播和不安全行为。在 $τ^2$-Bench 上的实验表明，NOD 相比基准模型实现了更高的任务成功率和关键动作精度。更重要的是，NOD 通过减少策略违规、工具幻觉和用户意图失配，提高了服务智能体的可靠性。

## Abstract
Large language model (LLM) agents have increasingly advanced service applications, such as booking flight tickets. However, these service agents suffer from unreliability in long-horizon tasks, as they often produce policy violations, tool hallucinations, and misaligned actions, which greatly impedes their real-world deployment. To address these challenges, we propose NOD (Navigator-Operator-Director), a heterogeneous multi-agent architecture for service agents. Instead of maintaining task state implicitly in dialogue context as in prior work, we externalize a structured Global State to enable explicit task state tracking and consistent decision-making by the Navigator. Besides, we introduce selective external oversight before critical actions, allowing an independent Director agent to verify execution and intervene when necessary. As such, NOD effectively mitigates error propagation and unsafe behavior in long-horizon tasks. Experiments on $τ^2$-Bench demonstrate that NOD achieves higher task success rates and critical action precision over baselines. More importantly, NOD improves the reliability of service agents by reducing policy violations, tool hallucinations, and user-intent misalignment.

*   **研究动机**：作者归纳了两个核心瓶颈：
    1.  **隐式状态脆弱性（Implicit State Fragility）**：任务状态仅存储在对话上下文中，随着对话增长，智能体难以准确追踪约束和进度。
    2.  **不可靠的内在自我修正（Unreliable Intrinsic Self-correction）**：让同一个模型修正自己的错误通常效果有限，尤其是在执行环境改变的动作时。
*   **整体含义**：通过将“状态追踪”与“动作执行”分离，并引入“外部监督”，将服务执行转变为一个结构化的控制问题。

### 2. 方法论：核心思想与技术细节
NOD 架构由三个专门的智能体角色组成，并引入了显式的**全局状态（Global State）**：
*   **Navigator（导航员）**：负责维护结构化的全局状态（JSON 格式），包括用户意图、约束条件、缺失信息、子任务进度、实体信息和对话摘要。它将隐式的上下文转化为显式的任务表示。
*   **Operator（操作员）**：基于 Navigator 提供的全局状态进行日常对话和提出动作建议。它不再直接处理原始长对话，而是基于经过整理的状态进行决策。
*   **Director（导演/监督员）**：作为安全网，仅在 **关键动作（Critical Actions）**（如修改订单、支付等改变外部环境的操作）执行前介入。
    *   **决策逻辑**：Director 评估动作是否符合策略和意图，返回三种指令：
        *   `PASS`：允许执行。
        *   `REVISE`：拦截动作，要求 Navigator 更新状态并让 Operator 重新提议。
        *   `ABORT`：判定动作极度危险或无法修复，直接终止。
*   **异构性**：通常使用较小的模型担任 Navigator 和 Operator 以节省成本，而使用最强的模型（Frontier Model）担任 Director 以确保监督质量。

### 6. 主要结论与发现
*   **性能提升**：NOD 在 SR 上最高提升了 **23.7** 个百分点，在 CAP 上最高提升了 **25.8** 个百分点。
*   **可靠性增强**：显著减少了策略违规和工具幻觉。
*   **分工明确**：实验发现 `REVISE` 指令主要负责提升任务成功率（纠偏），而 `ABORT` 指令主要负责提升安全性（拦截错误）。
*   **效率平衡**：Director 仅在关键动作时触发（约占总轮次的少数），在保证可靠性的同时避免了每一轮都调用高昂模型的成本。

### 7. 优点
*   **架构优雅**：将复杂的长程任务拆解为状态维护、执行和审计，逻辑清晰。
*   **显式状态**：通过外部化 JSON 状态，解决了 LLM 容易遗忘上下文约束的顽疾。
*   **针对性监督**：并非全量审计，而是针对“关键动作”审计，平衡了成本与安全。
*   **即插即用**：不需要微调模型，通过提示工程即可在现有 LLM 上实现。

### 8. 不足与局限
*   **Token 成本**：Navigator 每一轮都需要重新生成完整的 JSON 状态，对于极长对话，Token 消耗和延迟会增加。
*   **状态表示限制**：目前的全局状态基于预定义的 JSON 模式，可能无法涵盖所有复杂业务场景的细微差别。
*   **解析脆弱性**：依赖于模型输出完美的 JSON 格式，如果 Navigator 产生格式错误，可能导致整个流程中断。
*   **Director 依赖**：系统的最终安全性高度依赖于 Director 的能力，如果 Director 本身也产生幻觉，架构的防御力会下降。
