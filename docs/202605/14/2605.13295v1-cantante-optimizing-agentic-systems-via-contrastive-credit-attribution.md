## 摘要
基于大语言模型（LLM）的多智能体系统在软件工程、预测建模和检索增强生成等复杂的现实任务中表现出了强大的性能。然而，由于评分仅在系统层面可用，而控制智能体行为的参数是局部的，因此其配置的自动化仍然是一个结构性挑战。我们认为，优化这些系统从根本上说是一个信用分配（credit-assignment）问题。因此，我们提出了 CANTANTE，这是一个通过对比同一查询下多个联合配置的运行结果，将系统级奖励分解为每个智能体更新信号的框架。我们将其应用于提示词优化，将智能体提示词视为可学习的系统参数。我们在编程（MBPP）、数学推理（GSM8K）和多跳问答（HotpotQA）任务上，将 CANTANTE 与 GEPA 和 MIPROv2 进行了对比评估。在这些基准测试中，CANTANTE 在所有评估的优化器中获得了最佳的平均排名，并始终优于未优化的提示词。在 MBPP 上，它比最强的基准模型提高了 18.9 个百分点，在 GSM8K 上提高了 12.5 个百分点，同时推理成本更低。在 HotpotQA 上，它的表现与最强基准模型的差距保持在一个标准差之内。至关重要的是，我们的信用相关性分析证实，归因器产生了有意义的单智能体信号，而不是简单地重复全局系统评分。

## Abstract
LLM-based multi-agent systems have demonstrated strong performance across complex real-world tasks, such as software engineering, predictive modeling, and retrieval-augmented generation. Yet automating their configuration remains a structural challenge, as scores are available only at the system level, whereas the parameters governing agent behavior are local. We argue that optimizing these systems is fundamentally a credit-assignment problem. We therefore introduce CANTANTE, a framework that decomposes system-level rewards into per-agent update signals by contrasting rollouts of multiple joint configurations on the same query. We instantiate it for prompt optimization, treating agent prompts as learnable system parameters. We evaluate CANTANTE against GEPA and MIPROv2 on programming (MBPP), mathematical reasoning (GSM8K), and multi-hop question answering (HotpotQA). Across these benchmarks, CANTANTE achieves the best average rank among all evaluated optimizers and consistently outperforms unoptimized prompts. It improves over the strongest baseline by +18.9 percentage points on MBPP and +12.5 percentage points on GSM8K, while incurring a lower inference cost. It remains within one standard deviation of the strongest baseline on HotpotQA. Crucially, our credit correlation analysis confirms that the attributer produces meaningful per-agent signals rather than echoing the global system score.

### 1. 核心问题与研究动机
*   **核心问题**：如何自动化地优化基于大语言模型（LLM）的多智能体系统（MAS）配置（主要是提示词）。
*   **研究背景**：虽然多智能体系统在复杂任务中表现优异，但其配置通常依赖人工手动调优。
*   **技术痛点**：MAS 优化存在“结构性失配”——**信用分配（Credit Assignment）难题**。系统反馈是全局的（整个工作流的一个标量评分），而待优化的参数是局部的（每个智能体各自的提示词）。直接使用全局评分指导局部更新会导致信号模糊，难以识别具体是哪个智能体表现好或差。

### 2. 方法论：CANTANTE 框架
*   **核心思想**：引入**对比式组内归因（Contrastive In-Group Attribution）**机制。通过在相同查询（Query）下对比多个不同配置组合的执行轨迹，将全局奖励分解为针对单个智能体的局部更新信号。
*   **关键技术流程**：
    1.  **提议（Suggest）**：每个智能体的局部优化器（如 CAPO）提出 $K$ 个候选参数（提示词）。
    2.  **组装与执行**：将各智能体的候选参数配对成 $K$ 个联合配置，在任务查询上运行，产生执行轨迹（Traces）和全局评分。
    3.  **对比归因**：将同一查询下的多个执行轨迹组成对比组。使用一个强大的 LLM 作为“归因器”，分析轨迹差异，为每个智能体在每次运行中的表现分配 $[-1, +1]$ 范围内的信用分。
    4.  **局部更新**：各智能体的局部优化器接收归因后的信用分（而非原始全局分），进行下一轮参数迭代。
*   **优势**：即使不同配置的全局得分相同，归因器也能通过分析中间推理过程的质量差异（如是否引入了有用的逻辑、是否纠正了上游错误）来提供差异化的反馈信号。

### 3. 实验设计
*   **数据集与场景**：
    *   **MBPP (编程)**：测试条件循环结构（Planner-Executor-Validator）。
    *   **GSM8K (数学推理)**：测试并行集成结构（3个 Executor + 1个 Consensus）。
    *   **HotpotQA (多跳问答)**：测试检索增强结构（Retriever-Reader-Synthesizer-Checker）。
*   **基准（Benchmark）与对比方法**：
    *   对比了 **GEPA**（基于反思的进化算法）和 **MIPROv2**（基于贝叶斯优化的提示词优化器）。
    *   设置了 **Initial**（未优化的初始提示词）作为底线。
*   **模型选择**：下游任务智能体使用 Qwen3-30B，优化器和归因器使用 GPT-OSS-120B。

### 4. 资源与算力
*   **硬件**：使用 NVIDIA L40 GPU。
*   **算力消耗**：
    *   主实验耗时约 **100 GPU 小时**。
    *   消融实验耗时约 **30 GPU 小时**。
    *   总计约 **130 GPU 小时**。
*   **优化预算**：每个任务限制在 1000 万个 Token 的优化预算内。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了三种截然不同的 MAS 拓扑结构（循环、并行、检索增强），每组实验运行 3 个随机种子并计算标准差。
*   **消融实验**：非常充分。作者进行了包括“身份归因（直接传回全局分）”、“不同归因提示词”、“不同归因模型”、“不同局部优化器（如 EvoPrompt）”以及“不同数据集规模”在内的多项消融实验。
*   **客观性**：通过信用相关性分析（Spearman 相关系数）验证了归因信号的真实有效性，证明了性能提升并非偶然。

### 6. 主要结论与发现
*   **性能卓越**：CANTANTE 在所有评估的优化器中平均排名第一。在 MBPP 上比最强基准提升 **18.9%**，在 GSM8K 上提升 **12.5%**。
*   **推理高效**：优化后的系统在提升准确率的同时，推理时的 Token 消耗在 MBPP 和 GSM8K 上反而有所下降，实现了“更强且更省”。
*   **归因精准**：分析显示归因器能准确识别系统瓶颈（如 MBPP 中的 Validator 角色对最终得分贡献最大），并能克服位置偏见等干扰。
*   **鲁棒性**：即使在小样本（开发集仅 75 个样本）情况下，CANTANTE 依然能保持高效的优化能力。

### 7. 优点与亮点
*   **创新的归因机制**：将多智能体强化学习（MARL）中的信用分配思想引入 LLM 提示词优化，解决了 MAS 优化的核心结构性矛盾。
*   **对比学习的妙用**：通过组内对比（In-group comparison），在奖励信号稀疏或存在“平台期”时仍能提供有效的优化梯度。
*   **框架通用性**：CANTANTE 作为一个框架，可以兼容各种局部优化算法（如 CAPO, EvoPrompt 等），具有很强的扩展性。

### 8. 不足与局限
*   **拓扑固定**：该方法目前仅优化智能体的参数（提示词），不涉及 MAS 拓扑结构（如增加/删除智能体或改变连接方式）的自动设计。
*   **归因器能力边界**：归因质量高度依赖于所使用的 LLM 归因器。如果归因器模型能力弱于下游智能体，可能会产生错误的引导信号。
*   **上下文限制**：随着智能体数量增加，多个执行轨迹的拼接可能会超出归因器 LLM 的上下文窗口限制，影响其在大规模 MAS 中的扩展性。
