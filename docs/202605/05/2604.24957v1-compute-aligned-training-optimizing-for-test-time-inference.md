## 摘要
扩展测试时计算已成为增强大语言模型（LLM）性能的一种强大机制。然而，标准的后训练范式，如监督微调（SFT）和强化学习（RL），优化的是基础策略下单个样本的似然性，这与依赖聚合或过滤输出的测试时程序产生了不一致。在这项工作中，我们提出了计算对齐训练（Compute Aligned Training），旨在将训练目标与测试时策略相对齐。通过将推理策略概念化为基础策略上的算子，我们推导出了新的损失函数，这些函数能在应用上述策略时最大化性能。我们针对常见的测试时策略，在 SFT 和 RL 中实例化了这些损失函数。最后，我们提供了实证证据，表明这种训练方法相比标准训练显著提升了测试时的扩展性。

## Abstract
Scaling test-time compute has emerged as a powerful mechanism for enhancing Large Language Model (LLM) performance. However, standard post-training paradigms, Supervised Fine-Tuning (SFT) and Reinforcement Learning (RL), optimize the likelihood of individual samples under a base policy, creating a misalignment with test time procedures that rely on aggregated or filtered outputs. In this work, we propose Compute Aligned Training, which aligns training objectives with test-time strategies. By conceptualizing inference strategies as operators on the base policy, we derive new loss functions that maximize performance when said strategies are applied. We instantiate such loss functions for SFT and RL across common test time strategies. Finally, we provide empirical evidence that this training method substantially improves test time scaling over standard training.

*   **核心问题**：当前的训练范式（如 SFT 和 RL）与实际部署时的推理策略存在**目标失配（Misalignment）**。
*   **研究背景**：现代 LLM 部署常采用“搜索”策略（如 Pass@N、多数投票 MajVote、Best-of-N 验证），即生成多个候选答案并进行筛选或聚合。
*   **动机**：标准训练优化的是单次输出的似然度，这会导致模型在简单样本上过度自信，并压缩了搜索所需的探索空间（策略崩溃）。论文旨在提出一种通用的框架，让模型在训练时就“意识到”推理时会进行多次采样和聚合。

### 2. 方法论：计算对齐训练（CAT）
*   **核心思想**：将推理策略视为作用于基础策略 $\pi_\theta$ 上的**算子 $T$**。训练目标不再是最大化单次输出的概率，而是最大化经过算子转换后的“有效分布” $\tilde{\pi}_\theta$ 的性能。
*   **关键技术细节**：
    *   **统一算子框架**：定义 $\tilde{\pi}_\theta = T(\pi_\theta, \phi)$，其中 $\phi$ 是推理超参数（如采样预算 $N$）。
    *   **对角梯度近似（Diagonal Gradient Approximation）**：为了避免计算完整的雅可比矩阵（计算量巨大），论文证明了可以通过对标准梯度进行标量加权 $w$ 来近似 CAT 梯度。
    *   **梯度缩放因子 $w$**：
        *   **Pass@N**：权重随基础概率 $p$ 增加而单调递减，防止模型在已能稳过的问题上浪费容量。
        *   **多数投票（MajVote）**：权重集中在决策边界（即“胜负手”处），鼓励模型在竞争激烈的样本上建立共识。
        *   **Best-of-N (RL)**：权重取决于样本在奖励分布中的分位数，诱导“胜者全得”的动力学。
*   **算法流程**：在 SFT 或 RL（如 GRPO）过程中，根据当前样本的概率和预设的推理策略，计算权重 $w$，并将其乘到原始梯度上。

### 6. 主要结论与发现
*   **性能提升**：CAT 训练的模型在应用推理策略时，性能显著优于标准模型。例如，在 MATH 任务中，Pass@64 性能提升了约 7.8%。
*   **搜索税（Search Tax）**：对齐高预算推理的模型在单次输出（Pass@1）上往往会有轻微下降，但随着推理计算量的增加，其性能上限远高于标准模型。
*   **方差与稳定性**：在 RL 中，策略对齐梯度可能导致方差爆炸，论文提出的“对数加权”和“批次归一化”技术能有效稳定训练。
*   **通用性**：该方法在蛋白质序列设计中同样有效，能帮助模型跨越奖励函数的“死亡谷”，找到全局最优解。

### 7. 优点
*   **理论优雅**：将推理策略抽象为算子，提供了一个统一的数学框架来处理各种搜索算法。
*   **计算高效**：通过梯度加权实现，不需要在训练时进行昂贵的多次采样（相比于其他需要完整推理模拟的方法）。
*   **即插即用**：可以轻松集成到现有的 SFT 和 RL 流程中。

### 8. 不足与局限
*   **策略复杂度限制**：目前仅验证了 Pass@N、MajVote 等简单策略，对于更复杂的搜索（如 MCTS 或多步 Agent 策略），其算子形式可能难以推导。
*   **对角近似偏差**：虽然理论证明了其安全性，但在某些极端非竞争性策略下，忽略非对角项可能会导致梯度信号丢失。
*   **超参数敏感性**：如多数投票中的阈值 $k$ 需要预先设定或动态估计，对最终效果有较大影响。
