## 摘要
基于大型语言模型（LLM）构建的多智能体系统需要许多协调选择，这些选择难以事先确定：调用哪个技能协议、哪个智能体角色应执行子任务、每个角色绑定哪个模型、角色之间如何交互、何时使用检索或验证，以及何时完全省略某个步骤。这些选择与任务机制和操作约束相互作用，因此静态流水线和一次性模型比较仅提供了设计空间的一个有限视角。本文介绍了AgensFlow，这是一个开源框架，将多智能体协调视为部分可观测条件下的在线策略学习问题。该框架使协调决策可观测且可从重复轨迹中学习，而不是将技能、角色、模型、拓扑和评估选择视为固定的流水线设计。AgensFlow在两个语料库上进行了评估：分布式系统事件任务和安全咨询任务。评估显示了三个主要结果：在协调密集型任务类别上，学习型路由比固定管道基线达到了更高质量的工作点；skip:X将拓扑压缩隔离为基础架构的一个有意义部分；热启动策略图可以在保持平台质量的同时减少探索成本。总体而言，结果表明可学习、可审计的路由可以改善静态布线下的协调密集型多智能体工作流。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：将协调看作一个顺序决策过程，在每个路由步骤根据当前**折叠任务签名**(folded signature)从动作池中选择动作，完成后根据轨迹奖励更新策略图。签名表示状态抽象，使相似任务共享学习经验。
- **关键技术细节**：
  - **折叠任务签名 $s_t$**：由任务特征提取的**状态抽象**，包括：
    - 机制标签 $\ell_t$（如 straightforward、evidence_heavy、ambiguous 等，通过规则检测获得）；
    - 二进制 handoff 掩码 $z_t \in \{0,1\}^7$（标识 goal、subproblem、evidence、critique、verification、draft_answer、merged_answer 是否已填充）；
    - 桶化信念估计 $\beta(\hat{c}_t, \hat{u}_t, \hat{r}_t, \hat{e}_t)$（估计正确性、不确定性、矛盾风险、证据充分性）。
  - **动作空间** $a_t \in A(s_t)$ = {invoke(k, m)} ∪ {skip:X} ∪ {terminate}。其中 invoke(k, m) 表示绑定模型 m 调用技能协议 k；skip:X 表示跳过计划中尚未执行的单元 X；terminate 不直接被策略选择，而是由评估器或预算耗尽触发。
  - **奖励函数**：$r(t) = w_q Q(t) - w_c C(t) - w_\rho \rho(t)$，其中 $Q(t)$ 是**RelativeJudge**质量分（同类任务相对排序），$C(t)$ 是归一化 token 成本，$\rho(t)$ 是重试/失败惩罚。默认权重 $w_q=1.0$, $w_c=0.3$, $w_\rho=0.15$。
  - **策略学习**：使用**可靠性感知 UCB1** 选择动作：  
    $\text{score}(s,a) = \bar{r}(s,a) + c_s \sqrt{\frac{\log(N_s+1)}{N_{s,a}}} - \lambda f(s,a)$  
    其中 $\bar{r}$ 是平均回报，$N_s$ 是签名访问次数，$N_{s,a}$ 是动作访问次数，$f(s,a)$ 是失败率，$c_s$ 随着 $N_s$ 增加而退火（从 1.4 降至 0.5 下限），$\lambda=0.5$。未访问过的动作获得无穷大分数强制探索。
- **算法流程**（每任务生命周期）：
  1. 预检（API key、配额等）；
  2. 计算任务特征，通过规则检测生成机制标签，并计算 handoff 和信念 → 折叠为签名 $s_t$；
  3. 在每个路由步骤，UCB1 从合法动作集中选择动作（invoke 或 skip:X）；
  4. 执行选定动作（如调用 LLM 单元），更新 handoff 和信念状态；
  5. 重复 3~4 直到终止条件（评估器标记完成/预算耗尽/无合法动作/治理违规）；
  6. 轨迹完成后，RelativeJudge 将其与同类任务对比给出质量分 $Q(t)$，结合运营惩罚计算最终奖励 $r(t)$；
  7. 将奖励备份回策略图中对应的（签名, 动作）边，更新统计量。

### 3. 实验设计：使用的数据集/场景、benchmark、对比方法

- **语料库**：
  - **分布式系统事件语料**（60个任务）：涵盖 Paxos/Raft、逻辑时钟、Gossip、CRDT、一致性哈希、故障检测等分布式系统概念与事件推理。用于生成初始策略图（443节点）。
  - **合成安全咨询语料**（60个任务）：覆盖 6 个供应商对，设计为 8 个场景类（C1~C8），包括单文档、跨文档多源、模糊外域、协调密集型等。用于评估跨域迁移。
- **基准（baseline）**：固定 7 单元流水线（一次通过，无学习）。
- **对比方法（实验臂）**，均在 60 个安全咨询任务上评估：
  1. **固定基线 (baseline)**：固定流水线，单次运行；
  2. **无skip消融 (no-skip ablation)**：8 轮次，学习变体池但强制禁止 skip:X；
  3. **主实验 (main)**：8 轮次，启用 skip:X，冷启动；
  4. **热启动 (warm-start)**：8 轮次，启用 skip:X，从分布式系统策略图（443节点）初始化。
- **审计**：所有实验臂的最终轨迹被**跨族三法官**（Claude Haiku 4.5 + GPT-5.4 mini + Qwen3.6-flash）重新打分，并以相对排序分数作为审计依据。

### 6. 论文的主要结论与发现

1. **学习型路由优于固定基线**：主实验臂在三法官审计下，协调密集型类（C3、C7、C8）显著优于固定流水线（+0.18、+0.13、+0.16），而过程类类（C1、C5）接近持平。
2. **skip:X 是有效的拓扑压缩**：无skip消融臂相比主实验臂 token 成本大幅增加（+94%），且质量未超过主实验，证明跳过策略是性能提升的关键组成部分。
3. **热启动可降低探索成本并保持质量**：从分布式系统策略图热启动后，在主实验的 8 个 epoch 中 token 成本降低约 21%，三法官审计下平台质量增益（+0.012）虽小于单法官结果，但证明热启动确实减少了探索开销，且跨法官审计说明质量优势部分来自单法官偏差。
4. **奖励鲁棒性需要审计**：单法官信号下热启动看起来质量提升更大，但跨法官审计表明效果更温和，说明奖励信号本身也是协调问题的一部分，必须纳入系统设计而非外部评估步骤。
5. **所学习的策略并非简单地选择最便宜的变体**：系统在 7 个类上改变了默认的 solver_cot_haiku，根据每类结构选择不同的证据模板或简洁模板，体现出适应性。

### 7. 优点

- **方法创新**：将多智能体协调定义为在线策略学习问题，动作空间包括 skill×model 和拓扑跳过（skip:X），使得系统能够动态适应任务特征，无需手工编写流水线。
- **可审计性**：策略图持久化且可检查，每个（签名、动作）边记录了访问次数、平均奖励、方差、失败率，便于运营人员分析决策依据。
- **奖励鲁棒性设计**：引入跨法官平均、轴分解、置信加权，使奖励信号可观测、可审计，降低单法官偏差的影响。
- **迁移能力**：展示了在不同领域（分布式系统→安全咨询）之间策略图可以作为热启动先验，减少探索成本并保持性能。
- **开源实现**：提供了完整代码仓库，实验可复现。
