## 摘要
近期长程智能体任务的进展主要依赖于通过更强的模型、更好的工具和更有效的脚手架来扩展单个智能体。相比之下，对于横向扩展的理解则少得多：多个对等智能体，全部针对同一任务，是否可以在不依赖明确角色分工或工作流编排的情况下，成为额外的能力来源。我们研究了这个问题，并提出了AgentFugue，一个围绕共享推理中心构建的集体推理框架。当对等智能体并行探索同一任务时，中心记录每个智能体已确定、尝试或排除的简明笔记，并使每个智能体能够以对其当前搜索有用的形式，选择性地访问其他智能体发现的内容。这种设计将原本孤立的轨迹转变为可重用中间推理的互联生态，无需集中规划。我们将该中心实例化为一个即插即用的通信层，使用监督微调和端到端强化学习进行训练。在我们研究的具有挑战性的长程设置中，AgentFugue在强基线上取得了改进。我们的结果表明，集体推理可以将对等智能体系统的横向扩展转变为能力增益的独特来源，而不仅仅是消耗更多计算的一种方式。

## Abstract
Recent progress on long-horizon agentic tasks has been driven largely by scaling up individual agents through stronger models, better tools, and more effective scaffolding. In contrast, much less is understood about scaling out: whether multiple peer agents, all targeting the same task, can become an additional source of capability without relying on explicit role specialization or workflow orchestration. We study this question and propose AgentFugue, a collective reasoning framework built around a shared reasoning hub. As peer agents explore the same task in parallel, the hub records concise notes on what each agent has established, attempted, or ruled out, and enables each agent to selectively access what other agents have discovered in a form useful for its current search. This design turns otherwise isolated trajectories into a connected ecology of reusable intermediate reasoning without requiring centralized planning. We instantiate the hub as a plug-in communication layer, trained with supervised fine-tuning and end-to-end reinforcement learning. Across the challenging long-horizon settings we study, AgentFugue improves over strong baselines. Our results suggest that collective reasoning can turn scaling out peer agent systems into a distinct source of capability gains, rather than merely a way of spending more compute.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：构建一个**共享推理中心（Shared Reasoning Hub）**，作为外部通信层而非集中规划器。该中心记录每个智能体完成的交互片段（episode）的简明笔记（episode note），并允许其他智能体根据当前需求**选择性阅读**（intent-driven reading）队友的原始片段和合成摘要。这样，原本孤立的并行轨迹形成一个可重用中间推理的互联生态。

- **关键技术细节**：
  - **问题形式化**：定义任务的目标知识空间 \(K^*(x)\)；智能体 \(i\) 的局部轨迹 \(\tau_{i,t}\) 积累子空间 \(K(\tau_i)\)。多个智能体同时探索，期望碎片互补。
  - **共享推理中心（Hub）**：
    - **写入（Episode Writing）**：当智能体的本地上下文达到写预算（write budget，默认64K tokens），将当前片段压缩为笔记 \(z_{i,e} = M_{\text{write}}(\epsilon_{i,e})\)，然后笔记保留在上下文，原始片段被回收。
    - **上下文结构**：智能体工作上下文包含自身的已完成笔记、队友笔记和当前活跃交互，保持有界。
    - **意图驱动读取（Intent-driven Reading）**：智能体基于当前上下文判断需要队友信息时，发出请求 \((q_{i,t}, E_{i,t})\)，其中 \(q\) 是信息需求意图，\(E\) 是选择队友的笔记索引。Hub检索原始片段并合成读取结果 \(r_{i,t} = M_{\text{read}}(q_{i,t}, \{\epsilon_{j,e}\})\)，返回聚焦的证据或指导。
  - **Hub优化**：使用Qwen3.5-9B作为骨干，分两个阶段训练：
    1. **监督微调（SFT）**：用教师模型生成参考笔记和读取结果，通过LM损失训练 write 和 read 两个实例。
    2. **组相对策略优化（GRPO）**：在多智能体闭环中（冻结任务智能体），对Hub的输出进行端到端强化学习，奖励包含任务成功和长度奖励（鼓励短搜索路径）。优化目标使用组相对优势。
  - **扩展形式**：
    - **同质团队**：所有智能体使用相同骨干和配置，增益来自跨轨迹交互。
    - **异质团队**：智能体使用不同骨干（如Qwen、DeepSeek、GLM、Kimi），引入系统性偏差互补。

## 6. 论文的主要结论与发现

- **AgentFugue在所有三个基准上一致优于所有多智能体基线**：在Qwen骨干上平均准确率54.4%，比Swarm-Multi-Agent高5.4%，比Naive-Multi-Agent高5.9%；在DeepSeek骨干上达到65.0%，分别高7.4%和7.7%。增益贯穿检索、推理和广度任务。
- **同质团队缩放有效**：增加相同骨干的对等智能体可以提高每智能体性能，在N=5时饱和；团队级聚合（Pass@N、BoN等）持续提升。Hub将探索从独立搜索转向共享协调：每智能体搜索/访问调用减少，记忆调用增加。
- **异质团队缩放更有效**：不同骨干模型互补，弱模型获得两位数提升，强模型也获益（如Kimi+K2.5+K2.5提升约2-3%）。记忆访问频率远高于同质团队，显示异质性鼓励更深协作。
- **聚合策略不影响缩放趋势**：所有轻量级聚合规则（Pass、BoN、WMV等）均随团队大小提升，Pass-BoN差距提示更智能的聚合可以进一步利用hub输出。
- **hub上下文窗口大小是关键设计参数**：最优为32K（在N=2时），太小（16K）或太大（≥64K）都会降低性能。说明过大上下文会稀释hub的注意力。
- **失败模式**：附录E指出共享内存可能导致确认偏差（over-anchoring），当笔记强调某个候选的局部唯一性时，下游智能体可能忽略硬约束。需要结构化候选状态和硬失败门控。
