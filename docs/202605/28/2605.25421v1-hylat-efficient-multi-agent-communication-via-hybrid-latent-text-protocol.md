## 摘要
通信协议设计是基于大语言模型的多智能体系统中的核心挑战。现有的单通道方法面临固有的通信三难困境：基于文本的方法可解释但冗长，而基于潜在空间的方法高效但不透明且局限于单向工作流。受多通道通信理论启发，我们提出HyLaT，一种混合潜在-文本通信协议，通过潜在通道传输精细的认知信号以实现高效，同时以自然语言表达简洁的关键信号以保持可解释性和精确性。我们引入了一个两阶段训练框架，结合单智能体混合生成学习和多智能体交互协同训练，使智能体能够在多轮交互中生成和解释混合消息。实验表明，HyLaT在显著降低通信开销的同时保持竞争性的任务性能，并在各种设置下展现出强大的泛化性和鲁棒性。

## Abstract
Communication protocol design is a central challenge in large language model-based multi-agent systems. Existing single-channel approaches face an inherent communication trilemma: text-based methods are interpretable but verbose, while latent-space methods are efficient but opaque and limited to unidirectional workflows. Inspired by multi-channel communication theory, we propose HyLaT, a hybrid latent-text communication protocol that transmits elaborate cognitive signals through a latent channel for efficiency, while expressing concise critical signals in natural language to preserve interpretability and precision. We introduce a two-stage training framework combining single-agent hybrid generation learning and multi-agent interactive co-training, enabling agents to generate and interpret hybrid messages across multiple rounds of interaction. Experiments demonstrate that HyLaT reduces communication overhead significantly while maintaining competitive task performance, with strong generalization and robustness across diverse settings.

## 2. 论文提出的方法论

### 核心思想
- 设计**双通道通信协议**：潜在通道（连续向量序列）传递高密度中间推理；文本通道传递可读的最终结论。
- 通过**两阶段训练框架**使模型具备生成和解释混合消息的能力。

### 关键技术细节
- **双通道响应格式**：输出由 `[<bot> z1...zk <eot>]`（潜在部分）和 `[y1...ym]`（文本部分）拼接而成。潜在部分包含 k 个连续向量（k=6），通过自回归生成；文本部分为标准自然语言。
- **通道专业化**：训练数据包含详尽的推理过程（用于潜在通道）和简洁答案（用于文本通道），使模型自动学会分工，无需显式路由标注。
- **阶段 1：单智能体混合生成学习**
  - 同时进行两个前向传播：混合分支（自回归生成潜在向量后接文本答案）和纯文本分支（教师强制生成完整推理+答案）。
  - 引入**跨通道对齐损失**：在最终答案前的目标位置（如 "is"），将纯文本分支的隐藏状态对齐到混合分支的隐藏状态（仅传递梯度，停止梯度更新纯文本分支）。
  - 总损失：`L_stage1 = α L_hybrid + β L_text + γ L_align`，其中 α=1, β=1, γ=20。
- **阶段 2：多智能体交互共训练**

  - 对每个智能体计算阶段1的损失并聚合平均：`L_stage2 = (1/N) Σ (α L_hybrid^(i) + β L_text^(i) + γ L_align^(i))`。
  - 仅对最后一轮进行监督（因中间轮次答案可能错误）。

## 6. 论文的主要结论与发现

- HyLaT 在保持与全文本方法相当任务性能的同时，显著降低通信开销（token 数节省 10.6 倍，时间节省 6.8 倍 vs 最有效文本方法；3.6 倍 token 节省、2.5 倍时间节省 vs 最有效潜在方法）。
- 双通道设计优于任何单通道变体：纯文本导致效率低，纯潜在导致准确率下降和格式错误。
- 两阶段训练缺一不可：缺阶段1导致潜在表示不稳定，缺阶段2导致无法处理其他智能体的潜在信号。
- HyLaT 与 NL 智能体兼容，可混合部署；对潜在向量噪声鲁棒（σ≤0.5 性能稳定）。
- 可扩展性好：随智能体数量或交互轮次增加，HyLaT 的 token 消耗增长缓慢（而 NL 和 TextFullT 急剧增长）；通过扩大训练数据可进一步提升性能。
- 在社会模拟（重复信任游戏）中，HyLaT 以极低通信成本达到与 TextFullT 相当的社会推理能力。
- 注意力分析显示：潜在通道确实承载了跨智能体信息交换；PCA 表明潜在向量按数据集语义聚类。

## 7. 优点

- **创新性**：首次提出双通道通信协议，从结构上解决通信三难困境，而非像现有工作仅做压缩。
- **完整性**：从协议设计、训练框架到推理流程完整实现，并提供了可解释性分析。
- **实用性强**：保留文本通道使得与现有 NL 智能体兼容，便于增量部署；对噪声鲁棒。
- **实验充分**：对比基线全面（训练无关/训练相关、文本/潜在）；消融实验验证每个组件；在不同骨干、不同规模、不同任务上验证泛化性。
- **效率提升显著**：在 1B 模型上达到接近单智能体的运行时间，且性能优于大部分培训无关方法，接近全文本训练方法。

## 8. 不足与局限

- **模型规模限制**：所有实验仅在 1B~3B 参数模型上进行，未验证更大模型（如 7B、13B）是否同样受益。
- **部署规模限制**：HyLaT 的生成过程与现有推理框架（如 vLLM）不兼容，难以在大规模或长程多智能体场景中评估。
- **文本通道多样性受限**：训练数据主要为 QA 式辩论数据，且强制使用固定答案提示，限制了文本表达的行为范围；缺乏更丰富的社交或创意通信数据。
- **潜在通道不可解释**：虽保留文本输出，但潜在向量本身仍是黑箱；如何平衡潜在的可解码性与性能是开放问题。
- **训练数据构建开销**：阶段2需要借助更强模型（GPT-5）通过模拟生成数据，成本较高且依赖专有模型。
- **潜在偏差**：训练数据主要基于特定推理数据集，可能存在领域偏置；域外泛化测试中部分任务（如 ARC）提升有限。
