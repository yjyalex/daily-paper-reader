## 摘要
训练大语言模型作为检索增强推理代理通常将强化学习与从更强模型蒸馏出的SFT冷启动相结合。然而，这一范式忽略了两个基本因素：子技能之间的依赖结构，以及蒸馏并非获取能力的唯一途径。我们通过Plan研究这一方法，这是一种用于多跳检索的结构化代理行为，它在执行任何检索之前将问题分解为有序的子问题，从而使每一步搜索都能锚定到预先设计的子问题上，而不是因之前检索到的部分相关文档的影响而漂移。然而，在涵盖3B到14B参数的三个模型家族中，我们发现相同的奖励信号会引发质上不同的RL失败模式。这一现象表明，成功训练不仅取决于奖励设计，还依赖于模型特定的可行性条件：足够的初始熵、训练稳定性和必备的子技能。受此启发，我们提出了一种自举范式，其中小规模种子模型生成过滤后的轨迹，从而在任何目标模型中激活Plan，无需从外部更强模型进行蒸馏。我们的流程在每一个测试模型中都激活了Plan，并在多跳问答基准测试中持续优于竞争基线。

## Abstract
Training large language models as retrieval-augmented reasoning agents typically combines reinforcement learning with an SFT cold start distilled from a stronger model. However, this paradigm overlooks two fundamental factors: the dependency structure among sub-skills, and the possibility that distillation is not the only route to capability acquisition. We study this through Plan, a structured agentic behavior for multi-hop retrieval that decomposes a question into ordered sub-questions before any retrieval is performed, so that each search step can be anchored to a pre-designed sub-question instead of drifting under the influence of partially relevant documents retrieved earlier. However, across three model families spanning 3B to 14B parameters, we find that an identical reward signal induces qualitatively different RL failure modes. This phenomenon indicates that successful training hinges not only on reward design but also on model-specific feasibility conditions: sufficient initial entropy, training stability, and prerequisite sub-skills. Motivated by this, we propose a self-bootstrapping paradigm in which a small-scale seed model generates filtered trajectories that activate Plan in any target model, eliminating the need for distillation from an external stronger model. Our pipeline activates Plan across every tested model and consistently outperforms competitive baselines on multi-hop QA benchmarks.

  - **蒸馏并非唯一路径**：SFT冷启动数据必须来自更强模型吗？能否通过模型自身生成？
- 作者以**规划（Plan）** 行为作为研究载体，这是一种先分解问题为有序子问题再执行检索的结构化代理行为，能有效防止检索轨迹漂移。
- 核心动机：探究子技能依赖如何影响RL训练成败，以及能否用自举（self-bootstrapping）替代外部蒸馏。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想：Plan Before Search
- 模型在检索之前先生成一个全局计划（`<plan>`块），将多跳问题分解为有序子问题（Step 1, Step 2,...）。
- 之后每一步检索都锚定到对应的子问题，按顺序执行`<think>` → `<search>` → `<refine>`，最后给出答案`<answer>`。
- 相比纯粹反应式（reactive）方法，Plan避免了因部分相关文档累积导致的漂移。

### 2.2 轨迹结构
- 轨迹格式：`<plan>` → ( `<think>` → `<search>` → `<documents>` → `<refine>` )+ → `<answer>`。
- 检索到的`<documents>`在训练时被掩码不参与损失计算。

### 2.3 奖励设计（Plan-Aware Reward）
总奖励 R = R_ans + I[R_ans=0]·R_aux，其中R_aux = λ_fmt·R_fmt + λ_plan·R_plan。
- **结果奖励 R_ans**：预测答案与真实答案的F1分数。
- **格式奖励 R_fmt**：检查标签完整性（所有必需标签成对出现）及`<search>`前必须有`<think>`。得分0, 0.5, 1。
- **计划奖励 R_plan**：计算每个子问题p_k与对应`<think>`内容t_k的token级F1平均 Salign。采用阈值整形：若Salign > δ（δ=0.25），则R_plan=1；否则R_plan=Salign。防止模型直接复制计划内容。
- 辅助奖励仅在答案F1=0时生效，不干扰已部分正确轨迹的梯度。

### 2.4 自举训练管道（Self-Bootstrapping）
- **种子模型构造**：使用Qwen2.5-3B-Base，先经AutoRefine式RL学习精炼（refine），再在此基础上学习规划（plan）。此模型满足可行性条件（足够初始熵、训练稳定、有必备子技能）。
- **轨迹生成与过滤**：种子模型在训练集上生成Plan轨迹，经硬过滤（格式正确、答案cover-EM匹配）和软过滤（搜索轮次、查询多样性、精炼密度加权得分）后，按步数分桶采样得到4500条高质量轨迹。
- **激活目标模型**：任何目标模型（base或instruct，不同家族）先在这些轨迹上SFT（3个epoch），再在完整Plan奖励下进行RL精炼（200步）。

## 6. 论文的主要结论与发现

1. **Plan有效防止漂移**：在检索前分解问题，使每一步搜索锚定到子问题，显著提升多跳QA性能（MH-Avg提升约0.025~0.045）。
2. **直接RL训练Plan存在三种可行性条件**：足够的初始熵（低熵模型如指令调优模型会“先验崩溃”）、训练稳定性（高熵模型如Qwen3-4B-Base后期熵爆炸）、必备子技能（必须先掌握refine，否则P→R顺序训练崩塌）。
3. **自举可替代蒸馏**：一个满足条件的小种子模型（Qwen2.5-3B-Base）生成过滤轨迹后，SFT+RL可在任意目标模型中激活Plan能力，且性能优于来自72B模型的蒸馏数据（因分布内对齐更好）。
4. **奖励设计的重要性**：Plan奖励（阈值δ=0.25）是必要的，且阈值可防止复制collaepse；格式奖励确保结构。
