## 摘要
强化学习提升了LLM的推理能力，但PPO/GRPO通常使用固定的裁剪和解码温度，这使训练变得脆弱且需要大量调参。我们提出自适应组策略优化（AGPO），一种对GRPO的无评论家改进，利用组级统计量同时控制更新幅度和探索。AGPO使用共享的探针衍生统计状态来驱动两个控制器：(i)自适应裁剪，根据奖励离散度和偏度、探针投票熵、策略熵以及逐步KL漂移设置信任区域大小；(ii)双向自适应温度采样，根据相对于运行基线的中心不确定性，在基础温度周围加热或冷却解码。在九个英文和中文数学/STEM基准测试中，使用AGPO训练的Qwen2.5-14B在相同生成token预算下优于PPO/GRPO，在GSM8K上达到67.3%，在MATH上达到40.5%。优势可迁移到Llama-3-8B和Gemma-2-9B，消融实验证实两个模块互补。我们的实现公开在https://github.com/wandugu/paper_agpo。

## Abstract
Reinforcement learning improves LLM reasoning, but PPO/GRPO typically use fixed clipping and decoding temperature, which makes training brittle and tuning-heavy. We propose Adaptive Group Policy Optimization (AGPO), a critic-free refinement of GRPO that uses group-level statistics to control both update magnitude and exploration. AGPO uses a shared probe-derived statistical state to drive two controllers: (i) adaptive clipping, which sets the trust-region size from reward dispersion and skewness, probe vote entropy, policy entropy, and step-wise KL drift; and (ii) bidirectional adaptive temperature sampling, which heats or cools decoding around a base temperature according to centered uncertainty relative to a running baseline. On nine English and Chinese math/STEM benchmarks, Qwen2.5-14B trained with AGPO outperforms PPO/GRPO under the same generated-token budget, reaching 67.3% on GSM8K and 40.5% on MATH. Gains transfer to Llama-3-8B and Gemma-2-9B, and ablations confirm both modules are complementary. Our implementation is publicly available at https://github.com/wandugu/paper_agpo.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：PPO/GRPO 在 LLM 强化学习训练中通常使用固定的裁剪半径（ε）和解码温度（τ），导致训练过程脆弱、需要大量超参数调优（heavy tuning），且容易出现探索-利用不匹配（exploration–exploitation mismatch）和收敛不稳定问题（brittle convergence）。
- **背景**：强化学习（RL）能显著提升 LLM 的推理能力，但现有方法如 PPO（含价值 critic）和 GRPO（无 critic）均依赖固定超参数。在训练过程中，奖励离散度、答案分歧、策略熵和 KL 漂移会大幅变化，固定设置无法适应，导致早期更新不足、后期优化不稳定。
- **整体含义**：AGPO 旨在通过引入轻量级、无 critic 的自适应机制，利用训练过程中可观测的组级统计量动态调整裁剪半径和采样温度，以提升训练稳定性和最终推理性能，并降低调参成本。

## 2. 论文提出的方法论：核心思想、关键技术细节

### （1）核心思想
AGPO 是 GRPO 的无 critic 改进，核心是 **“一个探针，两个控制器”（one probe, two controllers）**：
- **探针阶段**：对每个 prompt 使用基础温度 τ_base 采样一组 rollout（G=8），计算组级统计量（奖励离散度、偏度、答案分歧等）。
- **两个控制器**：
  - **自适应裁剪（Adaptive Clipping）**：根据奖励离散度（ˆσ）、偏度（|κ̃₃|）、探针投票熵（E_probe）、策略熵（H(π)）和逐步 KL 漂移（̂D_step KL）动态设置裁剪半径 ε_adaptive，以控制更新幅度。
  - **双向自适应温度采样（ATS）**：根据相对运行基线的中心化不确定性（centered uncertainty）调整温度 τ_t，可高于或低于基础温度，以控制探索强度。

### （2）关键技术细节（公式）

**组归一化优势**：  
A_i = (r_i - μ_r) / (ˆσ + ε_A)，其中 ˆσ 为组内奖励标准差，μ_r 为组均值。

**自适应裁剪公式**：  
ε_adaptive = clip( ε_base * H(π_θ_old) * (1 + δ E_probe) / (1 + α ˆσ + ζ |κ̃₃| + γ ̂D_step KL), ε_min, ε_max )  
- 分母中 ˆσ、|κ̃₃|、̂D_step KL 越大，则 ε 越小（收紧信任区域）；分子中 H 和 E_probe 越大则 ε 越大（扩大更新范围）。  
- ̂D_step KL 来自上一更新步（EMA 平滑），避免循环依赖。

**自适应温度采样（ATS）**：  
U_t = w_r ˆσ + w_e E_probe + w_k |κ̃₃|（原始不确定性）  
中心化：êU_t = U_t - Ū_t（Ū_t 为 EMA 基线）  
τ_t = clip( τ_base · (1 + λ êU_t), τ_min, τ_max )  
- 当当前批次不确定性高于基线（êU_t > 0）时，加热温度增加探索；当低于基线时，冷却温度促进利用。

### （3）训练流程（两阶段）
- 每个更新步：先以 τ_base 采样探针组 → 计算统计量 → 更新 Ū_t → 设置 τ_t 和 ε_adaptive → 以新 τ_t 重采样训练 rollout → 用 AGPO 目标（式 7）更新策略 → 记录 ̂D_step KL 用于下一步。
- 探针 rollout 仅用于统计量，不参与梯度更新。

## 7. 优点：方法或实验设计上的亮点

- **无 critic 设计**：避免训练价值网络，节省内存和计算开销，简化训练流程。
- **共享探针统计**：一次探针同时驱动两个控制器，高效且机制一致。
- **双向温度调节**：基于基线中心化，允许温度“加热”或“冷却”，比单侧调节更灵活。
- **自适应裁剪基于多信号**：综合奖励离散度、偏度、策略熵、探针熵和 KL 漂移，信号互补，缓解单一信号的噪声。
- **实验设计严谨**：
  - 严格控制生成 token 预算（包括探针 token），公平对比。
  - 三种种子，报告均值方差。
  - 跨模型验证（3 种不同架构）。
  - 消融实验系统，包括 leave-one-out 和鲁棒性测试。
  - 训练时间对比和吞吐量统计，体现实用性。
- **开源实现**：提供 GitHub 仓库，便于复现和扩展。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制

1. **依赖组级统计量的稳定性**：G=8 的组粒度小，时刻估计（如偏度、方差）可能噪声较大。虽然 MAD/IQR 做了缓解，但极端小样本下统计量可信度低。
2. **奖励设计敏感**：论文提到“reward misspecification, reward hacking, and extremely sparse or adversarially noisy rewards remain open directions”。AGPO 的统计量直接依赖奖励，若奖励信号有偏差或稀疏，自适应效果可能受影响。
3. **仅测试推理任务**：所有 benchmark 都是数学/STEM 推理，未涉及对话、摘要、代码生成等更广泛的 RLHF 场景，泛化性有待验证。
4. **未分析超参数敏感性**：论文固定了多个权重（α, γ, δ, ζ, w_r, w_e, w_k, λ 等），未讨论这些超参数在不同任务或模型上的调整难度，可能仍需一定调参。
5. **训练预算较小**：300M token 相对于大型 LLM 的 RL 训练（如 GPT-4 级别）仍显少量，大规模训练下的表现未知。
6. **未对比更多自适应方法**：如 RLOO, Reinforce Leave-One-Out 等变体，但文中已覆盖了主要流派。
7. **理论分析较简略**：Theorem 1 为 idealized 证明，实际使用的 cosine 学习率调度、非凸目标等未完全纳入，可靠性有限。
8. **单节点实验**：2×A800 在分布式训练下可能遇到通信或扩展问题，未评估多节点表现。
