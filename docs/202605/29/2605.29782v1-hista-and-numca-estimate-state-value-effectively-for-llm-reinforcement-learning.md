## 摘要
强化学习通过奖励信号直接优化大语言模型的行为。在经典强化学习中，准确的状态价值估计对于稳定训练至关重要，但在LLM后训练中仍是一个未充分探索的挑战。本文引入状态价值估计基准（SVEB）来评估现有强化学习框架中的状态估计，并发现PPO等标准方法中的评论家会退化为粗糙的组平均基线。针对这一问题，我们提出两种技术：Numca利用数值跨度作为可分级里程碑进行状态价值估计；Hista则利用LLM的隐藏状态作为表示，对不相交的轨迹及其回报进行加权平均。大量实验表明，这两种方法都能提供更准确的状态价值估计，并在不同强化学习算法和模型大小下提升训练性能，且不会显著增加计算开销。

## Abstract
Reinforcement learning (RL) refines large language models (LLMs) by directly optimizing model behavior through reward signals. While accurate state value estimation is critical for stable training in classical RL, it remains an underexplored challenge in LLM post-training. In this work, we introduce the State Value Estimation Benchmark (SVEB) to assess state estimation within existing RL frameworks and show that critics in standard approaches like PPO collapse to a coarse group-average baseline. To address this, we propose two techniques: Numca, which leverages numerical spans as gradable milestones for state value estimation, and Hista, a framework that uses LLM's hidden states as representation to weighted average disjoint rollouts and their return. Extensive experiments demonstrate that both methods yield more accurate state value estimates and enhance training performance across different RL algorithms and model sizes without incurring significant computational overhead.

## 1. 核心问题与整体含义（研究动机与背景）

在LLM强化学习后训练中，准确的状态价值估计对稳定策略优化至关重要。然而，现有方法存在严重缺陷：经典PPO中的批评网络（critic）在LLM场景下退化，给出的状态价值估计与GRPO的组平均基线几乎无差异，无法提供细粒度的信用分配。该问题源于标准TD（时序差分）学习在LLM生成任务中难以捕捉中间状态的语义信息。作者提出两个方法——**Numca**（基于数值里程碑的启发式方法）和**Hista**（基于隐藏状态表示的通用框架），以在不显著增加计算开销的前提下提升状态价值估计准确性，从而改善RL训练效果。

## 2. 方法论

### 核心思想
- **Numca**：将数学推理视为稀疏奖励的目标达成问题。利用数值（整数、小数、分数等）作为自然里程碑，将包含相同里程碑集合的轨迹分组，并用这些轨迹的最终奖励平均值作为该里程碑状态的估计值。
- **Hista**：利用LLM的最后一层隐藏状态作为状态表示。通过MinDistance（最小距离）操作度量两个状态的相似度，选取k个最近邻轨迹，用它们最终奖励的概率加权平均（权重为距离的倒数）估计当前状态的价值。理论证明（定理5.5）该加权估计的绝对偏差不超过组平均基线。

### 关键技术细节
- **Numca算法**：定义数值模式集合P；对每个轨迹解析里程碑序列；维护字典存储每个里程碑抽象状态出现的次数和累计奖励；最终按平均奖励赋值给对应宏动作中的所有token。实现简单，用字典存储，计算量可忽略。
- **Hista流程**：①提取终态的最后一层隐藏状态；②通过指数移动平均（EMA）和压缩间隔φ将长序列压缩为短嵌入序列；③均匀采样间隔δ得到离散状态集合；④计算当前状态与组内其他状态间MinDistance；⑤选取k个最近邻，以距离倒数作为权重加权平均最终奖励，得到状态价值估计值。
- **理论保证**：定理5.2建立MinDistance与状态获得相同最终奖励概率的相关性；定理5.5证明概率加权估计器偏差不大于平均估计器。

## 6. 主要结论与发现

1. **PPO批评家退化**：在LLM RL中，PPO的批评家估计值基本等于组平均奖励，无法提供细粒度指导。SVEB证实了这一缺陷。
2. **Numca有效性**：在数学推理领域，Numca通过数值里程碑实现更精确的状态价值估计，显著提升RL训练稳定性和下游准确率（表3）。
3. **Hista通用性与理论保证**：Hista在SVEB的所有领域均优于GRPO和PPO，达到MCS@2水平的精度。定理5.5提供理论保证。将其集成到DAPO、CSIPO等算法中，在多数模型大小和数据集上带来持续提升（表5、6、7、8）。
4. **跨模型泛化**：Hista在Qwen2.5和Qwen3系列、指令微调与基础模型上均有效，说明隐藏状态表示质量足够。
5. **效率**：Hista几乎不增加训练时间，GPU内存与DAPO相当或更低。

## 7. 优点

- **方法创新**：将LLM隐藏状态作为状态表示用于信用分配，首次从状态价值估计角度系统解决LLM RL问题。
- **理论支撑**：定理5.2和5.5不仅给出相关性证明，还提供偏差改进的充分条件，使方法具有可解释性。
- **计算高效**：无需额外训练批评家或进行蒙特卡洛树搜索，仅利用已有轨迹的隐藏状态和奖励，轻量级实现。
- **实验全面**：覆盖多种任务、模型规模、算法和数据集，消融详尽，结论可靠。
- **通用性**：Hista不依赖领域先验，在数学、科学、编程、通用QA上均有效；Numca则专门针对数学推理极其高效。

## 8. 不足与局限

- **Numca的领域限制**：依赖数值模式识别，仅适用于含有明确数值里程碑的任务（如数学推理），在科学QA等无数值场景中退化为GRPO。
- **Hista的理论假设**：定理5.2依赖于“答案可从单一token解析”的假设（Assumption A.1），对开放编程任务不严格成立。尽管实验显示Hista在编程任务上仍有效，但缺乏对应理论支撑。
- **性能波动**：在部分实验（如Qwen3-1.7B + Hista）中出现下降，作者归因于RL随机性和O.O.D.问题，说明方法可能对模型和超参数敏感。
- **实验覆盖不足**：未涉及Agentic RL（如工具使用任务）、视觉语言模型、或更大规模模型（如>70B）。Agentic RL仅在未来工作提及。
- **实际部署挑战**：Hista需要对每个训练步计算所有状态间MinDistance，虽然在论文设定下高效，但对于超长序列（如>100K token）可能仍带来延迟，论文未给出极端条件性能。
- **SVEB基准局限**：参考值通过MCS@20获得，并非真值；且MCS@3的精度高于所有方法，说明状态价值估计仍有上升空间。
