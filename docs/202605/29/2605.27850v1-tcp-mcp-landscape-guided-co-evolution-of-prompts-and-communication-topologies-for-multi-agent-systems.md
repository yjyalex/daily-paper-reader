## 摘要
有效的多智能体系统无法通过孤立地选择提示或通信图来设计。智能体行为取决于其接收的信息，而通信边的有用性则取决于接收智能体如何解释和使用该信息。我们提出**TCP-MCP**（面向多智能体协作问题求解的拓扑耦合提示方法），这是一种将智能体提示和通信拓扑作为统一基因组进行搜索的协同进化框架。TCP-MCP使用初始化时的景观探测来校准早期搜索行为，然后依赖帕累托前沿诊断在三个目标（任务性能、令牌成本和结构复杂度）下调整探索行为。在所有方法均使用相同DeepSeek-V3.2骨干模型的情况下，TCP-MCP在MMLU-Pro、MMLU和GSM8K上分别达到82.66%、89.96%和96.61%的准确率。在这三个基准测试中，它始终优于自动图生成基线，并达到与辩论式系统相当的准确率，同时在其报告的工作点上使用的令牌数量比这些系统少多达5.69倍。这些结果表明，在受控评估中，协同进化提示和通信结构为实现成本感知和任务自适应的多智能体系统设计提供了一条实用途径。

## Abstract
Effective multi-agent systems cannot be designed by selecting prompts or communication graphs in isolation. Agent behavior depends on the information an agent receives, while the usefulness of a communication edge depends on how the receiving agent interprets and uses that information. We propose \textbf{TCP-MCP} (Topology-Coupled Prompting for Multi-Agent Collaborative Problem-Solving), a co-evolution framework that searches agent prompts and communication topologies as a unified genome. TCP-MCP uses an initialization-time landscape probe to calibrate early search behavior, and then relies on Pareto-front diagnostics to adapt exploration under three objectives: task performance, token cost, and structural complexity. Using the same DeepSeek-V3.2 backbone across all methods, TCP-MCP achieves 82.66\%, 89.96\%, and 96.61\% accuracy on MMLU-Pro, MMLU, and GSM8K, respectively. Across the three benchmarks, it consistently outperforms automated graph-generation baselines and achieves competitive accuracy relative to debate-style systems, while using up to 5.69$\times$ fewer tokens than those systems at the reported operating points. These results show that jointly evolving prompts and communication structure provides a practical route to cost-aware and task-adaptive multi-agent system design in controlled evaluations.

- **核心问题**：当前多智能体系统（MAS）的设计通常是碎片化的，即单独设计智能体的提示词（Prompts）或通信拓扑结构（Topologies）。作者指出，这种分离式设计忽略了二者之间本质上的强耦合关系。智能体的行为不仅取决于其自身的提示，还强烈依赖于其接收到的来自其他智能体的信息内容，而这些信息的流向正是由通信拓扑决定的。
- **整体含义**：因此，多智能体系统的构建应被视为一个**系统级的设计问题**。最有效的系统不能通过孤立地优化提示或拓扑来实现，而需要将二者作为相互依赖的关键变量，进行**联合优化**。本文正是基于这一核心洞察，提出了一个协同进化框架。

### 2. 方法论：核心思想、关键技术细节与算法流程
- **核心思想**：提出了 **TCP-MCP** 框架，将智能体的提示词和通信拓扑编码为一个统一的“基因组” `M = (G, P)`，并进行协同进化搜索。目标是同时优化三个目标：任务准确性（最大化）、令牌成本（最小化）和结构复杂度（最小化）。
- **关键技术细节**：
    - **统一基因组表示**：每个候选系统 `M` 包含一个通信有向图 `G` 和节点级角色-提示词分配 `P`。提示词是**拓扑感知**的，即在运行时根据节点的入邻居动态构建，即使存储的提示模板不变，行为也会因拓扑变化而改变。
    - **初始化景观探针**：在搜索开始时，计算适应度距离相关性（FDC），作为一个粗略的景观信号来校准早期搜索行为。
    - **协同进化算子**：
        - **结构交叉**：交换父代基因组的拓扑子图，而非提示片段。采用“提示最小继承（PMI）”原则，保留受体节点的提示，仅移植供体的子图和相关提示。
        - **变异算子**：包括局部拓扑编辑（增/删边、增/删节点）、角色变异（改变节点角色并重新生成提示）、以及由停滞信号触发的激进变异（引入更大的结构变化）。
    - **多目标环境选择**：使用改进的NSGA-II算法，结合动态可行性约束（保证候选系统有一定准确率）、基于令牌成本和结构复杂度的分箱生态位保持（防止种群聚集）、以及结构配额（避免系统退化到过于简单的结构）。
    - **帕累托前沿诊断与自适应控制**：通过监测超体积、间距、前沿间隙等诊断指标，计算一个响应指数，动态调整变异率和注入策略，平衡探索与利用。
- **算法流程**：初始化时通过拉丁超立方采样生成多样化的初始种群 → 评估并计算FDC → 进行多代进化（每代包括结构交叉、变异、帕累托选择）→ 监控帕累托前沿诊断指标进行自适应控制 → 最终输出一个在性能、成本和复杂度之间权衡的帕累托精英集。
