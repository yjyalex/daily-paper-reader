## 摘要
基于大语言模型（LLM）构建的多智能体系统需要诸多难以预先确定的协调选择：应调用哪个技能协议、哪个智能体角色应执行子任务、应为每个角色绑定哪个模型、角色间应如何交互、何时使用检索或验证、以及何时完全省略某个步骤。这些选择与任务机制和操作约束相互交织，因此静态流水线和一次性模型比较只能提供设计空间的有限视角。本文介绍了AgensFlow，这是一个开源框架，它将多智能体协调视为部分可观测条件下的在线策略学习问题。该框架使协调决策变得可观测，并能从重复轨迹中学习，而不是将技能、角色、模型、拓扑和评估选择视为固定流水线设计。 AgensFlow在两个语料库上进行了评估：分布式系统事件任务和安全咨询任务。评估显示了三个主要结果：在协调密集型类别上，学习到的路由比固定流水线基线达到了更高质量的工作点；skip:X将拓扑压缩隔离为基板的一个有意义组成部分；热启动策略图可以在保持平台质量的同时降低探索成本。总体而言，结果支持可学习、可审计的路由能够相较于静态布线改进协调密集型多智能体工作流程。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

- **背景**：现有MAS（如AutoGen、CAMEL）通过对话式协调，但协调决策与对话内容纠缠，难以重用和审计。工具学习（如Toolformer）和模块化专家协调（如RIM）提出了可学习的选择机制，但未统一处理技能、模型、拓扑和评估的联合交互。同时，智能体系统难以从最终答案直接评分，需要相对轨迹评估。

### 2. 方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：将多智能体协调视为**部分可观测序列决策过程**，学习一个基于**折叠签名**的在线策略。通过状态抽象（折叠签名）共享跨任务的价值估计，使用可靠性感知的UCB1算法选择动作，并通过轨迹级奖励备份更新策略图。
- **关键技术细节**：
  - **状态折叠签名** \( s_t = \phi(x_t, h_t, b_t) = (\ell_t, z_t, \beta(\hat{c}_t), \beta(\hat{u}_t), \beta(\hat{r}_t), \beta(\hat{e}_t)) \)，其中 \(\ell_t\) 是规则检测的任务机制标签（如straightforward, evidence_heavy, ambiguous, contradictory, high_risk, exploratory），\(z_t\) 是7维二进制交接掩码，\(\beta\) 是分桶的信念估计（正确性、不确定性、矛盾风险、证据充分性）。
  - **动作空间**：\( a_t \in A(s_t) = \{\text{invoke}(k,m)\} \cup \{\text{skip}:X\} \cup \{\text{terminate}\} \)，其中 \((k,m)\) 是技能协议绑定模型，skip:X 是省略仍计划中的可选单元。terminate 由条件隐式触发，不由策略直接选择。
  - **奖励函数**：\( r(t) = w_q Q(t) - w_c C(t) - w_\rho \rho(t) \)，\(Q(t)\) 是相对轨迹质量分数（RelativeJudge），\(C(t)\) 是归一化令牌成本，\(\rho(t)\) 是重试/失败惩罚。默认权重 \(w_q=1.0, w_c=0.3, w_\rho=0.15\)。
  - **策略图学习**：基于可靠性感知的UCB1变体：\(\text{score}(s,a) = \bar{r}(s,a) + c_s \sqrt{\frac{\log(N_s+1)}{N_{s,a}}} - \lambda f(s,a)\)，探索系数 \(c_s\) 随签名访问次数退化（从1.4降至0.5），失败惩罚 \(\lambda=0.5\)。
- **算法流程**（文字描述）：
  1. 预检（API密钥、配额等）。
  2. 计算任务特征，规则检测器产生机制标签，结合交接掩码和信念估计折叠为签名。
  3. 在每个路由步骤，可靠性UCB1从合法动作集中选择动作（调用某个单元或skip:X）。
  4. 执行动作，更新交接状态和信念，重新计算签名，重复直到终止触发。
  5. 轨迹完成后，RelativeJudge对同类任务轨迹进行相对评分（多轴分解），与成本、重试组成混合奖励，备份到策略图中。
- **额外机制**：RelativeJudge支持跨评判者平均（不同模型家族）、多轴分解（目标达成、依据、协调、恢复）、置信加权（基于评判者间标准差）。

### 3. 实验设计：数据集/场景、基准、对比方法

  - **安全咨询语料**（60个任务）：合成数据，覆盖8个场景类：C1程序性、C2单文档、C3跨文档多供应商、C4合成、C5语料外歧义、C6程序性派生、C7缓解正确性、C8跨供应商配对。用于评估转移和最终对比。
- **基准**：固定7单元流水线（baseline），单次运行无学习。
- **对比方法**（四个臂）：

### 6. 主要结论与发现
1. **学习路由优于固定流水线**：main arm在平台期（epoch 6-8）RJ(3j)为0.817，比baseline的0.749高0.068，token仅增加15%。在协调密集型类（C3、C7、C8）提升最显著（+0.13~+0.18），程序性类（C1、C5）持平或小幅下降。
2. **skip:X拓扑压缩有效**：no-skip ablation导致token增加94%（25,198 vs 12,960），说明跳过决策对成本控制很重要；main arm通过学习跳过有效压缩成本并提升质量。
3. **热启动降低探索成本，保持平台质量**：warm-start在平台期末令牌比冷启动少约10%，全运行少约21%，且RJ(3j)平台期0.829略高于冷启动0.817（+0.012），但单评判者显示更大提升（+0.052）。跨家庭审计显示单评判者信号存在偏差，但成本节约是评判者无关的。
4. **奖励信号鲁棒性重要**：单评判者与三评判者结果差异表明，评判者选择可改变结论。因此奖励审计应是协调基板的一部分。

### 7. 优点
- **方法论创新**：将MAS协调形式化为在线策略学习，使用折叠签名实现状态抽象，使跨任务经验共享；将拓扑选择（skip:X）纳入学习动作空间，而非固定流水线。

- **开放式框架**：代码开源，实验可复现，层间依赖清晰，易于扩展。
