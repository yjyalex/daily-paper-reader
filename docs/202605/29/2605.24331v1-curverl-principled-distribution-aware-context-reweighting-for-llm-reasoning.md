## 摘要
上下文或提示级重加权已成为带验证奖励的强化学习（RLVR）中提升大语言模型推理能力的核心算法杠杆，但关于最优加权的构成原则仍缺乏理解。我们通过将提示重加权表述为定义在通过率函数空间中的效用泛函的泛函导数，弥补了这一空白，从而生成了一个统一的最优性框架，涵盖了现有方案，包括REINFORCE和GRPO。基于此最优性框架，我们提出了一种基于分位数坐标变换的分布感知提示重加权方法，称为CurveRL，其中分配给每个提示的权重不取决于通过率的绝对值，而是取决于其在学习动态中反映通过率分布结构的排名和密度。跨多个基准的广泛实验表明，我们提出的CurveRL持续优于GRPO和其他RLVR基线。我们的研究将上下文分布控制确定为分析和设计提示重加权RLVR算法的一个原则性轴。代码已发布于https://github.com/zhyzmath/CurveRL。

## Abstract
Context or prompt-level reweighting has emerged as a central algorithmic lever in Reinforcement Learning with Verified Rewards (RLVR) for improving the reasoning capability of large language models, yet the principle determining what constitutes an optimal weighting remains poorly understood. We address this gap by formulating prompt reweighting as a functional derivative of a utility functional defined in the pass-rate function space, yielding a unified optimality framework that accommodates existing schemes, including REINFORCE and GRPO. Building on this optimality framework, we propose a distribution-aware prompt reweighting approach, called CurveRL, based on a quantile coordinate transform, in which the weight assigned to each prompt depends not on the absolute value of pass rates but on its rank and density to reflect the distributional structure of the pass rates in the learning dynamics. Extensive experiments across multiple benchmarks demonstrate that our proposed CurveRL consistently outperforms GRPO and other RLVR baselines. Our study identifies context-distribution control as a principled axis for analyzing and designing prompt-reweighted RLVR algorithms. The code is released in https://github.com/zhyzmath/CurveRL.

### 方法论：核心思想、关键技术细节、算法流程

#### 核心思想
- 将RLVR策略梯度更新统一写作：  
  `∇θ J(θ) = E_{x~d0}[w_θ(x) ∇_θ p_θ(x)]`，其中 `p_θ(x)` 为提示 x 的通过率。
- 定义**效用泛函** U_θ(p_θ(·))，其**泛函导数**（functional derivative）给出最优权重 `w^*_θ(x) = δU_θ/δp_θ(x)`。
- 现有方法（REINFORCE、GRPO、MaxRL）对应**逐点效用泛函**（点变换），权重仅依赖于 `p_θ(x)` 的绝对值，导致**权重坍塌**（weight collapse）——在 `p≈0` 或 `p≈1` 时权重无法区分不同提示。
- CurveRL 引入**分布感知效用泛函**：`U_θ(F_ref) = E_{x~d0}[ψ(F_ref(p_θ(x)))]`，其中 `F_ref` 是参考通过率分布的CDF，`ψ` 为扭曲函数（如 `ψ(u)=log u`）。权重变为 `w_θ(x) = f_ref(p_θ(x))/F_ref(p_θ(x))`，同时利用了通过率的**密度**（f_ref）和**分位数排名**（1/F_ref），反映分布几何。

#### 关键技术细节
- **参考分布估计**：使用滑动窗口（size = t0 × |B|）存储最近 t0 步的通过率，通过直方图估计 `f_ref` 和 `F_ref`。这使权重**数据驱动、自适应**，且避免了与当前 CDF 重合导致的退化（否则 `F_θ(p_θ(x))~Uniform` 会丢失信息）。
- **梯度估计**：对每个提示 x，采样 N 个响应，计算经验通过率 `ˆp`，赋权 `wt(ˆp)=ˆf_ref(ˆp)/ˆF_ref(ˆp)`，并用 `ˆp` 作为组级基线降低方差（`ˆg(x) = (1/N) Σ wt(ˆp)(ri - ˆp) ∇log π`）。
- **算法1** 描述了每步更新：1) 从滑动窗口估计密度与CDF；2) 对每个提示采样并计算权重；3) 仅保留 `ˆp∈(0,1)` 的样本（非零梯度）；4) 更新窗口。

#### 理论支撑
- **定理1**：任意逐点权重 w(p) 可唯一对应一个先验分布 `F_ref`（通过 `F_ref(p)=exp(-∫_p^1 w(t)dt)`）。CurveRL 不预设先验，而是动态估计。
- **命题2**：分布感知效用与当前策略效用之差受 1-Wasserstein 距离控制，证明方法具有几何感知性。
- **单调校准不变性**（Proposition 3）：对通过率施加任意单调递增变换，CurveRL 梯度保持不变，而逐点方法不满足。

### 实验设计

#### 数据集与任务
- **训练集**：POLARIS-53K（约5.3万个数学推理提示）。
- **测试基准（8个）**：
  - 主表：AIME 2025、BeyondAIME、HMMT 02/25、HMMT 02/26、MATH-500。

#### 对比方法
- **Base**：预训练模型直接采样（无RL训练）。
- **GRPO**：组相对策略优化（人口近似权重 `1/√(p(1-p))`）。
- **MaxRL**：最大似然RL（权重 `1/p`）。
- **CurveRL**：本文方法。

#### 指标
- **pass@1**（单次正确率）、**pass@k**（k次采样中至少一次正确的概率，k=2,…,1024/512）。通过 1000 次 bootstrap 估计 pass@k。

#### 模型与配置
- 基础模型：Qwen3-1.7B-Base、Qwen3-4B-Base。
- 训练超参数：batch size=256、rollouts per prompt=8、学习率1e-6、无KL惩罚/熵奖励、温度1.0、训练1000步。
- 评估：温度0.6、top-p 0.95，每个提示采样2048（1.7B）/1024（4B）次以降低方差。

### 主要结论与发现

1. **CurveRL 在 pass@1 和 pass@k 上一致优于 GRPO 和 MaxRL**，尤其在困难基准（如AIME、HMMT）上优势明显，且随k增大或模型规模增大优势扩大。
2. **缓解了 pass@k 退化问题**：以往方法（GRPO、MaxRL）可能损害 pass@1 或 pass@k 相比Base模型下降，而CurveRL在大多数设置下超过Base模型。
3. **分布感知权重自适应**：滑动窗口估计使权重动态调节，早期更多关注低通过率提示（高风险），后期适应通过率分布右移，避免了权重坍塌。
4. **扩宽了推理边界**：CurveRL将更多不可解提示转化为可解，并提升 medium 难度提示的过半数正确率（majority voting），有利于测试时扩展。
5. **提供原则性框架**：提示重加权应被视为上下文分布控制，最优性由效用泛函选择（风险偏好）决定，而非单一MLE原则。

### 优点

1. **理论创新**：首次将提示重加权形式化为泛函导数，统一了REINFORCE、GRPO、MaxRL等算法，并提示了其局限性（权重坍塌）。
2. **方法新颖**：引入分位数坐标变换和滑动窗口估计，使权重既具有分布几何感知又自适应数据，比逐点变换更鲁棒。
3. **实验扎实**：覆盖8个基准、两个模型、广泛k值、消融与机制分析，且通过共享框架（禁用KL/熵）隔离了混杂因素，公平性高。
4. **可解释性**：权重动态可视化清晰展示了分布运输过程（通过率密度右移）、权重自适应变化。
5. **实用性**：代码开源，超参数少（仅滑动窗口大小t0需调节），可直接替换现有RLVR训练中的权重部分。
