## 摘要
现代大语言模型强化学习（LLM RL）系统将采样生成与策略优化分离。这两个阶段预期会产生完全一致的 token 概率。然而，实现上的差异可能导致它们在相同的模型权重下为同一序列分配不同的值，从而引发训练-推理不匹配（TIM）。TIM 难以检测，因为它与离策漂移和常见的稳定机制交织在一起。在这项工作中，我们在一个零不匹配诊断设置（VeXact）中隔离了 TIM，并证明了微小的 token 级数值差异可以独立导致训练崩溃。我们进一步表明 TIM 改变了实际的优化问题，并确定了一系列可以缓解 TIM 的补救措施。我们的结果表明，TIM 并非良性的数值噪声，而是一种系统级的扰动，在分析 LLM RL 稳定性时应将其视为一阶因素。

## Abstract
Modern LLM RL systems separate rollout generation from policy optimization. These two stages are expected to produce token probabilities that match exactly. However, implementation differences can make them assign different values to the same sequence under the same model weights, inducing Training-Inference Mismatch (TIM). TIM is difficult to inspect because it is entangled with off-policy drift and common stabilization mechanisms. In this work, we isolate TIM in a zero-mismatch diagnostic setting (VeXact), and show that small token-level numerical disagreements can independently cause training collapse. We further show that TIM changes the effective optimization problem, and identify a set of remedies that could mitigate TIM. Our results suggest that TIM is not benign numerical noise, but a systems-level perturbation that should be treated as a first-order factor in analyzing LLM RL stability.

### 1. 核心问题与研究动机
*   **核心问题**：在现代 LLM RL 系统中，采样生成（Rollout）和策略优化（Training）通常由不同的引擎（如 vLLM 和 FSDP）执行。由于算子实现、计算顺序、并行策略及数值精度的差异，即使模型权重完全相同，两个阶段对同一序列计算出的 Token 概率也会存在微小差异。
*   **研究动机**：这种看似微小的数值噪声（TIM）往往与算法本身的离策漂移（Off-policy drift）交织在一起，导致 RL 训练极不稳定甚至崩溃。研究旨在隔离 TIM 的影响，探究其是否是导致训练失败的独立诱因，并评估现有缓解方案的有效性。

### 2. 方法论：VeXact 诊断环境
为了隔离 TIM，作者开发了 **VeXact**，一个实现“零不匹配”的轻量级采样引擎：
*   **核心思想**：通过统一采样和训练的底层实现，消除基础设施层面的数值偏差。
*   **关键技术细节**：
    *   **算子对齐**：使用与训练引擎（FSDP）完全一致的 HuggingFace 模型实现和内核。
    *   **批次不变内核（Batch-invariant Kernels）**：固定 GPU 内核的 Tiling（分块）和 Reduction（归约）顺序，避免因 Batch Size 改变或原子加法（Atomic Add）导致的非确定性数值结果。
    *   **确定性注意力机制**：禁用 FlashAttention 中的 KV 分割（KV splitting）以确保计算顺序一致。
*   **诊断指标**：定义 $\delta_t = \log \pi_{train} - \log \pi_{rollout}$ 来量化不匹配程度。

### 6. 主要结论与发现
*   **TIM 是独立诱因**：即使没有其他干扰，微小的 Token 级数值差异（最大可达 1.0 log 概率差）也足以导致 RL 训练崩溃。
*   **失败模式分析**：
    *   **重新计算模式**：由于数值偏差，优势函数加权的损失贡献出现正负失衡，改变了优化方向。
    *   **透传模式**：优化器会利用训练端前向传播中的数值伪影（Artifacts），导致更新后的策略在实际采样中无效（静默退化）。
*   **补丁有效性**：结合“Token 级截断”和“基于校正比率（$r_{corr}$）的序列级拒绝采样”可以最接近零不匹配的基准表现。

### 7. 优点
*   **系统级视角**：首次将 TIM 提升到“一阶干扰因素”的高度，而非视其为良性噪声。
*   **工具贡献**：开源了 VeXact，为社区提供了一个高精度的 LLM RL 诊断工具。
*   **机理清晰**：不仅展示了现象，还从优化目标（Zero-centered loss contribution）的角度解释了为什么微小偏差会放大。

### 8. 不足与局限
*   **任务覆盖面**：主要集中在数学推理任务（AIME），尚不清楚在创意写作或多轮对话等奖励信号更模糊的任务中 TIM 的影响是否同样显著。
*   **性能代价**：为了实现数值一致性，VeXact 牺牲了一定的推理吞吐量（如禁用 KV 分割），在超大规模生产环境中的适用性受限。
*   **补丁调优**：虽然证明了算法补丁有效，但这些补丁（如阈值设置）仍高度依赖经验，缺乏像 VeXact 这样从根本上解决问题的简洁性。
