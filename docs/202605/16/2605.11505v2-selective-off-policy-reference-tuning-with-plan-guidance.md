## 摘要
具有可验证奖励的强化学习有助于推理，但 GRPO 风格的方法在所有采样展开均失败的困难提示词上会陷入停滞。SORT 在不改变展开生成的情况下，为这些失败情况增加了一种修复更新：它从参考解中推导出计划，比较在有无该计划条件下的 token 概率，并为那些在计划引导下变得更具可预测性的 token 赋予更高的权重。这使得全错的提示词能够转化为具有选择性的、结构感知的学习信号，而非简单的统一模仿。在三种骨干网络和八个推理基准测试中，SORT 的表现优于 GRPO 和引导基线，且在较弱的模型上提升最为显著。

## Abstract
Reinforcement learning with verifiable rewards helps reasoning, but GRPO-style methods stall on hard prompts where all sampled rollouts fail. SORT adds a repair update for those failures without changing rollout generation: it derives a plan from the reference solution, compares token probabilities with and without that plan, and gives higher weight to tokens that become more predictable under plan conditioning. This turns all-wrong prompts into selective, structure-aware learning signals instead of uniform imitation. Across three backbones and eight reasoning benchmarks, SORT improves over GRPO and guidance baselines, with largest gains on weaker models.

### 1. 论文的核心问题与整体含义
**研究动机与背景：**
在具有可验证奖励的强化学习（RLVR）中，如 GRPO 算法，模型通过对比同一提示词（Prompt）下的多个采样结果来学习。然而，当面对极难的题目时，模型的所有采样可能全部错误（即“零奖励”或“全错”组）。在这种情况下，组内相对优势为零，导致梯度消失，模型无法从这些最具挑战性的失败中学习。现有的解决方法（如引入外部强模型轨迹或统一的 SFT 修复）往往会导致模型过度模仿表面格式或产生冗长的回答。本文旨在寻找一种既能利用参考答案修复失败，又能保持 token 选择性、避免盲目模仿的机制。

### 2. 论文提出的方法论：SORT
**核心思想：**
SORT（选择性离策参考微调）将全错的提示词视为学习的“前沿”，通过引入“计划引导”（Plan Guidance）来识别参考答案中哪些 token 承载了核心推理结构，并对这些 token 进行重点学习。

**关键技术细节：**
*   **延迟修复（Deferred Repair）：** 在 GRPO 训练期间，将所有采样均失败的提示词及其参考答案存入缓冲区。当缓冲区达到一定大小时，触发一次 SORT 辅助更新。
*   **计划提取（Plan Extraction）：** 利用当前模型从参考答案中提取出一个“推理计划”（包含关键决策、引理、步骤大纲等）。
*   **双重评分机制：** 对参考答案中的每个 token 进行两次概率评估：
    1.  $p_{base}$：仅基于原始提示词的预测概率。
    2.  $p_{plan}$：在提示词中加入“推理计划”作为上下文后的预测概率。
*   **计划显著性（Plan Salience）：** 如果一个 token 在加入计划后概率显著提升，说明该 token 包含了模型当前缺失的关键推理结构。
*   **几何平均权重（Geometric Weighting）：** 结合 DFT（动态微调）的思想，SORT 使用几何平均值作为 token 权重：$\omega = \text{sg}(\sqrt{p_{base} \cdot p_{plan}})$。这种权重既考虑了模型现有的支持度，又放大了计划引导下的关键 token。

### 7. 优点
*   **针对性强：** 专门解决了 GRPO 在困难样本上的“静默”问题。
*   **无需外部教师：** 仅利用模型自身生成的计划进行评分，降低了对超强模型（如 GPT-4）轨迹的依赖。
*   **Token 选择性：** 通过概率对比，自动识别并强化关键推理节点，避免了对平庸 token（如代数运算、格式词）的过度拟合。

### 8. 不足与局限
*   **依赖参考答案：** SORT 需要每个提示词都有经过验证的参考解，在缺乏标注答案的开放域任务中难以应用。
*   **计算开销：** 相比原生 GRPO，增加了计划生成和双重前向传播的计算负担（约 50% 的额外时间）。
*   **任务局限：** 目前仅在数学和逻辑推理任务上验证，对于创意写作或多轮对话等非确定性奖励任务的适用性尚待研究。
*   **计划质量：** 效果受限于模型自身提取计划的能力，如果模型太弱无法生成合理的计划，修复效果会打折扣。
