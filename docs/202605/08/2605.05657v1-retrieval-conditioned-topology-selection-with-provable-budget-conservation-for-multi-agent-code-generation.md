## 摘要
用于代码生成的多智能体大语言模型（LLM）系统面临一个根本性的路由问题：最优编排拓扑取决于待修改代码的结构复杂度，然而现有系统在选择拓扑时并未参考代码库。我们提出了检索引导的自适应编排（RGAO），该架构通过在选择编排拓扑之前从分层代码索引中提取结构复杂度向量来闭合这一环路。RGAO 在 Code-Agent 框架内运行，该框架的子智能体受具有六维预算向量的形式化合同约束。我们的核心贡献是将两个此前独立的研究方向——基于复杂度的 LLM 路由和形式化资源代数——结合起来，从而实现了一个两者单独都无法具备的特性：在检索条件下的动态拓扑选择中实现可证明的预算守恒。具体而言，我们的贡献包括：(1) 一个基于复杂度的拓扑路由器，将代理测量的误路由率从 30.1% 降低到 8.2%；(2) 一个具有结构归纳守恒定理的预算代数；以及 (3) 一个分层代码检索引擎。实证评估表明，该系统实现了亚毫秒级的 DAG 构建和线性树索引可扩展性。

## Abstract
Multi-agent LLM systems for code generation face a fundamental routing problem: the optimal orchestration topology depends on the structural complexity of the code under modification, yet existing systems select topologies without consulting the codebase. We present Retrieval-Guided Adaptive Orchestration (RGAO), an architecture that closes this loop by extracting a structural complexity vector from a hierarchical code index before selecting the orchestration topology. RGAO operates within Code-Agent, a multi-agent framework whose sub-agents are governed by formal contracts with six-dimensional budget vectors. Our headline contribution is the composition of two previously separate lines of work -- complexity-conditioned LLM routing and formal resource algebras -- yielding a property neither admits alone: provable budget conservation under retrieval-conditioned dynamic topology selection. Concretely we contribute: (1) a complexity-conditioned topology router that reduces proxy-measured misrouting from 30.1% to 8.2%; (2) a budget algebra with a structural-induction conservation theorem; and (3) a hierarchical code retrieval engine. Empirical evaluation demonstrates sub-millisecond DAG construction and linear tree-index scalability.

## 1. 核心问题与整体含义
**研究动机：**
当前的多智能体大语言模型（LLM）系统在处理代码生成任务时，通常采用固定的协作拓扑（如流水线或群组）。然而，任务的复杂性往往取决于代码库的结构（如单文件修复 vs. 跨模块重构）。现有的编排器（Orchestrator）大多仅根据用户查询的文本特征进行路由，而不参考代码库本身，导致了严重的效率损失或误路由（Misrouting）。

**核心问题：**
如何在执行代码生成任务前，通过感知代码库的结构复杂度来动态选择最优的智能体协作拓扑，并确保在动态编排过程中资源消耗（预算）是可控且守恒的？

## 2. 方法论
论文提出了 **RGAO（检索引导的自适应编排）** 架构，其核心思想是将代码检索信号引入路由决策，并结合形式化资源代数。

### A. 关键技术细节
1.  **结构复杂度向量 ($c$)：** 从分层代码索引中提取五个维度的信号：
    *   $d_{dep}$：最大依赖深度。
    *   $n_f$：涉及的文件数量。
    *   $n_s$：检索到的符号数量。
    *   $h_t$：树遍历深度。
    *   $\rho_x$：跨模块耦合率。
2.  **拓扑路由器：** 根据向量 $c$ 和查询 $q$，将任务路由至四种拓扑之一：
    *   **FAST PATH**：单体模型处理（简单任务）。
    *   **SUB AGENT**：单一专家智能体。
    *   **MULTI AGENT**：流水线或群组协作。
    *   **DEEP RESEARCH**：多阶段重度检索模式。
3.  **分层检索引擎：** 结合了 LATTICE 路径校准、KohakuRAG 多查询重构和 RepoGraph 依赖扩展，在树状索引上进行混合搜索。

### B. 预算代数与算法
*   **智能体合同 $\langle I, C, T, M \rangle$**：定义了指令、上下文（6维预算向量）、工具集（风险等级）和模型。
*   **6维预算向量**：包括迭代次数、调用次数、Token数、秒数、重试次数和移交次数。
*   **定理 1（预算守恒定理）**：通过结构归纳法证明，只要父节点的预算覆盖子节点预算之和，整个执行树的资源消耗绝不会超过根节点的初始设定。系统在执行前进行 $O(|V|+|E|)$ 的静态验证。

## 6. 主要结论与发现
1.  **路由精度大幅提升：** RGAO 将误路由率从正则基准的 **30.1% 降低至 8.2%**。
2.  **极低的系统开销：** 预算检查开销仅为 **0.65 $\mu$s**，DAG 构建时间小于 **0.01 ms**，远低于 LLM 推理延迟。
3.  **资源安全保障：** 静态验证能有效拦截预算超限的配置，避免了多智能体系统常见的“Token 爆炸”问题。
4.  **检索质量：** 结合子词重叠（TF-IDF）和依赖关系信号对代码检索至关重要。

## 8. 不足与局限
*   **基准测试不完整：** 核心性能数据来自代理环境（Proxy Harness），而非公认的 SWE-bench 官方榜单，存在环境偏差风险。
*   **阈值依赖：** RGAO 的路由决策依赖于手动调优的阈值，虽然经过了网格搜索，但在极端异构的代码库上可能需要重新校准。
*   **语言覆盖：** 结构分析依赖于 Tree-sitter 解析器，对于不支持的编程语言，其复杂度向量会退化。
*   **确定性假设：** 预算守恒定理假设工具成本是确定的，在处理具有高度随机性的工具调用时，只能提供“期望预算”保证。
