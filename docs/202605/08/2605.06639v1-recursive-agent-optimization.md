## 摘要
我们引入了递归智能体优化（Recursive Agent Optimization, RAO），这是一种用于训练递归智能体的强化学习方法。递归智能体能够递归地生成自身的新实例，并将子任务委派给这些实例。递归智能体实现了一种推理时扩展算法，使其能够通过分治法自然地扩展到更长的上下文，并泛化到更难的问题。RAO 提供了一种训练模型以充分利用此类递归推理的方法，教会智能体何时以及如何进行委派和通信。我们发现，以这种方式训练的递归智能体具有更高的训练效率，能够处理超出模型上下文窗口的任务，泛化到比训练任务更难的任务，并且与单智能体系统相比，能够减少实际运行时间。

## Abstract
We introduce Recursive Agent Optimization (RAO), a reinforcement learning approach for training recursive agents: agents that can spawn and delegate sub-tasks to new instantiations of themselves recursively. Recursive agents implement an inference-time scaling algorithm that naturally allows agents to scale to longer contexts and generalize to more difficult problems via divide-and-conquer. RAO provides a method to train models to best take advantage of such recursive inference, teaching agents when and how to delegate and communicate. We find that recursive agents trained in this way enjoy better training efficiency, can scale to tasks that go beyond the model's context window, generalize to tasks much harder than the ones the agent was trained on, and can enjoy reduced wall-clock time compared to single-agent systems.

### 2. 方法论：核心思想与关键技术
*   **核心思想**：训练一个单一的策略 $\pi_\theta$，使其在递归生成的执行树的每个节点上实例化。模型既要学会直接解决任务，也要学会生成有用的子任务委派给“副本”自己。
*   **关键技术细节**：
    *   **递归动作空间**：扩展了智能体的 Python REPL 环境，引入 `launch_subagent(goal, ...)` 异步函数。这允许智能体并行或串行地启动子智能体，并接收结构化返回结果。
    *   **局部节点奖励（Local Node Reward）**：每个节点的奖励由两部分组成：$R = \text{自身任务成功信号} + \lambda \times \text{子智能体平均成功率}$。这种设计提供了密集的信用分配（Credit Assignment），鼓励有效的委派。
    *   **策略优化目标**：在所有深度的任务分布上联合优化。为了防止深度节点过多导致梯度失衡，引入了**深度级逆频率加权（Depth-level Inverse-frequency Weighting）**。
    *   **优势估计**：使用“留一法”（Leave-one-out）基准来计算优势函数，将同一根任务下的不同尝试进行对比，确保估计的无偏性。

### 6. 主要结论与发现
*   **突破上下文限制**：递归智能体能处理远超其训练上下文窗口（如 8K）的任务，因为每个子智能体都有独立的上下文空间。
*   **更强的泛化性**：在简单/中等任务上训练的模型，通过递归能有效泛化到极难（深度达 10 层）的任务，而单智能体基线在此时完全失效。
*   **训练效率更高**：由于递归结构提供了自然的“课程学习”信号，递归智能体比单智能体学习速度更快。
*   **运行速度优势**：在可并行的任务中，递归智能体通过并发执行，实际运行时间比单智能体快达 2.5 倍。

### 7. 优点（亮点）
*   **端到端训练**：不同于以往只在推理时做递归，RAO 真正实现了对委派行为的强化学习优化。
*   **动态执行树**：支持非固定层级的、动态生成的递归结构，灵活性极高。
*   **异步支持**：通过 Python 的 `asyncio` 整合，使智能体能原生支持并发子任务，这在实际应用中极具价值。

### 8. 不足与局限
*   **顺序任务的开销**：在 DeepDive 等强顺序依赖的任务中，递归会显著增加延迟（比单智能体慢 18 倍），因为频繁的上下文切换和子智能体启动带来了额外开销。
*   **奖励依赖**：在缺乏自动验证器的领域（如 Oolong 和 DeepDive），高度依赖 LLM 作为裁判（LLM-judge）来提供子任务奖励，这可能引入偏见或增加训练成本。
*   **同质化假设**：目前主要训练单一模型在树的所有节点上运行，未来可能需要研究不同规模模型（如大模型带小模型）的异构递归协作。
