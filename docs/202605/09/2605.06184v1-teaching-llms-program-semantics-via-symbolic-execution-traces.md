## 摘要
我们介绍了一个基于 SV-COMP 2025 构建的评估框架，包含涵盖五种属性类型（内存安全、溢出、终止性、可达性、数据竞争）的 500 个 C 语言验证任务，并对来自六个家族的 14 个模型进行了评估。我们发现，较高的整体准确率掩盖了一个关键弱点：虽然大多数模型能可靠地确认属性成立，但违规检测的表现差异巨大，且随程序长度增加而急剧下降。为了弥补这一差距，我们利用形式化验证产物进行训练：在通用开源 C 代码上运行 Soteria 符号执行引擎，并使用生成的轨迹对 Qwen3-8B 进行持续预训练。仅使用约 3,000 条缺陷轨迹，结合推理时的思维链（CoT）推理，就将违规检测能力提升了 17 个百分点以上，在评估模型中实现了最均衡的准确率表现之一。在违规检测方面，经过训练的 8B 模型优于未开启思考模式、规模大 4 倍的 Qwen3-32B，且在整体准确率上与之接近。轨迹训练与思维链之间的交互具有超叠加效应：两者单独使用均无法带来显著提升，但结合使用效果显著。这种提升可以迁移到所有五种属性类型，包括训练轨迹未针对的类型。我们的 28 种配置实验证实，性能提升源于轨迹语义而非代码量，且轨迹的筛选和格式至关重要。

## Abstract
We introduce an evaluation framework of 500 C verification tasks across five property types (memory safety, overflow, termination, reachability, data races) built on SV-COMP 2025, and evaluate 14 models across six families. We find that high overall accuracy masks a critical weakness: while most models reliably confirm properties hold, violation detection varies widely and degrades sharply with program length. To close this gap, we train on formal verification artifacts: running the Soteria symbolic execution engine on generic open-source C code and using the resulting traces for continued pretraining of Qwen3-8B. Just ${\sim}$3,000 bug traces combined with chain-of-thought reasoning at inference time improve violation detection by over 17 percentage points, producing one of the most balanced accuracy profiles among evaluated models. On violation detection, the trained 8B model outperforms the 4$\times$ larger Qwen3-32B without thinking and approaches it in overall accuracy. The interaction between trace training and chain-of-thought is superadditive: neither alone provides meaningful gains, but their combination does. Improvements transfer across all five property types, including ones the training traces do not target. Our 28 configurations confirm the gains stem from trace semantics, not code volume, and that trace curation and format matter.

### 论文总结：通过符号执行轨迹向大语言模型教授程序语义

#### 1. 核心问题与整体含义
论文关注的核心问题是：**大语言模型（LLM）在理解底层代码（如 C 语言）语义方面的局限性**。尽管 LLM 在高级语言代码生成上表现出色，但在涉及内存安全、指针算术和未定义行为的 C 语言验证任务中，存在严重的能力不对称：
*   **“确认正确”容易，“检测违规”难**：模型能高效确认属性成立（Property Holds），但在检测属性违规（Property Violated，即找 Bug）时表现极差。
*   **性能随长度骤降**：随着代码行数增加，模型检测漏洞的准确率会急剧下降。
*   **研究动机**：探索是否可以通过形式化验证工具（符号执行）生成的中间产物作为训练数据，来增强模型对程序底层语义的理解。

#### 2. 方法论
核心思想是利用**符号执行（Symbolic Execution）**将程序的隐式语义显式化，并以此训练模型。
*   **数据生成流程**：
    1.  从开源数据集（CodeParrot）中筛选出 100 万个自包含的 C 语言文件。
    2.  运行 **Soteria**（一种快速的符号执行引擎）对这些代码进行分析。
    3.  提取符号执行轨迹（Traces），这些轨迹记录了执行路径、内存状态、分支条件以及导致属性违规的具体逻辑。
*   **关键技术细节**：
    *   **持续预训练（CPT）**：在 Qwen3-8B 基座模型上，使用带有 HTML 结构化标记的轨迹数据进行持续预训练。
    *   **轨迹筛选**：研究发现，仅使用约 3,000 条证明存在“显性 Bug”（Manifest Bugs）的轨迹效果最好。
    *   **推理时思维链（CoT）**：在推理阶段开启模型的“思考模式”（Thinking Mode），让模型在输出最终判定前进行内部推理。

#### 3. 实验设计
*   **Benchmark**：构建了一个包含 **500 个 C 语言验证任务**的评估框架，数据源自 **SV-COMP 2025**（软件验证竞赛）。
*   **属性分类**：涵盖内存安全、溢出、终止性、可达性、数据竞争五大类。
*   **对比方法**：
    *   评估了 14 个主流模型（包括 Claude 4.7、DeepSeek R1/V3.2、GPT-OSS、Qwen3 系列、Mistral 系列等）。
    *   对比了不同的训练配置：仅代码训练、全量轨迹训练、纯文本轨迹训练、人工/模型改写的非正式轨迹训练等。

#### 4. 资源与算力
*   **硬件**：使用了 **8 张 NVIDIA A100 GPU**（40GB 或 80GB 版本）。
*   **训练时长**：针对核心实验（3,208 条轨迹），训练过程非常高效，仅需约 **10 分钟**（单 epoch）。
*   **推理配置**：使用 vLLM 框架进行张量并行推理，最大输出长度设为 8,192 tokens。

#### 5. 实验数量与充分性
*   **实验规模**：论文总共测试了 **28 种不同的训练/推理配置**，涵盖了从数据规模、数据质量、轨迹格式到学习率等多个维度的消融实验。
*   **充分性与客观性**：
    *   实验设计非常严谨，通过对比“仅代码训练”和“代码+轨迹训练”，排除了性能提升仅源于代码暴露量增加的可能性。
    *   每组实验运行 4 次并取平均值，标准差较小，结果具有统计学意义。
    *   使用了独立于训练集的 SV-COMP 任务进行评估，确保了泛化能力的验证。

#### 6. 主要结论与发现
*   **超加性效应（Superadditive Effect）**：持续预训练（CPT）和思维链（CoT）单独使用的提升有限，但两者结合时，违规检测准确率提升了 **17.9 个百分点**。
*   **小数据胜过大数据**：精选的 3,000 条高质量 Bug 轨迹比 34,000 条全量轨迹效果更好，说明信号浓度至关重要。
*   **跨属性泛化**：虽然训练轨迹主要针对内存安全和溢出，但模型在终止性、数据竞争等未见过的属性上也表现出显著提升。
*   **以小博大**：经过训练的 8B 模型在违规检测任务上超越了规模大 4 倍的 32B 基座模型。

#### 7. 优点
*   **全自动且无标注**：利用现成的符号执行工具自动生成训练数据，无需人工标注，具有极强的可扩展性。
*   **格式的重要性**：发现保留工具输出的结构化格式（如 HTML 标签）有助于模型在 CoT 过程中进行逻辑锚定。
*   **深入的失败分析**：对模型错误进行了详细分类（如错过边缘情况、逻辑推导错误等），为后续研究提供了方向。

#### 8. 不足与局限
*   **解析失败率**：开启思考模式后，模型常因推理过长导致输出 Token 耗尽，约 10-14% 的任务无法得出最终判定。
*   **代码规模限制**：目前仅针对 400 行以内的单文件 C 程序，尚未验证在大型多文件项目中的表现。
*   **属性覆盖**：虽然有泛化，但内存安全等复杂属性的绝对准确率仍有提升空间（目前约 47%）。
*   **工具依赖**：性能提升受限于符号执行工具（Soteria）的解析能力和路径搜索深度。
