## 摘要
同策略自我蒸馏（self-OPD）通过让策略在特权上下文下进行自我教学，增强了具有可验证奖励的强化学习（RLVR）。我们发现，当这种引导覆盖整个响应时，全 Token KL 会在大部分冗余位置消耗梯度，并放大特权信息泄露，导致在长程数学训练中出现熵增、推理缩短以及分布外（OOD）性能退化。我们提出了 TRACE（Token-Routed Alignment for Critical rEasoning），它仅在标注者标记的关键片段上进行蒸馏：对正确 rollout 的关键片段进行前向 KL，对局部错误片段进行可选的反向 KL，并对所有剩余 Token 使用 GRPO，且 KL 通道在简短的预热后逐渐退火。我们的分析通过两种效应解释了 TRACE：前向 KL 为学生模型分配不足但受教师支持的 Token 提供了非消失的提升，而片段掩码和衰减则使累积的特权梯度暴露保持在有限范围内。在四个预留的数学基准测试以及 GPQA-Diamond 上，TRACE 相比 GRPO 平均提升了 2.76 个百分点，并在 GPQA-Diamond 上保留了 Qwen3-8B 基座模型的 OOD 分数，而 GRPO 和全 Token self-OPD 基准在这些任务上均出现了退化。在在线自我标注下，收益依然存在（+1.90 个百分点，约为强 API 收益的 69%），这减轻了关于 TRACE 仅仅是引入了外部标注者能力的担忧。在不同规模下，最佳的路由动作取决于基座模型：在 Qwen3-8B 上，它是针对关键片段的前向 KL；而在 Qwen3-1.7B 上，它转向了针对错误片段的反向 KL。

## Abstract
On-policy self-distillation (self-OPD) densifies reinforcement learning with verifiable rewards (RLVR) by letting a policy teach itself under privileged context. We find that when this guidance spans the full response, all-token KL spends gradients on mostly redundant positions and amplifies privileged-information leakage, causing entropy rise, shortened reasoning, and out-of-distribution degradation in long-horizon math training. We propose Token-Routed Alignment for Critical rEasoning (TRACE), which distills only on annotator-marked critical spans: forward KL on key spans of correct rollouts, optional reverse KL on localized error spans, and GRPO on all remaining tokens, with the KL channel annealed away after a short warm-up. Our analysis explains TRACE through two effects: forward KL provides non-vanishing lift to teacher-supported tokens that the student under-allocates, while span masking and decay keep cumulative privileged-gradient exposure finite. On four held-out math benchmarks plus GPQA-Diamond, TRACE improves over GRPO by 2.76 percentage points on average and preserves the Qwen3-8B base OOD score on GPQA-Diamond, where GRPO and all-token self-OPD baselines degrade. Gains persist under online self-annotation (+1.90 percentage points, about 69% of the strong-API gain), reducing the concern that TRACE merely imports external annotator capability. Across scales, the best routed action is base-dependent: on Qwen3-8B it is forward KL on key spans, while on Qwen3-1.7B it shifts to reverse KL on error spans.

### 1. 核心问题与整体含义
*   **研究背景**：在具有可验证奖励的强化学习（RLVR，如 GRPO）中，奖励信号通常是稀疏的（仅在轨迹末端给出标量评分）。为了加密训练信号，现有的“同策略自蒸馏”（Self-OPD）方法让模型在特权上下文（如正确答案、思维链提示）下生成逻辑值（Logits）来指导自身。
*   **核心问题**：论文指出，现有的全 Token 蒸馏存在“粒度错位”问题。在整个响应过程应用 KL 散度会导致：
    1.  **蒸馏税**：在 97% 的冗余 Token 上浪费梯度。
    2.  **特权信息泄露**：模型学会了依赖特权信号的“捷径”，导致在长程训练中出现熵值激增、推理长度缩短以及分布外（OOD）性能退化。
*   **研究动机**：寻找一种更精准的蒸馏单位，仅在学生模型需要修正的关键推理 Token 上进行指导。

### 2. 方法论
*   **核心思想**：提出 **TRACE**（Token-Routed Alignment for Critical rEasoning），通过特权标注器识别关键片段，并根据片段类型路由不同的 KL 动作，最后将蒸馏信号随训练进程退火。
*   **关键技术细节**：
    *   **关键片段标注**：使用一个特权标注器（可以是强 API 或模型自身）为每个 Rollout 标记“关键正确片段”（Key Spans）或“局部错误片段”（Error Spans），并给出粗粒度的诊断类型（如“分类讨论错误”）。
    *   **Token 路由机制**：
        *   **前向 KL (FKL)**：应用于正确 Rollout 的关键片段，旨在提升（Lift）那些受教师支持但被学生分配概率过低的 Token。
        *   **反向 KL (RKL)**：可选地应用于错误片段，旨在抑制学生模型过度自信的错误 Token。
        *   **GRPO**：所有非片段 Token 仅通过标准的 GRPO 算法进行训练。
    *   **退火与衰减**：KL 权重在简短的预热后线性衰减至零（约 40 步），确保长期训练由纯粹的强化学习驱动，避免特权信息泄露。
    *   **特权上下文隔离**：教师模型仅接收诊断类型标签（如 `case_split`），而不接收具体的片段文本或完整正确答案，以防止内容泄露。

### 3. 实验设计
*   **数据集**：使用 OpenThoughts-114k 的数学子集进行训练。
*   **基准测试（Benchmarks）**：
    *   **分布内（ID）**：MATH-500, AIME 2024, AIME 2025, AMC 2023。
    *   **分布外（OOD）**：GPQA-Diamond（研究生水平的科学问题）。
*   **对比方法**：
    *   **GRPO**：标准的强化学习基准。
    *   **SDPO / SRPO**：现有的全 Token 自蒸馏方法。
    *   **RLSD**：一种通过优势重加权的自蒸馏变体。
*   **模型规模**：在 Qwen3-8B（强基座）和 Qwen3-1.7B（弱基座）上分别验证。

### 4. 资源与算力
*   **硬件**：使用了 NVIDIA **H100 GPU**。
*   **框架**：基于 `verl` 强化学习框架。
*   **训练细节**：训练步数为 300 步，学习率为 $1 \times 10^{-5}$，使用了 AdamW 优化器。文中未明确给出总的 GPU 小时数，但提到在 H100 上进行了多规模模型的实验。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 1.7B 到 8B 的模型跨度，并在 5 个核心基准上进行了测试。
*   **消融实验**：非常充分。包括：
    *   **标注器质量**：对比了强 API、中等模型（32B）、静态基座模型以及**在线自我标注**（正在训练的模型自身）。
    *   **掩码定位**：对比了随机掩码、反向掩码和全 Token 掩码。

### 6. 主要结论与发现
*   **性能提升**：在 Qwen3-8B 上，TRACE 相比 GRPO 平均提升了 **2.76 个百分点**。
*   **OOD 鲁棒性**：TRACE 是唯一能在训练后保留基座模型在 GPQA-Diamond 上 OOD 得分的方法，而其他自蒸馏方法均出现了显著退化。
*   **路由反转现象**：在强基座（8B）上，对关键片段进行 FKL 效果最好；而在弱基座（1.7B）上，对错误片段进行 RKL 效果更好。这验证了理论：弱模型更多受局部过度自信困扰，而强模型更多受关键步骤分配不足困扰。
*   **自我标注可行性**：在线自我标注能保留约 69% 的强 API 收益，证明 TRACE 并非单纯依赖外部知识输入。

### 7. 优点
*   **诊断深入**：成功识别并解释了全 Token 蒸馏导致“推理萎缩”和“熵增”的内在机制。
*   **设计优雅**：通过简单的路由和退火机制解决了复杂的特权信息泄露问题。
*   **理论与实践结合**：提供了严谨的数学证明（如累积暴露界限、非消失提升效应）来支撑实验观察。
*   **通用性**：不依赖于特定的外部教师模型，具有很强的自我进化潜力。

### 8. 不足与局限
*   **领域局限性**：实验主要集中在数学和科学推理任务上，对于创意写作、多轮对话等非验证性任务的适用性尚待验证。

*   **超参数依赖**：退火步数（40步）和覆盖率上限（25%）等超参数可能需要根据不同的任务场景进行精细调整。
