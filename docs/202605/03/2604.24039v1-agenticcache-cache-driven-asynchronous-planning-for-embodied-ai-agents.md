## 摘要
具身智能体（Embodied AI agents）越来越多地依赖大语言模型（LLMs）进行规划，但每步调用 LLM 会带来严重的延迟和成本。在本文中，我们展示了具身任务表现出很强的规划局部性（plan locality），即下一个规划在很大程度上可以从当前规划中预测。基于此，我们引入了 AgenticCache，这是一个重用缓存规划以避免每步调用 LLM 的规划框架。在 AgenticCache 中，每个智能体查询频繁规划转换的运行时缓存，而后台缓存更新器（Cache Updater）则异步调用 LLM 以验证和细化缓存条目。在四个多智能体具身基准测试中，AgenticCache 在 12 种配置（4 个基准测试 x 3 个模型）下平均将任务成功率提高了 22%，模拟延迟降低了 65%，Token 使用量降低了 50%。因此，基于缓存的规划重用为实现低延迟、低成本的具身智能体提供了一条切实可行的路径。代码可在 https://github.com/hojoonleokim/MLSys26_AgenticCache 获取。

## Abstract
Embodied AI agents increasingly rely on large language models (LLMs) for planning, yet per-step LLM calls impose severe latency and cost. In this paper, we show that embodied tasks exhibit strong plan locality, where the next plan is largely predictable from the current one. Building on this, we introduce AgenticCache, a planning framework that reuses cached plans to avoid per-step LLM calls. In AgenticCache, each agent queries a runtime cache of frequent plan transitions, while a background Cache Updater asynchronously calls the LLM to validate and refine cached entries. Across four multi-agent embodied benchmarks, AgenticCache improves task success rate by 22% on average across 12 configurations (4 benchmarks x 3 models), reduces simulation latency by 65%, and lowers token usage by 50%. Cache-based plan reuse thus offers a practical path to low-latency, low-cost embodied agents. Code is available at https://github.com/hojoonleokim/MLSys26_AgenticCache.

#### 1. 核心问题与整体含义（研究动机和背景）
具身智能体（Embodied AI）通常遵循“感知-规划-执行”循环。目前主流方法使用大语言模型（LLM）作为推理核心，但**每步调用 LLM 会产生极高的推理延迟和经济成本**（实验显示规划占总运行时间的 70% 以上）。
论文观察到具身任务具有强烈的**“规划局部性”（Plan Locality）**：即下一个规划动作往往可以从当前状态和前一个动作中预测（例如：抓取物体后，下一步通常是移动到目标点）。受计算机体系结构中缓存和分支预测的启发，作者提出了 **AgenticCache** 框架，旨在通过重用缓存的规划转换来减少对 LLM 的频繁调用，实现低延迟、低成本的具身智能。

#### 2. 方法论：核心思想与关键技术
AgenticCache 的核心是**将规划过程从同步变为异步缓存驱动**。
*   **运行时缓存（Runtime Cache）：** 存储 2-gram 规划模式 $\langle P_i \to P_j \rangle$（如：`GoGrasp` $\to$ `Transport`）。每个条目包含任务状态元数据（如步数、持有物品数等）。
*   **评分与选择机制：** 智能体通过当前状态过滤候选规划，并根据复合得分 $S = C \cdot I$ 选择最佳动作。其中 $C$（Count）是局部转换频率，$I$（Importance）是全局确认率（由 LLM 验证的比例）。
*   **异步缓存更新器（Cache Updater）：** 在后台运行的 LLM 进程。它不阻塞执行，而是异步验证缓存的准确性。
    *   **确认（Confirmation）：** 若缓存预测正确，增强该条目权重并抑制冗余查询。
    *   **修正（Correction）：** 若缓存预测错误，立即替换当前执行的规划并更新缓存。
*   **预填充（Warm-Start）：** 可选步骤，利用少量离线成功轨迹初始化缓存，解决冷启动问题。

#### 7. 优点
*   **架构创新：** 巧妙地将计算机体系结构中的缓存思想引入具身智能规划，解决了 LLM 推理慢的痛点。
*   **极低开销：** 缓存结构极其轻量（KB 级别），对内存和计算资源几乎无额外负担。
*   **实用性强：** 兼容现有的 LLM 服务技术和多智能体框架，易于部署到实时性要求高的场景。

#### 8. 不足与局限
*   **多智能体冲突：** 在高度竞争或需要极精密同步的时刻，缓存重用可能导致智能体间的动作冲突（如重复取同一个物体），虽然更新器能修正，但存在滞后。
*   **局部性依赖：** 该方法依赖于任务的结构化规律。在完全随机、无序或高度开放的探索任务中，规划局部性可能变弱，导致缓存命中率下降。
*   **2-gram 限制：** 目前仅考虑了相邻两个规划的转换，对于需要更长历史依赖的复杂决策，可能需要更高阶（如 3-gram）或层次化的表示。
