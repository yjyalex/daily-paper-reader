## 摘要
强化学习通过奖励信号直接优化模型行为来微调大型语言模型。虽然在经典强化学习中准确的状态价值估计对于稳定训练至关重要，但在大型语言模型的后训练中，这一挑战尚未得到充分探索。在这项工作中，我们引入了状态价值估计基准（SVEB）来评估现有强化学习框架中的状态估计，并展示了PPO等标准方法中的评论家会坍缩为粗糙的组平均基线。为了解决这个问题，我们提出了两种技术：Numca，利用数值跨度作为可分级里程碑进行状态价值估计；以及Hista，一种使用LLM隐藏状态作为表示来加权平均不相交轨迹及其返回的框架。大量实验表明，这两种方法都能获得更准确的状态价值估计，并能在不同强化学习算法和模型大小下提升训练性能，且不会带来显著的计算开销。

## Abstract
Reinforcement learning (RL) refines large language models (LLMs) by directly optimizing model behavior through reward signals. While accurate state value estimation is critical for stable training in classical RL, it remains an underexplored challenge in LLM post-training. In this work, we introduce the State Value Estimation Benchmark (SVEB) to assess state estimation within existing RL frameworks and show that critics in standard approaches like PPO collapse to a coarse group-average baseline. To address this, we propose two techniques: Numca, which leverages numerical spans as gradable milestones for state value estimation, and Hista, a framework that uses LLM's hidden states as representation to weighted average disjoint rollouts and their return. Extensive experiments demonstrate that both methods yield more accurate state value estimates and enhance training performance across different RL algorithms and model sizes without incurring significant computational overhead.

## 2. 方法论

### 2.1 核心思想
- **Numca**（数值里程碑信用分配）：将数学推理视为稀疏奖励的目标达成问题，利用数值（整数、小数、分数）作为自然里程碑，将包含相同里程碑的轨迹的最终回报平均作为该抽象状态的价值。
- **Hista**（基于隐藏状态的状态价值估计）：利用LLM最后一层隐藏状态作为状态表示，通过MinDistance度量状态间相似度，使用k近邻的逆距离加权平均回报来估计当前状态价值。

### 2.2 关键技术细节
- **Numca**：
  - 定义数值模式集合 \( P \)，从状态中提取里程碑序列 \( M(s_t) \)。
  - 构建抽象状态 \( s^M_t \)，对每个抽象状态维护计数器与奖励和，最终价值 \( V(s^M) = \text{reward\_sum} / \text{count} \)。
  - 将价值均匀分配给该抽象状态对应的宏动作内所有token。
- **Hista**：
  - 从rollout的终端状态提取最后层隐藏状态序列，经EMA压缩和均匀采样得到有限状态空间。
  - 定义MinDistance：对两个长度不等的隐藏状态序列，逐个元素计算到另一序列的最小距离并求和。
  - 状态价值估计：\( V(s_t) = \frac{\sum_{i=1}^k \omega_i r_i}{\sum_{i=1}^k \omega_i} \), 其中 \( \omega_i = 1 / \text{MD}(s_t, s_i) \)。
  - **理论保证**（Theorem 5.5）：在温和假设下，概率加权估计的绝对偏差不大于组平均估计。

### 2.3 算法流程
- Numca：遍历所有rollout → 提取里程碑 → 更新字典 → 计算每个抽象状态的平均奖励。
- Hista：提取隐藏状态 → 压缩采样 → 计算MinDistance → 选取k近邻 → 加权平均。

## 6. 主要结论与发现

- **SVEB揭示**：PPO的评论家网络在LLM RL中退化为组平均基线（MAE几乎相同），无法提供细粒度指导。
- **Numca**在数学推理领域显著优于GRPO和PPO，但无法推广到无显式数值里程碑的领域。
- **Hista**在所有SVEB领域均优于GRPO（MAE降低0.017~0.056），在RL训练中一致提升下游benchmark（平均提升0.5%~2.3%），且与DAPO、CSIPO等结合效果稳定。
- **理论证明**（Theorem 5.5）支持Hista比组平均估计偏差更小。
- **效率**：计算开销与GRPO相当，额外延迟小于1秒/步。

- **轻量实用**：Numca仅需字典维护，Hista利用已有隐藏状态，无需额外训练或大量rollout。
