## 摘要
多智能体LLM系统通过结合多个智能体的输出来改进推理，但交互繁重的方法可能引入错误传播和高通信开销。当智能体交换原始响应或推理轨迹时，错误的中介推理可能被采纳并放大，导致自信但错误的共识；多轮通信也增加了令牌消耗、延迟和推理成本。本文提出了一种名为DarkForest的受控通信协调框架。DarkForest首先保持智能体独立，每个智能体在不看到其他智能体输出的情况下产生答案。然后，它将原始响应解析为结构化候选记录，将语义等价的候选记录分组为簇，并使用智能体可靠性、置信度、解析质量、支持模式可靠性和独立性校正来估计这些簇上的校准信念分布。协调器仅从具有受控通信的这个信念状态接收策略允许的证据。在六个推理基准上的实验表明，DarkForest实现了领先的整体质量，在基准指标上比最强基线提高了30.7%，与通信密集的基线相比，令牌消耗降低了高达6.5倍。

## Abstract
Multi-agent LLM systems improve reasoning by combining outputs from multiple agents, but interaction-heavy methods can introduce error propagation and high communication overhead. When agents exchange raw responses or reasoning traces, incorrect intermediate reasoning may be adopted and amplified, leading to confident but wrong consensus; multi-round communication also increases token consumption, latency, and inference cost. In this paper, we propose a controlled-communication coordination framework named DarkForest. DarkForest first keeps agents independent, so each agent produces an answer without seeing the others' outputs. It then parses the raw responses into structured candidate records, groups semantically equivalent candidates into clusters, and estimates a calibrated belief distribution over these clusters using agent reliability, confidence, parse quality, support-pattern reliability, and independence corrections. A coordinator receives only policy-permitted evidence from this belief state with controlled communication. Experiments on six reasoning benchmarks show that DarkForest achieves leading overall quality, improves the strongest baseline by up to 30.7\% on benchmark metrics, and reduces token consumption by up to $6.5\times$ compared with communication-heavy baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **多智能体 LLM 系统的现有缺陷**：现有方法（如辩论、角色扮演、工作流）通过智能体间的交互（交换完整推理轨迹或原始答案）来提升推理能力，但带来了两大问题：  
  1. **错误传播**：一个智能体的错误推理被其他智能体采纳、放大，最终形成自信但错误的共识。  
  2. **通信开销高**：多轮交换大幅增加 token 消耗、延迟和推理成本。  
- **关键发现**：作者在 MATH 数据集上验证，即使初始智能体独立输出中至少有一个包含正确答案（即候选可用性上限达 75.2%），现有协调方法（如 Debate、Self-Consistency 等）的最终准确率远低于此上限，表明它们常常丢弃或覆盖了独立产生的正确候选。  
- **研究目标**：回答“什么信息应该跨越智能体边界，以及何时披露能改善而不是污染最终决策”。作者从**不完全信息博弈论**获得启发，认为协作应通过显式的信息政策来调节，而非共享全部推理轨迹。

## 2. 论文提出的方法论

### 核心思想
DarkForest 是一个**受控通信协调框架**，遵循三条设计原则：  
1. **候选生成独立**：智能体在相互不观察对方输出的情况下独立生成答案。  
2. **聚合需校准**：对一致投票进行加权，权重基于历史可靠性、解析质量、置信度和智能体间的依赖性。  
3. **通信受控**：协调器仅接收策略允许的压缩证据，而非原始推理轨迹。

### 关键技术细节
- **独立生成与解析**：每个智能体 m_i 输出响应 r_i，经任务特定的解析器 P 转换为结构化观测 o_i = (a_i, z_i, c_i, v_i, q_i)，其中 z_i 是规范化后的候选，c_i 为置信度，v_i 为解析有效性。无效观测被排除。
- **候选聚类**：将语义等价的候选 z 聚类为簇 C_z = (z, S_z, π_z)，其中 S_z 是支持该候选的智能体索引集，π_z 是支持模式（即哪些智能体支持）。
- **校准信念构建**：簇 z 的证据得分为：  
  s(z) = R_{π_z} · Σ_{i∈S_z} α_i · ρ_i · δ_i · φ(c_i)  
  - α_i：智能体 i 的校准可靠性（通过有标签数据估计，带 Laplace 平滑）。  
  - R_{π_z}：支持模式 π_z 的校准可靠性（同样从校准数据估计）。  
  - ρ_i：解析质量惩罚（格式不良或可解析但不佳的输出降低权重）。  
  - δ_i：独立性折扣（已知相关的智能体不会被视为独立证据源）。  
  - φ(c_i) = 0.5 + c_i：置信度调制因子，范围 [0.5, 1.5]。  
  得分经 ReLU 和归一化得到后验似然分布 P(z|O)。并计算最大后验候选 z* 及其与第二候选的间隔 Δ。
- **不确定性识别**：若 P(z*|O) < τ_u 或 Δ < τ_Δ，则标记为不确定。
- **受控披露**：披露策略 Ω 决定向协调器暴露哪些信息（如候选标识符、支持模式、后验质量、不确定性指标等），通常不暴露完整原始轨迹。记录 disclosure cost T_cross = tokens(E)。
- **协调器与确定性护栏**：协调器 m_coord 接收输入 x 与暴露证据 E 后生成输出 ˆz_coord。然后，护栏仅在信念状态强支持某个候选（满足支持数、后验概率和间隔阈值）且与协调器冲突时，才将最终答案替换为 z*。否则采用协调器输出。

### 算法流程（文字描述）
1. 各智能体独立生成响应。  
2. 解析并规范化，剔除无效观测。  
3. 将规范化候选聚类。  
4. 利用校准参数计算各簇证据得分，归一化为后验分布。  
5. 根据披露策略选取信息暴露给协调器。  
6. 协调器产生候选。  
7. 应用确定性护栏：若信念状态高度信任某候选且与协调器冲突，则覆盖之；否则保留协调器结果。

## 6. 主要结论与发现

1. **DarkForest 实现领先或近优质量**：在 6 个基准上，DarkForest 收获 4 个最佳（MATH 76.80%、MMLU-Pro 58.38%、GPQA 39.90%、FinQA 程序准确率 11.33%）和 2 个次佳（HumanEval 84.00% 持平 MoA、LegalBench 68.00% 比 ReConcile 低 1%）。  
2. **显著降低 token 消耗**：相比 GoA-mean，DarkForest 在 MATH 上 token 减少约 66%（13.8k → 4.7k），在 MMLU-Pro 上减少 76%（24.6k → 5.9k），在 GPQA 上减少 75%（13.0k → 3.2k）。  
3. **控制通信比简单投票或辩论更有效**：DarkForest 超越多数投票和加权投票（见投票消融），并且比辩论式方法更稳定。  
4. **协调器与护栏互补**：去协调器后 LegalBench 下降，去护栏后两基准均下降，且护栏不增加 token 消耗。  
5. **披露更多文本不一定更好**：披露完整原始轨迹反而降低 GPQA 准确率（从 40% 降至 36.67%），增加 token 成本。  
6. **可扩展**：增加更多独立骨干可进一步提升性能，token 增加有限。  
7. **协调器不依赖特定模型**：更换协调器（如 Qwen-Coder）仍能取得良好结果，但弱协调器会降低质量。

## 7. 优点（方法或实验设计亮点）

- **零训练成本**：仅需少量校准数据估计参数，无需微调 LLM，实用性强。  
- **开源**：代码已公开，可复现。

## 8. 不足与局限

- **实验覆盖局限**：  
  - 仅测试 7B 左右规模模型，未验证在更大模型（如 70B、GPT-4 等）上的表现。  
  - 未与更复杂的工作流型多智能体系统（如 MetaGPT、AutoGen 完整对话循环）对比。  
- **校准依赖性**：需要每个任务有少量带标签校准数据，在完全零样本场景下需使用默认参数（文中提供默认值，但可能不如校准后可靠）。  
- **任务范围限制**：  
  - HumanEval 上仅达次优，说明代码生成可能更需丰富中间表示，简单候选摘要未必最优。  
  - 只测试了单轮答案的任务，未涉及需要多步协作或长期对话的场景。  
- **可能偏差**：智能体选择固定（Qwen2.5 系列 + 少量专业模型），结果可能受模型能力影响。  
- **未讨论**：智能体间存在系统级依赖（如同源训练数据）时，独立性校正（δ_i）的估计方法较为简化，实际应用可能需要更精细的依赖建模。
