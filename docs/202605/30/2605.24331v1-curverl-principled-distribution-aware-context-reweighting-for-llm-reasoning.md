## 摘要
上下文或提示级别的重加权已成为强化学习与验证奖励（RLVR）中用于提升大语言模型推理能力的关键算法杠杆，然而，关于什么构成最优加权的原则仍然知之甚少。我们通过将提示重加权定义为通过率函数空间中定义的效用泛函的泛函导数来弥补这一空白，从而产生一个统一的优化框架，该框架能够容纳现有方案，包括REINFORCE和GRPO。基于这一优化框架，我们提出了一种基于分位数坐标变换的分布感知提示重加权方法，称为CurveRL，其中分配给每个提示的权重不取决于通过率的绝对值，而是取决于其排名和密度，以反映学习动态中通过率的分布结构。跨多个基准的大量实验表明，我们提出的CurveRL始终优于GRPO和其他RLVR基线。我们的研究将上下文分布控制确定为分析和设计提示加权RLVR算法的一个原则性轴。代码发布于https://github.com/zhyzmath/CurveRL。

## Abstract
Context or prompt-level reweighting has emerged as a central algorithmic lever in Reinforcement Learning with Verified Rewards (RLVR) for improving the reasoning capability of large language models, yet the principle determining what constitutes an optimal weighting remains poorly understood. We address this gap by formulating prompt reweighting as a functional derivative of a utility functional defined in the pass-rate function space, yielding a unified optimality framework that accommodates existing schemes, including REINFORCE and GRPO. Building on this optimality framework, we propose a distribution-aware prompt reweighting approach, called CurveRL, based on a quantile coordinate transform, in which the weight assigned to each prompt depends not on the absolute value of pass rates but on its rank and density to reflect the distributional structure of the pass rates in the learning dynamics. Extensive experiments across multiple benchmarks demonstrate that our proposed CurveRL consistently outperforms GRPO and other RLVR baselines. Our study identifies context-distribution control as a principled axis for analyzing and designing prompt-reweighted RLVR algorithms. The code is released in https://github.com/zhyzmath/CurveRL.

## 1. 核心问题与整体含义（研究动机与背景）

- **研究动机**：在基于验证奖励的强化学习（RLVR）中，提示重加权（prompt reweighting）已成为提升大语言模型（LLM）推理能力的关键技术，但**什么构成最优的加权策略**缺乏理论原则的指导。现有的方法（如REINFORCE、GRPO、MaxRL等）各自基于启发式规则，均未能给出最优性的理论解释。
- **核心问题**：如何将提示重加权形式化为一类**上下文分布控制**（context distribution control）问题，并提出**最优加权**的数学定义。
- **研究含义**：本文通过泛函导数在通过率（pass-rate）函数空间中定义最优权重，统一现有不同加权方案，并基于此提出分布感知的CurveRL方法，从而将上下文分布控制确立为设计RLVR算法的一个原则性轴。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 将提示重加权视为在**策略依赖的上下文分布**（Policy-Reweighted Contextual Bandit, PRCB）下的优化问题。
- 最优权重由**效用泛函**（utility functional）对通过率函数 \(p_\theta(x)\) 的**泛函导数**（functional derivative）定义：
  \[
  w^*_\theta(x) = \frac{\delta U_\theta}{\delta p_\theta}(x)
  \]
  这比点态偏导（仅依赖局部信息）更具表达力，可捕捉全局分布结构。

### 2.2 关键技术细节
- **点态效用泛函** （如REINFORCE、GRPO、MaxRL）退化为例1，权重仅依赖绝对通过率值 \(p_\theta(x)\)，导致**权重坍塌**（weight collapse）：当多数提示通过率接近0或1时，权重无法区分学习潜力。
- **分布感知效用泛函**：引入参考分布 \(F_{\text{ref}}\)（通过滞后策略窗口估计），在**分位数坐标空间**定义效用：
  \[
  U_\theta(F_{\text{ref}}) = \mathbb{E}_{x\sim d_0}[\psi(F_{\text{ref}}(p_\theta(x)))]
  \]
  其中 \(\psi(u) = \log u\)（扭曲风险函数，强调低分位提示）。梯度为：
  \[
  \nabla_\theta U_\theta = \mathbb{E}_{x\sim d_0}\left[\frac{f_{\text{ref}}(p_\theta(x))}{F_{\text{ref}}(p_\theta(x))}\nabla_\theta p_\theta(x)\right]
  \]
  该权重等于**反向风险率**（reverse hazard rate），同时利用**排名**（通过 \(1/F_{\text{ref}}\)）和**密度**（通过 \(f_{\text{ref}}\)）信息。
- **算法CurveRL**（Algorithm 1）：
  - 维护一个滑动窗口 \(W\)（大小 \(t_0 \times |B|\)）存储历史通过率，用于估计 \(F_{\text{ref}}\) 和 \(f_{\text{ref}}\)（直方图估计）。
  - 对每个提示 \(x\)，采样 \(N\) 个回复，计算经验通过率 \(\hat{p}\)，若 \(\hat{p} \in (0,1)\)，则赋予权重 \(w_t(\hat{p}) = \hat{f}_{\text{ref}}(\hat{p}) / \hat{F}_{\text{ref}}(\hat{p})\)，并使用 \(\hat{p}\) 作为基线降低方差。
  - 梯度更新为 \( \frac{1}{|B|}\sum_x \frac{1}{N}\sum_i w_t(\hat{p}) (r_i - \hat{p}) \nabla_\theta \log \pi_\theta(y_i|x)\)。

### 2.3 理论优势
- **定理1**：任意点态权重可等价表示为某个先验分布 \(F_{\text{ref}}\) 下的分布感知权重，而CurveRL的 \(F_{\text{ref}}\) 是数据驱动的动态估计，更具适应性。
- **推论1**：梯度等价当且仅当参考分布为均匀分布，否则分布感知与点态权重不同。
- **单调校准不变性**（Proposition 3）：CurveRL对通过率的单调变换不变，鲁棒于校准误差。

## 6. 主要结论与发现

1. **CurveRL一致优于GRPO和MaxRL** ：在几乎所有基准和模型规模上，pass@1和pass@64均有提升；平均pass@64提升相对MaxRL为+5.9%（1.7B）和+9.7%（4B）。
2. **改善pass@1与pass@k的帕累托前沿**：未出现因提升pass@k而牺牲pass@1的情况（而GRPO和MaxRL有时会降低pass@1）。
3. **拓宽可解性边界**：CurveRL减少了不可解提示比例，将更多提示推向“中等”难度区间，使多数投票（majority voting）更可靠。
4. **权重自适应且数据驱动**：相比静态点态权重，CurveRL的权重根据回归率分布的演变动态调整，倾向于给低分位提示更高权重（更风险寻求），这与当前模型能力与数据集难度不匹配有关。
5. **分布感知权重的数学基础**：通过泛函导数、分位数坐标变换、Wasserstein几何等理论，统一解释了各类加权方法的本质。

## 7. 优点

- **理论创新**：为提示重加权提供了严格的泛函导数最优性框架，将点态方法视为特例，并解释了其局限性（权重坍塌）。
- **算法简洁**：CurveRL实现简单，仅需在GRPO基础上添加一个滑动窗口的密度/分位数估计。
- **实验全面且公平**：在8个基准、2个模型规模、多个k值下验证，控制变量仅改变加权规则；消融实验充分；可视化揭示了训练动态。
- **分析深入**：从难度分布、权重动态、校准不变性等多个角度剖析机制，并与经典理论（风险敏感控制、最优运输）建立联系。
