## 摘要
基于大语言模型（LLM）构建的多智能体系统需要众多难以预先确定的协调选择：调用哪个技能协议、哪个智能体角色应执行子任务、每个角色绑定哪个模型、角色之间如何交互、何时使用检索或验证、以及何时完全省略某个步骤。这些选择与任务场景和操作约束相互影响，因此静态流水线和一次性模型比较只能提供设计空间的有限视角。本文介绍AgensFlow，一个开源框架，它将多智能体协调视为部分可观测环境下的在线策略学习问题。该框架使协调决策变得可观测且可从重复轨迹中学习，而不是将技能、角色、模型、拓扑结构和评估选择视为固定的流水线设计。AgensFlow在两个数据集上进行了评估：分布式系统事件任务和安全公告任务。评估显示了三个主要结果：在协调密集型类别上，学习到的路由比固定流水线基线达到了更高的质量运行点；skip:X将拓扑压缩隔离为基板中有意义的一部分；热启动的策略图可以在保持平台质量的同时降低探索成本。总体而言，结果表明，与静态布线相比，可学习、可审计的路由可以改善协调密集型多智能体工作流。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

- **整体含义**：将智能体能力瓶颈从单个模型能力转向**动态协调策略**，提出**协调策略基板（Coordination-Policy Substrate）**，使系统能在线学习何时、如何组合技能、模型和拓扑。

## 2. 论文提出的方法论

### 2.1 核心思想

- 将多智能体协调建模为**部分可观测环境下的在线策略学习问题**。
- 系统不直接观测用户意图、任务难度等隐状态，而是构建**折叠签名（folded signature）** 作为可观测的抽象状态，使不同运行的相似状态共享学习经验。

### 2.2 关键技术与细节

- **状态表示**：折叠签名 \(s_t = \phi(x_t, h_t, b_t) = (\ell_t, z_t, \beta(\hat{c}_t), \beta(\hat{u}_t), \beta(\hat{r}_t), \beta(\hat{e}_t))\)，包含：
  - 规则检测的任务标签（如 `straightforward`, `evidence_heavy`, `ambiguous` 等）
  - 7 位二值手执状态掩码（目标、子问题、证据、批评、验证、草稿答案、合并答案是否已填充）
  - 四个连续信念估计（正确性、不确定性、矛盾风险、证据充分性）的桶化版本
- **动作空间**：\(a_t \in A(s_t) = \{\text{invoke}(k, m)\} \cup \{\text{skip}:X\} \cup \{\text{terminate}\}\)，其中 `(k,m)` 是技能协议与模型绑定，`skip:X` 表示跳过可选单元（使拓扑本身成为可学习变量）。
- **奖励函数**：\(r(t) = w_q Q(t) - w_c C(t) - w_\rho \rho(t)\)，其中 Q(t) 是 RelativeJudge 质量分，C(t) 是标准化 token 成本，ρ(t) 是重试/失败惩罚。默认权重 w_q=1.0, w_c=0.3, w_ρ=0.15。
- **策略学习**：使用**可靠性感知的 UCB1 变体**，公式：\(\text{score}(s,a) = \bar{r}(s,a) + c_s \sqrt{\ln(N_s+1)/N_{s,a}} - \lambda f(s,a)\)，其中探索系数 \(c_s\) 随签名访问次数 \(N_s\) 指数退火（从1.4衰减到0.5），失败惩罚系数 \(\lambda=0.5\)。
- **学习对象**：持久化的**策略图**，以折叠签名为键，存储每个（状态，动作）对的访问次数、平均奖励、方差、token 统计、失败计数。

### 2.3 算法流程（文字说明）

1. 预检（Pre-flight）关卡。
2. 计算任务特征，通过规则检测器生成任务标签，结合手执状态与信念估计折叠为签名。
3. 每个路由步骤：
   - 根据策略图（UCB1 分数）从合法动作集中选择动作（调用某技能-模型组合或 skip:X）。
   - 执行动作，更新手执状态与信念记录。
   - 重新计算签名，进入下一步。
4. 当评估器标记完成、预算耗尽、无合法动作或治理停止时终止。
5. 轨迹完成后，RelativeJudge 对同类别任务进行相对评分，与成本、失败惩罚组成混合奖励，回传给策略图中访问过的（s,a）边。

## 3. 实验设计

### 3.1 数据集与场景

- **分布式系统事件任务**（60个）：覆盖 Paxos/Raft、逻辑时钟、Gossip、CRDTs、一致性哈希、故障检测等，用于训练初始策略图（443节点）。
- **安全公告任务**（60个）：覆盖 6 个厂商对，设计为 8 个场景类：
  - C1 程序化、C2 单文档、C3 跨文档多厂商、C4 综合、C5 领域外模糊、C6 程序化衍生、C7 缓解正确性、C8 跨厂商对。

### 3.2 Benchmark 与对比方法

- **固定基线（baseline）**：固定 7 单元流水线，单次运行，无学习。
- **无 skip 消融（no-skip ablation）**：8 个 epoch，使用变体池学习，但强制关闭 skip:X 动作。
- **主方法（main）**：8 个 epoch，冷启动，skip:X 开启。
- **热启动（warm-start）**：8 个 epoch，skip:X 开启，从分布式系统策略图（443节点）初始化。

### 3.3 评估指标

- 在线奖励：使用 Claude Haiku 4.5 作为单裁判的 RelativeJudge 评分与混合奖励。
- 审计：最终轨迹由**三个模型（Claude Haiku 4.5 + GPT-5.4 mini + Qwen3.6-flash）** 组成的裁判组合进行后验交叉裁判评分，得出 3-judge audit 分数。

- **实验数量**：主要对比 4 个实验臂（基线、无skip消融、主冷启动、热启动），每个臂在 60 个任务上运行 8 个 epoch，产生多个数据点。此外还有跨域热启动、无skip消融、单裁判与三裁判审计对比。
- **充分性评估**：
  - **优点**：包含了基线、消融、冷启动、热启动四类对比，并且进行了跨域转移测试和奖励审计，设计较为全面。

## 6. 论文的主要结论与发现

1. **学习路由优于固定基线**：主方法在平台期的三裁判审计分 0.817  vs 基线 0.749，提升 +0.068，token 仅增加 15%。
2. **skip:X 可有效压缩并改善性能**：无skip消融 token 增加 94% 但质量未提升，说明跳过不必要的步骤是学习到的有用行为。
3. **热启动降低探索成本**：热启动在三裁判审计下平台期提升 +0.012，但 token 减少 ~10%，全运行减少 ~21%；单裁判显示更大质量优势，说明奖励信号存在偏差，需要审计。
4. **奖励审计是第一类问题**：单裁判与三裁判审计对热启动质量提升的结论差异明显，说明协调策略的设计依赖奖励的可靠性。
5. **增益集中在协调密集型类别**：在 C3（+0.18）、C8（+0.16）、C7（+0.13）上提升最大，程序化类（C1、C6）为统计持平。

## 7. 优点

- **创新视角**：将多智能体协调视为可学习的在线策略问题，而非固定管线设计。
- **可审计性**：策略图持久化、可 inspect，奖励信号通过跨裁判组合和维度分解实现审计。
- **将拓扑作为动作**：通过 skip:X 使省略步骤成为可学习的决策，增加了系统灵活性。
- **热启动机制**：展示了策略图可在不同领域间迁移，减少探索成本而不损失质量。
- **实验设计规范**：包含消融、跨域、审计、平台期比较，结果分析细致。
