## 摘要
大语言模型（LLMs）能够从自然语言生成代码，但其性能对提示表述高度敏感。我们提出一个基于强化学习的框架，将提示优化建模为序列决策问题。近端策略优化（PPO）智能体通过混合动作空间（结合直接生成、遗传词汇突变和语义重写）迭代改进提示，并由源自单元测试反馈的塑形奖励引导。我们使用CodeT5+、CodeLLaMA和DeepSeek-Coder作为冻结代码生成器，在MBPP+、HumanEval+和APPS上评估该框架。在500任务的MBPP+测试集上，PPO智能体分别达到57.58%、64.80%和85.50%的严格Pass@1分数，优于EPiC、Reflexion和Random-Hybrid。软Pass@1分别达到67.90%、73.10%和88.20%。在所有骨干模型上，HumanEval+和APPS也观察到类似改进。结果表明，具有塑形测试驱动奖励的强化学习提高了基于LLM的代码生成的功能正确性。

## Abstract
Large Language Models (LLMs) can generate code from natural language, but their performance is highly sensitive to prompt formulation. We propose a reinforcement-learning-based framework that models prompt refinement as a sequential decision-making problem. A Proximal Policy Optimization (PPO) agent iteratively improves prompts using a hybrid action space that combines direct generation, genetic lexical mutation and semantic rewriting, guided by shaped rewards derived from unit-test feedback. We evaluate the framework on MBPP+, HumanEval+, and APPS using CodeT5+, CodeLLaMA, and DeepSeek-Coder as frozen code generators. On the 500-task MBPP+ test set, the PPO agent achieves strict Pass@1 scores of 57.58%, 64.80%, and 85.50%, respectively, outperforming EPiC, Reflexion, and Random-Hybrid. Soft-Pass@1 reaches 67.90%, 73.10%, and 88.20%, respectively. Similar improvements are observed on HumanEval+ and APPS across all backbone models. The results demonstrate that reinforcement learning with shaped test-driven rewards improves functional correctness in LLM-based code generation.

## 2. 论文提出的方法论

### 核心思想
将提示优化建模为一个**序列决策问题**，使用**近端策略优化（PPO）** 智能体在每一步从混合动作空间中选择一个动作，对当前提示进行变换，然后由冻结的代码生成模型生成代码，根据单元测试结果计算塑形奖励，迭代训练以最大化累积奖励。

### 关键技术细节
- **状态表示**：当前提示通过 MiniLM 句子编码器嵌入为384维向量。
- **动作空间（三选一）**：
  - **直接生成（Direct Generation）**：保持提示不变，利用解码随机性探索不同输出。
  - **词汇突变（Lexical Mutation）**：基于遗传算法（EPiC启发），对提示进行token级交叉、洗牌和突变（概率0.2），通过测试通过率进行锦标赛选择。
  - **语义重写（Semantic Rewriting）**：使用预训练模型（基于LLaMA）和一个固定元提示进行结构化重写，并应用正则过滤保留关键词。
- **奖励函数**：基于测试通过率 ρ（通过数/总测试数）的塑形奖励：
  - ρ = 1.0 → 奖励 1.0
  - 0 < ρ < 1 → 奖励 ρ
  - ρ = 0 → 惩罚 -1
  - 执行失败（语法/运行时错误）→ 惩罚 -2
- **训练与评估流程**（见算法1和算法2）：
  - 对每个任务，初始提示嵌入后，智能体采样动作，变换提示，LLM生成代码，执行并计算奖励，存储转移并更新PPO策略。
  - 评估时，运行最多T步，记录是否达到严格通过（所有测试通过），并计算软通过率（逐步累积部分通过）。

## 6. 论文的主要结论与发现

1. **单步策略**中，语义重写通常最优，词汇突变次之，直接生成最弱。但单步改进有限，需要多步迭代。
2. **奖励塑造**显著优于稀疏二值奖励，塑形奖励带来更高Pass@1和训练稳定性，并允许智能体积累部分进步。
3. **多步RL（PPO）** 在所有数据集和骨干模型上均一致优于EPiC、Reflexion和Random-Hybrid。例如MBPP+测试集上，PPO在CodeT5+上严格Pass@1达57.58%（对比Random-Hybrid 31.12%）。
4. **Random-Hybrid的表现**（接近EPiC/Reflexion但低于PPO）证明性能提升来自**学习动作调度**，而非仅仅混合动作空间。
5. 在更具挑战性的APPS数据集上，虽然绝对分数下降，但PPO仍保持相对优势。
6. 实验确认了**强化学习结合塑形测试反馈**能有效提升LLM代码生成的功能正确性。
