## 摘要
多智能体LLM辩论提高了事实性和推理能力，但大多数方法选择固定的轮数，在简单问题上过度花费，在困难问题上投入不足。我们将Wald的序贯概率比检验（SPRT）适配为LLM辩论的插件式计算调控器。每轮结束后，LLM评判者对最新智能体立场输出一个[0,1]的共识分数；Wald监控器在Beta似然族下累积“有用收敛”相对于“尚未有用”的对数似然比，并在超出任一边界或达到最大轮数R_max时停止，返回一个截断的最佳结果。在独立同分布假设下，该规则继承了SPRT的第一类/第二类错误保证；在实际部署中，校准本身是更重要的对象，因为它估计评判者分数是否在给定领域中真正区分了有用和无益的收敛。我们评估了两个方向：（i）在校准的Beta模型下进行蒙特卡洛研究，刻画工作曲线、错误率、截断行为和敏感性；（ii）在200个MMLU和200个GSM8K样本上使用三个异构智能体（gpt-5、claude-opus-4-6、gemini-2.5-pro）和claude-opus-4-6评判者，使用不相交的40样本校准子集进行真实LLM评估。在GSM8K上，该规则平均1.01轮（4.06次LLM调用）达到97.0%准确率，而固定5轮辩论（15次调用）为99.0%：准确率降低2个百分点，调用次数降低3.7倍。在MMLU上，校准后的KL散度几乎降至0，该规则在99.5%的样本上以2.1倍成本截断。关键结论不是SPRT使辩论更准确，而是经典的序贯检验为多智能体LLM系统提供了一个低成本的计算控制和故障检测层。

## Abstract
Multi-agent LLM debate improves factuality and reasoning, but most recipes pick a fixed round count, over-spending on easy items and under-spending on hard ones. We adapt Wald's Sequential Probability Ratio Test (SPRT) as a plug-in compute governor for LLM debates. After each round, an LLM judge emits a [0,1] consensus score on the latest agent positions; a Wald monitor accumulates the log-likelihood ratio of "useful convergence" vs "not yet useful" under a Beta likelihood family, and stops when either boundary is crossed or returns a capped best-effort outcome at R_max. Under i.i.d. assumptions the rule inherits SPRT type-I/type-II error guarantees; in deployment the calibration itself is the more important object, since it estimates whether the judge score actually separates useful from unhelpful convergence in a given domain. We evaluate two tracks: (i) a Monte-Carlo study under calibrated Beta models characterising working curves, error rates, capping behaviour, and sensitivity; and (ii) a real-LLM evaluation on 200 attempted MMLU and 200 attempted GSM8K items with three heterogeneous agents (gpt-5, claude-opus-4-6, gemini-2.5-pro) and a claude-opus-4-6 judge, using disjoint 40-item calibration subsets. On GSM8K the rule stops in 1.01 average rounds (4.06 LLM calls) at 97.0% accuracy vs 99.0% for fixed-5 debate at 15 calls: a 3.7x call reduction at -2pp accuracy. On MMLU the calibrated KL collapses to about 0 and the rule caps on 99.5% of items at 2.1x cost. The takeaway is not that SPRT makes debate more accurate, but that a classical sequential test serves as a cheap compute-control and failure-detection layer for multi-agent LLM systems.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：当前多智能体LLM辩论通常采用固定轮次（如3-5轮），在简单问题上过早产生冗余调用，浪费计算资源；在困难问题上则可能无法充分收敛。固定轮次策略缺乏自适应能力。
- **核心问题**：如何在不牺牲太多准确率的前提下，动态决定辩论何时停止，以降低LLM调用成本，同时保证收敛信号的信息量足够。
- **背景**：Wald的序贯概率比检验（SPRT）是经典的自适应样本量假设检验方法，能够将第一类/第二类错误控制在预设水平，并最小化平均样本量。作者将其引入多智能体LLM辩论，作为计算调控器和故障检测层。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：将每轮辩论后LLM法官输出的共识分数 \(s_r \in [0,1]\) 视为序贯观测，利用SPRT在两个假设之间做决策：
  - \(H_0\)：智能体尚未收敛（不应停止，继续辩论）
  - \(H_1\)：智能体已经收敛（可以停止并输出答案）
- **关键技术细节**：
  - **Beta似然模型**：假设 \(s_r\) 服从Beta分布，在 \(H_0\) 和 \(H_1\) 下分布参数不同（通过一小批校准数据拟合）。
  - **累积对数似然比**：\(\Lambda_R = \sum_{r=1}^R \log \frac{f_1(s_r)}{f_0(s_r)}\)，其中 \(f_0, f_1\) 是两种假设下的Beta密度。
  - **停止阈值**：\(A = \log((1-\beta)/\alpha)\)，\(B = \log(\beta/(1-\alpha))\)，其中 \(\alpha, \beta\) 为目标第一类/第二类错误率（通常各取0.05）。
  - **决策规则**：
    - 若 \(\Lambda_R \ge A\)，接受 \(H_1\)（共识达成），停止辩论并输出聚合答案；
    - 若 \(\Lambda_R \le B\)，接受 \(H_0\)（无共识），停止并输出“无共识”标记，可转交更强大的模型或人工审核；
    - 否则继续下一轮。
  - **硬上限**：设置最大轮次 \(R_{\max}=8\)，超过后强制停止，标记为“截断（capped）”。
- **算法流程**（文字描述）：
  1. 校准：对每个任务领域，用40条不相交的辩论轨迹（已知正确答案）拟合 \(f_0\)（对错误答案的共识得分）和 \(f_1\)（对正确答案的共识得分）的Beta参数。
  2. 在每轮辩论后，LLM法官输出 \(s_r\)，累积对数似然比。
  3. 执行上述门槛比较，直到达到停止条件或 \(R_{\max}\)。
  4. 输出最终答案及决策类型（共识/无共识/截断）。

### 6. 论文的主要结论与发现

- **主要结论**：SPRT不再提高辩论准确率，而是作为低成本计算控制和故障检测层。当法官分数能区分正确与错误收敛时（如GSM8K），SPRT大幅减少调用次数（3.7倍）且准确率仅下降2pp；当法官分数无法区分时（如MMLU，KL≈0），SPRT正确拒绝提前停止，产生2.1倍成本增加，反而提示系统设计问题。
- **关键发现**：
  - 校准阶跃是成败关键：GSM8K校准显示很强分离（KL=9.8），MMLU校准几乎无分离（KL≈0）。
  - 在MMLU上，多数情况下（99.5%）达到上限，因为所有答案（正确或错误）都会得到高共识分数，法官无法分辨。
  - 模拟中，平均轮次比固定5轮减少约50%，错误率低于名义值；非i.i.d.时序相关性会轻微膨胀错误率（ρ=0.6时仍仅1.5倍）。
  - 简单阈值或稳定性规则远差于SPRT（准确性低或轮次高）。

- **方法优点**：

  - **校准导向**：不仅节省计算，还通过校准诊断揭示法官信号质量，提供部署前的安全阀门。
