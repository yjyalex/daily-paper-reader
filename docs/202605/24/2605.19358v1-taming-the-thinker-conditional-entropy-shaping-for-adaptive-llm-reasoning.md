## 摘要
基于熵的深度推理已成为提高大语言模型推理能力的一个有前途的方向，但现有方法往往要么不加区分地增加响应长度，要么以牺牲准确性为代价缩短响应。为了更好地平衡这一权衡，我们提出了条件熵塑造（CES），这是一个动态控制词级响应熵的框架，使大语言模型能够在简单问题上生成简洁的解决方案，同时鼓励在困难问题上进行更深入的探索。CES基于DAPO构建，将词级熵作为不确定性信号，并应用条件双向策略：在正确推理路径上惩罚高熵的“分叉点”词元以提高简洁性，在错误路径上则奖励这些词元以鼓励探索和纠错。我们在DeepSeek-R1-Distill-7B上实现了CES，并在12个数学基准上进行了评估。与DAPO相比，CES在持续提高平均准确率的同时减少了响应长度，补充实验在较小的1.5B骨干网络和领域外基准上显示出类似趋势。

## Abstract
Entropy-based deep reasoning has emerged as a promising direction for improving the reasoning capabilities of Large Language Models (LLMs), but existing methods often either increase response length indiscriminately or shorten responses at the cost of accuracy. To better balance this trade-off, we introduce Conditional Entropy Shaping (CES), a framework that dynamically controls token-level response entropy, enabling LLMs to produce concise solutions on simple problems while encouraging deeper exploration on hard ones. Built on DAPO, CES uses token-level entropy as an uncertainty signal and applies a conditional bidirectional policy: it penalizes high-entropy "forking point" tokens on correct reasoning paths to improve conciseness, and rewards them on incorrect paths to encourage exploration and error correction. We implement CES on DeepSeek-R1-Distill-7B and evaluate it on 12 mathematical benchmarks. CES consistently improves average accuracy while reducing response length relative to DAPO, and supplementary experiments show similar trends on a smaller 1.5B backbone and on out-of-domain benchmarks.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- 大语言模型（LLM）在复杂推理（如数学推导、代码生成）中展现出强大能力，但显式的链式思维（CoT）推理会导致生成大量 token，引发高延迟和计算成本，即“过度思考”（overthinking）现象——模型对简单问题（如“2+3=？”）也会产生冗长的推理步骤。
- 现有基于熵的深度推理方法，如奖励高熵 token 以鼓励探索（Cheng et al., 2025），能够提升准确性，但会进一步延长推理过程；而效率导向的方法（如惩罚过长正确答案）则可能牺牲复杂问题的性能。
- 核心矛盾：如何在保证准确性的同时控制推理长度，实现按需思考（adaptive reasoning）。
- 本文提出**条件熵塑造（Conditional Entropy Shaping, CES）**，旨在动态调节 token 级熵，使模型在简单问题上保持简洁，在困难问题上深入探索，从而**同时改善准确性和效率**。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 基于 DAPO 强化学习框架，将 token 级熵作为不确定性信号，根据推理路径正确性施加**条件双向控制**：
  - **正确路径**：对高熵“分叉点” token 施加**惩罚**（降低优势），鼓励模型更果断、简洁。
  - **错误路径**：对高熵 token 施加**奖励**（提高优势），激励模型探索替代路径、纠正错误。

### 关键技术细节
- **DAPO 框架**：组相对优势（group-relative advantage），组内归一化奖励，token 级策略梯度损失。
- **条件熵塑造三步法**：
  1. **组级计算**：生成 N 个响应，计算总奖励（R = 正确性奖励 + 格式奖励），组准确率 a = 正确响应比例，初始优势 A_i 基于总奖励组标准化。
  2. **动态选择高熵 token**：对每个响应 y_i，按比例 k_i = ⌊|y_i|·τ·b_i⌋ 选择熵最高的 token。b_i 根据正确性动态调整：正确时 b_i = a，错误时 b_i = 1-a。这使干预强度自适应问题难度（易题上更强惩罚，难题上更强奖励）。
  3. **熵基优势重塑**：对选中的高熵 token 修改优势：
     - 正确响应：A′ = A - β₁·H
     - 错误响应：A′ = A + β₂·H
     - 其他 token 保持不变。H 为 token 熵，β₁,β₂ 为缩放超参数。
- 熵项保留在计算图中，使模型能直接优化降低/提高熵的行为（区别于 Cheng et al. 的 detach 操作）。

- **CES 同时提升准确率和效率**：在 12 个数学基准上，平均准确率 72.1%（比 DAPO +2.5%），平均长度 1965 token（比 DAPO 缩短 411 token）。
