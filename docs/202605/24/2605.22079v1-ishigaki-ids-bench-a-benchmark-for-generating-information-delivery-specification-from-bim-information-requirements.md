## 摘要
大型语言模型（LLMs）被广泛用于生成结构化输出，如JSON、SQL和代码，但用于评估必须同时满足行业标准XML和领域词汇约束的生成的公共资源仍然有限。本文介绍了石垣-IDS-Bench，一个用于评估从建筑信息模型（BIM）信息需求生成信息交付规范（IDS）XML能力的基准测试。该基准包含166个由BIM/IDS专家创作和验证的示例，通过将83个实际场景扩展到日语和英语，相应的黄金IDS文件，以及输入格式、语言、回合设置、IFC版本和建筑领域的元数据。其评估结合了基于IDSAuditTool的可处理性、结构和内容审核，以及与黄金IDS文件的内容一致性评估。在10个LLM的零样本评估中，最佳模型在内容一致性上达到65.6%的宏F1，而只有27.7%的输出通过了内容审核。这些结果表明，当前的LLM可以将部分信息需求表达为IDS，但仍然难以稳定生成满足IDS标准和IFC词汇约束的XML。石垣-IDS-Bench支持比较评估、失败分析以及符合领域标准的约束结构化生成方法的开发。我们在GitHub和Hugging Face上以CC BY 4.0许可证发布评估脚本和基准数据。

## Abstract
Large language models (LLMs) are widely used to generate structured outputs such as JSON, SQL, and code, yet public resources remain limited for evaluating generation that must simultaneously satisfy industry-standard XML and domain vocabulary constraints. This paper presents Ishigaki-IDS-Bench, a benchmark for evaluating the ability to generate Information Delivery Specification (IDS) XML from Building Information Modeling (BIM) information requirements. The benchmark contains 166 BIM/IDS expert-authored and verified examples created by expanding 83 practical scenarios into Japanese and English, corresponding gold IDS files, and metadata for input format, language, turn setting, IFC version, and construction domain. Its evaluation combines IDSAuditTool-based Processability, Structure, and Content audits with content-agreement evaluation against gold IDS files. In zero-shot evaluation over 10 LLMs, the best model reaches 65.6% macro F1 for content agreement, while only 27.7% of outputs pass the Content audit. These results show that current LLMs can express part of the information requirements as IDS, but still struggle to stably generate XML that satisfies the IDS standard and IFC vocabulary constraints. Ishigaki-IDS-Bench supports comparative evaluation, failure analysis, and the development of constrained structured generation methods that conform to domain standards. We release the evaluation scripts and benchmark data under the CC BY 4.0 license on GitHub and Hugging Face.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
构建一个包含专家验证的输入-输出对（输入为 CSV 或自然语言需求，输出为黄金 IDS XML）的基准，并设计两阶段评估协议：第一阶段用 IDSAuditTool 评估形式有效性（可处理性、结构、内容），第二阶段用面（facet）级内容一致性评估与黄金 IDS 的匹配程度。

### 关键技术细节
- **数据集构建**：
  - 由 6 名 BIM/IDS 专家协作设计，收集约 100 个 IDS 用例候选，最终整理为 83 个场景，每个场景扩展为中英文两个版本，总共 166 个示例。
  - 每个示例附带元数据：输入格式（CSV/NL）、语言、回合设置（单轮/多轮）、IFC 版本、建筑领域。
  - 黄金 IDS 经过专家审核并通过 IDSAuditTool 验证。
- **两阶段评估协议**：
  - **第一阶段**：IDSAuditTool 输出的审计状态划分为三个指标：
    - **Processability**：所得 IDS 是否可被提取并进入完成状态（ok/structure-error/content-error）。
    - **Structure**：输出是否符合 IDS XML 结构和 XSD 模式。
    - **Content**：输出是否符合 IDS 标准和 IFC 词汇约束。
  - **第二阶段**：面（facet）评分器（facet scorer）比较黄金 IDS 与生成 IDS，针对 IFC 版本、实体（entity）、属性（attribute）、属性集（property）等面进行精确匹配，计算精确率、召回率、F1，并报告宏平均。
  - 不采用 LLM-as-judge，避免评估噪声。

### 关键约定
- 对于未明确指定属性集的情况，黄金 IDS 使用正则表达式 `^(?!(Pset_|Qto_)).+` 表示任意用户定义属性集（非标准规定，而是基准策略，在任务说明中明确告知所有模型）。

## 6. 论文的主要结论与发现

- **最佳模型**：GPT-5.5 在内容一致性面 F1 上达到 65.6%，但内容审计通过率仅 27.7%。
- **关键差距**：多数模型 Processability 高但 Content 低，表明 LLM 能输出可处理 XML，但难以同时满足 IDS 标准和 IFC 词汇约束。
- **模型间差异**：Kimi K2.6 的 Processability 和 Structure 高，但 Content 和面 F1 较低，说明形式有效性与内容恢复能力并不一致。
- **规模影响有限**：Qwen 系列从 8B 到 397B 规模扩大，面 F1 从 20.2% 提升至 26.8%，但 Content 通过率仅 10.2%~19.3%，表明单纯增大模型规模不足以解决领域标准约束。
- **多轮与 CSV 优势**：多轮设置（编辑现有 IDS）表现显著好于单轮；CSV 输入比自然语言输入更易映射到面和值约束。
- **主要失败类型**：
  1. 输出可提取为 XML 但违反 IDS XSD 或 IFC 词汇（Processability 与 Content 差距）。
  2. 属性集名称、基数、值约束表达错误频繁。
  3. 自然语言输入中目标与值约束对应关系模糊，导致缺少或过度生成面。

4. **多维度元数据**：支持按输入格式、语言、回合、IFC 版本、建筑领域进行细粒度分析，提高诊断价值。
