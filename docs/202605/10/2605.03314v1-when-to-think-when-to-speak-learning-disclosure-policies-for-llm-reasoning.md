## 摘要
在单流自回归接口中，相同的 token 既用于更新模型状态，又构成了不可逆的公开承诺。这种耦合产生了一种“沉默税”：额外的深思熟虑会推迟首个“任务相关”内容的出现，而简单的早期流式输出则面临过早承诺的风险，从而使后续生成产生偏差。我们引入了“并排（SxS）”交替推理，使“披露时机”成为标准自回归生成中一个可控的决策。SxS 在同一上下文中将部分披露与持续的私有推理交替进行，但仅在内容得到目前推理支持时才发布。为了在不激励填充内容的情况下学习这种节奏，我们通过将答案前缀与支持性推理前缀匹配，构建了蕴含对齐的交替轨迹，随后通过 SFT 训练以获取双重动作语义，并利用 RL 在新格式下恢复推理性能。在两种 Qwen3 架构/规模（MoE Qwen3-30B-A3B，稠密 Qwen3-4B）以及域内（AIME25）和域外（GPQA-Diamond）基准测试中，SxS 在 token 级代理指标（如更新间等待时间）下改善了准确率与内容延迟之间的帕累托权衡。

## Abstract
In single-stream autoregressive interfaces, the same tokens both update the model state and constitute an irreversible public commitment. This coupling creates a \emph{silence tax}: additional deliberation postpones the first \emph{task-relevant} content, while naive early streaming risks premature commitments that bias subsequent generations. We introduce \textbf{\emph{Side-by-Side (SxS)}} Interleaved Reasoning, which makes \emph{disclosure timing} a controllable decision within standard autoregressive generation. SxS interleaves partial disclosures with continued private reasoning in the same context, but releases content only when it is \emph{supported} by the reasoning so far. To learn such pacing without incentivizing filler, we construct entailment-aligned interleaved trajectories by matching answer prefixes to supporting reasoning prefixes, then train with SFT to acquire the dual-action semantics and RL to recover reasoning performance under the new format. Across two Qwen3 architectures/scales (MoE \textbf{Qwen3-30B-A3B}, dense \textbf{Qwen3-4B}) and both in-domain (AIME25) and out-of-domain (GPQA-Diamond) benchmarks, SxS improves accuracy--\emph{content-latency} Pareto trade-offs under token-level proxies (e.g., inter-update waiting).

### 论文总结：何时思考，何时发言——学习大语言模型推理的披露策略

#### 1. 核心问题与整体含义（研究动机和背景）
在标准的自回归大语言模型（LLM）中，生成的 token 既用于更新模型的内部状态，又作为不可逆的公开输出。这种“耦合”导致了**“沉默税（Silence Tax）”**问题：
*   **高延迟：** 为了保证准确性，模型需要进行长链条推理（CoT），导致用户在看到有效答案前必须等待漫长的推理过程。
*   **过早承诺风险：** 如果为了降低延迟而让模型过早输出答案，可能会因为缺乏充分推理而产生错误，且这些错误输出会进一步误导后续的生成。
论文旨在打破这种耦合，研究如何让模型自主决定“何时思考（私有推理）”以及“何时发言（公开披露）”，从而在不牺牲准确性的前提下优化响应速度。

#### 2. 方法论：Side-by-Side (SxS) 交替推理
核心思想是将披露时机建模为自回归过程中的一个**可控决策变量**。
*   **双重动作语义：** 引入 `think`（私有推理，用户不可见）和 `speak`（公开答案，用户可见）两种动作标签。所有 token 都在同一个上下文流中，但只有被标记为 `speak` 的部分会被实时推送给用户。
*   **蕴含对齐的监督数据构建（SFT 阶段）：**
    *   **分段与匹配：** 将原始的推理过程 $r$ 和答案 $a$ 分段。
    *   **蕴含检查：** 使用一个强大的模型（如 GPT-OSS-120B）作为评判者，检查当前的推理前缀是否足以支持（蕴含）某一段答案前缀。
    *   **交替轨迹：** 只有当推理进度足以支持答案时，才插入答案块，构建出“推理-答案-推理”交替的训练样本。
*   **强化学习恢复（RL 阶段）：**
    *   使用 **GRPO（群体相对策略优化）** 算法。
    *   **目标：** 修复 SFT 带来的分布偏移（推理过程被切碎），恢复并提升推理准确率。
    *   **奖励设计：** 主要基于结果正确性（Outcome-based），可选地加入“粒度奖励”以鼓励更频繁的交替输出。

#### 3. 实验设计
*   **模型：** Qwen3-30B-A3B（混合专家模型 MoE）、Qwen3-4B（稠密模型）。
*   **基准测试（Benchmarks）：**
    *   **域内（数学）：** AIME25。
    *   **域外（科学）：** GPQA-Diamond。
    *   **其他：** LiveCodeBench（代码）、KOR-Bench（逻辑推理）。

#### 5. 实验数量与充分性
*   **实验规模：** 涵盖了两种截然不同的架构（MoE 与 Dense）和两种参数规模，证明了方法的普适性。
*   **充分性：** 实验不仅关注最终准确率，还深入分析了推理块的粒度、RL 训练的动态过程以及域外泛化能力。
*   **客观性：** 通过对比 Base 模型、SFT 后模型及 RL 最终模型，清晰展示了“性能下降-恢复-超越”的曲线，评估较为客观。

#### 6 主要结论与发现
*   **打破帕累托边界：** SxS 在保持或提升 AIME25 准确率的同时，显著降低了用户感知的等待时间（AIRW 指标大幅下降）。
*   **RL 的必要性：** SFT 虽能教会模型交替格式，但会导致准确率骤降；RL 是恢复推理逻辑、实现性能反超的关键。
*   **域外鲁棒性：** 在 GPQA 任务上，SxS 表现出比标准 CoT 更强的抗遗忘能力，可能是因为交替格式强制了推理与答案的紧密耦合。
*   **可控性：** 通过奖励塑形，可以人为控制模型输出的频率（粒度），且不会严重损害准确率。

#### 7. 优点
*   **架构兼容性：** 无需修改模型结构，仅通过标签和数据工程即可实现，易于部署在现有自回归框架中。
*   **解决“填充”问题：** 不同于以往通过奖励减少 token 的方法，SxS 强调“有据可依”的披露，避免了模型生成无意义的填充词来刷低延迟指标。
*   **实时交互感：** 显著提升了长推理任务中的用户体验，使模型看起来在“边思考边汇报”。

#### 8. 不足与局限
*   **对齐成本高：** 预处理阶段需要调用大型模型进行大量的蕴含检查，计算开销较大且存在噪声。
*   **推理顺序限制：** 如果原始数据的答案顺序与推理逻辑不匹配（例如先给结论再给细节），SxS 的对齐机制可能会失效或退化。
*   **奖励函数单一：** 目前主要依赖结果奖励，对于“披露内容是否真的对用户有用”缺乏更细粒度的评估。
*   **潜在的训练不稳定性：** RL 阶段需要额外的恢复步数，且交替格式增加了序列长度和训练复杂性。
