## 摘要
为语言模型配备数十到数百个工具定义的智能体RAG系统面临一个关键资源冲突：工具模式占用了检索增强生成所需的相同上下文窗口。我们首次对这一工具-上下文权衡进行了系统研究，评估了14个模型（包括1.5B-32B参数的本地模型和一个前沿API模型），在三个上下文预算（8K、16K、32K）下使用28个工具定义进行了6,566次受控API调用。应用TSCG保守配置压缩（节省44-50%的模式令牌），我们观察到一种二元启用效应：在8K令牌时，JSON模式工具定义完全溢出上下文窗口，导致接近零的精确匹配（平均2.6%），而压缩模式恢复了RAG功能，所有八个模型的平均精确匹配提升为+20.5个百分点（在六个显示出完全启用的模型中为+24.7个百分点）。在32K时——两种格式都适合——五个测试模型中的四个显示差值<=1个百分点，确认该效应纯粹由预算驱动。在HotpotQA（50个多跳问题）上的外部验证显示，在相同溢出场景下精确匹配提升了+48个百分点。前沿扩展测试表明，JSON模式在大约494个工具时溢出，而压缩模式在超过800个工具时仍可运行。我们的结果确立了工具模式压缩作为受限上下文部署中智能体RAG的必要基础设施层。所有代码、数据和检查点均已公开。

## Abstract
Agentic RAG systems that equip language models with dozens to hundreds of tool definitions face a critical resource conflict: tool schemas consume the same context window needed for retrieval-augmented generation. We present the first systematic study of this tool-context trade-off, evaluating 14 models spanning 1.5B-32B local models plus one frontier API model across 6,566 controlled API calls at three context budgets (8K, 16K, 32K) with 28 tool definitions. Applying TSCG conservative-profile compression (44-50% schema token savings), we observe a binary enablement effect: at 8K tokens, JSON-schema tool definitions overflow the context window entirely, yielding near-zero EM (2.6% average), while compressed schemas restore RAG functionality with +20.5 pp average exact-match lift across all eight models (+24.7 pp among the six exhibiting full enablement). At 32K -- where both formats fit -- four of five tested models show delta <= 1 pp, confirming the effect is purely budget-driven. External validation on HotpotQA (50 multi-hop questions) shows +48 pp EM under the same overflow scenario. Frontier scaling tests demonstrate that JSON schemas overflow at ~494 tools while compressed schemas remain operational beyond 800 tools. Our results establish tool-schema compression as a necessary infrastructure layer for agentic RAG in constrained-context deployments. All code, data, and checkpoints are publicly available.

# 论文结构总结

## 1. 核心问题与整体含义
**研究动机与背景**：智能体RAG系统（Agentic RAG）需要同时容纳工具定义（Tool Schemas）和检索增强生成所需的上下文窗口。工具定义通常以冗长的JSON Schema形式呈现，每个工具消耗300–500 token。当工具数量达到28个时，JSON schema块占用约11,000 token，这使得在8K上下文预算下完全溢出，无法进行任何RAG。现有研究（如提示压缩）主要针对自然语言，忽略了结构化工具定义的压缩需求。本文首次系统研究工具–上下文权衡（tool–context trade-off），将工具schema压缩视为受限上下文部署中的必要基础设施层。

## 2. 方法论
- **核心思想**：通过确定性、规则驱动的工具schema压缩（TSCG），减少schema占用的token数，释放空间用于RAG chunks，从而在上下文预算紧张时恢复检索功能。
- **关键技术细节**：
  - 使用TSCG保守配置（conservative profile），保留描述信息，节省44–50%的schema token（28个工具时从约11,000 token降至约5,500 token）。
  - 上下文预算分配模型：\( B_{RAG} = B - B_{sys} - B_{schema}(f, n) - B_{hist} - B_{out} \)（公式1），其中 \( B \) 为总上下文窗口，\( B_{sys} \approx 350\) token，\( B_{hist}=1,500\) token，\( B_{out}=512\) token，\( B_{schema} \) 取决于格式 \( f \) 和工具数 \( n \)。
  - 在8K预算下，JSON格式导致 \( B_{RAG} < 0 \)（溢出），TSCG压缩后可将 \( B_{RAG} \) 恢复至约265–330 token，从而容纳至少1个RAG chunk。
- **算法流程**（文字说明）：采用ReAct风格的agent循环，最多3次迭代。每次迭代，系统将系统提示、工具schemas、RAG chunks（根据可用预算填充）和历史对话拼接后输入模型，模型决定调用工具或给出最终答案。唯一变量是schema格式（json vs tscg）及其导致的预算分配差异。

## 6. 主要结论与发现
1. **二元启用效应（Binary Enablement）**：
   - 在8K上下文预算下，JSON schema完全溢出（2.6%平均EM），TSCG压缩后恢复RAG功能，平均EM提升+20.5个百分点（六模型完全启用效果+24.7 pp）。
2. **预算驱动而非格式内在效应**：在32K（两种格式均适配）时，四个模型中三个的EM差异≤1 pp，符合预算再分配解释。
3. **外部验证**：HotpotQA上相同溢出场景，TSCG带来+48 pp EM增益。
4. **前沿扩展**：JSON schema在大约494个工具时溢出，TSCG在超过800个工具时仍可运行，运营范围扩展63%。
5. **上下文利用模型**：第一个RAG chunk提供最大边际增益（\( C(k) = C_{max}(1 - e^{-\lambda k}) + C_0 \)，\(\lambda\) 大表示饱和快），解释了为何压缩让少量chunk就能显著提升。
6. **分心稀释效应（Distractor Dilution）**：在16K预算下，TSCG提供更多chunk，但对小模型（≤8B）反而降低精度，因为额外内容成为干扰。

## 8. 不足与局限
- **基准合成性**：NovaTech-28是自建合成基准，工具定义和问题人工设计，真实场景可能更复杂（多轮对话、异构语料）。

- **小模型覆盖不足**：Tier C模型（≤7B）仅在16K下测试，分心稀释结论可能不具普适性。
- **前沿扩展局限**：仅使用Sonnet 4一个模型在200K上下文下测试，合成工具集可能偏离真实分布；未测试其他模型或真实MCP工具目录。
- **静态上下文填充**：实验采用固定填充策略，未评估动态检索或自适应chunk选择可能带来的不同交互效果。
- **统计效应**：16K下的无差异结果可能受限于合成基准的分辨率（chunk饱和后增益有限），在更难的检索任务中压缩可能仍有潜在价值。
