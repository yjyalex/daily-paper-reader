## 摘要
基于大语言模型的多智能体系统需要许多难以预先确定的协调选择：应该调用哪个技能协议？哪个智能体角色应该执行子任务？每个角色应该绑定哪个模型？角色之间应如何交互？何时使用检索或验证？何时完全省略某个步骤？这些选择与任务模式和操作约束相互作用，因此静态流水线和一次性的模型比较仅能提供设计空间的有限视角。本文介绍AgensFlow，一个开源框架，它将多智能体协调视为部分可观测性下的在线策略学习问题。该框架使协调决策可观察并可从重复轨迹中学习，而不是将技能、角色、模型、拓扑和评估选择视为固定的流水线设计。AgensFlow在两个语料库上进行了评估：分布式系统事件任务和安全咨询任务。评估展示了三个主要结果：学习到的路由在协调密集型类别上达到了比固定流水线基线更高质量的工作点；skip:X将拓扑压缩隔离为框架的有意义部分；热启动的策略图可以在保持平台质量的同时降低探索成本。总体而言，结果表明，与静态布线相比，可学习的、可审计的路由可以改进协调密集型多智能体工作流程。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

- **核心问题**：如何使多智能体系统的协调决策可观察、可学习、可审计，而非固定在一次性管道设计中？

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- 将多智能体协调建模为**部分可观测的序贯决策过程**，学习一个可复用的**策略图（policy graph）**，该图以折叠的任务签名（folded task signature）为状态，动作空间覆盖技能协议、模型绑定、拓扑选项（包括跳过步骤）。
- **状态表示**：折叠签名 \( s_t = \phi(x_t, h_t, b_t) \)，由规则检测器生成的场景标签（如 evidence_heavy、ambiguous）、二值握手掩码（handoff mask）以及四个连续信念估计（正确性、不确定性、矛盾风险、证据充分性）经过分桶后组成。
- **动作空间**：每个步骤选择调用一个技能+模型组合（如 solver_cot_haiku），或选择 `skip:X` 省略某个尚在规划中的单元，或触发终止（由评估器间接触发）。
- **奖励函数**：混合奖励 \( r(t) = w_q Q(t) - w_c C(t) - w_\rho \rho(t) \)，其中 \( Q(t) \) 为相对评估（RelativeJudge）质量得分，\( C(t) \) 为归一化令牌成本，\( \rho(t) \) 为重试/失败惩罚。默认权重 \( w_q=1.0, w_c=0.3, w_\rho=0.15 \)。
- **动作选择**：采用可靠性感知的 UCB1 变体，包含退火探索系数和失败惩罚项：\(\text{score}(s,a) = \bar{r}(s,a) + c_s \sqrt{\frac{\log(N_s+1)}{N_{s,a}}} - \lambda f(s,a)\)，其中探索系数随签名访问次数退火（从1.4到0.5，半衰期50次访问）。

### 关键技术细节
- **策略图**：持久化、可审计，记录每个(状态,动作)对的访问次数、平均奖励、方差、令牌统计和失败计数。
- **RelativeJudge**：同一任务类内的多个轨迹按照显式评分准则进行相对排序，产生[0,1]标量得分及子分数（目标达成、基础事实、协调、恢复）。支持跨模型家族裁判平均（如 Claude + GPT + Qwen），并基于裁判间分歧计算置信度加权。
- **跳过拓扑学习**：`skip:X` 作为一等动作，允许策略学习针对不同任务类省略某些步骤（如跳过检索或验证），将拓扑形状本身变成可学习的变量。
- **六层依赖结构**：从下到上为 Web搜索封装（层0）→ 模式/场景/技能注册（层1）→ 智能体/传输/追踪（层2）→ 策略图/路由器/LangGraph集成（层3）→ 奖励+裁判（层4）→ 预检/治理/报告（层5）。

### 数据集/场景
- **语料一**：60个分布式系统事件任务，覆盖Paxos/Raft、逻辑时钟、gossip协议、CRDT、一致性哈希、故障检测等概念和推理模式。

## 6. 主要结论与发现
1. **学习路由优于固定基线**：在主方案平台期（epoch 6-8），三裁判审计下 RelativeJudge 得分比固定基线高0.068（0.817 vs 0.749），令牌成本仅增加15%。
2. **拓扑压缩（skip:X）是框架的有效组成部分**：无跳过消融导致令牌成本增加94%（25,198 vs 12,960），而质量并未提升，表明跳过决策是提高效率的关键。
3. **热启动降低探索成本**：热启动从预处理策略图初始化，令牌成本降低约10%（平台期）至21%（全程），同时在三裁判审计下保持了平台质量（0.829 vs 0.817），且单裁判下的质量优势被审计削弱，凸显裁判审计的必要性。
4. **奖励审计是必要性设计**：跨家族三裁判审计改变了热启动质量优势的幅度（从单裁判的+0.055降低到+0.003），说明在线奖励信号不可靠，必须作为系统的一部分进行审计。
5. **协调密集型类别收益最大**：主方案在C3（跨文档多供应商，+0.18）、C8（跨供应商对，+0.16）、C7（缓解正确性，+0.13）上提升最大，程序性类别（C1、C5）没有明显提升或小幅下降。

## 7. 优点
- **框架设计创新**：将多智能体协调形式化为可学习的在线策略问题，并使拓扑（跳过步骤）成为一等动作，超越了固定流水线范式。
- **可审计性**：策略图记录所有状态-动作对的统计数据，奖励信号通过多裁判集成和置信度加权实现透明审计，这是实际部署中非常实用的特性。
- **跨域验证与迁移实验**：展示了同一框架在两个不同主题语料库上的转移能力，并量化了热启动的收益。
- **详细的类别级别分析**：按8个场景类呈现结果，揭示了不同任务类型对学习路由的响应差异，提供了深刻见解。

## 8. 不足与局限
- **拓扑覆盖有限**：仅评估了线性+跳过拓扑，未包括并行、分支/合并、层次化规划等更丰富的形式。
