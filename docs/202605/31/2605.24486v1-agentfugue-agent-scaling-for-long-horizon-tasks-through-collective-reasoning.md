## 摘要
近期在长期智能体任务上的进展主要通过更强大的模型、更好的工具和更有效的脚手架来扩展单个智能体。相比之下，对于横向扩展的理解要少得多：多个对等智能体，全部针对同一任务，能否成为能力增长的额外来源，而不依赖于明确的角色分工或工作流编排。我们研究这一问题，并提出AgentFugue，一个围绕共享推理中心构建的集体推理框架。当对等智能体并行探索同一任务时，中心记录每个智能体已建立、尝试或排除的简洁笔记，并使每个智能体能够以对当前搜索有用的形式选择性访问其他智能体的发现。这种设计将原本孤立的轨迹转化为可重用中间推理的互联生态系统，无需集中规划。我们将该中心实例化为一个即插即用的通信层，使用监督微调和端到端强化学习进行训练。在我们研究的具有挑战性的长期设置中，AgentFugue优于强基线。我们的结果表明，集体推理可以将对等智能体系统的横向扩展转化为能力增长的独特来源，而不仅仅是消耗更多计算的一种方式。

## Abstract
Recent progress on long-horizon agentic tasks has been driven largely by scaling up individual agents through stronger models, better tools, and more effective scaffolding. In contrast, much less is understood about scaling out: whether multiple peer agents, all targeting the same task, can become an additional source of capability without relying on explicit role specialization or workflow orchestration. We study this question and propose AgentFugue, a collective reasoning framework built around a shared reasoning hub. As peer agents explore the same task in parallel, the hub records concise notes on what each agent has established, attempted, or ruled out, and enables each agent to selectively access what other agents have discovered in a form useful for its current search. This design turns otherwise isolated trajectories into a connected ecology of reusable intermediate reasoning without requiring centralized planning. We instantiate the hub as a plug-in communication layer, trained with supervised fine-tuning and end-to-end reinforcement learning. Across the challenging long-horizon settings we study, AgentFugue improves over strong baselines. Our results suggest that collective reasoning can turn scaling out peer agent systems into a distinct source of capability gains, rather than merely a way of spending more compute.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在长期（long-horizon）智能体任务中，已有研究主要关注“垂直扩展”（scaling up）——通过更强的模型、更好的工具和更有效的脚手架提升单个智能体的能力。但对“横向扩展”（scaling out）——多个对等（peer）智能体同时处理同一任务，能否不依赖明确的角色分工或工作流编排，而成为能力增长的额外来源——理解甚少。
- **研究动机**：如果有多个智能体并行探索同一任务，它们各自可能发现不同的部分推理路径、中间证据或失败分支。如何让这些探索相互受益，而不仅仅是简单地增加计算量（如独立采样后汇总），是本文要回答的问题。
- **整体含义**：提出集体推理（collective reasoning）的概念，通过共享的推理中心（reasoning hub）连接智能体的轨迹，使中间进展可复用，从而将横向扩展转化为独立的能力增益来源，而非仅消耗更多计算。

## 2. 方法论

### 核心思想
- 构建一个**共享推理中心（shared reasoning hub）**，作为外部通信层，而非集中规划器。中心记录每个智能体完成的中期“情节”（episodes）的简要笔记，并允许其他智能体根据当前需求选择性访问这些笔记。这种设计使并行轨迹形成一个“连接的推理生态”，同时保持各智能体本地轨迹的独立性。

### 关键技术细节
- **情节写入与上下文驱逐**：当智能体的本地上下文窗口达到固定预算（如64K tokens）时，当前交互段被封闭为一个情节。中心模型将其压缩成情节笔记（episode note），记录已建立的内容、尝试过的分支、被排除的选项等。原始情节内容被驱逐，但保留在中心存储中供深度读取。智能体的工作上下文包含自己的历史笔记、队友的笔记以及当前未完成的交互段。
- **意图驱动读取**：智能体并非被动接收所有队友笔记。当它判断需要更深入查看某个队友的工作时，会向中心发送结构化请求，包含意图（如需要哪类信息）以及要检查的情节引用。中心检索原始内容并合成针对性反馈。
- **两阶段优化**：中心从Qwen3.5-9B初始化，分为`M_write`和`M_read`实例。
    1. **监督微调（SFT）**：使用教师模型生成参考笔记和参考读出力，以标准语言模型损失训练。
    2. **组相对策略优化（GRPO）**：在完整的多智能体循环中对齐中心，保持任务智能体冻结。对每个实例采样G个候选中心输出，计算组相对优势，并结合任务成功率和简洁性奖励进行优化。
- **两种横向扩展形式**：
    - **同质团队**：所有智能体使用相同模型和配置，增益必须来自交叉轨迹交互。
    - **异质团队**：智能体使用不同模型或设置，中心还可以调节互补优势。

## 3. 实验设计

### 数据集/基准
- **BrowseComp**：深度多跳网络搜索与跨文档证据聚合，需要短的事实性答案。使用200题随机样本（缩放研究时用100题子集）。
- **WideSearch**：强调广度证据收集，要求列举和整合多条并行信息。使用完整测试集。
- **HLE（Humanity's Last Exam）**：专家编写的多领域推理基准，侧重多步推理而非网络导航。使用200题随机样本。

### 对比方法
- **单智能体ReAct**：采用标准ReAct循环的多种前沿模型（Claude-Opus-4.5、Kimi-K2.5、Qwen3.5-35B-A3B、GLM-4.7、DeepSeek-v4-Flash等）。
- **单智能体DeepResearch**：具有扩展脚手架的专用长时研究系统（WebThinker、WebSailor、AgentFold、IterResearch、Tongyi-DeepResearch、OpenAI DeepResearch）。
- **多智能体系统**：
    - **Naive-Multi-Agent**：先规划→并行搜索→汇总元智能体。
    - **Swarm-Multi-Agent**：基于Kimi-K2.5 swarm设置，元智能体可动态创建和分配子智能体。
- 所有多智能体系统共享相同的每智能体工具堆栈和交互预算，以确保差异来自协调机制而非能力。

## 6. 主要结论与发现

- AgentFugue在三个基准上均显著优于所有基线，包括同质和异质团队设置。
- 横向扩展是有效的：增加同质智能体数量可提升每个智能体的平均性能，约在N=5时饱和；团队级聚合指标（Pass@N、BoN等）随N单调上升。
- 共享推理中心将孤立探索转化为协调：随着团队扩大，每智能体的搜索/访问调用减少，而内存调用增加，表明资源从重复探索转向结构化共享。
- 在异质团队中，弱骨干模型获益最多，强模型（如Kimi-K2.5）也有小幅度提升。异质团队产生的内存流量远高于同质团队，中心价值更大。
- 中心上下文窗口大小存在最佳点（32K），过小或过大都会降低性能（Table 2）。

## 7. 优点

- **问题新颖**：提出了横向扩展作为独立于纵向扩展的能力增长方向，突破了传统“角色分工”或“聚合采样”的框架。
- **方法简单有效**：通过共享中心以笔记形式交换中间进展，避免了集中规划和全广播通信，保持了智能体探索的多样性。
- **两阶段优化**：SFT加GRPO使中心不仅学会总结，还能生成对下游搜索真正有用的指导。
- **实验设计细致**：同时考察同质和异质团队，并分析多种聚合策略（Pass、BoN、MV、WMV、FewTool），全面验证了方法的鲁棒性。
- **案例研究深入**：成功案例展示了中心如何传递“失败地图”加速探索；失败案例揭示了确认偏差如何通过共享笔记被放大，为未来改进提供了方向。

- **资源开销未量化**：论文未报告训练中心所需的实际计算资源（GPU型号、数量、时长），透明性不足。
- **集体推理的失败模式**：如附录G所讨论，笔记质量低或过度自信可能导致误导结论在团队中传播；高显著性笔记可能降低探索多样性导致过早收敛。这些风险仅在案例研究中初步展示。
