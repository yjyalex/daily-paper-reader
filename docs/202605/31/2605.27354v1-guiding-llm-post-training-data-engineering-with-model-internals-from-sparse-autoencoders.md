## 摘要
模型内部状态编码了大语言模型（LLM）处理其训练数据时的丰富信息；然而，后训练数据工程主要依赖外部信号，而忽略了模型内部蕴含的丰富内在信号。我们提出SAERL，一个用于LLM强化学习（RL）的数据工程框架。它利用通过稀疏自编码器（SAE）提取的模型内部状态，对三个内在数据属性进行建模：多样性、难度和质量。每个属性对应一个具体的数据工程操作：通过SAE空间聚类并适度混合批次来控制批次多样性；通过难度代理实现从易到难的课程排序；通过质量探针进行数据过滤。SAERL在Qwen2.5-Math-1.5B上相比原始GRPO平均准确率提升3.00%，并在达到目标准确率时减少20%的训练步数，且在不同模型规模和RL算法上均有一致提升。实验表明，SAE可有效跨模型家族和规模迁移，作为一种轻量级且可复用的数据工程工具。这些结果证明，模型内部状态是后训练数据工程中强大且实用的信号来源。

## Abstract
Model internals encode rich information about how a large language model (LLM) processes its training data; however, post-training data engineering largely relies on external signals and ignores rich intrinsic signals lying in model internals. We propose SAERL, a data engineering framework for LLM reinforcement learning (RL). It models three intrinsic data properties: diversity, difficulty, and quality, using model internals extracted with Sparse Autoencoder (SAE), an advanced mechanistic interpretability tool. Each property grounds a concrete data engineering operation: SAE-space clustering with moderate batch mixing for batch diversity control, a difficulty proxy for easy-to-hard curriculum ordering, and a quality probe for data filtering. SAERL improves average accuracy by 3.00% over vanilla GRPO and reaches target accuracy with 20% fewer training steps on Qwen2.5-Math-1.5B, with consistent gains across model scales and RL algorithms. Experiments show that SAE transfers effectively across model families and scales, serving as a lightweight and reusable data engineering tool. These results demonstrate that model internals are a powerful and practical source of signals for post-training data engineering.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前LLM后训练（特别是强化学习）的数据工程严重依赖外部信号（如人类偏好、验证器结果、 rollout准确率等），这些信号成本高、难以全面获取，而模型内部状态蕴含的丰富反馈信号尚未被充分利用。
- **整体意义**：本文提出利用稀疏自编码器（SAE）提取模型内部表示，建模数据的多样性、难度和质量三种内在属性，并将其映射为具体的工程操作（批次混合、课程排序、数据过滤），从而为后训练数据工程提供一种轻量、可复用且跨模型迁移的内部信号来源。这项工作填补了“机械可解释性”与“实际训练优化”之间的空白。

## 2. 方法论

### 2.1 核心思想

SAERL框架将每个数据样本的token级SAE激活（分别从prompt和solution部分提取）通过均值池化与最大值池化聚合为统一表示 $\phi(x_i)$（包含960维SAE特征 + 26维元数据特征），然后基于该表示建模三个内在属性：

- **多样性**：通过MiniBatchKMeans在SAE空间聚类（K=10），并在批次构建时进行适度跨簇混合（尾部交换），平衡梯度一致性与覆盖度。
- **难度**：在少量标注子集上训练ElasticNet回归器预测难度分数，再通过簇级收缩校准得到最终排序分数，用于从易到难的课程排序。
- **质量**：在少量源标签上训练线性分类器（SGD优化）作为质量探针，输出属于目标分布的概率，用于过滤低质量样本。

### 2.2 关键技术细节

- SAE空间聚类：MiniBatchKMeans，K=10。
- 适度批次混合：每个批次与一个来自相邻课程阶段、不同主导簇的伙伴批次交换尾部h个样本。
- 难度校准：对原始预测分数应用全局映射，再叠加基于簇的收缩残差。
- 质量过滤：基于探针分数设定百分位阈值或Top-k选择。

### 2.3 算法流程（文字说明）

1. 对原始数据池，用SAE质量探针过滤，保留高概率为目标分布的样本。
2. 对过滤后的样本，在SAE空间聚类，并在每个簇内按校准难度从易到难排序，划分批次。
3. 构建全局课程：跨簇交错排序批次，并在同一课程阶段内对邻近批次进行适度尾部交换，形成最终训练顺序。
4. 将课程数据送入RL算法（GRPO/DAPO）训练。

## 6. 主要结论与发现

1. **性能提升**：SAERL在Qwen2.5-Math-1.5B上平均准确率较原始GRPO提升3.00%，在7B规模上也取得最佳平均结果；且在使用DAPO时同样有效。
2. **效率提升**：达到目标准确率所需的训练步数平均减少20%以上，在1.5B规模下SAERL_D平均仅需206步，而GRPO需470步。
3. **跨模型迁移**：一个在Qwen3-1.7B上训练的SAE可有效指导Qwen2.5-Math-1.5B和7B的后训练，证明SAE作为轻量级工具的跨家族泛化能力。
4. **质量过滤**：SAE探针以0.991 AUC区分DeepMath与原始语料，Top-50k选择可达到99.92%纯净度。
5. **可解释性**：SAE簇不直接对应人类主题标签，而是捕捉推理结构、格式、难度等课程相关属性；难度代理主要依赖SAE特征（前20特征中19个来自SAE），为课程构建提供了可审计路径。

## 7. 优点

- **方法创新**：首次系统性地将SAE这一机械可解释性工具应用于RL后训练数据工程，建模多样、难度、质量三种内在属性，填补了内部信号利用空白。
- **实用性强**：SAE可离线训练一次，跨模型家族和规模迁移，预处理开销低（全数据集编码<1 GPU小时），对比需要大量rollout的ADARFT等基线更高效。
- **实验严谨**：覆盖多个模型规模、两种RL算法、充分消融、批次大小分析、可解释性验证；对比基线包括使用不同能力信号的方法，公平客观。
- **可解释性收益**：SAE聚类和特征分析提供了数据难度的可审计解释，有助于理解课程构建逻辑。

## 8. 不足与局限

- **领域局限**：实验仅在数学推理与可验证奖励场景下进行，尚未验证在代码RL、agentic RL、工具使用、通用指令跟随等后训练设置中的适用性。
- **有监督依赖**：难度代理需要3,000个难度标签，质量探针需要源/分布标签，并非完全无监督。论文指出未来可探索更弱监督或自校准方法。
- **理论分析不充分**：对SAE距离与训练动力学的因果关系仅提供“优化级”解释（附录A的偏差-方差分解假设梯度与SAE表示局部Lipschitz连续），未提供严格的梯度级因果证明。
- **潜在偏差风险**：SAE激活编码了模型内部偏见，若用于优化不安全行为可能被滥用。论文仅在数学推理场景实验，建议安全筛选与人工监督。
- **实验规模**：最大模型7B，未验证更大规模（如70B+）的表现；batch size分析显示大batch下收益减小，趋势需进一步验证。
