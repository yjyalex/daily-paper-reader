## 摘要
有效的多智能体系统不能通过孤立地选择提示或通信图来设计。智能体行为取决于智能体接收到的信息，而通信边的有用性则取决于接收智能体如何解释和使用该信息。我们提出 \textbf{TCP-MCP}（面向多智能体协作问题求解的拓扑耦合提示），这是一个将智能体提示和通信拓扑作为统一基因组进行搜索的协同进化框架。TCP-MCP 使用初始化时的景观探针来校准早期搜索行为，然后依靠帕累托前沿诊断在三个目标（任务性能、令牌成本和结构复杂度）下调整探索。在相同 DeepSeek-V3.2 骨干网络上，TCP-MCP 在 MMLU-Pro、MMLU 和 GSM8K 上分别达到 82.66%、89.96% 和 96.61% 的准确率。在三个基准测试中，它一致地优于自动图生成基线，并与辩论式系统相比取得了有竞争力的准确率，同时在报告的工作点上使用的令牌数最多比这些系统少 5.69 倍。这些结果表明，在受控评估中，联合进化提示和通信结构为成本感知和任务自适应的多智能体系统设计提供了一条实用路径。

## Abstract
Effective multi-agent systems cannot be designed by selecting prompts or communication graphs in isolation. Agent behavior depends on the information an agent receives, while the usefulness of a communication edge depends on how the receiving agent interprets and uses that information. We propose \textbf{TCP-MCP} (Topology-Coupled Prompting for Multi-Agent Collaborative Problem-Solving), a co-evolution framework that searches agent prompts and communication topologies as a unified genome. TCP-MCP uses an initialization-time landscape probe to calibrate early search behavior, and then relies on Pareto-front diagnostics to adapt exploration under three objectives: task performance, token cost, and structural complexity. Using the same DeepSeek-V3.2 backbone across all methods, TCP-MCP achieves 82.66\%, 89.96\%, and 96.61\% accuracy on MMLU-Pro, MMLU, and GSM8K, respectively. Across the three benchmarks, it consistently outperforms automated graph-generation baselines and achieves competitive accuracy relative to debate-style systems, while using up to 5.69$\times$ fewer tokens than those systems at the reported operating points. These results show that jointly evolving prompts and communication structure provides a practical route to cost-aware and task-adaptive multi-agent system design in controlled evaluations.

# TCP-MCP 论文结构化总结

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：多智能体系统（MAS）的性能取决于智能体提示（prompt）与通信拓扑（communication topology）的**耦合关系**。现有方法通常孤立地设计提示或拓扑，忽略了二者的相互作用：一个提示在某种通信模式下有效，在另一种可能失败；一条通信边的价值取决于接收智能体如何解释和使用该信息。
- **整体含义**：MAS 设计应被视为提示与拓扑的**联合优化**问题，而非独立的提示工程或拓扑搜索。TCP-MCP 旨在解决这一耦合优化挑战，使系统能够同时适应任务需求、成本约束和结构复杂度。

## 2. 方法论

### 2.1 核心思想
- 将提示和通信拓扑编码为**统一基因组** `M = (G, P)`，其中 `G` 是有向图（节点=角色智能体，边=通信链接），`P` 是节点级角色-提示分配。
- 提示模板与拓扑分开存储，但**运行时根据拓扑上下文实例化**（BuildPrompt），使得同一提示在不同拓扑下行为不同。
- 优化目标：任务准确率（A）、token 成本（C）、结构复杂度（K = |V|+|E|），构成多目标最大化问题：`F(M) = (A(M), -C(M), -ln(1+K(G)))`。

### 2.2 关键技术细节
- **初始化景观探测**：计算偏好分数 `S(M) = A(M)^k / ((C/T0)^γ * K̄(G)^β)`，然后计算 Fitness Distance Correlation (FDC) 以校准早期搜索偏向。
- **结构交叉（PMI）**：交换拓扑子图而非提示片段。保留父节点的提示模板，移植节点继承捐赠者模板（若无则用默认提示）。通过精确角色匹配或边界度启发式选择交叉模块。
- **突变算子**：
  - 拓扑突变：边/节点的添加/删除（拒绝环）。
  - 角色突变：从领域角色池中选择新角色，并用 PromptMutator 重新生成提示。
  - 激进突变：在搜索停滞时引入更大结构变化（如链、树、星形拓扑），概率由自适应控制决定。
- **环境选择与多样性保持**：采用带约束支配的 NSGA-II，加入动态可行性阈值（τ = best_acc(t) - δ）。通过分箱小生境（bin-based niching）、结构配额、外部精英存档保持帕累托集多样性。
- **自适应控制**：利用跨代帕累托诊断（超体积、间距、前沿间隙、区域覆盖）综合成响应指数 s，映射为激进突变概率：`p_stag = σ(αs + β)`，动态调节探索强度。

## 6. 主要结论与发现

- TCP-MCP 在三个基准上均优于最强自动图生成基线 G-Designer（MMLU: +2.06, MMLU-Pro: +3.34, GSM8K: +0.80 准确率百分点）。
- 与高成本辩论式系统（LLM-Debate）相比，TCP-MCP 在相近准确率下 token 消耗最多减少 5.69 倍。
- **消融实验**：去除任何主要算子（交叉、突变）或采用顺序优化均导致性能下降，证明联合协同进化的必要性。
- **结构分析**：TCP-MCP 针对不同任务演化出不同的协作原型（GSM8K 紧凑，MMLU 强调验证，MMLU-Pro 需要定义锚定和分解）。
- **可迁移性**：搜索到的基因组在相关数据集和不同骨干上仍保持竞争力。

## 7. 优点

- **问题识别深刻**：明确揭示提示与拓扑的耦合性，突破孤立设计范式。
- **方法创新性**：统一基因组表示 + 初始化景观探测 + 帕累托自适应控制，是首个将两维度联合进化的框架。
- **实验设计严谨**：统一骨干模型、严格分离开发/测试集、报告帕累托前沿而非单点、消融实验覆盖主要组件。
- **实用价值**：提供成本感知的多目标设计空间，搜索结果可作为模板复用，降低重复搜索成本。
- **分析全面**：除主结果外，包含迁移分析、第二次评估、超体积曲线，增强结论稳健性。

## 8. 不足与局限

- **模型与任务覆盖有限**：仅使用 DeepSeek-V3.2 一个骨干，未探索其他 LLM 家族（如 LLaMA）或更广泛的任务类型（如生成式、多轮对话）。

- **静态拓扑**：实验仅使用进化的静态拓扑进行评估，未涉及动态通信或在线适应。
- **统计不确定性**：未进行多轮进化独立重复实验，消融差异可能部分源于随机性。作者明确承认这一局限。
- **操作点选取偏差**：虽然规则固定，但开发集上的帕累托选择可能因开发集规模小而波动。
- **应用限制**：未探索工具使用、外部记忆、长期任务等更复杂场景。
