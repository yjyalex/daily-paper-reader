## 摘要
上下文或提示级别的重加权已成为强化学习验证奖励（RLVR）中提升大语言模型推理能力的关键算法杠杆，然而关于最优权重的确定原则仍不清楚。我们通过将提示重加权形式化为通过率函数空间中的效用泛函的泛函导数，弥补了这一空白，构建了一个统一的最优性框架，涵盖了现有方案，包括REINFORCE和GRPO。基于此最优性框架，我们提出了一种基于分位数坐标变换的分布感知提示重加权方法，称为CurveRL，其中每个提示的权重不取决于通过率的绝对值，而是取决于其排名和密度，以反映学习动态中通过率的分布结构。在多个基准上的大量实验表明，我们提出的CurveRL始终优于GRPO和其他RLVR基线。我们的研究将上下文分布控制确定为分析和设计提示重加权RLVR算法的原则性轴。代码已在https://github.com/zhyzmath/CurveRL发布。

## Abstract
Context or prompt-level reweighting has emerged as a central algorithmic lever in Reinforcement Learning with Verified Rewards (RLVR) for improving the reasoning capability of large language models, yet the principle determining what constitutes an optimal weighting remains poorly understood. We address this gap by formulating prompt reweighting as a functional derivative of a utility functional defined in the pass-rate function space, yielding a unified optimality framework that accommodates existing schemes, including REINFORCE and GRPO. Building on this optimality framework, we propose a distribution-aware prompt reweighting approach, called CurveRL, based on a quantile coordinate transform, in which the weight assigned to each prompt depends not on the absolute value of pass rates but on its rank and density to reflect the distributional structure of the pass rates in the learning dynamics. Extensive experiments across multiple benchmarks demonstrate that our proposed CurveRL consistently outperforms GRPO and other RLVR baselines. Our study identifies context-distribution control as a principled axis for analyzing and designing prompt-reweighted RLVR algorithms. The code is released in https://github.com/zhyzmath/CurveRL.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：在基于验证奖励的强化学习（RLVR）中，提示重加权（prompt reweighting）已被广泛用于提升大语言模型的推理能力，但缺乏一个统一且原则性的理论框架来解释“什么样的权重是最优的”。现有方法（如GRPO、MaxRL）的权重设计多基于启发式，且彼此之间缺乏联系。
- **背景**：RLVR 的上下文赌博机结构允许直接操控提示分布（context distribution control），这是标准强化学习不具备的新信息获取轴。然而，传统统计最优性（如MLE）无法直接迁移，因为策略优化中数据分布随策略共同演变，而非固定。因此，需要从效用函数的角度重新定义最优权重。

## 2. 论文提出的方法论

### 2.1 核心思想

- 将提示重加权形式化为**通过率函数空间上的效用泛函对通过率函数的泛函导数**，得到最优权重 \(w_\theta^*(x)\) 的定义。
- 指出点状效用泛函（仅依赖局部通过率绝对值）存在“权重崩塌”问题：在训练早期或晚期，当通过率接近0或1时，权重无法区分具有不同学习潜力的提示。
- 提出**分布感知的效用泛函**：利用通过率的**分位数坐标变换**（Quantile Coordinate Transform），将权重与通过率的**排名和密度**（而非绝对值）挂钩，从而引入分布结构信息。

### 2.2 关键技术细节

- **定义**：效用泛函 \(U_\theta = \mathbb{E}_{x\sim d_0}[\psi(F_{\text{ref}}(p_\theta(x)))]\)，其中 \(F_{\text{ref}}\) 是参考分布（由旧策略滑动窗口估计）的CDF，\(\psi\) 是失真函数（本文取 \(\psi(u)=\log u\)）。
- **梯度形式**：\(\nabla_\theta U_\theta = \mathbb{E}_{x\sim d_0} \left[ \frac{f_{\text{ref}}(p_\theta(x))}{F_{\text{ref}}(p_\theta(x))} \nabla_\theta p_\theta(x) \right]\)，权重 \(w(p) = f_{\text{ref}}(p)/F_{\text{ref}}(p)\)，即逆风险率。
- **算法流程（CurveRL）**：
  1. 维护一个长度为 \(t_0 \times |B|\) 的滑动窗口，存储近期通过率历史。
  2. 每步使用直方图估计参考CDF \(F_{\text{ref}}\) 和密度 \(f_{\text{ref}}\)。
  3. 对每个提示采样N个响应，估计通过率 \(\hat{p}\)，计算权重 \(w_t(\hat{p})\)，并利用 \(\hat{p}\) 作组级基线构造梯度。
  4. 仅保留 \(\hat{p} \in (0,1)\) 的提示加入滑动窗口。
- **理论基础**：点状权重可视为特定先验分布 \(F_{\text{ref}}\) 下的特例（定理1）；分布感知权重具有单调校准不变性（命题3）。

## 6. 论文的主要结论与发现

1. **CurveRL一致优于基线**：在两个模型大小和所有8个基准上，CurveRL在pass@64上超过GRPO和MaxRL，且大多数情况下pass@1也保持优势或相当，改善了pass@1与pass@k的帕累托前沿。
2. **分布感知重加权的优势**：通过使用通过率的排名和密度（而非绝对值），CurveRL有效缓解了权重崩塌问题，尤其在早期（p≈0）和晚期（p≈1）阶段。
3. **扩展推理边界**：CurveRL降低了“不可解”提示的比例，并将更多提示推入中等难度区间（p>0.5），从而提升多数投票和Best-of-N性能。
4. **稳定且数据驱动的训练动态**：CurveRL产生更长的响应、更高的策略熵、更平滑的梯度范数，并通过滑动窗口自适应调整权重分布。

## 7. 优点

- **理论贡献**：首次从泛函导数的角度统一了提示重加权的最优性框架，清晰区分了点状权重与分布感知权重的本质差异。
- **创新方法**：将分位数坐标变换引入RLVR，利用通过率的分布结构（排名和密度），而非简单依赖绝对值，设计新颖。
- **实验严谨**：涵盖两个模型规模、8个基准、多种分析（pass@k缩放、难度分布、权重动态、超参数敏感性、训练动态），对比公平全面。
