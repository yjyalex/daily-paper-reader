## 摘要
多智能体大语言模型系统通过组合多个智能体的输出来提高推理能力，但交互密集型方法可能引入错误传播和高通信开销。当智能体交换原始响应或推理轨迹时，不正确的中间推理可能被采纳并放大，导致自信但错误的共识；多轮通信还会增加令牌消耗、延迟和推理成本。在本文中，我们提出了一种名为DarkForest的受控通信协调框架。DarkForest首先保持智能体独立，每个智能体在不看到其他智能体输出的情况下产生答案。然后将原始响应解析为结构化候选记录，将语义等价的候选记录分组为簇，并使用智能体可靠性、置信度、解析质量、支持模式可靠性和独立性校正来估计这些簇上的校准信念分布。协调器仅接收来自该信念状态的策略允许证据，且通信受控。在六个推理基准上的实验表明，DarkForest实现了领先的整体质量，在基准指标上比最强基线提高了高达30.7%，并且与通信密集型基线相比，令牌消耗最多减少了6.5倍。

## Abstract
Multi-agent LLM systems improve reasoning by combining outputs from multiple agents, but interaction-heavy methods can introduce error propagation and high communication overhead. When agents exchange raw responses or reasoning traces, incorrect intermediate reasoning may be adopted and amplified, leading to confident but wrong consensus; multi-round communication also increases token consumption, latency, and inference cost. In this paper, we propose a controlled-communication coordination framework named DarkForest. DarkForest first keeps agents independent, so each agent produces an answer without seeing the others' outputs. It then parses the raw responses into structured candidate records, groups semantically equivalent candidates into clusters, and estimates a calibrated belief distribution over these clusters using agent reliability, confidence, parse quality, support-pattern reliability, and independence corrections. A coordinator receives only policy-permitted evidence from this belief state with controlled communication. Experiments on six reasoning benchmarks show that DarkForest achieves leading overall quality, improves the strongest baseline by up to 30.7\% on benchmark metrics, and reduces token consumption by up to $6.5\times$ compared with communication-heavy baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有的大语言模型多智能体系统为了提升推理能力，往往采用自由交流、多轮辩论或角色扮演等交互密集型方法。这带来了两大严重缺陷：
    - **错误传播**：一旦某个智能体输出错误或有误导性的中间推理，其他智能体可能采纳、细化并放大该错误，导致系统自信地收敛到错误答案，且智能体间的“一致性”不再代表独立验证，而是模仿或污染的结果。
    - **通信开销巨大**：重复的消息交换大幅增加令牌消耗、延迟和推理成本，使得多智能体系统难以大规模部署。
- **研究动机**：作者通过实验发现（图1），在许多情况下，初始独立查询的智能体池中已经包含了正确答案（候选正确性上限高达75.2%），但现有协调方法（如辩论、自一致性、细化等）最终输出的准确率远低于此上限，说明协调机制丢弃或覆盖了有用的独立证据。因此，核心设计问题不是如何让智能体交流更多，而是如何控制哪些信息跨越智能体边界。
- **整体含义**：受到不完全信息博弈论的启发，作者提出协作不应等同于共享更多文本，而应是紧凑、可控、可验证证据的受控暴露。DarkForest框架正是遵循这一原则，通过结构化信念接口实现受控通信。

## 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：智能体独立生成候选答案；通过解析、聚类和校准信念估计进行聚合；协调器仅接收策略允许的紧凑证据，并辅以确定性护栏进行最终决策。整个设计遵循三个原则：候选生成独立、聚合校准、通信受控。

- **关键技术细节**：
    - **独立候选生成**：每个智能体独立接收输入，生成响应，不观察其他智能体的输出，保留推理时的条件独立性。
    - **解析与规范化**：原始输出经任务特定解析器转换为结构化观测 `oi = (ai, zi, ci, vi, qi)`（解析候选、规范表示、置信度、有效性、解析质量元数据）。语义等价的候选被规范化到同一表示。
    - **候选聚类**：将规范表示相同的有效候选归为同一簇，每个簇 `Cz = (z, Sz, πz)` 包含候选、支持集和支持模式（即哪些智能体支持它）。
    - **校准信念构建**：对每个候选簇 `Cz` 计算证据分数 `s(z) = Rπz * Σ(i∈Sz) αi * ρi * δi * ϕ(ci)`，其中：
        - `αi`：智能体 i 的校准可靠性（从校准数据中估计，含拉普拉斯平滑）。
        - `Rπz`：支持模式的校准可靠性（联合模式可靠性，区分两个互补智能体一致与两个高相关智能体一致的不同证据强度）。
        - `ρi`：解析质量惩罚（若输出畸形但可解析则降低权重）。
        - `δi`：独立性校正（对已知相关的智能体进行折扣，防止重复计数）。
        - `ϕ(ci) = 0.5 + ci`：置信度调制（有界仿射形式，将置信度映射为0.5-1.5的弱调制信号）。
    - 分数归一化为后验信念分布 `P(z|O)`，记录最高候选 `z*`、其后验质量、与第二名的差距等。
    - **受控披露**：披露策略决定向协调器暴露什么信息。默认只暴露候选标识符、规范候选、解析状态、置信度和紧凑信念摘要，不暴露完整原始痕迹。通信成本 `Tcross = tokens(E)` 作为可测量的设计变量。
    - **协调器与护栏**：协调器接收输入和披露证据，输出最终候选。然后应用确定性护栏：当信念状态强烈支持与协调器冲突的候选（满足 `|Sz*| ≥ k`，`P(z*|O) ≥ τp`，`margin ≥ τm`）时，覆盖协调器输出，否则保留协调器输出。护栏不引入额外模型调用。

- **算法流程**（Algorithm 1）：
    1. 独立生成：n个智能体分别生成响应。
    2. 解析与聚类：解析为结构化观测，聚类为候选簇。
    3. 信念构建：计算所有簇的分数和后验信念，确定最高候选及不确定性。
    4. 受控披露与协调：协调器基于披露证据生成输出。
    5. 确定性护栏：若最高候选被信任且与协调器冲突，则替换输出。

## 3. 实验设计

- **数据集/场景**：6个推理基准，覆盖数学（MATH）、代码生成（HumanEval）、通用知识推理（MMLU-Pro）、科学问答（GPQA）、金融推理（FinQA）、法律推理（LegalBench）。每个基准使用默认指标：精确匹配、Pass@1、准确率、执行精度/程序精度等。
- **Benchmark**：全部为标准公开数据集，其中4个通用基准使用Graph-of-Agents (GoA) 的官方子集，专业基准（FinQA、LegalBench）使用文本型子集或精确匹配可评估的样本。
- **对比方法**：6种基线方法——Debate（多轮辩论）、Self-Consistency（自一致性多数投票）、Refine（自我细化）、ReConcile（圆桌会议）、Mixture-of-Agents（MoA，层级聚合）、Graph-of-Agent（GoA，图结构协作，含Mean和Max两种聚合变体）。

## 6. 主要结论与发现

- 在6个基准上，DarkForest实现了最强或接近最强的任务质量，同时显著降低令牌消耗（比通信密集型基线最多降低6.5倍）。
- 在MATH上，DarkForest精确匹配达76.80%，比最强基线Self-Consistency提高5.00个绝对百分点（相对提升7.0%），比GoA-Mean提高5.40点。
- 在MMLU-Pro上，达到58.38%，比最强基线Debate提高2.52点；在FinQA上程序精度达11.33%（最优），执行精度15.67%（第二）。
- 协调器与护栏消融表明：护栏对于保护信念强烈支持的正确候选不被协调器丢弃至关重要（移除护栏后MATH下降1.4点，LegalBench下降2.4点）；协调器在标签异构的任务上更有用。
- 披露政策消融证明：暴露更多文本（推理摘要或完整原始痕迹）并不能提高质量，反而增加成本并可能降低准确率。
- 校准组件（智能体可靠性、支持模式可靠性等）一致提升质量，尤其在LegalBench上效果显著。
- 核心结论：多智能体LLM推理的关键不是让智能体交流更多，而是保留独立证据并控制跨边界的信息流。

## 8. 不足与局限

- **覆盖范围**：仅在7B规模模型上评估，未在更大模型（如70B/134B）或API模型（如GPT-4）上验证，泛化性存疑；专业领域基准（FinQA, LegalBench）样本量较小（300-500），可能不足以充分反映差异。
- **任务局限性**：在代码生成（HumanEval）上效果未达到最优，说明方法可能不适用于需要保留丰富程序细节的场景；在法律基准（LegalBench）上位居第二，与ReConcile差距很小但仍有差距。
- **独立性假设**：框架假设智能体生成独立（不相互观察），但实际中智能体可能因训练数据、架构等存在相关性，虽然进行了独立性校正，但校正效果高度依赖于校准数据的质量和代表性。
- **校准依赖**：需要离线校准步骤（使用已知标签的样本估计可靠性等参数），校准集的选择和大小可能影响最终性能；若任务分布偏移，校准参数可能不适用。
- **护栏设计**：确定性护栏依赖三个阈值（支持数、后验概率、边距），虽然实验显示阈值鲁棒，但默认值可能需要在不同任务上手动调整；护栏仅能替换为最高信念候选，无法进行更精细的合成。
- **未考虑一致性污染**：独立生成虽减少了运行时依赖，但智能体可能因共享的预训练知识而产生系统性偏差，框架未对此建模。
- **可扩展性分析有限**：仅在一个基准（FinQA）上测试了从3到5个智能体的扩展性，缺乏更广泛规模（如10+智能体）的分析。
