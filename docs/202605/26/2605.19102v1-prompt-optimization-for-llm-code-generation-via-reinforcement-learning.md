## 摘要
大型语言模型（LLMs）可以从自然语言生成代码，但其性能对提示表述高度敏感。我们提出了一种基于强化学习的框架，将提示优化建模为序列决策问题。使用近端策略优化（PPO）代理通过混合动作空间迭代改进提示，该混合动作空间结合了直接生成、遗传词汇变异和语义重写，并由源自单元测试反馈的塑造奖励引导。我们在MBPP+、HumanEval+和APPS上使用CodeT5+、CodeLLaMA和DeepSeek-Coder作为冻结代码生成器评估该框架。在500个任务的MBPP+测试集上，PPO代理分别达到了57.58%、64.80%和85.50%的严格Pass@1分数，优于EPiC、Reflexion和Random-Hybrid。Soft-Pass@1分别达到67.90%、73.10%和88.20%。在所有骨干模型上，HumanEval+和APPS也观察到类似的改进。结果表明，带有塑造测试驱动奖励的强化学习提高了基于LLM的代码生成的功能正确性。

## Abstract
Large Language Models (LLMs) can generate code from natural language, but their performance is highly sensitive to prompt formulation. We propose a reinforcement-learning-based framework that models prompt refinement as a sequential decision-making problem. A Proximal Policy Optimization (PPO) agent iteratively improves prompts using a hybrid action space that combines direct generation, genetic lexical mutation and semantic rewriting, guided by shaped rewards derived from unit-test feedback. We evaluate the framework on MBPP+, HumanEval+, and APPS using CodeT5+, CodeLLaMA, and DeepSeek-Coder as frozen code generators. On the 500-task MBPP+ test set, the PPO agent achieves strict Pass@1 scores of 57.58%, 64.80%, and 85.50%, respectively, outperforming EPiC, Reflexion, and Random-Hybrid. Soft-Pass@1 reaches 67.90%, 73.10%, and 88.20%, respectively. Similar improvements are observed on HumanEval+ and APPS across all backbone models. The results demonstrate that reinforcement learning with shaped test-driven rewards improves functional correctness in LLM-based code generation.

## 2. 方法论
### 2.1 核心思想
- 将提示优化视为一个**序列决策问题**，由PPO（近端策略优化）智能体在每一步选择提示变换动作，目标是最优化最终生成代码通过所有测试用例的概率。
- 智能体**状态**：当前提示的语义嵌入（使用MiniLM编码为384维向量）。
- **动作空间**：三个离散动作——直接生成（保留提示）、遗传词汇变异（基于EPiC思想）、语义重写（基于Reflexion思想）。
- **奖励函数**：基于生成代码在单元测试上的通过比例（pass ratio）设计成型奖励：
  - 全部通过：+1.0
  - 部分通过：ρ（0<ρ<1）
  - 全部失败：-1
  - 执行错误（如语法错误）：-2
- 每个编程任务为一个episode，直到完全正确或达到最大步数；使用PPO进行策略更新。

### 2.2 关键技术细节
- **词汇变异**（Lexical Mutation）：对提示进行token级操作（交叉、索引打乱、变异），使用锦标赛选择保留强变体，变异概率0.2。
- **语义重写**（Semantic Rewriting）：利用预训练语言模型（如LLaMA）通过固定元提示对原提示进行结构化改写，并用正则过滤保留关键词。
- **代码生成**：使用冻结的代码生成模型（CodeT5+、CodeLLaMA、DeepSeek-Coder）生成Python函数，并在沙箱中执行测试。
- **训练算法**：PPO（Proximal Policy Optimization），因其在LLM交互环境中稳定且样本高效。

### 2.3 算法流程（文字说明）
1. 从数据集中采样一个任务，包括初始提示和测试用例集。
2. 将初始提示嵌入为状态向量。
3. 循环直到完全正确或达到最大步数：
   - 智能体根据当前状态采样一个动作（直接生成/词汇变异/语义重写）。
   - 应用动作得到新提示。
   - 冻结代码生成器根据新提示生成代码。
   - 在沙箱中执行代码，计算测试通过比例。
   - 根据通过比例计算奖励（成型奖励）。
   - 将转换（状态、动作、奖励、新状态）存储。
   - 更新状态为嵌入后的新提示。
4. 在episode结束时，用PPO对智能体策略进行更新。
