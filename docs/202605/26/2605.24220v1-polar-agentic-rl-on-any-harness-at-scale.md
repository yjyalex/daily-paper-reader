## 摘要
语言智能体的强化学习日益依赖于管理长上下文、多轮工具使用和多智能体编排的自定义框架。然而，将这些框架移植到RL环境接口中仍然困难，并且常常丢失重要的训练信号。我们通过Polar弥合了这一差距，Polar是一个用于任意智能体框架上可扩展异步强化学习的rollout框架。Polar将智能体框架视为黑盒：它代理LLM API调用，记录令牌级别的模型交互，并重构令牌保真的轨迹用于训练。每个rollout节点高效地并行管理运行时预加载、智能体执行、轨迹重构和评估，暴露异步服务端点，可被独立训练器大规模消费。这种解耦设计使Polar对智能体框架、训练基础设施和RL算法均无感知，同时提高了长时间运行的智能体工作负载的计算利用率。我们通过流行的编码框架在软件工程任务上训练智能体来验证Polar。使用简单的GRPO，Polar在SWE-Bench Verified上使用Codex、Claude Code、Qwen Code和Pi框架分别将Qwen3.5-4B提升了22.6、4.8、0.6和6.2个百分点。我们进一步展示了Polar在自定义框架上进行离线数据生成以及消融轨迹重构策略。Polar重写了其先前的工作Prorl Agent，并已被注册为NeMo Gym环境之一。

## Abstract
Reinforcement learning for language agents increasingly depends on custom harnesses that manage long-running context, multi-turn tool use and multi-agent orchestration. However, porting these harnesses into RL environment interfaces remains difficult and often loses important training signals. We bridge this gap with polar, a rollout framework for scalable asynchronous RL over arbitrary agent harnesses. Polar treats the agent harness as a black box: it proxies LLM API calls, records token-level model interactions, and reconstructs token-faithful trajectories for training. Each rollout node efficiently manages runtime prewarming, agent execution, trajectory reconstruction, and evaluation in parallel, exposing asynchronous service endpoints that can be consumed by independent trainers at scale. This decoupled design makes Polar agnostic to agent harnesses, training infrastructure, and RL algorithms while improving compute utilization for long-running agent workloads. We validate polar by training agents on software-engineering tasks with popular coding harnesses. Using simple GRPO, polar improves Qwen3.5-4B by 22.6, 4.8, 0.6 and 6.2 points on SWE-Bench Verified with the Codex, Claude Code, Qwen Code and Pi harnesses, respectively. We further demonstrate Polar for offline data generation over custom harnesses and ablate trajectory reconstruction strategies. Polar rewrites its preceding work, Prorl Agent, and has been registered as one of NeMo Gym environments.

- **研究动机**：现有RL系统（如SkyRL-Agent、PRIME-RL）往往要求智能体适配RL基础设施，导致每次引入新框架都需要重新集成。而主流智能体框架（如Claude Code、Codex CLI）可能是闭源或非Python实现，无法直接嵌入训练管道。Polar旨在弥合这一鸿沟，提供一种对框架无关的、可扩展的异步rollout方案。
- **整体含义**：Polar提出一种新范式——**以智能体的LLM API调用作为RL的roolout接口**，通过代理（proxy）监听模型流量并重建令牌保真轨迹，从而将任何现有智能体框架原封不动地用于RL训练。这解决了agentic RL领域长期存在的系统整合难题。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：保持智能体框架完全不变，将LLM API边界作为观察和训练信号的获取点。Polar在框架与推理服务器之间插入一个模型API代理，代理记录每一次LLM调用的prompt、采样得到的tokens、log probabilities及响应，然后在外部重建出训练器可用的轨迹（trajectory）。
- **关键技术细节**：
  - **代理捕获**：代理兼容Anthropic、OpenAI Chat/Responses、Google等多种API格式，将请求归一化为OpenAI Chat格式后转发至本地推理服务，并记录完整的token-level数据（包括prompt token IDs、sampled response token IDs、finish reason、logprobs）。
  - **异步rollout分阶段执行**：Polar将rollout生命周期拆分为隔离的worker池：INIT（运行时初始化）、READY（缓冲就绪的运行时）、RUNNING（执行智能体）、POSTRUN（轨迹重建、评估、回调）。异步阶段设计使得CPU密集的运行时准备和长尾评估不阻塞GPU绑定的智能体执行。
  - **令牌保真的轨迹重建**：
    - **Per-request方法**（保守基线）：每个LLM completion生成一条独立trace，保留全部细节但碎片化。
    - **Prefix merging方法**：自动检测会话中保持只追加（append-only）的对话链，通过严格token前缀匹配合并相邻completions，仅保留行为策略的采样tokens作为可训练部分，中间插入的规范渲染tokens（如系统提示、工具输出）被标记为不可训练（loss mask置0）。保证了“每个可训练token都来自行为策略在rollout时的实际采样”。
  - **评估与奖励传播**：评估器（evaluator）注册在策略库中，可接收轨迹、会话产物及可刷新的运行时上下文。内置支持SWE-Bench harness评估、结果奖励广播等。
- **无需修改框架**：用户仅需提供一个小型“harness adapter”配置（如设置环境变量、shell命令），框架的启动和运行完全保持不变。

## 6. 主要结论与发现

- **Polar能够在不修改任何智能体框架的情况下，成功进行RL训练**：在四种编码框架上均观测到奖励提升，SWE-Bench Verified score全面增长。
- **大的提升出现在模型不熟悉的框架上**：Qwen3.5-4B在Codex框架上从3.8%提升至26.4%（+22.6点），而在其原生Qwen Code框架上从34.6%提升至35.2%（+0.6点）。这表明Polar可以适应不熟悉的工具模式和上下文策略。
- **Prefix merging轨迹重建策略显著提升训练效率**：相比per-request，将训练器更新次数减少83%，训练时间加速5.4倍，同时保持GPU利用率在87%以上。
- **Polar可兼用为离线数据生成工具**：使用122B模型在1,638个实例上获得了30.8%的接受率，生成了高质量SFT轨迹。
- **系统的核心贡献在于将rollout与训练解耦**，使得agent harness、训练基础设施和RL算法相互独立。

## 7. 优点

1. **真正的框架无关性**：通过LLM API代理而非修改框架内部，支持任何API兼容的智能体（包括闭源二进制或不同语言实现），极大降低了集成成本。
2. **令牌保真性**：prefix merging算法确保训练信号直接附着于行为策略实际采样的tokens，避免了重分词漂移问题，这是训练正确性的关键。
3. **异步分阶段架构**：runtime初始化、执行、后处理解耦，通过worker池和ready buffer避免CPU密集型操作阻塞GPU推理，适合长尾、长时rollout场景。
4. **可扩展性**：以rollout-as-a-service形式暴露异步API，能够与现有训练框架（如Slime）无缝集成，并支持分布式部署。
5. **实践验证充分**：在真实软件工程任务上使用多个流行框架进行了端到端训练，并公开了离线生成的数据集和代码。
6. **设计简洁且扩展性强**：trajectory builder、evaluator、runtime等均采用注册机制，可方便添加自定义策略。

## 8. 不足与局限

1. **实验范围有限**：仅针对SWE-Bench/软件工程任务，未评估在网页代理、桌面代理、多智能体协作等更广泛agentic RL场景中的适用性。
2. **基座模型单一**：所有实验仅使用Qwen3.5-4B（及离线数据生成用了122B），未在Llama、Mistral等其他主流模型上验证。
3. **缺乏与同类系统的直接比较**：虽然附录给出了定性对比表，但未提供与Agent Lightning、SkyRL-Agent等在相同benchmark上的定量性能对比，难以评估Polar在效率或训练效果上的相对优势。
4. **奖励设计简单**：在线RL仅使用二进制结果奖励（pass@1），未探索过程奖励、逐token奖励或session归一化，可能限制复杂任务下的训练效果，且per-request方法存在reward hacking风险（论文已指出）。
5. **离线数据生成实验仅使用二进制过滤**：只保留全部测试通过的轨迹，未区分半成功轨迹或利用失败案例进行偏好学习，也未分析数据质量对下游SFT的影响。
6. **对推理服务器依赖**：代理需要将请求转发到本地推理服务并支持logprobs返回，若推理后端不支持或性能不足，可能成为瓶颈。论文未讨论大规模部署下代理的负载和延迟特性。
7. **文档与代码可复现性**：虽有开源代码链接，但论文中未详细说明训练超参数搜索过程、硬件拓扑等，部分复现细节需参考相关仓库（如Slime示例）。
