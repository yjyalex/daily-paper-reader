## 摘要
基于结果的强化学习为语言智能体提供了稳定的优化基础，但其稀疏的轨迹级奖励对于哪些中间决策应该被强化或抑制提供的指导很少。在策略自蒸馏提供了密集的令牌级监督，然而现有的技能条件变体通常依赖于外部技能记忆或检索到的特权上下文，这些维护成本高昂，并且可能在与当前策略在多轮交互中产生的状态分布不匹配。我们提出OPID（On-Policy Skill Distillation），这是一个直接从已完成在策略轨迹中提取技能监督的框架。OPID将轨迹事后结果表示为分层技能：情节级技能捕获全局工作流或失败避免规则，而步骤级技能捕获关键时间步的局部决策知识。一种关键优先路由机制在识别出关键决策时使用步骤级技能，否则退回到情节级技能作为默认指导。选定的技能被注入到交互历史中，使得旧策略能够在原始上下文和技能增强上下文下对相同的采样响应重新评分。由此产生的对数概率偏移产生令牌级的自蒸馏优势，该优势与结果优势相结合用于策略优化。因此，OPID保留了强化学习作为主要训练目标，同时引入了密集的、分布匹配的事后监督。在ALFWorld、WebShop和基于搜索的QA上的实验表明，OPID在智能体性能、样本效率和鲁棒性上普遍优于仅结果强化学习和现有的技能蒸馏基线。我们的代码可在https://github.com/jinyangwu/OPID/tree/main获取。

## Abstract
Outcome-based reinforcement learning provides a stable optimization backbone for language agents, but its sparse trajectory-level rewards provide little guidance on which intermediate decisions should be reinforced or suppressed. On-policy self-distillation offers dense token-level supervision, yet existing skill-conditioned variants often rely on external skill memories or retrieved privileged context, which are costly to maintain and can be mismatched with the state distribution induced by the current policy in multi-turn interaction. We propose \textbf{OPID} (\textbf{O}n-\textbf{P}olicy Sk\textbf{i}ll \textbf{D}istillation), a framework that extracts skill supervision directly from completed on-policy trajectories. OPID represents trajectory hindsight as hierarchical skills: episode-level skills capture global workflows or failure-avoidance rules, while step-level skills capture local decision knowledge at critical timesteps. A critical-first routing mechanism uses step-level skills when critical decisions are identified and falls back to episode-level skills as default guidance otherwise. The selected skill is injected into the interaction history, allowing the old policy to re-score the same sampled response under both original and skill-augmented contexts. The resulting log-probability shift yields a token-level self-distillation advantage, which is combined with the outcome advantage for policy optimization. OPID thus preserves RL as the primary training objective while introducing dense, distribution-matched hindsight supervision. Experiments on ALFWorld, WebShop and Search-based QA demonstrate that OPID generally improves agent performance, sample efficiency, and robustness over outcome-only RL and existing skill-distillation baselines. Our code is available at https://github.com/jinyangwu/OPID/tree/main.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题根源**：基于结果的强化学习（如GRPO）为语言智能体提供稳定的优化骨干，但其奖励信号稀疏且延迟，仅在轨迹结束时给出成功/失败，无法为中间决策提供细化信用分配（credit assignment）。这使得智能体难以区分哪些中间行为导致了最终成败，尤其在长程交互任务中，早期错误可能被晚发现。
- **现有方法局限**：已有技能条件化的自蒸馏方法（如SDAR、Skill-SD）通过提供特权上下文（如外部技能库、检索到的规则）来生成令牌级监督，但存在两大痛点：
  1. **维护成本高**：需要维护技能记忆库和检索机制。
  2. **状态分布不匹配**：检索的技能可能与当前策略（在线策略）产生的实际状态分布不一致，导致错误指导。
- **OPID的出发点**：利用智能体自身在线策略产生的已完成轨迹作为“事后”知识来源，从中提取层级技能，在不依赖外部资源的前提下提供密集、分布匹配的监督信号。强化学习仍为主目标，蒸馏信号作为补充。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：将已完成在线策略轨迹转化为结构化、分层的自然语言技能，通过关键优先路由选择最合适的技能，注入交互历史，让旧策略对同一响应在有无技能增强的上下文下重新评分，利用对数概率差构建令牌级优势，与结果优势联合优化。
- **关键技术细节（三阶段流程）**：
  1. **在线策略技能提取（Stage 1）**：
     - 使用LLM分析器（Analyzer）解析每个在线策略轨迹（包含任务提示、步骤、观测、动作、结果），自动生成两类技能：
       - **情节级技能（Episode-level skill）**：概括全局工作流（成功轨迹）或失败避免规则（失败轨迹），作为各步骤的默认指导。
       - **步骤级技能（Step-level skill）**：仅针对关键决策步骤（如避免重复无效动作、选择下一步目标、修正错误子目标等）给出局部决策知识，稀疏且状态特异。
  2. **关键优先技能路由（Stage 2）**：
     - 对于非关键步骤，使用情节级技能；对于分析器识别出的关键步骤，使用步骤级技能。两者不叠加，避免冗余干扰。
     - 路由后的技能通过技能注入函数（skill-injection function）附加到交互历史中，形成增强上下文 `tilde{h}_{τ,t}`。
  3. **技能条件自蒸馏（Stage 3）**：
     - 对于每个轨迹步骤的每个响应令牌，使用旧策略（behavior policy）分别计算在原始上下文 `h` 和技能增强上下文 `tilde{h}` 下的对数概率。
     - 定义令牌级技能优势：`A_skill_{τ,t,ℓ} = log π_θold(y|tilde{h}) - log π_θold(y|h)`，正值表示技能使令牌更可能，负值反之。
     - 最终OPID优势结合结果优势（GRPO组相对优势）和技能优势：`A_OPID = A_ep + λ_skill * A_skill`。
     - 采用PPO剪枝的胜率损失进行优化，同时添加KL正则项。
- **训练-推理边界**：分析器、路由和技能评分仅在训练阶段使用；推理时智能体仅依赖原始交互历史，不增加任何开销。

## 6. 主要结论与发现
- **OPID持续增强结果RL**：在几乎所有模型-领域组合中，OPID相比于GRPO获得显著提升（例如Qwen2.5-3B在ALFWorld上 +9.3%，WebShop +10.9%，Search-based QA +8.6%），表明稀疏结果奖励可被密集技能监督有效补充。
