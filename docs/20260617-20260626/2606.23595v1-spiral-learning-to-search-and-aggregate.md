## 摘要
语言模型推理可以通过在测试时使用不同的原语（单条轨迹内的顺序推理、独立采样的并行轨迹以及将多条推理轨迹聚合为最终响应）进行推理计算的扩展而显著提升。然而，在后训练阶段，语言模型仅针对单条轨迹内的顺序推理进行优化。我们提出了顺序-并行-聚合强化学习（SPIRAL）框架，在该框架中，语言模型被训练使用所有三种原语作为统一推理计算流水线的一部分。具体而言，语言模型首先并行采样一组独立轨迹，每条轨迹通过顺序思维链推理生成，然后基于这些轨迹生成最终的聚合轨迹；所有组件均根据最终聚合响应的奖励进行端到端优化。为了训练该系统，SPIRAL使用集合强化学习来教导模型生成一组对聚合器集体有用的轨迹，并使用标准强化学习来教导模型将该集合聚合为改进的最终响应。我们在推理任务上的实验表明，SPIRAL有效扩展了推理计算，当所有三种计算原语都扩展时，其扩展效率比GRPO高出最多11倍，性能高出15%。

## Abstract
Language model reasoning can be substantially improved at test time via scaffolds that scale inference compute across different primitives -- sequential reasoning within a trace, independently sampled parallel traces, and aggregation of multiple reasoning traces into a final response. During post-training, however, language models are optimized only for sequential reasoning within a single trace. We introduce Sequential-Parallel-Aggregative Reinforcement Learning (SPIRAL), a framework in which a language model is trained to use all three primitives, as part of a unified inference compute pipeline. Concretely, the language model first samples a set of independent traces in parallel, each produced through sequential chain-of-thought reasoning, and then generates a final aggregation trace conditioned on those traces; all components are optimized end-to-end against the reward of the final aggregated response. To train this system, SPIRAL uses set reinforcement learning to teach models to produce a set of traces that are collectively useful for an aggregator and standard reinforcement learning to teach models to aggregate the set into improved final responses. Our experiments on reasoning tasks show that SPIRAL effectively scales with inference compute, outperforming GRPO by up to 11$\times$ scaling efficiency and 15% higher performance when all three compute primitives are scaled.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：语言模型推理可以通过测试时扩展推理计算（inference compute）来显著提升，但现有后训练（post-training）方法仅针对单一轨迹内的顺序推理（sequential reasoning）进行优化，忽视了并行采样（parallel traces）和轨迹聚合（aggregation）这两种同样重要的计算原语。这导致测试时实践者需要手工设计复杂的脚手架（scaffold）来协调多种原语，模型自身从未学会如何主动利用它们。
- **整体含义**：本文提出SPIRAL框架，旨在通过强化学习将三种推理计算原语（顺序、并行、聚合）统一在端到端训练中，让模型自动学习如何更有效地组织和利用测试时计算资源，从而突破现有推理性能瓶颈。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：将推理过程建模为一个两阶段生成流水线：先独立并行采样多个搜索轨迹（search traces），再基于这些轨迹生成一个聚合轨迹（aggregation trace）。仅使用最终聚合响应（即最终答案）的奖励来端到端优化所有轨迹。
- **关键技术细节**：
  - **问题形式化**：优化目标为 $\max_{\theta,\phi} \mathbb{E}_{y_{1:n}\sim\pi_\theta(\cdot|x)} \mathbb{E}_{y^*\sim\pi_\phi(\cdot|x,y_{1:n})}[r(x, y^*)]$。其中搜索轨迹由策略$\pi_\theta$生成，聚合轨迹由策略$\pi_\phi$生成（可共享参数）。
  - **梯度分解**：对目标求导得到两项梯度：
    - **Term 1（集合RL梯度）**：优化搜索轨迹集，使其对聚合器集体有用；使用集合强化学习（Set RL）的边际优势（marginal set advantage）进行信用分配。
    - **Term 2（标准RL梯度）**：优化聚合轨迹，使其能从给定轨迹集中合成出高质量答案；使用每组内部的标准化优势（within-set centered advantage）。
  - **具体算法流程**：
    1. 对每个输入$x$，采样$N_1$个搜索轨迹$y_1,\dots,y_{N_1}\sim\pi_\theta(\cdot|x)$。
    2. 从中均匀无放回地抽取$K$个大小为$n$的子集（sets）。
    3. 对每个子集$G_i$，采样$N_2$个聚合轨迹$y^{G_i}_j\sim\pi_\theta(\cdot|x, G_i)$。
    4. 根据聚合轨迹的奖励计算每个子集的平均奖励$\hat{f}_{\text{spiral}}(x,G_i)$，并计算跨子集基线$\hat{f}_{\text{spiral}}(x)$。
    5. 计算每个搜索轨迹的边际集合优势$A^\sharp_{\text{marg}}(x,y_i)$，以及每个聚合轨迹的组内优势$A(x,y^{G_i}_j)$。
    6. 将两种优势分别代入REINFORCE（使用重要性采样和Tinker库）进行策略更新。
  - **理论保证**：证明了当从有序元组中均匀采样时，集合RL梯度估计是无偏的（Proposition 3.1）。

## 6. 主要结论与发现
- **SPIRAL在Pass@k上比GRPO强**：性能提升最高达15%，缩放效率提升最高达11倍。

- **Token效率高**：在固定token预算下，SPIRAL结合递归自聚合的性能缩放最平滑有效。
