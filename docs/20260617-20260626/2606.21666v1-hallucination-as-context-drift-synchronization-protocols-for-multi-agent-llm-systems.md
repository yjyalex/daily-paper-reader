## 摘要
多智能体LLM系统经常产生无法仅用模型缺陷解释的幻觉输出。这类失败的一个重要原因并非模型能力不足，而是上下文漂移：并发智能体之间内部知识状态的分歧。当智能体带着对共享世界状态不匹配或过时的表征进入协作任务时，它们的联合推理会产生表现为幻觉的矛盾。我们定义了上下文分歧分数（CDS），这是一个轻量级标量度量，用于量化智能体对之间在空间、时间和任务维度上的知识状态差异，并提出了共享状态验证协议（SSVP），该协议允许智能体在联合推理前定期交换压缩的状态摘要并标记高分歧条件。我们使用Claude Haiku在两个领域（多智能体旅行和软件项目规划）评估了SSVP。在8个场景的受控实验中（旅行条件每组n=30，软件条件每组n=10），与无同步基线相比，朴素的全广播同步使幻觉率增加34%（HR：0.658 vs 0.492，p=0.0022，d=1.18），这是由于传播错误智能体状态带来的污染效应。SSVP避免了这种失败模式，同时显示出适度且一致的降低（HR：0.463，d=0.30），并且使用比全广播少58%的API调用实现了显著更低的幻觉率（p=0.0005，d=1.47）。污染效应在软件领域没有重复出现，所有条件都收敛到低HR（<0.2），证实了它特定于那些一个错误共享信念会在评估维度上级联传播的任务。我们的结果将幻觉缓解重新定义为分布式系统问题，并将上下文同步确立为多智能体LLM设计中的一级原语。

## Abstract
Multi-agent LLM systems routinely produce hallucinated outputs that cannot be explained by model deficiencies alone. A significant class of these failures arises not from model incapacity but from context drift: the divergence of internal knowledge states between concurrent agents. When agents enter a collaborative task with mismatched or stale representations of shared world state, their joint reasoning produces contradictions that manifest as hallucination. We define the Context Divergence Score (CDS), a lightweight scalar metric quantifying knowledge-state discrepancy between agent pairs across spatial, temporal, and task dimensions, and propose the Shared State Verification Protocol (SSVP), which lets agents periodically exchange compressed state summaries and flag high-divergence conditions before joint reasoning. We evaluate SSVP across two domains (multi-agent travel and software project planning) using Claude Haiku. In controlled experiments (n=30 per condition, travel; n=10, software) across 8 scenarios, naive full-broadcast synchronization increases hallucination rate by 34% above the no-sync baseline (HR: 0.658 vs. 0.492, p=0.0022, d=1.18), a contamination effect from propagating erroneous agent states. SSVP avoids this failure mode while showing modest, consistent reduction (HR: 0.463, d=0.30) and achieves significantly lower hallucination than full-broadcast (p=0.0005, d=1.47) using 58% fewer API calls. The contamination effect does not replicate in the software domain, where all conditions converge to low HR (<0.2), confirming it is specific to tasks where one erroneous shared belief cascades across evaluation dimensions. Our results reframe hallucination mitigation as a distributed systems problem and establish context synchronization as a first-class primitive in multi-agent LLM design.

- **背景**：现有幻觉缓解方法多集中于改进模型本身（如训练数据、RLHF、思维链等），但在多Agent部署中效果有限。本文提出**将幻觉缓解视为分布式系统问题**，认为核心在于Agent间的状态同步，而非单纯提升单Agent模型能力。

### 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：通过量化Agent间知识状态的分歧程度，并在分歧过大时触发选择性同步，避免错误信息扩散，从而减少因上下文不一致导致的幻觉。
- **关键技术细节**：
  - **Context Divergence Score (CDS)**：定义两个Agent i和j在时间t的上下文相似度度量（余弦距离），CDS = 1 - (c_i·c_j)/(||c_i||·||c_j||)，取值范围[0,2]，0表示完全相同，2表示最大分歧。CDS具有模型无关、O(1)计算、阈值可调等特点。
  - **Shared State Verification Protocol (SSVP)**：轻量级协调协议，Agent定期（或检测到关键事件时）生成压缩的上下文摘要，广播后计算系统级CDS；若超过阈值τ（实验设为0.25），则暂停联合推理，让高分歧对交换完整上下文，通过ContextMerge提示Agent在自身状态与传入摘要间进行裁决（基于时间戳和信息质量），然后重新嵌入合并后的上下文，继续联合推理。协议开销为O(nλ)。
- **算法流程（文字说明）**：初始化每个Agent的上下文向量→每个Agent独立执行推理步骤→每隔Δt或检测到任务关键事件时，每个Agent生成上下文摘要并广播→计算所有Agent对的CDS→若系统级CDS > τ，则暂停联合推理，高分歧对交换完整上下文，通过ContextMerge裁决矛盾→更新合并后的上下文→恢复联合推理。

### 7. 优点：方法或实验设计上的亮点
- **问题视角新颖**：将多Agent幻觉重新定义为分布式系统同步问题，而非纯语言模型缺陷，开辟了新的缓解方向。
- **轻量级实用度量CDS**：O(1)计算、模型无关、可在线使用，基于嵌入向量实现语义距离衡量，易于部署。
- **SSVP设计巧妙**：阈值触发的选择性同步避免了全广播的污染风险，同时保持低开销；ContextMerge提示让Agent主动裁决矛盾，不盲目覆盖，保留显式推理步骤。
- **严格受控的实验设计**：注入明确的不匹配、随机化场景、独立LLM评判、统计显著性检验，结果可解释性强。

### 8. 不足与局限
- **嵌入保真度**：384维向量可能无法完全捕捉复杂Agent状态，细微但关键的分歧可能被遗漏。
- **ContextMerge裁决错误**：当前仅验证了注入的不匹配明确（权威性明显）的情况；当冲突双方看似同样可信时（例如均来自不同但有效的API数据），裁决可能出错。论文建议未来集成源归因和置信度评分。
- **规模限制**：实验仅涉及3个Agent，MAST发现失败率随Agent数量非线性增加，SSVP在更大规模下可能需要层级化传播。
- **任务覆盖有限**：仅测试结构化规划任务（旅行/软件），开放域或动态任务可能场景不同，CDS动态变化可能不同。
- **统计显著性不足**：SSVP vs No-Sync的HR降低未达到统计显著（p=0.257），仅方向性一致，需更大样本验证。
- **算力信息缺失**：未报告推理所需GPU/训练资源，其他研究者难以验证计算成本。
- **污染效应非普遍**：仅在特定类型任务中发现，限制了方法的普适性声明。论文对此诚实承认，但需更多领域验证。
